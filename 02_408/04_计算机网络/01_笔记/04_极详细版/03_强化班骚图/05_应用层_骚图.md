## 🌍 主题 1：应用层概述

### 1.1 应用层 4 大功能

```
1. 文件传输：FTP/TFTP
2. 邮件：SMTP/POP3/IMAP
3. Web：HTTP/HTTPS
4. 域名解析：DNS
```

### 1.2 网络应用模型

| 模型 | 架构 | 例子 |
|------|------|------|
| **C/S** | 客户端/服务器 | Web/邮件 |
| **P2P** | 对等 | BT/电驴 |
| **混合** | 两者结合 | 现代应用 |

### 1.3 C/S vs P2P

| 特性 | C/S | P2P |
|------|-----|-----|
| 服务器压力 | 大 | 小 |
| 客户端要求 | 低 | 高 |
| 扩展性 | 差 | 好 |
| 可靠性 | 服务器故障影响大 | 节点故障影响小 |

## 🌐 主题 2：DNS 域名系统

### 2.1 DNS 作用

```
域名 → IP 地址
例：www.baidu.com → 110.242.68.66
```

### 2.2 域名结构

```
       .（根）
       │
   ┌───┴───┐
   com     cn
   │       │
baidu  sina  edu
   │
www mail
```

### 2.3 顶级域名分类

| 类别 | 例子 | 含义 |
|------|------|------|
| **国家顶级** | .cn/.us/.jp | 国家 |
| **通用顶级** | .com/.net/.org | 通用 |
| **基础结构** | .arpa | 反向解析 |

### 2.4 DNS 服务器类型

| 类型 | 英文 | 作用 |
|------|------|------|
| 根 | Root | 顶级域指引 |
| 顶级域 | TLD | 顶级域管理 |
| 权威 | Authoritative | 域内解析 |
| 本地 | Local | 缓存 + 转发 |

### 2.5 5 大 DNS 报文

```
1. 查询报文
2. 响应报文
3. 头部：12 字节
   - ID（2 字节）
   - 标志（2 字节）
   - 4 个计数（各 2 字节）
4. 问题部分
5. 回答/权威/附加部分
```

### 2.6 DNS 查询 2 大方式

| 方式 | 流程 | 特点 |
|------|------|------|
| **递归** | 本地 → 根 → 顶级 → 权威 → 顶级 → 根 → 本地 | 客户全包 |
| **迭代** | 本地问根 → 根给顶级 → 本地问顶级 → 顶级给权威 → 本地问权威 | 客户自助 |

```
递归查询：
  客户 → 本地 DNS → 根 → 顶级 → 权威 → 顶级 → 根 → 本地 DNS → 客户
  （代为查询）

迭代查询：
  客户 → 本地 DNS
  本地 DNS → 根（"问顶级"）
  本地 DNS → 顶级（"问权威"）
  本地 DNS → 权威（"给你 IP"）
  本地 DNS → 客户
```

### 2.7 DNS 资源记录

| 类型 | 含义 |
|------|------|
| A | 域名 → IPv4 |
| AAAA | 域名 → IPv6 |
| CNAME | 别名 |
| MX | 邮件服务器 |
| NS | 域名服务器 |
| PTR | IP → 域名 |
| SOA | 起始授权 |

## 🌍 主题 3：HTTP 协议

### 3.1 HTTP 特点

```
1. 无连接：每次请求独立
2. 无状态：服务器不保留客户端状态
3. 媒体独立：可传输任意类型数据
4. 面向事务：一问一答
```

### 3.2 HTTP URL 格式

```
http://host[:port]/path[:params][?query]
例：http://www.baidu.com:80/index.html
```

### 3.3 HTTP 报文结构

```
请求报文：
  请求行：方法 URL 版本
  首部行：多个 Key: Value
  空行
  实体主体（可选）

响应报文：
  状态行：版本 状态码 短语
  首部行
  空行
  实体主体
```

### 3.4 HTTP 8 大方法

| 方法 | 含义 | 幂等 |
|------|------|------|
| **GET** | 获取资源 | ✔ |
| **POST** | 提交数据 | ❌ |
| **PUT** | 上传资源 | ✔ |
| **DELETE** | 删除 | ✔ |
| **HEAD** | 类似 GET，无实体 | ✔ |
| **OPTIONS** | 选项 | ✔ |
| **TRACE** | 回显请求 | ✔ |
| **CONNECT** | 隧道 | ❌ |

### 3.5 HTTP 状态码

| 状态码 | 类别 | 例子 |
|--------|------|------|
| 1xx | 信息 | 100 Continue |
| 2xx | 成功 | 200 OK, 201 Created |
| 3xx | 重定向 | 301 Moved, 304 Not Modified |
| 4xx | 客户端错 | 400 Bad Request, 404 Not Found |
| 5xx | 服务器错 | 500 Internal Error, 503 Unavailable |

### 3.6 HTTP 版本演进

| 版本 | 年份 | 特点 |
|------|------|------|
| HTTP/0.9 | 1991 | 仅 GET |
| HTTP/1.0 | 1996 | 多种方法，无状态 |
| HTTP/1.1 | 1997 | 长连接，管道 |
| HTTP/2.0 | 2015 | 多路复用，二进制 |
| HTTP/3.0 | 2018 | QUIC/UDP |

### 3.7 HTTP 持久连接

```
非持久连接：每次请求建立 TCP 连接
持久连接：一次 TCP 连接可发多次请求
HTTP/1.1 默认持久
```

### 3.8 HTTP 报文头

| 头 | 含义 |
|----|------|
| Host | 主机名 |
| Connection | 连接方式 |
| Cookie | 客户端状态 |
| User-Agent | 浏览器信息 |
| Content-Type | 数据类型 |
| Content-Length | 数据长度 |
| Accept | 可接受类型 |

## 🔒 主题 4：HTTPS

### 4.1 HTTPS = HTTP + SSL/TLS

```
HTTP 默认 80 端口，明文
HTTPS 默认 443 端口，加密
SSL/TLS 在传输层与应用层之间
```

### 4.2 4 大加密技术

```
1. 对称加密：加密解密同密钥（AES/DES）
2. 非对称加密：公钥+私钥（RSA/ECC）
3. 散列函数：单向（SHA/MD5）
4. 数字签名：身份验证
```

### 4.3 HTTPS 工作流程

```
1. 客户端：Hello（支持的加密算法）
2. 服务器：Hello（选定算法）+ 证书
3. 客户端：验证证书
4. 客户端：用公钥加密"预主密钥"发送
5. 服务器：用私钥解密
6. 双方：根据预主密钥生成会话密钥
7. 用对称密钥加密通信
```

### 4.4 SSL/TLS 4 大阶段

```
握手：协商算法、认证身份
密钥导出：生成会话密钥
数据传输：加密通信
关闭：释放连接
```

## 📧 主题 5：电子邮件协议

### 5.1 邮件系统组成

```
用户代理（UA）：用户读写邮件
邮件服务器：发送、接收、存储邮件
邮件协议：SMTP/POP3/IMAP
```

### 5.2 邮件收发过程

```
发件人 UA → SMTP → 发送服务器
发送服务器 → SMTP → 接收服务器
接收服务器 → POP3/IMAP → 收件人 UA
```

### 5.3 SMTP（简单邮件传输协议）

```
TCP 25 端口
推协议：发送方主动推
命令：HELO/MAIL FROM/RCPT TO/DATA/QUIT
只支持 ASCII 文本
```

### 5.4 POP3（邮局协议第 3 版）

```
TCP 110 端口
拉协议：客户端主动拉
2 种模式：
  下载并删除：看完删
  下载并保留：可重看
```

### 5.5 IMAP（互联网邮件访问协议）

```
TCP 143 端口
比 POP3 更强
支持部分下载、文件夹管理
在线操作
```

### 5.6 三大邮件协议对比

| 协议 | 角色 | 端口 | 模式 |
|------|------|------|------|
| SMTP | 发送 | 25 | 推 |
| POP3 | 接收 | 110 | 拉 |
| IMAP | 接收 | 143 | 拉（更强） |

### 5.7 MIME

```
MIME：多用途互联网邮件扩展
使邮件支持：图片/音频/视频/非 ASCII
放在邮件头说明
```

## 📂 主题 6：FTP 文件传输

### 6.1 FTP 特点

```
TCP 协议
20：数据端口
21：控制端口
双连接：控制 + 数据
```

### 6.2 FTP 主动 vs 被动模式

| 模式 | 数据连接发起方 | 服务器端口 |
|------|--------------|-----------|
| **主动 (PORT)** | 服务器主动连客户端 | 20 → 客户端 N |
| **被动 (PASV)** | 客户端主动连服务器 | 服务器 N（>1024） |

```
主动模式：
  客户端 → 服务器 21（控制）
  服务器 20 → 客户端 N（数据，被服务器主动连）

被动模式：
  客户端 → 服务器 21（控制）
  服务器告知端口 P
  客户端 → 服务器 P（数据，客户端主动连）
```

### 6.3 FTP 常用命令

```
USER 用户名
PASS 密码
LIST 列表
RETR 下载
STOR 上传
DELE 删除
QUIT 退出
```

### 6.4 TFTP（简单文件传输协议）

```
UDP 69 端口
只支持文件传输
无认证、无目录
适合简单网络（如 PXE）
```

## 🌐 主题 7：其他应用协议

### 7.1 DHCP（动态主机配置协议）

```
UDP 67/68
4 步：DISCOVER → OFFER → REQUEST → ACK
自动分配 IP
租约机制
```

### 7.2 Telnet（远程登录）

```
TCP 23 端口
明文传输
不安全
基本被 SSH 取代
```

### 7.3 SSH（安全壳）

```
TCP 22 端口
加密传输
替代 Telnet
```

### 7.4 SNMP（简单网络管理协议）

```
UDP 161/162
网络设备管理
MIB：管理信息库
5 大操作：GET/SET/GET-NEXT/GET-RESPONSE/TRAP
```

### 7.5 SMB/CIFS

```
Windows 文件共享
445 端口
Linux 用 Samba
```

## 🔐 主题 8：P2P 应用

### 8.1 P2P 特点

```
无（或少）服务器
每个节点既是客户端又是服务器
扩展性好
BitTorrent（BT）是最流行 P2P 协议
```

### 8.2 BT 工作原理

```
1. 种子文件（.torrent）含文件信息和 Tracker
2. 客户端连 Tracker 获取 peers
3. 客户端同时从多个 peers 下载
4. 完成后可作为 peer 服务他人
```

### 8.3 P2P 文件分发分析

```
N 个节点下载 F 比特文件
C/S 模式：服务器带宽受限
P2P 模式：分发时间 = max(F/us, NF/(us + Σui))
  us = 服务器上传带宽
  ui = 节点 i 上传带宽
```

## ⚠️ 8 大易错点

1. ❌ **DNS 用 TCP？** 错误！DNS 主要用 **UDP 53**，区域传送用 TCP
2. ❌ **HTTP 1.1 短连接？** 错误！默认**长连接**
3. ❌ **HTTPS 默认 80？** 错误！HTTPS 默认 **443**
4. ❌ **SMTP 拉协议？** 错误！SMTP 是**推**协议
5. ❌ **POP3 可管理文件夹？** 错误！POP3 不能，要 **IMAP**
6. ❌ **FTP 单一连接？** 错误！FTP **双连接**（控制+数据）
7. ❌ **P2P 无服务器？** 错误！P2P 也有 Tracker 等
8. ❌ **HTTP 无状态 = 不安全？** 错误！无状态仅指服务器不保留会话
