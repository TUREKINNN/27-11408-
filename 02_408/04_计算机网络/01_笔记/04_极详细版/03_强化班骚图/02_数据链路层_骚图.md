## 📦 主题 1：数据链路层功能

### 1.1 5 大功能

```
1. 封装成帧：网络层数据报加头加尾
2. 透明传输：解决特殊字符问题
3. 差错控制：检错 + 纠错（CRC/海明）
4. 流量控制：协调收发双方速率
5. 可靠传输：保证数据无错不丢不乱
```

### 1.2 帧结构

```
┌──────┬─────────┬──────┬─────────┬──────┐
│ 帧头 │ 数据字段 │ 帧尾 │   FCS   │  帧  │
│ SOI  │ Payload │ EOI  │ 校验码  │  标志 │
└──────┴─────────┴──────┴─────────┴──────┘
```

## 🎯 主题 2：封装成帧

### 2.1 4 大成帧方法

| 方法 | 原理 | 优缺点 |
|------|------|--------|
| **字符计数法** | 头部放帧长度 | 错位即崩溃 |
| **字符填充** | 特殊字符界定 | 简单，效率低 |
| **零比特填充** | 01111110 标志 | 高效，HDLC 用 |
| **违规编码法** | 用无效编码界定 | 高速链路 |

### 2.2 零比特填充（HDLC）

```
标志：01111110
发送方：发现 5 个连续 1，填 1 个 0
接收方：发现 5 个连续 1 后 0，删该 0

例：原数据 011011111110011111100
   填后  011011111011001111101000 （最后是标志）
```

## 🔍 主题 3：差错控制

### 3.1 三大检错码

| 码 | 检错能力 | 编码效率 |
|----|---------|---------|
| **奇偶校验** | 1 位错 | 低 |
| **CRC** | 所有奇数位错 + 突发错 | 高 |
| **海明码** | 2 位错 + 纠 1 位 | 中 |

### 3.2 CRC 计算

```
步骤：
1. 设生成多项式 G(x) 阶为 r
2. 发送数据后补 r 个 0
3. 模 2 除以 G(x)，得余数
4. 余数 = FCS，附加到数据末尾发送
5. 接收方用 G(x) 整除，余 0 正确
```

> - 补 4 个 0：1101 0000
> - 11010000 ÷ 10011：
>   - 11010000
>   - 10011
>   - -----
>   - 10010
>   - 10011
>   - -----
>   - 00001
>   - 00000
>   - -----
>   - 00001
>   - 00000
>   - -----
>   - 00001
>   - 00000
>   - -----
>   - 00001 余 0001（但 CRC 是 4 位）
>   - 实际余数 = 0010（重新计算）
> - 发送：1101 0010

### 3.3 CRC 检错能力

```
- 可检测所有奇数位错
- 可检测所有长度 ≤ r 的突发错
- 不能保证 100% 检错
```

## 🛡️ 主题 4：流量控制 + 可靠传输

### 4.1 4 大可靠传输协议

| 协议 | 发送窗口 | 接收窗口 | 信道利用率 | 复杂度 |
|------|---------|---------|----------|--------|
| **停止-等待** | 1 | 1 | 低 | 最简 |
| **后退 N 帧 (GBN)** | > 1 | 1 | 中 | 中 |
| **选择重传 (SR)** | > 1 | > 1 | 高 | 最复杂 |

### 4.2 停止-等待协议

```
发送方:  发帧 0
        ↓ 等待 ACK 0
        ↓ 超时重发
        ↓ 收到 ACK 0
        发帧 1
        ...

接收方:  收帧 0
        发 ACK 0
        收帧 1
        发 ACK 1
        ...
```

> 帧长 L bit，速率 C bps，单程时延 R s
> 周期 T = L/C + 2R
> 利用率 U = (L/C) / (L/C + 2R)

### 4.3 后退 N 帧（GBN）

```
发送窗口 W = 2^n - 1
接收窗口 W = 1

累积确认：ACK n 表示 n 及之前全部正确收到
超时：重传该帧及之后所有已发送的帧
```

#### GBN 行为

```
发送方连续发 0,1,2,3,4,5
假设 2 丢失：
接收方丢弃 3,4,5
发送方超时重传 2,3,4,5
```

### 4.4 选择重传（SR）

```
发送窗口 W = 2^(n-1)
接收窗口 W = 2^(n-1)

每个帧独立 ACK
仅重传丢失的帧
接收方缓存乱序帧
```

#### SR 行为

```
发送方发 0,1,2,3,4
假设 2 丢失：
接收方缓存 3,4，发 ACK 0,1,3,4
发送方超时只重传 2
收到 2 后，接收方按序交付 0-4
```

### 4.5 三大协议对比

| 协议 | 窗口 | 重传 | 接收方缓存 | 利用率 |
|------|------|------|----------|--------|
| 停止-等待 | 1,1 | 1 帧 | 否 | 低 |
| GBN | >1,1 | 多帧 | 否 | 中 |
| SR | >1,>1 | 1 帧 | 是 | 高 |

## 🔌 主题 5：介质访问控制（MAC）

### 5.1 信道划分 3 大方式

| 方式 | 英文 | 原理 |
|------|------|------|
| **频分复用** | FDM | 频率分割 |
| **时分复用** | TDM | 时间分割 |
| **码分复用** | CDM | 码型分割 |

### 5.2 随机访问 4 大协议

| 协议 | 英文 | 核心思想 |
|------|------|---------|
| **ALOHA** | ALOHA | 自由发，冲突重传 |
| **时隙 ALOHA** | Slotted ALOHA | 按时隙发 |
| **CSMA** | Carrier Sense | 先听后发 |
| **CSMA/CD** | CSMA/Collision Detection | 先听后发边发边听 |
| **CSMA/CA** | CSMA/Collision Avoidance | 先听后发边发边避 |

### 5.3 ALOHA 协议

```
纯 ALOHA：吞吐量 S = G × e^(-2G)，最大 1/(2e) ≈ 18.4%
时隙 ALOHA：吞吐量 S = G × e^(-G)，最大 1/e ≈ 36.8%
G = 总负载 = 帧率 × 帧时间
```

### 5.4 CSMA/CD（以太网）

```
Carrier Sense Multiple Access with Collision Detection
步骤：
1. 监听信道：忙则等待
2. 空闲：发帧
3. 边发边听：检测冲突
4. 冲突：停止发，Jam 信号
5. 二进制退避算法：随机等待
6. 重传

最小帧长 = 2τ × 带宽
  τ = 单程传播时延
```

> - τ = 1000 / 2×10⁸ = 5μs
> - 最小帧长 = 2 × 5μs × 10Mbps = 100 bit
> - 10Mbps 以太网规定最小帧长 64 字节 = 512 bit ✓

### 5.5 CSMA/CA（无线局域网）

```
Carrier Sense Multiple Access with Collision Avoidance
特点：
1. 预约信道：RTS/CTS 握手
2. ACK 确认：收到才确认
3. 退避算法：CW 窗口

为什么不用 CSMA/CD？
- 无线：近远效应，无法检测冲突
- 采用 CA 主动避免
```

### 5.6 二进制指数退避

```
第 i 次冲突后：
  从 [0, 1, ..., 2^i - 1] 选随机数 r
  等待 r × 512 比特时间
  i 增加到 10 后封顶（1023）
  重传 16 次后放弃
```

## 🌐 主题 6：局域网

### 6.1 3 大 LAN 标准

| 标准 | 速率 | 介质 |
|------|------|------|
| **以太网** | 10Mbps ~ 100Gbps | 双绞线/光纤 |
| **令牌环** | 4/16 Mbps | 令牌 |
| **FDDI** | 100 Mbps | 光纤 |

### 6.2 MAC 地址

```
格式：48 bit = 24 位 OUI + 24 位 NIC 序列
表示：xx:xx:xx:xx:xx:xx (16 进制)
唯一性：全球唯一
类型：
  单播：第 8 位 = 0
  多播：第 8 位 = 1
  广播：FF:FF:FF:FF:FF:FF
```

### 6.3 以太网帧结构

```
┌─────────┬──────┬──────────┬──────┬──────┐
│ 前导码  │ DA   │   SA     │ Type │ Data │ FCS │
│ 8 字节  │ 6 字 │ 6 字节   │ 2 字 │ 46-  │ 4 字│
│  7+1    │ 节   │          │ 节   │ 1500 │ 节 │
└─────────┴──────┴──────────┴──────┴──────┴──────┘
最小帧 64 字节（512 bit）
最大帧 1518 字节
```

## 🖧 主题 7：交换机

### 7.1 交换机 3 大功能

```
1. 学习：记录源 MAC 和端口 → CAM 表
2. 转发：查 CAM 表转发
3. 泛洪：CAM 表无目标，广播所有端口
```

### 7.2 交换机特点

| 特性 | 描述 |
|------|------|
| 工作层 | 数据链路层 |
| 端口带宽 | 独享 |
| 冲突域 | 每个端口一个 |
| 广播域 | 所有端口（VLAN 可分） |
| 转发方式 | 直通 / 存储转发 / 碎片隔离 |

### 7.3 3 大转发方式

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **直通** | 收 6 字节即转发 | 速度快 | 错帧也转 |
| **存储转发** | 收全帧检错后再发 | 错帧丢弃 | 延迟大 |
| **碎片隔离** | 收 64 字节后转 | 折中 | 中等 |

## ⚠️ 8 大易错点

1. ❌ **CRC 能纠错？** 错误！CRC 只能**检错**
2. ❌ **GBN 和 SR 接收窗口相同？** 错误！GBN=1，SR>1
3. ❌ **CSMA/CD 用于无线？** 错误！无线用 CSMA/CD 不行，用 CA
4. ❌ **交换机能隔离广播域？** 错误！默认不能，要 VLAN
5. ❌ **MAC 地址全球唯一且不变？** 错误！可软件改
6. ❌ **海明码能纠 2 位？** 错误！海明码只能**检 2 位 + 纠 1 位**
7. ❌ **停止-等待信道利用率高？** 错误！利用率很低
8. ❌ **以太网最小帧 = 64？** 错误！最小 64 字节 = 512 bit
