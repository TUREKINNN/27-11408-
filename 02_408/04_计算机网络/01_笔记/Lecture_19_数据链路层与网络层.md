## 1. 数据链路层

### 1.1 数据链路层的功能

<span style="background-color:#ccc1d9">**数据链路层**</span>：负责相邻节点间的帧传输。

**功能**：

| 功能 | 描述 |
|------|------|
| **帧同步** | 识别帧的开始和结束 |
| **差错检测** | 检测传输错误 |
| **流量控制** | 控制发送速率 |
| **介质访问控制** | 解决冲突 |

### 1.2 帧的结构

```
┌──────────┬──────────┬──────────────┬──────────┬──────────┐
│ 帧头     │ 地址     │ 数据         │ 校验     │ 帧尾     │
│ 标志     │ 字段     │ 字段         │ 字段     │ 标志     │
└──────────┴──────────┴──────────────┴──────────┴──────────┘
```

### 1.3 差错检测

#### 1.3.1 奇偶校验

**原理**：添加一位校验位，使1的个数为奇数或偶数。

**缺点**：只能检测奇数个错误。

#### 1.3.2 循环冗余校验（CRC）

**原理**：多项式除法，余数作为校验码。

**步骤**：

1. 选择生成多项式G(x)
2. 在数据后添加r个0（r = G(x)次数）
3. 用模2除法计算余数
4. 余数作为CRC校验码

**优点**：检测能力强。

### 1.4 流量控制

#### 1.4.1 停止等待协议

```
发送方发送一帧，等待确认后再发送下一帧
```

**优点**：简单。

**缺点**：效率低。

#### 1.4.2 滑动窗口协议

```
发送方可以连续发送多个帧，接收方确认多个帧
```

**优点**：效率高。

**缺点**：复杂。

### 1.5 介质访问控制

#### 1.5.1 CSMA/CD

<span style="background-color:#ccc1d9">**CSMA/CD**</span>：载波监听多点接入/碰撞检测。

**步骤**：

1. 监听信道
2. 空闲时发送
3. 检测碰撞
4. 碰撞后退避

**适用**：以太网。

#### 1.5.2 CSMA/CA

<span style="background-color:#ccc1d9">**CSMA/CA**</span>：载波监听多点接入/碰撞避免。

**步骤**：

1. 监听信道
2. 空闲时等待随机时间
3. 发送前发送RTS
4. 收到CTS后发送数据

**适用**：无线局域网。

## 2. 网络层

### 2.1 网络层的功能

<span style="background-color:#ccc1d9">**网络层**</span>：负责分组的路由和转发。

**功能**：

| 功能 | 描述 |
|------|------|
| **路由选择** | 选择最佳路径 |
| **分组转发** | 转发数据包 |
| **拥塞控制** | 控制网络流量 |
| **地址管理** | IP地址管理 |

### 2.2 IP协议

#### 2.2.1 IPv4地址

**格式**：32位，点分十进制。

**分类**：

| 类别 | 范围 | 用途 |
|------|------|------|
| **A类** | 1-126 | 大型网络 |
| **B类** | 128-191 | 中型网络 |
| **C类** | 192-223 | 小型网络 |
| **D类** | 224-239 | 组播 |
| **E类** | 240-255 | 保留 |

#### 2.2.2 子网划分

**目的**：将一个网络划分为多个子网。

**方法**：借用主机位作为子网位。

**示例**：

```
C类地址：192.168.1.0/24
划分子网：借用2位主机位
子网掩码：255.255.255.192
子网数：4个
每个子网主机数：62个
```

#### 2.2.3 IPv6地址

**格式**：128位，十六进制，冒号分隔。

**特点**：

- 地址空间大
- 支持自动配置
- 内置安全

### 2.3 ICMP协议

<span style="background-color:#ccc1d9">**ICMP**</span>：网际控制报文协议。

**功能**：

- 差错报告
- 网络诊断

**常见报文**：

| 类型 | 描述 |
|------|------|
| **回显请求/响应** | ping命令 |
| **目的地不可达** | 目标地址不可达 |
| **超时** | 生存时间过期 |

### 2.4 ARP协议

<span style="background-color:#ccc1d9">**ARP**</span>：地址解析协议。

**功能**：将IP地址转换为MAC地址。

**工作过程**：

1. 发送ARP请求广播
2. 目标主机回复ARP响应
3. 更新ARP缓存

## 3. 路由算法

### 3.1 路由算法的分类

| 类型 | 描述 |
|------|------|
| **静态路由** | 手动配置路由表 |
| **动态路由** | 自动更新路由表 |

### 3.2 常见路由算法

#### 3.2.1 距离向量算法

```
每个节点向邻居广播距离向量
```

**特点**：

- 简单
- 收敛慢
- 可能产生路由环路

#### 3.2.2 链路状态算法

```
每个节点广播链路状态信息
```

**特点**：

- 收敛快
- 无环路
- 计算复杂

#### 3.2.3 最短路径优先（SPF）

```
使用Dijkstra算法计算最短路径
```

**特点**：

- 最优路径
- 计算量大

### 3.3 路由协议

| 协议 | 类型 | 适用范围 |
|------|------|----------|
| **RIP** | 距离向量 | 小型网络 |
| **OSPF** | 链路状态 | 中大型网络 |
| **BGP** | 路径向量 | 互联网 |

## 4. 拥塞控制

### 4.1 拥塞的定义

<span style="background-color:#ccc1d9">**拥塞**</span>：网络流量超过处理能力，导致延迟增加、丢包率上升。

### 4.2 拥塞控制方法

| 方法 | 描述 |
|------|------|
| **端到端控制** | TCP拥塞控制 |
| **网络层控制** | 流量整形、准入控制 |
| **链路层控制** | 队列管理 |

### 4.3 队列管理

#### 4.3.1 FIFO队列

```
先进先出
```

**优点**：简单。

**缺点**：无法区分优先级。

#### 4.3.2 优先级队列

```
高优先级优先处理
```

**优点**：支持优先级。

**缺点**：低优先级可能饥饿。

#### 4.3.3 加权公平队列（WFQ）

```
按权重分配带宽
```

**优点**：公平，支持优先级。

## 5. 典型例题

### 例题 1：CRC计算

**题目**：数据为101010，生成多项式G(x) = x^3 + x + 1。计算CRC校验码。

**解答**：

```
G(x) = x^3 + x + 1 = 1011

数据：101010
添加3个0：101010000

模2除法：
101010000 ÷ 1011 = 100101，余数为011

CRC校验码：011

发送帧：101010011
```

**总结**：CRC通过多项式除法生成校验码。

**来源**：408历年真题，考查方式：计算题

---

### 例题 2：子网划分

**题目**：将B类地址172.16.0.0划分为30个子网，求子网掩码和每个子网的主机数。

**解答**：

```
B类地址默认掩码：255.255.0.0（/16）

需要30个子网，2^5 = 32 >= 30，所以借用5位主机位

子网掩码：/16 + 5 = /21 = 255.255.248.0

每个子网主机位数：32 - 21 = 11位

每个子网主机数：2^11 - 2 = 2046个
```

**总结**：子网划分需要借用主机位。

**来源**：408历年真题，考查方式：计算题

---

### 例题 3：ARP工作过程

**题目**：描述ARP协议的工作过程。

**解答**：

```
1. 主机A需要发送数据给主机B，已知B的IP地址，需要获取B的MAC地址
2. 主机A发送ARP请求广播，包含A的IP和MAC，以及目标IP
3. 网络中所有主机接收ARP请求
4. 只有主机B识别到目标IP是自己，发送ARP响应单播给主机A
5. 主机A收到响应，更新ARP缓存
6. 主机A可以发送数据给主机B
```

**总结**：ARP通过广播请求和单播响应获取MAC地址。

**来源**：408历年真题，考查方式：简答题
