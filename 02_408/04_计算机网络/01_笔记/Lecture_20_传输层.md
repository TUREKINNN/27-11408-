## 1. 传输层的功能

### 1.1 传输层的定义

<span style="background-color:#ccc1d9">**传输层**</span>：负责端到端的通信。

### 1.2 传输层的功能

| 功能 | 描述 |
|------|------|
| **端到端通信** | 建立应用程序间的连接 |
| **分段与重组** | 将数据分段传输，到达后重组 |
| **流量控制** | 控制发送速率 |
| **拥塞控制** | 控制网络流量 |
| **可靠传输** | 保证数据可靠到达 |

### 1.3 端口号

**定义**：标识应用程序的编号。

**分类**：

| 范围 | 用途 |
|------|------|
| **0-1023** | 知名端口 |
| **1024-49151** | 注册端口 |
| **49152-65535** | 动态端口 |

**常见端口**：

| 端口 | 协议 | 用途 |
|------|------|------|
| **80** | HTTP | 网页服务 |
| **443** | HTTPS | 安全网页服务 |
| **21** | FTP | 文件传输 |
| **22** | SSH | 安全远程登录 |
| **25** | SMTP | 邮件发送 |

## 2. UDP协议

### 2.1 UDP的特点

<span style="background-color:#ccc1d9">**UDP**</span>：用户数据报协议，无连接，不可靠。

**特点**：

| 特点 | 描述 |
|------|------|
| **无连接** | 不需要建立连接 |
| **不可靠** | 不保证数据到达 |
| **面向报文** | 保留报文边界 |
| **开销小** | 头部开销小 |

### 2.2 UDP的报文结构

```
┌─────────────────────────────────────────────┐
│ 源端口 │ 目的端口 │ 长度 │ 校验和 │         │
│ 16位   │ 16位    │ 16位 │ 16位   │         │
├─────────────────────────────────────────────┤
│                 数据                         │
└─────────────────────────────────────────────┘
```

### 2.3 UDP的适用场景

| 场景 | 描述 |
|------|------|
| **实时应用** | 视频、音频 |
| **简单请求** | DNS查询 |
| **广播/组播** | 多播通信 |

## 3. TCP协议

### 3.1 TCP的特点

<span style="background-color:#ccc1d9">**TCP**</span>：传输控制协议，面向连接，可靠。

**特点**：

| 特点 | 描述 |
|------|------|
| **面向连接** | 需要三次握手建立连接 |
| **可靠传输** | 保证数据可靠到达 |
| **面向字节流** | 无报文边界 |
| **流量控制** | 滑动窗口 |
| **拥塞控制** | 慢启动、拥塞避免 |

### 3.2 TCP的报文结构

```
┌─────────────────────────────────────────────┐
│ 源端口 │ 目的端口 │                         │
│ 16位   │ 16位    │                         │
├─────────────────────────────────────────────┤
│              序列号（32位）                  │
├─────────────────────────────────────────────┤
│              确认号（32位）                  │
├─────────────────────────────────────────────┤
│ 数据偏移 │ 保留 │ U A P R S F │ 窗口大小   │
│ 4位     │ 6位  │ R C S S Y I │ 16位       │
│         │      │ G K H T N N │            │
├─────────────────────────────────────────────┤
│ 校验和 │ 紧急指针 │                         │
│ 16位   │ 16位    │                         │
├─────────────────────────────────────────────┤
│              选项（0-40字节）                │
├─────────────────────────────────────────────┤
│                 数据                         │
└─────────────────────────────────────────────┘
```

**标志位**：

| 标志 | 描述 |
|------|------|
| **URG** | 紧急指针有效 |
| **ACK** | 确认号有效 |
| **PSH** | 推送数据 |
| **RST** | 重置连接 |
| **SYN** | 同步序列号 |
| **FIN** | 结束连接 |

### 3.3 TCP的连接管理

#### 3.3.1 三次握手

```
客户端 → 服务器：SYN, seq=x
服务器 → 客户端：SYN+ACK, seq=y, ack=x+1
客户端 → 服务器：ACK, seq=x+1, ack=y+1
```

**作用**：

1. 确认双方的发送能力
2. 确认双方的接收能力
3. 同步序列号

#### 3.3.2 四次挥手

```
客户端 → 服务器：FIN, seq=x
服务器 → 客户端：ACK, ack=x+1
服务器 → 客户端：FIN, seq=y
客户端 → 服务器：ACK, ack=y+1
```

**作用**：

1. 客户端请求关闭连接
2. 服务器确认关闭请求
3. 服务器请求关闭连接
4. 客户端确认关闭请求

### 3.4 TCP的可靠传输

#### 3.4.1 序列号和确认号

**序列号**：标识数据段的位置。

**确认号**：期望收到的下一个字节的序号。

#### 3.4.2 重传机制

**超时重传**：超过超时时间重传。

**快速重传**：收到3个重复确认后重传。

#### 3.4.3 流量控制

<span style="background-color:#ccc1d9">**滑动窗口**</span>：接收方告知发送方可以发送的数据量。

**原理**：

1. 接收方设置窗口大小
2. 发送方根据窗口大小发送数据
3. 收到确认后滑动窗口

### 3.5 TCP的拥塞控制

#### 3.5.1 慢启动

```
拥塞窗口从1开始，每次收到确认加倍
```

**特点**：快速增长。

#### 3.5.2 拥塞避免

```
拥塞窗口达到阈值后，每次收到确认加1
```

**特点**：线性增长。

#### 3.5.3 快速重传和快速恢复

```
收到3个重复确认后：
1. 快速重传丢失的报文段
2. 降低阈值
3. 进入快速恢复状态
```

**特点**：快速恢复，避免慢启动。

## 4. TCP与UDP的比较

### 4.1 对比

| 对比项 | TCP | UDP |
|--------|-----|-----|
| **连接** | 面向连接 | 无连接 |
| **可靠性** | 可靠 | 不可靠 |
| **顺序** | 保证顺序 | 不保证顺序 |
| **流量控制** | 有 | 无 |
| **拥塞控制** | 有 | 无 |
| **开销** | 大 | 小 |
| **适用场景** | 文件传输 | 实时应用 |

## 5. 典型例题

### 例题 1：TCP三次握手

**题目**：描述TCP三次握手的过程。

**解答**：

```
第一次握手：
客户端 → 服务器：SYN=1, seq=x

第二次握手：
服务器 → 客户端：SYN=1, ACK=1, seq=y, ack=x+1

第三次握手：
客户端 → 服务器：ACK=1, seq=x+1, ack=y+1

作用：
1. 客户端确认服务器的接收能力
2. 服务器确认客户端的发送和接收能力
3. 客户端确认服务器的发送能力
4. 同步双方的序列号
```

**总结**：三次握手确保双方通信能力正常。

**来源**：408历年真题，考查方式：简答题

### 例题 2：TCP拥塞控制

**题目**：描述TCP拥塞控制的过程。

**解答**：

```
1. 慢启动：
   - 拥塞窗口cwnd从1开始
   - 每次收到确认，cwnd加倍
   - 直到cwnd达到阈值ssthresh

2. 拥塞避免：
   - cwnd达到阈值后
   - 每次收到确认，cwnd加1
   - 线性增长

3. 快速重传：
   - 收到3个重复确认
   - 立即重传丢失的报文段

4. 快速恢复：
   - 设置ssthresh = cwnd / 2
   - 设置cwnd = ssthresh + 3
   - 进入拥塞避免状态
```

**总结**：TCP拥塞控制通过慢启动和拥塞避免平衡吞吐量和稳定性。

**来源**：408历年真题，考查方式：简答题

### 例题 3：TCP和UDP对比

**题目**：比较TCP和UDP的特点。

**解答**：

```
TCP：
- 面向连接
- 可靠传输
- 保证顺序
- 有流量控制
- 有拥塞控制
- 开销大
- 适用：文件传输、网页浏览

UDP：
- 无连接
- 不可靠传输
- 不保证顺序
- 无流量控制
- 无拥塞控制
- 开销小
- 适用：视频通话、DNS查询
```

**总结**：TCP适合需要可靠性的场景，UDP适合实时性要求高的场景。

**来源**：408历年真题，考查方式：简答题
