## 一、按章节分类

### 第一章 计算机网络概述

本章是计算机网络的基础入门章节，主要介绍计算机网络的基本概念、分类方式、性能指标以及体系结构等内容。虽然分值占比不高，但概念理解对后续章节的学习至关重要。

| 题型 | 题目数量 | 难度 | 解题方法 |
|------|----------|------|----------|
| 概念题 | 15道 | ★★ | 网络分类、性能指标计算 |
| 计算题 | 10道 | ★★★ | 时延计算、信道利用率 |

**核心知识点**：
- 计算机网络的定义与功能（数据通信、资源共享、分布式处理等）
- 网络的分类（按覆盖范围：LAN、MAN、WAN；按拓扑结构：星型、环型、总线型、网状型）
- 计算机网络的性能指标（速率、带宽、吞吐量、时延、时延带宽积、往返时间RTT、信道利用率）
- 分组交换与电路交换、报文交换的对比
- 计算机网络体系结构（OSI七层模型、TCP/IP四层模型、五层参考模型）
- 协议、接口、服务的概念
- 面向连接服务与无连接服务的区别
- 确认与重传机制

**典型例题详解**：

例题1：假设一个数据包长度为10000bit，从源主机到目的主机共经过3段链路，每段链路长度为100km，信号传播速率为2×10^8 m/s，链路带宽为1Mbps，求总时延。

解题过程：
- 传播时延 = 距离/传播速率 = 100×10³ / (2×10^8) = 0.5ms（每段）
- 传输时延 = 数据长度/带宽 = 10000 / (1×10^6) = 10ms（每段）
- 总时延 = 3×(0.5 + 10) = 31.5ms（忽略排队时延和处理时延）

例题2：若信道带宽为10Mbps，信号传播时延为5ms，数据帧长度为10000bit，求信道利用率。

解题过程：
- 发送时延 = 10000 / (10×10^6) = 1ms
- 往返时延 RTT = 2×5ms = 10ms
- 信道利用率 = 发送时延 / (发送时延 + RTT) = 1 / (1 + 10) ≈ 9.1%

### 第二章 物理层

物理层是OSI模型的最底层，负责在物理介质上传输原始比特流。本章需要掌握各种编码方式、调制技术、奈奎斯特准则和香农定理等核心内容。

| 题型 | 题目数量 | 难度 | 解题方法 |
|------|----------|------|----------|
| 概念题 | 10道 | ★★ | 编码方式、调制技术 |
| 计算题 | 15道 | ★★★ | 奈奎斯特/香农定理应用 |

**核心知识点**：
- 物理层的功能与特性（机械特性、电气特性、功能特性、过程特性）
- 数据通信的基础知识（信号、信道、带宽、码元、波特率、比特率）
- 基带传输与频带传输
- 编码方式：
  - 归零编码（RZ）与非归零编码（NRZ）
  - 反向非归零编码（NRZI）
  - 曼彻斯特编码（Manchester Encoding）
  - 差分曼彻斯特编码
  - 4B/5B编码
- 调制技术：
  - 调幅（ASK）
  - 调频（FSK）
  - 调相（PSK）
  - 正交振幅调制（QAM）
- 奈奎斯特定理：最大数据传输率 = 2W × log₂(V) bps
- 香农定理：最大数据传输率 = W × log₂(1 + S/N) bps
- 信噪比的计算：SNR(dB) = 10 × log₁₀(S/N)
- 传输介质（双绞线、同轴电缆、光纤、无线传输）
- 物理层设备（中继器、集线器）

**典型例题详解**：

例题1：某信道带宽为4kHz，信噪比为30dB，求该信道的最大数据传输率。

解题过程：
- 信噪比 SNR(dB) = 30dB
- S/N = 10^(30/10) = 1000
- 根据香农定理：C = W × log₂(1 + S/N) = 4000 × log₂(1001) ≈ 4000 × 10 = 40kbps

例题2：若采用曼彻斯特编码，数据传输速率为10Mbps，则信号速率（波特率）为多少？

解题过程：
- 曼彻斯特编码每个比特对应两个码元
- 信号速率 = 2 × 数据速率 = 2 × 10Mbps = 20Mbaud

**CSMA/CD与CSMA/CA对比**：

| 特性 | CSMA/CD | CSMA/CA |
|------|---------|---------|
| 应用场景 | 有线局域网 | 无线局域网 |
| 冲突检测方式 | 边发边检测 | 发送前避免冲突 |
| 最小帧长 | 64字节 | 无最小帧长要求 |
| 退避算法 | 截断二进制指数退避 | 二进制指数退避 |
| 确认机制 | 无 | 有ACK确认 |

**典型例题详解**：

例题1：某以太网数据传输速率为100Mbps，信号传播速度为2×10^8 m/s，求CSMA/CD的最小帧长和最大网络长度。

解题过程：
- 最小帧长 = 2 × 传输速率 × 最大传播时延
- 以太网规定最大网络长度为2500m（10BASE5）
- 最大传播时延 = 2500 / (2×10^8) = 12.5μs
- 最小帧长 = 2 × 100×10^6 × 12.5×10^-6 = 2500bit ≈ 312字节
- 但实际上以太网最小帧长固定为64字节（512bit）

例题2：使用生成多项式 G(x) = x^4 + x + 1，对数据 1010011 进行CRC编码。

解题过程：
- 生成多项式 G(x) = x^4 + x + 1，对应二进制 10011
- 数据 1010011 后面补4个0：10100110000
- 用 10011 进行模2除法
- 余数为 0110
- 最终发送数据：10100110110

**子网划分公式汇总**：

1. 子网数量 = 2^n（n为借用的主机位数，不考虑全0全1子网时为 2^n - 2）
2. 每个子网的主机数 = 2^m - 2（m为剩余的主机位数）
3. 子网掩码 = 网络位全1 + 子网位全1 + 主机位全0
4. 子网地址 = IP地址 AND 子网掩码
5. 广播地址 = 子网地址 OR (NOT 子网掩码)
6. 可用IP范围 = 子网地址 + 1 到 广播地址 - 1

**CIDR记法**：
- /8 表示前8位是网络位，对应子网掩码 255.0.0.0
- /16 表示前16位是网络位，对应子网掩码 255.255.0.0
- /24 表示前24位是网络位，对应子网掩码 255.255.255.0

**路由协议对比**：

| 特性 | RIP | OSPF | BGP |
|------|-----|------|-----|
| 类型 | 内部网关 | 内部网关 | 外部网关 |
| 算法 | 距离向量 | 链路状态 | 路径向量 |
| 度量 | 跳数 | 开销（带宽） | 策略 |
| 更新方式 | 周期性广播 | 事件触发 | 事件触发 |
| 收敛速度 | 慢 | 快 | 中等 |
| 最大跳数 | 15 | 无限制 | 无限制 |
| 传输层协议 | UDP | IP直接封装 | TCP |
| 端口号 | 520 | 89 | 179 |

**典型例题详解**：

例题1：某单位申请到一个C类地址 192.168.1.0/24，需要划分成6个子网，每个子网至少有20台主机，求子网掩码。

解题过程：
- 需要6个子网，至少需要借3位（2^3 = 8 ≥ 6）
- 每个子网主机数 = 2^5 - 2 = 30 ≥ 20，满足要求
- 子网掩码 = 255.255.255.224（即 /27）
- 各子网地址：192.168.1.0/27, 192.168.1.32/27, 192.168.1.64/27, 192.168.1.96/27, 192.168.1.128/27, 192.168.1.160/27

例题2：已知CIDR地址块 206.0.64.0/18，求该地址块的最小地址、最大地址和包含的IP地址数。

解题过程：
- /18 表示前18位是网络位
- 206.0.64.0 = 206.0.01000000.00000000
- 最小地址：206.0.64.0
- 最大地址：206.0.127.255
- IP地址数 = 2^(32-18) = 2^14 = 16384个

  - UDP首部格式（源端口、目的端口、长度、校验和）

  - TCP特点（面向连接、可靠、面向字节流、全双工）

**TCP与UDP对比**：

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接方式 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |

| 通信方式 | 全双工 | 一对一、一对多、多对多 |
| 应用场景 | HTTP、FTP、SMTP | DNS、RTP、DHCP |

**TCP三次握手详解**：

```
客户端                              服务器
  |                                   |
  |------- SYN=1, seq=x ------------>|  (第一次握手)
  |                                   |
  |<---- SYN=1, ACK=1, seq=y,        |  (第二次握手)
  |      ack=x+1 -------------------->|
  |                                   |
  |------- ACK=1, seq=x+1,           |  (第三次握手)
  |      ack=y+1 -------------------->|
  |                                   |
  |<---------- 数据传输 ------------->|
```

**为什么需要三次握手？**
- 防止已失效的连接请求报文到达服务器，导致错误连接
- 三次握手可以确认双方的收发能力正常

**TCP四次挥手详解**：

```
客户端                              服务器
  |                                   |
  |------- FIN=1, seq=u ------------>|  (第一次挥手)
  |                                   |
  |<---- ACK=1, ack=u+1 -------------|  (第二次挥手)
  |                                   |
  |<---- FIN=1, seq=w ---------------|  (第三次挥手)
  |                                   |
  |------- ACK=1, ack=w+1 ---------->|  (第四次挥手)
  |                                   |
  | (等待2MSL)                        |
```

**为什么需要等待2MSL？**
- 确保最后一个ACK能够到达服务器
- 确保本次连接的所有报文都从网络中消失

**TCP拥塞控制算法详解**：

1. **慢开始（Slow Start）**：
   - 初始拥塞窗口 cwnd = 1 MSS
   - 每收到一个ACK，cwnd翻倍（指数增长）
   - 直到 cwnd ≥ ssthresh（慢开始门限）
   - 进入拥塞避免阶段

2. **拥塞避免（Congestion Avoidance）**：
   - cwnd 每个RTT增加1 MSS（线性增长）
   - 发生超时：ssthresh = cwnd / 2，cwnd = 1，重新慢开始

3. **快重传（Fast Retransmit）**：
   - 连续收到3个重复ACK
   - 立即重传丢失的报文段

4. **快恢复（Fast Recovery）**：
   - ssthresh = cwnd / 2
   - cwnd = ssthresh（不是1）
   - 直接进入拥塞避免阶段

**典型例题详解**：

例题1：TCP连接的初始阈值为32 MSS，传输轮次1-8的拥塞窗口大小分别为1、2、4、8、16、32、33、34 MSS。分析各阶段的拥塞控制策略。

解题过程：
- 第1-6轮次：cwnd从1到32，指数增长，属于慢开始阶段
- 第6轮次：cwnd = ssthresh = 32，切换到拥塞避免
- 第7-8轮次：cwnd从33到34，线性增长，属于拥塞避免阶段

例题2：若TCP的拥塞窗口为24 MSS时发生超时，之后的拥塞窗口如何变化？

解题过程：
- 超时时：ssthresh = 24 / 2 = 12 MSS
- cwnd 重置为 1 MSS
- 重新开始慢开始：1 → 2 → 4 → 8 → 12（到达阈值）
- 切换到拥塞避免：13 → 14 → 15 → ...

**HTTP状态码分类**：

| 状态码 | 含义 | 常见状态码 |
|--------|------|------------|
| 1xx | 信息性状态码 | 100 Continue |
| 2xx | 成功 | 200 OK, 201 Created |
| 3xx | 重定向 | 301 Moved Permanently, 302 Found, 304 Not Modified |
| 4xx | 客户端错误 | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found |
| 5xx | 服务器错误 | 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable |

**DNS查询过程详解**：

当用户在浏览器中输入 www.example.com 时：

1. 浏览器检查自身DNS缓存
2. 操作系统检查本地DNS缓存（hosts文件）
3. 向本地DNS服务器发起递归查询
4. 本地DNS服务器向根DNS服务器发起迭代查询
5. 根DNS服务器返回顶级域DNS服务器地址
6. 本地DNS服务器向顶级域DNS服务器查询
7. 顶级域DNS服务器返回权威DNS服务器地址
8. 本地DNS服务器向权威DNS服务器查询
9. 权威DNS服务器返回IP地址
10. 本地DNS服务器将结果返回给客户端并缓存

**FTP主动模式与被动模式对比**：

| 特性 | 主动模式（PORT） | 被动模式（PASV） |
|------|------------------|------------------|
| 数据连接建立 | 服务器主动连接客户端 | 客户端主动连接服务器 |
| 防火墙友好 | 不友好 | 友好 |
| 客户端在NAT后 | 有问题 | 正常 |
| 服务器在NAT后 | 正常 | 有问题 |

**典型例题详解**：

例题1：简述从输入URL到页面显示的完整过程。

解题过程：
1. DNS解析：域名 → IP地址
2. TCP连接：三次握手建立连接
3. 发送HTTP请求
4. 服务器处理请求并返回HTTP响应
5. 浏览器解析HTML、CSS、JavaScript
6. 渲染页面
7. 连接关闭（四次挥手）

## 三、解题方法总结

### 题型1：子网划分

**解题步骤**：
1. 确定需要的子网数量和主机数量
2. 计算需要借位的数量（n ≥ log₂(子网数)）
3. 验证每个子网的主机数是否满足要求（2^m - 2 ≥ 主机数）
4. 写出子网掩码
5. 列出各子网地址范围（起始地址、结束地址、广播地址）

**典型例题**：
某单位需要划分5个子网，每个子网至少20台主机，求子网掩码。

解题：
- 5个子网需要至少3位（2^3 = 8 ≥ 5）
- 每个子网20台主机需要至少5位（2^5 - 2 = 30 ≥ 20）
- 如果是C类地址，网络位24位 + 子网位3位 = 27位
- 子网掩码：255.255.255.224 或 /27

**易错点**：
- 注意全0和全1子网是否可用（现代网络通常可用）
- 广播地址 = 子网地址 OR (NOT 子网掩码)
- 可用主机地址 = 广播地址 - 1

### 题型2：TCP拥塞控制

**解题步骤**：
1. 确定初始窗口大小和阈值
2. 判断当前阶段（慢开始、拥塞避免、快恢复）
3. 根据事件（收到ACK、超时、收到3个重复ACK）计算窗口变化
4. 画出窗口变化曲线图

**典型例题**：
TCP连接的初始阈值为16 MSS，画出前10个RTT的拥塞窗口变化图。

解题：
- RTT 1-4：慢开始，cwnd = 1, 2, 4, 8
- RTT 5：cwnd = 16，达到阈值，切换到拥塞避免
- RTT 6-10：拥塞避免，cwnd = 17, 18, 19, 20, 21

**易错点**：
- 慢开始是指数增长，拥塞避免是线性增长
- 超时时阈值减半，窗口重置为1
- 快重传后进入快恢复，不是慢开始

### 题型3：路由表分析

**解题步骤**：
1. 分析网络拓扑结构
2. 运行路由算法（RIP使用距离向量，OSPF使用链路状态）
3. 填写路由表（目的网络、子网掩码、下一跳、接口）
4. 验证路由正确性（检查环路、检查连通性）

**典型例题**：
给定网络拓扑，使用RIP协议计算各路由器的路由表。

解题：
1. 初始化：每个路由器只知道直连网络
2. 第一轮交换：路由器向邻居发送自己的路由表
3. 更新路由表：根据收到的信息更新（Bellman-Ford算法）
4. 重复直到收敛

**易错点**：
- RIP最大跳数为15，16表示不可达
- OSPF使用Dijkstra算法，需要画出最短路径树
- BGP是路径向量协议，需要考虑AS路径

### 题型4：CRC校验计算

**解题步骤**：
1. 确定生成多项式 G(x)，得到除数
2. 在数据后面补 r 个0（r为生成多项式的最高次数）
3. 进行模2除法（异或运算）
4. 得到余数，替换补的0
5. 发送端发送数据 + 余数

**典型例题**：
使用 G(x) = x^4 + x + 1 对数据 1101011011 进行CRC编码。

解题：
- G(x) = x^4 + x + 1，对应二进制 10011
- 数据 1101011011 后面补4个0：11010110110000
- 进行模2除法，得到余数 1110
- 最终发送数据：11010110111110

### 题型5：IP分片计算

**解题步骤**：
1. 确定MTU（最大传输单元）
2. 计算每个分片的数据部分长度（必须是8字节的倍数）
3. 设置标识、标志、片偏移
4. 计算每个分片的总长度

**典型例题**：
一个4000字节的数据报（20字节首部 + 3980字节数据），经过MTU为1500字节的网络，如何分片？

解题：
- MTU = 1500字节，最大数据部分 = 1500 - 20 = 1480字节
- 分片1：数据1480字节，偏移0，MF=1
- 分片2：数据1480字节，偏移185（1480/8），MF=1
- 分片3：数据1020字节，偏移370（2960/8），MF=0

## 五、易错题汇总

### 1. 子网划分中全0全1子网是否可用

**易错点**：
- 传统观点认为全0和全1子网不可用
- 现代网络（CIDR）通常允许使用
- 考试中需要根据题目要求判断

**正确理解**：
- 全0子网：子网地址中主机位全为0，如192.168.1.0/27
- 全1子网：子网地址中主机位全为1，如192.168.1.224/27
- RFC 1812允许使用全0和全1子网

**解题技巧**：
- 题目明确说"不考虑全0全1子网"时，子网数 = 2^n - 2
- 题目没有说明时，通常按可用计算

### 2. CSMA/CD最小帧长计算

**易错点**：
- 最小帧长 = 2 × 传输速率 × 最大传播时延
- 以太网规定最小帧长为64字节（512bit）
- 计算时注意单位换算（Mbps、m、μs）

**正确公式**：
- 最小帧长（bit）= 2 × 数据传输速率（bps）× 最大传播时延（s）
- 最大传播时延 = 最大网络长度 / 信号传播速度
- 以太网信号传播速度约为 2×10^8 m/s

**典型错误**：
- 错误：用单程传播时延而不是往返时延
- 错误：忘记将结果转换为字节（除以8）

### 3. TCP三次握手状态转换

**易错点**：
- 客户端状态：CLOSED → SYN_SENT → ESTABLISHED
- 服务器状态：LISTEN → SYN_RCVD → ESTABLISHED
- 第三次握手时客户端进入ESTABLISHED状态

**正确状态转换**：

```
客户端状态转换：
CLOSED → SYN_SENT → ESTABLISHED

服务器状态转换：
CLOSED → LISTEN → SYN_RCVD → ESTABLISHED
```

**常见错误**：
- 错误：认为服务器在收到SYN后直接进入ESTABLISHED
- 正确：服务器需要收到第三个ACK后才进入ESTABLISHED

### 4. 拥塞控制窗口变化规律

**易错点**：
- 慢开始：指数增长（1, 2, 4, 8, 16, ...）
- 拥塞避免：线性增长（+1 MSS/RTT）
- 超时后：阈值减半，窗口重置为1
- 快重传后：阈值减半，窗口设为新阈值（不是1）

**正确规律**：

```
初始：cwnd = 1, ssthresh = 初始值

慢开始阶段：
cwnd: 1 → 2 → 4 → 8 → ... → ssthresh
每个RTT翻倍

拥塞避免阶段：
cwnd: ssthresh → ssthresh+1 → ssthresh+2 → ...
每个RTT增加1

超时事件：
ssthresh = cwnd / 2
cwnd = 1
重新慢开始

快重传事件（3个重复ACK）：
ssthresh = cwnd / 2
cwnd = ssthresh
直接进入拥塞避免
```

### 5. ARP协议工作原理

**易错点**：
- ARP请求是广播，ARP响应是单播
- ARP缓存有过期时间
- ARP只在同一子网内使用

**正确流程**：
1. 主机A检查ARP缓存，查找主机B的MAC地址
2. 如果缓存中没有，发送ARP请求（广播）
3. 主机B收到ARP请求，发送ARP响应（单播）
4. 主机A收到ARP响应，更新ARP缓存

### 6. IP分片与重组

**易错点**：
- 分片在目的主机重组，中间路由器只分片不重组
- 每个分片的数据部分必须是8字节的倍数（最后一个分片除外）
- MF标志：1表示后面还有分片，0表示这是最后一个分片
- 片偏移以8字节为单位

**正确计算**：
- 最大数据部分 = MTU - IP首部长度（通常20字节）
- 分片数 = ceil(总数据长度 / 最大数据部分)
- 每个分片的总长度 = IP首部长度 + 该分片数据长度

### 7. DNS查询类型

**易错点**：
- 递归查询：客户端只发一次请求，DNS服务器负责完成全部查询
- 迭代查询：DNS服务器返回下一级DNS服务器地址，客户端继续查询
- 通常客户端到本地DNS是递归，本地DNS到其他DNS是迭代

**正确理解**：

```
客户端 → 本地DNS：递归查询
本地DNS → 根DNS：迭代查询
本地DNS → 顶级域DNS：迭代查询
本地DNS → 权威DNS：迭代查询
```

### 8. TCP序列号与确认号

**易错点**：
- 序列号（Seq）：本报文段数据的第一个字节的编号
- 确认号（Ack）：期望收到的下一个字节的编号
- 确认号 = 对方序列号 + 数据长度

**正确计算**：
- 发送方发送：Seq = x, 数据长度 = 100
- 接收方确认：Ack = x + 100
- 发送方发送：Seq = x + 100

### 9. HTTP持久连接

**易错点**：
- HTTP/1.0默认非持久连接，需要设置 Connection: Keep-Alive
- HTTP/1.1默认持久连接，需要设置 Connection: Close 关闭
- 持久连接可以减少TCP连接建立的开销

**正确理解**：
- 非持久连接：每个请求/响应都需要单独的TCP连接
- 持久连接：多个请求/响应可以共享同一个TCP连接
- 管线化：持久连接可以不等待响应就发送下一个请求

### 10. RIP路由环路问题

**易错点**：
- RIP使用水平分割、毒性逆转、触发更新等机制防止环路
- 最大跳数为15，16表示不可达
- 每30秒广播一次完整路由表

**防止环路机制**：
- 水平分割：不向来源接口发送路由信息
- 毒性逆转：向来源接口发送度量值为16的路由
- 触发更新：路由变化时立即发送更新
- 抑制计时器：防止频繁更新

## 六、公式速查表

### 1. 性能指标计算

```
总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
发送时延 = 数据长度 / 信道带宽
传播时延 = 信道长度 / 信号传播速率
信道利用率 = 发送时延 / (发送时延 + 往返时延)
```

### 2. 物理层计算

```
奈奎斯特定理：最大数据传输率 = 2W × log₂(V)
香农定理：最大数据传输率 = W × log₂(1 + S/N)
信噪比：SNR(dB) = 10 × log₁₀(S/N)
波特率与比特率：比特率 = 波特率 × log₂(码元状态数)
```

### 3. 数据链路层计算

```
最小帧长 = 2 × 传输速率 × 最大传播时延
CRC校验：数据 ÷ 生成多项式 = 商 + 余数
停等协议效率：η = 1 / (1 + 2a)，其中a = 传播时延/发送时延
滑动窗口效率：GBN: η = (n) / (1 + 2a)，SR: η = (n) / (1 + a)
```

### 4. 网络层计算

```
子网数量 = 2^n（n为子网位数）
每个子网主机数 = 2^m - 2（m为主机位数）
子网掩码 = 网络位全1 + 子网位全1 + 主机位全0
子网地址 = IP地址 AND 子网掩码
广播地址 = 子网地址 OR (NOT 子网掩码)
CIDR地址块大小 = 2^(32 - 前缀长度)
```

### 5. 传输层计算

```
TCP窗口大小：min(rwnd, cwnd)
慢开始：cwnd = 2^n（n为RTT数）
拥塞避免：cwnd = ssthresh + n（n为RTT数）
序列号计算：Ack = Seq + 数据长度
```
