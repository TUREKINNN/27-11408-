# 4.2 路由算法与路由协议

> 路由算法是网络层的核心，本质是**图的最短路径问题**。路由协议定义了路由器之间如何交换路由信息。大纲要求掌握：RIP、OSPF、BGP。

---

## 一、路由算法

### 1.1 路由算法与路由协议的关系

```
路由协议 → 定义路由器之间如何交换信息 → 作为路由算法的输入
路由算法 → 计算最佳转发路径 → 输出路由表
```

### 1.2 路由算法分类

| 类型 | 特点 | 适用场景 | 对应协议 |
|------|------|---------|---------|
| **静态路由** | 管理员手工配置 | 小型网络 | - |
| **动态路由** | 自动适应网络变化 | 大型网络 | RIP/OSPF/BGP |

### 1.3 距离-向量路由算法（Bellman-Ford）

> $$d_x(\text{net1}) = \min\{c(x,v) + d_v(\text{net1})\}$$
> 其中 $v$ 是 $x$ 的所有邻居，$c(x,v)$ 是 $x$ 到邻居 $v$ 的距离，$d_v(\text{net1})$ 是 $v$ 到目的网络的最短距离。

**特点**：
- 每个路由器只需要知道**邻居到目的网络的距离**
- 不需要知道完整的网络拓扑
- 周期性地与邻居交换路由信息
- 收敛较慢，可能出现**无穷计数**问题

> **RIP 协议**使用距离-向量路由算法。

### 1.4 链路状态路由算法（Dijkstra）

> 每台路由器都了解**完整的网络拓扑结构**（带权图），然后用 **Dijkstra 算法**计算到所有目的网络的最短路径。

**特点**：
- 每个路由器需要知道**全网拓扑**
- 当链路状态变化时，向所有路由器**泛洪**通告
- 收敛快，不会出现无穷计数
- 计算开销较大

> **OSPF 协议**使用链路状态路由算法。

### 1.5 两种算法对比

| 特性 | 距离-向量 | 链路状态 |
|------|----------|---------|
| 信息范围 | 只知道邻居信息 | 知道全网拓扑 |
| 算法 | Bellman-Ford | Dijkstra |
| 收敛速度 | 慢 | 快 |
| 通信开销 | 低（只与邻居交换） | 高（泛洪到所有节点） |
| 无穷计数 | 可能出现 | 不会出现 |
| 对应协议 | RIP | OSPF |

## 二、分层次的路由协议

### 2.1 自治系统（AS）

> 在单一技术管理下的一组路由器，使用统一的路由协议。互联网被划分为多个 AS。

| 路由协议类型 | 作用范围 | 对应协议 |
|------------|---------|---------|
| **IGP**（内部网关协议） | AS 内部 | RIP、OSPF |
| **EGP**（外部网关协议） | AS 之间 | BGP |

---

## 三、RIP 协议（Routing Information Protocol）

### 3.1 基本特点

| 特性 | 说明 |
|------|------|
| 类型 | 距离-向量协议 |
| 度量 | **跳数**（Hop Count），每经过一个路由器+1 |
| 最大跳数 | **15**，16 表示不可达 |
| 适用范围 | 小型自治系统 |
| 交换对象 | 仅与**直接相邻**的路由器交换 |
| 交换内容 | 本路由器的**完整路由表** |
| 交换周期 | **30 秒** |
| 超时时间 | **180 秒**（未收到更新则标记不可达） |
| 传输方式 | 使用 **UDP**，端口号 **520** |

### 3.2 路由表项

每条路由表项包含三个关键字段：

```
<目的网络 N, 距离 d, 下一跳路由器地址 X>
```

### 3.3 RIP 的更新规则

收到邻居 X 发来的 RIP 报文后：

1. **修改报文**：将所有"下一跳"改为 X，所有"距离"加 1
2. **逐项处理**：
   - 若路由表中**没有**目的网络 N → **添加**新条目
   - 若有 N 且下一跳是 X → **替换**（以最新消息为准）
   - 若有 N 且下一跳不是 X → 若新距离更小则**更新**，否则不变
3. **超时处理**：180 秒未收到邻居更新 → 距离设为 16（不可达）

### 3.4 RIP 的优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单 | 最大跳数限制为15，不适合大型网络 |
| 开销小 | 收敛慢，可能出现无穷计数 |
| 配置简单 | 周期性广播占用带宽 |

> 为加快收敛，可引入**触发更新**机制：当网络拓扑变化时立即通告，不必等待30秒。

## 四、OSPF 协议（Open Shortest Path First）

### 4.1 基本特点

| 特性 | 说明 |
|------|------|
| 类型 | **链路状态**协议 |
| 度量 | 链路**代价**（Cost），通常与带宽成反比 |
| 算法 | **Dijkstra** 最短路径算法 |
| 适用范围 | 大型自治系统 |
| 交换对象 | 向**所有路由器**泛洪链路状态信息 |
| 传输方式 | 直接使用 **IP**（协议号 89），不用 TCP/UDP |

### 4.2 OSPF 的区域（Area）

为减少通信开销，OSPF 将 AS 划分为多个**区域**：

```
         ┌──────────────┐
         │   Area 0     │  ← 主干区域
         │  (Backbone)  │
         └──┬───┬───┬───┘
            │   │   │
        ┌───┘   │   └───┐
        ▼       ▼       ▼
    ┌──────┐ ┌──────┐ ┌──────┐
    │Area 1│ │Area 2│ │Area 3│
    └──────┘ └──────┘ └──────┘
```

- **区域内部路由器**：只知道本区域的拓扑
- **区域边界路由器（ABR）**：连接不同区域
- **自治系统边界路由器（ASBR）**：连接不同 AS

### 4.3 OSPF 的分组类型

| 分组类型 | 功能 |
|---------|------|
| **Hello** | 发现和维护邻居关系 |
| **DBD** | 数据库描述，交换链路状态摘要 |
| **LSR** | 链路状态请求 |
| **LSU** | 链路状态更新（携带 LSA） |
| **LSAck** | 链路状态确认 |

### 4.4 OSPF 的工作过程

1. 使用 **Hello 分组**发现邻居
2. 交换 **DBD 分组**，同步链路状态数据库
3. 链路状态变化时，泛洪 **LSU 分组**
4. 使用 **Dijkstra 算法**计算最短路径树
5. 生成**路由表**

---

## 五、BGP 协议（Border Gateway Protocol）

### 5.1 基本特点

| 特性 | 说明 |
|------|------|
| 类型 | **路径-向量**协议 |
| 作用 | AS 之间的路由选择 |
| 传输方式 | 使用 **TCP**，端口号 **179** |
| 度量 | 基于**策略**，不只是最短距离 |

### 5.2 BGP 的两种连接

| 连接类型 | 说明 |
|---------|------|
| **eBGP** | 不同 AS 之间的 BGP 连接 |
| **iBGP** | 同一 AS 内部的 BGP 连接 |

### 5.3 BGP 报文

| 报文类型 | 功能 |
|---------|------|
| **OPEN** | 建立 BGP 连接 |
| **UPDATE** | 通告/撤销路由 |
| **KEEPALIVE** | 保持连接活跃 |
| **NOTIFICATION** | 报告错误 |

### 5.4 BGP 的路由选择

BGP 不仅仅看"最短路径"，还要考虑：
- AS 级别的跳数
- **策略约束**（如：不经过某些 AS）
- 本地偏好值
- MED 值

---

## 六、三种路由协议对比

| 特性 | RIP | OSPF | BGP |
|------|-----|------|-----|
| 类型 | 距离-向量 | 链路状态 | 路径-向量 |
| 作用范围 | AS 内部 | AS 内部 | AS 之间 |
| 度量 | 跳数（最大15） | 链路代价 | 策略+路径 |
| 传输层协议 | UDP (520) | IP (89) | TCP (179) |
| 收敛速度 | 慢 | 快 | - |
| 适用规模 | 小型 | 大型 | 全球互联网 |
| 算法 | Bellman-Ford | Dijkstra | 路径选择 |

---

## 考点总结

> 1. **距离-向量算法**：只知道邻居信息，Bellman-Ford，RIP 使用
> 2. **链路状态算法**：知道全网拓扑，Dijkstra，OSPF 使用
> 3. **RIP**：跳数度量，最大15跳，UDP传输，30秒交换周期
> 4. **OSPF**：链路代价度量，Dijkstra算法，IP直接传输，支持区域划分
> 5. **BGP**：AS间协议，TCP传输，基于策略选择路径
> 6. IGP（RIP/OSPF）在 AS 内部，EGP（BGP）在 AS 之间
> 7. RIP 更新规则：没有→添加，同一邻居→替换，不同邻居→比较更新
