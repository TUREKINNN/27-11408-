# 3.4 流量控制与可靠传输机制

> 王道考研·计算机网络 第三章 数据链路层

---

## 一、基本概念

### 1.1 流量控制

限制发送方的发送速率，使接收方来得及接收。

### 1.2 可靠传输

确保接收方能够正确、有序地收到所有发送的数据。


---

## 二、滑动窗口机制

### 2.1 发送窗口 $W_T$

发送方当前允许发送的帧的范围。

```
发送窗口 W_T = 4
待发送帧: A B C D E F G H I J K L
          ├───────┤
          ↑ 允许发送  ↑ 窗口外暂不允许发送
```

### 2.2 接收窗口 $W_R$

接收方当前允许接收的帧的范围。

```
接收窗口 W_R = 2
待接收帧: A B C D E F G H I J K L
          ├───┤
          ↑ 允许接收  ↑ 窗口外的帧直接丢弃
```

### 2.3 窗口滑动条件

- **发送窗口滑动**：收到已发送帧的确认
- **接收窗口滑动**：收到按序到达的帧

---

## 三、三种协议

### 3.1 停止-等待协议（SW）

**特点**：$W_T = 1, W_R = 1$

```
发送方A                          接收方B
  │── 发送帧 0 ──────────────────►│
  │                                │
  │◄─────── ACK 1 ───────────────│
  │                                │
  │── 发送帧 1 ──────────────────►│
  │                                │
  │◄─────── ACK 0 ───────────────│
```

**信道利用率**：

$$
U = \frac{T_D}{T_D + RTT + T_A}
$$

其中：$T_D$ = 发送时延，$T_A$ = 确认帧发送时延，$RTT$ = 往返时延

> 发送方发送帧后启动计时器，超时未收到 ACK 则重传。

### 3.2 后退 N 帧协议（GBN）

**特点**：$W_T > 1, W_R = 1$

- 发送方可连续发送多个帧
- 接收方只接受**按序到达**的帧
- 若某帧出错，接收方丢弃该帧及其后续所有帧
- 发送方超时后，从出错帧开始**全部重传**

```
发送方A                          接收方B
  │── 帧 0 ─────────────────────►│ ✓
  │── 帧 1 ─────────────────────►│ ✗ (丢失)
  │── 帧 2 ─────────────────────►│ ✗ (丢弃，非按序)
  │── 帧 3 ─────────────────────►│ ✗ (丢弃，非按序)
  │                                │
  │◄─────── ACK 0 ───────────────│ (确认帧0)
  │                                │
  │ (超时，重传帧1,2,3)            │
  │── 帧 1 ─────────────────────►│ ✓
  │── 帧 2 ─────────────────────►│ ✓
  │── 帧 3 ─────────────────────►│ ✓
  │◄─────── ACK 3 ───────────────│
```

**窗口大小约束**：$W_T \leq 2^n - 1$（$n$ 为序号位数）

**信道利用率**：

$$
U = \frac{W_T \times T_D}{T_D + RTT + T_A}
$$

### 3.3 选择重传协议（SR）

**特点**：$W_T > 1, W_R > 1`

- 发送方可连续发送多个帧
- 接收方可以**缓存**乱序到达的帧
- 发送方只**重传出错的帧**

```
发送方A                          接收方B
  │── 帧 0 ─────────────────────►│ ✓ 缓存
  │── 帧 1 ─────────────────────►│ ✗ (丢失)
  │── 帧 2 ─────────────────────►│ ✓ 缓存
  │── 帧 3 ─────────────────────►│ ✓ 缓存
  │                                │
  │◄─── ACK 0, NAK 1, ACK 2,3 ──│
  │                                │
  │── 帧 1 ─────────────────────►│ ✓ 交付帧0,1,2,3
  │◄─────── ACK 1 ───────────────│
```

**窗口大小约束**：$W_T \leq 2^{n-1}$，$W_R \leq 2^{n-1}$

---

## 四、三种协议对比

| 特性 | SW | GBN | SR |
|------|-----|-----|-----|
| 发送窗口 | $W_T = 1$ | $W_T > 1$ | $W_T > 1$ |
| 接收窗口 | $W_R = 1$ | $W_R = 1$ | $W_R > 1$ |
| 重传方式 | 重传出错帧 | 重传出错帧及后续所有帧 | 只重传出错帧 |
| 接收方缓存 | 不需要 | 不需要 | 需要 |
| 信道利用率 | 低 | 较高 | 最高 |
| 实现复杂度 | 简单 | 较简单 | 复杂 |

---

## 五、信道利用率分析

### 5.1 停止-等待协议

$$
U_{SW} = \frac{T_D}{T_D + RTT + T_A}
$$

### 5.2 GBN 协议

$$
U_{GBN} = \frac{W_T \times T_D}{T_D + RTT + T_A}
$$

> 当 $W_T$ 足够大时，$U \to 1$

### 5.3 最大窗口计算

> 给定信道利用率要求，求最小窗口大小。

**例**：设 $T_D = 1$ms，$RTT = 20$ms，$T_A \approx 0$，要求信道利用率 $\geq 80\%$，求 $W_T$ 的最小值。

$$
U = \frac{W_T \times 1}{1 + 20 + 0} = \frac{W_T}{21} \geq 0.8
$$

$$
W_T \geq 16.8 \implies W_T = 17
$$

## 六、考点总结

| 考点 | 频率 | 重点 |
|------|------|------|
| 三种协议对比 | ⭐⭐⭐ | 窗口大小、重传机制 |
| 窗口大小约束 | ⭐⭐⭐ | GBN: $2^n-1$，SR: $2^{n-1}$ |
| 信道利用率计算 | ⭐⭐⭐ | 公式应用 |
| 超时重传机制 | ⭐⭐ | 计时器设置 |

---

> **Q**：数据链路层使用 GBN 协议，序号位数为 3，发送窗口最大为多少？
>
> **A**：$W_{T,max} = 2^3 - 1 = 7$
