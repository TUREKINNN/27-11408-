# 3.6 局域网

> 局域网（LAN）是在有限地理范围内将多台计算机互联的网络。本节涵盖局域网基本概念、以太网（IEEE 802.3）、无线局域网（IEEE 802.11）和 VLAN。

---

## 一、局域网与 IEEE 802 标准

### 1.1 IEEE 802 的层次划分

IEEE 802 标准将数据链路层分为两个子层：

| 子层 | 全称 | 功能 |
|------|------|------|
| **LLC** | Logical Link Control 逻辑链路控制 | 为网络层提供服务（名存实亡） |
| **MAC** | Medium Access Control 介质访问控制 | 帧的封装/解封、差错检测、介质访问控制 |

> 现代以太网中，LLC层的功能已被弱化，实际使用中主要关注MAC层。

### 1.2 局域网的拓扑结构

| 拓扑 | 特点 |
|------|------|
| 星形 | 以集线器/交换机为中心，最常用 |
| 总线形 | 传统以太网，使用同轴电缆 |
| 环形 | 令牌环网 |
| 树形 | 集线器级联 |

## 二、以太网（IEEE 802.3）

### 2.1 两种 MAC 层标准

| 标准 | 推出者 | 帧格式差异 |
|------|--------|----------|
| **DIX Ethernet V2** | DEC、Intel、Xerox | 类型字段（Type） |
| **IEEE 802.3** | IEEE | 长度字段（Length） |

> 市面上常用 **V2 标准**。

### 2.2 以太网 MAC 帧格式（V2 标准）


| 字段 | 长度 | 说明 |
|------|------|------|
| **目的地址** | 6 字节（48bit） | 接收方 MAC 地址，全1为广播帧 |
| **源地址** | 6 字节（48bit） | 发送方 MAC 地址 |
| **类型** | 2 字节 | 指明网络层协议（如 0x0800=IP） |
| **数据** | 46\~1500 字节 | IP 数据报，太长分片，太短填充 |
| **FCS** | 4 字节 | CRC 校验码 |

**帧长限制**：
- 最短帧长：**64 字节**（数据部分最少 46 字节）
- 最长帧长：**1518 字节**（数据部分最多 1500 字节）

**帧定界**：
- 前导码（Preamble）：8 字节，用于时钟同步（曼彻斯特编码）
- 帧结尾：采用**违规编码法**（曼彻斯特编码中不跳变的信号）

> 以太网使用 CSMA/CD 协议，需要保证在发送完一帧之前能检测到冲突。因此帧长不能小于最短帧长。

### 2.3 IEEE 802.3 标准帧格式

与 V2 标准的区别：
- 第3个字段为**长度/类型**字段
- 当值 ≤ 0x0600 时，表示数据部分长度
- 当值 > 0x0600 时，表示上层协议类型（与 V2 兼容）

### 2.4 以太网的通信方式

| 连接方式 | 通信模式 |
|---------|---------|
| 交换机连接 | 通常支持**全双工** |
| 集线器连接 | 仅支持**半双工** |
| 同轴电缆 | 仅支持**半双工** |
| 光纤 | 只支持**全双工** |

## 三、CSMA/CD 协议

### 3.1 基本概念

**CSMA/CD** = Carrier Sense Multiple Access with Collision Detection
**载波监听多路访问 / 碰撞检测**

工作流程：
1. **先听后发**（载波监听）
2. **边听边发**（碰撞检测）
3. **冲突停发**（检测到冲突立即停止发送）
4. **随机重传**（等待随机时间后重传）

### 3.2 争用期（Collision Window）

> 争用期 = **2 × 最大单向传播时延** = $2\tau$
> 
> 一个节点最多需要花一个争用期的时间才能确信自己"占领信道"。

**最短帧长**：

$$\text{最短帧长} = 2\tau \times \text{信道带宽}$$

> A、B 相距 6000m，信号传播速度 $2 \times 10^8$ m/s，信道带宽 10Mbps：
> - 最大单向传播时延 $\tau = 6000 / (2 \times 10^8) = 30\mu s$
> - 争用期 = $2 \times 30 = 60\mu s$
> - 最短帧长 = $60\mu s \times 10\text{Mbps} = 600\text{bit}$

### 3.3 截断二进制指数退避算法

发生冲突后，随机等待时间的算法：
1. 确定基本退避时间：$2\tau$
2. 从 $[0, 1, ..., 2^k - 1]$ 中随机选择 $r$，其中 $k = \min(\text{重传次数}, 10)$
3. 等待时间 = $r \times 2\tau$
4. 重传 16 次仍失败，放弃发送

## 四、CSMA/CA 协议

### 4.1 为什么无线网不用 CSMA/CD？

无线网络中存在**隐蔽站问题**，无法做到"边听边发"，因此使用 **CSMA/CA**（Collision Avoidance，碰撞避免）。

### 4.2 CSMA/CA 工作流程

1. 发送前先**监听信道**
2. 如果信道空闲，等待 **DIFS** 时间后发送
3. 如果信道忙，执行**退避算法**
4. 接收方正确接收后返回 **ACK**
5. 使用 **RTS/CTS** 机制解决隐蔽站问题

## 五、无线局域网（IEEE 802.11）

### 5.1 基本概念

| 术语 | 说明 |
|------|------|
| **BSS** | 基本服务集（Basic Service Set） |
| **AP** | 接入点（Access Point），如 WiFi 路由器 |
| **STA** | 站点（Station），如手机、笔记本 |
| **SSID** | 服务集标识符，即 WiFi 名称 |

### 5.2 两种架构

| 架构 | 说明 |
|------|------|
| **有基础设施** | 通过 AP 连接（最常见的 WiFi） |
| **无基础设施** | Ad hoc 模式，设备直接互联 |

## 六、VLAN（虚拟局域网）

### 6.1 基本概念

> 在逻辑上将一个物理局域网划分为多个**广播域**，不同 VLAN 之间的广播帧不会互相传播。

### 6.2 VLAN 的实现

- 在以太网帧中增加 **VLAN 标签**（Tag），4 字节
- VLAN 标签包含 **VID**（VLAN ID），12 bit，可标识 4096 个 VLAN
- **交换机端口**可配置为属于某个 VLAN

| 端口类型 | 说明 |
|---------|------|
| **Access 端口** | 只属于一个 VLAN，连接终端设备 |
| **Trunk 端口** | 可承载多个 VLAN 的数据，交换机之间互联 |

## 考点总结

> 1. IEEE 802 将数据链路层分为 LLC 和 MAC 两个子层
> 2. **以太网 MAC 帧**：6+6+2+N+4（目的+源+类型/长度+数据+FCS），数据 46\~1500B
> 3. 最短帧长 64B，与 CSMA/CD 的争用期有关
> 4. **CSMA/CD**：先听后发、边听边发、冲突停发、随机重传
> 5. 争用期 = 2τ，最短帧长 = 争用期 × 带宽
> 6. CSMA/CA 用于无线网络，使用 ACK 和 RTS/CTS
> 7. VLAN 通过标签划分广播域，VID 12bit
