# 3.2 组帧

> 数据链路层将网络层的 IP 数据报（分组）封装成**帧**（Frame）。帧 = 帧首 + 数据部分 + 帧尾。组帧的核心问题是：如何在比特流中确定一帧的**开始**和**结束**。

---

## 一、四种组帧方法

### 1.1 字符计数法

**原理**：在每个帧开头，用一个**定长计数字段**表示帧长（包括计数字段本身）。

```
[5] [数据...] [6] [数据...] [7] [数据...]
 ↑帧长=5字节    ↑帧长=6字节    ↑帧长=7字节
```

> 任何一个计数字段出错，都会导致**后续所有帧无法定界**，因此实际中很少使用。

### 1.2 字节填充法（面向字节的协议）

**原理**：使用特殊字符标记帧的开始和结束。

| 控制字符 | 作用 |
|---------|------|
| **SOH**（Start of Header） | 帧开始定界符 |
| **EOT**（End of Transmission） | 帧结束定界符 |

**问题**：如果数据部分恰好包含 SOH 或 EOT 字符怎么办？

**解决方案**：**转义字符法**

- 使用特殊**转义字符 ESC**
- 如果数据中出现 SOH/EOT/ESC，就在前面插入一个 ESC
- 接收方收到 ESC 后，去掉 ESC 并将后面的字符作为数据

```
数据: [SOH] → 发送: [ESC][SOH]
数据: [EOT] → 发送: [ESC][EOT]
数据: [ESC] → 发送: [ESC][ESC]
```

### 1.3 零比特填充法（面向比特的协议）

**原理**：使用特殊的比特模式 `01111110` 作为帧的开始和结束标志。

**填充规则**：
1. 发送方：扫描数据部分，每遇到连续 **5 个 1**，就自动在后面**填充一个 0**
2. 接收方：收到连续 5 个 1 后，去掉后面的 0

```
原始数据: 0110111111111111110
发送数据: 0110111110111110111110
                            ↑ 填充的0
```

> - 实现简单，硬件容易实现
> - 不依赖于字符编码方案
> - HDLC 协议使用此方法

### 1.4 违规编码法

**原理**：利用物理层编码中不会出现的"违规"信号模式作为帧定界符。

> 以太网使用曼彻斯特编码，每个码元中间必须有跳变。如果出现"中间不跳变"的信号，就是违规编码，可用于帧定界。

---

## 二、组帧方法对比

| 方法 | 定界方式 | 适用场景 | 缺点 |
|------|---------|---------|------|
| 字符计数法 | 计数字段 | - | 一个出错，后续全错 |
| 字节填充法 | SOH/EOT + ESC | 面向字节的协议（PPP） | 依赖字符编码 |
| 零比特填充法 | 01111110 标志 | 面向比特的协议（HDLC） | 需要扫描数据 |
| 违规编码法 | 违规信号 | 以太网 | 依赖物理层编码 |

---

## 考点总结

> 1. 字符计数法的缺点：一个计数字段出错导致后续所有帧无法定界
> 2. 字节填充法使用转义字符 ESC 解决数据中出现定界符的问题
> 3. **零比特填充法**：连续5个1插入0，HDLC协议使用
> 4. 违规编码法：利用物理层编码的违规模式，以太网使用
> 5. PPP 协议使用字节填充法，HDLC 使用零比特填充法
