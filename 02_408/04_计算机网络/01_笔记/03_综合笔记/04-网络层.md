| 功能 | 说明 |
|------|------|
| **路由选择** | 确定分组从源到目的的最佳路径 |

## 核心知识

### 4.2 IP地址 ⭐⭐⭐

#### 4.2.1 IP地址分类

| 类别 | 网络号位数 | 范围 | 默认子网掩码 | 说明 |
|------|-----------|------|-------------|------|
| **A类** | 8位（0开头） | 1.0.0.0 ~ 126.255.255.255 | 255.0.0.0 (/8) | 大型网络 |
| **B类** | 16位（10开头） | 128.0.0.0 ~ 191.255.255.255 | 255.255.0.0 (/16) | 中型网络 |
| **C类** | 24位（110开头） | 192.0.0.0 ~ 223.255.255.255 | 255.255.255.0 (/24) | 小型网络 |
| **D类** | 1110开头 | 224.0.0.0 ~ 239.255.255.255 | — | 组播地址 |
| **E类** | 1111开头 | 240.0.0.0 ~ 255.255.255.255 | — | 保留 |

**特殊IP地址**：

| 地址 | 用途 |
|------|------|
| 0.0.0.0 | 本网络本主机 |
| 255.255.255.255 | 本网络广播 |
| 127.x.x.x | 环回地址（本机测试） |
| 网络号全0+主机号 | 本网络特定主机 |
| 网络号+主机号全1 | 特定网络广播 |

#### 4.2.2 子网划分 ⭐⭐⭐

**子网掩码**：将IP地址分为网络号和主机号。

**计算步骤**：
1. 将IP地址和子网掩码转换为二进制
2. **按位与**运算 → 得到**网络地址**
3. 网络地址中主机位全1 → **广播地址**
4. 主机数 = $2^n - 2$（n为主机位数，减去网络地址和广播地址）

**完整例题**：

**题目**：IP地址为 192.168.10.68，子网掩码为 255.255.255.240，求网络地址、广播地址、可用主机范围、可用主机数。

**解答**：
```
IP地址：  192.168.10.01000100（68 = 64+4 = 01000100）
子网掩码：255.255.255.11110000（240 = 11110000）

网络地址（按位与）：192.168.10.01000000 = 192.168.10.64
广播地址（主机位全1）：192.168.10.01001111 = 192.168.10.79
可用主机范围：192.168.10.65 ~ 192.168.10.78
可用主机数：2⁴ - 2 = 14台
```

### 4.3 CIDR与路由聚合 ⭐⭐⭐

**CIDR（无类别域间路由）**：消除了传统的A、B、C类划分。

**记法**：IP地址/前缀长度，如 `192.168.10.68/28`

**路由聚合（超网）**：将多个子网合并为一个较大的地址块。

**例**：将以下4个子网聚合成一个CIDR块：
```
192.168.10.0/24
192.168.11.0/24
192.168.12.0/24
192.168.13.0/24
```

分析：
```
10 = 00001010
11 = 00001011
12 = 00001100
13 = 00001101
```
共同前缀：`00001`（5位），加上前两个字节的16位，共21位

聚合结果：`192.168.8.0/21`

> ⚠️ 注意：实际聚合时要找最长共同前缀，不是简单地取平均

### 4.4 NAT（网络地址转换）

将私有IP地址转换为公有IP地址。

| 私有地址范围 | 类别 |
|-------------|------|
| 10.0.0.0 ~ 10.255.255.255 | A类 |
| 172.16.0.0 ~ 172.31.255.255 | B类 |
| 192.168.0.0 ~ 192.168.255.255 | C类 |

**NAPT**：使用IP+端口号进行转换，多个私有地址共用一个公有地址。

### 4.5 ARP协议 ⭐⭐⭐

**作用**：IP地址 → MAC地址

**工作过程**：
```
场景：主机A要发数据给同一子网的主机B

1. A检查ARP缓存中是否有B的MAC地址
2. 若没有，A广播ARP请求：
   "谁的IP是192.168.1.2？请告诉192.168.1.1"
3. B收到后单播ARP应答：
   "192.168.1.2的MAC是 XX-XX-XX-XX-XX-XX"
4. A收到应答，更新ARP缓存
```

**跨子网的ARP**：
```
A(子网1) → 路由器 → B(子网2)

A发给B时，目标MAC地址填的是路由器接口的MAC地址（不是B的MAC）
路由器收到后，用ARP获取B的MAC，再转发
```

> 🔑 **关键结论**：IP数据报中的源/目的IP始终不变，但MAC地址在每一跳都会变化！

### 4.6 DHCP

**动态主机配置协议**，基于UDP，客户端/服务器模式。

```
DHCP过程（4步）：
1. 客户端广播：DHCP Discover（发现）
2. 服务器单播/广播：DHCP Offer（提供）
3. 客户端广播：DHCP Request（请求）
4. 服务器单播：DHCP ACK（确认）
```

### 4.7 ICMP协议

**网际控制报文协议**，用于报告差错和网络探测。

| 类型 | 报文 | 说明 |
|------|------|------|
| 差错报告 | 终点不可达 | 路由器无法转发 |
| | 源点抑制 | 拥塞丢弃 |
| | 时间超过 | TTL=0 |
| | 参数问题 | 首部有误 |
| | 改变路由（重定向） | 建议走更优路径 |
| 询问 | 回送请求/回答 | ping命令 |
| | 时间戳请求/回答 | 获取时间 |

> ⚠️ ICMP报文封装在**IP数据报**的数据部分（不是独立的传输层协议）

### 4.8 IPv6

| 特性 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32位（4字节） | **128位（16字节）** |
| 表示 | 点分十进制 | 冒号十六进制 |
| 首部长度 | 可变（20~60字节） | **固定40字节** |
| 分片 | 路由器和主机都可分片 | **仅源端分片** |
| 校验和 | 有 | **取消**（交给上层） |
| ARP | 有 | 用**NDP**替代 |

**IPv6地址缩写**：
```
完整：2001:0DB8:0000:0000:0000:0000:0000:0001
缩写：2001:DB8::1  （连续的0用::代替，只能用一次）
```

### 4.9 路由算法 ⭐⭐⭐

#### 4.9.1 距离向量算法（RIP）

| 特性 | 说明 |
|------|------|
| 算法 | Bellman-Ford |
| 度量 | **跳数**（最大15跳，16=不可达） |
| 更新方式 | 周期性广播（每30秒）整个路由表 |
| 收敛速度 | **慢**（好消息传播快，坏消息传播慢） |
| 问题 | **无穷计数问题**（count-to-infinity） |
| 协议 | RIP（基于UDP，端口522） |

**RIP更新规则**：
$$d(x,y) = \min\{d(x,v) + d(v,y)\} \quad \text{（对所有邻居v）}$$

#### 4.9.2 链路状态算法（OSPF）

| 特性 | 说明 |
|------|------|
| 算法 | **Dijkstra** |
| 度量 | 带宽、延迟等（综合度量） |
| 更新方式 | **洪泛法**发送链路状态更新（LSA） |
| 收敛速度 | **快** |
| 区域划分 | 支持分层（Area概念） |
| 协议 | OSPF（直接用IP，协议号89） |

**OSPF工作过程**：
1. 每个路由器向所有邻居发送**Hello**报文
2. 建立邻居关系后，交换**链路状态通告（LSA）**
3. 所有路由器拥有相同的**链路状态数据库（LSDB）**
4. 每个路由器运行Dijkstra算法，计算最短路径树
5. 将最优路由加入路由表

#### 4.9.3 路径向量算法（BGP）

| 特性 | 说明 |
|------|------|
| 应用场景 | **自治系统间**（AS间路由） |
| 度量 | 路径向量（记录经过的AS序列） |
| 更新方式 | 仅在变化时发送增量更新 |
| 连接类型 | eBGP（AS间）、iBGP（AS内） |
| 协议 | BGP（基于TCP，端口179） |

#### 三种路由协议对比 ⭐⭐

| 比较项 | RIP | OSPF | BGP |
|--------|-----|------|-----|
| 类型 | 内部网关（IGP） | 内部网关（IGP） | 外部网关（EGP） |
| 算法 | 距离向量 | 链路状态 | 路径向量 |
| 度量 | 跳数 | 代价（带宽等） | 策略+路径 |
| 传输 | UDP | IP | **TCP** |
| 更新 | 周期性广播 | 事件触发洪泛 | 事件触发 |
| 收敛 | 慢 | **快** | 较快 |
| 规模 | 小型网络 | 大型网络 | **互联网** |
| 环路防止 | 水平分割、毒性逆转 | 无环（Dijkstra） | AS路径检测 |

### 4.10 路由器结构

```
路由器 = 输入端口 + 交换结构 + 输出端口 + 路由处理器

输入端口：物理层处理 → 数据链路层处理 → 查表转发
交换结构：内存交换 / 总线交换 / 交叉开关
输出端口：排队 → 调度 → 数据链路层处理 → 物理层处理
路由处理器：运行路由协议、维护路由表
```

**三种交换结构**：

| 类型 | 速度 | 说明 |
|------|------|------|
| 内存交换 | 慢 | 最早方式，通过CPU转发 |
| 总线交换 | 中等 | 共享总线，同一时刻只有一对端口通信 |
| **交叉开关** | **最快** | 多对端口可同时通信 |

## 深入理解

### 为什么需要子网划分？

A类地址有1600多万个主机地址，实际使用中浪费严重。子网划分允许将一个大网络划分为多个小网络，提高地址利用率。

### RIP的无穷计数问题

```
A -- 1 -- B -- 1 -- C

初始：A到C的距离 = 2（通过B）

若C失效：
第1轮：B发现C不可达，设d(B,C)=16
       A不知道，告诉B："我到C的距离是2"
       B误认为可以通过A到达C，设d(B,C)=3
第2轮：A从B知道d(B,C)=3，设d(A,C)=4
...如此循环直到距离增长到16
```

**解决方法**：水平分割、毒性逆转、触发更新

### OSPF为什么比RIP收敛快？

- RIP只与邻居交换信息，需要**逐步传播**到全网
- OSPF用**洪泛法**，所有路由器同时收到链路状态变化，**同步更新**

## 典型例题

### 例题1：子网划分

**题目**：某公司申请了一个C类地址200.100.10.0，需要划分5个子网，每个子网至少20台主机。求子网掩码、各子网地址。

**解答**：
- 需要5个子网 → 至少需要3位子网号（$2^3 = 8 \geq 5$）
- 每个子网20台主机 → 至少需要5位主机号（$2^5 - 2 = 30 \geq 20$）
- 总共需要3+5=8位，C类有8位主机号，刚好够
- 子网掩码：255.255.255.**224**（11100000 = 224）

| 子网 | 网络地址 | 可用IP范围 | 广播地址 |
|------|----------|-----------|----------|
| 子网1 | 200.100.10.0/27 | .1 ~ .30 | .31 |
| 子网2 | 200.100.10.32/27 | .33 ~ .62 | .63 |
| 子网3 | 200.100.10.64/27 | .65 ~ .94 | .95 |
| 子网4 | 200.100.10.96/27 | .97 ~ .126 | .127 |
| 子网5 | 200.100.10.128/27 | .129 ~ .158 | .159 |
| 子网6~8 | 备用 | — | — |

### 例题2：CIDR路由聚合

**题目**：路由器R的路由表中有以下表项，求可聚合的最大CIDR块：
```
192.168.16.0/24
192.168.17.0/24
192.168.18.0/24
192.168.19.0/24
```

**解答**：
```
16 = 00010000
17 = 00010001
18 = 00010010
19 = 00010011
共同前缀：000100（6位）+ 前两字节16位 = 22位
聚合结果：192.168.16.0/22
```

### 例题3：ARP协议分析

**题目**：主机A(192.168.1.1/24)向主机B(192.168.1.2/24)发送数据，描述ARP过程。

**解答**：
1. A和B在同一子网（前24位相同）
2. A查ARP缓存，无B的MAC地址
3. A广播ARP请求（目的MAC=FF-FF-FF-FF-FF-FF）
4. B收到后，单播ARP应答给A
5. A收到应答，将B的IP-MAC映射存入ARP缓存
6. A用B的MAC地址封装帧，发送数据

### 例题4：路由算法选择

**题目**：某互联网服务提供商（ISP）需要在不同自治系统间交换路由信息，应使用（）协议，同一AS内部应使用（）或（）协议。

**解答**：
- AS间：**BGP**（外部网关协议）
- AS内：**OSPF**（链路状态）或 **RIP**（距离向量）

## 易错点

1. ⚠️ 子网划分时，全0和全1的子网是否可用取决于具体实现
2. ⚠️ ARP请求是**广播**，ARP应答是**单播**
3. ⚠️ IP地址在传输过程中**不变**，MAC地址**每跳都变**
4. ⚠️ RIP最大跳数为15，16表示不可达
5. ⚠️ OSPF直接用IP封装（协议号89），RIP用UDP封装
6. ⚠️ BGP用**TCP**连接（端口179），这是唯一使用TCP的路由协议
7. ⚠️ ICMP报文封装在IP数据报中，不是传输层协议
8. ⚠️ CIDR中/28表示前28位是网络号，主机号有4位，可用主机数=$2^4-2=14$

### 网络层功能的深入理解

网络层为传输层提供服务，将传输层的数据封装成"**IP数据报**"。网络中的路由器根据IP数据包首部中的**源IP地址**和**目的IP地址**进行"分组转发"。因此网络层实现了"**主机到主机**"的传输。

**异构网络互联**：
- **异构**：每个网络的拓扑结构不同、物理层和数据链路层的实现不同、主机类型不同
- **中继系统**：
  - 物理层中继系统：转发器、集线器
  - 数据链路层中继系统：网桥、交换机
  - 网络层中继系统：路由器
  - 网络层以上的中继系统：网关
- **路由器**也称为**网关**（网络适配器称为网卡）

**路由与转发**：
- **路由选择**：各个路由器之间相互配合，规划IP数据报（分组）的最佳转发路径，每个路由器需要运转"路由协议"，最终生成各自的"路由表"
- **分组转发**：一台路由器，根据自己的转发表，将收到的IP数据报从合适的接口转发出去（转发表 = 精简版路由表，更精简的数据结构有助于快速检索）

**重要结论**：
- 接入网络的每台主机至少拥有一个IP地址
- 通常，路由器的每个接口都需要分配一个IP地址（最新技术标准已取消）
- **路由器与主机之间必须有IP地址，路由器与路由器之间可以没有IP地址**
- 一个路由器有多个转发接口，每个接口背后都是一个网络适配器，因此**路由器有多个MAC地址**

**拥塞控制**：
- **拥塞原因**：网络上产生了过量分组，超负荷，引起网络性能下降
- **拥塞现象**：网络上的分组数增加，但吞吐量减少
- **开环控制**（静态方法）：在部署网络时，就提前设计好预防拥塞的方法，一旦网络系统开始运行，就不再修改
- **闭环控制**（动态方法）：动态监视网络状态，及时发现哪里发生阻塞，并将阻塞信息传递给相关路由器，相关路由器及时调整"路由表"

### IPv4分组的补充

**IP数据报首部重要字段**：

| 字段 | 说明 |
|------|------|
| **协议** | 区分网络层使用的IP协议版本（v4、v6） |
| **首部长度** | 以×4B为单位 |
| **总长度** | 包含首部、数据部分，以×1B为单位 |
| **标识** | 由源主机生成，通常是自增序列 |
| **标志** | MF=1表示后面还有分片，MF=0表示这是最后一个分片；DF=1表示不允许被分片，DF=0表示允许被分片 |
| **片偏移** | 表示数据部分在"被分片前的位置"，以×8B为单位 |
| **生存时间TTL** | 数据报在网络中通过的路由器的最大值，每经过一个路由器就减1，TTL=0时直接丢弃并向源主机发送ICMP报文 |
| **协议** | TCP=6，UDP=17 |
| **首部校验和** | 每个路由器仅校验首部，不校验数据部分 |

**IP数据报分片的重要结论**：
- 每个分片都是一个可以被单独转发的IP数据报，都包含首部
- IP数据报的"**分片**"可能在**源主机、或任何一个路由器**中发生
- 只有**目的主机**才会对分片进行"**重组**"
- 各分片有可能乱序到达目的主机
- 由于首部的"片偏移"字段是以×8B为单位，**除了最后一个分片外，其他每个分片的数据部分必须是8B的整数倍**

### IPv4地址的补充

**特殊IP地址**：
- 全1全1只在本网络广播（**各路由器均不转发，直接丢弃**）
- 如果一个网络中，主机号占N bit，**那么这个网络中，最多支持2ᴺ-2台主机&路由器**

**重要结论**：
- 在那个年代，要求**每台主机、每个路由器接口被分配的IP地址都是全球唯一的**
- **路由器和路由器连接的接口可以不分配IP地址，但路由器和其他节点连接的接口必须分配IP地址**
- 从属于**同一个网络**的所有主机、路由器接口的IP地址**"网络号"都相同**
- 当一台**新主机接入网络**时，需要给它**分配一个IP地址**、并配置**"默认网关"**
- **在一个主机上允许有两个IP地址，但这两个IP地址属于两个网络**

### 子网划分与子网掩码的补充

**子网划分原理**：
- 假设原本主机号占n bit，那么可以将前k bit作为子网号，用剩余的n-k bit作为主机号
- 这样就能划分出2ᴷ个子网（每个子网包含的IP地址块大小相等）
- 子网划分前，IP地址为两级结构 = <网络号，主机号>
- 子网划分后，IP地址为三级结构 = <网络号，子网号，主机号>
- 每个子网地址中，主机号不能分配为全0/全1（全0表示子网本身，全1为子网广播地址）

**子网掩码的作用**：
- 用子网掩码、IP地址"逐位与"，算出<网络号, 子网号>（可合称为"**网络前缀**"）
- 只有网络前缀相同的IP地址，才归属于同一个网络（或子网）

**重要结论**：
- 若一个网络内部进行子网划分，则这个网络中的**每台主机、每个路由器接口都需要配置IP地址、默认网关、子网掩码**
- 如果一台路由器支持子网划分技术，那么在它的转发表中，需要包含<目的网络号，子网掩码，转发接口>

**主机转发IP数据报过程**：
1. **判断目的主机和本机是否属于同一个网络**：检查本机IP地址和目的IP地址的网络前缀是否相同（需要用本机配置的子网掩码"逐位与"）
2. **将IP数据报封装成MAC帧并发送到链路上**：
   - 如果目的主机与本机属于同一个网络：就通过ARP协议找到目的主机的MAC地址，再将IP数据报封装成帧，并将帧发送给目的主机
   - 如果目的主机与本机不属于同一个网络：就通过ARP协议找到默认网关的MAC地址，再将IP数据报封装成帧，并将帧发送给默认网关

**路由器转发IP数据报过程**：
1. 路由器的某个接口收到一个IP数据报
2. 对IP数据报首部进行校验，并从中找到**目的IP地址**
3. **查"转发表"**：转发表的表项包含<目的网络号，子网掩码，转发接口>；检查目的IP地址与每个表项能否匹配（将目的IP地址、子网掩码"逐位与"，匹配表项中的目的网络号）；注：至少"默认路由"表项一定是可以匹配成功的
4. **转发**：根据查转发表的结果，将IP数据报从匹配的接口转发出去；注：如果匹配的"转发接口"和该IP数据报的入口相同，就不用再把IP数据报转发回去，直接丢弃

**默认路由**：目的网络号全0，子网掩码全0（可以使得目的IP与子网掩码作与运算后可以与目的网络号相匹配）。在路由器转发表中，如果所有表项都不匹配，那么将从"默认路由"转发出去。

### CIDR的补充

**CIDR中的子网划分**：

**1. 定长子网划分**：与传统的子网划分技术同理，在一个CIDR地址块中，把主机号前k bit抠出来作为定长子网号，这样就能划分出2ᴷ个子网（每个子网包含的IP地址块大小相等）。**最多划分多少子网，留出2位**。缺点：每个子网都一样大，不够灵活，IP地址利用率低。

**2. 变长子网划分**：在一个CIDR地址块中，划分子网时，**子网号长度不固定**（每个子网包含的IP地址块大小不同），更灵活。

**CIDR地址块的划分技巧**：可以利用类似于"从根到叶构造二叉哈夫曼树"
- 原始CIDR地址块作为根节点（假设可以自由分配的主机号占h bit）
- **每个分支节点必须同时拥有左右孩子**，左0，右1
- **每个叶子节点对应一个子网**，根据根节点到达叶子节点的路径来分析子网对应的IP地址块范围
- 整棵树的**高度不超过h-1**（因为即使最小的子网也至少要保留2 bit主机号）
- 每个可分配的地址个数 = N-**2**

### 路由聚合的补充

**路由聚合/构成超网**：对于一个路由转发表，若几条路由表项的**转发接口相同**，**部分网络前缀也相同**，则可以将这几条路由表项聚合为一条。

**优点**：可以减少路由表的大小，加快查询速度，减少转发时延
**缺点**：可能会引入额外的无效地址（冗余地址）

**最长前缀匹配原则**：当多个表项匹配时，选择前缀最长的表项进行转发。

### NAT的补充

**私有IP地址（内网IP）**：
- 10.0.0.0 ~ 10.255.255.255
- 172.16.0.0 ~ 172.31.255.255
- 192.168.0.0 ~ 192.168.255.255
- 只允许分配给局域网内部的节点，不允许分配给互联网上的节点
- 每个局域网内部都可以自行分配这些私有地址
- **私有地址是可复用的，只要求局域网内唯一，不要求全球唯一**

**全球IP地址（外网IP）**：
- 通常由ISP提供，全球唯一
- 外网IP是一个局域网与外界通信时所需使用的IP地址

**NAT路由器**：NAT表、转发表
- NAT表中记录了外网和内网的映射关系【内网IP:端口号 <——> 外网IP:端口号】
- **NAT路由器包含传输层的功能**（因为端口号是传输层的概念）
- 一个IP数据报从内网转发到外网，会更改源IP地址、源端口号
- 从外网转发到内网，会更改目的IP地址、目的端口号
- **在NAT路由器要转换对IP地址和端口号进行转换**

### ARP协议的补充

**ARP表**：（存储在ARP缓存中）
- 记录了<IP地址、MAC地址>之间的映射关系
- 一个数据结构（**每台主机、每台路由器都有自己的ARP表**）
- 需要**定期更新**ARP表项

**ARP工作过程**：
1. **ARP请求分组**：内容包括我的IP地址、我的MAC地址、我想找的IP地址；封装进MAC帧（帧目的地址=全1，源地址=Y）【广播帧】
2. **ARP响应分组**：内容包括我的IP地址、我的MAC地址（响应的同时也会更改ARP表）；封装进入MAC帧（帧目的地址=Y，源地址=V）【单播帧】
- **源ARP、目的ARP请求收到响应后会修改ARP表**
- **路由器收到请求也会修改ARP表，然后再转发**
- **ARP存放的是本局域网上的各主机和路由器的IP地址和MAC地址**

### DHCP的补充

**DHCP协议作用**：给刚接入网络的主机动态分配IP地址、配置默认网关、子网掩码

**DHCP使用客户/服务器模型（C/S）**：
- **DHCP客户**：就是新接入网络的主机（希望获得IP地址等配置）
- **DHCP服务器**：就是负责分配IP地址的那台主机，管理一系列IP地址池
- 在家庭网络中，通常由家庭路由器兼职"DHCP服务器"
- 在一个大型网络内可以有多台DHCP服务器
- DHCP是应用层协议，基于UDP。**客户UDP端口号=68、服务器UDP端口号=67**

### IPv6的补充

**IPv4和IPv6的主要区别**：
1. IPv6将地址从**32位（4B）**扩大到**128位（16B）**，更大的地址空间
2. IPv6将IPv4的**校验和字段彻底移除**，以减少每跳的处理时间
3. IPv6将IPv4的可选字段移出首部，变成了**扩展首部**，成为灵活的首部格式（路由器通常不对扩展首部进行检查，大大提高了路由器的处理效率）
4. IPv6支持**即插即用**（即自动配置），**不需要DHCP协议**
5. IPv6首部长度必须是**8B的整数倍**，IPv4首部是**4B的整数倍**
6. **IPv6只能在主机处分片**，IPv4可以在路由器和主机处分片
7. ICMPv6附加报文类型"分组过大"支持实时视像等要求，保证一定的带宽和时延的应用
8. IPv6取消了协议字段，改成下一个首部字段
9. IPv6取消了总长度字段，改用有效载荷长度字段
10. IPv6取消了服务类型字段

**IPv6地址表示形式**：
- **一般形式**：冒号十六进制记法，如4BF5:AA12:0216:FEBC:BA5F:039A:BE9A:2170
- **压缩形式**：每个区域开头0可省略、结尾0不可省略
- **零压缩**：一连串连续的0可以被一对冒号取代，如FF05:0:0:0:0:0:0:B3 → FF05::B3（**双冒号表示法在一个地址中仅可出现一次**）

**IPv6基本地址类型**：
- **单播**：一对一通信，可做源地址+目的地址
- **多播**：一对多通信，可做目的地址
- **任播**：一对多中的一个通信，可做目的地址

**IPv6向IPv4的过渡策略**：
- **双栈协议**：在一台设备上**同时启用IPv4协议栈和IPv6协议栈**
- **隧道技术**：使用隧道传递的数据/负载可以是不同协议的数据帧/包，隧道协议将其它协议的数据帧/包**重新封装**然后通过隧道发送

### 路由算法与路由协议的补充

**自治系统AS**：互联网采用分层次的路由选择协议，把整个互联网划分成为许多较小的自治系统

**路由协议分类**：
- **内部网关协议IGP**（在一个自治系统内部使用的路由选择协议）：RIP、OSPF
- **外部网关协议EGP**（在不同自治系统之间使用的路由选择协议）：BGP

**协议封装**：
- **RIP——UDP、应用层**
- **OSPF——IP数据报、网络层**
- **BGP——TCP、应用层**

### RIP的补充

**核心原理**：Bellman-Ford算法（**距离向量算法**），类似于图的广度优先

**工作过程**：
- **Who**：每个路由器仅告诉邻居"距离向量"
- **How**：告诉邻居们，从本路由器到达每一个目的网络的距离
- **When**：自治系统刚启动时，定期交换；收敛后：定期交换+距离向量有更新就交换

**特点**：
1. RIP仅和相邻路由器交换信息
2. 路由器交换的信息是当前本路由器所知道的全部信息，即自己现在的路由表
3. 按固定时间间隔交换路由信息，然后路由器根据收到的路由信息更新路由表
4. RIP只适用于小型互联网

**重要结论**：
- 路由器在刚刚开始工作时，它的路由表是空的，然后路由器就得出到直接相连的几个网络的距离（这些距离定义为1）
- 经过若干次的更新后，所有的路由器最终都会知道到达本自治系统中任何一个网络的最短距离和下一跳路由器的地址（**先修改自身，再发出通告**）
- **每个路由器除了自身的路由表以外，还有邻居的路由向量**
- 当3分钟还没有收到相邻路由器的更新数据表，则把此相邻路由器记为不可达的路由器，即把距离设置为16
- **好消息传播的快，坏消息传播的慢**

### OSPF的补充

**核心原理**：基于数据结构的"图"，迪杰斯特拉算法（**链路状态路由算法**），每台路由器都要构造整个自治系统的网络拓扑结构

**工作过程**：
- **Who**：每个路由器告诉自治系统内的所有路由器"本路由器相邻的所有路由器的链路状态"
- **How**：洪泛法，通知本自治系统的所有路由器
- **When**：新结点刚加入时，找邻居抄作业；收敛后：定期交换+相邻节点、路径信息有变化时

**洪泛法**：路由器通过所有输出端口向所有相邻的路由器发送信息，而每一个相邻路由器又再将此信息发往其所有的相邻路由器（但不再发送给刚刚发来信息的那个路由器）

**链路状态**：说明本路由器都和哪些路由器相邻，以及该链路的"度量"（度量用来表示费用、距离、时延、带宽）

**重要结论**：
- 由于各路由器之间频繁地交换链路状态信息，因此所有的路由器最终都能建立一个链路状态数据库，这个数据库实际上就是全网的拓扑结构图
- 协议RIP的每一个路由器虽然知道到所有的网络的距离以及下一跳路由器，但却不知道全网的拓扑结构
- **OSPF将一个自治系统再划分为若干个更小的范围（区域）**，好处是把利用洪泛法交换链路状态信息的范围局限于每一个区域而不是整个的自治系统，减少了整个网络上的通信量
- **边的权值由系统自己设置，与TTL没有关系**
- **OSPF更新收敛的很快**

### BGP的补充

**边界网关协议**：
- 在一个AS中有两个不同功能的路由器，**边界路由器**和**内部路由器**
- 在边界路由器与边界路由器之间建立的是**eBGP连接**
- 在内部路由器与内部路由器之间建立的是**iBGP连接**
- BGP采用**路径向量路由选择协议**
- BGP的AS路径选择：**优先AS最少**，在AS相同的情况下选最快出本AS的路径
- **BGP交换的网络可达性信息：到达某个网络所经过的路径**

### 网络层设备的补充

**路由器的组成和功能**：
- 路由器是一种具有多个输入/输出端口的专用计算机，其任务是连接不同的网络（连接异构网络）并完成分组转发
- 在多个逻辑网络（即多个广播域）互连时必须使用路由器
- 从结构上看，路由器由**路由选择**（网络层）和**分组转发**（物理层+数据链路层）两部分构成
- **路由选择处理机用来计算路由信息**

**直接交付与间接交付**：
- **直接交付**：在同一个网络中，可以不通过路由器
- **间接交付**：不在同一个网络中，需要经过路由器

**设备互联能力**：
- **路由器**：可以互联两个不同网络层协议的网段
- **网桥**：可以互联两个物理层和链路层不同的网段
- **集线器**：不能互联两个物理层不同的网段
