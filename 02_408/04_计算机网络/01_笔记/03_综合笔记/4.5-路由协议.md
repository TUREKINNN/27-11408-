# 4.5 路由协议

> 王道考研·计算机网络 第四章 网络层

---

## 一、路由算法

### 1.1 距离向量算法（DV）

每个路由器维护到所有目的网络的**距离向量**（距离+下一跳），定期与邻居交换。

**Bellman-Ford 方程**：

$$
D_x(y) = \min_v \{c(x,v) + D_v(y)\}
$$

其中 $c(x,v)$ 是 $x$ 到邻居 $v$ 的代价，$D_v(y)$ 是 $v$ 到 $y$ 的最短距离。

**特点**：
- ✅ 简单，开销小
- ❌ 收敛慢，存在"好消息传播快，坏消息传播慢"的问题
- ❌ 可能产生**路由环路**

### 1.2 链路状态算法（LS）

每个路由器获取全网拓扑，用 **Dijkstra 算法**计算最短路径。

**步骤**：
1. 发现邻居，测量链路代价
2. 构造链路状态分组（LS 分组）
3. 向全网洪泛 LS 分组
4. 每个路由器获得完整拓扑，运行 Dijkstra 算法

**特点**：
- ✅ 收敛快，无环路
- ❌ 开销大，需要更多存储和计算

## 二、路由协议分类

```
路由协议
├── 内部网关协议 IGP（AS 内部）
│   ├── RIP（基于距离向量）
│   └── OSPF（基于链路状态）
└── 外部网关协议 EGP（AS 之间）
    └── BGP
```

---

## 三、RIP 协议 ⭐⭐

### 3.1 基本特点

- 基于**距离向量算法**
- 使用**跳数**作为度量（最大 15 跳，16 表示不可达）
- 每 **30 秒**与邻居交换完整路由表
- 适用于小型网络

### 3.2 RIP 报文

- 使用 **UDP**，端口 520
- 请求报文：请求邻居的路由信息
- 响应报文：通告自己的路由表

### 3.3 RIP 的问题

**慢收敛 / 无穷计数问题**：

```
A ──1── B ──1── C
        │
        D (到C的链路断开)

问题：B 从 A 学到去 C 的路径（经过 B 自己），
形成路由环路：B→A→B→A→...
```

**解决方法**：
- **水平分割**：不把从某邻居学到的路由再通告给该邻居
- **毒性逆转**：将从某邻居学到的路由设为无穷大（16）再通告回去
- **触发更新**：链路变化时立即通知，不等 30 秒

---

## 四、OSPF 协议 ⭐⭐⭐

### 4.1 基本特点

- 基于**链路状态算法**
- 使用 **Dijkstra** 最短路径算法
- 使用**代价（Cost）**作为度量（可基于带宽）
- 每当链路状态变化时才发送更新
- 支持**分区（Area）**，减少开销

### 4.2 OSPF 分组类型

| 类型 | 名称 | 说明 |
|------|------|------|
| 1 | Hello | 发现和维护邻居 |
| 2 | Database Description | 描述链路状态数据库摘要 |
| 3 | Link State Request | 请求特定链路状态信息 |
| 4 | Link State Update | 发送链路状态更新 |
| 5 | Link State Ack | 确认收到更新 |

### 4.3 OSPF vs RIP

| 对比项 | RIP | OSPF |
|--------|-----|------|
| 算法 | 距离向量 | 链路状态 |
| 度量 | 跳数（最大15） | 代价（基于带宽） |
| 更新方式 | 定期（30s） | 触发更新 |
| 收敛速度 | 慢 | 快 |
| 规模 | 小型网络 | 大型网络 |
| 传输层 | UDP | IP（直接封装） |

---

## 五、BGP 协议 ⭐⭐

### 5.1 基本概念

- **边界网关协议**，用于**自治系统（AS）之间**的路由选择
- 基于**路径向量算法**
- 使用 **TCP**，端口 179

### 5.2 BGP 两种类型

| 类型 | 说明 |
|------|------|
| **eBGP** | 不同 AS 之间交换路由 |
| **iBGP** | 同一 AS 内部传递外部路由 |

### 5.3 BGP 报文

| 报文 | 功能 |
|------|------|
| OPEN | 建立 BGP 连接 |
| UPDATE | 通告新路由或撤销旧路由 |
| KEEPALIVE | 保持连接（心跳） |
| NOTIFICATION | 报告错误 |

### 5.4 BGP 路由选择

BGP 可能学到到达同一目的网络的多条路径，选择策略（按优先级）：
1. 本地偏好（Local Preference）
2. AS 路径最短
3. 最近的下一跳路由器
4. ...

## 六、三种协议对比

| 对比项 | RIP | OSPF | BGP |
|--------|-----|------|-----|
| 类型 | IGP | IGP | EGP |
| 算法 | DV | LS | 路径向量 |
| 传输层 | UDP | IP | TCP |
| 度量 | 跳数 | 代价 | 策略 |
| 适用规模 | 小 | 中/大 | AS间 |

---

## 七、考点总结

| 考点 | 频率 | 重点 |
|------|------|------|
| RIP 协议 | ⭐⭐ | 跳数限制、水平分割 |
| OSPF 协议 | ⭐⭐⭐ | Dijkstra 算法、分组类型 |
| BGP 协议 | ⭐⭐ | AS 间路由、TCP 连接 |
| DV vs LS 算法 | ⭐⭐ | 收敛性、环路问题 |

---

> **Q**：RIP 协议中，最大跳数是多少？超过此值表示什么？
>
> **A**：最大跳数为 **15**，超过 15（即 16）表示**不可达**。这也是 RIP 适用于小型网络的原因。
