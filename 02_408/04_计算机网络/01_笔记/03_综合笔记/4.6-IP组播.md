# 4.6 IP 组播

> IP 组播（Multicast）实现**一对多**通信，将数据从一个源发送到多个特定的接收方，比单播更高效。

---

## 一、组播的基本概念

### 1.1 三种通信方式对比

| 方式 | 发送方 → 接收方 | 效率 |
|------|----------------|------|
| **单播** | 一对一 | 发送方需发送 N 份数据 |
| **广播** | 一对所有 | 所有主机都收到 |
| **组播** | 一对多（特定组） | 只有组内成员收到 |

### 1.2 组播的优势

- 发送方只需发送**一份**数据
- 网络中的路由器负责复制和转发
- 只有加入组播组的主机才接收数据

---

## 二、IP 组播地址

### 2.1 IPv4 组播地址范围

- D 类地址用于组播：**224.0.0.0 \~ 239.255.255.255**
- 首 4 位固定为 **1110**

### 2.2 常用组播地址

| 地址 | 用途 |
|------|------|
| 224.0.0.1 | 所有主机 |
| 224.0.0.2 | 所有路由器 |
| 224.0.0.5 | OSPF 所有路由器 |
| 224.0.0.6 | OSPF 指定路由器 |

### 2.3 组播 MAC 地址

组播 IP 地址映射到 MAC 地址：
- MAC 地址前 25 位固定为 `01-00-5E`
- 第 25 位为 0
- 后 23 位来自 IP 地址的低 23 位

---

## 三、IGMP（Internet Group Management Protocol）

### 3.1 功能

IGMP 用于管理主机加入/离开组播组：

| 操作 | 说明 |
|------|------|
| **加入** | 主机发送 IGMP 报文，告知路由器自己要加入某组播组 |
| **离开** | 主机发送 IGMP 报文，告知路由器自己要离开某组播组 |
| **查询** | 路由器周期性查询本网段是否有组播组成员 |

### 3.2 IGMP 的位置

- IGMP 工作在主机与**直接相连的路由器**之间
- IGMP 不负责组播数据在路由器之间的转发

---

## 四、组播路由协议

### 4.1 组播路由的目标

建立从源到所有组播组成员的**组播转发树**。

### 4.2 常见组播路由协议

| 协议 | 特点 |
|------|------|
| **DVMRP** | 基于距离-向量 |
| **MOSPF** | 基于 OSPF，链路状态 |
| **PIM-SM** | 稀疏模式，适合大规模网络 |
| **PIM-DM** | 密集模式，适合小规模网络 |

---

## 考点总结

> 1. 组播实现一对多通信，比单播高效
> 2. IPv4 组播地址范围：224.0.0.0 \~ 239.255.255.255（D 类）
> 3. 组播 MAC 地址前缀：01-00-5E
> 4. IGMP 管理主机加入/离开组播组，工作在主机与直接路由器之间
> 5. 组播路由协议负责路由器之间的组播转发树构建
