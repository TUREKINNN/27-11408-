# 3.5 介质访问控制

> 王道考研·计算机网络 第三章 数据链路层

---

## 一、基本概念

**介质访问控制（Medium Access Control, MAC）**：多个节点共享同一个广播信道时，可能发生信号冲突。需要控制各节点对传输介质的访问。

```
介质访问控制
├── 信道划分（静态分配）
│   ├── 频分复用 FDM
│   ├── 时分复用 TDM
│   ├── 波分复用 WDM
│   └── 码分复用 CDM
├── 随机访问（动态分配）
│   ├── ALOHA
│   ├── CSMA
│   ├── CSMA/CD
│   └── CSMA/CA
└── 轮询访问（动态分配）
    └── 令牌传递协议
```

---

## 二、信道划分介质访问控制

### 2.1 频分复用（FDM）

将信道的总频带划分为多个子频带，分配给不同用户。

```
频率 ↑
     │  用户C
     │  ─────
     │  用户B
     │  ─────
     │  用户A
     │  ─────
     └──────────────► 时间
```

- ✅ 每个用户拥有固定的子频带，可同时发送数据
- ❌ 频带资源固定分配，空闲时浪费

### 2.2 时分复用（TDM）

将时间划分为等长的 TDM 帧，每个帧分为 $m$ 个时隙，分配给 $m$ 个用户。

```
时间 ──────────────────────────►
      │ A │ B │ C │ D │ A │ B │ C │ D │
      └───┴───┴───┴───┴───┴───┴───┴───┘
          TDM帧1              TDM帧2
```

- ❌ 每个用户最多只能获得 $\frac{1}{m}$ 的带宽
- ❌ 用户空闲时时隙浪费

**统计时分复用（STDM）**：动态按需分配时隙，提高利用率。

### 2.3 波分复用（WDM）

在光纤通信中使用，不同用户使用不同波长的光信号。

### 2.4 码分复用（CDM）/ 码分多址（CDMA）

每个用户分配一个唯一的 $m$ 位**码片序列（Chip Sequence）**。

**规则**：
- 发送 1 → 发送码片序列
- 发送 0 → 发送码片序列的反码
- 各用户的码片序列必须**正交**（内积为 0）

**内积计算**：

$$
S \cdot T = \frac{1}{m}\sum_{i=1}^{m} S_i \times T_i = 0
$$

**数据还原**：接收到的信号与某用户的码片序列做内积：
- 结果为 +1 → 收到的是 1
- 结果为 -1 → 收到的是 0
- 结果为 0 → 没有发送数据

> 设站点 A 的码片序列：`+1 -1 -1 +1 +1 +1 -1 +1`
> 设站点 B 的码片序列：`+1 -1 +1 -1 +1 +1 +1 -1`
>
> A 发送 1，B 发送 0 →
> A 发送码片序列，B 发送码片序列的反码
> 信道上的信号 = A的码片 + B码片的反码

## 三、随机访问介质访问控制

### 3.1 ALOHA 协议

**纯 ALOHA**：有数据就直接发送，冲突后随机等待重传。

**时隙 ALOHA**：只在时隙开始时发送，减少冲突概率。

### 3.2 CSMA 协议（载波监听多路访问）

**核心思想**：发送前先**监听**信道。

| 协议 | 监听策略 | 信道空闲 | 信道忙 |
|------|---------|---------|--------|
| **1-坚持 CSMA** | 持续监听 | 立即发送 | 继续监听直到空闲 |
| **非坚持 CSMA** | 不持续监听 | 立即发送 | 等待随机时间再监听 |
| **p-坚持 CSMA** | 持续监听 | 以概率 $p$ 发送 | 等到下一时隙再判断 |

### 3.3 CSMA/CD 协议（有线网络 ⭐⭐⭐）

**适用于**：以太网（有线广播信道）

**核心思想**：**边发送边监听**，检测到冲突立即停止发送。

#### 算法流程

```
1. 监听信道是否空闲
   ├─ 空闲 → 发送数据
   └─ 忙 → 持续监听直到空闲
2. 发送数据的同时检测冲突
   ├─ 无冲突 → 发送完毕
   └─ 检测到冲突 → 停止发送 → 发送干扰信号 → 随机等待后重新监听
```

#### 关键参数

**最小帧长**：确保在发送完帧之前能检测到冲突。

$$
\text{最小帧长} = 2 \times \text{传播时延} \times \text{数据传输速率}
$$

$$
L_{min} = 2 \tau \times R
$$

> 以太网最小帧长 = **64 字节**（512 bit），传播时延约 25.6 μs

**碰撞检测时间**：最多 $2\tau$（端到端往返传播时延）

**截断二进制指数退避算法**：
- 发生第 $i$ 次冲突后，从 $\{0, 1, ..., 2^{\min(i,10)}-1\}$ 中随机选择等待时间
- 等待时间 = 随机数 × $2\tau$
- 最大重试次数 = 16，超过则放弃

#### 信道利用率

$$
S = \frac{1}{1 + a(2e^{-1} - 1)}
$$

其中 $a = \frac{\tau}{T_0}$（传播时延与发送时延之比），$e$ 为自然常数。

### 3.4 CSMA/CA 协议（无线网络 ⭐⭐⭐）

**适用于**：IEEE 802.11（WiFi）

#### 为什么不能用 CSMA/CD？

无线网络存在**隐蔽站问题**：A 和 C 都在 B 的通信范围内，但 A 和 C 不在彼此的通信范围内，导致 A 和 C 同时向 B 发送数据时产生冲突，但 A 和 C 检测不到。

#### 核心机制

**CA = Collision Avoidance**（碰撞避免）

```
发送方A                           接收方B         隐蔽站C
  │── RTS ─────────────────────►│                │
  │◄────────── CTS ─────────────│                │
  │                                │◄──── CTS ────│（听到CTS，退让）
  │═══════ 数据 ═══════════════►│                │
  │◄────────── ACK ─────────────│                │
```

**关键步骤**：
1. **RTS（Request to Send）**：发送方请求发送
2. **CTS（Clear to Send）**：接收方允许发送
3. **数据传输**：发送数据帧
4. **ACK**：接收方确认

**NAV 机制**：其他站收到 RTS/CTS 后，设置**网络分配矢量**，在此期间不发送数据。

**退避算法**：冲突后等待随机时间再重发。

## 四、轮询访问：令牌传递协议

**令牌环网**：各站点通过一个逻辑环连接，只有持有**令牌**的站点才能发送数据。

```
    A ──── B
    │      │
    D ──── C
   (令牌: A→B→C→D→A)
```

- ✅ 无冲突，确定性访问
- ❌ 令牌开销，单点故障

---

## 五、CSMA/CD vs CSMA/CA 对比

| 对比项 | CSMA/CD | CSMA/CA |
|--------|---------|---------|
| 适用网络 | 有线以太网 | 无线 WiFi |
| 冲突处理 | **检测**冲突并停止发送 | **避免**冲突（RTS/CTS） |
| 监听方式 | 边发边听 | 发送前监听 |
| 隐蔽站问题 | 不存在 | 存在，用 RTS/CTS 解决 |
| 最小帧长 | 64 字节 | 无此要求 |

## 六、考点总结

| 考点 | 频率 | 重点 |
|------|------|------|
| CSMA/CD 协议 | ⭐⭐⭐ | 最小帧长、退避算法 |
| CSMA/CA 协议 | ⭐⭐⭐ | RTS/CTS、隐蔽站问题 |
| CDMA 计算 | ⭐⭐⭐ | 码片序列内积 |
| 信道划分方法 | ⭐⭐ | FDM、TDM、CDM 对比 |
| CSMA/CD vs CSMA/CA | ⭐⭐⭐ | 核心区别 |

---

> **Q**：以太网数据传输率为 100 Mbps，信号传播速率为 $2 \times 10^8$ m/s，CSMA/CD 的最小帧长为多少字节？若两站点间最大距离为 1000m，最小帧长为多少？
>
> **A**：
> $$\tau = \frac{1000}{2 \times 10^8} = 5 \times 10^{-6} \text{s} = 5\mu s$$
> $$L_{min} = 2\tau \times R = 2 \times 5 \times 10^{-6} \times 100 \times 10^6 = 1000 \text{bit} = 125 \text{字节}$$
> 但以太网标准规定最小帧长为 **64 字节**。
