# 6.2 DNS 系统

> 王道考研·计算机网络 第六章 应用层

---

## 一、DNS 概述

### 1.1 什么是 DNS

**DNS（Domain Name System，域名系统）**：将域名解析为 IP 地址的分布式数据库系统。

- 使用 **UDP**，端口号 **53**
- 采用**客户-服务器**模式

### 1.2 为什么需要 DNS

- IP 地址难以记忆（如 202.108.22.5）
- 域名便于记忆（如 www.baidu.com）
- DNS 自动完成域名 → IP 地址的转换

## 二、域名结构

### 2.1 层次化域名

```
              . (根域名)
             ╱ ╲
           .com  .cn   .org  .edu  .net  ...
          ╱        ╲
       baidu      tsinghua
        ╱            ╲
      www           www

完整域名: www.baidu.com.
         ↑ 注意末尾有个点（根域名）
```

### 2.2 域名层次

| 层次 | 说明 | 示例 |
|------|------|------|
| 根域名 | 最顶层（通常省略） | `.` |
| 顶级域名 | 国家/类别 | `.com` `.cn` `.edu` |
| 二级域名 | 组织/机构 | `baidu` `tsinghua` |
| 三级域名 | 主机/服务 | `www` `mail` |

### 2.3 域名服务器层次

```
根域名服务器（13个，编号A~M）
    │
    ├── 顶级域名服务器（.com, .cn, .org...）
    │       │
    │       ├── 权威域名服务器（baidu.com, tsinghua.edu.cn...）
    │       │
    │       └── ...
    │
    └── 本地域名服务器（ISP 提供）
```

## 三、DNS 解析过程 ⭐⭐⭐

### 3.1 递归查询 vs 迭代查询

| 类型 | 说明 |
|------|------|
| **递归查询** | 本地 DNS 服务器代替主机完成全部查询 |
| **迭代查询** | 本地 DNS 服务器逐级向各级域名服务器查询 |

### 3.3 DNS 缓存

为了提高效率，DNS 使用**缓存**机制：

- **浏览器缓存**：浏览器将域名→IP 缓存在本地
- **操作系统缓存**：操作系统 DNS 客户端缓存
- **本地 DNS 缓存**：本地域名服务器缓存查询结果
- **各级域名服务器缓存**：减少对根服务器的查询

> DNS 记录有 **TTL（Time To Live）**，过期后缓存失效，需要重新查询。

## 四、DNS 记录类型

| 类型 | 含义 | 示例 |
|------|------|------|
| **A** | 域名 → IPv4 地址 | `www.baidu.com → 202.108.22.5` |
| **AAAA** | 域名 → IPv6 地址 | — |
| **CNAME** | 域名别名 | `www.baidu.com → baidu.com` |
| **MX** | 邮件交换记录 | `baidu.com → mail.baidu.com` |
| **NS** | 域名服务器记录 | `baidu.com → ns1.baidu.com` |
| **PTR** | IP → 域名（反向解析） | — |

---

> **Q**：主机访问 www.example.com，本地 DNS 缓存中没有记录，请描述完整的 DNS 解析过程。
>
> **A**：
> 1. 主机向**本地域名服务器**发送查询请求（递归查询）
> 2. 本地 DNS 向**根域名服务器**查询（迭代查询）
> 3. 根 DNS 返回 `.com` 顶级域名服务器地址
> 4. 本地 DNS 向 `.com` 顶级域名服务器查询
> 5. 顶级 DNS 返回 `example.com` 权威域名服务器地址
> 6. 本地 DNS 向 `example.com` 权威域名服务器查询
> 7. 权威 DNS 返回 www.example.com 的 IP 地址
> 8. 本地 DNS 将结果返回给主机，并缓存结果
