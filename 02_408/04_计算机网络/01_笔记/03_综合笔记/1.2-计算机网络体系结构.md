# 1.2 计算机网络体系结构

> 王道考研·计算机网络 第一章 计算机网络体系结构

---

## 一、分层的设计思想

### 1.1 为什么要分层？

将庞大而复杂的问题，转化为若干较小的局部问题。

**快递网络类比**：
```
┌─────────────────────────────────────────────┐
│  第5层：顾客（货物打包）                       │
│  第4层：快递驿站（取件、派件）                  │
│  第3层：仓库部门（冷链存储、路由）              │
│  第2层：运输部门（包裹运输）                    │
│  第1层：公路/铁路等基础设施                     │
└─────────────────────────────────────────────┘
```

### 1.2 计算机网络要完成的功能

- **差错控制**：确保数据在传输过程中不出错
- **流量控制**：发送端的发送速率必须使接收端来得及接收
- **分段和重装**：发送端将数据划分为更小的单位，接收端还原
- **复用和分用**：多个高层会话复用一条低层连接
- **连接建立和释放**：交换数据前先建立逻辑连接

## 二、三种网络体系结构

### 2.1 OSI 参考模型（7层）

> 国际标准化组织 ISO 提出，"法律上的标准"

```
┌────────────────────────────┐
│  7  应用层    (Application)  │
│  6  表示层    (Presentation) │
│  5  会话层    (Session)      │
│  4  运输层    (Transport)    │
│  3  网络层    (Network)      │
│  2  数据链路层 (Data Link)    │
│  1  物理层    (Physical)     │
└────────────────────────────┘
```

**记忆口诀**：**物联网叔会使用**（物、联、网、叔（输）、会、使、用）

### 2.2 TCP/IP 模型（4层）

> 美国国防部 ARPANET 项目的成果，"事实上的标准"

```
┌────────────────────────────┐
│  4  应用层    (Application)  │
│  3  传输层    (Transport)    │
│  2  网际层    (Internet)     │  ← 也称 IP 层
│  1  网络接口层 (Network Interface) │
└────────────────────────────┘
```

### 2.3 五层模型（教学用）

综合前两者，便于学习和理解：

```
┌────────────────────────────┐
│  5  应用层                   │
│  4  传输层                   │
│  3  网络层                   │
│  2  数据链路层                │
│  1  物理层                   │
└────────────────────────────┘
```

### 2.4 三种模型对比

| OSI 参考模型 | TCP/IP 模型 | 五层模型 |
|:---:|:---:|:---:|
| 应用层 | — | — |
| 表示层 | 应用层 | 应用层 |
| 会话层 | — | — |
| 运输层 | 传输层 | 传输层 |
| 网络层 | 网际层 | 网络层 |
| 数据链路层 | — | 数据链路层 |
| 物理层 | 网络接口层 | 物理层 |

## 三、OSI 各层功能详解

### 3.1 物理层（第1层）

**任务**：实现相邻节点之间**比特（0或1）**的传输

- 定义电路接口参数（形状、尺寸、引脚数等）
- 定义传输信号的含义、电气特征（如 5V 表示 1，1V 表示 0）

```
结点A ──── 比特流(01010...) ──── 结点B
```

### 3.2 数据链路层（第2层）

**任务**：确保相邻节点之间的链路逻辑上无差错

- **差错控制**：检错+纠错，或检错+丢弃+重传
- **流量控制**：协调两个结点的速率
- 数据单位：**帧（Frame）**

```
┌────────┬────────────────┬────────┐
│ 帧首部  │    数据部分     │ 帧尾部  │
└────────┴────────────────┴────────┘
```

### 3.3 网络层（第3层）

**任务**：把"分组"从源结点转发到目的结点

- **路由选择**：构造并维护路由表，决定最佳路径
- **分组转发**：将分组从合适的端口转发出去
- **拥塞控制**：发现网络拥塞并采取措施
- 数据单位：**分组（Packet）** 或 **数据报（Datagram）**

### 3.4 传输层（第4层）

**任务**：实现**端到端**通信（进程到进程，"端"指端口）

- **复用**：多个进程共享同一网络连接
- **分用**：接收端将数据分发给正确的进程
- 数据单位：**报文段（Segment）**

```
进程A (端口:8899) ─────────── 进程B (端口:1763)
      │                            │
      └──── 传输层（端到端）─────────┘
```

### 3.5 会话层（第5层）

**任务**：管理进程间会话

- 会话管理（建立、维护、终止会话）
- 采用**检查点机制**，通信失效时从检查点恢复

### 3.6 表示层（第6层）

**任务**：解决不同主机上信息表示不一致的问题

- 数据格式转换（编码转换、压缩/解压、加密/解密）

### 3.7 应用层（第7层）

**任务**：实现特定的网络应用

- 功能最复杂且多样
- 包含各种应用协议（HTTP、FTP、SMTP 等）

## 四、TCP/IP 模型 vs OSI 模型

### 4.1 OSI 模型的缺陷

1. **专家先行**：先制定标准再实现，缺乏实践经验
2. **标准复杂**：协议实现复杂，运行效率低
3. **层间重复**：表示层和会话层功能较少，被合并到应用层

### 4.2 TCP/IP 模型的优势

1. **实践先行**：从 ARPANET 实际经验中总结
2. **简洁高效**：只有4层，去除不必要的层次
3. **灵活适应**：网络接口层不作具体规定，适应各种网络硬件

### 4.3 关键区别

| 对比项 | OSI 模型 | TCP/IP 模型 |
|--------|---------|-------------|
| 层次数 | 7层 | 4层 |
| 网络层 | 面向连接+无连接 | 仅无连接（IP） |
| 传输层 | 仅面向连接 | 面向连接(TCP)+无连接(UDP) |
| 通用性 | 先标准后实现 | 先实现后标准 |

## 五、核心概念

### 5.1 实体

在分层结构中，第 $n$ 层中的活动元素（软件+硬件）称为**第 $n$ 层实体**。

- 不同机器上的同一层称为**对等层**
- 同一层的实体称为**对等实体**

### 5.2 协议

**网络协议（Network Protocol）**：控制对等实体之间进行通信的规则的集合。

协议三要素：
- **语法**：数据与控制信息的结构或格式
- **语义**：需要发出何种控制信息，完成何种动作
- **同步**：事件实现顺序的详细说明

### 5.3 接口（服务访问点 SAP）

同一结点内相邻两层之间交换信息的连接点。

### 5.4 服务

下层为上层提供服务，上层使用下层的服务。

```
上层 ──请求──► 下层
     ◄──指示──
     ──响应──►
     ◄──证实──
```

## 六、常见网络设备的功能层次

```
┌─────────────────────────────────────┐
│  7 应用层                            │  ← 主机（实现全部7层）
│  6 表示层                            │
│  5 会话层                            │
│  4 传输层                            │
│  3 网络层 ─────────────────────────│  ← 路由器（实现1~3层）
│  2 数据链路层 ─────────────────────│  ← 交换机（实现1~2层）
│  1 物理层 ────────────────────────│  ← 集线器（实现1层）
└─────────────────────────────────────┘
```

| 设备 | 工作层次 | 功能 |
|------|---------|------|
| **集线器** | 物理层 | 比特转发 |
| **交换机** | 数据链路层 | 帧转发，基于 MAC 地址 |
| **路由器** | 网络层 | 分组转发，基于 IP 地址 |

## 七、考点总结

| 考点 | 频率 | 重点 |
|------|------|------|
| OSI 七层模型 | ⭐⭐⭐ | 各层功能、数据单位 |
| TCP/IP 四层模型 | ⭐⭐⭐ | 与 OSI 的区别 |
| 协议三要素 | ⭐⭐ | 语法、语义、同步 |
| 实体、协议、服务 | ⭐⭐ | 概念辨析 |
| 网络设备层次 | ⭐⭐⭐ | 集线器/交换机/路由器 |

---

> **Q**：在 OSI 参考模型中，实现端到端可靠传输的是哪一层？
>
> **A**：**传输层（第4层）**。传输层负责进程到进程的通信，可提供可靠传输服务。注意"端到端"指端口到端口，区别于数据链路层的"点到点"。

> **Q**：路由器工作在 OSI 的哪一层？交换机呢？
>
> **A**：路由器工作在**网络层（第3层）**，交换机工作在**数据链路层（第2层）**。
