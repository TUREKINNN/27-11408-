# 3.1 数据链路层的功能

> 王道考研·计算机网络 第三章 数据链路层

---

## 一、数据链路层的地位

```
┌────────────────────────────────────────────┐
│  5  应用层                                  │
│  4  传输层                                  │
│  3  网络层          IP数据报（分组）          │
│  2  数据链路层   ┌─帧首─┬─数据─┬─帧尾─┐    │
│  1  物理层      10100…110100（比特流）       │
└────────────────────────────────────────────┘
```

- **数据链路层** 使用 **物理层** 提供的"比特传输"服务
- **数据链路层** 为 **网络层** 提供服务，将 IP 数据报封装成**帧**，传输给下一个相邻结点

---

## 二、基本概念

### 2.1 链路 vs 数据链路

| 概念 | 定义 |
|------|------|
| **链路（Link）** | 从一个结点到相邻结点的一段物理线路（传输介质+物理层） |
| **数据链路（Data Link）** | 在物理链路上增加通信协议后形成的逻辑链路 |

### 2.2 帧（Frame）

数据链路层的协议数据单元（PDU）。

```
┌──────────┬──────────────────┬──────────┐
│  帧首部   │     数据部分      │  帧尾部   │
│ (Header)  │   (Payload)      │ (Trailer) │
└──────────┴──────────────────┴──────────┘
```

## 三、数据链路层的功能

### 3.1 为网络层提供服务

| 服务类型 | 说明 | 可靠性 |
|---------|------|--------|
| **无确认的无连接服务** | 不建立连接，不确认，不重传 | 低（以太网） |
| **有确认的无连接服务** | 不建立连接，但有确认和重传 | 中（无线网络） |
| **有确认的面向连接服务** | 建立连接，有确认和重传 | 高 |

### 3.2 链路管理

- 连接的建立、维持和释放
- 主要用于面向连接的服务

### 3.3 帧定界（组帧）

确定帧的起始和结束位置。

**四种组帧方法**：
1. **字符计数法**：帧首部用一个字段标明帧的字符数
2. **字符填充法**：用特殊字符标记帧的开始和结束
3. **零比特填充法**：用 `01111110` 标记帧的边界
4. **违规编码法**：利用物理层编码的违规模式

### 3.4 流量控制

限制发送方的发送速率，使接收方来得及接收。

> - 数据链路层：控制**相邻结点**之间的流量（点到点）
> - 传输层：控制**端到端**的流量（进程到进程）

### 3.5 差错控制

检测和纠正传输过程中产生的比特差错。

**差错类型**：
- **位错**：某比特位 0→1 或 1→0
- **帧错**：帧丢失、帧重复、帧失序

**检错编码**：
- **奇偶校验码**：在数据后附加1位校验位
- **CRC 校验码**：循环冗余校验，检错能力强

**纠错编码**：
- **海明码**：能检测并纠正单比特错误

## 四、CRC 循环冗余校验（重点）

### 4.1 原理

发送方和接收方约定一个**生成多项式 $G(x)$**。

**发送方**：
1. 将待发送数据 $D(x)$ 左移 $r$ 位（$r$ = 生成多项式最高次幂）
2. 用 $D(x) \cdot x^r$ 除以 $G(x)$，得到余数 $R(x)$
3. 发送 $D(x) \cdot x^r + R(x)$

**接收方**：
1. 将收到的数据除以 $G(x)$
2. 余数为 0 → 无差错；余数非 0 → 有差错

### 4.2 计算示例

> 设数据 $D = 101001$，生成多项式 $G(x) = x^3 + x + 1$（即 1011）

**步骤**：
1. $D$ 左移 3 位：`101001000`
2. 模 2 除法：`101001000 ÷ 1011`

```
          100100
    ─────────────
1011│101001000
     1011
     ────
      00101
      00000
      ─────
       01010
       00000
       ─────
        10100
        1011
        ────
         00100
         00000
         ─────
          01000
          00000
          ─────
           1000
           1011
           ────
            011  ← 余数 R = 011
```

3. 发送：`101001011`（原始数据 + 余数）

> CRC 只能**检错**，不能**纠错**。数据链路层通常使用 CRC 进行检错，发现差错就丢弃帧。
