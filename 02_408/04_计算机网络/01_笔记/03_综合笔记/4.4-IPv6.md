# 4.4 IPv6

> IPv6 是 IPv4 的下一代协议，主要解决 IPv4 地址耗尽问题。IPv6 使用 **128 位**地址，提供几乎无限的地址空间。

---

## 一、IPv6 与 IPv4 的对比

| 特性 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | **32 bit**（4B） | **128 bit**（16B） |
| 地址表示 | 点分十进制 | **冒号十六进制** |
| 地址数量 | 约 43 亿 | 约 $3.4 \times 10^{38}$ |
| 首部长度 | 可变（20\~60B） | **固定 40B** |
| 分片 | 路由器和发送方都可以 | 只在**发送方**分片 |
| 校验和 | 有 | **无**（交给上层） |
| ARP | 有 | 用**邻居发现协议**替代 |
| DHCP | 有 | 支持**无状态自动配置** |

---

## 二、IPv6 地址表示

### 2.1 冒号十六进制记法

```
IPv6: 2001:0DB8:0000:0000:0000:0000:1428:57AB
```

### 2.2 简化规则

1. **省略前导零**：`0DB8` → `DB8`
2. **连续全零段用 :: 代替**（只能使用一次）：

```
2001:DB8:0000:0000:0000:0000:1428:57AB
→ 2001:DB8::1428:57AB
```

---

## 三、IPv6 地址类型

| 类型 | 说明 | 前缀 |
|------|------|------|
| **单播** | 一对一通信 | - |
| **组播** | 一对多通信 | FF00::/8 |
| **任播** | 一对最近的一个（新增） | - |

> IPv6 中不再有广播地址，广播功能由组播替代。

---

## 四、IPv6 首部格式

固定 40 字节，包含以下字段：

| 字段 | 长度 | 说明 |
|------|------|------|
| 版本 | 4 bit | 值为 6 |
| 流量类别 | 8 bit | 类似 IPv4 的 TOS |
| 流标签 | 20 bit | 标识同一"流"的分组 |
| 有效载荷长度 | 16 bit | 数据部分长度（不含首部） |
| 下一个首部 | 8 bit | 类似 IPv4 的协议字段 |
| 跳数限制 | 8 bit | 类似 IPv4 的 TTL |
| 源地址 | 128 bit | 发送方 IPv6 地址 |
| 目的地址 | 128 bit | 接收方 IPv6 地址 |

> - 取消了首部长度字段（固定 40B）
> - 取消了校验和字段（交给上层处理）
> - 取消了分片相关字段（只在发送方分片）

---

## 五、IPv4 到 IPv6 的过渡

### 5.1 双栈技术

路由器同时运行 IPv4 和 IPv6 协议栈，根据目的地址选择使用哪种协议。

### 5.2 隧道技术

将 IPv6 数据报封装在 IPv4 数据报中传输：

```
IPv4首部 | IPv6首部 | IPv6数据
← 封装 →
```

---

## 考点总结

> 1. IPv6 地址 128 bit，冒号十六进制表示
> 2. IPv6 取消了广播，用组播替代
> 3. IPv6 首部固定 40B，取消了校验和、分片字段
> 4. IPv6 支持无状态自动配置（不需要 DHCP）
> 5. 过渡技术：双栈和隧道
> 6. :: 只能使用一次，用于省略连续全零段
