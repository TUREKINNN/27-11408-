## 1. 计算机网络的基本概念

### 1.1 计算机网络的定义

<span style="background-color:#ccc1d9">**计算机网络**</span>：将地理位置不同的计算机通过通信设备和通信线路连接起来，实现资源共享和信息交换的系统。

### 1.2 计算机网络的功能

| 功能 | 描述 |
|------|------|
| **资源共享** | 共享硬件、软件、数据 |
| **数据通信** | 传输数据和信息 |
| **分布式处理** | 协同完成任务 |
| **提高可靠性** | 冗余备份 |

### 1.3 计算机网络的分类

#### 1.3.1 按覆盖范围分类

| 类型 | 覆盖范围 | 特点 |
|------|----------|------|
| **局域网（LAN）** | 几公里内 | 高速，低延迟 |
| **城域网（MAN）** | 城市范围 | 中等规模 |
| **广域网（WAN）** | 跨城市/国家 | 规模大，延迟高 |
| **互联网（Internet）** | 全球范围 | 连接多个网络 |

#### 1.3.2 按拓扑结构分类

| 类型 | 描述 |
|------|------|
| **总线型** | 所有节点连接到一条总线 |
| **星型** | 所有节点连接到中心节点 |
| **环型** | 节点连成环状 |
| **树型** | 层次结构 |
| **网状** | 任意节点相连 |

## 2. 网络分层模型

### 2.1 分层的意义

<span style="background-color:#8db3e2">**分层原则**：每层实现一个独立的功能，层间通过接口通信，每层使用下层提供的服务。</span>

**优点**：

1. **模块化**：每层独立设计和实现
2. **灵活性**：修改一层不影响其他层
3. **可维护性**：便于故障排查和升级

### 2.2 OSI七层模型

#### 2.2.1 模型结构

```
┌─────────────────────────────────────────────┐
│ 7. 应用层                                    │
├─────────────────────────────────────────────┤
│ 6. 表示层                                    │
├─────────────────────────────────────────────┤
│ 5. 会话层                                    │
├─────────────────────────────────────────────┤
│ 4. 传输层                                    │
├─────────────────────────────────────────────┤
│ 3. 网络层                                    │
├─────────────────────────────────────────────┤
│ 2. 数据链路层                                │
├─────────────────────────────────────────────┤
│ 1. 物理层                                    │
└─────────────────────────────────────────────┘
```

#### 2.2.2 各层功能

| 层次 | 名称 | 功能 |
|------|------|------|
| **第7层** | 应用层 | 为应用程序提供网络服务 |
| **第6层** | 表示层 | 数据格式转换、加密解密 |
| **第5层** | 会话层 | 建立和管理会话 |
| **第4层** | 传输层 | 端到端通信、流量控制 |
| **第3层** | 网络层 | 分组转发、路由选择 |
| **第2层** | 数据链路层 | 帧的传输、差错检测 |
| **第1层** | 物理层 | 比特传输、介质访问 |

### 2.3 TCP/IP四层模型

#### 2.3.1 模型结构

```
┌─────────────────────────────────────────────┐
│ 4. 应用层                                    │
├─────────────────────────────────────────────┤
│ 3. 传输层                                    │
├─────────────────────────────────────────────┤
│ 2. 网际层                                    │
├─────────────────────────────────────────────┤
│ 1. 网络接口层                                │
└─────────────────────────────────────────────┘
```

#### 2.3.2 与OSI模型的对应关系

| TCP/IP | OSI |
|--------|-----|
| **应用层** | 应用层、表示层、会话层 |
| **传输层** | 传输层 |
| **网际层** | 网络层 |
| **网络接口层** | 数据链路层、物理层 |

## 5. 网络的性能指标

### 5.1 带宽

**定义**：单位时间内传输的数据量。

**单位**：bps（比特/秒）

### 5.2 延迟

**定义**：数据从源到目的的时间。

**组成**：

| 组成 | 描述 |
|------|------|
| **传输延迟** | 数据传输时间 |
| **传播延迟** | 信号传播时间 |
| **处理延迟** | 设备处理时间 |
| **排队延迟** | 等待时间 |

### 5.3 吞吐量

**定义**：单位时间内成功传输的数据量。

### 5.4 丢包率

**定义**：丢失的数据包占总数据包的比例。

## 6. ISO/OSI参考模型与TCP/IP模型的比较

### 6.1 相同点

1. 都是分层模型
2. 都基于协议栈
3. 都支持异构网络互联

### 6.2 不同点

| 对比项 | OSI | TCP/IP |
|--------|-----|--------|
| **层数** | 7层 | 4层 |
| **应用层** | 3层 | 1层 |
| **网络层** | 1层 | 1层（网际层） |
| **链路层** | 2层 | 1层（网络接口层） |
| **面向连接** | 传输层 | 传输层（TCP） |
| **无连接** | 网络层 | 网络层（IP） |

## 7. 典型例题

### 例题 1：OSI七层模型

**题目**：描述OSI七层模型各层的功能。

**解答**：

```
1. 物理层：比特传输，介质访问
2. 数据链路层：帧的传输，差错检测
3. 网络层：分组转发，路由选择
4. 传输层：端到端通信，流量控制
5. 会话层：建立和管理会话
6. 表示层：数据格式转换，加密解密
7. 应用层：为应用程序提供网络服务
```

**总结**：OSI模型是网络协议的标准分层模型。

**来源**：408历年真题，考查方式：简答题

---

### 例题 2：TCP/IP模型

**题目**：比较OSI模型和TCP/IP模型的异同。

**解答**：

```
相同点：
- 都是分层模型
- 都基于协议栈
- 都支持异构网络互联

不同点：
- OSI 7层，TCP/IP 4层
- OSI应用层分为3层，TCP/IP合并为1层
- OSI严格分层，TCP/IP允许跨层调用
```

**总结**：TCP/IP是实际应用的标准。

**来源**：408历年真题，考查方式：简答题

### 例题 3：网络延迟计算

**题目**：数据长度为1000字节，带宽为10Mbps，传播距离为1000km，传播速度为2×10^8 m/s。求总延迟。

**解答**：

```
数据长度 = 1000字节 = 8000比特

传输延迟 = 8000 / 10×10^6 = 0.0008秒 = 0.8ms

传播延迟 = 1000×10^3 / 2×10^8 = 0.005秒 = 5ms

总延迟 = 传输延迟 + 传播延迟 = 0.8ms + 5ms = 5.8ms
```

**总结**：网络延迟由传输延迟和传播延迟组成。

**来源**：408历年真题，考查方式：计算题
