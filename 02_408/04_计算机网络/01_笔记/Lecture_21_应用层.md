## 1. 应用层的功能

### 1.1 应用层的定义

<span style="background-color:#ccc1d9">**应用层**</span>：为应用程序提供网络服务。

### 1.2 应用层的功能

| 功能 | 描述 |
|------|------|
| **网络服务** | 提供各种网络服务 |
| **数据交换** | 应用程序间的数据交换 |
| **用户接口** | 用户与网络的接口 |

---

## 2. HTTP协议

### 2.1 HTTP的特点

<span style="background-color:#ccc1d9">**HTTP**</span>：超文本传输协议。

**特点**：

| 特点 | 描述 |
|------|------|
| **无连接** | 每次请求建立连接，完成后断开 |
| **无状态** | 不保存请求状态 |
| **基于TCP** | 使用TCP传输 |
| **明文传输** | 数据不加密 |

### 2.2 HTTP的请求方法

| 方法 | 描述 |
|------|------|
| **GET** | 请求获取资源 |
| **POST** | 提交数据 |
| **PUT** | 更新资源 |
| **DELETE** | 删除资源 |
| **HEAD** | 获取资源头部 |

### 2.3 HTTP的状态码

| 状态码 | 类别 | 描述 |
|--------|------|------|
| **1xx** | 信息 | 请求已接收，继续处理 |
| **2xx** | 成功 | 请求成功 |
| **3xx** | 重定向 | 需要进一步操作 |
| **4xx** | 客户端错误 | 请求有误 |
| **5xx** | 服务器错误 | 服务器出错 |

**常见状态码**：

| 状态码 | 描述 |
|--------|------|
| **200** | OK | 请求成功 |
| **301** | Moved Permanently | 永久重定向 |
| **302** | Found | 临时重定向 |
| **400** | Bad Request | 请求错误 |
| **401** | Unauthorized | 未授权 |
| **403** | Forbidden | 禁止访问 |
| **404** | Not Found | 资源未找到 |
| **500** | Internal Server Error | 服务器错误 |

### 2.4 HTTP的报文结构

#### 2.4.1 请求报文

```
请求行：GET /index.html HTTP/1.1
请求头：Host: www.example.com
        User-Agent: Mozilla/5.0
        Accept: text/html
空行
请求体：（POST请求有）
```

#### 2.4.2 响应报文

```
状态行：HTTP/1.1 200 OK
响应头：Date: Mon, 01 Jan 2024 00:00:00 GMT
        Content-Type: text/html
        Content-Length: 1024
空行
响应体：<html>...</html>
```

### 2.5 HTTPS

<span style="background-color:#ccc1d9">**HTTPS**</span>：安全超文本传输协议。

**特点**：

- 数据加密传输
- 使用SSL/TLS协议
- 端口443

**加密方式**：

| 方式 | 描述 |
|------|------|
| **对称加密** | 双方使用相同密钥 |
| **非对称加密** | 公钥加密，私钥解密 |
| **数字证书** | 验证服务器身份 |

## 3. DNS协议

### 3.1 DNS的功能

<span style="background-color:#ccc1d9">**DNS**</span>：域名系统，将域名转换为IP地址。

### 3.2 DNS的层次结构

```
根域名服务器
├── 顶级域名服务器（.com, .cn, .org）
│   └── 权威域名服务器（example.com）
│       └── 本地域名服务器
```

### 3.3 DNS的查询过程

#### 3.3.1 递归查询

```
客户端 → 本地DNS → 根DNS → 顶级DNS → 权威DNS
```

#### 3.3.2 迭代查询

```
客户端 → 本地DNS（返回下一个服务器地址）
客户端 → 根DNS（返回顶级DNS地址）
客户端 → 顶级DNS（返回权威DNS地址）
客户端 → 权威DNS（返回IP地址）
```

### 3.4 DNS的记录类型

| 类型 | 描述 |
|------|------|
| **A** | 域名到IPv4地址 |
| **AAAA** | 域名到IPv6地址 |
| **CNAME** | 别名记录 |
| **MX** | 邮件服务器 |
| **NS** | 域名服务器 |

## 4. FTP协议

### 4.1 FTP的功能

<span style="background-color:#ccc1d9">**FTP**</span>：文件传输协议。

**功能**：

- 上传文件
- 下载文件
- 管理文件

### 4.2 FTP的连接

**控制连接**：传输命令，端口21。

**数据连接**：传输数据，端口20。

### 4.3 FTP的工作模式

#### 4.3.1 主动模式

```
服务器主动连接客户端的数据端口
```

#### 4.3.2 被动模式

```
服务器等待客户端连接数据端口
```

## 5. SMTP协议

### 5.1 SMTP的功能

<span style="background-color:#ccc1d9">**SMTP**</span>：简单邮件传输协议。

**功能**：发送邮件。

### 5.2 SMTP的过程

```
1. 客户端连接服务器端口25
2. 发送EHLO命令
3. 发送AUTH命令（认证）
4. 发送MAIL FROM命令
5. 发送RCPT TO命令
6. 发送DATA命令
7. 发送邮件内容
8. 发送QUIT命令
```

### 5.3 POP3和IMAP

| 协议 | 描述 |
|------|------|
| **POP3** | 下载邮件到本地 |
| **IMAP** | 在服务器上管理邮件 |

## 6. 其他应用层协议

### 6.1 DHCP

<span style="background-color:#ccc1d9">**DHCP**</span>：动态主机配置协议。

**功能**：自动分配IP地址。

**过程**：

```
1. 客户端发送DHCPDISCOVER广播
2. 服务器发送DHCPOFFER响应
3. 客户端发送DHCPREQUEST请求
4. 服务器发送DHCPACK确认
```

### 6.2 SNMP

<span style="background-color:#ccc1d9">**SNMP**</span>：简单网络管理协议。

**功能**：管理网络设备。

### 6.3 SSH

<span style="background-color:#ccc1d9">**SSH**</span>：安全外壳协议。

**功能**：安全远程登录。

## 7. 典型例题

### 例题 1：HTTP请求方法

**题目**：描述HTTP的主要请求方法。

**解答**：

```
GET：请求获取资源
POST：提交数据
PUT：更新资源
DELETE：删除资源
HEAD：获取资源头部信息

区别：
- GET：幂等，参数在URL中
- POST：非幂等，参数在请求体中
- PUT：幂等，更新整个资源
- DELETE：幂等，删除资源
```

**总结**：不同请求方法有不同的语义。

**来源**：408历年真题，考查方式：简答题

### 例题 2：DNS查询过程

**题目**：描述DNS递归查询的过程。

**解答**：

```
1. 客户端向本地DNS服务器发送查询请求
2. 本地DNS服务器检查缓存，若有则返回
3. 若无缓存，向根DNS服务器发送查询
4. 根DNS服务器返回顶级DNS服务器地址
5. 本地DNS服务器向顶级DNS服务器发送查询
6. 顶级DNS服务器返回权威DNS服务器地址
7. 本地DNS服务器向权威DNS服务器发送查询
8. 权威DNS服务器返回IP地址
9. 本地DNS服务器缓存并返回给客户端
```

**总结**：DNS查询通过层次化服务器完成。

**来源**：408历年真题，考查方式：简答题

**题目**：说明常见HTTP状态码的含义。

```
200 OK：请求成功
301 Moved Permanently：永久重定向
302 Found：临时重定向
400 Bad Request：请求错误
401 Unauthorized：未授权
403 Forbidden：禁止访问
404 Not Found：资源未找到
500 Internal Server Error：服务器错误

分类：
- 1xx：信息
- 2xx：成功
- 3xx：重定向
- 4xx：客户端错误
- 5xx：服务器错误
```

**总结**：状态码表示请求的处理结果。
