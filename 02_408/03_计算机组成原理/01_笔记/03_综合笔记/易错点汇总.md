# 计算机组成原理易错点汇总

## 一、浮点数易错点

### 1. IEEE 754阶码偏置值混淆

**易错点：** 混淆单精度和双精度的偏置值

| 精度 | 阶码位数 | 偏置值 |
|------|----------|--------|
| 单精度 | 8位 | 127 (01111111) |
| 双精度 | 11位 | 1023 (01111111111) |

**正确计算：** 实际阶码 = 存储阶码 - 偏置值

**例：** 存储阶码 10000010 (130)
- 单精度：实际阶码 = 130 - 127 = 3
- 如果错误使用1023：实际阶码 = 130 - 1023 = -893 ❌

### 2. 隐藏位（前导1）遗漏

**易错点：** 忘记IEEE 754规格化数有隐藏的前导1

**正确格式：** $V = (-1)^S \times \mathbf{1}.M \times 2^{E-127}$

**错误理解：** $V = (-1)^S \times 0.M \times 2^{E-127}$ ❌

**例：** 尾数字段为 10000000000000000000000
- 正确：尾数 = 1.10000000000000000000000 = 1.5
- 错误：尾数 = 0.10000000000000000000000 = 0.5 ❌

### 3. 阶码全0和全1的特殊含义

**易错点：** 不理解阶码全0和全1的特殊处理

| 阶码E | 尾数M | 含义 |
|-------|-------|------|
| 00000000 | 0 | ±0 |
| 00000000 | ≠0 | 非规格化数（无隐藏位） |
| 11111111 | 0 | ±∞ |
| 11111111 | ≠0 | NaN |

**重要：** 非规格化数没有隐藏位！

### 4. 浮点数溢出判断错误

**易错点：** 只看尾数溢出，忘记检查阶码溢出

**溢出类型：**
- **上溢：** 阶码超过最大值（如单精度 > 127）
- **下溢：** 阶码低于最小值（如单精度 < -126）

**正确流程：**
1. 先进行尾数运算
2. 规格化（可能改变阶码）
3. 最后检查阶码是否溢出

### 5. 对阶方向错误

**易错点：** 对阶时移动方向搞反

**正确规则：** 小阶向大阶看齐
- 小阶的尾数右移
- 阶码增大直到等于大阶

**错误做法：** 大阶向小阶看齐 ❌

**例：** 阶码 2 和 5 对阶
- 正确：阶码2的尾数右移3位，阶码变为5
- 错误：阶码5的尾数左移3位，阶码变为2 ❌

### 6. 规格化方向混淆

**易错点：** 左规和右规的条件搞反

| 规格化 | 条件 | 操作 |
|--------|------|------|
| 左规 | 尾数形如 0.0xxx 或 1.1xxx | 尾数左移，阶码减1 |
| 右规 | 尾数形如 01.xxx 或 10.xxx | 尾数右移，阶码加1 |

**记忆：** 左规 = 尾数太小需要左移放大

## 二、Cache映射易错点

### 1. 地址划分计算错误

**易错点：** 主存地址划分时各字段位数计算错误

**正确方法：**
```
块内偏移位数 = log₂(块大小)
行号/组号位数 = log₂(Cache行数/组数)
标记位数 = 主存地址位数 - 行号位数 - 块内偏移位数
```

**例：** 主存32位，Cache 4KB，块大小64B，直接映射
- 块内偏移 = log₂(64) = 6位
- Cache行数 = 4KB / 64B = 64行
- 行号 = log₂(64) = 6位
- 标记 = 32 - 6 - 6 = 20位

### 2. 直接映射组号计算

**易错点：** 混淆直接映射和组相联的组号计算

**直接映射：**
$$Cache行号 = 主存块号 \mod Cache行数$$

**组相联映射：**
$$Cache组号 = 主存块号 \mod Cache组数$$

**区别：** 直接映射是行号，组相联是组号

### 3. 命中判断条件

**易错点：** 判断Cache命中时遗漏条件

**命中条件（直接映射）：**
1. 有效位 = 1
2. 标记字段匹配

**命中条件（组相联）：**
1. 有效位 = 1
2. 在该组的某个行中找到匹配的标记

### 4. 写策略理解错误

**易错点：** 混淆写直达和写回的时机

| 写策略 | 写命中 | 写缺失 |
|--------|--------|--------|
| 写直达 | 同时写Cache和主存 | 写分配/非写分配 |
| 写回 | 只写Cache，标记脏位 | 写分配/非写分配 |

**写回优势：** 减少写主存次数
**写直达优势：** Cache和主存数据一致

### 5. 替换算法选择

**易错点：** 不同映射方式使用不同替换算法

| 映射方式 | 是否需要替换算法 |
|----------|----------------|
| 直接映射 | 不需要（固定位置） |
| 全相联 | 需要（可放任意位置） |
| 组相联 | 需要（组内选择） |

### 6. Cache大小计算遗漏

**易错点：** 只计算数据部分，忽略标记和标志位

**完整Cache行大小：**
```
| 有效位(1bit) | 脏位(1bit) | 标记(Tag bits) | 数据(Block size) |
```

**总大小 = 行数 × (有效位 + 脏位 + 标记 + 数据)**

## 三、流水线易错点

### 1. 流水线段数与级数混淆

**易错点：** 混淆流水线段数k和任务数n

- **k（段数）：** 流水线的功能段数量
- **n（任务数）：** 需要处理的任务数量

**执行时间公式：** $T = (n + k - 1) \times \Delta t$

### 2. 非均匀流水线计算

**易错点：** 对非均匀流水线直接使用均匀流水线公式

**正确方法：**
$$T = \sum_{i=1}^{k} \Delta t_i + (n-1) \times \max(\Delta t_i)$$

**原因：** 流水线的吞吐率取决于最慢的段

### 3. 加速比计算错误

**易错点：** 分母使用错误

**正确公式：**
$$S = \frac{T_{非流水}}{T_{流水}} = \frac{n \times \sum_{i=1}^{k} \Delta t_i}{T_{流水}}$$

**注意：** 非流水时间是串行执行所有段的时间

### 4. 流水线冲突类型混淆

**易错点：** 混淆三种数据冲突

| 冲突类型 | 全称 | 含义 | 解决方法 |
|----------|------|------|----------|
| RAW | Write After Read | 后续指令读取前面指令写入的数据 | 数据转发 |
| WAR | Read After Write | 后续指令写入前面指令读取的位置 | 寄存器重命名 |
| WAW | Write After Write | 两条指令写入同一位置 | 寄存器重命名 |

**常见：** RAW冲突最常见，需要数据转发解决

### 5. 数据转发时机错误

**易错点：** 不清楚数据何时可以转发

**转发时机：**
- EX段结果 → EX段输入（同一周期）
- MEM段结果 → EX段输入（下一周期）

**不能转发的情况：**
- Load指令的数据需要等待MEM段完成

### 6. 分支预测错误率

**易错点：** 不会计算分支预测的性能影响

**正确计算：**
$$实际吞吐率 = 理想吞吐率 \times (1 - 预测错误率 \times 惩罚周期数)$$

## 四、寻址方式易错点

### 1. 基址寻址与变址寻址混淆

**易错点：** 不清楚两者的应用场景

| 寻址方式 | 基址寄存器 | 变址寄存器 | 应用场景 |
|----------|-----------|-----------|----------|
| 基址寻址 | BR（操作系统设置） | - | 程序重定位、多道程序 |
| 变址寻址 | - | IX（用户设置） | 数组访问、循环 |
| 基址+变址 | BR | IX | 二维数组访问 |

**区别：** 基址寻址的BR由操作系统管理，变址寻址的IX由用户程序管理

### 2. 相对寻址偏移量计算

**易错点：** 偏移量的计算基准搞错

**正确：** EA = (PC) + A，其中PC是下一条指令地址

**错误：** EA = 当前指令地址 + A ❌

**例：** 当前指令地址 1000，指令长度 4，偏移量 20
- 正确：EA = 1004 + 20 = 1024
- 错误：EA = 1000 + 20 = 1020 ❌

### 3. 间接寻址层次

**易错点：** 混淆一次间接和多次间接

- **一次间接：** EA = (A)，A指向的单元存放EA
- **多次间接：** EA = ((A))，需要多次访存

**访存次数：**
- 直接寻址：1次
- 一次间接：2次
- 多次间接：n次

### 4. 堆栈寻址方向

**易错点：** 混淆栈增长方向

| 类型 | 栈增长方向 | 入栈操作 | 出栈操作 |
|------|-----------|----------|----------|
| 向下增长栈 | 高地址→低地址 | SP减小 | SP增大 |
| 向上增长栈 | 低地址→高地址 | SP增大 | SP减小 |

**x86使用向下增长栈**

## 五、IO系统易错点

### 1. 中断响应条件

**易错点：** 不清楚中断响应的完整条件

**中断响应条件：**
1. 有中断请求
2. CPU允许中断（中断允许位 = 1）
3. 当前指令执行完毕
4. 没有更高优先级的中断正在处理

### 2. 中断优先级与屏蔽

**易错点：** 混淆中断优先级和中断屏蔽

| 概念 | 含义 |
|------|------|
| 中断优先级 | 决定哪个中断先被响应 |
| 中断屏蔽 | 决定哪些中断可以被暂时忽略 |

**中断屏蔽字：**
- 1 = 屏蔽该级中断
- 0 = 允许该级中断

### 3. 向量中断与非向量中断

**易错点：** 混淆两种中断类型

| 类型 | 中断服务程序地址来源 | 特点 |
|------|---------------------|------|
| 向量中断 | 中断向量表 | 响应快，硬件实现 |
| 非向量中断 | 软件查询 | 响应慢，灵活 |

### 4. DMA三种传送方式

**易错点：** 混淆DMA的三种传送方式

| 方式 | CPU状态 | 特点 |
|------|---------|------|
| 停止CPU访问 | CPU停止 | 简单，CPU利用率低 |
| 周期挪用 | CPU暂停一个周期 | CPU利用率高，DMA慢 |
| 交替访问 | CPU和DMA交替 | 无冲突，需要双端口存储器 |

### 5. DMA与中断对比

**易错点：** 不清楚DMA和中断的本质区别

| 对比项 | 中断方式 | DMA方式 |
|--------|----------|---------|
| 数据传送单位 | 字/字节 | 数据块 |
| CPU参与 | 每个字都参与 | 仅开始和结束 |
| 适用设备 | 低速设备 | 高速设备 |
| 中断作用 | 数据传送 | 传送结束通知 |

## 六、其他高频易错点

### 1. 存储器层次理解错误

**易错点：** 不理解各层次的特点

```
寄存器 ← 最快、最小、最贵
   ↓
Cache ← 快、小、贵
   ↓
主存 ← 中等速度、中等大小
   ↓
辅存 ← 最慢、最大、最便宜
```

### 2. SRAM与DRAM区别

**易错点：** 混淆两者的特性

| 特性 | SRAM | DRAM |
|------|------|------|
| 存储元件 | 触发器 | 电容 |
| 是否需要刷新 | 不需要 | 需要 |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 成本 | 高 | 低 |
| 应用 | Cache | 主存 |

### 3. 总线带宽计算

**易错点：** 忽略总线开销

**正确公式：**
$$带宽 = \frac{数据量}{传输时间 + 协议开销}$$

**例：** 数据宽度32位，时钟频率100MHz，每个传输需要2个时钟周期
$$带宽 = \frac{32bit}{2 \times 10ns} = \frac{4B}{20ns} = 200MB/s$$

### 4. 指令字长与存储字长

**易错点：** 混淆指令字长和存储字长

| 概念 | 含义 | 关系 |
|------|------|------|
| 存储字长 | 存储器一次读写的位数 | 由硬件决定 |
| 指令字长 | 一条指令的位数 | 可以是存储字长的整数倍 |
| 机器字长 | CPU一次能处理的位数 | 通常等于数据总线宽度 |

### 5. 微指令与机器指令

**易错点：** 混淆微指令和机器指令

| 概念 | 执行者 | 作用 |
|------|--------|------|
| 机器指令 | CPU执行 | 程序员可见的指令 |
| 微指令 | 控制器执行 | 产生控制信号 |
| 微程序 | 微指令序列 | 实现一条机器指令 |

**关系：** 一条机器指令 = 一个微程序 = 多条微指令
