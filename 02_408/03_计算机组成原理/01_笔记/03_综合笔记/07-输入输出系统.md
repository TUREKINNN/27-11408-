# 07 输入输出系统

> **选择题**常考各种I/O方式的特点对比；**大题**常考中断响应条件判断和DMA传送时间计算。

---

## 一、I/O接口

### 1.1 I/O接口的功能

| 功能 | 说明 |
|------|------|
| **数据缓冲** | 缓冲CPU与外设的速度差异 |
| **设备选择** | 通过地址译码选择特定设备 |
| **信号转换** | 电平转换、串/并转换、A/D、D/A |
| **控制命令传递** | 将CPU的控制命令传送给设备 |
| **状态返回** | 将设备状态返回给CPU |
| **数据格式转换** | 串/并转换等 |

### 1.2 I/O接口的类型

| 分类依据 | 类型 |
|----------|------|
| 数据传送方式 | **并行接口** / **串行接口** |
| 是否可编程 | **可编程接口** / **不可编程接口** |
| 按功能选择方式 | **可选址接口** / **不可选址接口** |

### 1.3 I/O接口的基本结构

```
              数据总线
                ↑↓
    ┌───────────────────────────┐
    │         I/O接口           │
    │                           │
    │  ┌─────┐    ┌─────────┐  │
    │  │数据  │    │ 控制/   │  │
    │  │缓冲  │    │ 状态    │  │
    │  │寄存器│    │ 寄存器  │  │
    │  └──┬──┘    └────┬────┘  │
    │     │            │       │
    │  ┌──┴──┐    ┌────┴────┐  │
    │  │地址  │    │ 控制    │  │
    │  │译码  │    │ 逻辑    │  │
    │  └─────┘    └────┬────┘  │
    └──────────────────┬───────┘
                       ↓
                  设备侧
```

- **数据缓冲寄存器**（数据端口）：存放输入/输出数据
- **状态寄存器**：存放设备状态（忙/就绪/错误等）
- **控制寄存器**：存放CPU发来的控制命令

## 二、端口编址方式

### 2.1 统一编址（存储器映射I/O）

- I/O端口的地址**编入主存地址空间**中
- 用**访存指令**（如LOAD/STORE）即可访问I/O端口
- I/O端口和主存共享地址空间

### 2.2 独立编址（I/O映射I/O）

- I/O端口有**独立的地址空间**（与主存地址空间分离）
- 需要**专用I/O指令**（如IN/OUT）访问

### 2.3 对比表

| 对比项 | 统一编址 | 独立编址 |
|--------|---------|---------|
| **地址空间** | 与主存共享 | 独立地址空间 |
| **访问指令** | 通用访存指令（MOV等） | 专用I/O指令（IN/OUT） |
| **指令功能** | 可用全部访存指令（灵活） | 仅专用指令（功能有限） |
| **主存空间** | 减少可用主存空间 | 不影响主存空间 |
| **译码电路** | 复杂（需区分主存和I/O） | 简单（地址空间小） |
| **程序可读性** | 较差（看不出是I/O操作） | 较好 |
| **代表系统** | MIPS、ARM、RISC-V | x86 |

## 三、程序查询方式（Programmed I/O）

### 3.1 工作原理

CPU**主动查询**设备状态，数据传送完全由CPU控制。

### 3.2 流程图

```
        ┌───────────┐
        │  CPU发出  │
        │  读命令   │
        └─────┬─────┘
              ↓
        ┌───────────┐
    ┌──→│ 读设备状态 │
    │   └─────┬─────┘
    │         ↓
    │    ┌─────────┐
    │    │就绪？    │──否──→┐
    │    └────┬────┘       │
    │    是↓                │
    │   ┌──────────┐       │
    │   │ 从接口    │       │
    │   │ 读数据    │       │
    │   └────┬─────┘       │
    │        ↓              │
    │   ┌──────────┐       │
    │   │数据送CPU  │       │
    │   └────┬─────┘       │
    │        ↓              │
    │   ┌──────────┐       │
    │   │修改计数器  │←──────┘
    │   │/地址指针  │
    │   └────┬─────┘
    │        ↓
    │   ┌──────────┐
    │   │传送完成？  │──否──→回到"读设备状态"
    │   └────┬────┘
    │    是↓
    │   ┌──────────┐
    │   │   结束    │
    │   └──────────┘
```

### 3.3 特点

- **优点**：接口简单，硬件需求少
- **缺点**：CPU利用率低（忙等待），不能处理突发事件
- **适用**：低速设备、简单系统

## 四、程序中断方式

### 4.1 中断的基本概念

- 设备准备好后，主动向CPU发送**中断请求**
- CPU在**当前指令执行完毕**后响应中断
- 无需忙等待，CPU与设备可**并行工作**

### 4.2 中断响应条件


| 条件 | 说明 |
|------|------|
| ① 中断源有中断请求 | 设备发出中断请求信号 |
| ② CPU允许中断（开中断） | **中断允许触发器 IF = 1**（STI指令置1，CLI指令清0） |
| ③ 一条指令执行完毕 | 不是在指令执行中间响应 |

> - 关中断状态（IF=0）
> - 指令执行过程中
> - 某些不可中断的指令（如关中断指令本身、特权指令）
> - 更高级中断源正在被处理（取决于优先级）

### 4.3 中断判优

#### 硬件判优

| 方式 | 说明 |
|------|------|
| **硬件排队器** | 用门电路实现优先级编码，同时多个请求时选优先级最高的 |
| **链式判优** | 类似链式总线仲裁，串行查询 |

#### 软件判优

- 用程序逐个查询中断源（类似轮询），灵活但速度慢

#### 中断优先级

```
高 ← 优先级 → 低

硬件故障中断 > 程序性中断 > DMA > 外部设备中断 > 用户程序中断
```

### 4.4 中断处理过程

```
1. 关中断
2. 保存断点（PC和PSW压栈）
3. 识别中断源（判优确定最高优先级中断源）
4. 保存现场（通用寄存器内容压栈）
5. 中断服务（执行中断服务程序）
6. 恢复现场（出栈恢复寄存器）
7. 开中断
8. 中断返回（IRET，恢复PC和PSW）
```

### 4.5 中断向量表

- **中断向量**：中断服务程序的入口地址
- **中断向量表**：存放所有中断向量的表，通常位于内存低地址区
- **中断类型号** → 查中断向量表 → 得到入口地址 → 跳转执行

```
地址      内容（中断向量）
0x0000    中断0的服务程序入口地址
0x0004    中断1的服务程序入口地址
0x0008    中断2的服务程序入口地址
  ...
```

> CPU收到中断类型号n后，访问地址 **n × 4**（x86）处的中断向量，得到入口地址并跳转。

### 4.6 多重中断

- **单重中断**：中断处理过程中不响应新的中断
- **多重中断（中断嵌套）**：处理过程中可以响应更高优先级的中断

实现多重中断的条件：
1. 保存现场后**开中断**（IF=1）
2. 有**中断屏蔽**机制（中断屏蔽字）

### 4.7 中断屏蔽字

| 含义 | 说明 |
|------|------|
| 屏蔽位 = 1 | 屏蔽该级中断（不响应） |
| 屏蔽位 = 0 | 允许该级中断 |

> 中断源A的屏蔽字 = `1110` 表示：在A的中断处理中，允许中断D（优先级低于A），屏蔽B、C（优先级高于或等于A）。

## 五、DMA方式

### 5.1 DMA的概念

**DMA（Direct Memory Access）**：数据在外设和主存之间直接传送，**不经过CPU**，仅在传送前和传送后需要CPU干预。

### 5.2 DMA控制器组成

```
┌───────────────────────────────────────┐
│            DMA 控制器                  │
│                                       │
│  ┌────┐  ┌────┐  ┌────┐  ┌───────┐  │
│  │MAR │  │ WC │  │ DR │  │控制逻辑│  │
│  │    │  │    │  │    │  │       │  │
│  │地址│  │计数│  │数据│  │状态/   │  │
│  │寄存│  │器  │  │缓冲│  │命令    │  │
│  │器  │  │    │  │    │  │寄存器  │  │
│  └────┘  └────┘  └────┘  └───────┘  │
└───────────────────────────────────────┘
```

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **MAR** | Memory Address Register | 存放主存地址，每传一个字自动 +1 |
| **WC** | Word Count | 存放传送字数，每传一个字自动 -1，减到0时发出中断 |
| **DR** | Data Register | 数据缓冲寄存器，暂存传送数据 |
| **控制逻辑** | - | 控制DMA传送过程、与CPU协调总线使用权 |

### 5.3 DMA传送过程

```
┌─────────────────────────────────────────────┐
│              DMA传送过程                     │
│                                             │
│  ① 预处理（CPU完成）                         │
│     - 向DMA控制器写入：起始地址(MAR)、       │
│       字数(WC)、传送方向、启动命令            │
│                                             │
│  ② 数据传送（DMA控制器完成，CPU暂停/让出总线）│
│     a. 设备读一个字 → DR                     │
│     b. DMA请求总线                           │
│     c. 获得总线后：DR → M[MAR]              │
│     d. MAR + 1, WC - 1                      │
│     e. 若WC ≠ 0，回到a；否则转③             │
│                                             │
│  ③ 后处理                                   │
│     - WC减到0 → DMA向CPU发出中断请求         │
│     - CPU响应中断，做结束处理                 │
└─────────────────────────────────────────────┘
```

### 5.4 CPU与DMA分时使用总线

三种方式：

| 方式 | 说明 |
|------|------|
| **停止CPU法** | DMA传送期间CPU完全停止访存，**CPU利用率最低** |
| **周期挪用法（窃取）** | DMA利用CPU不访存的时钟周期传送数据；若冲突，DMA优先 | 
| **交替访存法** | CPU和DMA交替使用总线周期，不需要总线请求/释放开销 |

> - CPU正在执行指令不访存时 → DMA传送
> - CPU也要访存时 → DMA优先，CPU暂停一个周期
> - 这一个周期被DMA"挪用"了

### 5.5 DMA传送时间计算

> 设主存带宽为 W 字/秒，需传送 N 个字：
> - **停止CPU法**：$T_{DMA} = N \times T_{存取周期}$
> - **周期挪用法**：$T_{DMA} = N \times T_{存取周期}$（但CPU只是偶而被暂停）
>
> **例题**：主存存取周期为1μs，DMA要传送1000个字：
> - 停止CPU法：CPU暂停 $1000 \times 1μs = 1000μs$
> - 周期挪用法：每次挪用1个存取周期（1μs），共1000个周期

## 六、I/O方式对比总结

| 对比项 | 程序查询 | 程序中断 | DMA | 通道 |
|--------|---------|---------|-----|------|
| **数据传送** | CPU | CPU | 硬件（DMA控制器） | 通道处理器 |
| **CPU干预** | 每次传送 | 每次传送 | 仅首尾 | 仅启动和结束 |
| **CPU与外设并行** | ❌ 不能 | ✅ 部分 | ✅ 较好 | ✅ 最好 |
| **数据传送单位** | 字 | 字 | 数据块 | 数据块/一组块 |
| **适用设备** | 低速 | 中低速 | 高速 | 高速 |
| **硬件复杂度** | 低 | 中 | 高 | 最高 |
| **CPU利用率** | 最低 | 中 | 高 | 最高 |
| **传送控制** | 软件（程序） | 软件+硬件 | 硬件 | 通道程序 |

## 七、通道

### 7.1 通道的概念

**通道**是一种特殊的处理器，有自己的指令系统（**通道指令**），能执行通道程序来控制I/O操作。

### 7.2 通道类型

| 对比项 | 字节多路通道 | 数组选择通道 | 数组多路通道 |
|--------|------------|------------|------------|
| **数据单位** | 字节 | 数据块 | 数据块 |
| **共享性** | 多个设备**分时**共享 | 同一时间**只服务一个设备** | 多个设备**分时**共享 |
| **适用设备** | 低速设备（键盘、打印机） | 高速设备（磁盘） | 高速设备（磁盘） |
| **并行性** | 多个设备可交替传送 | 无并行 | 多个设备可交替传送 |
| **效率** | 低速设备利用率高 | 高速设备充分利用 | 综合最优 |

> - **字节多路**：每次传一个字节，多个低速设备分时共享
> - **数组选择**：选定一个高速设备后，独占通道直到传完整块
> - **数组多路**：每次传一个数据块后切换到下一个设备（结合前两者优点）

### 7.3 通道工作过程

```
CPU → 启动通道（送通道程序首地址）
  ↓
通道 → 从主存取通道指令
  ↓
通道 → 执行通道指令，控制设备I/O
  ↓
通道 → 传送结束或异常 → 向CPU发中断
  ↓
CPU → 做结束处理
```

---

> - 端口编址：统一编址用访存指令，独立编址用IN/OUT指令
> - 中断响应三条件：**有请求 + 开中断(IF=1) + 指令执行完**
> - DMA组成：MAR（地址）+ WC（计数）+ DR（数据缓冲）+ 控制逻辑
> - CPU与DMA总线使用：停止CPU法、周期挪用法、交替访存法
> - I/O方式选择：速度从低到高 → 程序查询 → 中断 → DMA → 通道
