## 一、基本概念

### 1.1 计算机系统的定义

**计算机系统** = 硬件 + 软件

- **硬件（Hardware）**：计算机的物理实体，如CPU、存储器、I/O设备等
- **软件（Software）**：各类程序和文档的集合
- **固件（Firmware）**：将软件固化在硬件中（如BIOS），兼具软硬件特征

### 1.2 计算机的层次结构

```
高级语言虚拟机  → 高级语言程序（C/Java）
汇编语言虚拟机  → 汇编语言程序
操作系统虚拟机  → 系统调用
传统机器（ISA层）→ 机器指令  ← 软硬件分界面
微程序机器      → 微指令
硬件逻辑层      → 硬件电路
```

> **关键概念**：ISA（指令集架构）是软硬件的交界面。ISA之上是软件世界，ISA之下是硬件世界。

### 1.3 冯·诺依曼体系结构

**核心思想**：存储程序——将程序和数据以同等地位存储在存储器中。

| 特征 | 具体内容 |
|------|----------|
| ① 五大部件 | 运算器、控制器、存储器、输入设备、输出设备 |
| ② 存储程序 | 指令和数据以二进制形式存储在存储器中 |
| ③ 按地址访问 | 指令和数据按地址访问 |
| ④ 指令格式 | 指令 = 操作码 + 地址码 |
| ⑤ 以运算器为中心 | 早期以运算器为中心（现代已改为以存储器为中心） |

**与哈佛结构的区别**：
- **冯·诺依曼结构**：指令和数据存储在同一个存储器中，通过同一总线传输
- **哈佛结构**：指令和数据分别存储在不同的存储器中，使用独立总线

### 1.4 计算机性能指标

| 指标 | 定义 | 公式 |
|------|------|------|
| **主频（Clock Rate）** | CPU时钟频率，单位Hz/GHz | — |
| **时钟周期** | 主频的倒数 | T = 1/f |
| **CPI** | 每条指令执行所需的时钟周期数 | CPI = 时钟周期数 / 指令数 |
| **MIPS** | 每秒百万条指令 | MIPS = 指令数 / (执行时间 × 10⁶) |
| **CPU执行时间** | 执行程序所需时间 | T_cpu = 指令数 × CPI × 时钟周期 |
| **IPC** | 每个时钟周期执行的指令数 | IPC = 1 / CPI |

## 二、核心知识

### 2.1 冯·诺依曼体系结构详解

**现代计算机结构图（文字描述）**：
```
            ┌─────────────┐
            │   控制器(CU) │ ←── 取指令、分析指令、发出控制信号
            └──────┬──────┘
                   │ 控制信号
      ┌────────────┼────────────┐
      ▼            ▼            ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 运算器(ALU)│ │ 主存储器  │ │ I/O设备  │
└──────────┘ └──────────┘ └──────────┘
```

**以存储器为中心的改进**：将I/O设备直接连接到存储器（通过DMA），避免所有数据都经过运算器。

### 2.2 性能计算方法

**CPU执行时间公式推导**：

```
T_cpu = 程序中指令总数 × CPI × T_clock

其中：
  CPI = Σ(各类指令的CPI_i × 各类指令所占比例_i)

例：若程序中 60%的指令CPI=2, 40%的指令CPI=4
  则平均CPI = 0.6×2 + 0.4×4 = 1.2 + 1.6 = 2.8
```

**MIPS的计算**：

```
MIPS = I_c / (T_cpu × 10⁶) = f / (CPI × 10⁶)

其中 I_c = 指令总数, f = 主频(Hz)
```

**Amdahl定律**（系统优化）：

```
加速比 S = 1 / [(1-f) + f/S_fe]

其中：
  f = 可优化部分占总执行时间的比例
  S_fe = 可优化部分的加速倍数
```

### 2.3 计算机硬件组成

**五大部件功能**：

| 部件 | 功能 |
|------|------|
| **运算器（ALU）** | 执行算术运算和逻辑运算 |
| **控制器（CU）** | 取指令、分析指令、执行指令 |
| **主存储器** | 存储程序和数据 |
| **输入设备** | 将外部信息转换为机器能识别的形式 |
| **输出设备** | 将机器处理结果转换为人能理解的形式 |

> **MAR（地址寄存器）**：存放要访问的存储单元地址
> **MDR（数据寄存器）**：存放从存储器读出或要写入的数据
> MAR的位数决定寻址空间，MDR的位数等于存储字长

## 三、深入理解

### 3.1 为什么冯·诺依曼结构要"以存储器为中心"？

- **原始结构**：所有数据I/O都经过运算器→运算器成为瓶颈
- **改进结构**：I/O设备直接与主存交互→提高系统效率
- 现代计算机的DMA技术正是这一思想的体现

### 3.2 CPI与MIPS的局限性

| 指标 | 局限性 |
|------|--------|
| CPI | 不同指令集的CPI无法直接比较 |
| MIPS | 不同指令集的MIPS值不可比；同一计算机执行不同程序MIPS也不同 |
| MFLOPS | 仅适用于浮点运算，不适用于整数程序 |

> **最可靠的性能指标是程序的执行时间**（benchmark），而非任何单一指标。

### 3.3 固件与微程序

固件是"用软件方法实现硬件功能"的典型。微程序控制器中，每条机器指令对应一段微程序，微程序存储在控制存储器中，这就是固件。

## 五、典型例题

### 例题1：概念辨析（选择题）

**题目**：下列关于冯·诺依曼计算机工作方式的描述中，正确的是（）
A. 控制器从存储器中取出指令并执行
B. 指令和数据必须存放在不同的存储器中
C. 程序运行前，指令和数据均存储在存储器中
D. 指令和数据以十进制形式存储

**思路**：逐一分析每个选项是否符合冯·诺依曼体系结构的特征。

**详细解答**：
- A：正确但不完整，控制器取指→译码→执行
- B：错误。冯·诺依曼结构中指令和数据存放在**同一**存储器中（哈佛结构才分开）
- C：✅ 正确。这是"存储程序"的核心思想——运行前全部存入
- D：错误。存储的是**二进制**形式

**答案**：C

**总结**：冯·诺依曼的核心特征是"存储程序"——程序和数据以二进制形式存储在同一存储器中。

---

### 例题2：计算题

**题目**：某计算机主频为500MHz，程序A包含指令1000万条，其中指令类甲占60%（CPI=2），指令类乙占30%（CPI=3），指令类丙占10%（CPI=5）。求：
（1）程序A的平均CPI
（2）程序A的CPU执行时间
（3）若将丙类指令优化到CPI=3，求加速比

**思路**：先求加权平均CPI，再用公式 T = n × CPI × T_clock。

**详细解答**：

**（1）平均CPI**：
```
CPI_avg = 0.6 × 2 + 0.3 × 3 + 0.1 × 5
        = 1.2 + 0.9 + 0.5 = 2.6
```

**（2）CPU执行时间**：
```
T_clock = 1/500MHz = 2 × 10⁻⁹ s = 2ns
T_cpu = 10⁷ × 2.6 × 2 × 10⁻⁹ = 0.052 s = 52ms
```

**（3）Amdahl定律求加速比**：
```
优化后 CPI_new = 0.6×2 + 0.3×3 + 0.1×3 = 1.2+0.9+0.3 = 2.4
加速比 = T_old / T_new = CPI_old / CPI_new = 2.6/2.4 ≈ 1.083
```

**总结**：性能优化要看"瓶颈在哪里"，丙类只占10%，即使优化很多，整体提升也有限。

### 例题3：综合分析题

**题目**：某机器主频为1GHz，现有两种方案：
- 方案A：CPI=2，需要执行1.0×10⁹条指令
- 方案B：CPI=4，需要执行0.5×10⁹条指令

问：哪种方案更快？快多少？

**思路**：分别计算两种方案的执行时间，比较大小。

**详细解答**：
```
T_A = 1.0×10⁹ × 2 × (1/10⁹) = 2.0 s
T_B = 0.5×10⁹ × 4 × (1/10⁹) = 2.0 s
```
两者执行时间相同，性能相同。

**总结**：不能只看CPI或指令数，必须看最终执行时间。CPI小但指令多、CPI大但指令少，可能性能一样。

---

### 例题4：历年真题

**题目（2015年408）**：计算机硬件能够直接执行的是（）
Ⅰ. 机器语言程序  Ⅱ. 汇编语言程序  Ⅲ. 硬件描述语言程序

A. 仅Ⅰ  B. 仅Ⅰ和Ⅱ  C. 仅Ⅰ和Ⅲ  D. 全部

**思路**：硬件直接执行的只有机器语言。汇编语言需要汇编器翻译，HDL需要综合/编译。

**详细解答**：答案 **A**。只有机器语言（二进制指令）能被CPU直接执行。

**总结**：ISA层是软硬件分界面，其上的语言都需要翻译。

## 六、易错点

1. ❌ "冯·诺依曼结构以运算器为中心" → ✅ 早期是，现代已改为以**存储器**为中心
2. ❌ MIPS值越大说明计算机越快 → ✅ 不同指令集的MIPS不可比
3. ❌ CPI是固定的 → ✅ 不同指令的CPI不同，需要加权平均
4. ❌ 主频高的CPU一定更快 → ✅ 还取决于CPI和指令数
5. ❌ 固件就是硬件 → ✅ 固件是软件固化在硬件中，兼具两者特征

### 计算机发展历程补充

#### 计算机硬件发展四代

| 代次 | 时间 | 逻辑元件 | 速度(次/秒) | 内存 | 外存 |
|------|------|----------|-------------|------|------|
| 第一代 | 1946-1957 | **电子管** | 几千~几万 | 汞延迟线、磁鼓 | 穿孔卡片、纸带 |
| 第二代 | 1958-1964 | **晶体管** | 几万~几十万 | 磁芯存储器 | 磁带 |
| 第三代 | 1964-1971 | **中小规模集成电路** | 几十万~几百万 | 半导体存储器 | 磁带、磁盘 |
| 第四代 | 1972-至今 | **大规模/超大规模集成电路** | 上千万~万亿 | 半导体存储器 | 磁盘、光盘、半导体存储器 |

**关键历史节点**：
- **ENIAC（1946）**：第一台电子数字计算机，占地约170平方米，耗电量150千瓦，运算速度5000次加法/秒，使用**电子管**作为逻辑元件
- **EDVAC**：第一台采用**冯·诺依曼结构**的计算机
- **摩尔定律**：集成电路上可容纳的晶体管数目约每隔**18个月**增加一倍，整体性能也提升一倍

#### 微处理器发展里程碑

| 微处理器 | 机器字长 | 年份 | 晶体管数目 |
|----------|----------|------|-----------|
| 8080 | 8位 | 1974 | — |
| 8086 | 16位 | 1979 | 2.9万 |
| 80386 | 32位 | 1985 | 27.5万 |
| Pentium | 64位 | 1993 | 310万 |
| Pentium IV | 64位 | 2000 | 4200万 |

> 微型计算机的发展以**微处理器技术**为标志。机器字长是计算机一次整数运算所能处理的二进制位数。

### 冯·诺依曼结构深层理解

#### 早期冯·诺依曼机的6个特征

1. 计算机由**五大部件**组成（运算器、控制器、存储器、输入设备、输出设备）
2. 指令和数据以**同等地位**存于存储器，可按地址寻访
3. 指令和数据用**二进制**表示
4. 指令由**操作码 + 地址码**组成
5. **存储程序**（核心思想）
6. 以**运算器**为中心（早期特征）

> 在计算机系统中，软件和硬件在**逻辑上是等效的**。例如：对于乘法运算，可以设计专门的硬件电路实现，也可以用软件方式执行多次加法来实现。

#### 现代计算机结构改进

```
早期：以运算器为中心 → 所有数据I/O都经过运算器 → 运算器成为瓶颈
现代：以存储器为中心 → I/O设备直接与主存交互 → 提高系统效率
```

**现代计算机层次**：
```
CPU = 运算器 + 控制器
主机 = CPU + 主存储器
I/O设备（外设）= 输入设备 + 输出设备
```

### 主存储器内部结构详解

#### 存储器核心组件

| 组件 | 全称 | 功能 | 类比 |
|------|------|------|------|
| **存储体** | — | 存放二进制数据 | 货架（放包裹） |
| **MAR** | Memory Address Register | 存放要访问的存储单元地址 | 取件号（告诉店员取哪个） |
| **MDR** | Memory Data Register | 存放从存储器读出或要写入的数据 | 柜台（传递包裹的地方） |

#### 存储相关概念

| 概念 | 定义 |
|------|------|
| **存储单元** | 每个存储单元存放一串二进制代码 |
| **存储字（word）** | 存储单元中二进制代码的组合 |
| **存储字长** | 存储单元中二进制代码的位数 |
| **存储元** | 存储二进制的电子元件，每个存储元可存 1 bit |

> - MAR的位数决定**寻址范围**：MAR = n 位 → 最多有 $2^n$ 个存储单元
> - MDR的位数 = **存储字长**：MDR = m 位 → 每个存储单元存放 m 位数据
> - 存储容量 = $2^n \times m$ 位
> - 例：MAR = 4位，MDR = 16位 → 存储容量 = $2^4 \times 16 = 256$ bit

#### 计算机硬件工作原理概述

**数据通路**：
```
输入设备 → 存储器 → 运算器 → 存储器 → 输出设备
                ↑
            控制器（指挥各部件协调工作）
```

**控制器的功能**：
- 取指令：从存储器中取出指令
- 分析指令：对指令进行译码
- 执行指令：发出控制信号，指挥各部件执行操作

**运算器的功能**：
- 执行**算术运算**（加减乘除）
- 执行**逻辑运算**（与或非）
- 包含ALU（算术逻辑单元）、通用寄存器、状态寄存器等

### 软件发展概述

```
机器语言 → 汇编语言 → 高级语言（FORTRAN → PASCAL → C++ → Java）
操作系统：DOS → Windows → macOS/Linux
```

**软件分类**：
- **系统软件**：操作系统、DBMS、标准程序库、网络软件、语言处理程序
- **应用软件**：用户使用的各种应用程序（如QQ、游戏等）

> **计算机性能的好坏取决于"软"、"硬"件功能的总和。**
