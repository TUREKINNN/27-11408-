# 7.3.3 DMA 方式

## 一、DMA 控制器的基本概念

**DMA（Direct Memory Access）**：主存和高速 I/O 设备之间有一条**直接数据通路**，数据传送不经过 CPU。

### DMA 控制器的角色
- DMA 控制器是一种**特殊的 I/O 控制器**
- CPU 向 DMA 控制器指明：输入/输出方向、传送数据量、主存地址、外设地址

### 传送过程
1. **传送前**：CPU 进行预处理（设置参数）
2. **传送时**：DMA 控制器自动控制，每次传送 1 个字
3. **传送后**：一整块数据传送完毕后，向 CPU 发中断请求，CPU 进行后处理

## 二、DMA 控制器的组成

| 部件 | 功能 |
|------|------|
| **主存地址寄存器 AR** | 存放要交换数据的主存地址 |
| **传送长度计数器 WC** | 记录传送数据长度，溢出时表示传送完毕，自动发中断请求 |
| **数据缓冲寄存器 DR** | 暂存每次传送的数据 |
| **DMA 请求触发器** | I/O 设备准备好数据后置位，向 DMA 控制器发送请求 |
| **控制/状态逻辑** | 指定传送方向，修改传送参数，协调 DMA 请求和 CPU 响应 |

> DMA 传送过程中，DMA 控制器**接管 CPU 的地址总线、数据总线和控制总线**，CPU 的主存控制信号被禁止。传送结束后恢复 CPU 一切权利。

## 三、DMA 传送过程（以数据输入为例）

```
① I/O 设备准备好数据 → 写入数据缓冲寄存器 DR
② DR 写满 → 向 DMA 控制器发送 DMA 请求
③ DMA 控制器向总线发送总线请求
④ CPU 响应 → 将总线控制权交给 DMA 控制器
⑤ DMA 控制器接管总线，完成一次数据传送
⑥ 修改主存地址和传送计数
⑦ 全部传送完毕 → 发出中断请求 → CPU 后处理
```

## 四、DMA 传送方式

当 CPU 和 DMA 控制器同时访问主存时可能冲突，有三种解决方法：

### 1. 停止 CPU 访存
- 控制简单
- CPU 处于不工作/保持状态
- **缺点**：未充分发挥 CPU 对主存的利用率

### 2. DMA 与 CPU 交替访存
- 将一个 CPU 周期分为 C1 和 C2 两个子周期
- C1 专供 DMA 访存，C2 专供 CPU 访存
- **优点**：不需要总线使用权的申请、建立和归还过程
- **缺点**：硬件逻辑更复杂

### 3. 周期挪用（周期窃取）
- DMA 访问主存有三种可能情况：
  1. CPU 此时不访存（不冲突）
  2. CPU 正在访存（存取周期结束后让出总线）
  3. CPU 与 DMA 同时请求访存（**I/O 访存优先**）

## 五、DMA 方式的特点

1. 主存与 CPU 的固定联系**脱钩**，主存既可被 CPU 访问，又可被外设访问
2. 数据块传送时，主存地址确定、传送计数等都由**硬件电路直接实现**
3. 主存中要开辟**专用缓冲区**，及时供给和接收外设数据
4. DMA 传送速度快，CPU 和外设**并行工作**，提高系统效率
5. DMA 在传送前要通过**程序进行预处理**，结束后要通过**中断方式进行后处理**

## 六、DMA 方式与中断方式对比

| 对比项 | 中断方式 | DMA 方式 |
|--------|----------|----------|
| **数据传送** | 程序控制（程序切换 + 保存恢复现场 + 传送） | 硬件控制（CPU 只需预处理和后处理） |
| **响应时机** | 指令执行周期结束后响应 | 每个机器周期结束均可，总线空闲时即可响应 |
| **适用场景** | CPU 控制，低速设备 | DMA 控制器控制，高速设备 |
| **优先级** | 低于 DMA | 高于中断 |
| **异常处理** | 能处理异常事件 | 仅传送数据 |
