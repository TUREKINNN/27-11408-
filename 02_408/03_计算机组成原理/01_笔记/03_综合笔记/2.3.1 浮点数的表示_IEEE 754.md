# 浮点数的表示 — IEEE 754 标准

## 一、从科学计数法理解浮点数

### 1.1 浮点数的基本概念

$$
N = (-1)^S \times M \times R^E
$$

| 符号 | 含义 |
|------|------|
| $S$ | 符号（0 正 1 负） |
| $M$ | 尾数（Mantissa），影响精度 |
| $E$ | 阶码（Exponent），反映小数点的实际位置 |
| $R$ | 基数（Radix），二进制中 R = 2 |

**规格化**：确保尾数的最高位非 0 数位刚好在小数点之前（二进制中即小数点前为 1）。

### 1.2 定点数的局限性

- 定点整数和定点小数的表示范围有限
- 机器字长位数有限，不能无限制增加数据长度
- 浮点数在**位数不变**的情况下可以大幅增加数据表示范围

## 二、IEEE 754 标准格式 ⭐⭐⭐⭐⭐

### 2.1 浮点数格式

IEEE 754 标准定义了以下浮点数格式：

| 类型 | 符号 | 阶码 | 尾数 | 总位数 | C语言类型 |
|------|------|------|------|--------|----------|
| 单精度 | 1 bit | 8 bit | 23 bit | 32 bit | `float` |
| 双精度 | 1 bit | 11 bit | 52 bit | 64 bit | `double` |
| 扩展精度 | 1 bit | 15 bit | 64 bit | 80 bit | `long double` |

### 2.2 float 单精度浮点型的存储格式

```
| 1 bit | 8 bit | 23 bit |
| 符号S | 阶码E | 尾数M  |
```

**存储规则：**

1. **符号位 S**：0 表示正，1 表示负
2. **尾数 M**：规定小数点位置在 23 bit 之前。默认存储**规格化尾数**，小数点前的 **1 省略（隐含）**，这样增加了 1 bit 的实际精度
3. **阶码 E**：用**移码**表示，规定**偏置值为 127**（即 Bias = 127 = $2^7 - 1$）
4. **基数 R**：不需要存储，规定为 2

### 2.3 阶码的移码表示

**将十进制真值转换为偏置值为 M 的移码：**
1. 将十进制真值 + 偏置值
2. 按"无符号整数"规则转换为指定位数

**示例：** 阶码真值 = +2

$$
E = 2 + 127 = 129 = 10000001_B
$$

### 2.4 double 双精度浮点型

```
| 1 bit | 11 bit | 52 bit |
| 符号S | 阶码E  | 尾数M  |
```

- 偏置值为 **1023**（即 $2^{10} - 1$）
- 阶码真值 = +2 → $E = 2 + 1023 = 1025 = 10000000001_B$

## 三、IEEE 754 完整推导 ⭐⭐⭐⭐⭐

### 3.1 真值转换为 IEEE 754 格式

**步骤：**
1. 将真值转换为二进制
2. 规格化为 $1.xxxxx \times 2^E$ 的形式
3. 确定符号位 S
4. 计算阶码 $E_{存储} = E_{真值} + 127$（float）或 $+ 1023$（double）
5. 尾数去掉隐含的 1，只存储小数部分

**示例 1：** 将 $-6.75$ 转换为 float 格式

**Step 1**：转为二进制
$$
-6.75 = -110.11_B
$$

**Step 2**：规格化
$$
-110.11_B = -1.1011 \times 2^2
$$

**Step 3**：确定各字段
- $S = 1$（负数）
- $E_{真值} = 2$，$E_{存储} = 2 + 127 = 129 = 10000001_B$
- 尾数（去掉隐含的 1）：$10110000000000000000000$

**Step 4**：组合

```
1 10000001 10110000000000000000000
S 阶码E    尾数M（23位）
```

### 3.2 IEEE 754 格式转换为真值

**步骤：**
1. 提取 S, E, M
2. 还原阶码真值：$E_{真值} = E_{存储} - 127$
3. 还原尾数（加上隐含的 1）：$1.M$
4. 计算真值：$(-1)^S \times 1.M \times 2^{E_{真值}}$

**示例 2：** 将 `0 10000000 10000000000000000000000` 转换为真值

- $S = 0$（正数）
- $E_{存储} = 128$，$E_{真值} = 128 - 127 = 1$
- 尾数：$1.1000...0 = 1.1_B = 1.5$

$$
N = (+1) \times 1.5 \times 2^1 = 3.0
$$

## 四、IEEE 754 的特殊值

### 4.1 规格化数

- 阶码 E 不全为 0 且不全为 1
- 真值：$(-1)^S \times 1.M \times 2^{E-Bias}$

### 4.2 非规格化数（阶码全 0）

- 阶码 E 全为 0，尾数 M 不全为 0
- 真值：$(-1)^S \times 0.M \times 2^{1-Bias}$（注意隐含的是 0 而不是 1）
- 表示**非常接近 0** 的数（次正规数）

### 4.3 零

- 阶码 E 全为 0，尾数 M 全为 0
- $+0$：`0 00000000 000...0`
- $-0$：`1 00000000 000...0`

### 4.4 无穷大

- 阶码 E 全为 1，尾数 M 全为 0
- $+\infty$：`0 11111111 000...0`
- $-\infty$：`1 11111111 000...0`

### 4.5 NaN（非数）

- 阶码 E 全为 1，尾数 M 不全为 0
- 表示"不是一个数"（如 $0/0$, $\infty - \infty$）

---

## 五、float 的表示范围与精度

### 5.1 表示范围

- **最小规格化正数**：$1.0 \times 2^{-126} \approx 1.175 \times 10^{-38}$
- **最大规格化正数**：$(2 - 2^{-23}) \times 2^{127} \approx 3.403 \times 10^{38}$

### 5.2 精度

- 尾数 23 位 + 1 位隐含 = 24 位有效二进制位
- 约 $24 \times \log_{10}2 \approx 7.22$ 位十进制有效数字

### 5.3 阶码范围

- 存储值范围：$1 \sim 254$（0 和 255 用于特殊值）
- 真值范围：$-126 \sim +127$

## 六、408 考点总结

1. **IEEE 754 格式的结构**：1 + 8 + 23（float）或 1 + 11 + 52（double）
2. **偏置值**：float = 127，double = 1023
3. **隐含位**：规格化数隐含 1，非规格化数隐含 0
4. **真值与机器数的互相转换**是必考计算题
5. **特殊值**（0, ∞, NaN）的判别
6. 阶码全 0 和全 1 有特殊含义
7. 尾数的隐含存储增加了 1 bit 精度
