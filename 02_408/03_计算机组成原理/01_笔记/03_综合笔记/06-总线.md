# 06 总线

> **选择题**常考各仲裁方式的特点对比（控制线数、优先级、灵活性）。

---

## 一、总线的定义与分类

### 1.1 总线的定义

**总线**是一组能为多个部件**分时共享**的公共信息传送线路。

- **分时**：同一时刻只能有一个部件发送数据到总线上
- **共享**：总线上可以挂接多个部件，各部件通过总线交换信息

### 1.2 总线的分类

#### 按功能（信号类型）分

| 类型 | 作用 | 宽度 |
|------|------|------|
| **数据总线** | 传送数据 | 与机器字长/数据宽度有关 |
| **地址总线** | 传送地址 | 决定寻址空间（n位→2ⁿ个地址） |
| **控制总线** | 传送控制信号和时序信号 | 各种独立的控制线 |

#### 按位置分

| 类型 | 连接部件 | 说明 |
|------|----------|------|
| **CPU内部总线** | CPU内部各寄存器和ALU | 速度最快 |
| **系统总线** | CPU、主存、I/O接口 | 通常所说的"总线" |
| **I/O总线** | 各I/O设备接口 | 如PCI、USB |

#### 系统总线按信号类型细分

```
系统总线
├── 数据总线（Data Bus, DB）：双向
├── 地址总线（Address Bus, AB）：单向，CPU→外
└── 控制总线（Control Bus, CB）：每根线单向，总体双向
```

### 1.3 总线结构

```
┌──────────┐    系统总线    ┌──────────┐
│   CPU    │──┬────────┬──│ 主存(MM) │
│          │  │数据总线 │  │          │
│  ┌───┐  │  │地址总线 │  └──────────┘
│  │CU │  │  │控制总线 │
│  └───┘  │  └────┬───┘
└──────────┘       │
            ┌──────┴──────┐
            │  I/O接口    │
            │ ┌───┐ ┌───┐ │
            │ │网卡│ │显卡│ │
            │ └───┘ └───┘ │
            └─────────────┘
```

## 二、总线仲裁

### 2.1 为什么需要仲裁

总线是共享资源，多个设备可能同时请求使用总线。**总线仲裁器（Bus Arbiter）**根据优先级决定哪个设备获得总线控制权。

### 2.2 四种仲裁方式

#### 链式查询（Daisy Chain）

```
        ┌───────┐   BR（总线请求）
        │ 总线  │←──────────────────┐
        │ 仲裁器│                   │
        │       │←──┐  ┌──┐  ┌──┐  │
        └──┬────┘   │  │D1│  │D2│  │
     BG ───┤        └──┤  ├──┤  ├──┘
           ↓          └──┘  └──┘
        ┌──────┐   ←──── BS（总线忙）
        │  D0  │
        └──────┘
```

- **BR**（总线请求）：所有设备共用一条请求线，向仲裁器发出请求
- **BG**（总线同意）：串行传递，从优先级最高的设备开始逐个查询
- **BS**（总线忙）：获得使用权的设备置位

#### 计数器定时查询

```
        ┌───────┐   BR
        │ 总线  │←──────────┐
        │ 仲裁器│            │
        │  计数器│   设备地址  │
        └──┬────┘────────────┘
           ↓       ↑设备地址线
    ┌──┐ ┌──┐ ┌──┐
    │D0│ │D1│ │D2│
    └──┘ └──┘ └──┘
         ←── BS
```

- **设备地址线**：代替BG线，仲裁器通过设备地址线发送计数值
- 设备将自己的编号与总线上的值比较，匹配则获得总线

#### 独立请求方式

```
        ┌───────┐
  BR0 ──→│       │←── BR1
  BR2 ──→│ 总线  │←── BR3
        │ 仲裁器│
  BG0 ←──│       │──→ BG1
  BG2 ←──│       │──→ BG3
        └───────┘
         ←── BS
```

- 每个设备有**独立的请求线和同意线**
- 仲裁器内部用优先级编码器选择

#### 分布式仲裁

- **无中央仲裁器**，每个设备自行判断优先级
- 各设备将自己的优先级发到总线上，竞争失败的自动退出

### 2.3 仲裁方式对比表

| 对比项 | 链式查询 | 计数器定时查询 | 独立请求 | 分布式仲裁 |
|--------|---------|-------------|---------|-----------|
| **控制线数** | **3**（BR+BG+BS） | **2+⌈log₂N⌉**（BR+BS+设备地址线） | **2N+1**（N个BR+N个BG+BS） | 各实现不同 |
| **优先级** | 固定（离仲裁器近的优先级高） | 灵活（计数器起始值可变） | 灵活（硬件编程） | 灵活 |
| **设备增减** | 容易，直接串接 | 较容易 | 困难，需增加控制线 | 较容易 |
| **故障敏感性** | 高（BG链路断则后续设备不可用） | 低 | 低 | 低 |
| **速度** | 慢（串行传递） | 中等 | 快（并行判断） | 快 |

> - 链式查询只需 **3根** 控制线，最少
> - 独立请求灵活性最好、速度最快，但控制线最多（**2N+1**）
> - 链式查询的缺点：对电路故障敏感，低优先级设备可能"饥饿"

## 三、总线事务与定时

### 3.1 总线事务

一次总线事务包括四个阶段：

1. **申请分配**：设备发出请求，仲裁获得使用权
2. **寻址**：获得使用权的设备通过总线发送地址和命令
3. **传数**：数据通过数据总线传送
4. **结束**：释放总线，让出控制权

### 3.2 同步定时

- 使用**统一的时钟信号**协调总线操作
- 所有事件在时钟信号的固定时刻发生
- **优点**：速度快，逻辑简单
- **缺点**：灵活性差，不能适应速度差异大的设备

```
时钟  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
     ─┘  └──┘  └──┘  └──┘  └──┘  └──

地址  ──┤ 地址有效  ├──┤  ──────────────
数据  ──┤          ├──┤ 数据有效  ├────
读/写 ──┤ 命令有效  ├──┤          ├────
```

### 3.3 异步定时

不使用统一时钟，采用**握手（Handshaking）**协议协调通信双方。

#### 全互锁握手过程

```
主模块                    从模块
  │                         │
  │─── 请求(REQ) ─────────→│
  │                         │ (准备数据)
  │                         │
  │←── 应答(ACK) ──────────│
  │ (撤销请求)               │ (撤销应答)
  │                         │
  │─── 撤销REQ ────────────→│
  │                         │
  │←── 撤销ACK ────────────│
  │    (一次传送完成)         │
```

**全互锁**特点：
- 主模块发出REQ → 从模块检测到后发出ACK → 主模块检测到ACK后撤销REQ → 从模块检测到REQ撤销后撤销ACK
- **每一步都必须等对方确认**，可靠性最高但速度最慢

#### 三种互锁方式对比

| 方式 | 特点 | 速度 |
|------|------|------|
| **全互锁** | 请求和应答完全互锁，最可靠 | 最慢 |
| **半互锁** | 只要求一方互锁 | 中等 |
| **不互锁** | 请求和应答有固定时间间隔 | 最快，但不安全 |

### 3.4 半同步定时

- 结合同步和异步的优点
- 以时钟为基准，但增加**WAIT信号**（从模块来不及响应时插入等待周期）

## 四、总线带宽

### 4.1 总线带宽公式

$$BW = \frac{W}{T}$$

其中：
- **BW**：总线带宽（Bus Bandwidth），单位为 B/s（字节/秒）
- **W**：总线数据传输量（字节）
- **T**：传输所用时间

### 4.2 总线带宽计算

对于同步总线：

$$BW = \frac{总线宽度（位）}{时钟周期} \times \frac{1}{8} \text{ (B/s)}$$

或

$$BW = \frac{总线宽度（字节）}{时钟周期} \text{ (B/s)}$$

### 4.3 影响总线带宽的因素

| 因素 | 影响 |
|------|------|
| **总线宽度** | 越宽，每周期传输数据越多 |
| **时钟频率** | 越高，单位时间内传输次数越多 |
| **传输方式** | 突发传输（burst）可减少地址开销 |
| **信号线数量** | 地址/数据是否复用 |

> 某总线宽度为32位，时钟频率为100MHz：
> - $BW = 32/8 \times 100 \times 10^6 = 400$ MB/s

## 五、总线标准（了解）

| 总线标准 | 特点 |
|----------|------|
| **ISA** | 早期PC总线，16位 |
| **PCI** | 并行总线，支持即插即用 |
| **PCIe** | 串行、点对点、多通道（lanes），目前主流 |
| **USB** | 通用串行总线，支持热插拔 |
| **SATA** | 串行硬盘接口 |

---

> - 总线三要素：**分时共享**，包含数据/地址/控制三类信号线
> - 仲裁方式对比：链式查询（3线）→ 计数器（2+log₂N线）→ 独立请求（2N+1线）→ 分布式
> - 同步定时靠**时钟**，异步定时靠**握手**
> - 带宽公式：$BW = W/T$
