# 5.2 指令执行过程

## 一、指令周期

**指令周期**：CPU 从主存中每取出并执行一条指令所需的全部时间。

### 层次结构
```
指令周期 = 若干机器周期（CPU周期）
机器周期 = 若干时钟周期（T周期/CPU时钟周期，CPU操作的最基本单位）
```

> 每个指令周期内机器周期数可以不等，每个机器周期内的节拍数也可以不等。

## 二、指令周期的四个阶段

| 周期 | 触发器 | 访存目的 |
|------|--------|----------|
| **取指周期 FE** | FE=1 | 取指令 |
| **间址周期 IND** | IND=1 | 取有效地址 |
| **执行周期 EX** | EX=1 | 取操作数 |
| **中断周期 INT** | INT=1 | 保存程序断点 |

### 不同指令包含的周期
| 指令类型 | 包含的周期 |
|----------|-----------|
| 空指令 NOP | 取指周期 |
| 加法指令 ADD | 取指周期 + 执行周期 |
| 乘法指令 MUL | 取指周期 + 执行周期 |
| 间接寻址指令 | 取指 + 间址 + 执行 |
| 带中断的指令 | 取指 + [间址] + 执行 + 中断 |

## 三、各周期的数据流

### 1. 取指周期
```
1. (PC) → MAR           # 当前指令地址送至 MAR
2. 1 → R                # CU 发出读信号
3. M(MAR) → MDR         # 从主存取出指令到 MDR
4. (MDR) → IR           # 指令送入 IR
5. (PC)+1 → PC          # PC 指向下一条指令
```

### 2. 间址周期
```
1. Ad(IR) → MAR         # 指令地址码送入 MAR
2. 1 → R                # CU 发出读信号
3. M(MAR) → MDR         # 从主存取出有效地址
4. (MDR) → Ad(IR)       # 有效地址送回指令地址码字段
```

### 3. 执行周期
- 根据 IR 中的操作码和操作数，通过 ALU 产生执行结果
- **不同指令的执行周期操作不同**，没有统一的数据流向

### 4. 中断周期
```
1. (SP)-1 → SP, (SP) → MAR   # 保存断点地址（堆栈方式）
   或 a → MAR                  # a 为断点保存地址
2. 1 → W                       # CU 发出写信号
3. (PC) → MDR                  # 断点（PC 内容）送入 MDR
4. 向量地址 → PC                # 中断服务程序入口地址送入 PC
```

## 四、指令执行方案

| 方案 | 特点 |
|------|------|
| **单指令周期** | 所有指令用相同执行时间；串行执行；取决于最慢指令 |
| **多指令周期** | 不同类型指令用不同执行步骤；串行执行；需要更复杂硬件 |
| **流水线方案** | 每个时钟周期启动一条指令；**并行执行**；效率最高 |
