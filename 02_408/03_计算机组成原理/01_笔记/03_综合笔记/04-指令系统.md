## 一、基本概念

### 1.1 指令的基本格式

```
指令 = 操作码 + 地址码（0~3个）

三地址指令：OP  A1  A2  A3    → A1 op A2 → A3
二地址指令：OP  A1  A2        → A1 op A2 → A1
一地址指令：OP  A             → ACC op A → ACC（或单操作数指令）
零地址指令：OP                → 如停机指令、空操作
```

**定长操作码**：所有指令的操作码位数相同，译码简单但空间利用率低。
**扩展操作码**：不同指令的操作码位数不同，空间利用率高但译码复杂。

**扩展操作码的计算**：
```
若k种地址数，n条指令：
  n条三地址指令用了 n 个编码 → 剩余 (2^操作码位数 - n) 个
  剩余编码分配给二地址指令，以此类推
```

### 1.2 寻址方式

| 寻址方式 | 有效地址 | 特点 |
|----------|----------|------|
| **立即寻址** | 操作数在指令中（无EA） | 速度快，但操作数固定 |
| **直接寻址** | EA = A（地址字段即有效地址） | 简单，但寻址范围受地址位数限制 |
| **间接寻址** | EA = (A)（地址字段指向的单元存放EA） | 寻址范围大，但访存次数多 |
| **寄存器寻址** | 操作数在寄存器中 | 速度最快 |
| **寄存器间接寻址** | EA = (R)（寄存器中存地址） | 灵活，比间接寻址快 |
| **偏移寻址** | EA = A + (R) | 变址/基址/相对寻址的统一描述 |
| **基址寻址** | EA = (BR) + A | 面向系统，支持程序重定位 |
| **变址寻址** | EA = (IX) + A | 面向用户，适合数组访问 |
| **相对寻址** | EA = (PC) + A | 程序内部跳转 |
| **堆栈寻址** | 操作数在栈顶 | 隐含寻址，支持表达式计算 |

### 1.3 CISC vs RISC

| 特性 | CISC | RISC |
|------|------|------|
| 指令数量 | 多（200+条） | 少（通常<100条） |
| 指令长度 | 变长 | 定长 |
| 寻址方式 | 多种 | 少，主要是Load/Store |
| 访存方式 | 任何指令都可访存 | 只有Load/Store指令访存 |
| 流水线 | 难以实现高效流水线 | 天然适合流水线 |
| 控制方式 | 微程序控制为主 | 硬布线控制为主 |
| 代表 | x86 | ARM, MIPS, RISC-V |

## 二、核心知识

### 2.1 扩展操作码设计

**设计原则**：
```
1. 短操作码不能是长操作码的前缀（前缀码性质）
2. 用操作码的全1编码作为扩展标志

例：4位操作码
  0000~1110 → 15条零地址指令（留1111作为扩展标志）
  1111 0000~1111 1110 → 15条一地址指令
  1111 1111 0000~... → 继续扩展
```

**指令字长的确定**：
```
操作码位数 ≥ ⌈log₂(指令条数)⌉
地址码总位数 = 地址码个数 × 每个地址的位数
指令字长 = 操作码位数 + 地址码总位数
```

### 2.2 各寻址方式详解

**基址寻址 vs 变址寻址**：

| 对比 | 基址寻址 | 变址寻址 |
|------|----------|----------|
| 基址/变址寄存器内容 | 由操作系统设定，用户不可改变 | 由用户设定 |
| 用途 | 程序重定位、多道程序 | 数组遍历、字符串处理 |
| 指令中地址码的含义 | 偏移量（较短） | 基地址（程序起始地址） |

**数组访问示例**：
```
假设数组A从地址1000开始，每个元素4字节

基址寻址：BR=1000, A=i×4 → 访问A[i]
变址寻址：IX=i×4, A=1000 → 访问A[i]

变址寻址更适合遍历：IX从0递增，每次+4，A固定为1000
```

### 2.3 指令执行过程

```
取指周期：
  1. PC → MAR → 地址总线 → 存储器
  2. 存储器 → 数据总线 → MDR → IR
  3. PC + 1 → PC（或PC + 指令长度）

间址周期（如有间接寻址）：
  1. IR中的地址码 → MAR → 存储器
  2. 取出有效地址 → MDR → MAR

执行周期：
  根据操作码执行相应操作

中断周期（如有中断）：
  1. 关中断
  2. 保存PC到堆栈
  3. 将中断服务程序入口地址 → PC
```

### 2.4 数据对齐

```
字节对齐：
  - 1字节数据：任意地址
  - 2字节数据：起始地址为2的倍数
  - 4字节数据：起始地址为4的倍数
  - 8字节数据：起始地址为8的倍数

未对齐访问可能导致：
  - 需要两次访存（数据跨Cache行）
  - 某些架构直接报错
```

## 三、深入理解

### 3.1 为什么RISC要用定长指令？

- **定长指令**：每条指令长度相同 → 取指、译码简单 → 流水线效率高
- **变长指令**：长度不同 → 取指边界不确定 → 流水线气泡

### 3.2 Load/Store架构的优势

RISC中只有Load/Store指令可以访存，ALU指令只能操作寄存器。这样：
- 简化了指令格式（寄存器地址只需3~5位）
- 减少了访存次数（寄存器访问只需1个时钟周期）
- 便于编译器优化

### 3.3 寻址方式对指令系统设计的影响

寻址方式越多 → 指令越灵活 → 但硬件越复杂 → 指令长度越长
寻址方式越少 → 指令越简单 → 硬件越简洁 → 需要更多指令完成同一任务

## 五、典型例题

### 例题1：概念辨析（选择题）

**题目**：以下关于RISC的说法中，错误的是（）
A. RISC通常采用定长指令
B. RISC通常采用硬布线控制器
C. RISC的寻址方式种类多于CISC
D. RISC适合采用流水线技术

**详细解答**：
- A：✅ 正确
- B：✅ 正确
- C：❌ RISC寻址方式**少于**CISC
- D：✅ 正确

**答案**：C

---

### 例题2：计算题（寻址方式）

**题目**：设某计算机指令格式为 `[OP|X|A]`，其中X为寻址方式字段，A为形式地址。已知 (R1)=1000H, (PC)=2000H, [2000H]=3000H, [3000H]=4000H。

当X=0（直接寻址）、X=1（间接寻址）、X=2（寄存器寻址）、X=3（相对寻址）、X=4（变址寻址，IX=R1）、A=100H时，分别求有效地址EA和操作数。

**详细解答**：

```
直接寻址(X=0)：EA = A = 0100H
间接寻址(X=1)：EA = (A) = (0100H) → 需要知道0100H的内容
寄存器寻址(X=2)：EA无意义，操作数 = (R1) = 1000H
相对寻址(X=3)：EA = (PC) + A = 2000H + 100H = 2100H
变址寻址(X=4)：EA = (R1) + A = 1000H + 100H = 1100H
```

**总结**：关键是正确使用各寻址方式的EA计算公式。

### 例题3：综合分析题（扩展操作码）

**题目**：某计算机指令字长16位，有三地址、二地址、一地址三种指令格式。若操作码固定4位，最多可定义16条指令。现需要15条三地址指令、15条二地址指令、15条一地址指令。设计扩展操作码方案。

**详细解答**：

```
操作码4位，最多编码16个（0000~1111）

方案：
  三地址指令（4位OP + 3×4位地址 = 16位，刚好用完）：
    0000~1110 → 15条三地址指令
    1111 → 扩展标志

  但这样没有剩余位给二地址指令了！
  
  需要减小地址码位数：
  若地址码改为3位：
    三地址：4位OP + 3×3位 = 13位 ≤ 16位 → OK
    二地址：4位OP + 2×3位 = 10位 → 用1111扩展后继续
    一地址：4位OP + 3位 = 7位 → 继续扩展
```

实际上题目给出的操作码4位可能不够，需要更灵活的设计。

**总结**：扩展操作码的核心是前缀码性质，设计时要注意每层的编码空间分配。

---

### 例题4：历年真题

**题目（2016年408）**：某计算机按字节编址，采用小端方式存放数据。若寄存器R中内容为12345678H，则存储器地址2000H开始存放的内容为（存储器按字编址，字长32位）

**详细解答**：
```
小端存储：低位字节存放在低地址
12345678H → 78H存放在2000H，56H存放在2001H，34H存放在2002H，12H存放在2003H
```

**答案**：从2000H开始：78H, 56H, 34H, 12H

**总结**：大端/小端的区别在于字节存放顺序，小端是低对低、高对高。

## 六、易错点

1. ❌ 基址寻址和变址寻址是一回事 → ✅ 基址面向系统（基址寄存器不可改），变址面向用户
2. ❌ 寄存器寻址没有有效地址 → ✅ 寄存器寻址的操作数在寄存器中，不需要访存
3. ❌ 立即寻址的操作数存在存储器中 → ✅ 操作数直接在指令中，不需要访存
4. ❌ RISC一定比CISC快 → ✅ 取决于具体实现，CISC在某些场景下可能更好
5. ❌ 间接寻址只需一次访存 → ✅ 间接寻址需要**两次**访存（先取地址，再取操作数）
