## 一、基本概念

### 1.1 存储器层次结构

```
        ┌─────────┐
        │ 寄存器  │ ← 最快、最小、最贵
        ├─────────┤
        │ Cache   │ ← SRAM
        ├─────────┤
        │ 主存    │ ← DRAM
        ├─────────┤
        │ 虚拟存储│ ← 磁盘/SSD
        └─────────┘
        速度↓ 容量↑ 成本↓
```

**设计原则**：程序局部性原理（时间局部性 + 空间局部性）使得层次结构有效。

### 1.2 存储器分类

| 分类方式 | 类型 |
|----------|------|
| 按介质 | 半导体存储器、磁表面存储器、光存储器 |
| 按读写 | ROM（只读）、RAM（随机读写） |
| 按存取方式 | 随机存取（RAM/ROM）、顺序存取（磁带）、直接存取（磁盘） |
| 按易失性 | 易失（SRAM/DRAM）、非易失（ROM/Flash） |

### 1.3 SRAM vs DRAM

| 特性 | SRAM | DRAM |
|------|------|------|
| 存储原理 | 双稳态触发器（6个晶体管） | 电容充放电（1个晶体管+1个电容） |
| 是否需要刷新 | 否 | 是（通常64ms刷新一次） |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 功耗 | 大 | 小 |
| 成本 | 高 | 低 |
| 用途 | Cache | 主存 |

### 1.4 存储器主要技术指标

| 指标 | 定义 | 公式 |
|------|------|------|
| **容量** | 存储器能存储的二进制位数 | 字数 × 字长 |
| **存取时间（T_a）** | 从启动读/写到完成的时间 | — |
| **存取周期（T_m）** | 连续两次独立存取的最小间隔 | T_m ≥ T_a（含恢复时间） |
| **带宽（W）** | 单位时间内存取的数据量 | W = 字长 / T_m |

## 二、核心知识

### 2.1 主存储器扩展

**位扩展**：增加字长（如用多片8位芯片拼成32位）
```
将多片芯片的地址线、控制线并联，数据线分别接到总线的不同位段
```

**字扩展**：增加字数（如用多片芯片扩展存储容量）
```
地址线分为两部分：低位直接连芯片，高位通过译码器选片（片选信号CS）
```

**字位同时扩展**：既增加字数又增加字长
```
先位扩展组成"一行"，再字扩展增加"行数"，用译码器选行
```

### 2.2 双端口存储器和多体交叉存储器

**多体交叉存储器**：

| 方式 | 特点 |
|------|------|
| **高位交叉编址** | 连续地址在同一存储体 → 不能并行访问 |
| **低位交叉编址** | 连续地址在不同存储体 → 可以流水式访问 |

**低位交叉存储器带宽计算**：
```
设m个存储体，每个体存取周期为T，总线传输周期为τ
若 T = mτ，则每隔τ启动一个体，可实现连续传输
最大带宽 = m × (字长 / T)
```

### 2.3 Cache 原理

**基本思想**：利用局部性原理，将主存中近期可能被访问的数据副本放入Cache。

**地址映射方式**：

| 映射方式 | 主存地址划分 | 特点 |
|----------|-------------|------|
| **直接映射** | Tag + 行号 + 块内偏移 | 简单，但冲突率高 |
| **全相联映射** | Tag + 块内偏移 | 冲突率最低，但硬件复杂 |
| **组相联映射** | Tag + 组号 + 块内偏移 | 折中方案（n路组相联） |

**Cache地址结构（以直接映射为例）**：
```
主存地址 = [Tag(高位)] [行号(中间)] [块内偏移(低位)]

Cache行数 = 2^(行号位数)
块大小 = 2^(偏移位数) 字节
```

**替换策略**：

| 策略 | 说明 |
|------|------|
| **FIFO** | 先进先出 |
| **LRU** | 最近最少使用（实际最常用） |
| **LFU** | 最不经常使用 |
| **随机替换** | 随机选择 |

**写策略**：

| 策略 | 写命中 | 写不命中 |
|------|--------|----------|
| **写直达（Write Through）** | 同时写Cache和主存 | 写分配/非写分配 |
| **写回（Write Back）** | 只写Cache，设脏位 | 写分配/非写分配 |

### 2.4 Cache 性能计算

**Cache命中率**：
```
h = 命中次数 / 总访问次数
```

**平均访问时间**：
```
T_avg = h × T_cache + (1-h) × T_miss

考虑写操作的修改位：
  写回法：T_avg = h × T_c + (1-h) × [T_m]（写回时额外开销）

对于两级Cache：
  T_avg = h₁ × T₁ + (1-h₁) × h₂ × T₂ + (1-h₁)(1-h₂) × T_m
```

**Cache缺失代价分析（3C模型）**：
- **Compulsory（强制缺失）**：首次访问，冷启动
- **Capacity（容量缺失）**：Cache太小装不下
- **Conflict（冲突缺失）**：直接映射/组相联的映射冲突

### 2.5 虚拟存储器

**页式虚拟存储器**：

```
虚地址 = [虚页号(高位)] [页内偏移(低位)]

地址转换过程：
1. 用虚页号查页表（TLB → Page Table）
2. 得到实页号（物理页帧号）
3. 实地址 = [实页号] + [页内偏移]

Page Table Entry (PTE) 包含：
  - 实页号
  - 有效位（是否在主存中）
  - 脏位（是否被修改）
  - 访问权限
```

**TLB（快表）**：
- 存放在Cache中的页表项副本
- 命中率通常很高（>99%）
- TLB + Cache + 主存的三级访问

### 2.6 虚实地址转换 + Cache访问的综合流程

```
CPU发出虚地址
  │
  ├─→ TLB查找 ─── 命中 → 得到实页号 → 拼接实地址 → Cache查找
  │                                                  ├─ 命中 → 完成
  │                                                  └─ 缺失 → 访问主存
  │
  └─→ TLB缺失 → 查Page Table
                  ├─ 在主存（有效位=1）→ 得到实页号 → 更新TLB → Cache查找
                  └─ 不在主存 → 缺页中断 → 从磁盘调入主存
```

## 三、深入理解

### 3.1 为什么要层次结构？

- 速度和容量是一对矛盾：速度快的存储器容量小且贵
- 局部性原理保证了层次结构的有效性：
  - 时间局部性：刚访问的数据可能很快再次被访问 → Cache保留副本
  - 空间局部性：访问某地址附近的数据也可能被访问 → Cache按块传输

### 3.2 Cache行大小的权衡

- **行太小**：空间局部性利用不足，命中率低
- **行太大**：传输时间增加，且可能浪费带宽
- **经验值**：通常32~128字节

### 3.3 映射方式与硬件复杂度

```
直接映射 → 只需比较1个Tag → 硬件最简单
n路组相联 → 需要比较n个Tag → 复杂度中等
全相联 → 需要比较所有Tag → 硬件最复杂
```

### 3.4 写直达 vs 写回的本质区别

- **写直达**：主存始终最新，但总线流量大（每次写都要写主存）
- **写回**：只在Cache行被替换时写回主存，总线流量小，但一致性维护复杂

## 五、典型例题

### 例题1：概念辨析（选择题）

**题目**：下列关于Cache的叙述中，正确的是（）
A. Cache的容量通常大于主存
B. Cache命中率只与Cache容量有关
C. 提高Cache命中率的方法之一是增大块大小
D. 全相联映射方式下不需要替换算法

**详细解答**：
- A：❌ Cache容量远小于主存
- B：❌ 命中率与容量、映射方式、替换策略、块大小都有关
- C：✅ 增大块大小可以利用空间局部性提高命中率（但有上限）
- D：❌ 全相联也需要替换策略（因为有多个位置可选）

**答案**：C

### 例题2：计算题

**题目**：某计算机Cache有8行，采用直接映射方式，块大小为16字节，主存地址为32位。
（1）求Cache地址中Tag、行号、偏移各占多少位？
（2）主存地址 `0x00001A3C` 映射到Cache的哪一行？Tag值是多少？

**思路**：直接映射地址结构 = Tag + 行号 + 块内偏移。

**详细解答**：

**（1）地址划分**：
```
块大小 = 16B → 偏移 = log₂(16) = 4 位
Cache行数 = 8 → 行号 = log₂(8) = 3 位
Tag = 32 - 4 - 3 = 25 位
```

**（2）地址0x00001A3C的映射**：
```
十六进制转二进制：0000 0000 0000 0000 0001 1010 0011 1100
最后4位偏移：1100
行号位（第4~6位）：001 = 第1行
Tag（第7~31位）：0000 0000 0000 0000 0000 001 101 = 0x0000000D（取高位25位）
```

**答案**：映射到第1行，Tag = `0x0000000D`。

### 例题3：综合分析题（Cache命中率计算）

**题目**：某计算机Cache采用4路组相联，Cache共16KB，块大小为64B，主存地址32位。程序访问以下地址序列（十进制）：0, 64, 128, 256, 0, 64, 128, 0, 64, 128。采用LRU替换策略。
（1）求Cache的组数和地址划分
（2）逐个分析每次访问是命中还是缺失
（3）求命中率

**思路**：先根据已知条件推导Cache结构，再逐个分析访问序列。

**详细解答**：

**（1）Cache结构**：
```
Cache总行数 = 16KB / 64B = 256 行
4路组相联 → 组数 = 256 / 4 = 64 组
偏移位 = log₂(64) = 6 位
组号位 = log₂(64) = 6 位
Tag位 = 32 - 6 - 6 = 20 位
```

**（2）分析各地址**：
```
0   → 块地址0   → 组号0, Tag0   → 缺失（放入组0）
64  → 块地址1   → 组号1, Tag0   → 缺失（放入组1）
128 → 块地址2   → 组号2, Tag0   → 缺失（放入组2）
256 → 块地址4   → 组号4, Tag0   → 缺失（放入组4）
0   → 块地址0   → 组号0, Tag0   → 命中！
64  → 块地址1   → 组号1, Tag0   → 命中！
128 → 块地址2   → 组号2, Tag0   → 命中！
0   → 块地址0   → 组号0, Tag0   → 命中！
64  → 块地址1   → 组号1, Tag0   → 命中！
128 → 块地址2   → 组号2, Tag0   → 命中！
```

**（3）命中率**：
```
总访问次数 = 10
命中次数 = 6
命中率 h = 6/10 = 60%
```

**总结**：4路组相联在本题中容量足够装入所有活跃块，因此第一次访问后全部命中。

### 例题4：历年真题

**题目（2019年408）**：某计算机主存地址空间大小为256MB，按字节编址，Cache有16个Cache行，块大小为32B，采用2路组相联映射。若Cache初始为空，CPU依次访问地址0x00000000, 0x00000020, 0x00000040, ..., 0x000003E0（等差数列，公差0x20）。

求Cache访问的命中率。

**详细解答**：
```
地址序列：0x0, 0x20, 0x40, 0x60, ..., 0x3E0
共 0x3E0/0x20 + 1 = 32 个地址

Cache结构：
  块大小 = 32B → 偏移 = 5位
  16行 / 2路 = 8组 → 组号 = 3位
  Tag = 高位

分析每个地址所属的组：
  块地址 = 地址/32
  0→组0, 1→组1, 2→组2, 3→组3, 4→组4, 5→组5, 6→组6, 7→组7
  8→组0, 9→组1, 10→组2, ...

8组 × 2路 = 16个位置，但要装入32个不同的块
→ 第17~32次访问会替换掉前16个块
→ 替换后再次访问原来的块时会缺失

结果：16次命中（第17~32次访问对应前16次放的块）
命中率 = 16/32 = 50%
```

**总结**：当活跃工作集超过Cache容量时，会发生冲突缺失和容量缺失。

## 六、易错点

1. ❌ Cache行大小和块大小是两个概念 → ✅ 在408考试中，**行=块**，一个Cache行存放一个主存块
2. ❌ 直接映射不需要替换策略 → ✅ 直接映射中同一位置的替换是确定性的（无选择），不需要替换算法
3. ❌ 命中率可以达到100% → ✅ 理论上不可能（总有强制缺失），实际通常95%~99%
4. ❌ 虚地址和实地址长度相同 → ✅ 虚地址位数通常≥实地址位数
5. ❌ 脏位对写直达有意义 → ✅ 脏位是**写回**策略使用的（标记是否需要写回主存）
6. ❌ 多体交叉存储器按高位编址可以提高带宽 → ✅ 应为**低位**交叉编址
