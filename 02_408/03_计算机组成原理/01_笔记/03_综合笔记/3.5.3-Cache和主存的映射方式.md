# Cache和主存的映射方式

## 1. 问题背景

Cache容量很小，主存容量很大。需要解决三个核心问题：

| 问题 | 解决方案 |
|------|----------|
| 如何区分Cache与主存的数据块对应关系？ | Cache和主存的映射方式 |
| Cache满了怎么办？ | 替换算法 |
| CPU修改了Cache中的数据副本，如何确保主存数据一致性？ | Cache写策略 |

> **关键前提**：每次被访问的主存块，一定会被立即调入Cache。

---

## 2. Cache行的基本结构

每个Cache行（块）包含：

```
┌─────────┬──────────────┬──────────────┐
│ 有效位  │   标记(Tag)   │   数据块     │
│ (1bit)  │              │              │
└─────────┴──────────────┴──────────────┘
```

- **有效位**：初始为0，表示该Cache行是否有效
- **标记(Tag)**：记录对应的主存块号（或其部分）
- **数据块**：存储的实际数据

---

## 3. 三种映射方式

### 3.1 全相联映射（Fully Associative）

**规则**：主存中的任意块可以放到Cache的任意位置。

**地址结构**：

```
┌──────────────────────┬──────────────┐
│     主存块号(高位)      │   块内地址(低位) │
└──────────────────────┴──────────────┘
```

**示例**：主存256MB，Cache有8行，行长64B

- 主存地址 = 28位
- 块内地址 = log₂(64) = 6位
- 主存块号 = 28 - 6 = 22位
- 主存块数 = 256MB / 64B = 2²² = 4,194,304块

**访存过程**：

1. CPU给出28位主存地址
2. 取前22位作为主存块号
3. **并行比较**Cache中所有行的标记
4. 若标记匹配且有效位=1 → Cache命中，用后6位访问块内单元
5. 若未命中或有效位=0 → 访问主存

**Cache行结构**：

```
┌─────────┬──────────────────┐
│ 有效位  │   标记(22位)      │
└─────────┴──────────────────┘
```

**特点**：
- ✅ Cache利用率最高，冲突率最低
- ❌ 需要比较所有行的标记，硬件成本高（需要大量比较器）
- ❌ 标记位数多，存储开销大

---

### 3.2 直接映射（Direct Mapped）

**规则**：每个主存块只能放到Cache的一个特定位置。

**映射公式**：

$$\text{Cache行号} = \text{主存块号} \mod \text{Cache总行数}$$

**地址结构**：

```
┌──────────────────┬──────────────┬──────────────┐
│     标记(Tag)      │   行号(Index)  │  块内地址     │
│   (高位)           │   (中位)       │   (低位)      │
└──────────────────┴──────────────┴──────────────┘
```

**示例**：主存256MB，Cache有8行，行长64B

- 主存地址 = 28位
- 块内地址 = 6位
- 行号 = log₂(8) = 3位
- 标记 = 28 - 6 - 3 = 19位

**地址拆分**：

```
28位地址 = [19位标记][3位行号][6位块内地址]
```

**访存过程**：

1. CPU给出28位主存地址
2. 取中间3位确定Cache行号
3. 比较该行的标记与地址高19位
4. 若标记匹配且有效位=1 → Cache命中
5. 若未命中 → 访问主存

**特点**：
- ✅ 硬件简单，只需1个比较器
- ✅ 标记位数少
- ❌ Cache利用率低，冲突率高
- ❌ 多个主存块映射到同一Cache行时会频繁替换

**标记优化**：
- 若Cache总行数 = 2ⁿ，则主存块号末尾n位直接反映它在Cache中的位置
- 只需将主存块号的其余位作为标记即可

---

### 3.3 组相联映射（Set Associative）

**规则**：Cache分为若干组，每个主存块可以放到特定分组中的任意一个位置。

**映射公式**：

$$\text{组号} = \text{主存块号} \mod \text{分组数}$$

**地址结构**：

```
┌──────────────────┬──────────────┬──────────────┐
│     标记(Tag)      │   组号(Index)  │  块内地址     │
│   (高位)           │   (中位)       │   (低位)      │
└──────────────────┴──────────────┴──────────────┘
```

**示例**：主存256MB，Cache有8行，行长64B，采用2路组相联

- 分组数 = 8 / 2 = 4组
- 主存地址 = 28位
- 块内地址 = 6位
- 组号 = log₂(4) = 2位
- 标记 = 28 - 6 - 2 = 20位

**地址拆分**：

```
28位地址 = [20位标记][2位组号][6位块内地址]
```

**访存过程**：

1. CPU给出28位主存地址
2. 取中间2位确定组号
3. **并行比较**该组内所有行的标记（本例为2个）
4. 若标记匹配且有效位=1 → Cache命中
5. 若未命中 → 访问主存

**术语说明**：
- **n路组相联**：每组包含n个Cache行
- 全相联映射 = "Cache总行数"路组相联
- 直接映射 = 1路组相联

**特点**：
- ✅ 兼顾了全相联和直接映射的优点
- ✅ 冲突率较低，硬件成本适中
- 是实际应用中最常用的映射方式

---

## 4. 三种映射方式对比

| 特性 | 全相联映射 | 直接映射 | 组相联映射 |
|------|-----------|---------|-----------|
| 映射规则 | 随意放 | 放固定位置 | 放到特定分组 |
| 比较器数量 | Cache行数个 | 1个 | 每组内行数个 |
| 标记位数 | 主存块号位数 | 主存块号位数-log₂(Cache行数) | 主存块号位数-log₂(组数) |
| 冲突率 | 最低 | 最高 | 适中 |
| 硬件成本 | 最高 | 最低 | 适中 |
| 适用场景 | 小容量Cache | 大容量Cache | 实际应用最广泛 |

---

## 5. 计算题要点

### 地址位数计算

```
主存地址位数 = log₂(主存容量)
块内地址位数 = log₂(块大小)
主存块号位数 = 主存地址位数 - 块内地址位数
主存块数 = 主存容量 / 块大小
```

### 直接映射标记位数

```
标记位数 = 主存块号位数 - log₂(Cache行数)
```

### 组相联映射标记位数

```
标记位数 = 主存块号位数 - log₂(组数)
组数 = Cache行数 / 路数
```

### 判断是否命中

1. 根据地址拆分找到对应的Cache行/组
2. 检查有效位是否为1
3. 比较标记是否匹配

---

## 6. 典型例题

**题目**：某计算机主存地址空间大小为256MB，按字节编址，其数据Cache有8个Cache行，行长64B。

**解答**：

1. **全相联映射**：
   - 地址：[22位主存块号][6位块内地址]
   - 标记 = 22位
   - 访存需比较8个标记

2. **直接映射**：
   - 地址：[19位标记][3位行号][6位块内地址]
   - 标记 = 19位
   - 访存只需比较1个标记

3. **2路组相联映射**：
   - 组数 = 8/2 = 4
   - 地址：[20位标记][2位组号][6位块内地址]
   - 标记 = 20位
   - 访存需比较2个标记

---

## 7. 考研重点

1. **地址拆分**：给定主存地址，能正确拆分出标记、行号/组号、块内地址
2. **判断命中**：根据Cache内容判断给定地址是否命中
3. **计算比较器**：不同映射方式需要的比较器数量
4. **标记位数**：不同映射方式的标记位数计算
5. **综合应用**：映射方式+替换算法+写策略的综合分析
