# 5.1 CPU 的功能和基本结构

## 一、CPU 的功能

CPU = **运算器** + **控制器**

- 指令控制：取指令、分析指令、执行指令
- 操作控制：产生各种操作控制信号
- 时间控制：对各种操作加以时间上的控制
- 数据加工：对数据进行算术和逻辑运算
- 中断处理：对异常和中断进行处理

## 二、运算器的基本组成

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ACC** | Accumulator（累加器） | 存放操作数或运算结果 |
| **MQ** | Multiple-Quotient Register（乘商寄存器） | 乘除运算时存放操作数或结果 |
| **X** | 通用操作数寄存器 | 存放操作数 |
| **ALU** | Arithmetic and Logic Unit（算术逻辑单元） | 通过内部复杂电路实现算术、逻辑运算 |

### 运算中各寄存器的分工

| 运算 | ACC | MQ | X |
|------|-----|----|----|
| **加法** | 被加数、和 | — | 加数 |
| **减法** | 被减数、差 | — | 减数 |
| **乘法** | 乘积高位 | 乘数、乘积低位 | 被乘数 |
| **除法** | 被除数、余数 | 商 | 除数 |

## 三、控制器的基本组成

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **CU** | Control Unit（控制单元） | 分析指令，给出控制信号 |
| **IR** | Instruction Register（指令寄存器） | 存放**当前正在执行**的指令 |
| **PC** | Program Counter（程序计数器） | 存放**下一条指令地址**，有自动 +1 功能 |

### 完成一条指令的过程

```
取指令 → 分析指令 → 执行指令
  PC        IR         CU
  ↓         ↓          ↓
 取指      译码       执行
```

## 四、CPU 的工作过程示例

以计算 y = a×b + c 为例：

```
地址  操作码    地址码         注释
0     000001    0000000101     取数a至ACC
1     000100    0000000110     乘b得ab，存于ACC中
2     000011    0000000111     加c得ab+c，存于ACC中
3     000010    0000001000     将ab+c存于主存单元
4     000110    0000000000     停机
5     0000000000000010         原始数据a=2
6     0000000000000011         原始数据b=3
7     0000000000000001         原始数据c=1
8     0000000000000000         原始数据y=0
```

### 取数指令执行过程（地址0）
1. (PC)→MAR → MAR=0
2. 1→R（读信号）
3. M(MAR)→MDR → MDR=000001 0000000101
4. (MDR)→IR → IR=000001 0000000101
5. OP(IR)→CU → CU 分析得知是"取数"指令
6. Ad(IR)→MAR → MAR=5
7. 1→R
8. M(MAR)→MDR → MDR=2
9. (MDR)→ACC → ACC=2

### 乘法指令执行过程（地址1）
1. (PC)→MAR → MAR=1
2. 取指 → MDR=000100 0000000110
3. (MDR)→IR
4. OP(IR)→CU → "乘法"指令
5. Ad(IR)→MAR → MAR=6
6. M(MAR)→MDR → MDR=3
7. (MDR)→MQ → MQ=3
8. (ACC)→X → X=2
9. (MQ)×(X)→ACC → ACC=6
