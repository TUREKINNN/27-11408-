# Cache 的基本原理

## 一、为什么需要 Cache？

### 1.1 速度矛盾

CPU 速度 >> 主存速度，CPU 经常需要等待主存数据。

### 1.2 局部性原理

#### 时间局部性

在最近的未来要用到的信息，很可能是**现在正在使用的信息**。

- 例如：循环结构的指令代码

#### 空间局部性

在最近的未来要用到的信息，很可能与现在正在使用的信息在**存储空间上是邻近的**。

- 例如：数组元素、顺序执行的指令代码

> 基于局部性原理，可以把 CPU 目前访问的地址"周围"的部分数据放到 Cache 中。

---

## 二、Cache 的工作原理

### 2.1 基本结构

```
CPU ←→ Cache（SRAM，小而快）←→ 主存（DRAM，大而慢）
```

- Cache 用 **SRAM** 实现，速度快，成本高
- Cache 通常集成在 **CPU 内部**
- Cache 比内存快约 **60 倍**

### 2.2 工作流程

1. CPU 要访问某个地址的数据
2. 先查看 Cache 中是否已有该数据（**命中**）
3. 若命中，直接从 Cache 读取（速度快）
4. 若未命中，从主存读取，并将该数据所在的**整个块**调入 Cache

### 2.3 分块机制

将主存的存储空间"**分块**"，主存与 Cache 之间以"**块**"为单位进行数据交换。

| 术语 | 含义 |
|------|------|
| **块/行** | Cache 中的数据块也称为"行"（Line） |
| **页/页面/页框** | 操作系统中，主存的一个块也称为一个页 |
| **块大小** | 通常为 1KB 或 64B 等 |
| **块内地址** | 块内偏移量，由块大小决定位数 |

**主存地址的划分：**

$$
\text{主存地址} = \underbrace{\text{块号}}_{高位} + \underbrace{\text{块内地址}}_{低位}
$$

## 三、Cache 性能分析 ⭐⭐⭐

### 3.1 基本参数

| 符号 | 含义 |
|------|------|
| $t_c$ | 访问一次 Cache 所需时间 |
| $t_m$ | 访问一次主存所需时间 |
| $H$ | 命中率（CPU 欲访问的信息已在 Cache 中的比率） |
| $M = 1 - H$ | 缺失率（未命中率） |

### 3.2 平均访问时间

**方式一：先访问 Cache，未命中再访问主存**

$$
t = H \cdot t_c + (1 - H)(t_c + t_m)
$$

**方式二：同时访问 Cache 和主存，命中则停止访问主存**

$$
t = H \cdot t_c + (1 - H) \cdot t_m
$$

### 3.3 计算示例

**例题**：假设 Cache 的速度是主存的 5 倍，命中率为 95%，求采用 Cache 后性能提高多少。

设 Cache 存取周期为 $t$，则主存存取周期为 $5t$。

**方式二（同时访问）：**

$$
t_{avg} = 0.95t + 0.05 \times 5t = 1.2t
$$

$$
\text{性能提升} = \frac{5t}{1.2t} \approx 4.17 \text{ 倍}
$$

**方式一（先 Cache 后主存）：**

$$
t_{avg} = 0.95t + 0.05 \times (t + 5t) = 1.25t
$$

$$
\text{性能提升} = \frac{5t}{1.25t} = 4 \text{ 倍}
$$

## 四、Cache 需要解决的三个问题

| 问题 | 解决方案 |
|------|---------|
| 如何区分 Cache 与主存的数据块对应关系？ | **映射方式**（直接映射、全相联映射、组相联映射） |
| Cache 很小，主存很大，Cache 满了怎么办？ | **替换算法**（LRU、FIFO、RAND、LFU） |
| CPU 修改了 Cache 中的数据副本，如何确保主存一致性？ | **写策略**（写回法、全写法） |
