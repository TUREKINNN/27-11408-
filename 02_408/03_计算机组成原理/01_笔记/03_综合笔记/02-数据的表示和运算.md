## 一、基本概念

### 1.1 进制与编码基础

**进制转换**：
- 十进制 → 二进制：整数部分除2取余（逆序），小数部分乘2取整（顺序）
- 八进制/十六进制与二进制互转：每3位/4位二进制对应1位

**BCD码（8421码）**：
- 每4位二进制表示1位十进制数（0000~1001对应0~9）
- 运算后若大于9或有进位，需+6修正

### 1.2 定点数表示

| 编码方式 | n位范围 | 零的表示 | 特点 |
|----------|---------|----------|------|
| 无符号整数 | 0 ~ 2ⁿ-1 | 00...0（唯一） | 全部位表示数值 |
| 原码 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | +0和-0两种 | 符号位+绝对值 |
| 反码 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | +0和-0两种 | 正数不变，负数按位取反 |
| 补码 | -2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1) | 00...0（唯一） | 正数不变，负数=反码+1 |
| 移码 | -2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1) | 100...0（唯一） | 补码符号位取反，用于浮点数阶码 |

**真值与机器数的关系**（以8位为例）：

```
真值 +5  → 原码 0_0000101 → 反码 0_0000101 → 补码 0_0000101 → 移码 1_0000101
真值 -5  → 原码 1_0000101 → 反码 1_1111010 → 补码 1_1111011 → 移码 0_1111011
```

### 1.3 浮点数表示

**一般格式**：
```
N = (-1)^S × M × R^E

其中：
  S：符号位（0正1负）
  M：尾数（定点小数），决定精度
  E：阶码（定点整数），决定范围
  R：基数（隐含，通常为2）
```

**IEEE 754 单精度（32位）格式**：

```
| 1位  |  8位   |   23位   |
| S |   E    |     M     |
  31  30~23    22~0

真值 = (-1)^S × 1.M × 2^(E-127)

特殊值：
  E=0, M=0: ±0
  E=255, M=0: ±∞
  E=255, M≠0: NaN
  E=0, M≠0: 非规格化数 = (-1)^S × 0.M × 2^(-126)
```

**IEEE 754 双精度（64位）格式**：

```
| 1位  |  11位   |   52位   |
| S |    E     |     M     |

真值 = (-1)^S × 1.M × 2^(E-1023)
```

## 二、核心知识

### 2.1 补码加减运算的推导

**补码加法公式**（证明略，核心思想：模运算）：

```
[X+Y]_补 = [X]_补 + [Y]_补  (mod 2^n)
```

**补码减法公式**（由加法推导）：

```
[X-Y]_补 = [X]_补 + [-Y]_补  (mod 2^n)

其中 [-Y]_补 的求法：将 [Y]_补 所有位（包括符号位）取反，末位加1
```

**溢出检测方法**：

| 方法 | 条件 | 说明 |
|------|------|------|
| **单符号位法** | 两个操作数同号，结果异号 | 仅对加法有效 |
| **双符号位法（变形补码）** | S₁ ≠ S₂（两个符号位不同） | 00正, 01正溢, 10负溢, 11负 |
| **进位位法** | C_s ⊕ C_1 = 1 | C_s为符号位进位，C_1为最高数值位进位 |

### 2.2 移位运算

| 移位类型 | 说明 | 适用场景 |
|----------|------|----------|
| 逻辑左移/右移 | 高位/低位移出，空位补0 | 无符号数 |
| 算术左移 | 高位移出，低位补0（等价乘2） | 有符号数 |
| 算术右移 | 低位移出，高位补符号位（等价除2） | 有符号数 |

### 2.3 IEEE 754 浮点数运算步骤

**加减法（以单精度为例）**：

```
步骤1：对阶
  - 求阶差 ΔE = E_x - E_y
  - 小阶向大阶看齐：小阶的尾数右移|ΔE|位，阶码加上|ΔE|
  （注意：右移丢失的是精度，应保留附加位用于舍入）

步骤2：尾数加减
  - 对齐后的尾数直接进行定点加减法
  - 包含隐含的1（规格化数）

步骤3：规格化
  - 左规：结果形如00.0xx...x → 尾数左移直到1.xx形式，阶码减相应值
  - 右规：结果形如1x.xxxx...x（溢出） → 尾数右移1位，阶码加1

步骤4：舍入
  - 就近舍入（Round to Nearest）：0舍1入
  - 向零舍入（Truncate）
  - 向+∞舍入 / 向-∞舍入

步骤5：溢出判断
  - 阶码上溢（E≥255）→ 结果溢出
  - 阶码下溢（E≤0时左规结果E<0）→ 结果为0
```

**乘法**：
```
1. 阶码相加：E_结果 = E_x + E_y - 偏置值（127）
2. 尾数相乘：M_结果 = M_x × M_y
3. 规格化、舍入、溢出判断
```

**除法**：
```
1. 阶码相减：E_结果 = E_x - E_y + 偏置值（127）
2. 尾数相除：M_结果 = M_x / M_y
3. 规格化、舍入、溢出判断
```

### 2.4 定点乘法

**原码一位乘法**：
```
步骤：
1. 初始化：部分积P=0，乘数寄存器存放Y
2. 检查乘数最低位：
   - 若为1：P = P + X（被乘数）
   - 若为0：P = P + 0
3. P和乘数一起右移1位
4. 重复n次
5. 结果的符号 = X的符号 ⊕ Y的符号
```

**补码Booth乘法（两位一乘）**：

```
核心思想：利用等式
  2Y = 2(y_{n-1}·2^{n-1} + ... + y_0·2^0)
  Y = -y_{n-1}·2^{n-1} + Σ(y_i - y_{i+1})·2^i

编码表（y_{i+1} - y_i 的值）：
  00 → 0（不操作）
  01 → +X（加被乘数）
  10 → -X（减被乘数）
  11 → 0（不操作）
```

### 2.5 浮点数的表示范围

| 格式 | 最小正规格化数 | 最大规格化数 | 精度（十进制有效位） |
|------|---------------|-------------|---------------------|
| 单精度（32位） | 2^(-126) ≈ 1.18×10⁻³⁸ | 2^127×(2-2⁻²³) ≈ 3.40×10³⁸ | ~7位 |
| 双精度（64位） | 2^(-1022) ≈ 2.23×10⁻³⁰⁸ | 2^1023×(2-2⁻⁵²) ≈ 1.80×10³⁰⁸ | ~16位 |

## 三、深入理解

### 3.1 为什么用补码而不是原码？

| 对比 | 原码 | 补码 |
|------|------|------|
| 加法电路 | 需要判断符号决定加/减 | 直接加法，统一处理 |
| 零的表示 | +0和-0两种 | 唯一表示 |
| 范围 | -(2ⁿ⁻¹-1)~+(2ⁿ⁻¹-1) | -2ⁿ⁻¹~+(2ⁿ⁻¹-1)（多表示一个负数） |
| 硬件复杂度 | 高 | 低 |

**结论**：补码简化了加减运算的硬件实现，是计算机内部表示有符号整数的标准方式。

### 3.2 浮点数为什么要规格化？

- **不规格化**的数（如0.001×2³）浪费了有效位，降低了精度
- **规格化**确保尾数最高有效位为1，最大限度利用有限的尾数位
- IEEE 754的"隐含1"进一步节省了一位存储空间

### 3.3 定点数与浮点数的本质区别

```
定点数：小数点位置固定 → 范围有限，但精度均匀
浮点数：小数点位置浮动 → 范围巨大，但精度不均匀
  - 浮点数越靠近0，相邻可表示数之间的间隔越小（精度越高）
  - 浮点数越远离0，相邻可表示数之间的间隔越大（精度越低）
```

## 五、典型例题

### 例题1：概念辨析（选择题）

**题目**：一个8位补码整数能表示的范围是（）
A. -127 ~ +127  B. -128 ~ +127  C. -256 ~ +255  D. -128 ~ +128

**思路**：补码范围公式 -2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1)。

**详细解答**：n=8，范围 = -2⁷ ~ +(2⁷-1) = -128 ~ +127。答案 **B**。

**总结**：补码比原码多表示一个最小负数（-128），因为+0和-0共用一个编码。

---

### 例题2：计算题（IEEE754表示）

**题目**：将十进制数 -0.625 转换为IEEE 754单精度浮点数表示。

**思路**：先转二进制→规格化→确定S、E、M→填入32位。

**详细解答**：

**Step 1**：转二进制
```
0.625 × 2 = 1.25 ... 1
0.25  × 2 = 0.5  ... 0
0.5   × 2 = 1.0  ... 1
∴ 0.625 = 0.101₂
∴ -0.625 = -0.101₂
```

**Step 2**：规格化
```
-0.101₂ = -1.01₂ × 2⁻¹
```

**Step 3**：确定各字段
```
S = 1（负数）
E = -1 + 127 = 126 = 01111110₂
M = 01000000000000000000000（隐含1去掉，尾数23位）
```

**Step 4**：组合
```
1 01111110 01000000000000000000000
= 1011 1111 0010 0000 0000 0000 0000 0000
= BF200000H
```

**总结**：IEEE754编码的关键步骤——规格化后分离S、E、M，E要加偏置值。

---

### 例题3：综合分析题（浮点数加减）

**题目**：已知 X = +0.1101 × 2², Y = +0.1011 × 2¹，用浮点加法求 X+Y。
（假设基数为2，尾数4位，阶码3位，采用双符号位补码）

**思路**：按5步流程执行。

**详细解答**：

**Step 1：对阶**
```
E_x = 2, E_y = 1, ΔE = 2-1 = 1
小阶向大阶看齐：Y的尾数右移1位，E_y改为2
Y' = +0.01011₂ × 2² → 截断尾数为4位+保护位：0.0101₂
```

**Step 2：尾数加减**
```
  X尾数：00.1101
+ Y'尾数：00.0101
=        01.0010（双符号位）
```

**Step 3：规格化**
```
双符号位为01，说明尾数溢出，需右规
尾数右移1位：00.1001，阶码加1：E = 2+1 = 3
```

**Step 4：舍入**
```
截断，无额外舍入操作
```

**Step 5：溢出判断**
```
阶码E=3未溢出，结果有效
```

**结果**：X + Y = +0.1001 × 2³ = +0.1001 × 8 = 4.5（验证：2.625+1.375=4.0... 实际需精确计算）

**总结**：对阶是最关键的一步，一定要小阶向大阶对齐；双符号位可以同时检测溢出和判断正负。

### 例题4：历年真题

**题目（2017年408）**：IEEE 754单精度浮点数 `C0A00000H` 对应的真值是（）

**思路**：拆分S、E、M三个字段，代入公式。

**详细解答**：
```
十六进制：C0A00000
二进制：1100 0000 1010 0000 0000 0000 0000 0000

S = 1（负数）
E = 1000 0001 = 129，实际指数 = 129-127 = 2
M = 0100 0000 0000 0000 0000 000

实际尾数 = 1.M = 1.01₂ = 1.25

真值 = (-1)¹ × 1.25 × 2² = -1.25 × 4 = -5.0
```

**答案**：-5.0

**总结**：逆向拆解IEEE754时，注意隐含的1和偏置值的减法。

## 六、易错点

1. ❌ 补码右移时高位补0 → ✅ 应补**符号位**（算术右移）
2. ❌ 浮点数对阶时大阶向小阶看齐 → ✅ **小阶向大阶**看齐（右移丢失低位，精度损失更小）
3. ❌ IEEE754偏置值是128 → ✅ 单精度偏置值是**127**
4. ❌ IEEE754中所有数都有隐含的1 → ✅ 非规格化数（E=0）隐含0
5. ❌ 溢出和上溢是同一概念 → ✅ 定点数溢出≠浮点数上溢
6. ❌ 双符号位溢出判断时01和10都代表溢出 → ✅ 01正溢，10负溢，不能搞反
7. ❌ 浮点数乘法需要先对阶 → ✅ 乘法**不需要对阶**，直接阶码相加、尾数相乘

# 反码

反码：若符号位为0，则反码与原码相同  
若符号位为1，则数值位全部取反

$$

# 补码

补码：正数的补码 = 原码

负数的补码 = 反码末位 + 1（要考虑进位）

将负数补码转回原码的方法相同：尾数取反，末位+1

$$
\begin{array}{l} x = + 1 9 D \quad [ x ] _ {\text {原}} = 0, 0 0 1 0 0 1 1 \\ [ \mathbf {x} ] _ {\text {反}} = \mathbf {0}, 0 0 1 0 0 1 1 \\ [ \mathrm{x} ] _ {\text {补}} = 0, 0 0 1 0 0 1 1 \\ \end{array}
$$

$$
