# 5.6.1 指令流水线——基本概念与性能指标

## 一、指令流水的定义

一条指令的执行过程可以分成多个阶段，每个阶段用到的硬件不同：

```
取指（IF）→ 分析（ID）→ 执行（EX）
```

- **取指**：根据 PC 访问主存，取出指令送到 IR
- **分析**：译码操作码，形成有效地址 EA，取出操作数
- **执行**：根据操作码完成指令功能，将结果写到寄存器或主存

## 二、三种执行方式对比

设取指、分析、执行各阶段时间相等为 t，n 条指令：

| 执行方式 | 总耗时 | 公式 |
|----------|--------|------|
| **顺序执行** | 3nt | T = n × 3t |
| **一次重叠** | (1+2n)t | T = 3t + (n-1)×2t |
| **二次重叠（流水线）** | (2+n)t | T = 3t + (n-1)×t |

### 顺序执行
- 传统冯·诺依曼机
- 优点：控制简单，硬件代价小
- 缺点：速度慢，功能部件利用率低

### 一次重叠执行
- 程序执行时间缩短 1/3
- 功能部件利用率明显提高
- 需要较大硬件开销

### 二次重叠执行（流水线）
- 与顺序执行相比，执行时间缩短近 **2/3**
- 理想情况下，处理机中同时有 3 条指令在执行
- 可以分成 4 个或 5 个阶段（5 个阶段较常见）

## 四、流水线的性能指标

### 1. 吞吐率（TP）

单位时间内流水线完成的任务数量：

$$TP = \frac{n}{T_k}$$

- 一条指令分为 k 个阶段，每个阶段耗时 Δt
- 流水线实际运行时间：$T_k = (k + n - 1)\Delta t$

$$TP = \frac{n}{(k+n-1)\Delta t}$$

- 当 n → ∞ 时，**最大吞吐率**：$TP_{max} = \frac{1}{\Delta t}$

### 2. 加速比（S）

不使用流水线时间 / 使用流水线时间：

$$S = \frac{T_0}{T_k}$$

- 顺序执行：$T_0 = nk\Delta t$
- 流水线：$T_k = (k+n-1)\Delta t$

$$S = \frac{kn}{k+n-1}$$

- 当 n → ∞ 时，**最大加速比**：$S_{max} = k$

### 3. 效率（E）

流水线设备利用率：

$$E = \frac{n \text{ 个任务占用的时空区有效面积}}{n \text{ 个任务所用时间与 } k \text{ 个流水段围成的时空区总面积}} = \frac{T_0}{kT_k}$$

- 当 n → ∞ 时，**最高效率**：$E_{max} = 1$

## 五、理想情况的前提

1. 各阶段花费时间**相同**
2. 每个阶段结束后能**立即进入下一阶段**
