# 4.2.2.1 数据寻址（一）——直接、间接、寄存器寻址

## 一、指令寻址 vs 数据寻址

| | 指令寻址 | 数据寻址 |
|---|----------|----------|
| 目标 | 下一条指令的地址 | 本条指令地址码指明的真实地址 |
| 由谁给出 | 始终由 PC 给出 | 由寻址方式决定 |

## 二、数据寻址的基本格式

```
| 操作码（OP） | 寻址特征 | 形式地址（A） |
```

- **形式地址 A**：指令中直接给出的地址
- **有效地址 EA**：操作数的真实地址
- **寻址特征**：指明采用哪种寻址方式

## 三、直接寻址

**EA = A**（形式地址 A 就是有效地址）

```
| OP | 直接寻址特征 | A |
```

| 项目 | 内容 |
|------|------|
| 访存次数 | 取指1次 + 执行1次 = **2次** |
| 优点 | 简单，执行阶段仅访存一次，不需专门计算操作数地址 |
| 缺点 | A 的位数限制寻址范围；操作数地址不易修改 |

## 四、间接寻址

**EA = (A)**（形式地址 A 指向的存储单元中存放的才是有效地址）

```
| OP | 间接寻址特征 | A |  →  M[A] 中存放 EA  →  M[EA] 中存放操作数
```

| 项目 | 内容 |
|------|------|
| 访存次数 | 取指1次 + 执行2次 = **3次**（一次间址） |
| 优点 | 可扩大寻址范围（EA 位数 > A 位数）；便于编制程序（方便子程序返回） |
| 缺点 | 执行阶段多次访存（一次间址需2次访存） |

## 五、寄存器寻址

**EA = Rᵢ**（指令中直接给出寄存器编号，操作数在该寄存器中）

```
| OP | 寄存器寻址特征 | Rᵢ编号 |
```

| 项目 | 内容 |
|------|------|
| 访存次数 | 取指1次 + 执行0次 = **1次** |
| 优点 | 不访问主存，只访问寄存器；指令字短、执行速度快；支持向量/矩阵运算 |
| 缺点 | 寄存器价格昂贵，数量有限 |

## 六、寄存器间接寻址

**EA = (Rᵢ)**（寄存器中存放的是操作数所在主存单元的地址）

```
| OP | 寄存器间接特征 | Rᵢ编号 |
```

| 项目 | 内容 |
|------|------|
| 访存次数 | 取指1次 + 执行1次 = **2次** |
| 特点 | 比一般间接寻址速度快，但执行阶段仍需访存 |

## 七、隐含寻址

- 不明显给出操作数的地址，指令中**隐含**着操作数的地址
- 优点：缩短指令字长
- 缺点：需增加存储操作数或隐含地址的硬件

## 八、立即寻址

**A 本身就是操作数**（立即数，一般采用补码形式）

```
| OP | # | A（立即数） |
```

| 项目 | 内容 |
|------|------|
| 访存次数 | 取指1次 + 执行0次 = **1次** |
| 优点 | 执行阶段不访存，执行时间最短 |
| 缺点 | A 的位数限制立即数范围（n 位补码：-2^(n-1) ~ 2^(n-1)-1） |

## 九、总结对比

| 寻址方式 | 有效地址 | 访存次数（执行期间） |
|----------|----------|----------------------|
| 隐含寻址 | 程序指定 | 0 |
| 立即寻址 | A 即是操作数 | 0 |
| 直接寻址 | EA = A | 1 |
| 一次间接寻址 | EA = (A) | 2 |
| 寄存器寻址 | EA = Rᵢ | 0 |
| 寄存器间接寻址 | EA = (Rᵢ) | 1 |
