# 4.2.2.2 数据寻址（二）——偏移寻址

## 一、偏移寻址的共同特点

以某个地址作为**起点**，形式地址视为**偏移量**：

$$EA = (起点寄存器) + A$$

三种偏移寻址的区别在于**偏移的起点**不同：

| 寻址方式 | 公式 | 起点 |
|----------|------|------|
| 基址寻址 | EA = (BR) + A | 程序的起始存放地址 |
| 变址寻址 | EA = (IX) + A | 程序员自己决定 |
| 相对寻址 | EA = (PC) + A | PC 所指地址 |

## 二、基址寻址

**EA = (BR) + A**

- BR（Base address Register）：基址寄存器
- 可以是**专用基址寄存器**，也可以指定某个**通用寄存器**作为基址寄存器

### 关键特征
- **面向操作系统**：BR 的内容由操作系统或管理程序确定
- 程序执行过程中 **BR 不变**（作为基地址），**A 可变**（作为偏移量）

### 作用：程序浮动
```
程序从地址 0 开始存放：变量a实际地址 = 0 + 5 = 5
程序从地址 100 开始存放：变量a实际地址 = 100 + 5 = 105（BR=100）
```
- 无需修改指令中的地址码即可实现程序在内存中的浮动
- 便于**多道程序设计**

### 优点
1. 可扩大寻址范围（BR 位数 > A 位数）
2. 便于多道程序设计和浮动程序编制

## 三、变址寻址

**EA = (IX) + A**

- IX（Index Register）：变址寄存器（专用或通用寄存器）
- **面向用户**：程序执行过程中 **IX 可由用户改变**（作为偏移量），**A 不变**（作为基地址）

### 对比基址寻址

| | 基址寻址 | 变址寻址 |
|---|----------|----------|
| 不变的 | BR（基地址） | A（基地址，数组首地址） |
| 可变的 | A（偏移量） | IX（偏移量，循环变量） |
| 面向 | 操作系统 | 用户 |

### 作用：数组处理
```
for(int i=0; i<10; i++)
    sum += a[i];

设数组首地址 A = 7，IX 从 0 递增到 9
EA = 7+0, 7+1, 7+2, ..., 7+9  → 依次访问 a[0]~a[9]
```
- 特别适合编制**循环程序**

## 四、基址+变址复合寻址

**EA = (IX) + (BR) + A**

- 实际应用中常需要多种寻址方式复合使用
- 例如：程序装入内存后 BR 指向程序起始地址，A 指向数组首地址（相对于程序起始），IX 作为循环偏移

## 五、相对寻址

**EA = (PC) + A**

- A 是相对于 **PC 所指地址**的位移量（可正可负，补码表示）
- ⚠️ 注意：取出当前指令后 PC 已经 +"1"，所以 A 是相对于下一条指令的偏移

### 示例
```
当前指令存放地址 = 1000
若指令字长 = 2B，则取指后 PC = 1002
若指令字长 = 4B，则取指后 PC = 1004
```

### 作用：程序段浮动
```
直接寻址：条件跳转跳到绝对地址 2 → 程序位置变动后会出错
相对寻址：条件跳转跳到 (PC) + (-4) → 无论程序放在哪里都能正确跳转
```

### 优点
- 操作数地址随 PC 值变化而变化，与指令地址之间总相差一个固定值
- 便于**一段代码在程序内部的浮动**
- 广泛应用于**转移指令**

## 六、完整总结

| 寻址方式 | 有效地址 | 访存次数（执行期间） | 特点 |
|----------|----------|----------------------|------|
| 隐含寻址 | 程序指定 | 0 | 缩短指令字长 |
| 立即寻址 | A 即是操作数 | 0 | 执行最快，A 限制范围 |
| 直接寻址 | EA = A | 1 | 简单，范围受限 |
| 一次间接寻址 | EA = (A) | 2 | 扩大范围，多次访存 |
| 寄存器寻址 | EA = Rᵢ | 0 | 最快，寄存器昂贵 |
| 寄存器间接寻址 | EA = (Rᵢ) | 1 | 需访存但比间接快 |
| 基址寻址 | EA = (BR)+A | 1 | 程序浮动，面向 OS |
| 变址寻址 | EA = (IX)+A | 1 | 数组处理，面向用户 |
| 相对寻址 | EA = (PC)+A | 1 | 转移指令，代码浮动 |
