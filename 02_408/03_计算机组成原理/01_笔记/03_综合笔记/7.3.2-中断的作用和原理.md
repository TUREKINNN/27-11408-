# 7.3.2 中断的作用和原理

## 一、中断的基本概念

**程序中断**：在计算机执行现行程序的过程中，出现某些急需处理的异常情况或特殊请求，CPU 暂时中止现行程序，转去对这些情况进行处理，处理完毕后 CPU 自动返回到原程序断点处继续执行。

### 工作流程
1. **中断请求**：中断源向 CPU 发送中断请求信号
2. **中断响应**：满足响应条件 + 中断判优
3. **中断处理**：中断隐指令 + 中断服务程序

## 二、中断请求的分类

| 类型 | 说明 |
|------|------|
| **非屏蔽中断** | 关中断时也会被响应（如掉电） |
| **可屏蔽中断** | 关中断时不会被响应 |

- **IF（Interrupt Flag）**：存在 PSW 中
  - IF=1：开中断（允许中断）
  - IF=0：关中断（不允许中断）
- 关中断的作用：实现**原子操作**

## 三、中断请求标记

- 每个中断源设置**中断请求标记触发器 INTR**
- INTR=1 表示中断源有请求
- 触发器可组成**中断请求标记寄存器**，可集中在 CPU 中或分散在各中断源中

### CPU 响应中断的 3 个条件
1. 中断源有中断请求
2. CPU 允许中断（开中断）
3. 一条指令执行完毕，且没有更紧迫的任务

> CPU 响应中断的时间：**每条指令执行阶段的结束时刻**

## 四、中断判优

### 实现方式
- **硬件实现**：硬件排队器（设置在 CPU 中或分散在各中断源中）
- **软件实现**：查询程序

### 优先级设置
1. 硬件故障中断 > 软件中断
2. 非屏蔽中断 > 可屏蔽中断
3. DMA 请求 > I/O 设备传送的中断请求
4. 高速设备 > 低速设备
5. 输入设备 > 输出设备
6. 实时设备 > 普通设备

## 五、中断处理过程

### 1. 中断隐指令（CPU 自动完成）

> 并不是一条具体的指令，而是 CPU 检测到中断请求时自动完成的一系列动作

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | **关中断** | 保护中断现场期间不被新中断打断 |
| ② | **保存断点** | 将 PC 内容保存（堆栈或指定单元），保证能正确返回 |
| ③ | **引出中断服务程序** | 取出中断服务程序入口地址→PC |

引出中断服务程序的方法：
- **软件查询法**
- **硬件向量法**：硬件产生向量地址 → 由向量地址找到入口地址

### 硬件向量法
```
排队器输出 → 中断向量地址形成部件 → 向量地址（中断类型号）→ 中断向量表 → 入口地址
```

### 2. 中断服务程序

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | **保护现场** | 保存通用寄存器和状态寄存器内容（如 ACC 的值） |
| ② | **中断服务（设备服务）** | 主体部分，如将字符代码送入打印机缓冲区 |
| ③ | **恢复现场** | 通过出栈/取数指令将保存的信息送回寄存器 |
| ④ | **中断返回** | 通过中断返回指令回到原程序断点处 |

## 六、总结

```
中断隐指令                    中断服务程序
┌──────────────┐          ┌──────────────┐
│ 1. 关中断     │          │ 1. 保护现场   │
│ 2. 保存断点   │    →     │ 2. 中断服务   │
│ 3. 引出服务程序│          │ 3. 恢复现场   │
└──────────────┘          │ 4. 中断返回   │
                          └──────────────┘
```

- **单重中断**：执行中断服务程序时不响应新的中断请求
