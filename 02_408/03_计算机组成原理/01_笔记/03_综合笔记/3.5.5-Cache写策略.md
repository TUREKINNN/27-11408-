# Cache写策略

## 1. 问题背景

CPU修改了Cache中的数据副本，如何确保主存中数据母本的一致性？

> **注意**：读操作不会导致Cache和主存的数据不一致，因此只讨论**写操作**的情况。

---

## 2. 写命中策略

### 2.1 写回法（Write-Back）

**规则**：当CPU对Cache写命中时，只修改Cache的内容，而不立即写入主存，只有当此块被换出时才写回主存。

**Cache行结构**：

```
┌─────────┬─────┬──────────────┬──────────────┐
│ 有效位  │脏位  │   标记(Tag)   │   数据块     │
│ (1bit)  │(1bit)│              │              │
└─────────┴─────┴──────────────┴──────────────┘
```

- **脏位（Dirty Bit）**：表示该块是否被修改过
  - 0：未被修改，替换时无需写回主存
  - 1：已被修改，替换时必须写回主存

**工作流程**：

```
写命中时：
  只修改Cache内容
  设置脏位 = 1

替换时：
  若脏位 = 0 → 直接替换，无需写回
  若脏位 = 1 → 先将Cache块写回主存，再替换
```

**特点**：
- ✅ 减少了访存次数，速度快
- ❌ 存在数据不一致的隐患
- ❌ 需要额外的脏位

---

### 2.2 全写法（Write-Through，写直通法）

**规则**：当CPU对Cache写命中时，必须把数据同时写入Cache和主存。

**写缓冲（Write Buffer）**：

```
CPU → Cache + 写缓冲(FIFO队列) → 主存
```

- 写缓冲由SRAM实现
- CPU写入Cache和写缓冲后即可继续执行
- 由专门的控制电路将写缓冲中的数据逐个写回主存

**特点**：
- ✅ 能保证数据一致性
- ✅ 实现简单
- ❌ 访存次数增加，速度变慢
- ❌ 若写操作频繁，写缓冲可能饱和导致阻塞

---

## 3. 写不命中策略

### 3.1 写分配法（Write-Allocate）

**规则**：当CPU对Cache写不命中时，把主存中的块调入Cache，在Cache中修改。

**工作流程**：

```
写不命中 → 将主存块调入Cache → 在Cache中修改
```

**通常搭配写回法使用**。

---

### 3.2 非写分配法（Not-Write-Allocate）

**规则**：当CPU对Cache写不命中时只写入主存，不调入Cache。

**工作流程**：

```
写不命中 → 直接写入主存，不调入Cache
```

**通常搭配全写法使用**。

---

## 4. 写策略组合

| 写命中策略 | 写不命中策略 | 组合名称 | 特点 |
|-----------|-------------|---------|------|
| 写回法 | 写分配法 | 写回+写分配 | 减少访存，数据可能不一致 |
| 全写法 | 非写分配法 | 全写+非写分配 | 保证一致性，访存次数多 |

---

## 5. 多级Cache的写策略

现代计算机常采用多级Cache：

```
CPU ← → L1 Cache ← → L2 Cache ← → 主存(DRAM)
```

**各级之间的写策略**：

| 层次 | 写策略 |
|------|--------|
| L1 Cache ↔ L2 Cache | 全写法 + 非写分配法 |
| L2 Cache ↔ 主存 | 写回法 + 写分配法 |

**设计原则**：
- 离CPU越近，速度越快，容量越小
- 离CPU越远，速度越慢，容量越大
- L1与L2之间：速度快，采用全写法保证一致性
- L2与主存之间：速度差异大，采用写回法减少访存

---

## 6. 三种I/O控制方式对比（与写策略相关）

| 控制方式 | 数据传送路径 | CPU参与程度 |
|----------|-------------|------------|
| 程序查询方式 | I/O ↔ CPU ↔ 主存 | CPU全程参与 |
| 程序中断方式 | I/O ↔ CPU ↔ 主存 | CPU参与数据传送 |
| DMA方式 | I/O ↔ 主存（直接通路） | CPU只参与预处理和后处理 |

---

## 7. 典型例题

**题目**：某计算机系统中，Cache采用写回法和写分配法，主存容量为4MB，Cache容量为8KB，块大小为64B。

**问题1**：若Cache采用直接映射，计算地址位数分配。

**解答**：
- 主存地址 = log₂(4MB) = 22位
- 块内地址 = log₂(64B) = 6位
- Cache行数 = 8KB / 64B = 128 = 2⁷
- 行号 = 7位
- 标记 = 22 - 6 - 7 = 9位

**问题2**：若某Cache行的有效位=1，脏位=1，标记=100000001B，该行对应主存地址是多少？

**解答**：
- 主存块号 = 标记 + 行号 = 100000001B + 行号
- 若行号为0，则主存块号 = 100000001B = 257
- 主存地址 = 257 × 64 = 16448

---

## 8. Cache写策略总结

```
                    ┌─ 写回法（Write-Back）
                    │   - 只改Cache，替换时写回
                    │   - 需要脏位
                    │   - 速度快，可能不一致
        ┌─ 写命中 ──┤
        │           └─ 全写法（Write-Through）
        │               - 同时写Cache和主存
        │               - 用写缓冲加速
        │               - 保证一致性
写策略 ──┤
        │           ┌─ 写分配法（Write-Allocate）
        │           │   - 调入Cache再修改
        └─ 写不命中 ─┤
                    └─ 非写分配法（Not-Write-Allocate）
                        - 直接写主存，不调入Cache
```

---

## 9. 考研重点

1. **写回法与全写法的区别**：脏位的作用、写缓冲的作用
2. **写分配法与非写分配法的区别**：写不命中时是否调入Cache
3. **写策略组合**：写回法通常搭配写分配法，全写法通常搭配非写分配法
4. **多级Cache的写策略**：不同层次采用不同策略
5. **综合应用**：映射方式+替换算法+写策略的综合分析
