# 5.6.3 五段式指令流水线

## 一、机器周期的设置

五段式流水线：

```
IF → ID → EX → M → WB
取指  译码  执行  访存  写回
```

- 流水线每个功能段部件后面都要有一个**缓冲寄存器（锁存器）**，保存本段执行结果，提供给下一流水段
- 各段实际耗时可能不同（如 100ns, 80ns, 70ns, 50ns, 50ns）
- 为方便设计，以**最长耗时为准**（此处取 100ns）
- 理想情况下，每个机器周期只消耗一个时钟周期

## 二、五类常见指令的执行过程

### 1. 运算类指令

| 阶段 | 操作 |
|------|------|
| IF | 根据 PC 从指令 Cache 取指令至锁存器 |
| ID | 取出操作数至锁存器 |
| EX | 运算，结果存入 EX 段锁存器 |
| M | **空段** |
| WB | 将运算结果写回指定寄存器 |

示例：
- `ADD Rs, Rd` → (Rs)+(Rd)→Rd
- `ADD #996, Rd` → 996+(Rd)→Rd
- `SHL Rd` → (Rd)<<<2→Rd（算术左移）

### 2. LOAD 指令

| 阶段 | 操作 |
|------|------|
| IF | 根据 PC 取指令 |
| ID | 基址寄存器值→锁存器 A，偏移量→Imm |
| EX | 计算有效地址 |
| M | 从数据 Cache 取数到锁存器 |
| WB | 将取出的数写回寄存器 |

示例：`LOAD Rd, 996(Rs)` → (996+(Rs))→Rd

> RISC 处理器只有 LOAD 和 STORE 指令才能访问主存

### 3. STORE 指令

| 阶段 | 操作 |
|------|------|
| IF | 根据 PC 取指令 |
| ID | 基址寄存器值→A，偏移量→Imm，要存的数→B |
| EX | 计算有效地址，B 的内容→锁存器 Store |
| M | 写入数据 Cache |
| WB | **空段** |

示例：`STORE Rs, 996(Rd)` → Rs→(996+(Rd))

### 4. 条件转移指令

| 阶段 | 操作 |
|------|------|
| IF | 根据 PC 取指令 |
| ID | 比较的两个数→A、B；偏移量→Imm |
| EX | 比较两个数 |
| M | 将目标 PC 值写回 PC（WrPC 段） |
| WB | **空段** |

示例：
- `beq Rs, Rt, #偏移量` → 若(Rs)==(Rt)，则 PC = PC + 指令字长 + 偏移量×指令字长
- `bne Rs, Rt, #偏移量` → 若(Rs)!=(Rt)，则 PC = PC + 指令字长 + 偏移量×指令字长

> 转移类指令常采用**相对寻址**。IF 段结束后 PC 会自动 +"1"

### 5. 无条件转移指令

| 阶段 | 操作 |
|------|------|
| IF | 根据 PC 取指令 |
| ID | 偏移量→Imm |
| EX | 将目标 PC 值写回 PC（WrPC 段） |
| M | **空段** |
| WB | **空段** |

示例：`jmp #偏移量` → PC = PC + 指令字长 + 偏移量×指令字长

> WrPC 段耗时比 EX 段更短，可安排在 EX 时间内完成。WrPC 段越早完成，越能避免控制冲突。

## 三、流水线约束

> **只有上一条指令进入 ID 段后，下一条指令才能开始 IF 段**，否则会覆盖 IF 段锁存器的内容。

## 四、例题分析

**题目**：假设指令流水线采用"按序发射，按序完成"方式，没有转发技术，同一寄存器读写不能在同一时钟周期内进行。执行 `x=a+b`：

```
I1: LOAD R1, [a]      M[a] → R1
I2: LOAD R2, [b]      M[b] → R2
I3: ADD R1, R2        (R1)+(R2) → R2
I4: STORE R2, [x]     (R2) → M[x]
```

**I3 的 ID 段被阻塞原因**：I3 与 I1、I2 存在**数据相关**——I3 需要读取 R1、R2，但 I1、I2 尚未将结果写回寄存器

**I4 的 IF 段被阻塞原因**：I4 的 IF 段必须在 I3 进入 ID 段后才能开始，否则会覆盖 IF 段锁存器的内容
