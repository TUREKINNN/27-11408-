# 7.1 I/O 系统和 I/O 控制方式

## 一、I/O 系统基本概念

**I/O 设备**：可以将数据输入到计算机，或接收计算机输出数据的外部设备

| 类型 | 示例 |
|------|------|
| 输入设备 | 鼠标、键盘 |
| 输出设备 | 显示器、打印机 |
| 输入/输出设备 | 硬盘、光盘（外存设备） |

## 二、I/O 接口（I/O 控制器）

**I/O 接口**：又称 I/O 控制器、设备控制器，负责协调主机与外部设备之间的数据传输

- I/O 控制器就是一块芯片，常被集成在主板上（南桥芯片内部）
- 不同设备有不同标准的 I/O 接口（USB、SATA 3.0 等）

### I/O 接口中的寄存器
- **数据寄存器**：暂存传输数据
- **控制寄存器**：存放控制命令
- **状态寄存器**：存放设备状态

## 三、I/O 控制方式

### 1. 程序查询方式

```
CPU 执行程序 → 启动 I/O → CPU 查询等待并传输 I/O 数据 → CPU 执行程序
```

- CPU 不断轮询检查 I/O 控制器中的**状态寄存器**
- 检测到"已完成"后，从数据寄存器取出输入数据
- **CPU 利用率低**，CPU 和 I/O 串行工作

### 2. 程序中断方式

```
CPU 执行程序 → 启动 I/O → I/O 准备 → 中断请求 → CPU 处理中断服务程序 → CPU 执行程序
```

- 等待 I/O 时 CPU 可以先执行其他程序
- I/O 完成后向 CPU 发出**中断请求**
- 适合低速设备

> 思考：快速 I/O 设备（如磁盘）每准备好一个字就中断一次 → CPU 大量时间处理中断，利用率严重下降

### 3. DMA 方式

```
CPU 执行程序 → 启动 I/O → I/O 准备 → DMA 请求 → CPU 让出总线 → DMA 传送 → 传送完成中断
```

- **DMA（Direct Memory Access）**：主存与高速 I/O 之间有直接数据通路
- CPU 向 DMA 控制器发出读/写命令（主存地址、磁盘地址、数据量）
- DMA 控制器自动控制数据读写，**每完成一整块数据**（如 1KB）才向 CPU 发一次中断
- DMA 控制器与主存每次传送 **1 个字**

### 4. 通道控制方式

- 通道 = "弱鸡版 CPU"，能识别并执行通道指令
- CPU 向通道发出 I/O 指令，指明通道程序位置和 I/O 设备
- 通道执行通道程序，控制 I/O 设备完成一系列任务
- 任务完成后通道向 CPU 发中断请求

## 四、四种控制方式对比

| 方式 | CPU 参与程度 | 数据传送 | 适用场景 |
|------|-------------|----------|----------|
| 程序查询 | 全程参与 | CPU 控制 | 低速设备 |
| 程序中断 | 间断参与 | CPU 控制 | 低速设备 |
| DMA | 仅预处理+后处理 | DMA 控制器控制 | 高速设备 |
| 通道 | 仅发 I/O 指令 | 通道控制 | 大量 I/O 设备 |

## 五、I/O 系统组成

### I/O 硬件
- 外部设备、I/O 接口、I/O 总线等

### I/O 软件
- 驱动程序、用户程序、管理程序等
- 通过 **I/O 指令**和**通道指令**实现主机和 I/O 设备的信息交换

#### I/O 指令格式
```
| 操作码 | 命令码 | 设备码 |
```
- 操作码：CPU 要对 I/O 接口做什么
- 命令码：I/O 接口要对设备做什么
- 设备码：对哪个设备操作

#### 通道指令
- 通道能识别的指令，通道程序提前编制好放在主存中
- CPU 执行 I/O 指令对通道发出命令，由通道执行通道指令代替 CPU 管理 I/O 设备
