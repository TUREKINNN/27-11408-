# 4.1 指令的基本格式

## 一、指令的定义

- **指令（机器指令）**：指示计算机执行某种操作的命令，是计算机运行的**最小功能单位**
- **指令系统（指令集）**：一台计算机的所有指令的集合
- 一台计算机只能执行自己指令系统中的指令（如 x86 架构、ARM 架构）

## 二、指令格式

一条指令 = **操作码（OP）** + **地址码（A）**

| 字段 | 含义 |
|------|------|
| 操作码（OP） | 指明用户要干什么（加减乘除、停机、中断等） |
| 地址码（A） | 指明对谁进行操作（0~4个地址码） |

根据地址码数目不同，分为：

### 1. 零地址指令
```
| OP |
```
- 不需要操作数：空操作、停机、关中断等
- **堆栈计算机**：两个操作数隐含存放在栈顶和次栈顶，计算结果压回栈顶
  - 数据结构采用"后缀表达式"

### 2. 一地址指令
```
| OP | A₁ |
```
- **单操作数**：`OP(A₁)→A₁`，如加1、减1、取反、求补
  - 访存3次：取指→读A₁→写A₁
- **双操作数（一个隐含）**：`(ACC)OP(A₁)→ACC`
  - 访存2次：取指→读A₁

### 3. 二地址指令
```
| OP | A₁（目的） | A₂（源） |
```
- 指令含义：`(A₁)OP(A₂)→A₁`
- 访存4次：取指→读A₁→读A₂→写A₁

### 4. 三地址指令
```
| OP | A₁ | A₂ | A₃（结果） |
```
- 指令含义：`(A₁)OP(A₂)→A₃`
- 访存4次：取指→读A₁→读A₂→写A₃

### 5. 四地址指令
```
| OP | A₁ | A₂ | A₃（结果） | A₄（下址） |
```
- 指令含义：`(A₁)OP(A₂)→A₃`，`A₄` = 下一条将要执行指令的地址
- 正常情况下取指令后 PC+1；四地址指令执行后将 PC 修改为 A₄ 所指地址

> **地址码位数的影响**：n 位地址码的直接寻址范围 = 2ⁿ。若指令总长度固定，地址码数量越多，寻址能力越差。

## 三、指令按指令长度分类

| 概念 | 定义 |
|------|------|
| **指令字长** | 一条指令的总长度（可能变） |
| **机器字长** | CPU 一次整数运算所能处理的二进制数据位数（与 ALU 相关） |
| **存储字长** | 一个存储单元中的二进制代码位数（通常与 MDR 位数相同） |

- **定长指令字结构**：所有指令长度相等
- **变长指令字结构**：各种指令长度不等
- 指令字长会影响取指令所需时间（如双字长指令需两次访存）

## 四、指令按操作码长度分类

| 类型 | 特点 |
|------|------|
| **定长操作码** | 所有指令操作码长度相同，译码电路简单，灵活性低。n 位 → 2ⁿ 条指令 |
| **可变长操作码** | 各指令操作码长度可变，译码电路复杂，灵活性高 |

- **扩展操作码指令格式**：定长指令字结构 + 可变长操作码，不同地址数的指令使用不同长度的操作码

## 五、指令按操作类型分类

| 类型 | 示例 |
|------|------|
| **数据传送** | LOAD（存储器→寄存器）、STORE（寄存器→存储器） |
| **算术逻辑** | 加减乘除、与或非异或、位操作 |
| **移位操作** | 算术移位、逻辑移位、循环移位 |
| **转移操作** | 无条件转移JMP、条件转移JZ/JO/JC、CALL/RETURN、Trap |
| **输入输出** | CPU 寄存器与 I/O 端口之间的数据传送 |
