# 05 中央处理器

> **大题**常考流水线性能计算（吞吐率/加速比）和冒险判断与处理方案。

---

## 一、CPU的功能与基本结构

### 1.1 CPU的功能

| 功能 | 说明 |
|------|------|
| **指令控制** | 取指令、分析指令、执行指令（程序的顺序控制） |
| **操作控制** | 产生各种操作控制信号，指挥各部件协调工作 |
| **时间控制** | 对各种操作加以时间上的控制 |
| **数据加工** | 对数据进行算术运算和逻辑运算 |
| **中断处理** | 处理异常情况和特殊请求 |

### 1.2 CPU的基本结构

```
┌─────────────────────────────────────────────────┐
│                      CPU                         │
│  ┌──────────┐  ┌──────────────┐  ┌───────────┐  │
│  │  运算器   │  │   控制器      │  │ 寄存器组   │  │
│  │          │  │              │  │           │  │
│  │ ALU      │  │ PC           │  │ 通用寄存器 │  │
│  │ 暂存器   │  │ IR           │  │ 暂存寄存器 │  │
│  │ 移位器   │  │ MAR          │  │ 状态寄存器 │  │
│  │ ACC      │  │ 时序系统     │  │ 地址寄存器 │  │
│  │ 状态寄存器│  │ 微操作信号   │  │ ...       │  │
│  └──────────┘  └──────────────┘  └───────────┘  │
│                  ↕ 内部总线 ↕                     │
└─────────────────────┬───────────────────────────┘
                      ↕
              系统总线 (数据/地址/控制)
```

#### 运算器
- **ALU**：执行算术和逻辑运算
- **暂存器**：暂存操作数和运算结果
- **累加器（ACC）**：暂存运算结果
- **程序状态字寄存器（PSW）**：存放标志位（CF/ZF/SF/OF等）

#### 控制器
- **程序计数器（PC）**：存放下一条指令的地址，自动 +1（或由转移指令修改）
- **指令寄存器（IR）**：存放当前正在执行的指令
- **存储器地址寄存器（MAR）**：存放访存地址
- **存储器数据寄存器（MDR）**：存放读/写的数据
- **时序系统**：产生时序信号
- **微操作信号发生器**：产生微操作控制信号

#### 寄存器组
- **用户可见寄存器**：通用寄存器、地址寄存器、条件码寄存器
- **用户不可见寄存器**：MAR、MDR、IR、暂存寄存器

> PC存放的是**下一条指令的地址**，MAR存放的是**当前要访问的存储单元地址**。取指阶段，PC的内容送MAR；执行阶段访存时，指令中的地址码字段送MAR。

## 二、数据通路

### 2.1 数据通路的概念

**数据通路**：数据在功能部件之间传送的路径，包括数据通路上流经的部件（如ALU、寄存器、多路选择器等）。

### 2.2 寄存器传输级（RTL）描述

用寄存器传输语言描述微操作：

| 符号 | 含义 |
|------|------|
| `R0 ← R1` | 将R1的内容传送到R0 |
| `R0 ← R0 + R1` | R0和R1的内容相加，结果送R0 |
| `R0 ← M[R1]` | 以R1为地址读存储器，结果送R0 |
| `M[R1] ← R0` | 将R0的内容写入以R1为地址的存储单元 |
| `PC ← PC + 1` | PC自增 |

### 2.3 典型数据通路示例

```
取指周期微操作序列：
T1: MAR ← (PC)
T2: MDR ← M[MAR], PC ← (PC) + 1
T3: IR ← (MDR)

执行 ADD R1, [addr] 的微操作序列：
T1: MAR ← addr（地址码字段）
T2: MDR ← M[MAR]
T3: R1 ← (R1) + (MDR)
```

## 三、硬布线控制器 vs 微程序控制器

### 3.1 硬布线控制器

- **原理**：用组合逻辑电路（门电路、触发器等）直接产生微操作控制信号
- **微操作信号** = f（指令操作码、时序信号、标志状态、外来信号）
- **特点**：速度快，但设计不规整、修改/扩展困难

### 3.2 微程序控制器

- **原理**：仿照程序设计方法，将每条机器指令的微操作序列编成一段**微程序**，存入**控制存储器（CM）**
- **微指令格式**：

| 字段 | 说明 |
|------|------|
| **操作控制字段** | 产生各种微操作控制信号（直接编码/字段直接编码/字段间接编码） |
| **顺序控制字段** | 指出下一条微指令的地址（下地址字段 / 判断测试字段） |

- **微程序存储**：控制存储器（CM），ROM实现，按地址访问

### 3.3 对比表

| 对比项 | 硬布线控制器 | 微程序控制器 |
|--------|------------|------------|
| **实现方式** | 组合逻辑电路 | 控制存储器 + 微指令 |
| **速度** | 快（门电路延迟小） | 较慢（需访问CM） |
| **设计规整性** | 不规整，设计复杂 | 规整，类似程序设计 |
| **可扩展性** | 差，修改需重新设计硬件 | 好，修改微程序即可 |
| **适用场景** | RISC（指令简单、格式规整） | CISC（指令复杂、功能多样） |
| **调试** | 困难 | 容易 |
| **成本** | 大规模指令系统成本高 | 需要CM，增加硬件成本 |

> - **直接编码（水平型）**：每一位控制一个微操作，速度快但微指令字长
> - **字段直接编码**：将微指令分为若干字段，每段独立编码，需**字段互斥**
> - **字段间接编码**：一个字段的含义受另一字段影响

## 四、指令流水线

### 4.1 基本原理

将一条指令的执行过程分为 **k 个阶段**，每个阶段由独立的硬件完成，各阶段重叠执行。

典型五级流水线：

```
阶段：  IF    ID    EX    MEM   WB
       取指  译码  执行  访存  写回
```

### 4.2 时空图

```
指令  时间→
      |  1  |  2  |  3  |  4  |  5  |  6  |  7  |  8  |  9  |
指令1  [IF]  [ID]  [EX]  [MEM] [WB]
指令2        [IF]  [ID]  [EX]  [MEM] [WB]
指令3              [IF]  [ID]  [EX]  [MEM] [WB]
指令4                    [IF]  [ID]  [EX]  [MEM] [WB]
指令5                          [IF]  [ID]  [EX]  [MEM] [WB]
```

> 流水线满载后，每个时钟周期完成一条指令。

### 4.3 性能指标推导

设：**k** = 流水线级数，**n** = 指令条数，**τ** = 流水线时钟周期

#### 各阶段时间

- 每个阶段耗时均为 τ（取最长阶段为瓶颈）
- **非流水线**执行 n 条指令总时间：$T_{np} = n \cdot k \cdot \tau$
- **流水线**执行 n 条指令总时间：

$$T_{pipeline} = k \cdot \tau + (n-1) \cdot \tau = (k + n - 1) \cdot \tau$$

> 第一条指令需要 k 个时钟周期"装入"流水线，之后每条指令只需 1 个时钟周期。

#### 吞吐率（Throughput）

$$TP = \frac{n}{T_{pipeline}} = \frac{n}{(k + n - 1) \cdot \tau}$$

- 当 $n \to \infty$ 时，$TP_{max} = \frac{1}{\tau}$（最大吞吐率）

#### 加速比（Speedup）

$$S = \frac{T_{np}}{T_{pipeline}} = \frac{n \cdot k \cdot \tau}{(k + n - 1) \cdot \tau} = \frac{k \cdot n}{k + n - 1}$$

- 当 $n \to \infty$ 时，$S_{max} = k$（最大加速比等于流水线级数）

#### 流水线效率（Efficiency）

$$E = \frac{S}{k} = \frac{n}{k + n - 1} = \frac{TP}{TP_{max}}$$

> 某四级流水线，τ=5ns，执行100条指令：
> - $T = (4 + 100 - 1) \times 5 = 515$ ns
> - $TP = 100 / 515 \approx 0.194$ 条/ns
> - $S = 4 \times 100 / 103 \approx 3.88$
> - $E = 3.88 / 4 \approx 97.1\%$

## 五、流水线冒险及处理

### 5.1 结构冒险（资源冲突）

- **原因**：多条指令同时需要同一硬件资源
- **典型**：指令和数据共用一个存储器（取指和访存冲突）
- **解决**：
  - 资源复制：分离指令存储器和数据存储器（**哈佛结构**）
  - 插入暂停（stall），等待资源释放

### 5.2 数据冒险

#### 三种类型

| 类型 | 全称 | 含义 | 例子 |
|------|------|------|------|
| **RAW** | Read After Write | 后续指令读取前面指令还未写入的数据 | `ADD R1,R2,R3` 后 `SUB R4,R1,R5` |
| **WAR** | Write After Read | 后续指令写入前面指令还未读取的数据（乱序执行时出现） | `ADD R1,R2,R3` 后 `SUB R2,R4,R5` |
| **WAW** | Write After Write | 后续指令写入前面指令还未写入的目标（乱序执行时出现） | `ADD R1,R2,R3` 后 `MUL R1,R4,R5` |


#### RAW举例

```
ADD R1, R2, R3    # 写R1在WB阶段
SUB R4, R1, R5    # 读R1在ID阶段 → 冲突！
```

### 5.3 控制冒险

- **原因**：分支指令改变程序执行流，后续取指地址不确定
- **损失**：分支指令到确定目标地址之间的时钟周期（分支损失）
- **典型**：条件分支、无条件跳转、函数调用/返回

### 5.4 冒险处理方法

#### 转发/旁路（Forwarding/Bypassing）

```
不等写回，直接将结果从流水线寄存器转发到需要的位置：

EX/MEM ──→ ALU输入（解决EX-EX转发）
MEM/WB ──→ ALU输入（解决MEM-EX转发）

   IF    ID    EX    MEM   WB
         ↓ 读R1     ↑
   ADD   R1, R2, R3
   SUB   R4, R1, R5
         ↑ 转发 ─────┘
         (从EX/MEM寄存器直接送ALU输入)
```

#### 插入气泡（Stall/Bubble）

```
当转发无法解决时，插入空操作（NOP），暂停流水线：

   ADD   IF    ID    EX    MEM   WB
   SUB         IF    [stall] ID   EX   MEM   WB
   AND               IF    [stall] ID  EX   MEM   WB
```

> 适用于 Load-Use 冒险：`LW R1, 0(R2)` 后 `ADD R3, R1, R4`，数据在MEM阶段才可用。

#### 分支预测

| 方式 | 说明 |
|------|------|
| **静态预测-总是不转** | 预测分支不跳转，继续顺序取指 |
| **静态预测-总是转** | 预测分支跳转，总是取目标地址指令 |
| **静态预测-按方向** | 向后跳转预测为转（循环），向前跳转预测不转 |
| **动态预测-1位** | 根据上次执行结果预测（简单但错误率高） |
| **动态预测-2位** | 用2位饱和计数器，需连续两次误判才改变预测方向 |

#### 2位饱和计数器状态转换

```
00(强不转) → 01(弱不转) → 10(弱转) → 11(强转)
    ↑ 实际转                      实际不转 ↓
    └──────────── 反向转移 ───────────────┘
```

### 5.5 冒险处理总结

| 冒险类型 | 原因 | 处理方法 |
|----------|------|----------|
| 结构冒险 | 资源冲突 | 资源复制（哈佛结构）、流水线暂停 |
| 数据冒险 RAW | 数据依赖 | 转发/旁路、插入气泡、编译器调度 |
| 数据冒险 WAR/WAW | 乱序执行相关 | 寄存器重命名 |
| 控制冒险 | 分支改变执行流 | 分支预测、延迟分支、提前计算 |

> 1. 给出指令序列，画出流水线时空图，判断冒险类型
> 2. 计算吞吐率、加速比、效率
> 3. 加入冒险处理（转发/气泡）后的实际执行时间

---

> - 流水线总时间：$T = (k + n - 1) \cdot \tau$
> - 吞吐率：$TP = \frac{n}{(k+n-1)\tau}$
> - 加速比：$S = \frac{kn}{k+n-1}$
> - 效率：$E = \frac{n}{k+n-1}$

## 六、超标量与动态流水线（考纲新增）

### 6.1 超标量处理器

**超标量（Superscalar）**：每个时钟周期可同时发射**多条指令**的处理器。

| 项目 | 说明 |
|------|------|
| 原理 | 配备多套功能部件，同时执行多条指令 |
| 指令级并行 | 通过硬件动态发现指令间的并行性 |
| 实现 | 指令窗口 + 动态调度 + 多发射 |

**超标量 vs 流水线**：
- 流水线：每个周期取**一条**指令，分阶段执行
- 超标量：每个周期取**多条**指令，多条流水线并行

### 6.2 动态流水线

**动态流水线**：流水线的各段可以**不按固定顺序**执行不同指令。

与静态流水线的区别：
- 静态：同一时刻只能执行一种功能组合
- 动态：各段独立工作，可同时处理不同功能

## 七、多处理器基本概念（考纲新增）

### 7.1 指令集体系结构分类（Flynn分类）

| 类型 | 全称 | 说明 | 举例 |
|------|------|------|------|
| **SISD** | 单指令单数据 | 传统单处理器 | 早期计算机 |
| **SIMD** | 单指令多数据 | 一条指令处理多个数据 | GPU、向量处理器 |
| **MISD** | 多指令单数据 | 实际不存在 | — |
| **MIMD** | 多指令多数据 | 多个处理器独立执行 | 多核CPU、集群 |

### 7.2 硬件多线程

**概念**：通过硬件支持在一个物理处理器上同时维护多个线程的上下文。

| 类型 | 说明 |
|------|------|
| **细粒度多线程** | 每个时钟周期切换线程（轮流执行） |
| **粗粒度多线程** | 只在遇到长延迟事件（如Cache缺失）时切换 |
| **同时多线程（SMT）** | 一个时钟周期内同时执行多个线程的指令（如Intel超线程） |

### 7.3 多核处理器（Multi-core）

**概念**：单个芯片上集成**多个独立的处理器核心**。

```
┌────────────────────────────────────┐
│            多核处理器               │
│  ┌─────────┐  ┌─────────┐         │
│  │ Core 0  │  │ Core 1  │  ...    │
│  │ L1 Cache│  │ L1 Cache│         │
│  └────┬────┘  └────┬────┘         │
│       └─────┬──────┘              │
│         L2 Cache (共享)            │
│             │                     │
│         L3 Cache (共享)            │
│             │                     │
│         主存接口                   │
└────────────────────────────────────┘
```

### 7.4 共享内存多处理器（SMP）

**SMP（Symmetric Multi-Processing）**：多个处理器**共享同一主存**，通过总线或交叉开关互连。

| 特点 | 说明 |
|------|------|
| 对称性 | 所有处理器地位相同 |
| 共享内存 | 统一地址空间 |
| 通信方式 | 通过读写共享内存 |
| 缓存一致性 | 需要Cache一致性协议（如MESI） |
