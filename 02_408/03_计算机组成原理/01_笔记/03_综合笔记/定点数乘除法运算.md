# 定点数乘除法运算

> [[02_408/03_计算机组成原理/01_笔记/03_综合笔记/02-数据的表示和运算]]

## 定点数乘法

### 原码一位乘法

#### 思想

从最低位开始，逐位与被乘数相乘，右移累加器。

#### 步骤

```
① 初始化：部分积 = 0，乘数寄存器
② 循环 n 次（n 为数值位）：
   - 乘数最低位为 1：部分积 += 被乘数
   - 部分积和乘数一起右移一位
③ 符号位单独处理：结果符号 = 两数符号异或
```

#### 特点

- 简单直观
- 需要 n 次加法和 n 次移位
- 符号位单独处理

### 补码一位乘法（Booth 算法）

#### 思想

通过编码减少加法次数。

#### 编码规则

观察乘数相邻两位的变化：
| 当前位 | 右边位 | 操作 |
|--------|--------|------|
| 0 | 0 | 部分积右移 |
| 0 | 1 | 部分积 += [y]补，右移 |
| 1 | 0 | 部分积 += [-y]补，右移 |
| 1 | 1 | 部分积右移 |

#### 步骤

```
① 初始化：部分积 = 0，乘数末位补 0
② 循环 n 次：
   - 按规则决定是否加减被乘数
   - 部分积和乘数一起算术右移
③ 末位不移位
```

#### 特点

- 适合补码运算
- 减少加法次数
- 处理有符号数方便

## 定点数除法

### 恢复余数法

#### 思想

试减法，若结果为负则恢复余数。

#### 步骤

```
① 初始化：余数 = 0
② 循环 n 次：
   - 余数和被除数左移一位
   - 余数 -= 除数
   - 若结果 < 0：恢复余数（+= 除数），商 0
   - 若结果 >= 0：商 1
③ 符号位单独处理
```

#### 缺点

- 余数恢复需要额外步骤
- 速度较慢

### 加减交替法（不恢复余数法）

#### 思想

若减法结果为负，**下次加除数**；若为正，**下次减除数**。

#### 步骤

```
① 初始化：余数 = 被除数
② 循环 n 次：
   - 余数和商左移一位
   - 若余数 >= 0：余数 -= 除数，商 1
   - 若余数 < 0：余数 += 除数，商 0
③ 符号位单独处理
```

#### 特点

- 不需要恢复余数
- 速度比恢复余数法快
- 加减交替进行

## 乘法器实现

### 阵列乘法器

- 多个加法器并行排列
- 速度快，硬件开销大
- 适合高速运算

### 乘法器优化

1. **进位保留加法器**（CSA）
2. **Wallace 树**
3. **流水线乘法器**

## 对比表

| 方法 | 加法次数 | 移位次数 | 复杂度 |
|------|---------|---------|--------|
| 原码一位乘 | n | n | 简单 |
| Booth 算法 | ≤ n | n | 中等 |
| 恢复余数法 | 2n | n | 简单 |
| 加减交替法 | n+1 | n | 中等 |
