## 1. 指令格式

### 1.1 指令的基本格式

<span style="background-color:#ccc1d9">**指令格式**</span>：操作码 + 地址码。

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│   操作码     │   地址码1    │   地址码2    │   地址码3    │
│     OP       │      A1      │      A2      │      A3      │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

### 1.2 地址码的分类

| 类型 | 地址数量 | 示例 |
|------|----------|------|
| **零地址指令** | 0 | NOP |
| **一地址指令** | 1 | INC A |
| **二地址指令** | 2 | ADD A, B |
| **三地址指令** | 3 | ADD A, B, C |
| **多地址指令** | >3 | 复杂指令 |

### 1.3 操作码的设计

#### 1.3.1 固定长度操作码

```
操作码位数固定，指令条数 = 2^n
```

**优点**：译码简单，速度快。

**缺点**：浪费空间，扩展困难。

#### 1.3.2 扩展操作码

```
短操作码：常用指令
长操作码：不常用指令
```

**优点**：节省空间，支持更多指令。

**缺点**：译码复杂。

**设计原则**：

1. 短操作码不能是长操作码的前缀
2. 常用指令使用短操作码

## 3. 指令系统的性能要求

### 3.1 完备性

**定义**：指令系统应包含所有必要的指令。

**衡量**：能否用现有指令实现任意运算。

### 3.2 有效性

**定义**：用少量指令和时间完成任务。

**衡量**：指令的执行效率。

### 3.3 规整性

**定义**：指令格式和操作码设计统一。

**衡量**：指令的对称性和规律性。

### 3.4 兼容性

**定义**：不同型号计算机的指令系统兼容。

**衡量**：新机型能否运行旧机型程序。

## 4. RISC与CISC

### 4.1 CISC

<span style="background-color:#ccc1d9">**CISC**</span>：复杂指令系统计算机。

**特点**：

- 指令数量多，功能复杂
- 寻址方式多样
- 指令长度不固定
- 强调硬件实现复杂功能
- 代表：Intel x86系列

### 4.2 RISC

<span style="background-color:#ccc1d9">**RISC**</span>：精简指令系统计算机。

**特点**：

- 指令数量少，功能简单
- 寻址方式简单
- 指令长度固定
- 强调软件实现复杂功能
- 代表：ARM、MIPS

### 4.3 RISC与CISC的比较

| 对比项 | CISC | RISC |
|--------|------|------|
| **指令数量** | 多（200+） | 少（50左右） |
| **指令长度** | 不固定 | 固定 |
| **寻址方式** | 多（10+） | 少（3-4种） |
| **CPI** | 高（2-15） | 低（1-2） |
| **编译器** | 简单 | 复杂 |
| **硬件复杂度** | 高 | 低 |

### 4.4 RISC的关键技术

#### 4.4.1 寄存器窗口技术

**思路**：函数调用时使用不同的寄存器组，减少栈操作。

#### 4.4.2 指令流水线

**思路**：多条指令重叠执行，提高效率。

#### 4.4.3 延迟加载技术

**思路**：在加载指令后插入无关指令，隐藏延迟。

#### 4.4.4 分支预测

**思路**：预测分支方向，提前执行。

## 5. 指令的执行过程

### 5.1 指令周期

<span style="background-color:#ccc1d9">**指令周期**</span>：从取指令到执行完的时间。

```
指令周期 = 取指周期 + 间址周期 + 执行周期 + 中断周期
```

### 5.2 取指周期

```
1. PC → MAR → 地址总线
2. 发读命令 → 主存
3. 数据 → MDR → IR
4. PC + 1 → PC
```

### 5.3 间址周期

```
1. IR的地址码 → MAR → 地址总线
2. 发读命令 → 主存
3. 数据 → MDR → IR的地址码字段
```

### 5.4 执行周期

```
根据IR的操作码执行相应操作：
- 算术运算：ALU运算
- 数据传送：寄存器/存储器操作
- 程序控制：修改PC
```

### 5.5 中断周期

```
1. 关中断
2. 保存断点：PC → MDR → 主存
3. 保存状态：PSW → MDR → 主存
4. 中断服务程序地址 → PC
```

## 6. 指令的流水执行

### 6.1 流水线的概念

<span style="background-color:#ccc1d9">**指令流水线**</span>：将指令执行过程分解为多个阶段，并行执行。

### 6.2 流水线的阶段划分

```
取指（IF）→ 译码（ID）→ 执行（EX）→ 访存（MEM）→ 写回（WB）
```

### 6.3 流水线的性能指标

#### 6.3.1 吞吐率

$$
\text{吞吐率} = \frac{\text{完成指令数}}{\text{总时间}}
$$

#### 6.3.2 加速比

$$
\text{加速比} = \frac{\text{非流水线时间}}{\text{流水线时间}}
$$

#### 6.3.3 效率

$$
\text{效率} = \frac{\text{n个任务占用的时空区}}{\text{n个任务实际占用的时空区}}
$$

### 6.4 流水线的冲突

#### 6.4.1 结构冲突

**原因**：硬件资源不足。

**解决**：增加资源或暂停指令。

#### 6.4.2 数据冲突

**原因**：指令间存在数据依赖。

**解决**：数据旁路、指令重排、暂停。

#### 6.4.3 控制冲突

**原因**：分支指令导致PC不确定。

**解决**：分支预测、延迟分支。

## 7. 典型例题

### 例题 1：指令格式设计

**题目**：设计一个二地址指令系统，操作码4位，地址码12位，指令字长32位。求可表示的指令条数和寻址范围。

**解答**：

```
操作码：4位，可表示2^4 = 16种操作

地址码：12位，寻址范围2^12 = 4KB

指令格式：
┌──────┬────────────┬────────────┐
│ OP   │    A1      │    A2      │
│ 4位  │    12位    │    12位    │
└──────┴────────────┴────────────┘

指令条数：16条
寻址范围：4KB
```

**总结**：指令格式设计需要平衡操作种类和寻址范围。

**来源**：408历年真题，考查方式：计算题

### 例题 2：RISC与CISC对比

**题目**：比较RISC和CISC的特点。

**解答**：

```
CISC特点：
- 指令数量多，功能复杂
- 寻址方式多样
- 指令长度不固定
- CPI高
- 硬件复杂度高

RISC特点：
- 指令数量少，功能简单
- 寻址方式简单
- 指令长度固定

```

**总结**：RISC通过精简指令提高执行效率。

**来源**：408历年真题，考查方式：简答题

### 例题 3：流水线性能计算

**题目**：某5段流水线，每段延迟1ns，求100条指令的执行时间。

**解答**：

```
非流水线时间 = 100 × 5 × 1ns = 500ns

流水线时间 = (5 + 100 - 1) × 1ns = 104ns

加速比 = 500 / 104 ≈ 4.81
```

**总结**：流水线能显著提高指令执行效率。

**来源**：408历年真题，考查方式：计算题
