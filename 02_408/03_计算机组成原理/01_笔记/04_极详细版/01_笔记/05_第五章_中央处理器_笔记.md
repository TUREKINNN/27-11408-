# 第五章 CPU 结构与功能

## 5.1 CPU 的基本结构

### 5.1.1 CPU 内部组成

```
┌─────────────────────────────────────────────────┐
│                    CPU                          │
├─────────────────────────────────────────────────┤
│   ┌───────────────┐  ┌──────────────────┐       │
│   │   运算器 ALU   │  │   控制器 CU      │       │
│   │ ACC/MQ/X/PSW  │  │ IR/PC/MAR/MDR   │       │
│   └───────────────┘  └──────────────────┘       │
│              ┌──────────────────┐              │
│              │   寄存器组        │              │
│              └──────────────────┘              │
└─────────────────────────────────────────────────┘
```

### 5.1.2 运算器与控制器的组成

| 部件 | 所属 | 功能 |
|:-----|:-----|:-----|
| **ALU** | 运算器 | 算术/逻辑运算 |
| **ACC** | 运算器 | 累加器，存放结果 |
| **MQ** | 运算器 | 乘商寄存器 |
| **X** | 运算器 | 操作数寄存器 |
| **PSW** | 运算器 | 程序状态字（标志位） |
| **CU** | 控制器 | 控制单元，产生控制信号 |
| **IR** | 控制器 | 指令寄存器，存放当前指令 |
| **PC** | 控制器 | 程序计数器，下条指令地址 |
| **MAR** | 控制器 | 地址寄存器 |
| **MDR** | 控制器 | 数据寄存器 |

> 常见标志位：CF（进位）、OF（溢出）、ZF（零）、SF（符号）、PF（奇偶）

## 5.2 指令周期

### 5.2.1 指令周期的 4 个阶段

| 阶段 | 触发条件 | 主要工作 |
|:-----|:---------|:---------|
| **取指周期（IF）** | 每条指令都有 | PC→MAR→读主存→MDR→IR，PC+1 |
| **间址周期（IND）** | 间接寻址时 | 取操作数有效地址 |
| **执行周期（EX）** | 每条指令都有 | ALU 运算、读写数据、修改 PC |
| **中断周期（INT）** | 有中断请求 | 保存断点，转中断服务 |

### 5.2.2 不同指令的周期

| 指令类型 | 周期数 |
|:---------|:-------|
| **无操作数指令**（NOP） | 取指 + 执行 |
| **直接寻址指令** | 取指 + 执行 |
| **间接寻址指令** | 取指 + 间址 + 执行 |
| **算术逻辑指令** | 取指 + 执行 |
| **访存指令** | 取指 + 执行（含访存） |
| **控制指令** | 取指 + 执行（修改 PC） |
| **中断发生** | + 中断周期 |

> - **直接寻址** 只需要取指+执行，**不需要间址**
> - **间接寻址** 才有间址周期

## 5.3 时序系统

### 5.3.1 时序层次

| 层次 | 含义 |
|:-----|:-----|
| **指令周期** | 完整执行一条指令的时间 |
| **机器周期（CPU 周期）** | 完成一个基本操作（取指、访存）的时间 |
| **时钟周期（节拍）** | 最小时间单位 |

> 1 个指令周期 = 多个机器周期
> 1 个机器周期 = 多个时钟周期

### 5.3.2 三种控制方式

| 控制方式 | 特点 | 优点 | 缺点 |
|:---------|:-----|:-----|:-----|
| **同步控制** | 统一时钟控制 | 简单、可靠 | 速度慢 |
| **异步控制** | 握手信号协调 | 速度快 | 复杂 |
| **联合控制** | 同步+异步 | 折中 | 折中 |

## 5.4 微程序控制器

### 5.4.1 基本概念

> **微程序**：由**微指令**组成的程序，用于**解释执行**机器指令。

> **微指令**：一个时钟周期内完成的**微操作**集合，是微程序的基本单位。

> **微命令**：控制各部件完成**微操作**的最小控制信号。

### 5.4.2 微指令格式

```
┌──────────────────┬─────────────────────┐
│   微操作控制字段  │   顺序控制字段       │
│   （操作控制）    │   （下址）           │
└──────────────────┴─────────────────────┘
```

### 5.4.3 微指令的编码方式

| 编码方式 | 特点 | 优点 | 缺点 |
|:---------|:-----|:-----|:-----|
| **直接编码（水平）** | 1 位 1 微操作 | 速度快、并行性高 | 位数多 |
| **字段直接编码** | 分组编码 | 节省位数 | 速度稍慢 |
| **字段间接编码** | 字段含义由其他字段决定 | 更省位 | 复杂 |
| **垂直编码** | 类似机器指令 | 短 | 速度慢 |

> - 互斥的微操作放在**同一字段**（一个字段内不能同时出现）
> - 相容的微操作放在**不同字段**

### 5.4.4 微程序的执行过程

```
1. 根据指令操作码查微指令地址（微地址）
2. 读取微指令到微指令寄存器（CMAR/CMDR）
3. 执行微指令的微操作
4. 根据顺序控制字段确定下条微指令地址
5. 重复 1-4 直到机器指令执行完毕
```

> - **微地址**：微指令在控制存储器中的**地址**
> - **微地址形成方式**：计数器法（下址字段法）、断定法

## 5.5 硬布线控制器

### 5.5.1 硬布线控制

> **硬布线控制**：用**组合逻辑电路**直接产生控制信号。

**控制信号生成公式**：

$$
C = f(	ext{IR}, 	ext{PC}, 	ext{PSW}, 	ext{时序信号})
$$

### 5.5.2 微程序 vs 硬布线

| 对比项 | 微程序控制 | 硬布线控制 |
|:-------|:----------|:----------|
| **速度** | 慢（取微指令） | **快**（组合逻辑）|
| **灵活性** | **高**（改 ROM 即可） | 低（需改电路）|
| **复杂度** | 规则、规整 | 不规整、复杂 |
| **可维护性** | **好** | 差 |
| **适合** | CISC、复杂指令 | RISC、简单指令 |
| **代表** | x86 | RISC-V、MIPS |

> - 微程序控制器**常考**：微地址形成、微指令格式、编码方式
> - 硬布线控制器只需了解**优缺点**

## 5.6 流水线技术（深入）

### 5.6.1 流水线的分类

| 类型 | 描述 |
|:-----|:-----|
| **指令流水线** | 指令级并行（IF-ID-EX-MEM-WB）|
| **算术流水线** | 运算级并行（如浮点运算）|
| **处理器级流水线** | 又称宏流水线 |
| **超标量流水线** | 多条流水线并行（多发射）|
| **超流水线** | 细分流水段（提高主频）|
| **超长指令字（VLIW）** | 多操作打包成一条长指令 |

### 5.6.2 性能分析

**吞吐率（TP）**：

$$
TP = rac{n}{(k + n - 1) \cdot \Delta t}
$$

**加速比（S）**：

$$
S = rac{k \cdot n \cdot \Delta t}{(k + n - 1) \cdot \Delta t} = rac{kn}{k + n - 1}
$$

**效率（E）**：

$$
E = rac{n \cdot k \cdot \Delta t}{(k + n - 1) \cdot k \cdot \Delta t} = rac{n}{k + n - 1}
$$

> - 当 $n 	o \infty$ 时，$S 	o k$，$E 	o 1$
> - 当 $n = k$ 时，$S = k^2/(2k-1)$，$E = 1/2$

### 5.6.3 三大冲突及解决

| 冲突类型 | 原因 | 解决方案 |
|:---------|:-----|:---------|
| **结构冲突** | 硬件资源争用 | 增加资源（双端口存储）、分时使用 |
| **数据冲突** | 指令间数据依赖 | 旁路/转发、暂停、指令重排 |
| **控制冲突** | 分支跳转 | 分支预测、延迟槽、动态预测 |

**数据冲突三种类型**：
- **RAW（Read After Write）**：真相关（必须解决）
- **WAR（Write After Read）**：反相关
- **WAW（Write After Write）**：输出相关

> - **RAW** 是唯一必须解决的冲突
> - WAR/WAW 在**按序流水线**中不会发生

## 5.7 RISC CPU 设计

### 5.7.1 RISC 五大特点

1. **精简指令集**：指令少而简单
2. **定长指令**：简化译码
3. **Load/Store 架构**：仅 Load/Store 访存
4. **大量寄存器**：通常 32 个通用寄存器
5. **硬布线控制 + 流水线**

### 5.7.2 RISC CPU 结构

```
┌─────────────────────────────────────────────────────┐
│                    RISC CPU                          │
├─────────────────────────────────────────────────────┤
│  指令缓存（I-Cache）  │  数据缓存（D-Cache）          │
├─────────────────────────────────────────────────────┤
│                   寄存器组（32个通用寄存器）          │
├─────────────────────────────────────────────────────┤
│              算术逻辑单元（ALU）                      │
├─────────────────────────────────────────────────────┤
│              指令流水线（IF-ID-EX-MEM-WB）           │
└─────────────────────────────────────────────────────┘
```

> - 寄存器窗口（减少栈操作）
> - 指令流水线
> - 延迟加载
> - 分支预测
> - 硬布线控制
