# 第四章 指令系统

## 4.1 指令格式

### 4.1.1 指令的基本格式

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│   操作码     │   地址码1    │   地址码2    │   地址码3    │
│     OP       │      A1      │      A2      │      A3      │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

### 4.1.2 按地址数分类

| 类型 | 地址数 | 示例 | 说明 |
|:-----|:-------|:-----|:-----|
| **零地址指令** | 0 | NOP, RET | 操作数隐含在栈中 |
| **一地址指令** | 1 | INC A, JMP X | 单操作数 |
| **二地址指令** | 2 | ADD A, B | A ← A OP B |
| **三地址指令** | 3 | ADD A, B, C | A ← B OP C |
| **多地址指令** | >3 | 复杂指令 | 大型机专用 |

### 4.1.3 操作码设计

| 方式 | 特点 |
|:-----|:-----|
| **固定长度操作码** | 译码简单、速度快，但浪费空间 |
| **扩展操作码** | 短码常用、长码不常用，节省空间 |

**扩展操作码设计原则**：
1. 短操作码**不能**是长操作码的**前缀**
2. 常用指令使用**短操作码**
3. 留出扩展空间

> 指令字 16 位，地址码 12 位，4 位扩展操作码
> - OP = 0000：零地址指令（16 条短操作码）
> - OP = 0001~1110：单地址指令
> - OP = 1111：双地址指令

---

## 4.2 指令的分类

| 类别 | 典型指令 | 功能 |
|:-----|:--------|:-----|
| **数据传送类** | MOV, LOAD, STORE, PUSH, POP | 数据搬运 |
| **算术运算类** | ADD, SUB, MUL, DIV, INC, DEC | 算术运算 |
| **逻辑运算类** | AND, OR, NOT, XOR, TEST | 逻辑运算 |
| **移位操作类** | SHL, SHR, SAL, SAR, ROL, ROR | 移位 |
| **程序控制类** | JMP, JZ/JNZ, CALL, RET, INT | 控制流 |
| **输入输出类** | IN, OUT | I/O 操作 |
| **串操作类** | MOVS, CMPS, SCAS | 字符串 |

> - **SHL/SHR**：逻辑移位，**无符号数**
> - **SAL/SAR**：算术移位，**有符号数**（SAR 补符号位）
> - **ROL/ROR**：循环移位

## 4.3 指令系统的性能要求

| 性能 | 含义 |
|:-----|:-----|
| **完备性** | 指令系统应**包含所有必要**的指令 |
| **有效性** | 用**少指令**和**短时间**完成任务 |
| **规整性** | 指令格式和操作码**统一**、**对称** |
| **兼容性** | 新机型能运行**旧机型的程序** |
| **可扩展性** | 保留**操作码扩展空间** |

## 4.4 RISC 与 CISC

### 4.4.1 CISC

> **CISC（Complex Instruction Set Computer）**：复杂指令系统计算机。
> - 指令数量多（200+），功能复杂
> - 寻址方式多样（10+ 种）
> - **指令长度不固定**
> - 强调**硬件**实现复杂功能
> - 代表：x86（Intel/AMD）

### 4.4.2 RISC

> **RISC（Reduced Instruction Set Computer）**：精简指令系统计算机。
> - 指令数量少（~50），功能简单
> - 寻址方式少（3~4 种）
> - **指令长度固定**
> - 强调**软件**实现复杂功能
> - 代表：ARM、MIPS、RISC-V

### 4.4.3 RISC vs CISC

| 对比项 | CISC | RISC |
|:-------|:-----|:-----|
| **指令数量** | 多（200+） | 少（~50） |
| **指令长度** | **不固定** | **固定** |
| **寻址方式** | 多（10+） | 少（3-4 种） |
| **CPI** | 高（2-15） | 低（1-2） |
| **指令周期数** | 多 | 少（多 1 周期执行） |
| **寄存器数** | 少 | 多（≥32） |
| **编译器** | 简单 | 复杂 |
| **硬件复杂度** | 高 | 低 |
| **流水线** | 难实现 | 易实现 |

### 4.4.4 RISC 关键技术

| 技术 | 作用 |
|:-----|:-----|
| **寄存器窗口** | 函数调用用不同寄存器组，减少栈操作 |
| **指令流水线** | 多指令重叠执行，提高效率 |
| **延迟加载** | Load 后插入无关指令，隐藏延迟 |
| **分支预测** | 预测分支方向，提前执行 |
| **硬布线控制** | 用组合逻辑实现，**速度快** |

> 设有 $n$ 条指令，$k$ 段流水线，每段时间 $t$：

$$
T = (k + n - 1) 	imes t
$$

## 4.5 指令的执行过程

### 4.5.1 指令周期

> **指令周期**：从取指令到执行完的时间，包含 4 个子周期：

$$
	ext{指令周期} = 	ext{取指周期} + 	ext{间址周期} + 	ext{执行周期} + 	ext{中断周期}
$$

### 4.5.2 四个子周期详解

**取指周期（IF）**：

```
1. PC → MAR → 地址总线
2. 发读命令 → 主存
3. 数据 → MDR → IR
4. PC + 1 → PC
```

**间址周期（IND）**（取有效地址）：

```
1. IR 的地址码 → MAR
2. 发读命令 → 主存
3. 数据 → MDR → IR 的地址码字段
```

**执行周期（EX）**：

```
根据 OP 执行：
- 算术运算：ALU 运算
- 数据传送：寄存器/存储器
- 程序控制：修改 PC
```

**中断周期（INT）**：

```
1. 关中断
2. 保存断点：PC → MDR → 主存
3. 保存状态：PSW → MDR → 主存
4. 中断服务程序地址 → PC
```

> - **取指周期**：每条指令都有
> - **间址周期**：当有间接寻址时
> - **执行周期**：每条指令都有
> - **中断周期**：仅当有中断请求时

## 4.6 指令流水线

### 4.6.1 流水线基本概念

> **指令流水线**：将指令执行过程分解为多个阶段，**多条指令并行执行**。

### 4.6.2 五段流水线标准划分

```
取指（IF）→ 译码（ID）→ 执行（EX）→ 访存（MEM）→ 写回（WB）
```

| 段 | 功能 |
|:---|:-----|
| **IF** | 取指令，PC → IR，PC+1 |
| **ID** | 译码 + 读寄存器 |
| **EX** | ALU 运算 / 计算地址 |
| **MEM** | 访存（Load/Store）|
| **WB** | 结果写回寄存器 |

### 4.6.3 流水线性能指标

**吞吐率（TP）**：

$$
TP = rac{	ext{完成指令数}}{	ext{总时间}}
$$

**加速比（S）**：

$$
S = rac{	ext{非流水线时间}}{	ext{流水线时间}}
$$

**效率（E）**：

$$
E = rac{	ext{任务实际占用的时空区}}{	ext{m 个段的总时空区}}
$$

### 4.6.4 流水线的三大冲突

| 冲突类型 | 原因 | 解决方案 |
|:---------|:-----|:---------|
| **结构冲突** | 硬件资源不足（多指令争用同一资源）| 增加资源（如双端口存储器）、暂停 |
| **数据冲突** | 指令间数据依赖 | 旁路（转发）、指令重排、暂停（气泡）|
| **控制冲突** | 分支指令导致 PC 不确定 | 分支预测、延迟分支、动态预测 |

**数据冲突的三种类型**：
- **RAW（Read After Write）**：真数据相关
- **WAR（Write After Read）**：反相关
- **WAW（Write After Write）**：输出相关

> - 把 EX 段结果**直接转发**到下一条指令的 EX 段
> - 无需等待 WB 段写回
> - 可解决**大部分 RAW 冲突**

## 4.7 典型例题

### 例题 1：指令格式设计

**题目**：设计二地址指令系统，操作码 4 位，地址码 12 位，指令字 32 位。求指令条数和寻址范围。

**解答**：

```
操作码：4 位 → 2^4 = 16 条指令
地址码：12 位 → 寻址范围 2^12 = 4KB
指令格式：
┌──────┬────────────┬────────────┐
│ OP   │    A1      │    A2      │
│ 4位  │    12位    │    12位    │
└──────┴────────────┴────────────┘
```

**总结**：指令格式设计在**操作种类**和**寻址范围**间平衡。

### 例题 2：RISC vs CISC

**题目**：比较 RISC 和 CISC 的特点。

**解答**：见 4.4.3 对比表。

**总结**：RISC 通过**精简指令**提高执行效率。

### 例题 3：流水线性能计算

**题目**：5 段流水线，每段 1ns，求 100 条指令的执行时间。

**解答**：

```
流水线时间 = (5 + 100 - 1) × 1ns = 104ns
非流水线时间 = 100 × 5 × 1ns = 500ns
加速比 = 500 / 104 ≈ 4.81
```

**总结**：流水线能显著提高指令执行效率。
