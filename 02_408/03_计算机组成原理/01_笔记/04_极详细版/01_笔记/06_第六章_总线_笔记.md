# 第六章 总线

## 6.1 总线概述

### 6.1.1 总线的定义

> **总线（Bus）**：计算机中**多个部件共享的传输介质**，用于在部件间传输数据、地址和控制信号。

### 6.1.2 总线的分类

| 分类方式 | 类型 | 描述 |
|:--------|:-----|:-----|
| **按功能** | 数据总线 | 传输**数据**（双向） |
|           | 地址总线 | 传输**地址**（单向） |
|           | 控制总线 | 传输**控制信号**（如读/写） |
| **按时序** | 同步总线 | 由统一时钟控制 |
|           | 异步总线 | 由握手信号协调 |
| **按位置** | 内部总线 | CPU 内部（寄存器、ALU 之间） |
|           | 系统总线 | CPU、主存、I/O 之间 |
|           | 通信总线 | 计算机系统之间 |

### 6.1.3 总线的性能指标

| 指标 | 含义 |
|:-----|:-----|
| **总线带宽** | 单位时间内传输的数据量（MB/s） |
| **总线宽度** | 总线能同时传输的位数（位宽） |
| **总线频率** | 总线时钟频率（Hz） |
| **总线负载** | 连接到总线的设备数 |

> $$\text{带宽} = \text{总线宽度} \times \text{总线频率} / 8 \text{ (B/s)}$$

---

## 6.2 总线仲裁

> **总线仲裁**：当多个设备同时请求总线使用权时，决定**哪个设备获得使用权**。

### 6.2.1 集中仲裁方式

| 方式 | 描述 | 优先级 |
|:-----|:-----|:-------|
| **链式查询** | 设备顺序连接 | 离控制器**最近的**优先级**最高** |
| **计数器定时查询** | 计数器循环查询 | 灵活可调 |
| **独立请求** | 每个设备独立请求线 | 灵活，速度快 |

### 6.2.2 分布仲裁方式

- **自举分布仲裁**：每个设备有自己的仲裁器
- **冲突检测**：检测到冲突时让优先级高的设备继续
- **并行仲裁**：多个设备**并行**请求

> - 优点：电路简单，**优先级固定**
> - 缺点：**优先级不可改变**（电路固定），故障敏感

---

## 6.3 总线传输过程

### 6.3.1 总线传输的 4 个阶段

```
1. 申请阶段：主设备请求总线
2. 仲裁阶段：总线仲裁器决定使用权
3. 传输阶段：主从设备交换数据
4. 释放阶段：主设备释放总线
```

### 6.3.2 总线定时方式

| 方式 | 特点 |
|:-----|:-----|
| **同步定时** | 统一时钟控制，事件出现在**固定时刻** |
| **异步定时** | 由**握手信号**协调，无固定时刻 |
| **半同步定时** | 同步+异步结合 |

> - 同步：所有设备以**同一时钟**工作
> - 异步：设备速度不同时，**握手**协调

## 6.4 典型例题

### 例题 1：总线带宽计算

**题目**：总线宽度 32 位，时钟频率 100 MHz，求总线带宽。

**解答**：
$$
\ext{带宽} = 32 \imes 100 \imes 10^6 / 8 = 400 \ext{ MB/s}
$$

**总结**：带宽 = 宽度 × 频率 / 8。

### 例题 2：总线仲裁方式

**题目**：比较链式查询和独立请求的优缺点。

**解答**：

| 方式 | 优点 | 缺点 |
|:-----|:-----|:-----|
| 链式查询 | 电路简单 | 优先级固定、故障敏感 |
| 独立请求 | 优先级可编程、速度快 | 电路复杂 |

**总结**：选择仲裁方式需在**速度**和**复杂度**间权衡。

## 6.5 知识回顾与重要考点

> 1. **总线分类**：数据/地址/控制总线（按功能）
> 2. **总线带宽公式**：宽度 × 频率 / 8
> 3. **总线仲裁方式**：链式查询 / 计数器定时查询 / 独立请求
> 4. **总线传输 4 阶段**：申请、仲裁、传输、释放
> 5. **总线定时方式**：同步、异步、半同步

> - 链式查询的优先级**离仲裁器最近者最高**
> - 独立请求速度最快
> - 同步总线简单，异步总线灵活
