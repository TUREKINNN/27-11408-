# 第七章 输入/输出（I/O）系统

## 7.1 I/O 系统概述

### 7.1.1 I/O 系统的组成

```
I/O 系统 = I/O 软件 + I/O 硬件
       ├─ I/O 软件：驱动程序、用户程序
       └─ I/O 硬件：I/O 设备、I/O 接口、控制器
```

### 7.1.2 I/O 设备分类

| 分类 | 设备 |
|:-----|:-----|
| **输入设备** | 键盘、鼠标、扫描仪、读卡器 |
| **输出设备** | 显示器、打印机、绘图仪 |
| **存储设备** | 磁盘、磁带、光盘 |
| **通信设备** | 调制解调器、网卡 |

### 7.1.3 I/O 接口的功能

| 功能 | 描述 |
|:-----|:-----|
| **数据缓冲** | 解决 CPU 与 I/O 设备速度不匹配 |
| **命令转换** | 转换命令格式 |
| **设备选择** | 通过地址选通特定设备 |
| **状态反馈** | 向 CPU 报告设备状态 |
| **信号转换** | 转换电平、时序 |

## 7.2 I/O 控制方式

### 7.2.1 程序查询方式

> CPU 不断**查询** I/O 设备状态，准备就绪后传输数据。

**特点**：
- CPU 与 I/O **串行**工作
- CPU 利用率**低**（大量时间用于等待）
- 适合简单系统

### 7.2.2 程序中断方式

> I/O 设备**主动通知** CPU，CPU 暂停当前程序处理 I/O。

**中断处理过程**：
```
1. I/O 设备准备就绪 → 发出中断请求
2. CPU 响应中断（若允许）
3. 保存断点和 PSW
4. 跳转到中断服务程序
5. 执行 I/O 操作
6. 恢复现场，返回断点
```

**特点**：
- CPU 与 I/O **部分并行**
- 适合**中低速** I/O
- 每次 I/O 都要**保存/恢复现场**

### 7.2.3 DMA 方式

> **DMA**：数据传送由 **DMA 控制器**完成，CPU 仅参与初始化和结束处理。

**DMA 过程**：
```
1. CPU 初始化 DMA 控制器（传送地址、长度）
2. DMA 控制器获得总线控制权
3. DMA 直接在 I/O 与主存间传送数据
4. 传送完毕，DMA 发出中断通知 CPU
```

**特点**：
- CPU 与 I/O **完全并行**（传送期间）
- 适合**高速** I/O（如磁盘、网卡）
- 传送以**块**为单位

### 7.2.4 通道方式

> **通道**：专门处理 I/O 的**处理器**，能执行通道程序。

**特点**：
- 通道独立于 CPU 工作
- 适合**大型机**
- 通道程序由**通道指令**组成

### 7.2.5 四种方式对比

| 方式 | CPU 干预 | 并行度 | 适用 |
|:-----|:---------|:-------|:-----|
| **程序查询** | 全程干预 | 最低 | 简单系统 |
| **程序中断** | 每次 I/O 干预 | 中 | 中低速 I/O |
| **DMA** | 仅初始化/结束 | 高 | 高速 I/O（块传送） |
| **通道** | 仅发指令 | 最高 | 大型系统 |

## 7.3 中断系统

### 7.3.1 中断分类

| 类型 | 来源 | 是否可屏蔽 |
|:-----|:-----|:----------|
| **硬件中断** | I/O 设备 | 可屏蔽/不可屏蔽 |
| **软件中断** | 异常、程序指令 | 可屏蔽 |
| **内中断（异常）** | CPU 内部 | 不可屏蔽 |

### 7.3.2 中断优先级

- **不可屏蔽中断 > 可屏蔽中断**
- **高速设备 > 低速设备**
- **实时设备 > 普通设备**

### 7.3.3 中断处理过程

```
1. 中断请求：设备发出中断请求信号
2. 中断响应：CPU 检查是否允许中断
3. 保护现场：保存 PC、PSW 等
4. 中断识别：识别中断源
5. 转入服务：跳转到中断服务程序
6. 恢复现场：恢复 PC、PSW
7. 中断返回：继续执行原程序
```

> - **屏蔽中断**：处理当前中断时屏蔽其他中断
> - **中断嵌套**：允许高优先级中断打断低优先级

## 7.4 磁盘存储

### 7.4.1 磁盘结构

```
磁盘 = 盘片 + 磁道 + 扇区
     ├─ 盘片：存储介质（圆形）
     ├─ 磁道：盘片上的同心圆
     └─ 扇区：磁道上的弧段（最小读写单位）
```

### 7.4.2 磁盘性能指标

| 指标 | 公式 |
|:-----|:-----|
| **存取时间** | = 寻道时间 + 旋转延迟 + 传输时间 |
| **寻道时间** | 磁头移动到目标磁道的时间 |
| **旋转延迟** | 等待目标扇区旋转到磁头下（=1/2 旋转周期）|
| **传输时间** | 读写数据的时间 |

> 平均寻道时间 10 ms，旋转速度 7200 RPM，传输 1 个扇区（512B）：
> - 旋转延迟 = 1/(7200/60) / 2 = 4.17 ms
> - 传输时间 ≈ 0
> - 平均存取时间 ≈ 10 + 4.17 = 14.17 ms

### 7.4.3 磁盘调度算法

| 算法 | 思想 | 特点 |
|:-----|:-----|:-----|
| **FCFS** | 先来先服务 | 公平但效率低 |
| **SSTF** | 最短寻道时间优先 | 可能饥饿 |
| **SCAN** | 磁头单向扫描 | 不会饥饿 |
| **C-SCAN** | 循环扫描 | 等待时间均匀 |
| **FSCAN** | SCAN 改进 | 周期性扫描 |

> - 远离当前磁道的请求可能**长期等待**（饥饿）
> - **SCAN 算法**通过单向扫描避免饥饿

### 7.4.4 磁盘阵列（RAID）

| 级别 | 描述 | 冗余 |
|:-----|:-----|:-----|
| **RAID 0** | 条带化 | 无 |
| **RAID 1** | 镜像 | 100% 冗余 |
| **RAID 5** | 条带化 + 奇偶校验 | 1 盘冗余 |
| **RAID 6** | 双奇偶校验 | 2 盘冗余 |
| **RAID 10** | 1 + 0 组合 | 高 |

## 7.5 典型例题

### 例题 1：DMA 与中断对比

**题目**：比较 DMA 方式与程序中断方式的优缺点。

**解答**：

| 方式 | 优点 | 缺点 |
|:-----|:-----|:-----|
| 程序中断 | 适合中低速、字节级传送 | 每次中断都要保护现场 |
| DMA | 适合高速、块传送 | 硬件复杂 |

**总结**：选择方式取决于**数据量**和**速度要求**。

### 例题 2：磁盘调度

**题目**：磁头当前位置 100，磁道请求序列：55, 58, 39, 18, 90, 160, 150, 38, 184。求 SSTF 算法访问顺序和总移动距离。

**解答**：
```
当前位置 100
SSTF: 90 → 58 → 55 → 39 → 38 → 18 → 150 → 160 → 184
移动：(100-90) + (90-58) + (58-55) + (55-39) + (39-38)
    + (38-18) + (150-18) + (160-150) + (184-160)
    = 10 + 32 + 3 + 16 + 1 + 20 + 132 + 10 + 24 = 248
```

**总结**：SSTF 优先选最近磁道，但可能饥饿。

## 7.6 知识回顾与重要考点

> 1. **I/O 控制 4 方式**：查询/中断/DMA/通道
> 2. **DMA 工作原理**：传送期间 CPU 不干预
> 3. **中断处理流程**：保护现场→服务→恢复现场
> 4. **磁盘性能指标**：存取时间 = 寻道 + 旋转 + 传输
> 5. **磁盘调度算法**：FCFS / SSTF / SCAN / C-SCAN
> 6. **RAID 级别**：0/1/5/6/10
> 7. **通道方式**：专门处理 I/O 的处理器

> - DMA 适用于**块传送**，中断适用于**字节传送**
> - 旋转延迟 = 1/2 旋转周期
> - SSTF 存在**饥饿**问题
