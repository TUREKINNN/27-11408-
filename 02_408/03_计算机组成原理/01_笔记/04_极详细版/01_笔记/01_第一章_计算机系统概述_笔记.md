# 第一章 计算机系统概述

## 1.1 计算机的基本组成

### 1.1.1 硬件的五大部件

> 现代计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

| 部件 | 功能 |
|:-----|:-----|
| **运算器** | 执行算术运算和逻辑运算 |
| **控制器** | 控制计算机各部件协同工作 |
| **存储器** | 存储程序和数据 |
| **输入设备** | 向计算机输入数据 |
| **输出设备** | 将计算机处理结果输出 |

> **「运控存输入输出」**——运算器、控制器、存储器、输入、输出

### 1.1.2 CPU 的组成


**运算器内部组成**：
- **ALU**：算术逻辑单元，执行算术/逻辑运算
- **ACC**：累加寄存器，存放操作数/运算结果
- **MQ**：乘商寄存器，乘除法时使用
- **X**：操作数寄存器，存放操作数

**控制器内部组成**：
- **CU**：控制单元，分析指令、发出控制信号
- **IR**：指令寄存器，存放当前执行的指令
- **PC**：程序计数器，存放下一条指令地址
- **MAR**：地址寄存器，存放访问存储器的地址
- **MDR**：数据寄存器，存放与存储器交换的数据

## 1.2 冯·诺依曼结构

### 1.2.1 冯·诺依曼思想

> **存储程序（Stored-Program）**：程序和数据以二进制形式同等存放在存储器中，计算机按程序顺序自动执行。

### 1.2.2 冯·诺依曼结构六大特点

1. 计算机由**五大部件**组成
2. **指令和数据以同等地位**存于存储器
3. 指令和数据**均用二进制**表示
4. 指令由**操作码**和**地址码**组成
5. 指令在存储器中**按顺序存放**
6. **以运算器为中心**（现代改进为以存储器为中心）

### 1.2.3 冯·诺依曼瓶颈

> CPU 与存储器之间数据传输带宽有限，成为系统性能瓶颈。解决方案：采用**Cache**、**流水线**、**并行**等技术。

> - 冯氏以**运算器**为中心
> - 现代以**存储器**为中心（CPU 内部数据流向更高效）
> - 两者都遵循"存储程序"思想，本质相同

---

## 1.3 计算机系统的层次结构

### 1.3.1 层次划分

| 层次 | 名称 | 核心部件 | 归属 |
|:-----|:-----|:---------|:-----|
| **L6** | 应用语言级 | 应用程序 | 软件 |
| **L5** | 高级语言级 | 高级语言 | 软件 |
| **L4** | 汇编语言级 | 汇编指令 | 软件 |
| **L3** | 操作系统级 | 系统调用 | 软件/硬件交界面 |
| **L2** | 传统机器级 | 机器指令 | 硬件 |
| **L1** | 微程序机器级 | 微指令 | 硬件（微程序控制器） |

### 1.3.2 各层功能

- **L1~L2（硬件层）**：执行机器指令
- **L3（系统软件）**：管理硬件资源
- **L4~L5（语言层）**：提供编程接口
- **L6（应用层）**：解决具体问题

> - **编译**：高级语言 → 汇编语言/机器语言
> - **解释**：逐条翻译并执行
> - **翻译**：源代码 → 目标代码

## 1.4 指令系统

### 1.4.1 指令的定义

> 指令是计算机能执行的基本操作命令，一组指令的集合称为**指令系统/指令集（ISA）**。

### 1.4.2 指令格式

```
┌──────────────┬──────────────┐
│  操作码 OP   │   地址码 A   │
└──────────────┴──────────────┘
```

- **操作码（OP）**：表示指令的操作类型
- **地址码（A）**：表示操作数地址

### 1.4.3 指令分类

| 类型 | 描述 |
|:-----|:-----|
| **数据传送类** | 数据在寄存器/存储器间传送（如 MOV） |
| **算术运算类** | 加、减、乘、除 |
| **逻辑运算类** | 与、或、非、异或 |
| **移位操作类** | 左移、右移、循环移位 |
| **程序控制类** | 跳转、调用、返回 |
| **输入输出类** | 与外设交互 |

### 1.4.4 指令字长

- **单字长指令**：一个机器字长
- **半字长指令**：半个机器字长
- **双字长指令**：两个机器字长

> - **定长指令字**：译码简单、速度快（如 RISC）
> - **变长指令字**：灵活、节省空间（如 x86）

## 1.5 寻址方式

### 1.5.1 寻址方式定义

> **寻址方式**：寻找**操作数**或**指令地址**的方式。寻址方式的设计直接决定指令系统的灵活性和效率。

### 1.5.2 七大寻址方式

| 寻址方式 | 有效地址（EA） | 特点 |
|:---------|:--------------|:-----|
| **立即寻址** | 操作数 = 指令中立即数 | 速度最快，操作数固定 |
| **直接寻址** | EA = A（指令中地址码） | 简单，地址空间有限 |
| **间接寻址** | EA = (A)（访问两次内存） | 扩大寻址范围 |
| **寄存器寻址** | 操作数在 Ri 中 | 速度极快 |
| **寄存器间接寻址** | EA = (Ri) | 寄存器+间接 |
| **相对寻址** | EA = PC + A | 程序可浮动 |
| **基址寻址** | EA = BR + A | 多道程序设计 |
| **变址寻址** | EA = IX + A | 数组/循环 |

> - **基址寻址**：基址寄存器内容**不变**，地址码变 → 系统管理用
> - **变址寻址**：变址寄存器内容**可变**，地址码固定 → 用户编程用
> - **相对寻址**：以**PC 当前值**为基准，常用于转移指令

### 1.5.3 寻址方式比较

| 寻址方式 | 优点 | 缺点 |
|:---------|:-----|:-----|
| 立即寻址 | 速度快 | 操作数固定 |
| 直接寻址 | 简单 | 地址空间有限 |
| 间接寻址 | 地址空间大 | 访问内存两次 |
| 寄存器寻址 | 速度最快 | 寄存器数有限 |
| 相对寻址 | 程序可浮动 | 偏移量有限 |
| 基址寻址 | 多道程序 | 需基址寄存器 |
| 变址寻址 | 数组操作方便 | 需变址寄存器 |

## 1.6 计算机性能指标

### 1.6.1 基本性能指标

| 指标 | 含义 |
|:-----|:-----|
| **CPU 时钟频率** | CPU 每秒执行的时钟周期数（Hz） |
| **CPI** | 每条指令的平均时钟周期数（Cycles Per Instruction） |
| **MIPS** | 每秒百万条指令数 |
| **FLOPS** | 每秒浮点运算次数 |

### 1.6.2 核心公式

**CPU 执行时间**：

$$
T_{CPU} = rac{	ext{指令数} 	imes 	ext{CPI}}{	ext{时钟频率}}
$$

**MIPS 计算**：

$$
	ext{MIPS} = rac{	ext{时钟频率}}{	ext{CPI} 	imes 10^6}
$$

**CPU 时钟周期**：

$$
T = rac{1}{f}
$$

### 1.6.3 其他性能指标

- **吞吐量**：单位时间处理的数据量
- **响应时间**：从请求到响应的时间
- **利用率**：资源被使用的时间比例
- **带宽**：单位时间传输的数据量

> 某 CPU 时钟频率 2 GHz，CPI = 2，则：
> - MIPS = 2×10⁹ / (2×10⁶) = **1000 MIPS**
> - 每条指令耗时 = 1ns

## 1.7 典型例题

### 例题 1：指令格式设计

**题目**：设计一个指令格式，包含 6 位操作码、5 位寄存器号、21 位偏移量，求各类容量。

**解答**：

```
┌──────┬──────┬────────────────┐
│ OP   │ R    │     offset     │
│ 6位  │ 5位  │      21位      │
└──────┴──────┴────────────────┘
```

- 操作码：6 位 → $2^6 = 64$ 种操作
- 寄存器号：5 位 → $2^5 = 32$ 个寄存器
- 偏移量：21 位 → $\pm 2^{20} = \pm 1	ext{M}$ 的寻址范围

**总结**：指令格式设计需在**操作种类、寄存器数、寻址范围**三者间平衡。

### 例题 2：寻址方式计算

**题目**：基址寄存器 BR = 1000H，变址寄存器 IX = 2000H，指令中地址码 A = 3000H，分别求直接/基址/变址的有效地址。

**解答**：

| 寻址方式 | 公式 | EA |
|:---------|:-----|:--|
| **直接寻址** | EA = A | 3000H |
| **基址寻址** | EA = BR + A | 1000H + 3000H = 4000H |
| **变址寻址** | EA = IX + A | 2000H + 3000H = 5000H |

**总结**：不同寻址方式的有效地址计算公式不同。

### 例题 3：CPU 性能计算

**题目**：某 CPU 时钟频率 2 GHz，CPI = 2，求 MIPS。

**解答**：

$$
	ext{MIPS} = rac{f}{	ext{CPI} 	imes 10^6} = rac{2 	imes 10^9}{2 	imes 10^6} = 1000 	ext{ MIPS}
$$

**总结**：MIPS 是衡量 CPU 运算速度的重要指标，仅适用于**定长指令**情形。

## 1.8 知识回顾与重要考点

> 1. **冯·诺依曼结构特点**（5 大特点 + 存储程序思想）
> 2. **CPU 内部组成**：运算器（ALU/ACC/MQ/X）+ 控制器（CU/IR/PC/MAR/MDR）
> 3. **计算机层次结构**：6 层结构，硬件/软件边界
> 4. **指令格式**：定长 vs 变长；扩展操作码
> 5. **寻址方式**：7 种寻址的有效地址计算
> 6. **性能计算**：CPU 时间、MIPS、CPI 三者关系

> - **冯·诺依曼瓶颈** ≠ 计算机整体速度瓶颈
> - **CPU 时钟频率** 不等于 **CPU 执行速度**（还看 CPI）
> - **基址寻址** 操作系统用，**变址寻址** 用户程序用
