## 🧠 主题 1：CPU 结构与功能

### 1.1 CPU 基本结构

```
┌──────────────────────────────┐
│           CPU                 │
│  ┌────────┐    ┌──────────┐  │
│  │ 寄存器 │    │   ALU    │  │
│  │  堆    │    │  算逻单元│  │
│  └────────┘    └──────────┘  │
│  ┌─────────────────────────┐ │
│  │     CU 控制器            │ │
│  │  (Control Unit)         │ │
│  └─────────────────────────┘ │
│  ┌─────────────────────────┐ │
│  │   中断系统/异常处理      │ │
│  └─────────────────────────┘ │
└──────────┬───────────────────┘
           │ 系统总线
    ┌──────┴──────┐
    │             │
   主存          IO
```

### 1.2 CPU 寄存器分类

| 寄存器 | 缩写 | 功能 |
|--------|------|------|
| **程序计数器** | PC | 存放下一条指令地址 |
| **指令寄存器** | IR | 存放当前执行指令 |
| **累加器** | ACC | 算术运算主操作数 |
| **通用寄存器** | GPR | 多种用途 |
| **状态寄存器** | PSW/FLAGS | 存放标志位 |
| **地址寄存器** | MAR | 访存地址 |
| **数据寄存器** | MDR | 访存数据缓冲 |
| **缓冲寄存器** | DR | 暂存数据 |
| **堆栈指针** | SP | 栈顶地址 |

### 1.3 标志寄存器（PSW）核心标志

```
ZF (Zero Flag): 零标志，结果 = 0
CF (Carry Flag): 进位/借位标志
OF (Overflow Flag): 溢出标志，带符号运算
SF (Sign Flag): 符号标志
PF (Parity Flag): 奇偶标志
```

## 🔄 主题 2：指令执行过程

### 2.1 冯·诺依曼结构

```
程序指令 = 数据，统一编址
CPU ──── 主存 ──── IO
        ↑  ↑
        └──┴── 共享总线
```

### 2.2 指令周期

```
┌─────────────────────────────────────┐
│       指令周期 = 4 阶段             │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐       │
│  │取指│→│译码│→│执行│→│写回│       │
│  │IF  │ │ID  │ │EX  │ │WB  │       │
│  └────┘ └────┘ └────┘ └────┘       │
└─────────────────────────────────────┘
```

### 2.3 完整数据通路

```
        PC ─→ MAR ─→ 主存 ─→ MDR ─→ IR
        ↑                                  │
        │                                  ↓
       PC+1                            译码
                                         │
                                         ↓
                          GPR[Ri] ─→ ALU ─→ GPR[Rj]
                                         │
                                         ↓
                                       标志位
```

### 2.4 数据通路例题

> 1. PC → MAR（送地址）
> 2. 1 → R（读命令）
> 3. M(MAR) → MDR（读主存）
> 4. MDR → IR（送指令寄存器）
> 5. PC + 1 → PC（PC 加 1）
> 6. OP(IR) → CU（译码）

## 🎛️ 主题 3：控制器

### 3.1 控制器两大流派

| 流派 | 思想 | 优缺点 |
|------|------|--------|
| **硬布线控制器** | 组合逻辑产生控制信号 | 速度快，难设计，难修改 |
| **微程序控制器** | 微指令序列产生控制信号 | 规整，易修改，慢 |

### 3.2 微程序控制器

```
┌─────────────┐
│  机器指令 M  │
└──────┬──────┘
       ↓
┌──────────────────┐
│ 微程序（多条微指令）│
│ 微指令 = 微操作 + 下地址│
└──────┬───────────┘
       ↓
┌──────────────────┐
│  控制信号 C       │
└──────────────────┘

微指令存储于控制存储器（CM）
```

### 3.3 微指令格式

```
┌──────────────────────┬────────────┐
│  操作控制字段（微操作）│ 顺序控制字段│
│  Operation Field     │ Address    │
│  22 位               │  下地址 8 位│
└──────────────────────┴────────────┘
```

### 3.4 微指令编码方式

| 方式 | 思想 | 优缺点 |
|------|------|--------|
| **直接表示** | 1 位 1 微操作 | 简单，微指令长 |
| **字段直接编码** | 互斥微操作合一段 | 微指令短，需译码 |
| **混合编码** | 部分直接部分字段 | 折中 |

### 3.5 微程序控制器例题

> - 假设 CM 容量为 4 × 30 位
> - 每条微指令 30 位：操作字段 22 位 + 下地址 8 位
> - 容量 = 4 × 30 = 120 bit
> - 实际机器指令数 = 微程序条数

### 3.6 硬布线 vs 微程序对比

| 特性 | 硬布线 | 微程序 |
|------|--------|--------|
| 速度 | 快 | 慢 |
| 设计 | 复杂 | 规整 |
| 修改 | 难 | 易 |
| 适用 | RISC | CISC |
| 实现 | 组合逻辑 | 控制存储器 |

## 🚌 主题 4：总线系统

### 4.1 总线分类

| 分类标准 | 类型 |
|---------|------|
| **位置** | 内部总线 / 系统总线 / I/O 总线 |
| **功能** | 数据总线 / 地址总线 / 控制总线 |
| **传输方式** | 串行 / 并行 |
| **时序** | 同步 / 异步 |

### 4.2 系统总线三大总线

```
┌──────────┐
│   DB     │ 数据总线（双向）
│ Data Bus │ 位数 = 字长
└──────────┘

┌──────────┐
│   AB     │ 地址总线（单向）
│ Addr Bus │ 位数 = 地址空间
└──────────┘

┌──────────┐
│   CB     │ 控制总线（双向）
│ Ctrl Bus │ 读写/中断/总线请求等
└──────────┘
```

### 4.3 总线仲裁 3 大方式

#### 4.3.1 集中式仲裁

| 方式 | 思想 | 优缺点 |
|------|------|--------|
| **链式查询** | 总线授权 BG 串联 | 优先级固定，靠前优先 |
| **计数器定时** | 计数器轮询 | 优先级动态 |
| **独立请求** | 每个设备 BR/BG | 灵活，线多 |

#### 4.3.2 链式查询 ASCII

```
         总线仲裁器
            │
        ┌───┴────┐
       BR1      BR2     BR3
        │        │       │
        └────┬───┴───┬───┘
             │   │   │
        ┌────┴───┴───┴────┐
        │ 设备 1, 2, 3    │
        └─────────────────┘
        
总线授权 BG：
设备1 → 设备2 → 设备3
（按顺序传递，先到先得）
```

#### 4.3.3 独立请求

```
设备 1: BR1 ──→ ┐
设备 2: BR2 ──→ ├──→ 总线仲裁器 → BGx → 设备 x
设备 3: BR3 ──→ ┘
```

### 4.4 总线传输 4 阶段

```
申请分配：设备请求总线，仲裁器分配
寻址阶段：主设备发送从设备地址
传输阶段：数据交换
结束阶段：释放总线
```

### 4.5 总线性能指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **总线带宽** | 频率 × 位宽 | 传输速度 |
| **总线周期** | 1 次传输时间 | 一次完整传输 |
| **总线利用率** | 实际传输 / 总周期 | 使用效率 |

### 4.6 同步 vs 异步总线

| 类型 | 特点 |
|------|------|
| **同步** | 统一时钟控制，速度快，难扩展 |
| **异步** | 握手信号，可变速度，慢 |

## 💾 主题 5：IO 控制方式

### 5.1 4 大 IO 控制方式

| 方式 | 思想 | 优缺点 | 适用 |
|------|------|--------|------|
| **程序查询** | CPU 轮询设备 | 简单，CPU 忙等 | 简单设备 |
| **中断** | 设备主动通知 CPU | CPU 可并行 | 通用 |
| **DMA** | DMA 控制器接管总线 | 不打扰 CPU | 高速块设备 |
| **通道** | 通道独立处理 | 最强 | 大型机 |

### 5.2 4 大方式对比

```
效率：程序查询 < 中断 < DMA < 通道
CPU 干预：程序查询 > 中断 > DMA > 通道
硬件成本：程序查询 < 中断 < DMA < 通道
```

### 5.3 中断处理流程

```
设备 ──中断请求──→ CPU
                    │
                    ↓
              响应中断
                    │
                    ↓
              保存现场（PC/PSW）
                    │
                    ↓
              查中断向量
                    │
                    ↓
              执行中断服务程序
                    │
                    ↓
              恢复现场
                    │
                    ↓
              返回主程序
```

### 5.4 中断优先级

```
多个中断同时请求：按优先级响应
处理方式：
1. 嵌套中断：高优先级打断低优先级
2. 屏蔽中断：CPU 关闭中断响应
3. 排队：硬件 / 软件排队
```

### 5.5 DMA 详解

#### DMA 三大传送方式

| 方式 | 思想 | 优缺点 |
|------|------|--------|
| **停止 CPU 访存** | DMA 期间 CPU 不能访存 | 简单，CPU 停顿 |
| **周期挪用** | DMA 挪用 1 个存储周期 | CPU 慢但仍工作 |
| **DMA 与 CPU 交替** | 总线分时 | 硬件复杂 |

#### DMA 控制器

```
DMA 控制器
┌──────────┐
│ MAR: 主存地址│
│ WC:  字数  │ 
│ DR:   数据  │  → 主存
│ CR:   控制  │
└──────────┘
```

#### DMA 工作流程

```
1. CPU 初始化 DMA（传送方向/地址/字数）
2. 设备准备数据，发 DMA 请求
3. DMA 控制器向总线申请
4. 获得总线，DMA 直接读写主存
5. 传送完毕，DMA 发中断给 CPU
6. CPU 处理后续
```

### 5.6 通道

```
通道 = 轻量级 CPU，专门处理 IO
有自己的指令系统（通道程序）
可控制多台设备
类型：
  字节多路通道：多个字符设备
  数组多路通道：多个块设备
  选择通道：单一高速设备
```

## 🖥️ 主题 6：磁盘 IO

### 6.1 磁盘性能参数

```
转速 r: 转/分钟
平均寻道时间 Ts: 磁头移动时间
旋转延迟 Tr: 旋转半圈时间 = 1/(2r) 分钟 = 30/r 秒
传输率: 每秒字节数 = 每道字节数 × 转速
```

### 6.2 磁盘读写时间

```
T_总 = Ts + 0.5 × Tr + Tt
     = 寻道时间 + 半圈旋转 + 数据传输

Tt = 字节数 / 传输率
```

> - Tr = 60 / 7200 / 2 = 4.17 ms
> - 传输率 = 1MB × 7200/60 = 120 MB/s
> - Tt = 4KB / 120MB/s = 0.033 ms
> - Ts 假设 5 ms
> - **T_总 ≈ 9.2 ms**

### 6.3 4 大磁盘调度算法

| 算法 | 思想 | 优缺点 |
|------|------|--------|
| **FCFS** | 先来先服务 | 公平，慢 |
| **SSTF** | 最短寻道优先 | 快，可能饥饿 |
| **SCAN** | 电梯算法 | 较均匀 |
| **C-SCAN** | 循环扫描 | 更均匀 |

### 6.4 减少 IO 时间方法

1. **磁盘调度**：优化寻道时间
2. **提前读 (Read-Ahead)**：预读后续块
3. **延迟写 (Write-Behind)**：合并多次写
4. **高速缓存**：磁盘 Cache

## ⚠️ 8 大易错点

1. ❌ **CPU 包含主存？** 错误！CPU 不含主存，CPU 与主存通过总线相连
2. ❌ **PC 存放当前指令？** 错误！PC 存放**下一条**指令地址
3. ❌ **微程序 = 程序？** 错误！微程序由微指令组成，存于控制存储器
4. ❌ **微指令 = 机器指令？** 错误！1 条机器指令 = 1 段微程序（多微指令）
5. ❌ **DMA 时 CPU 完全不工作？** 错误！周期挪用时 CPU 仍可工作
6. ❌ **中断越多越好？** 错误！中断多则开销大，应适度
7. ❌ **SCAN = 最短寻道？** 错误！SSTF 才是，SCAN 防饥饿
8. ❌ **磁盘访问 = 寻道时间？** 错误！还有旋转 + 传输
