## 📋 主题 1：指令格式

### 1.1 指令基本格式

```
┌──────────┬──────────┬──────────┬───┬──────────┐
│  操作码   │  地址码1  │  地址码2  │…  │  地址码n  │
│  (OP)    │   (A1)   │   (A2)   │   │   (An)   │
└──────────┴──────────┴──────────┴───┴──────────┘
     ↑
   表明做什么操作
```

### 1.2 指令字长

| 类型 | 字长 | 特点 |
|------|------|------|
| **定长指令字** | 所有指令等长 | 译码简单，浪费空间 |
| **变长指令字** | 指令长度可变 | 节省空间，译码复杂 |

### 1.3 操作码扩展

```
定长操作码：n 位操作码 → 最多 2ⁿ 条指令

扩展操作码：4 位 OP
┌────┬────────────────┐
│OP=│  其他字段       │
│1111│   ...          │
└────┴────────────────┘
将 OP = 1111 用作扩展标志
```

### 1.4 4 地址、3 地址、2 地址、1 地址、0 地址指令

| 类型 | 格式 | 说明 | 用途 |
|------|------|------|------|
| **4 地址** | OP A1 A2 A3 A4 | 结果存 A3，下条地址 A4 | 理论 |
| **3 地址** | OP A1 A2 A3 | 主流 | RISC |
| **2 地址** | OP A1 A2 | 结果存 A1 | CISC |
| **1 地址** | OP A1 | 隐含 ACC | 累加器型 |
| **0 地址** | OP | 隐含栈 | 栈机器 |

## 🎯 主题 2：寻址方式（7 大寻址）

### 2.1 7 大寻址方式汇总

| # | 寻址方式 | 有效地址 EA | 特点 | 速度 |
|---|---------|-----------|------|------|
| 1 | **立即寻址** | EA = A（操作数在指令中） | 最快 | ⚡⚡⚡ |
| 2 | **直接寻址** | EA = A | 简单 | ⚡⚡ |
| 3 | **寄存器寻址** | EA = Ri（操作数在寄存器） | 快 | ⚡⚡⚡ |
| 4 | **寄存器间接** | EA = (Ri) | 灵活 | ⚡⚡ |
| 5 | **间接寻址** | EA = (A) | 多级间接，慢 | ⚡ |
| 6 | **基址寻址** | EA = A + (Rb) | 程序重定位 | ⚡ |
| 7 | **变址寻址** | EA = A + (Rx) | 数组处理 | ⚡ |
| 8 | **相对寻址** | EA = A + (PC) | 转移指令 | ⚡ |

### 2.2 各寻址方式 ASCII 示意

```
1. 立即寻址：操作数就在指令里
   指令：[OP | 123]    操作数 = 123
   ↑
   最直接，但操作数大小受限

2. 直接寻址：操作数地址在指令里
   指令：[OP | 1000]   操作数 = MEM[1000]
   ↑                     ↑
   地址                  访存1次

3. 寄存器寻址：操作数在寄存器
   指令：[OP | R1]     操作数 = R1
   ↑
   不访存，最快

4. 寄存器间接：
   指令：[OP | R1]     操作数 = MEM[R1]
   ↑                     ↑
   R1存的是地址          访存1次

5. 间接寻址：操作数地址的地址
   指令：[OP | 1000]   1000 → 2000 → MEM[2000]
   ↑      ↑            ↑
   1000   2000          访存2次

6. 基址寻址：用于重定位
   指令：[OP | A | Rb] EA = A + (Rb)
   ↑       ↑    ↑       ↑
   偏移量  通用  基址寄存器

7. 变址寻址：用于数组
   指令：[OP | A | Rx] EA = A + (Rx)
   ↑       ↑    ↑       ↑
   基地址  索引  自动增减

8. 相对寻址：用于转移
   指令：[OP | offset]  EA = PC + offset
   ↑       ↑             ↑
   偏移量  跳转范围      PC当前
```

### 2.3 寻址方式对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| 立即 | 最快，无访存 | 范围小 |
| 直接 | 简单 | 地址固定 |
| 寄存器 | 快，无需访存 | 寄存器有限 |
| 寄存器间接 | 灵活，地址可变 | 多1次访存 |
| 间接 | 灵活，可指针链 | 多次访存 |
| 基址 | 利于重定位 | 需硬件支持 |
| 变址 | 利于数组 | 需硬件支持 |
| 相对 | 利于转移 | 范围受限 |

## ⚖️ 主题 3：CISC vs RISC

### 3.1 CISC vs RISC 全面对比

| 特性 | CISC（复杂指令集） | RISC（精简指令集） |
|------|-------------------|-------------------|
| 指令数 | 多（100-250） | 少（<100） |
| 指令字长 | 不定长 | 定长 |
| 寻址方式 | 多（5-20 种） | 少（1-5 种） |
| 指令格式 | 多于 4 种 | 少于 4 种 |
| 访存指令 | 多 | 少（Load/Store） |
| 周期 | 多周期 | 单周期 |
| 控制器 | 微程序 | 硬连线 |
| 流水线 | 难实现 | 易实现 |
| 编译器 | 简单 | 复杂 |
| 代表 | x86, Motorola 68K | MIPS, ARM, RISC-V |
| 目标 | 减少指令数 | 提升速度 |

### 3.2 CISC 特点详解

```
CISC 目标：强化指令功能，减少程序指令数
设计哲学：用一条复杂指令完成多步操作
实现：微程序控制器
缺点：指令复杂，流水线难
```

### 3.3 RISC 特点详解

```
RISC 目标：简化指令，提升 CPI（每条指令周期数）
设计哲学：每条指令 1 周期，Load/Store 架构
实现：硬连线控制器 + 大量寄存器
优点：流水线高效，编译器优化
代表：MIPS 5 段流水线：
  IF → ID → EX → MEM → WB
```

### 3.4 RISC 五大特点

1. **Load/Store 架构**：只有 Load/Store 访存
2. **大量寄存器**：32 个以上通用寄存器
3. **指令定长**：32 位等长
4. **单周期执行**：每条指令 1 周期
5. **硬连线控制**：用组合逻辑而非微程序

## 🏃 主题 4：指令流水线

### 4.1 5 段流水线（MIPS）

```
┌─────────────────────────────────────────────────┐
│ IF │ ID │ EX │ MEM │ WB │                       │
│取指│译码│执行│访存 │写回│                       │
└─────────────────────────────────────────────────┘

IF (Instruction Fetch): 取指
ID (Instruction Decode): 译码 + 读寄存器
EX (Execute): 执行 / 计算地址
MEM (Memory Access): 访存
WB (Write Back): 写回寄存器
```

### 4.2 流水线时空图

```
时钟周期：    1    2    3    4    5    6    7    8
指令 1:    [IF  ][ID  ][EX  ][MEM ][WB  ]
指令 2:         [IF  ][ID  ][EX  ][MEM ][WB  ]
指令 3:              [IF  ][ID  ][EX  ][MEM ][WB  ]
指令 4:                   [IF  ][ID  ][EX  ][MEM ][WB  ]
```

### 4.3 流水线性能公式

```
T(流水线) = (n + k - 1) × Δt
n = 指令数
k = 流水段数
Δt = 最长段耗时

加速比 S = T(顺序) / T(流水线)
       = n × k × Δt / [(n + k - 1) × Δt]
       = n × k / (n + k - 1)

吞吐率 TP = n / [(n + k - 1) × Δt]    指令/秒
最大吞吐率 = 1 / Δt

效率 E = TP / 最大吞吐率
       = n / (n + k - 1)
```

### 4.4 三大冒险（Hazard）

| 类型 | 原因 | 解决办法 |
|------|------|----------|
| **结构冒险** | 硬件资源冲突 | 资源重复 / 停顿 |
| **数据冒险** | 后指令用到前指令结果 | 转发 / 旁路 / 停顿 |
| **控制冒险** | 转移指令改变 PC | 延迟槽 / 预测 / 预取 |

### 4.5 数据冒险详解

```
RAW (Read After Write) 真数据相关
WAR (Write After Read) 反相关
WAW (Write After Write) 输出相关
```

#### RAW 处理方法

```
┌────────────────────────────────────────┐
│ I1: ADD R1, R2, R3                      │
│ I2: SUB R4, R1, R5      ← 数据冒险     │
└────────────────────────────────────────┘

解决：转发（Forwarding / Bypassing）
将 I1 的 EX 结果直接转发到 I2 的 EX 输入
无需停顿 1 个周期
```

### 4.6 控制冒险处理

```
1. 延迟槽 (Delay Slot)
   转移指令后插入 1 条必执行的指令
   
2. 静态预测
   总是预测跳转 / 总是预测不跳
   
3. 动态预测
   1-bit 预测：记录上次结果
   2-bit 预测：状态机
   
4. 分支目标缓冲 (BTB)
   缓存 (PC, 目标地址, 历史)
```

### 4.7 流水线例题

> - 顺序执行：10 × 5 = 50 周期
> - 流水线：(10 + 5 - 1) = 14 周期
> - **加速比 = 50 / 14 ≈ 3.57**

> - 最大吞吐率 = 1 / 100ns = 10⁷ 条/s
> - 100 条指令实际：T = (100 + 4 - 1) × 100ns = 10300ns
> - 实际吞吐率 = 100 / 10300ns ≈ 9.7 × 10⁶ 条/s
> - **效率 = 9.7 / 10 = 97%**

## 📊 主题 5：指令类型

### 5.1 5 类指令

| 类型 | 例子 | 占比 |
|------|------|------|
| **数据传送** | MOV, LOAD, STORE | 最高 |
| **算术逻辑** | ADD, SUB, AND, OR | 高 |
| **移位** | SHL, SHR | 中 |
| **转移** | JMP, JZ, CALL, RET | 中 |
| **输入输出** | IN, OUT | 低 |

### 5.2 转移指令分类

| 类别 | 寻址 | 范围 |
|------|------|------|
| **无条件转移** | JMP target | 全地址 |
| **条件转移** | JZ / JNZ | 相对寻址 |
| **调用** | CALL | 全地址 |
| **返回** | RET | 栈顶 |

## ⚠️ 8 大易错点

1. ❌ **变址寻址 = 基址寻址？** 错误！基址 EA = A + (Rb)（Rb 不变 A 变），变址 EA = A + (Rx)（A 不变 Rx 变）
2. ❌ **相对寻址是相对 A？** 错误！相对寻址相对 **PC**
3. ❌ **RISC 指令少 = 性能差？** 错误！RISC 通过流水线反而更快
4. ❌ **CISC 一定用微程序？** 错误！现代 CISC 也可用硬布线
5. ❌ **流水线周期 = 1 段耗时？** 错误！流水线周期 = **最长段**耗时
6. ❌ **数据冒险无法避免？** 错误！可用**转发**避免停顿
7. ❌ **WAW 是真数据相关？** 错误！RAW 才是，WAW/WAR 是名相关
8. ❌ **延迟槽能消除控制冒险？** 错误！只是**掩盖**，转移仍需 1 周期
