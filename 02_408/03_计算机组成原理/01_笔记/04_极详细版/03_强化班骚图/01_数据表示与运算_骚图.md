## 📐 主题 1：数制转换

### 1.1 四大数制

| 数制 | 基数 | 数码 | 标识 | 示例 |
|------|------|------|------|------|
| 二进制 | 2 | 0,1 | B | 1010B |
| 八进制 | 8 | 0-7 | O/Q | 12O |
| 十进制 | 10 | 0-9 | D | 10D |
| 十六进制 | 16 | 0-9,A-F | H | AH |

### 1.2 转换速算

```
二进制 ↔ 八进制：3 位一组
二进制 ↔ 十六进制：4 位一组
任意 → 十进制：按权展开 Σ di × r^i
十进制 → 任意：除基取余（整数部分） / 乘基取整（小数部分）
```

### 1.3 二进制数对照表

| 8 | 1000 | 8 |
| 15 | 1111 | F |
| 16 | 0001 0000 | 10 |
| 127 | 0111 1111 | 7F |
| 128 | 1000 0000 | 80 |
| 255 | 1111 1111 | FF |

## 🔢 主题 2：原码 / 反码 / 补码 / 移码

### 2.1 四大码制定义（n 位字长，最高位为符号位）

| 码制 | 正数 | 负数 | 0 的表示 | 范围 |
|------|------|------|----------|------|
| **原码** | 0+数值 | 1+数值 | +0/-0 两种 | -（2ⁿ⁻¹-1）~ 2ⁿ⁻¹-1 |
| **反码** | 0+数值 | 1+按位取反 | +0/-0 两种 | 同上 |
| **补码** | 0+数值 | 1+按位取反+1 | 唯一 0 | -2ⁿ⁻¹ ~ 2ⁿ⁻¹-1 |
| **移码** | 1+数值 | 0+数值 | 唯一 | -2ⁿ⁻¹ ~ 2ⁿ⁻¹-1 |

> - 原码：符号位 + 数值位
> - 反码：负数按位取反
> - 补码：负数反码 +1
> - 移码：补码**符号位取反**

### 2.2 8 位码制对照表

| 真值 | 原码 | 反码 | 补码 | 移码 |
|------|------|------|------|------|
| +127 | 0111 1111 | 0111 1111 | 0111 1111 | 1111 1111 |
| +1 | 0000 0001 | 0000 0001 | 0000 0001 | 1000 0001 |
| +0 | 0000 0000 | 0000 0000 | 0000 0000 | 1000 0000 |
| -0 | 1000 0000 | 1111 1111 | 0000 0000 | 1000 0000 |
| -1 | 1000 0001 | 1111 1110 | 1111 1111 | 0111 1111 |
| -127 | 1111 1111 | 1000 0000 | 1000 0001 | 0000 0001 |
| -128 | 无法表示 | 无法表示 | 1000 0000 | 0000 0000 |

> - 8 位补码能表示 **-128**（1000 0000）
> - 但原码和反码都**无法**表示 -128
> - 移码表示 -128 = 0000 0000

### 2.3 补码核心公式

```
[x+y]补 = [x]补 + [y]补   (mod 2ⁿ)
[x-y]补 = [x]补 + [-y]补  (mod 2ⁿ)
[-y]补 = ~[y]补 + 1 = 连同符号位取反加 1
```

### 2.4 真值还原（补码 → 真值）

```
正数：直接转
负数：符号位不变，数值部分按位取反后 + 1
或：对补码再求一次补码（连同符号位一起取反+1）
```

> 1. 符号位 1 → 负数
> 2. 数值位取反：0000 0011
> 3. +1：0000 0100 = 4
> 4. 真值 = **-4**

## ⚙️ 主题 3：定点数与浮点数

### 3.1 定点数

```
定点整数：符号位 + n-1 位整数位（范围：-2ⁿ⁻¹ ~ 2ⁿ⁻¹-1）
定点小数：符号位 + n-1 位小数位（范围：-（1-2⁻ⁿ⁺¹）~ 1-2⁻ⁿ⁺¹）
```

### 3.2 浮点数表示

```
N = (-1)^S × M × 2^E

S：符号位（0 正 1 负）
M：尾数（有效数字，规格化后 1 ≤ M < 2）
E：阶码（含阶符）
```

### 3.3 规格化浮点数

| 码制 | 正数规格化 | 负数规格化 |
|------|----------|----------|
| 原码 | 尾数 0.1xxxx | 尾数 1.1xxxx |
| 补码 | 尾数 0.1xxxx | 尾数 1.0xxxx |
| 反码 | 尾数 0.1xxxx | 尾数 1.0xxxx |

> - 补码尾数形如 1.0xxx 时为规格化（因为 -0.5 = 1.1000...0 最小）
> - 而 1.1xxx 不是规格化（不是最简形式）

## 💎 主题 4：IEEE 754 浮点数（⭐⭐⭐⭐⭐）

### 4.1 IEEE 754 三种格式

| 格式 | 总位数 | 符号位 S | 阶码 E | 尾数 M | 偏置 Bias |
|------|--------|---------|--------|--------|-----------|
| 单精度 float | 32 | 1 | 8 | 23 | 127 |
| 双精度 double | 64 | 1 | 11 | 52 | 1023 |
| 半精度 | 16 | 1 | 5 | 10 | 15 |

### 4.2 IEEE 754 编码规则

```
真值 = (-1)^S × (1.M) × 2^(E - Bias)
      └ 符号  └ 隐含 1   └ 偏移后指数
```

### 4.3 4 类特殊值

| 类型 | 阶码 E | 尾数 M | 真值 |
|------|--------|--------|------|
| **零** | 全 0 | 全 0 | ±0（符号位决定） |
| **非规格化** | 全 0 | 非 0 | (-1)^S × 0.M × 2^(1-Bias) |
| **规格化** | [1, 2ⁿ-2] | 任意 | (-1)^S × 1.M × 2^(E-Bias) |
| **无穷大** | 全 1 | 全 0 | ±∞ |
| **NaN** | 全 1 | 非 0 | 非数值 |

### 4.4 IEEE 754 单精度浮点数结构

```
位号  31      30  23      22       0
    ┌────┬─────────┬───────────────┐
    │ S  │   E(8)  │     M(23)     │
    └────┴─────────┴───────────────┘
    符号  阶码       尾数（隐含 1.）
```

### 4.5 IEEE 754 例题详解

>
> 1. **绝对值转二进制**：13.625 = 1101.101B
> 2. **规格化**：1101.101 = 1.101101 × 2³
> 3. **S = 1**（负数）
> 4. **E = 3 + 127 = 130 = 1000 0010**
> 5. **M = 101101**（23 位：1011 0100 0000 0000 0000 000）
>
> 完整：1 10000010 10110100000000000000000

>
> 1. **S = 0**（正数）
> 2. **E = 130 - 127 = 3**
> 3. **M = 0.01 0000... = 1.25**
> 4. **真值 = +1.25 × 2³ = 10**

## 🧮 主题 5：ALU 算术逻辑单元

### 5.1 ALU 核心运算

```
加法器：半加器、全加器、行波进位加法器
超前进位加法器：并行进位，速度快
减法：A - B = A + (-B)补 = A + ~B + 1
乘法：原码一位乘、补码一位乘（Booth 算法）
除法：原码恢复余数法、不恢复余数法（加减交替法）
```

### 5.2 溢出判断

| 方法 | 公式 | 适用 |
|------|------|------|
| **单符号位** | 参加运算的两个数符号相同，结果符号与原操作数不同 → 溢出 | 带符号 |
| **双符号位** | 结果双符号位 01=正溢出，10=负溢出，00/11 正确 | 带符号 |
| **进位位** | 最高位进位 ≠ 次高位进位 → 溢出 | 带符号 |
| **无符号** | 进位 = 1 → 溢出 | 无符号 |

### 5.3 逻辑运算

```
与 AND：F = A · B
或 OR：F = A + B
非 NOT：F = ~A
异或 XOR：F = A ⊕ B
同或 XNOR：F = ~(A ⊕ B)
```

### 5.4 一位全加器

```
S = A ⊕ B ⊕ Cin
Cout = (A · B) + (Cin · (A ⊕ B))
       └ AB同1   └ 必有进位  └ A⊕B=1且Cin=1
```

## ✅ 主题 6：数据校验码

### 6.1 奇偶校验码

```
奇校验：数据位 + 1 校验位，使总 1 的个数为奇数
偶校验：数据位 + 1 校验位，使总 1 的个数为偶数
能检错 1 位，不能纠错
```

### 6.2 海明码（Hamming Code）

```
海明码位数关系：2^r ≥ k + r + 1
  k = 信息位，r = 校验位

校验位 P_i 放在 2^(i-1) 位
分组：第 i 位（从 1 起）被所有 P_j 校验，其中 2^(j-1) 能整除 i
纠错：S = P1×1 + P2×2 + ... = 错误位位置
```

> - 2^r ≥ 8 + r + 1
> - r = 4 时：16 ≥ 13 ✓
> - **需要 4 位海明校验位**

### 6.3 循环冗余校验（CRC）

```
步骤：
1. 设生成多项式 G(x) 阶为 r
2. 信息位后补 r 个 0
3. 做模 2 除法（异或）
4. 余数即为 CRC 校验位
5. 接收端整除 G(x) → 余 0 无错
```

> - 补 4 个 0：1101 0000
> - 模 2 除：11010000 ÷ 10011 = 1100 余 0010
> - 发送：1101 0010
> - 接收整除 G(x) 余 0 正确

| # | 公式 | 名称 | 应用 |
|---|------|------|------|
| 1 | 真值 = (-1)^S × 1.M × 2^(E-Bias) | IEEE 754 还原 | 浮点解码 |
| 2 | E_实际 = E_存储 - Bias | 阶码偏移 | 浮点解码 |
| 3 | [x+y]补 = [x]补 + [y]补 (mod 2ⁿ) | 补码加法 | 补码运算 |
| 4 | [-y]补 = ~[y]补 + 1 | 求负补 | 补码减法 |
| 5 | 2^r ≥ k + r + 1 | 海明码位数 | 校验码设计 |
| 6 | G(x) 阶数 r = CRC 位数 | CRC | 校验码 |
| 7 | 规格化补码：正 0.1xxx / 负 1.0xxx | 规格化 | 浮点 |
| 8 | 双符号位 01 正溢/10 负溢 | 溢出判断 | 补码运算 |

## ⚠️ 8 大易错点

1. ❌ **补码 = 原码取反 + 1？** 错误！符号位**不变**，只对**数值位**取反 + 1
2. ❌ **-0 的补码 = 1000 0000？** 错误！-0 的补码 = 0000 0000（与 +0 相同）
3. ❌ **移码 = 补码？** 错误！移码 = 补码**符号位取反**
4. ❌ **IEEE 754 尾数有 24 位？** 错误！尾数域 23 位 + 隐含 1 = 24 位精度
5. ❌ **E=全 0 是规格化最小？** 错误！E=全 0 是非规格化，规格化最小 E=1
6. ❌ **浮点比较：直接比较字面？** 错误！必须**先对齐阶码**再比较尾数
7. ❌ **CRC 一定能纠错？** 错误！CRC 只能**检错**，不能纠错
8. ❌ **奇偶校验能纠错？** 错误！只能**检 1 位错**，不能纠错
