## 1. 存储器的分类

### 1.1 按存储介质分类

| 类型 | 特点 | 例子 |
|------|------|------|
| **半导体存储器** | 速度快，功耗低 | RAM、ROM |
| **磁表面存储器** | 容量大，价格低 | 硬盘、软盘 |
| **光存储器** | 容量大，非易失 | CD、DVD、蓝光 |

### 1.2 按存取方式分类

| 类型 | 特点 | 例子 |
|------|------|------|
| **随机存取存储器（RAM）** | 任意地址访问 | SRAM、DRAM |
| **只读存储器（ROM）** | 只读，非易失 | MROM、PROM、EPROM、EEPROM |
| **顺序存取存储器** | 顺序访问 | 磁带 |
| **直接存取存储器** | 直接定位+顺序访问 | 磁盘 |

### 1.3 按在计算机中的作用分类

| 类型 | 位置 | 特点 |
|------|------|------|
| **寄存器** | CPU内部 | 速度最快，容量最小 |
| **Cache** | CPU和主存之间 | 速度快，容量小 |
| **主存** | CPU外部 | 速度较快，容量较大 |
| **辅存** | 主存外部 | 速度慢，容量大 |

## 2. 主存与辅存

### 2.1 主存的基本结构

<span style="background-color:#ccc1d9">**主存**</span>：计算机运行时直接使用的存储器。

```
┌─────────────────────────────────────┐
│           存储体（存储单元）         │
├─────────────────────────────────────┤
│      地址寄存器 MAR（地址输入）     │
├─────────────────────────────────────┤
│      数据寄存器 MDR（数据输入输出） │
├─────────────────────────────────────┤
│        控制电路（读写控制）         │
└─────────────────────────────────────┘
```

### 2.2 主存的技术指标

| 指标 | 定义 |
|------|------|
| **存储容量** | 存储单元总数 |
| **存储字长** | 每个存储单元的位数 |
| **存取时间** | 从发出读写命令到完成的时间 |
| **存储周期** | 连续两次访问的最小时间间隔 |
| **带宽** | 单位时间内传输的数据量 |

### 2.3 辅存的特点

<span style="background-color:#ccc1d9">**辅存**</span>：用于存储大量数据，速度较慢但容量大。

**特点**：

- 容量大，价格低
- 非易失性
- 速度慢，需通过I/O访问
- 作为主存的扩展

## 3. 半导体随机存取存储器

### 3.1 SRAM

<span style="background-color:#ccc1d9">**SRAM**</span>：静态随机存取存储器。

**特点**：

- 用触发器存储数据
- 速度快，无需刷新
- 功耗大，集成度低

**应用**：Cache

### 3.2 DRAM

<span style="background-color:#ccc1d9">**DRAM**</span>：动态随机存取存储器。

**特点**：

- 用电容存储数据
- 速度较慢，需要刷新
- 功耗小，集成度高

**应用**：主存

### 3.3 DRAM的刷新

**刷新方式**：

1. **集中刷新**：一段时间内集中刷新所有行
2. **分散刷新**：每读写一次刷新一行
3. **异步刷新**：结合集中和分散

## 4. 只读存储器

### 4.1 ROM的分类

| 类型 | 特点 |
|------|------|
| **MROM** | 厂商掩膜写入，不可修改 |
| **PROM** | 用户一次性写入 |
| **EPROM** | 紫外线擦除，可多次写入 |
| **EEPROM** | 电擦除，可逐字节修改 |
| **Flash** | 电擦除，块级操作 |

### 4.2 ROM的应用

- 存储固定程序（BIOS）
- 存储常量数据
- 嵌入式系统

## 5. Cache基本原理

### 5.1 Cache的定义

<span style="background-color:#ccc1d9">**Cache**</span>：高速缓冲存储器，位于CPU和主存之间。

**目的**：解决CPU和主存之间的速度差距。

### 5.2 程序访问的局部性原理

<span style="background-color:#8db3e2">**局部性原理**：程序访问的数据和指令在时间和空间上具有局部性。</span>

**时间局部性**：最近访问的数据可能再次被访问。

**空间局部性**：访问的数据附近的数据可能被访问。

### 5.3 Cache的工作原理

```
CPU访问数据：
1. 先访问Cache
2. 命中：直接从Cache读取
3. 未命中：从主存读取，同时写入Cache
```

### 5.4 Cache的命中率

$$
\text{命中率} h = \frac{\text{Cache命中次数}}{\text{总访问次数}}
$$

$$
\text{平均访问时间} = h \cdot T_c + (1-h) \cdot T_m
$$

### 5.5 Cache的映射方式

#### 5.5.1 直接映射

```
Cache行号 = 主存块号 mod Cache行数
```

**特点**：简单，冲突率高。

#### 5.5.2 全相联映射

```
主存块可放入任意Cache行
```

**特点**：灵活，冲突率低，成本高。

#### 5.5.3 组相联映射

```
主存块放入指定组的任意行
组号 = 主存块号 mod 组数
```

**特点**：平衡直接映射和全相联映射。

### 5.6 Cache的替换策略

| 策略 | 描述 |
|------|------|
| **LRU** | 替换最近最少使用的块 |
| **LFU** | 替换最不经常使用的块 |
| **FIFO** | 替换最先进入的块 |
| **随机** | 随机替换 |

### 5.7 Cache的写策略

| 策略 | 描述 |
|------|------|
| **写直达** | 同时写入Cache和主存 |
| **写回** | 只写入Cache，替换时写回主存 |

## 6. 虚拟存储器

### 6.1 虚拟存储器的定义

<span style="background-color:#ccc1d9">**虚拟存储器**</span>：将主存和辅存统一管理，为用户提供一个比实际主存大得多的地址空间。

### 6.2 虚拟存储器的管理方式

#### 6.2.1 分页管理

```
虚拟地址 = 页号 + 页内偏移
```

**特点**：页大小固定，管理简单。

#### 6.2.2 分段管理

```
虚拟地址 = 段号 + 段内偏移
```

**特点**：段大小可变，便于保护和共享。

#### 6.2.3 段页式管理

```
虚拟地址 = 段号 + 页号 + 页内偏移
```

**特点**：结合分页和分段的优点。

### 6.3 页表

<span style="background-color:#ccc1d9">**页表**</span>：记录虚拟页号到物理页号的映射关系。

```
页表项 = 物理页号 + 有效位 + 修改位 + 访问位
```

### 6.4 地址变换过程

```
1. 根据虚拟页号查页表
2. 若有效位为1，得到物理页号
3. 物理地址 = 物理页号 × 页大小 + 页内偏移
4. 若有效位为0，发生缺页中断
```

### 6.5 快表（TLB）

<span style="background-color:#ccc1d9">**TLB**</span>：页表的Cache，加速地址变换。

**工作原理**：

1. 先查TLB
2. 命中：直接得到物理页号
3. 未命中：查页表，同时更新TLB

## 7. 多级存储体系

### 7.1 存储层次

```
寄存器 → Cache → 主存 → 辅存
```

| 层次 | 容量 | 速度 | 价格 |
|------|------|------|------|
| **寄存器** | 最小 | 最快 | 最贵 |
| **Cache** | 小 | 快 | 较贵 |
| **主存** | 中 | 中 | 中等 |
| **辅存** | 最大 | 最慢 | 最便宜 |

### 7.2 性能分析

$$
\text{平均访问时间} = T_1 + (1-h_1) \cdot T_2 + (1-h_1)(1-h_2) \cdot T_3 + ...
$$

## 8. 典型例题

### 例题 1：Cache命中率计算

**题目**：某系统Cache容量为2KB，主存容量为256KB，块大小为16B，采用直接映射。求主存地址为12345H的块映射到Cache的哪一行。

**解答**：

```
Cache行数 = 2KB / 16B = 128行
主存块号 = 12345H / 16 = 1234H
Cache行号 = 1234H mod 128 = 1234H mod 80H = 34H = 52

映射到Cache第52行。
```

**总结**：直接映射的行号计算使用取模运算。

**来源**：408历年真题，考查方式：计算题

---

### 例题 2：页表地址变换

**题目**：某系统页面大小为4KB，虚拟地址为123456H，页表项大小为4B，页表起始地址为1000H。求物理地址。

**解答**：

```
页大小 = 4KB = 2^12
页内偏移 = 123456H & 0FFFh = 456H
虚拟页号 = 123456H >> 12 = 123H

页表项地址 = 1000H + 123H × 4 = 1000H + 49CH = 149CH

假设页表项内容为567H，则物理页号 = 567H

物理地址 = 567H × 4096 + 456H = 567456H
```

**总结**：地址变换需要通过页表查找物理页号。

**来源**：408历年真题，考查方式：计算题

---

### 例题 3：平均访问时间计算

**题目**：某系统Cache访问时间为10ns，主存访问时间为100ns，命中率为95%。求平均访问时间。

**解答**：

```
平均访问时间 = h × Tc + (1-h) × Tm
            = 0.95 × 10 + 0.05 × 100
            = 9.5 + 5
            = 14.5ns
```

**总结**：提高命中率可以显著降低平均访问时间。

**来源**：408历年真题，考查方式：计算题
