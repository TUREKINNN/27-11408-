## 1. CPU的基本结构

### 1.1 CPU的组成

<span style="background-color:#ccc1d9">**CPU**</span>：中央处理器，计算机的核心部件。

```
┌─────────────────────────────────────────────────┐
│                    CPU                          │
├─────────────────────────────────────────────────┤
│              ┌───────────────┐                  │
│              │   运算器 ALU   │                  │
│              │ ┌───────────┐ │                  │
│              │ │ ACC/MQ/X  │ │                  │
│              │ └───────────┘ │                  │
│              └───────────────┘                  │
│              ┌───────────────┐                  │
│              │   控制器 CU   │                  │
│              │ ┌───────────┐ │                  │
│              │ │ IR/PC/    │ │                  │
│              │ │ MAR/MDR   │ │                  │
│              │ └───────────┘ │                  │
│              └───────────────┘                  │
│              ┌───────────────┐                  │
│              │    寄存器组   │                  │
│              └───────────────┘                  │
└─────────────────────────────────────────────────┘
```

### 1.2 运算器的组成

| 部件 | 功能 |
|------|------|
| **ALU** | 执行算术和逻辑运算 |
| **ACC** | 累加器，存放运算结果 |
| **MQ** | 乘商寄存器 |
| **X** | 操作数寄存器 |
| **PSW** | 程序状态字，记录运算状态 |

### 1.3 控制器的组成

| 部件 | 功能 |
|------|------|
| **CU** | 控制单元，产生控制信号 |
| **IR** | 指令寄存器，存放当前指令 |
| **PC** | 程序计数器，存放下条指令地址 |
| **MAR** | 地址寄存器，存放访问地址 |
| **MDR** | 数据寄存器，存放数据 |

## 2. 指令周期

### 2.1 指令周期的概念

<span style="background-color:#ccc1d9">**指令周期**</span>：从取指令到执行完的时间。

### 2.2 指令周期的阶段

| 阶段 | 功能 |
|------|------|
| **取指周期** | 从主存取出指令 |
| **间址周期** | 取操作数有效地址 |
| **执行周期** | 执行指令操作 |
| **中断周期** | 处理中断 |

### 2.3 不同指令的周期

| 指令类型 | 周期 |
|----------|------|
| **无操作数指令** | 取指 + 执行 |
| **直接寻址指令** | 取指 + 执行 |
| **间接寻址指令** | 取指 + 间址 + 执行 |

---

## 3. 时序系统

### 3.1 时序信号的作用

<span style="background-color:#ccc1d9">**时序信号**</span>：控制各部件协调工作的时间信号。

### 3.2 时序信号的层次

| 层次 | 描述 |
|------|------|
| **机器周期** | 完成一个基本操作的时间 |
| **时钟周期** | 最小时间单位 |
| **节拍** | 时钟周期内的细分 |

### 3.3 同步控制方式

**特点**：所有操作由统一时钟控制。

**优点**：简单，可靠。

**缺点**：速度慢。

### 3.4 异步控制方式

**特点**：操作之间用握手信号协调。

**优点**：速度快。

**缺点**：复杂。

### 3.5 联合控制方式

**特点**：结合同步和异步。

## 4. 微程序控制

### 4.1 微程序的概念

<span style="background-color:#ccc1d9">**微程序**</span>：由微指令组成的程序，用于解释执行机器指令。

### 4.2 微指令格式

```
┌──────────────┬───────────────────┐
│  微操作控制  │    顺序控制字段   │
│    字段      │                   │
└──────────────┴───────────────────┘
```

### 4.3 微指令的编码方式

#### 4.3.1 直接编码

**特点**：每一位代表一个微操作。

**优点**：速度快。

**缺点**：位数多。

#### 4.3.2 字段直接编码

**特点**：将微操作分组，每组用编码表示。

**优点**：节省位数。

**缺点**：速度稍慢。

#### 4.3.3 字段间接编码

**特点**：某些字段的编码含义由其他字段决定。

**优点**：更节省位数。

**缺点**：更复杂。

### 4.4 微程序的执行过程

```
1. 根据指令操作码查微指令地址
2. 读取微指令到微指令寄存器
3. 执行微指令的微操作
4. 根据顺序控制字段确定下条微指令地址
5. 重复直到指令执行完毕
```

## 5. 硬布线控制

### 5.1 硬布线控制的概念

<span style="background-color:#ccc1d9">**硬布线控制**</span>：用逻辑电路直接产生控制信号。

### 5.2 控制信号的产生

```
控制信号 = f(IR, PC, PSW, 时序信号)
```

### 5.3 硬布线控制的特点

**优点**：速度快。

**缺点**：复杂，不易修改。

### 5.4 微程序控制与硬布线控制的比较

| 对比项 | 微程序控制 | 硬布线控制 |
|--------|------------|------------|
| **速度** | 慢 | 快 |
| **灵活性** | 高 | 低 |
| **复杂度** | 低 | 高 |
| **可维护性** | 好 | 差 |

## 6. 流水线技术

### 6.1 流水线的基本概念

<span style="background-color:#ccc1d9">**流水线**</span>：将指令执行分解为多个阶段，并行执行。

### 6.2 流水线的种类

| 类型 | 描述 |
|------|------|
| **指令流水线** | 指令级并行 |
| **算术流水线** | 运算级并行 |
| **超标量流水线** | 多条流水线并行 |
| **超流水线** | 细分流水段 |

### 6.3 流水线的性能分析

#### 6.3.1 吞吐率

$$
\text{吞吐率} = \frac{n}{(k + n - 1) \cdot \Delta t}
$$

其中 $n$ 为指令数，$k$ 为流水段数，$\Delta t$ 为段延迟。

#### 6.3.2 加速比

$$
\text{加速比} = \frac{k \cdot n \cdot \Delta t}{(k + n - 1) \cdot \Delta t} = \frac{k \cdot n}{k + n - 1}
$$

#### 6.3.3 效率

$$
\text{效率} = \frac{\text{使用时间}}{\text{总时间}} = \frac{n \cdot k \cdot \Delta t}{(k + n - 1) \cdot k \cdot \Delta t} = \frac{n}{k + n - 1}
$$

### 6.4 流水线的冲突处理

#### 6.4.1 数据冲突

**解决方法**：

1. **数据旁路**：直接从执行阶段传送到需要的阶段
2. **暂停流水线**：插入气泡
3. **编译器优化**：指令重排

#### 6.4.2 控制冲突

**解决方法**：

1. **分支预测**：预测分支方向
2. **延迟分支**：分支后插入延迟槽
3. **动态分支预测**：根据历史记录预测

#### 6.4.3 结构冲突

**解决方法**：

1. **增加硬件资源**：多端口存储器
2. **时间片复用**：分时使用资源

## 7. RISC CPU设计

### 7.1 RISC的特点

| 特点 | 描述 |
|------|------|
| **精简指令** | 指令数量少，功能简单 |
| **固定长度** | 指令长度统一 |
| **Load/Store架构** | 只有Load/Store访问内存 |
| **大量寄存器** | 减少访存次数 |
| **流水线** | 充分利用流水线 |

### 7.2 RISC CPU的结构

```
┌─────────────────────────────────────────────────────┐
│                    RISC CPU                          │
├─────────────────────────────────────────────────────┤
│  指令缓存（I-Cache）  │  数据缓存（D-Cache）          │
├─────────────────────────────────────────────────────┤
│                   寄存器组（32个通用寄存器）          │
├─────────────────────────────────────────────────────┤
│              算术逻辑单元（ALU）                      │
├─────────────────────────────────────────────────────┤
│              指令流水线（IF-ID-EX-MEM-WB）           │
└─────────────────────────────────────────────────────┘
```

## 8. 典型例题

### 例题 1：指令周期分析

**题目**：分析一条加法指令的指令周期，包含取指、间址、执行阶段。

**解答**：

```
取指周期：
1. PC → MAR
2. 读主存 → MDR → IR
3. PC + 1 → PC

间址周期（间接寻址）：
1. IR地址码 → MAR
2. 读主存 → MDR → IR地址码

执行周期：
1. IR地址码 → MAR
2. 读主存 → MDR → ACC
3. ACC + MDR → ACC
```

**总结**：不同寻址方式的指令周期包含不同阶段。

**来源**：408历年真题，考查方式：综合题

---

### 例题 2：流水线性能计算

**题目**：某4段流水线，每段延迟2ns，执行100条指令。求吞吐率、加速比和效率。

**解答**：

```
吞吐率 = 100 / [(4 + 100 - 1) × 2] = 100 / 206 ≈ 0.485 条/ns

加速比 = (4 × 100) / (4 + 100 - 1) = 400 / 103 ≈ 3.88

效率 = 100 / (4 + 100 - 1) = 100 / 103 ≈ 0.97 = 97%
```

**总结**：流水线能显著提高CPU的执行效率。

**来源**：408历年真题，考查方式：计算题

---

### 例题 3：微指令编码

**题目**：设计微指令格式，包含8个微操作，使用字段直接编码，分两组，每组4个微操作。

**解答**：

```
每组4个微操作需要2位编码（2^2 = 4）

微指令格式：
┌──────┬──────┬──────────┐
│ 组1  │ 组2  │  顺序控制 │
│ 2位  │ 2位  │    4位   │
└──────┴──────┴──────────┘

总位数：2 + 2 + 4 = 8位

对比直接编码：8位（每个微操作1位）

节省：0位

但如果微操作数量增加，字段编码优势明显。
```

**总结**：字段编码能有效减少微指令长度。

**来源**：408历年真题，考查方式：综合题
