## 1. 计算机的基本组成

### 1.1 计算机硬件的五大部件

<span style="background-color:#ccc1d9">**五大部件**</span>：运算器、控制器、存储器、输入设备、输出设备。

| 部件 | 功能 |
|------|------|
| **运算器** | 执行算术运算和逻辑运算 |
| **控制器** | 控制计算机各部件协同工作 |
| **存储器** | 存储程序和数据 |
| **输入设备** | 向计算机输入数据 |
| **输出设备** | 将计算机处理结果输出 |

### 1.2 CPU的组成

<span style="background-color:#ccc1d9">**CPU**</span>：由运算器和控制器组成。

- **运算器**：ALU（算术逻辑单元）、ACC（累加器）、MQ（乘商寄存器）、X（操作数寄存器）
- **控制器**：CU（控制单元）、IR（指令寄存器）、PC（程序计数器）、MAR（地址寄存器）、MDR（数据寄存器）

## 2. 冯·诺依曼结构

### 2.1 冯·诺依曼的基本思想

<span style="background-color:#8db3e2">**冯·诺依曼思想**：存储程序，程序和数据以二进制形式存储在同一存储器中。</span>

### 2.2 冯·诺依曼结构的特点

1. **计算机由五大部件组成**
2. **指令和数据以同等地位存于存储器**
3. **指令和数据均用二进制表示**
4. **指令由操作码和地址码组成**
5. **指令在存储器中按顺序存放**
6. **以运算器为中心**

### 2.3 冯·诺依曼瓶颈

<span style="background-color:#ccc1d9">**冯·诺依曼瓶颈**</span>：CPU与存储器之间的速度差距，限制了计算机性能。

## 3. 计算机系统的层次结构

### 3.1 层次结构的划分

| 层次 | 名称 | 核心部件 |
|------|------|----------|
| L1 | 微程序机器级 | 微指令 |
| L2 | 传统机器级 | 机器指令 |
| L3 | 操作系统级 | 系统调用 |
| L4 | 汇编语言级 | 汇编指令 |
| L5 | 高级语言级 | 高级语言 |
| L6 | 应用语言级 | 应用程序 |

### 3.2 各层次的功能

- **L1-L2**：硬件实现，执行机器指令
- **L3**：操作系统，管理资源
- **L4-L5**：语言处理，方便编程
- **L6**：应用层，解决具体问题

## 4. 指令系统

### 4.1 指令的定义

<span style="background-color:#ccc1d9">**指令**</span>：计算机能执行的基本操作命令。

### 4.2 指令格式

```
┌──────────────┬──────────────┐
│  操作码 OP   │   地址码 A   │
└──────────────┴──────────────┘
```

**操作码**：表示指令要执行的操作类型。

**地址码**：表示操作数的地址。

### 4.3 指令的分类

| 类型 | 描述 |
|------|------|
| **数据传送类** | 数据在寄存器和存储器之间传送 |
| **算术运算类** | 执行算术运算（加、减、乘、除） |
| **逻辑运算类** | 执行逻辑运算（与、或、非、异或） |
| **移位操作类** | 数据移位 |
| **程序控制类** | 控制程序执行顺序（跳转、调用） |
| **输入输出类** | 与外部设备交互 |

### 4.4 指令字长

- **单字长指令**：一个机器字长
- **半字长指令**：半个机器字长
- **双字长指令**：两个机器字长

## 5. 寻址方式

### 5.1 寻址方式的定义

<span style="background-color:#ccc1d9">**寻址方式**</span>：寻找操作数或指令地址的方式。

### 5.2 常见寻址方式

#### 5.2.1 立即寻址

```
指令 = OP + 立即数
```

**特点**：操作数就在指令中，无需访问内存。

#### 5.2.2 直接寻址

```
指令 = OP + 操作数地址
```

**特点**：指令中直接给出操作数地址。

#### 5.2.3 间接寻址

```
指令 = OP + 操作数地址的地址
```

**特点**：指令中给出的是操作数地址的地址。

#### 5.2.4 寄存器寻址

```
指令 = OP + 寄存器号
```

**特点**：操作数在寄存器中，速度快。

#### 5.2.5 寄存器间接寻址

```
指令 = OP + 寄存器号（寄存器中存操作数地址）
```

**特点**：寄存器中存储的是操作数的地址。

#### 5.2.6 相对寻址

```
有效地址 = PC + 偏移量
```

**特点**：以PC为基准，适合程序浮动。

#### 5.2.7 基址寻址

```
有效地址 = 基址寄存器 + 偏移量
```

**特点**：适合多道程序设计。

#### 5.2.8 变址寻址

```
有效地址 = 变址寄存器 + 偏移量
```

**特点**：适合数组操作。

### 5.3 寻址方式的比较

| 寻址方式 | 优点 | 缺点 |
|----------|------|------|
| **立即寻址** | 速度快 | 操作数固定 |
| **直接寻址** | 简单 | 地址空间有限 |
| **间接寻址** | 地址空间大 | 访问内存两次 |
| **寄存器寻址** | 速度快 | 寄存器数量有限 |
| **相对寻址** | 程序浮动 | 偏移量有限 |
| **基址寻址** | 多道程序 | 需要基址寄存器 |
| **变址寻址** | 数组操作 | 需要变址寄存器 |

## 6. 计算机的性能指标

### 6.1 基本性能指标

| 指标 | 定义 |
|------|------|
| **CPU时钟频率** | CPU每秒执行的时钟周期数 |
| **CPI** | 每条指令的平均时钟周期数 |
| **MIPS** | 每秒执行的百万条指令数 |
| **FLOPS** | 每秒执行的浮点运算次数 |

### 6.2 计算关系

$$
\text{CPU执行时间} = \frac{\text{指令数} \times \text{CPI}}{\text{时钟频率}}
$$

$$
\text{MIPS} = \frac{\text{时钟频率}}{\text{CPI} \times 10^6}
$$

### 6.3 其他性能指标

- **吞吐量**：单位时间处理的数据量
- **响应时间**：从请求到响应的时间
- **利用率**：资源使用时间的比例

## 7. 典型例题

### 例题 1：指令格式设计

**题目**：设计一个指令格式，包含操作码、寄存器号、偏移量。

**解答**：

```
┌──────┬──────┬────────────────┐
│ OP   │ R    │     offset     │
│ 6位  │ 5位  │      21位      │
└──────┴──────┴────────────────┘
```

```
操作码：6位，可表示2^6 = 64种操作
寄存器号：5位，可表示2^5 = 32个寄存器
偏移量：21位，可表示±2^20 = ±1M的地址范围
```

**总结**：指令格式设计需要平衡操作种类、寄存器数量和寻址范围。

**来源**：408历年真题，考查方式：综合题

---

### 例题 2：寻址方式计算

**题目**：已知基址寄存器内容为1000H，变址寄存器内容为2000H，指令中地址码为3000H，分别计算直接寻址、基址寻址、变址寻址的有效地址。

**解答**：

```
直接寻址：EA = 3000H

基址寻址：EA = 基址寄存器 + 地址码
            = 1000H + 3000H = 4000H

变址寻址：EA = 变址寄存器 + 地址码
            = 2000H + 3000H = 5000H
```

**总结**：不同寻址方式的有效地址计算方式不同。

**来源**：408历年真题，考查方式：选择题

---

### 例题 3：CPU性能计算

**题目**：某CPU时钟频率为2GHz，CPI为2，求MIPS。

**解答**：

```
MIPS = 时钟频率 / (CPI × 10^6)
     = 2 × 10^9 / (2 × 10^6)
     = 1000
```

**总结**：MIPS是衡量CPU运算速度的重要指标。

**来源**：408历年真题，考查方式：计算题
