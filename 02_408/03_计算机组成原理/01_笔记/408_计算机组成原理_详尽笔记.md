# 第一章 计算机系统概述

## 1.1 计算机的基本组成

### 1.1.1 硬件的五大部件

> 现代计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

| 部件 | 功能 |
|:-----|:-----|
| **运算器** | 执行算术运算和逻辑运算 |
| **控制器** | 控制计算机各部件协同工作 |
| **存储器** | 存储程序和数据 |
| **输入设备** | 向计算机输入数据 |
| **输出设备** | 将计算机处理结果输出 |

> **「运控存输入输出」**——运算器、控制器、存储器、输入、输出

### 1.1.2 CPU 的组成


**运算器内部组成**：
- **ALU**：算术逻辑单元，执行算术/逻辑运算
- **ACC**：累加寄存器，存放操作数/运算结果
- **MQ**：乘商寄存器，乘除法时使用
- **X**：操作数寄存器，存放操作数

**控制器内部组成**：
- **CU**：控制单元，分析指令、发出控制信号
- **IR**：指令寄存器，存放当前执行的指令
- **PC**：程序计数器，存放下一条指令地址
- **MAR**：地址寄存器，存放访问存储器的地址
- **MDR**：数据寄存器，存放与存储器交换的数据

---

## 1.2 冯·诺依曼结构

### 1.2.1 冯·诺依曼思想

> **存储程序（Stored-Program）**：程序和数据以二进制形式同等存放在存储器中，计算机按程序顺序自动执行。

### 1.2.2 冯·诺依曼结构六大特点

1. 计算机由**五大部件**组成
2. **指令和数据以同等地位**存于存储器
3. 指令和数据**均用二进制**表示
4. 指令由**操作码**和**地址码**组成
5. 指令在存储器中**按顺序存放**
6. **以运算器为中心**（现代改进为以存储器为中心）

### 1.2.3 冯·诺依曼瓶颈

> CPU 与存储器之间数据传输带宽有限，成为系统性能瓶颈。解决方案：采用**Cache**、**流水线**、**并行**等技术。

> - 冯氏以**运算器**为中心
> - 现代以**存储器**为中心（CPU 内部数据流向更高效）
> - 两者都遵循"存储程序"思想，本质相同

## 1.3 计算机系统的层次结构

### 1.3.1 层次划分

| 层次 | 名称 | 核心部件 | 归属 |
|:-----|:-----|:---------|:-----|
| **L6** | 应用语言级 | 应用程序 | 软件 |
| **L5** | 高级语言级 | 高级语言 | 软件 |
| **L4** | 汇编语言级 | 汇编指令 | 软件 |
| **L3** | 操作系统级 | 系统调用 | 软件/硬件交界面 |
| **L2** | 传统机器级 | 机器指令 | 硬件 |
| **L1** | 微程序机器级 | 微指令 | 硬件（微程序控制器） |

### 1.3.2 各层功能

- **L1~L2（硬件层）**：执行机器指令
- **L3（系统软件）**：管理硬件资源
- **L4~L5（语言层）**：提供编程接口
- **L6（应用层）**：解决具体问题

> - **编译**：高级语言 → 汇编语言/机器语言
> - **解释**：逐条翻译并执行
> - **翻译**：源代码 → 目标代码

## 1.4 指令系统

### 1.4.1 指令的定义

> 指令是计算机能执行的基本操作命令，一组指令的集合称为**指令系统/指令集（ISA）**。

### 1.4.2 指令格式

```
┌──────────────┬──────────────┐
│  操作码 OP   │   地址码 A   │
└──────────────┴──────────────┘
```

- **操作码（OP）**：表示指令的操作类型
- **地址码（A）**：表示操作数地址

### 1.4.3 指令分类

| 类型 | 描述 |
|:-----|:-----|
| **数据传送类** | 数据在寄存器/存储器间传送（如 MOV） |
| **算术运算类** | 加、减、乘、除 |
| **逻辑运算类** | 与、或、非、异或 |
| **移位操作类** | 左移、右移、循环移位 |
| **程序控制类** | 跳转、调用、返回 |
| **输入输出类** | 与外设交互 |

### 1.4.4 指令字长

- **单字长指令**：一个机器字长
- **半字长指令**：半个机器字长
- **双字长指令**：两个机器字长

> - **定长指令字**：译码简单、速度快（如 RISC）
> - **变长指令字**：灵活、节省空间（如 x86）

## 1.5 寻址方式

### 1.5.1 寻址方式定义

> **寻址方式**：寻找**操作数**或**指令地址**的方式。寻址方式的设计直接决定指令系统的灵活性和效率。

### 1.5.2 七大寻址方式

| 寻址方式 | 有效地址（EA） | 特点 |
|:---------|:--------------|:-----|
| **立即寻址** | 操作数 = 指令中立即数 | 速度最快，操作数固定 |
| **直接寻址** | EA = A（指令中地址码） | 简单，地址空间有限 |
| **间接寻址** | EA = (A)（访问两次内存） | 扩大寻址范围 |
| **寄存器寻址** | 操作数在 Ri 中 | 速度极快 |
| **寄存器间接寻址** | EA = (Ri) | 寄存器+间接 |
| **相对寻址** | EA = PC + A | 程序可浮动 |
| **基址寻址** | EA = BR + A | 多道程序设计 |
| **变址寻址** | EA = IX + A | 数组/循环 |

> - **基址寻址**：基址寄存器内容**不变**，地址码变 → 系统管理用
> - **变址寻址**：变址寄存器内容**可变**，地址码固定 → 用户编程用
> - **相对寻址**：以**PC 当前值**为基准，常用于转移指令

### 1.5.3 寻址方式比较

| 寻址方式 | 优点 | 缺点 |
|:---------|:-----|:-----|
| 立即寻址 | 速度快 | 操作数固定 |
| 直接寻址 | 简单 | 地址空间有限 |
| 间接寻址 | 地址空间大 | 访问内存两次 |
| 寄存器寻址 | 速度最快 | 寄存器数有限 |
| 相对寻址 | 程序可浮动 | 偏移量有限 |
| 基址寻址 | 多道程序 | 需基址寄存器 |
| 变址寻址 | 数组操作方便 | 需变址寄存器 |

## 1.6 计算机性能指标

### 1.6.1 基本性能指标

| 指标 | 含义 |
|:-----|:-----|
| **CPU 时钟频率** | CPU 每秒执行的时钟周期数（Hz） |
| **CPI** | 每条指令的平均时钟周期数（Cycles Per Instruction） |
| **MIPS** | 每秒百万条指令数 |
| **FLOPS** | 每秒浮点运算次数 |

### 1.6.2 核心公式

**CPU 执行时间**：

$$
T_{CPU} = rac{	ext{指令数} 	imes 	ext{CPI}}{	ext{时钟频率}}
$$

**MIPS 计算**：

$$
	ext{MIPS} = rac{	ext{时钟频率}}{	ext{CPI} 	imes 10^6}
$$

**CPU 时钟周期**：

$$
T = rac{1}{f}
$$

### 1.6.3 其他性能指标

- **吞吐量**：单位时间处理的数据量
- **响应时间**：从请求到响应的时间
- **利用率**：资源被使用的时间比例
- **带宽**：单位时间传输的数据量

> 某 CPU 时钟频率 2 GHz，CPI = 2，则：
> - MIPS = 2×10⁹ / (2×10⁶) = **1000 MIPS**
> - 每条指令耗时 = 1ns

## 1.7 典型例题

### 例题 1：指令格式设计

**题目**：设计一个指令格式，包含 6 位操作码、5 位寄存器号、21 位偏移量，求各类容量。

**解答**：

```
┌──────┬──────┬────────────────┐
│ OP   │ R    │     offset     │
│ 6位  │ 5位  │      21位      │
└──────┴──────┴────────────────┘
```

- 操作码：6 位 → $2^6 = 64$ 种操作
- 寄存器号：5 位 → $2^5 = 32$ 个寄存器
- 偏移量：21 位 → $\pm 2^{20} = \pm 1	ext{M}$ 的寻址范围

**总结**：指令格式设计需在**操作种类、寄存器数、寻址范围**三者间平衡。

### 例题 2：寻址方式计算

**题目**：基址寄存器 BR = 1000H，变址寄存器 IX = 2000H，指令中地址码 A = 3000H，分别求直接/基址/变址的有效地址。

**解答**：

| 寻址方式 | 公式 | EA |
|:---------|:-----|:--|
| **直接寻址** | EA = A | 3000H |
| **基址寻址** | EA = BR + A | 1000H + 3000H = 4000H |
| **变址寻址** | EA = IX + A | 2000H + 3000H = 5000H |

**总结**：不同寻址方式的有效地址计算公式不同。

### 例题 3：CPU 性能计算

**题目**：某 CPU 时钟频率 2 GHz，CPI = 2，求 MIPS。

**解答**：

$$
	ext{MIPS} = rac{f}{	ext{CPI} 	imes 10^6} = rac{2 	imes 10^9}{2 	imes 10^6} = 1000 	ext{ MIPS}
$$

**总结**：MIPS 是衡量 CPU 运算速度的重要指标，仅适用于**定长指令**情形。

# 第二章 数据表示与运算

## 2.1 数制与编码

### 2.1.1 数制的概念

> **数制**：用一组固定的符号和统一的规则表示数值的方法。常见的有二进制、八进制、十进制、十六进制。

### 2.1.2 常用数制

| 数制 | 基数 | 符号 | 示例 |
|:-----|:-----|:-----|:-----|
| **二进制** | 2 | 0, 1 | 1010B |
| **八进制** | 8 | 0-7 | 12O |
| **十进制** | 10 | 0-9 | 10D |
| **十六进制** | 16 | 0-9, A-F | 0AH |

### 2.1.3 数制转换

**任意进制 → 十进制**：按权展开

$$
(N)_b = \sum_{i=-m}^{n-1} d_i \cdot b^i
$$

**十进制 → 二进制**：
- **整数部分**：除 2 取余，**逆序**排列
- **小数部分**：乘 2 取整，**顺序**排列

---

## 2.2 定点数

### 2.2.1 定点数的定义

> **定点数**：小数点位置**固定不变**的数。分为**定点整数**和**定点小数**。

### 2.2.2 四种机器码编码

| 编码 | 正数 | 负数 | 范围（含 $n$ 位数值） |
|:-----|:-----|:-----|:---------------------|
| **原码** | 0 + 数值位 | 1 + 数值位 | $-(1-2^{-n}) \sim (1-2^{-n})$ |
| **反码** | 0 + 数值位 | 1 + 各位取反 | 同原码 |
| **补码** | 0 + 数值位 | 反码 + 1 | $-1 \sim (1-2^{-n})$ |
| **移码** | 补码 + $2^{n-1}$ | 用于阶码 | $-2^{n-1} \sim (2^{n-1}-1)$ |

**核心公式**：
- $[X+Y]_	ext{补} = [X]_	ext{补} + [Y]_	ext{补}$
- $[X-Y]_	ext{补} = [X]_	ext{补} + [-Y]_	ext{补}$
- $[-Y]_	ext{补} = 	ext{连符号位取反} + 1$（或 $[Y]_	ext{补} - 2^n$）

> - 移码 = 补码符号位取反
> - 移码**便于比较大小**（指数码部分）
> - 移码主要用于**浮点数阶码**

### 2.2.3 溢出判断

**方法 1：双符号位法（变形补码）**

| 双符号位 | 含义 |
|:--------|:-----|
| 00 | 正数，无溢出 |
| 01 | 正溢出（上溢） |
| 10 | 负溢出（下溢） |
| 11 | 负数，无溢出 |

**方法 2：单符号位进位法**
- 最高位进位 $\oplus$ 符号位进位 = 1 → 溢出
- 即 $C_1 \oplus C_2 = 1$ 时溢出

> - 最高位进位 $C_1$ 与符号位进位 $C_2$ **不同时** → 溢出
> - 模 4 补码 = 双符号位补码，溢出判断直观

## 2.3 浮点数

### 2.3.1 浮点数表示

$$
N = M 	imes r^E
$$

- **M**：尾数（有效数字）
- **r**：基数（通常为 2）
- **E**：阶码（指数）

### 2.3.2 IEEE 754 标准

```
┌────┬──────────┬─────────────────────────────┐
│ S  │    E     │              M              │
│ 1位 │  8/11位  │        23/52位              │
└────┴──────────┴─────────────────────────────┘
```

**单精度（32 位）**：

| 字段 | 长度 | 说明 |
|:-----|:-----|:-----|
| **S** | 1 | 符号位 |
| **E** | 8 | 阶码，偏移量 127 |
| **M** | 23 | 尾数（隐含 1） |

**双精度（64 位）**：

| 字段 | 长度 | 说明 |
|:-----|:-----|:-----|
| **S** | 1 | 符号位 |
| **E** | 11 | 阶码，偏移量 1023 |
| **M** | 52 | 尾数（隐含 1） |

> 规格化数隐含最高位 1，**实际尾数精度 = 24 位/53 位**。

### 2.3.3 IEEE 754 特殊值

| 阶码 E | 尾数 M | 表示的值 |
|:-------|:-------|:---------|
| 0 | 0 | ±0（由符号位决定） |
| 0 | ≠ 0 | 非规格化数 |
| 1~254 (单) | 任意 | 规格化数 $\pm 1.M 	imes 2^{E-127}$ |
| 255 (单) | 0 | ±∞ |
| 255 (单) | ≠ 0 | NaN（非数） |

### 2.3.4 浮点数运算步骤

**加法/减法五步**：

1. **对阶**：小阶向大阶看齐
2. **尾数求和差**：定点数加法/减法
3. **规格化**：调整为 $1/2 \le |M| < 1$
4. **舍入**：处理多余位数（0 舍 1 入/恒置 1）
5. **溢出判断**：检查阶码是否溢出

**对阶规则**：

- $\Delta E = E_1 - E_2 > 0$：$M_2$ 右移 $\Delta E$ 位，$E_2 += \Delta E$
- $\Delta E = E_1 - E_2 < 0$：$M_1$ 右移 $-\Delta E$ 位，$E_1 += -\Delta E$
- **小阶向大阶看齐**，**尾数右移**

**规格化**：
- **左规**：尾数左移，阶码减 1（直到最高位为 1）
- **右规**：尾数右移，阶码加 1（尾数溢出时）

> - 浮点数加法**不满足结合律**：$(a+b)+c 
e a+(b+c)$
> - 浮点数加法**不满足分配律**

## 2.4 算术逻辑单元（ALU）

### 2.4.1 一位全加器

**输出**：

$$
S_i = A_i \oplus B_i \oplus C_{i-1}
$$

**进位**：

$$
C_i = A_i B_i + (A_i \oplus B_i) C_{i-1}
$$

### 2.4.2 多位加法器

**串行进位加法器**：
- 低位进位串行传到高位
- 速度慢（每级都有一个 $T$ 延迟）
- 共 $nT$ 延迟

**并行进位加法器（CLA）**：

| 概念 | 公式 | 含义 |
|:-----|:-----|:-----|
| **进位生成项 $G_i$** | $A_i B_i$ | 本位必产生进位 |
| **进位传递项 $P_i$** | $A_i \oplus B_i$ | 本位能否传递进位 |
| **进位公式** | $C_i = G_i + P_i C_{i-1}$ | 各位并行计算 |

> - 4 位并行进位加法器仅需约 $2T$ 延迟
> - 16 位以上需采用**分组并行进位**

## 2.5 乘法与除法

### 2.5.1 原码一位乘法

**思想**：符号位单独处理，数值位逐位相乘。

$$
S = S_1 \oplus S_2, \quad |P| = |X| 	imes |Y|
$$

### 2.5.2 补码一位乘法（Booth 算法）

**思想**：根据相邻两位 $Y_i Y_{i-1}$ 决定加减操作。

| $Y_i Y_{i-1}$ | 操作 |
|:--------------|:-----|
| 00 | +0 |
| 01 | +$|X|$ |
| 10 | $-|X|$ |
| 11 | +0 |

> - 统一处理**正负数**，无需单独处理符号
> - 适合计算机实现

### 2.5.3 除法

| 方法 | 思路 | 特点 |
|:-----|:-----|:-----|
| **恢复余数法** | 余数 < 0 时 + 除数 | 步骤多、速度慢 |
| **加减交替法** | 余数 < 0 时左移 + 除数 | 步骤统一、速度快 |

## 2.6 数据的存储与排列

### 2.6.1 大端序 vs 小端序

> - **大端序（Big Endian）**：**高位字节**存低地址（人类阅读习惯）
> - **小端序（Little Endian）**：**低位字节**存低地址（x86 采用）

**示例**：0x12345678 存储在 1000H 起始地址

| 模式 | 1000H | 1001H | 1002H | 1003H |
|:-----|:------|:------|:------|:------|
| 大端 | 12 | 34 | 56 | 78 |
| 小端 | 78 | 56 | 34 | 12 |

### 2.6.2 边界对齐

> **边界对齐**：数据存储在地址是其大小**整数倍**的位置。

- **优点**：访问速度快（1 次总线操作）
- **缺点**：可能浪费空间

> int (4 字节) 起始地址：0x0000, 0x0004, 0x0008 ...

## 2.7 典型例题

### 例题 1：补码加减运算

**题目**：$X = 0.1011$，$Y = -0.0101$，求 $[X+Y]_	ext{补}$ 和 $[X-Y]_	ext{补}$。

**解答**：

```
[X]补 = 0.1011
[Y]补 = 1.1011
[-Y]补 = 0.0101

X+Y：
  0.1011
+ 1.1011
--------
 10.0110 → 0.0110（溢出位丢弃）

X-Y：
  0.1011
+ 0.0101
--------
  0.1110
```

**总结**：补码运算将减法转化为加法。

### 例题 2：IEEE 754 转换

**题目**：十进制 10.5 → IEEE 754 单精度浮点数。

**解答**：

$$
10.5 = 1010.1B = 1.0101 	imes 2^3
$$

| 字段 | 值 |
|:-----|:--|
| **S** | 0（正数）|
| **E** | $3 + 127 = 130 = 10000010B$ |
| **M** | 0101 0000 ... (23 位) |

```
结果：0 10000010 01010000000000000000000 = 41280000H
```

**总结**：IEEE 754 是浮点数的工业标准。

### 例题 3：浮点数加法

**题目**：$X = 0.1101 	imes 2^{10}$，$Y = 0.1011 	imes 2^{01}$，求 $X + Y$。

**解答**：

```
① 对阶：ΔE = 10 - 01 = 1
   Y 尾数右移 1 位：0.01011 × 2^10

② 尾数相加：
   0.11010
 + 0.01011
 --------
   1.00101

③ 规格化（左规）：
   1.00101 → 0.100101 × 2^11

④ 舍入：0.1001 × 2^11
```

**总结**：浮点数加法的关键是**对阶**和**规格化**。

# 第三章 存储器层次结构

## 3.1 存储器的分类

### 3.1.1 按存储介质分类

| 类型 | 特点 | 例子 |
|:-----|:-----|:-----|
| **半导体存储器** | 速度快，功耗低 | RAM、ROM |
| **磁表面存储器** | 容量大，价格低 | 硬盘、磁带 |
| **光存储器** | 容量大，非易失 | CD、DVD、蓝光 |

### 3.1.2 按存取方式分类

| 类型 | 特点 | 例子 |
|:-----|:-----|:-----|
| **随机存取存储器（RAM）** | 任意地址访问 | SRAM、DRAM |
| **只读存储器（ROM）** | 只读，非易失 | MROM、PROM、EPROM、Flash |
| **顺序存取存储器（SAM）** | 顺序访问 | 磁带 |
| **直接存取存储器（DAM）** | 直接定位+顺序访问 | 磁盘 |

### 3.1.3 按在计算机中的作用分类

| 类型 | 位置 | 特点 |
|:-----|:-----|:-----|
| **寄存器** | CPU 内部 | 速度最快，容量最小 |
| **Cache** | CPU 与主存之间 | 速度快，容量小 |
| **主存** | CPU 外部 | 速度较快，容量较大 |
| **辅存** | 主存外部 | 速度慢，容量大 |

---

## 3.2 主存的基本结构

### 3.2.1 主存结构

```
┌─────────────────────────────────────┐
│           存储体（存储单元）         │
├─────────────────────────────────────┤
│      地址寄存器 MAR（地址输入）     │
├─────────────────────────────────────┤
│      数据寄存器 MDR（数据输入输出） │
├─────────────────────────────────────┤
│        控制电路（读写控制）         │
└─────────────────────────────────────┘
```

### 3.2.2 主存的技术指标

| 指标 | 定义 |
|:-----|:-----|
| **存储容量** | 存储单元总数 |
| **存储字长** | 每个存储单元的位数 |
| **存取时间** | 从发出读写命令到完成的时间 |
| **存储周期** | 连续两次访问的最小时间间隔 |
| **带宽** | 单位时间内传输的数据量 |

> - **存取时间 $T_a$**：单次访问所需时间
> - **存储周期 $T_m$**：连续两次访问的**最小间隔**（含恢复时间）
> - 通常 $T_m > T_a$

---

## 3.3 半导体随机存取存储器

### 3.3.1 SRAM 与 DRAM 对比

| 比较项 | **SRAM（静态）** | **DRAM（动态）** |
|:-------|:----------------|:----------------|
| **存储元件** | 触发器（双稳态） | 电容 |
| **刷新** | **不需要** | **需要定期刷新** |
| **速度** | 快 | 较慢 |
| **集成度** | 低 | 高 |
| **功耗** | 大 | 小 |
| **价格** | 高 | 低 |
| **应用** | Cache | 主存 |

### 3.3.2 DRAM 的刷新方式

| 刷新方式 | 描述 | 特点 |
|:---------|:-----|:-----|
| **集中刷新** | 一段时间内集中刷新所有行 | 存在"死区"（刷新期间不能访问）|
| **分散刷新** | 每读写一次刷新一行 | 速度降低，无死区 |
| **异步刷新** | 2ms 内每行刷新一次，分散进行 | 折中方案，最常用 |

> - 刷新由 DRAM 内部电路完成
> - 刷新操作对 CPU **不可见**
> - 刷新期间不能进行读写操作

## 3.4 只读存储器（ROM）

### 3.4.1 ROM 的分类

| 类型 | 特点 |
|:-----|:-----|
| **MROM** | 厂商掩膜写入，**不可修改** |
| **PROM** | 用户**一次性**写入 |
| **EPROM** | 紫外线擦除，可**多次**写入 |
| **EEPROM** | 电擦除，**逐字节**修改 |
| **Flash** | 电擦除，**块级**操作（速度比 EEPROM 快） |

> - **NOR Flash**：支持随机读，**写慢**，常用于 BIOS
> - **NAND Flash**：连续读，**写快**，常用于 U 盘、SSD

## 3.5 Cache 基本原理

### 3.5.1 局部性原理

> 程序访问的数据和指令在**时间**和**空间**上具有局部性。
> - **时间局部性**：最近访问的数据可能**再次**被访问
> - **空间局部性**：访问的数据**附近**的数据可能被访问

### 3.5.2 Cache 的工作原理

```
CPU 访问数据：
1. 先访问 Cache
2. 命中（Hit）：直接从 Cache 读取
3. 未命中（Miss）：从主存读取，同时写入 Cache
```

### 3.5.3 命中率与平均访问时间

**命中率**：

$$
h = rac{	ext{Cache 命中次数}}{	ext{总访问次数}}
$$

**平均访问时间**：

$$
T_a = h \cdot T_c + (1 - h) \cdot T_m
$$

> 设有两级 Cache（C1, C2），平均访问时间：

$$
T_a = h_1 T_{c1} + (1-h_1) h_2 T_{c2} + (1-h_1)(1-h_2) T_m
$$

### 3.5.4 Cache 的三种映射方式

| 映射方式 | 规则 | 优点 | 缺点 |
|:---------|:-----|:-----|:-----|
| **直接映射** | `Cache行号 = 主存块号 mod Cache行数` | 硬件简单，访问快 | 冲突率高 |
| **全相联映射** | 主存块可放任意 Cache 行 | 灵活，冲突率低 | 查找慢，硬件贵 |
| **组相联映射** | `组号 = 主存块号 mod 组数` | 折中方案 | 复杂度中等 |

> - **直接映射**：标记 + 行号 + 块内地址
> - **全相联映射**：标记 + 块内地址
> - **组相联映射**：标记 + 组号 + 块内地址

### 3.5.5 Cache 替换策略

| 策略 | 全称 | 描述 | 命中率 |
|:-----|:-----|:-----|:------|
| **LRU** | Least Recently Used | 替换最近最少使用 | 较高 |
| **LFU** | Least Frequently Used | 替换最不经常使用 | 较高 |
| **FIFO** | First In First Out | 替换最先进入 | 一般 |
| **Random** | 随机替换 | 随机选择 | 最低 |

> - 全相联/组相联 Cache 中 LRU 实现复杂
> - 一般采用**伪 LRU**（如"树形伪 LRU"）

### 3.5.6 Cache 写策略

| 策略 | 描述 | 优点 | 缺点 |
|:-----|:-----|:-----|:-----|
| **写直达（Write Through）** | 同时写 Cache 和主存 | 一致性好 | 访存次数多 |
| **写回（Write Back）** | 只写 Cache，替换时写主存 | 访存次数少 | 一致性需维护 |
| **写一次（Write Once）** | 第一次写直达，之后写回 | 折中 | 复杂 |

> 写回策略下，每个 Cache 行需 1 位**脏位**，标记是否被修改过。

## 3.6 虚拟存储器

### 3.6.1 虚拟存储器定义

> **虚拟存储器**：将主存和辅存统一管理，为用户提供一个比**实际主存大得多**的地址空间。

### 3.6.2 三种管理方式

| 方式 | 虚拟地址结构 | 特点 |
|:-----|:------------|:-----|
| **分页** | 页号 + 页内偏移 | 页大小固定，**无外碎片**，有内碎片 |
| **分段** | 段号 + 段内偏移 | 段大小可变，便于保护和共享 |
| **段页式** | 段号 + 页号 + 页内偏移 | 结合分页和分段优点 |

### 3.6.3 页表与 TLB

**页表项结构**：

```
┌──────────┬────────┬────────┬────────┬───┐
│ 物理页号 │ 有效位 │ 修改位 │ 访问位 │ 保护位 ...
└──────────┴────────┴────────┴────────┴───┘
```

**快表（TLB, Translation Lookaside Buffer）**：

> **TLB**：页表的 Cache，存储最近使用的页表项。
> - **TLB 命中**：直接得到物理页号（极快）
> - **TLB 未命中**：查主存页表，并更新 TLB

### 3.6.4 地址变换过程

```
1. 根据虚拟页号查 TLB
2. TLB 命中 → 得到物理页号 → 拼接物理地址
3. TLB 未命中 → 查主存页表
   3.1 有效位 = 1 → 得到物理页号，更新 TLB
   3.2 有效位 = 0 → **缺页中断**（从辅存调页）
4. 物理地址 = 物理页号 × 页大小 + 页内偏移
```

### 3.6.5 缺页中断

> 1. 保护 CPU 现场
> 2. 查页表，确定缺页所在辅存位置
> 3. 调页（可能需换出旧页）
> 4. 更新页表
> 5. 恢复 CPU 现场，重新执行缺页指令

## 3.7 多级存储体系

### 3.7.1 存储层次金字塔

```
     寄存器        ← 最快、最贵、最小
      ↓
     Cache         ← 快、贵、小
      ↓
      主存         ← 中、中、中
      ↓
      辅存         ← 慢、便宜、大
```

| 层次 | 容量 | 速度 | 价格/位 |
|:-----|:-----|:-----|:-------|
| **寄存器** | 几 B~KB | 极快（ns）| 最高 |
| **Cache** | KB~MB | 快（ns）| 高 |
| **主存** | MB~GB | 中（100ns）| 中 |
| **辅存** | GB~TB | 慢（ms）| 低 |

### 3.7.2 平均访问时间（多级）

$$
T_a = T_1 + (1-h_1) T_2 + (1-h_1)(1-h_2) T_3 + \ldots
$$

> $T_a = h \cdot T_c + (1-h) \cdot T_m$

> - 命中率 $h$ 越大，平均访问时间**越接近** $T_c$
> - 顺序：先访问快的，再访问慢的

## 3.8 典型例题

### 例题 1：Cache 映射计算

**题目**：Cache 容量 2KB，主存容量 256KB，块大小 16B，直接映射。求主存地址 12345H 映射到 Cache 哪一行。

**解答**：

```
Cache 行数 = 2KB / 16B = 128 行
主存块号 = 12345H / 16 = 1234H
Cache 行号 = 1234H mod 128 = 1234H mod 80H = 34H = 52
```

**总结**：直接映射的行号计算使用**取模运算**。

### 例题 2：页表地址变换

**题目**：页面大小 4KB，虚拟地址 123456H，页表项 4B，页表起始 1000H，求物理地址（设页表项内容 567H）。

**解答**：

```
页大小 = 4KB = 2^12
页内偏移 = 123456H & 0FFFH = 456H
虚拟页号 = 123456H >> 12 = 123H

页表项地址 = 1000H + 123H × 4 = 1000H + 49CH = 149CH
物理页号 = 567H
物理地址 = 567H × 4096 + 456H = 567456H
```

**总结**：地址变换通过**页表**查找物理页号。

### 例题 3：平均访问时间计算

**题目**：Cache 访问时间 10ns，主存 100ns，命中率 95%，求平均访问时间。

**解答**：

$$
T_a = h \cdot T_c + (1-h) \cdot T_m = 0.95 	imes 10 + 0.05 	imes 100 = 14.5 	ext{ns}
$$

**总结**：提高命中率可显著降低平均访问时间。

## 4.1 指令格式

### 4.1.1 指令的基本格式

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│   操作码     │   地址码1    │   地址码2    │   地址码3    │

### 4.1.2 按地址数分类

### 4.1.3 操作码设计

| 方式 | 特点 |
|:-----|:-----|
| **固定长度操作码** | 译码简单、速度快，但浪费空间 |
| **扩展操作码** | 短码常用、长码不常用，节省空间 |

1. 短操作码**不能**是长操作码的**前缀**

3. 留出扩展空间

> 指令字 16 位，地址码 12 位，4 位扩展操作码
> - OP = 0000：零地址指令（16 条短操作码）
> - OP = 0001~1110：单地址指令
> - OP = 1111：双地址指令

## 4.4 RISC 与 CISC

### 4.4.1 CISC

> **CISC（Complex Instruction Set Computer）**：复杂指令系统计算机。
> - 指令数量多（200+），功能复杂
> - 寻址方式多样（10+ 种）
> - **指令长度不固定**
> - 强调**硬件**实现复杂功能
> - 代表：x86（Intel/AMD）

### 4.4.2 RISC

> **RISC（Reduced Instruction Set Computer）**：精简指令系统计算机。
> - 指令数量少（~50），功能简单
> - 寻址方式少（3~4 种）
> - **指令长度固定**
> - 强调**软件**实现复杂功能
> - 代表：ARM、MIPS、RISC-V

### 4.4.3 RISC vs CISC

| 对比项 | CISC | RISC |
|:-------|:-----|:-----|
| **指令数量** | 多（200+） | 少（~50） |
| **指令长度** | **不固定** | **固定** |
| **寻址方式** | 多（10+） | 少（3-4 种） |
| **CPI** | 高（2-15） | 低（1-2） |
| **指令周期数** | 多 | 少（多 1 周期执行） |
| **寄存器数** | 少 | 多（≥32） |
| **编译器** | 简单 | 复杂 |
| **硬件复杂度** | 高 | 低 |
| **流水线** | 难实现 | 易实现 |

### 4.4.4 RISC 关键技术

| 技术 | 作用 |
|:-----|:-----|
| **寄存器窗口** | 函数调用用不同寄存器组，减少栈操作 |
| **指令流水线** | 多指令重叠执行，提高效率 |
| **延迟加载** | Load 后插入无关指令，隐藏延迟 |
| **分支预测** | 预测分支方向，提前执行 |
| **硬布线控制** | 用组合逻辑实现，**速度快** |

> 设有 $n$ 条指令，$k$ 段流水线，每段时间 $t$：

$$
T = (k + n - 1) 	imes t
$$

## 4.5 指令的执行过程

### 4.5.1 指令周期

> **指令周期**：从取指令到执行完的时间，包含 4 个子周期：

$$
	ext{指令周期} = 	ext{取指周期} + 	ext{间址周期} + 	ext{执行周期} + 	ext{中断周期}
$$

### 4.5.2 四个子周期详解

**取指周期（IF）**：

```
1. PC → MAR → 地址总线
2. 发读命令 → 主存
3. 数据 → MDR → IR
4. PC + 1 → PC
```

**间址周期（IND）**（取有效地址）：

```
1. IR 的地址码 → MAR
2. 发读命令 → 主存
3. 数据 → MDR → IR 的地址码字段
```

**执行周期（EX）**：

```
根据 OP 执行：
- 算术运算：ALU 运算
- 数据传送：寄存器/存储器
- 程序控制：修改 PC
```

**中断周期（INT）**：

```
1. 关中断
2. 保存断点：PC → MDR → 主存
3. 保存状态：PSW → MDR → 主存
4. 中断服务程序地址 → PC
```

> - **取指周期**：每条指令都有
> - **间址周期**：当有间接寻址时
> - **执行周期**：每条指令都有
> - **中断周期**：仅当有中断请求时

---

## 4.6 指令流水线

### 4.6.1 流水线基本概念

> **指令流水线**：将指令执行过程分解为多个阶段，**多条指令并行执行**。

### 4.6.2 五段流水线标准划分

```
取指（IF）→ 译码（ID）→ 执行（EX）→ 访存（MEM）→ 写回（WB）
```

| 段 | 功能 |
|:---|:-----|
| **IF** | 取指令，PC → IR，PC+1 |
| **ID** | 译码 + 读寄存器 |
| **EX** | ALU 运算 / 计算地址 |
| **MEM** | 访存（Load/Store）|
| **WB** | 结果写回寄存器 |

### 4.6.3 流水线性能指标

**吞吐率（TP）**：

$$
TP = rac{	ext{完成指令数}}{	ext{总时间}}
$$

**加速比（S）**：

$$
S = rac{	ext{非流水线时间}}{	ext{流水线时间}}
$$

**效率（E）**：

$$
E = rac{	ext{任务实际占用的时空区}}{	ext{m 个段的总时空区}}
$$

### 4.6.4 流水线的三大冲突

| 冲突类型 | 原因 | 解决方案 |
|:---------|:-----|:---------|
| **结构冲突** | 硬件资源不足（多指令争用同一资源）| 增加资源（如双端口存储器）、暂停 |
| **数据冲突** | 指令间数据依赖 | 旁路（转发）、指令重排、暂停（气泡）|
| **控制冲突** | 分支指令导致 PC 不确定 | 分支预测、延迟分支、动态预测 |

**数据冲突的三种类型**：
- **RAW（Read After Write）**：真数据相关
- **WAR（Write After Read）**：反相关
- **WAW（Write After Write）**：输出相关

> - 把 EX 段结果**直接转发**到下一条指令的 EX 段
> - 无需等待 WB 段写回
> - 可解决**大部分 RAW 冲突**

## 4.7 典型例题

### 例题 1：指令格式设计

**题目**：设计二地址指令系统，操作码 4 位，地址码 12 位，指令字 32 位。求指令条数和寻址范围。

**解答**：

```
操作码：4 位 → 2^4 = 16 条指令
地址码：12 位 → 寻址范围 2^12 = 4KB
指令格式：
┌──────┬────────────┬────────────┐
│ OP   │    A1      │    A2      │
│ 4位  │    12位    │    12位    │
└──────┴────────────┴────────────┘
```

**总结**：指令格式设计在**操作种类**和**寻址范围**间平衡。

### 例题 2：RISC vs CISC

**题目**：比较 RISC 和 CISC 的特点。

**解答**：见 4.4.3 对比表。

**总结**：RISC 通过**精简指令**提高执行效率。

### 例题 3：流水线性能计算

**题目**：5 段流水线，每段 1ns，求 100 条指令的执行时间。

**解答**：

```
流水线时间 = (5 + 100 - 1) × 1ns = 104ns
非流水线时间 = 100 × 5 × 1ns = 500ns
加速比 = 500 / 104 ≈ 4.81
```

**总结**：流水线能显著提高指令执行效率。

# 第五章 CPU 结构与功能

## 5.1 CPU 的基本结构

### 5.1.1 CPU 内部组成

```
┌─────────────────────────────────────────────────┐
│                    CPU                          │
├─────────────────────────────────────────────────┤
│   ┌───────────────┐  ┌──────────────────┐       │
│   │   运算器 ALU   │  │   控制器 CU      │       │
│   │ ACC/MQ/X/PSW  │  │ IR/PC/MAR/MDR   │       │
│   └───────────────┘  └──────────────────┘       │
│              ┌──────────────────┐              │
│              │   寄存器组        │              │
│              └──────────────────┘              │
└─────────────────────────────────────────────────┘
```

### 5.1.2 运算器与控制器的组成

| 部件 | 所属 | 功能 |
|:-----|:-----|:-----|
| **ALU** | 运算器 | 算术/逻辑运算 |
| **ACC** | 运算器 | 累加器，存放结果 |
| **MQ** | 运算器 | 乘商寄存器 |
| **X** | 运算器 | 操作数寄存器 |
| **PSW** | 运算器 | 程序状态字（标志位） |
| **CU** | 控制器 | 控制单元，产生控制信号 |
| **IR** | 控制器 | 指令寄存器，存放当前指令 |
| **PC** | 控制器 | 程序计数器，下条指令地址 |
| **MAR** | 控制器 | 地址寄存器 |
| **MDR** | 控制器 | 数据寄存器 |

> 常见标志位：CF（进位）、OF（溢出）、ZF（零）、SF（符号）、PF（奇偶）

---

## 5.2 指令周期

### 5.2.1 指令周期的 4 个阶段

| 阶段 | 触发条件 | 主要工作 |
|:-----|:---------|:---------|
| **取指周期（IF）** | 每条指令都有 | PC→MAR→读主存→MDR→IR，PC+1 |
| **间址周期（IND）** | 间接寻址时 | 取操作数有效地址 |
| **执行周期（EX）** | 每条指令都有 | ALU 运算、读写数据、修改 PC |
| **中断周期（INT）** | 有中断请求 | 保存断点，转中断服务 |

### 5.2.2 不同指令的周期

| 指令类型 | 周期数 |
|:---------|:-------|
| **无操作数指令**（NOP） | 取指 + 执行 |
| **直接寻址指令** | 取指 + 执行 |
| **间接寻址指令** | 取指 + 间址 + 执行 |
| **算术逻辑指令** | 取指 + 执行 |
| **访存指令** | 取指 + 执行（含访存） |
| **控制指令** | 取指 + 执行（修改 PC） |
| **中断发生** | + 中断周期 |

> - **直接寻址** 只需要取指+执行，**不需要间址**
> - **间接寻址** 才有间址周期

---

## 5.3 时序系统

### 5.3.1 时序层次

| 层次 | 含义 |
|:-----|:-----|
| **指令周期** | 完整执行一条指令的时间 |
| **机器周期（CPU 周期）** | 完成一个基本操作（取指、访存）的时间 |
| **时钟周期（节拍）** | 最小时间单位 |

> 1 个指令周期 = 多个机器周期
> 1 个机器周期 = 多个时钟周期

### 5.3.2 三种控制方式

| 控制方式 | 特点 | 优点 | 缺点 |
|:---------|:-----|:-----|:-----|
| **同步控制** | 统一时钟控制 | 简单、可靠 | 速度慢 |
| **异步控制** | 握手信号协调 | 速度快 | 复杂 |
| **联合控制** | 同步+异步 | 折中 | 折中 |

---

## 5.4 微程序控制器

### 5.4.1 基本概念

> **微程序**：由**微指令**组成的程序，用于**解释执行**机器指令。

> **微指令**：一个时钟周期内完成的**微操作**集合，是微程序的基本单位。

> **微命令**：控制各部件完成**微操作**的最小控制信号。

### 5.4.2 微指令格式

```
┌──────────────────┬─────────────────────┐
│   微操作控制字段  │   顺序控制字段       │
│   （操作控制）    │   （下址）           │
└──────────────────┴─────────────────────┘
```

### 5.4.3 微指令的编码方式

| 编码方式 | 特点 | 优点 | 缺点 |
|:---------|:-----|:-----|:-----|
| **直接编码（水平）** | 1 位 1 微操作 | 速度快、并行性高 | 位数多 |
| **字段直接编码** | 分组编码 | 节省位数 | 速度稍慢 |
| **字段间接编码** | 字段含义由其他字段决定 | 更省位 | 复杂 |
| **垂直编码** | 类似机器指令 | 短 | 速度慢 |

> - 互斥的微操作放在**同一字段**（一个字段内不能同时出现）
> - 相容的微操作放在**不同字段**

### 5.4.4 微程序的执行过程

```
1. 根据指令操作码查微指令地址（微地址）
2. 读取微指令到微指令寄存器（CMAR/CMDR）
3. 执行微指令的微操作
4. 根据顺序控制字段确定下条微指令地址
5. 重复 1-4 直到机器指令执行完毕
```

> - **微地址**：微指令在控制存储器中的**地址**
> - **微地址形成方式**：计数器法（下址字段法）、断定法

---

## 5.5 硬布线控制器

### 5.5.1 硬布线控制

> **硬布线控制**：用**组合逻辑电路**直接产生控制信号。

**控制信号生成公式**：

$$
C = f(	ext{IR}, 	ext{PC}, 	ext{PSW}, 	ext{时序信号})
$$

### 5.5.2 微程序 vs 硬布线

| 对比项 | 微程序控制 | 硬布线控制 |
|:-------|:----------|:----------|
| **速度** | 慢（取微指令） | **快**（组合逻辑）|
| **灵活性** | **高**（改 ROM 即可） | 低（需改电路）|
| **复杂度** | 规则、规整 | 不规整、复杂 |
| **可维护性** | **好** | 差 |
| **适合** | CISC、复杂指令 | RISC、简单指令 |
| **代表** | x86 | RISC-V、MIPS |

> - 微程序控制器**常考**：微地址形成、微指令格式、编码方式
> - 硬布线控制器只需了解**优缺点**

## 5.6 流水线技术（深入）

### 5.6.1 流水线的分类

| 类型 | 描述 |
|:-----|:-----|
| **指令流水线** | 指令级并行（IF-ID-EX-MEM-WB）|
| **算术流水线** | 运算级并行（如浮点运算）|
| **处理器级流水线** | 又称宏流水线 |
| **超标量流水线** | 多条流水线并行（多发射）|
| **超流水线** | 细分流水段（提高主频）|
| **超长指令字（VLIW）** | 多操作打包成一条长指令 |

### 5.6.2 性能分析

**吞吐率（TP）**：

$$
TP = rac{n}{(k + n - 1) \cdot \Delta t}
$$

**加速比（S）**：

$$
S = rac{k \cdot n \cdot \Delta t}{(k + n - 1) \cdot \Delta t} = rac{kn}{k + n - 1}
$$

**效率（E）**：

$$
E = rac{n \cdot k \cdot \Delta t}{(k + n - 1) \cdot k \cdot \Delta t} = rac{n}{k + n - 1}
$$

> - 当 $n 	o \infty$ 时，$S 	o k$，$E 	o 1$
> - 当 $n = k$ 时，$S = k^2/(2k-1)$，$E = 1/2$

### 5.6.3 三大冲突及解决

| 冲突类型 | 原因 | 解决方案 |
|:---------|:-----|:---------|
| **结构冲突** | 硬件资源争用 | 增加资源（双端口存储）、分时使用 |
| **数据冲突** | 指令间数据依赖 | 旁路/转发、暂停、指令重排 |
| **控制冲突** | 分支跳转 | 分支预测、延迟槽、动态预测 |

**数据冲突三种类型**：
- **RAW（Read After Write）**：真相关（必须解决）
- **WAR（Write After Read）**：反相关
- **WAW（Write After Write）**：输出相关

> - **RAW** 是唯一必须解决的冲突
> - WAR/WAW 在**按序流水线**中不会发生

## 5.7 RISC CPU 设计

### 5.7.1 RISC 五大特点

1. **精简指令集**：指令少而简单
2. **定长指令**：简化译码
3. **Load/Store 架构**：仅 Load/Store 访存
4. **大量寄存器**：通常 32 个通用寄存器
5. **硬布线控制 + 流水线**

### 5.7.2 RISC CPU 结构

```
┌─────────────────────────────────────────────────────┐
│                    RISC CPU                          │
├─────────────────────────────────────────────────────┤
│  指令缓存（I-Cache）  │  数据缓存（D-Cache）          │
├─────────────────────────────────────────────────────┤
│                   寄存器组（32个通用寄存器）          │
├─────────────────────────────────────────────────────┤
│              算术逻辑单元（ALU）                      │
├─────────────────────────────────────────────────────┤
│              指令流水线（IF-ID-EX-MEM-WB）           │
└─────────────────────────────────────────────────────┘
```

> - 寄存器窗口（减少栈操作）
> - 指令流水线
> - 延迟加载
> - 分支预测
> - 硬布线控制

## 5.8 典型例题

### 例题 1：指令周期分析

**题目**：分析一条加法指令（含间址）的指令周期。

**解答**：

```
取指周期：
1. PC → MAR
2. 读主存 → MDR → IR
3. PC + 1 → PC

### 例题 3：微指令字段编码

**题目**：8 个微操作，字段直接编码分两组，每组 4 个，求微指令总位数。

# 第六章 总线

## 6.1 总线的基本概念

### 6.1.1 总线的定义

> **总线**：连接计算机各部件的**公共通信线路**，用于传输数据、地址和控制信号。

### 6.1.2 总线的特点

| 特点 | 含义 |
|:-----|:-----|
| **共享性** | 多个设备**共享**同一总线 |
| **分时性** | 同一时刻**只能有一个设备**占用总线发送 |
| **标准化** | 统一的接口标准 |

### 6.1.3 总线分类（按功能）

| 类型 | 传输内容 | 方向 |
|:-----|:--------|:-----|
| **数据总线** | 传输数据 | **双向** |
| **地址总线** | 传输地址 | **单向**（CPU→内存/I/O） |
| **控制总线** | 传输控制/时序信号 | 一般单向 |

### 6.1.4 总线分类（按层次）

| 层次 | 范围 | 速度 |
|:-----|:-----|:-----|
| **片内总线** | CPU 芯片内部 | 最快 |
| **系统总线** | 主板各部件（CPU/内存/I/O） | 中等 |
| **通信总线** | 计算机之间（USB/网线） | 较慢 |
| **I/O 总线** | CPU 与外设 | 慢 |

## 6.2 总线的性能指标

### 6.2.1 三大性能指标

| 指标 | 定义 |
|:-----|:-----|
| **总线宽度** | 数据总线的**位数**（8/16/32/64 位） |
| **总线频率** | 总线工作的**时钟频率**（Hz） |
| **总线带宽** | 单位时间内传输的**数据量**（B/s） |

### 6.2.2 总线带宽公式

$$
	ext{总线带宽} = rac{	ext{总线宽度} 	imes 	ext{总线频率}}{8} \quad (	ext{B/s})
$$

> 总线宽 32 位，频率 66 MHz：

$$
	ext{带宽} = rac{32 	imes 66 	imes 10^6}{8} = 264 	ext{ MB/s}
$$

### 6.2.3 其他指标

- **总线延迟**：从发起传输到完成的时间
- **总线负载**：可连接的设备数（受信号衰减限制）
- **总线复用**：多路分时共享总线（如地址/数据复用）

## 6.3 总线仲裁

### 6.3.1 仲裁的概念

> **总线仲裁**：解决**多个设备同时申请总线**的冲突，决定哪个设备获得使用权。

### 6.3.2 集中式仲裁

**① 链式查询方式（菊花链）**

```
仲裁器 → 设备1 → 设备2 → ... → 设备n
```

- 越靠近仲裁器，**优先级越高**
- 优点：简单、成本低
- 缺点：优先级**固定**、故障**敏感性**强

**② 计数器定时查询**

- 仲裁器从 0 开始**计数**，每个设备检查计数值是否等于自己编号
- 优点：优先级**可调整**
- 缺点：复杂

**③ 独立请求方式**

- 每个设备有**独立的请求线**和**应答线**
- 优点：**响应快**、优先级灵活
- 缺点：线数多、**成本高**

| 方式 | 优先级灵活性 | 速度 | 成本 |
|:-----|:------------|:-----|:-----|
| 链式查询 | 固定 | 慢 | 低 |
| 计数器定时 | 可变 | 中 | 中 |
| 独立请求 | 灵活 | **快** | **高** |

### 6.3.3 分布式仲裁

- **无**专门仲裁器，各设备**自行协商**
- 优点：灵活、可靠（无单点故障）
- 缺点：复杂

---

## 6.4 总线操作与时序

### 6.4.1 总线操作类型

| 类型 | 描述 |
|:-----|:-----|
| **读操作** | 从主存/设备读数据 |
| **写操作** | 向主存/设备写数据 |
| **读-修改-写** | 先读再写（不可分割）|
| **写后读** | 先写再读（不可分割）|
| **中断响应** | 处理中断请求 |
| **I/O 操作** | 与外设交换数据 |

### 6.4.2 同步时序

- 由**统一时钟**控制
- 每个操作占用**固定**的时钟周期
- 优点：简单、可靠
- 缺点：速度慢、适应性差

> ```
> T1: CPU 发地址 + 读命令
> T2: 地址稳定，等待响应
> T3: 主存数据送总线
> T4: CPU 读数据，撤销命令
> ```

### 6.4.3 异步时序

- 用**握手信号**协调
- 优点：速度快、适应性强
- 缺点：复杂

| 异步方式 | 特点 | 可靠性 |
|:---------|:-----|:-------|
| **不互锁** | 主设备不等待应答 | 最低 |
| **半互锁** | 主设备等应答后撤销请求 | 中 |
| **全互锁** | 双方等对方撤销 | 最高 |

> **全互锁 > 半互锁 > 不互锁**（可靠性）

### 6.4.4 半同步时序

- 结合同步（时钟）和异步（等待信号）的优点
- 引入 **WAIT 信号**，可插入等待周期

# 第七章 输入/输出（I/O）系统

> 本章覆盖 2026 考纲 I/O 部分：**I/O 系统基本概念、I/O 接口（功能与编址）、I/O 控制方式（程序查询/中断/DMA/通道）、I/O 设备分类与典型外设、中断系统（中断判优/响应/服务）**。本章与操作系统"设备管理"高度关联，但本课程侧重**硬件层**实现原理。

## 7.1 I/O 系统基本概念

> I/O 系统由三部分构成：
> 1. **I/O 设备（外部设备）**：键盘、显示器、磁盘、打印机等
> 2. **I/O 接口（I/O 控制器）**：连接 CPU 与 I/O 设备的"桥梁"
> 3. **CPU** + **总线**：协调 I/O 操作的指挥中枢

### 1. I/O 设备分类

| 分类维度 | 类型 | 示例 |
|:--------|:----|:----|
| 按使用特性 | 输入设备 / 输出设备 / 存储设备 / 网络设备 | 键盘、显示器、磁盘、网卡 |
| 按信息组织 | 字符设备 / 块设备 | 键盘（字符）、磁盘（块） |
| 按传输速率 | 低速设备 / 中速设备 / 高速设备 | 键盘（<1KB/s）、打印机（~10KB/s）、磁盘（~100MB/s） |
| 按共享属性 | 独占设备 / 共享设备 / 虚拟设备 | 打印机、磁盘、SPOOLing 模拟的打印机 |

### 2. I/O 控制方式的演进

> | 控制方式 | CPU 介入度 | 数据传输单位 | 优点 | 缺点 |
> |:----|:----|:----|:----|:----|
> | **程序查询方式**（Polling） | 全程参与 | 字（1~n 字节） | 接口简单、控制容易 | CPU 长期等待，利用率极低 |
> | **程序中断方式**（Interrupt） | 传输前后参与，传输时 CPU 可执行其他任务 | 字 | CPU 与 I/O 并行，效率较高 | 每字都要中断、频繁保存恢复现场 |
> | **DMA 方式** | 传输前后参与，传输时由 DMA 控制器接管总线 | 块（一组数据） | 适合高速块设备批量传输 | 控制逻辑较复杂 |
> | **通道方式**（Channel） | 极少参与（通道执行通道程序） | 块 / 块组 | 适合大量高速设备，CPU 几乎不干预 | 硬件复杂，成本高 |

## 7.2 I/O 接口（I/O 控制器）

### 2.1 I/O 接口的功能

> 1. **数据缓冲**：解决 CPU 与外设**速度不匹配**问题（DR 暂存数据）
> 2. **数据格式转换**：串/并转换、电平转换
> 3. **地址译码与端口选择**：识别 CPU 选中的 I/O 端口
> 4. **传送控制命令和状态信息**：
>    - 控制寄存器（CR）：接收 CPU 命令
>    - 状态寄存器（SR）：向 CPU 反馈设备状态
> 5. **中断请求与 DMA 请求**：向 CPU 发中断信号或向 DMA 控制器申请总线

### 2.2 I/O 接口的典型结构

```
        CPU ↔ 系统总线 ↔ I/O 接口 ↔ 外部设备
                            ↑
                ┌─────┬─────┼─────┬─────┐
                │ CR  │ DR  │ 状态│ 数据│
                │ 控制│ 数据│ 寄存│ 缓冲│
                │ 寄存│ 缓冲│ 器  │ 寄存│
                └─────┴─────┴─────┴─────┘
```

> - **数据缓冲寄存器 DR** 与 **数据端口** 是**同一个东西**（不同书不同叫法）。
> - I/O 接口中可能含**控制寄存器、状态寄存器、数据寄存器**等多组寄存器，每组寄存器占一个 I/O 端口地址。

### 2.3 I/O 端口编址

> | 方式 | 特点 | 优缺点 |
> |:----|:----|:----|
> | **独立编址（I/O 隔离）** | I/O 端口有**独立地址空间**，使用专门的 `IN` / `OUT` 指令 | 优点：I/O 与内存访问互不干扰；缺点：需专门指令，I/O 指令少 |
> | **统一编址（内存映射 I/O）** | I/O 端口与内存**共用地址空间**，使用访存指令 | 优点：可用丰富指令集；缺点：占用内存地址空间，且程序中不易区分访存/访 I/O |

> - 8086：**独立编址**（10 位端口地址，64K 端口）
> - 51 单片机 / ARM：**统一编址**
> - 现代 PC 多数采用**混合**：部分 I/O 独立编址，部分设备采用内存映射 I/O（MMIO）

### 2.4 I/O 接口的类型

> - **程序控制接口**：适用于程序查询/程序中断方式（如 8255 并行接口、8251 串行接口）
> - **DMA 接口**：适用于 DMA 方式（如 8237 DMA 控制器）
> - **通道接口**：适用于通道方式（高端服务器专用）

## 7.3 程序查询方式

> 1. CPU 向 I/O 设备发出**读/写命令**，将 I/O 接口的"启动位"置 1
> 2. CPU **循环读取**状态寄存器 SR，**轮询**设备是否就绪（Ready = 1）
> 3. 设备就绪后，CPU 从数据寄存器 DR **读写数据**
> 4. 完成后，CPU 继续执行其他任务

```c
// 程序查询方式：读一个数据字
while (STATUS_REG & 0x01 == 0)        // 轮询：等待设备就绪
    ;
DATA = DATA_REG;                       // 设备就绪，读取数据
```

> - **优点**：接口电路简单，控制逻辑容易
> - **缺点**：CPU 与 I/O 设备**完全串行工作**，CPU 利用率极低
> - **适用**：极低速设备（如 LED、开关），或调试场合

## 7.4 程序中断方式

### 4.1 中断的基本概念

> | 类别 | 来源 | 是否可屏蔽 | 典型例子 |
> |:----|:----|:----|:----|
> | **内中断（异常 / Exception）** | CPU 内部 | 多数不可屏蔽 | 缺页、除 0、溢出、断点 |
> | **外中断（中断 / Interrupt）** | CPU 外部 | 可分可屏蔽 | I/O 设备、定时器、电源故障 |
> | **可屏蔽中断 INTR** | 外设 | **可屏蔽**（IF=0 屏蔽） | 一般 I/O 设备 |
> | **不可屏蔽中断 NMI** | 关键外设 | **不可屏蔽** | 电源故障、内存奇偶校验错 |

### 4.2 中断处理流程（5 步）

> 1. **中断请求（IRQ）**：外设准备好后，通过中断控制器向 CPU 发送中断请求
> 2. **中断判优**：当多个中断同时请求时，**按优先级**排队（链式 / 集中式 / 分布式）
> 3. **中断响应（CPU 自动完成）**：
>    - 关中断（IF = 0）
>    - 保存断点（PC 压栈）和程序状态（PSW 压栈）
>    - 取出**中断服务程序入口地址**（中断向量），送至 PC
> 4. **中断服务**（中断服务程序）：
>    - 保护现场（保存通用寄存器的值）
>    - 中断处理（真正的 I/O 操作）
>    - 恢复现场（还原寄存器）
>    - 开中断（IF = 1）
>    - 中断返回（`IRET` 指令：恢复 PC 和 PSW）
> 5. **返回原程序**：CPU 继续执行被中断的指令

### 4.3 中断判优（仲裁）

> 1. **硬件判优（速度最快）**：
>    - **链式查询**：优先级由硬件连线固定（位置靠前优先级高）
>    - **独立请求**：每个设备有独立的请求线和响应线
>    - **集中式轮询（计数器）**：CPU 通过计数器查询
> 2. **软件判优**：用程序轮询设备优先级（灵活但慢）
> 3. **分级仲裁**：与操作系统协同，CPU 决定屏蔽哪些中断

### 4.4 多重中断（中断嵌套）

> - 当 CPU 正在处理某一级中断时，若有**更高级**中断请求 → 暂停当前中断服务，转去处理更高级中断（**中断嵌套**）
> - 同级或低级中断请求 → 等当前中断处理完再响应
> - 实现嵌套的关键：**中断允许触发器** + **中断屏蔽寄存器**

> 计算机有 5 级中断 L1 > L2 > L3 > L4 > L5，优先级递减。正在执行 L3 中断服务程序时，同时到来 L1、L2、L4、L5 请求。
> **处理顺序**：L1 → L2 → L3（继续）→ L4 → L5
> 即先按优先级处理完所有**更高级**的中断，再回到原中断，最后处理低级中断。

### 4.5 中断向量与中断向量表

> - **中断向量**：中断服务程序的**入口地址**（CS:IP）
> - **中断向量表**：存放所有中断向量的内存区域（通常在内存最低端 0~1KB 区间）
> - 中断响应时，CPU 通过**中断类型号** × 4 找到向量表项，取出 CS 和 IP，跳转执行

## 7.5 DMA 方式

### 5.1 DMA 的基本概念

> **DMA 控制器**在 I/O 设备与主存之间建立**直接数据通路**，**无需 CPU 介入**（传输时）。整个传输过程由 DMA 控制器管理，CPU 仅在传输前/后参与。

> | 比较维度 | 中断方式 | DMA 方式 |
> |:----|:----|:----|
> | 数据传送单位 | 字（1~n 字节） | **数据块** |
> | CPU 介入 | 每个字都中断 CPU | 仅**开始 + 结束**介入 |
> | 设备与主存路径 | 设备 → CPU 寄存器 → 主存 | 设备 → **DMA 控制器** → 主存 |
> | 适用场景 | 中低速字符设备 | **高速块设备**（磁盘、网卡） |

### 5.2 DMA 控制器的组成

```c
typedef struct DMAC {
    int MAR;        // 主存地址寄存器：主存缓冲区起始地址
    int DC;         // 数据计数器：本次传输的字数（每传送 1 个字 DC 自动 -1）
    int DAR;        // 设备地址寄存器（I/O 设备）
    int CR;         // 控制/状态寄存器：传送方向、启动、结束等
} DMAC;
```

> - **主机与 DMA 的联络**：HRQ（Hold Request）、HLDA（Hold Acknowledge）
> - **DMA 与设备的联络**：DREQ（DMA Request）、DACK（DMA Acknowledge）

### 5.3 DMA 传送过程

> 1. **预处理**（CPU 执行）：设置 MAR、DC、CR，启动 DMA
> 2. **数据传送**（DMA 控制器执行）：
>    - 设备就绪 → 发 DREQ → DMA 发 HRQ → CPU 响应（HLDA）→ 接管总线控制权
>    - 每次从设备读 1 字 → 写入主存 MAR 指定单元 → MAR++、DC--
>    - 重复直到 DC = 0 → DMA 释放总线
> 3. **后处理**（CPU 执行）：DMA 发**中断请求**，CPU 执行中断服务程序进行善后（校验、决定下次操作）

### 5.4 DMA 传送方式

> 1. **停止 CPU 访问主存**：DMA 占用总线时，CPU 释放总线控制权（效率较低）
> 2. **周期挪用（周期窃取）**：DMA 偷 1 个总线周期完成 1 个字的传输，其余时间 CPU 继续工作（**最常用**）
> 3. **DMA 与 CPU 交替访问**：将 1 个 CPU 周期分成 2 个时间片，DMA 与 CPU 交替使用总线（硬件复杂）

> - DMA 每次挪用 1 个存取周期，挪用后立即释放总线
> - 每挪用一个周期，CPU 就**暂停** 1 个存取周期
> - 若设备传输速率为 $V$，主存周期为 $T$，则挪用 1 个字的 CPU 暂停时间为 $T$

## 7.6 通道控制方式

> **通道是一种专门负责 I/O 的处理器（IOP）**，有自己的**通道程序**（由通道指令组成），可以执行更复杂的 I/O 操作。
> - 通道的**类型**：
>   - **字节多路通道**：适合大量低速字符设备（如多个终端）
>   - **选择通道**：适合单个高速设备（如单个磁盘）
>   - **数组多路通道**：适合多个高速块设备（如多个磁盘）

> | | DMA | 通道 |
> |:----|:----|:----|
> | 控制对象 | 一个 I/O 设备 | 多个 I/O 设备 |
> | 指令系统 | 无 | 有自己的**通道指令**（通道程序） |
> | 智能性 | 低（需 CPU 详细设置） | 高（独立处理 I/O） |
> | 适用 | 单一高速块传输 | 大规模 I/O 集中管理 |

## 7.7 I/O 设备与典型外设

### 7.1 常见 I/O 设备特性

> | 设备 | 类型 | 关键性能指标 |
> |:----|:----|:----|
> | **键盘** | 字符输入 | 响应速度、手感 |
> | **鼠标** | 指点输入 | DPI、刷新率 |
> | **显示器** | 输出 | 分辨率、刷新率、色彩深度 |
> | **打印机** | 输出 | 速度、分辨率、彩色能力 |
> | **磁盘（硬盘）** | 块存储 | 容量、平均寻道时间、转速、传输率 |
> | **SSD（固态硬盘）** | 块存储 | IOPS、读写速度、寿命（DWPD） |
> | **网卡** | 网络 I/O | 带宽（Mbps / Gbps）、延迟 |

### 7.2 显示器参数

> - **分辨率**：水平像素 × 垂直像素（如 1920×1080）
> - **刷新率**：每秒刷新的帧数（Hz）
> - **色彩深度**：每个像素的位数（8 bit、24 bit、32 bit）
> - **显示存储器（VRAM）容量 ≥ 分辨率 × 色彩深度 / 8**
>   例：1920×1080×24bit = 1920×1080×3 ≈ 6.2 MB

## 7.8 典型例题

> 某系统使用程序查询方式读取 1000 个数据字，每字查询需 100 个时钟周期，传输 1 个字需 1 个时钟周期。改用中断方式，每字中断响应 + 服务需 200 个时钟周期。
> **解**：
> - 程序查询总时间：1000 × (100 + 1) = 101,000 周期
> - 中断方式总时间：1000 × (200 + 1) = 201,000 周期（**反而更慢**！）
> - **结论**：本例中查询方式优于中断方式，因为数据传输本身很快（1 周期），中断开销太大。
> - **实际意义**：当数据块小、设备慢 → 中断更有优势；当数据块大、传输快 → DMA 更优。

> 某主存存取周期 100 ns，磁盘传输率 2 MB/s，使用 DMA 方式传输 1 KB 数据。
> **解**：
> - 传输 1 KB 所需时间 = 1024 / (2 × 1024 × 1024) ≈ 488 μs
> - CPU 被挪用的周期数 = 1024 个周期
> - CPU 被挪用时间 = 1024 × 100 ns = 102.4 μs
> - 挪用比例 = 102.4 / 488 ≈ **21%**

> 一台显示器分辨率 1024×768，色彩深度 16 bit（每像素 2 字节），刷新率 85 Hz。
> **解**：
> - VRAM 容量 = 1024 × 768 × 2 = 1,572,864 字节 ≈ **1.5 MB**
> - 显存带宽需求 = 1.5 MB × 85 Hz ≈ **127.5 MB/s**

> 某机 4 级中断 A > B > C > D。CPU 执行主程序时 A、B、D 同时请求，CPU 在处理 B 时 C 请求到达。处理顺序？
> **解**：
> - 主程序：先响应 A（最优先）→ 然后响应 B（次高）→ B 处理中 C 到达，**B 比 C 优先**，B 完成后才响应 C → 最后响应 D
> - 顺序：A → B → C → D

## 7.9 知识回顾

> - I/O 系统 = **I/O 设备 + I/O 接口 + CPU**。
> - 四种 I/O 控制方式：**程序查询 → 程序中断 → DMA → 通道**，CPU 介入度递减，效率递增。
> - **中断处理 5 步**：请求 → 判优 → 响应 → 服务 → 返回。**中断嵌套**保证高优先级优先响应。
> - **DMA 三大组成**：MAR（主存地址）+ DC（数据计数）+ CR（控制状态）；**周期挪用**最常用。
> - 通道 = 专门 I/O 处理器，可独立执行通道程序，控制多台设备。
> - I/O 端口编址：**独立编址**（专门 IN/OUT 指令）vs **统一编址**（访存指令）。
