## 1. 总线的基本概念

### 1.1 总线的定义

<span style="background-color:#ccc1d9">**总线**</span>：连接计算机各部件的公共通信线路。

### 1.2 总线的特点

| 特点 | 描述 |
|------|------|
| **共享性** | 多个设备共享总线 |
| **分时性** | 同一时刻只能有一个设备传输 |
| **标准化** | 统一的接口标准 |

### 1.3 总线的分类

#### 1.3.1 按功能分类

| 类型 | 描述 |
|------|------|
| **数据总线** | 传输数据，双向 |
| **地址总线** | 传输地址，单向 |
| **控制总线** | 传输控制信号 |

#### 1.3.2 按层次分类

| 类型 | 描述 |
|------|------|
| **片内总线** | CPU内部 |
| **系统总线** | 主板上各部件 |
| **通信总线** | 计算机之间 |

---

## 2. 总线的分类

### 2.1 片内总线

**定义**：CPU芯片内部的总线。

**特点**：速度快，距离短。

### 2.2 系统总线

**定义**：连接计算机主要部件的总线。

**分类**：

| 类型 | 描述 |
|------|------|
| **ISA总线** | 早期总线，16位 |
| **PCI总线** | 32位/64位，高速 |
| **PCI Express** | 串行，高速 |
| **AGP总线** | 图形专用 |

### 2.3 通信总线

**定义**：计算机之间通信的总线。

**分类**：

| 类型 | 描述 |
|------|------|
| **并行总线** | 多线并行传输 |
| **串行总线** | 单线串行传输 |

---

## 3. 总线的性能指标

### 3.1 总线宽度

**定义**：数据总线的位数。

**常见宽度**：8位、16位、32位、64位。

### 3.2 总线频率

**定义**：总线工作的时钟频率。

**常见频率**：66MHz、133MHz、266MHz、533MHz。

### 3.3 总线带宽

**定义**：单位时间内传输的数据量。

$$
\text{总线带宽} = \frac{\text{总线宽度} \times \text{总线频率}}{8}
$$

### 3.4 总线延迟

**定义**：从发起传输到完成的时间。

## 4. 总线仲裁

### 4.1 总线仲裁的概念

<span style="background-color:#ccc1d9">**总线仲裁**</span>：解决多个设备同时申请总线的冲突。

### 4.2 集中式仲裁

**特点**：由专门的仲裁机构统一管理。

#### 4.2.1 链式查询方式

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│  仲裁器  │───>│ 设备1    │───>│ 设备2    │───> ...
└──────────┘    └──────────┘    └──────────┘
```

**特点**：优先级由位置决定，越靠近仲裁器优先级越高。

**优点**：简单，成本低。

**缺点**：优先级固定，可靠性差。

#### 4.2.2 计数器定时查询方式

```
仲裁器计数，每个设备检查计数值是否等于自己的编号。
```

**特点**：优先级可动态调整。

**优点**：灵活。

**缺点**：复杂，成本高。

#### 4.2.3 独立请求方式

```
每个设备有独立的请求和应答线。
```

**特点**：优先级可灵活设置。

**优点**：响应快，优先级灵活。

**缺点**：复杂，成本高。

### 4.3 分布式仲裁

**特点**：无专门仲裁器，各设备自行协商。

**优点**：灵活，可靠。

**缺点**：复杂。

## 5. 总线操作和时序

### 5.1 总线操作的类型

| 类型 | 描述 |
|------|------|
| **读操作** | 从主存/设备读取数据 |
| **写操作** | 向主存/设备写入数据 |
| **读-修改-写操作** | 先读再写 |
| **中断响应** | 响应中断请求 |

### 5.2 同步时序

**特点**：由统一时钟控制。

**优点**：简单，可靠。

**缺点**：速度慢，适应性差。

```
时钟：  ──┬──┬──┬──┬──┬──
         │  │  │  │  │  │
地址：  ──┴──┴──┴──┴──┴──
         ↑
      地址稳定

数据：  ───────────┴──┴──
                     ↑
                  数据稳定
```

### 5.3 异步时序

**特点**：用握手信号协调。

**优点**：速度快，适应性强。

**缺点**：复杂。

#### 5.3.1 不互锁方式

```
主设备发请求 → 从设备发应答 → 完成
```

**特点**：应答后主设备立即撤销请求。

#### 5.3.2 半互锁方式

```
主设备发请求 → 从设备发应答 → 主设备撤销请求 → 从设备撤销应答
```

**特点**：主设备等待应答后撤销请求。

#### 5.3.3 全互锁方式

```
主设备发请求 → 从设备发应答 → 主设备撤销请求 → 从设备撤销应答
```

**特点**：双方都等待对方撤销信号。

### 5.4 半同步时序

**特点**：结合同步和异步。

## 6. 总线标准

### 6.1 ISA总线

**特点**：

- 16位数据总线
- 8MHz频率
- 16MB寻址空间
- 已淘汰

### 6.2 PCI总线

**特点**：

- 32位/64位数据总线
- 33MHz/66MHz频率
- 即插即用
- 支持多设备

### 6.3 PCI Express

**特点**：

- 串行传输
- 点对点连接
- 高速（PCIe 4.0: 16GT/s）
- 支持热插拔

### 6.4 USB

**特点**：

- 串行传输
- 即插即用
- 支持热插拔
- 多设备连接

## 7. 总线的设计考虑

### 7.1 总线宽度的选择

**原则**：平衡性能和成本。

**影响**：

- 宽度越大，带宽越高
- 宽度越大，成本越高

### 7.2 总线频率的选择

**原则**：考虑设备速度。

**影响**：

- 频率越高，带宽越高
- 频率越高，干扰越大

### 7.3 总线负载

**问题**：设备过多导致信号衰减。

**解决**：

- 使用缓冲器
- 限制设备数量

## 8. 典型例题

### 例题 1：总线带宽计算

**题目**：某总线宽度为32位，频率为66MHz，求带宽。

**解答**：

```
总线带宽 = (32 × 66 × 10^6) / 8
         = (2.112 × 10^9) / 8
         = 264 × 10^6 B/s
         = 264 MB/s
```

**总结**：总线带宽与宽度和频率成正比。

**来源**：408历年真题，考查方式：计算题

---

### 例题 2：总线仲裁方式

**题目**：比较链式查询、计数器定时查询和独立请求三种仲裁方式。

**解答**：

```
链式查询：
- 优点：简单，成本低
- 缺点：优先级固定，可靠性差

计数器定时查询：
- 优点：优先级可调整
- 缺点：复杂，成本较高

独立请求：
- 优点：响应快，优先级灵活
- 缺点：复杂，成本高
```

**总结**：不同仲裁方式各有优劣，需根据需求选择。

**来源**：408历年真题，考查方式：简答题

---

### 例题 3：同步时序分析

**题目**：分析同步总线的读操作时序。

**解答**：

```
T1：CPU发地址和读命令
T2：地址稳定，等待主存响应
T3：主存将数据送上数据总线
T4：CPU读取数据，撤销命令

时钟周期：4个

特点：
- 每个操作占用固定周期
- 简单可靠
- 但可能浪费时间
```

**总结**：同步时序的操作时间固定。

**来源**：408历年真题，考查方式：综合题
