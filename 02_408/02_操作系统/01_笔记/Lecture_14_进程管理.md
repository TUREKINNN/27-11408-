## 1. 进程同步与互斥

### 1.1 进程同步的概念

<span style="background-color:#ccc1d9">**进程同步**</span>：协调多个进程的执行顺序，使它们按预期的方式运行。

### 1.2 进程互斥的概念

<span style="background-color:#ccc1d9">**进程互斥**</span>：多个进程不能同时访问共享资源。

### 1.3 临界区

<span style="background-color:#ccc1d9">**临界区**</span>：进程中访问共享资源的代码段。

### 1.4 临界区的访问规则

1. **空闲让进**：临界区空闲时允许进程进入
2. **忙则等待**：临界区忙时进程等待
3. **有限等待**：等待时间有限
4. **让权等待**：等待时释放CPU

## 2. 信号量机制

### 2.1 信号量的定义

<span style="background-color:#ccc1d9">**信号量**</span>：一个整数变量，用于实现进程同步和互斥。

### 2.2 P、V操作

**P操作（申请）**：

```
P(S):
    S = S - 1
    if S < 0:
        阻塞进程，加入等待队列
```

**V操作（释放）**：

```
V(S):
    S = S + 1
    if S <= 0:
        唤醒等待队列中的一个进程
```

### 2.3 信号量的分类

| 类型 | 描述 |
|------|------|
| **互斥信号量** | 初值为1，实现互斥 |
| **同步信号量** | 初值为0或n，实现同步 |
| **资源信号量** | 初值为资源数量 |

### 2.4 信号量的应用

#### 2.4.1 实现互斥

```
mutex = 1

进程A:          进程B:
P(mutex)        P(mutex)
临界区          临界区
V(mutex)        V(mutex)
```

#### 2.4.2 实现同步

```
S = 0

进程A:          进程B:
执行操作A       P(S)
V(S)            执行操作B（必须在A之后）
```

## 3. 经典同步问题

### 3.1 生产者-消费者问题

**问题描述**：生产者生产产品放入缓冲区，消费者从缓冲区取出产品。

**信号量设置**：

```
empty = n    // 空缓冲区数量
full = 0     // 满缓冲区数量
mutex = 1    // 互斥访问缓冲区

生产者:
    P(empty)
    P(mutex)
    放入产品
    V(mutex)
    V(full)

消费者:
    P(full)
    P(mutex)
    取出产品
    V(mutex)
    V(empty)
```

### 3.2 读者-写者问题

**问题描述**：多个读者可以同时读，但写者必须独占。

**信号量设置**：

```
rw_mutex = 1    // 写者互斥
count_mutex = 1 // 计数器互斥
count = 0       // 读者数量

读者:
    P(count_mutex)
    count = count + 1
    if count == 1:
        P(rw_mutex)
    V(count_mutex)
    读操作
    P(count_mutex)
    count = count - 1
    if count == 0:
        V(rw_mutex)
    V(count_mutex)

写者:
    P(rw_mutex)
    写操作
    V(rw_mutex)
```

### 3.3 哲学家就餐问题

**问题描述**：5个哲学家围坐一圈，需要左右筷子吃饭。

**信号量设置**：

```
chopstick[5] = {1, 1, 1, 1, 1}

哲学家i:
    P(chopstick[i])
    P(chopstick[(i+1)%5])
    吃饭
    V(chopstick[i])
    V(chopstick[(i+1)%5])
```

**死锁问题**：所有哲学家同时拿起左边筷子。

**解决方案**：

1. 最多4个哲学家同时拿筷子
2. 奇数哲学家先拿左边，偶数先拿右边

## 4. 管程

### 4.1 管程的定义

<span style="background-color:#ccc1d9">**管程**</span>：一种高级同步机制，将共享资源和操作封装在一起。

### 4.2 管程的组成

| 组成 | 描述 |
|------|------|
| **共享变量** | 共享资源的数据 |
| **操作过程** | 对共享资源的操作 |
| **条件变量** | 实现进程等待和唤醒 |

### 4.3 条件变量

```
wait(): 阻塞进程，释放管程
signal(): 唤醒等待的进程
```

### 4.4 管程的特点

- 同一时刻只有一个进程在管程内执行
- 条件变量用于同步

## 5. 死锁

### 5.1 死锁的定义

<span style="background-color:#ccc1d9">**死锁**</span>：多个进程互相等待对方释放资源，导致所有进程都无法继续执行。

### 5.2 死锁的必要条件

<span style="background-color:#8db3e2">**死锁必要条件**：四个条件同时满足时发生死锁。</span>

1. **互斥条件**：资源只能被一个进程占用
2. **请求与保持条件**：进程持有资源并请求新资源
3. **不剥夺条件**：资源不能被强制剥夺
4. **循环等待条件**：进程形成循环等待链

### 5.3 死锁的处理策略

| 策略 | 描述 |
|------|------|
| **预防死锁** | 破坏必要条件 |
| **避免死锁** | 动态检测，避免进入不安全状态 |
| **检测死锁** | 定期检测，发现后解除 |
| **解除死锁** | 终止进程或剥夺资源 |

### 5.4 死锁预防

**破坏互斥条件**：允许资源共享。

**破坏请求与保持条件**：一次性申请所有资源。

**破坏不剥夺条件**：允许剥夺资源。

**破坏循环等待条件**：资源编号，按顺序申请。

### 5.5 死锁避免

<span style="background-color:#ccc1d9">**银行家算法**</span>：判断分配资源后是否安全。

**安全状态**：存在一个进程执行顺序，所有进程都能完成。

**不安全状态**：不存在这样的执行顺序。

### 5.6 银行家算法

**步骤**：

1. 初始化可用资源向量
2. 找到一个进程，其需求向量 <= 可用向量
3. 分配资源，执行完成后释放资源
4. 重复直到所有进程完成

### 5.7 死锁检测

**资源分配图**：

- 进程节点
- 资源节点
- 请求边：进程→资源
- 分配边：资源→进程

**死锁检测**：检测图中是否存在环路。

### 5.8 死锁解除

| 方法 | 描述 |
|------|------|
| **终止进程** | 终止一个或多个进程 |
| **剥夺资源** | 剥夺进程的资源 |
| **回滚** | 回滚到之前的状态 |

## 6. 进程通信

### 6.1 进程通信的定义

<span style="background-color:#ccc1d9">**进程通信**</span>：进程之间交换数据。

### 6.2 进程通信的方式

| 方式 | 描述 |
|------|------|
| **共享内存** | 共享一块内存区域 |
| **消息传递** | 通过消息队列传递数据 |
| **管道** | 半双工通信 |
| **套接字** | 网络通信 |

### 6.3 消息传递

**发送消息**：

```
send(receiver, message)
```

**接收消息**：

```
receive(sender, message)
```

### 6.4 管道

**特点**：

- 半双工
- 先进先出
- 只能在父子进程间使用

## 7. 典型例题

### 例题 1：生产者-消费者问题

**题目**：使用信号量实现生产者-消费者问题，缓冲区大小为n。

**解答**：

```
semaphore empty = n;   // 空缓冲区数量
semaphore full = 0;    // 满缓冲区数量
semaphore mutex = 1;   // 互斥锁

producer() {
    while (true) {
        produce_item();
        P(empty);
        P(mutex);
        insert_item();
        V(mutex);
        V(full);
    }
}

consumer() {
    while (true) {
        P(full);
        P(mutex);
        remove_item();
        V(mutex);
        V(empty);
        consume_item();
    }
}
```

**总结**：需要三个信号量，分别控制空缓冲区、满缓冲区和互斥。

**来源**：408历年真题，考查方式：综合题

### 例题 2：银行家算法

**题目**：系统有3类资源A(10)、B(5)、C(7)，当前分配和需求如下：

```
   Allocation    Max
   A B C         A B C
P0: 0 1 0       7 5 3
P1: 2 0 0       3 2 2
P2: 3 0 2       9 0 2
P3: 2 1 1       2 2 2
P4: 0 0 2       4 3 3

Available: 3 3 2
```

判断系统是否安全。

**解答**：

```
检查各进程需求：
P0: 7-0=7, 5-1=4, 3-0=3 → 需求(7,4,3) > Available(3,3,2)
P1: 3-2=1, 2-0=2, 2-0=2 → 需求(1,2,2) <= Available(3,3,2) ✓

分配给P1，执行完释放：
Available = (3,3,2) + (2,0,0) = (5,3,2)

P3: 2-2=0, 2-1=1, 2-1=1 → 需求(0,1,1) <= Available(5,3,2) ✓

分配给P3，执行完释放：
Available = (5,3,2) + (2,1,1) = (7,4,3)

P4: 4-0=4, 3-0=3, 3-2=1 → 需求(4,3,1) <= Available(7,4,3) ✓

分配给P4，执行完释放：
Available = (7,4,3) + (0,0,2) = (7,4,5)

P2: 9-3=6, 0-0=0, 2-2=0 → 需求(6,0,0) <= Available(7,4,5) ✓

分配给P2，执行完释放：
Available = (7,4,5) + (3,0,2) = (10,4,7)

P0: 7-0=7, 5-1=4, 3-0=3 → 需求(7,4,3) <= Available(10,4,7) ✓

安全序列：P1 → P3 → P4 → P2 → P0

系统安全。
```

**总结**：银行家算法通过检查安全序列判断系统是否安全。

**来源**：408历年真题，考查方式：综合题

### 例题 3：死锁必要条件

**题目**：说明死锁的四个必要条件。

**解答**：

```
1. 互斥条件：资源只能被一个进程占用
2. 请求与保持条件：进程持有资源并请求新资源
3. 不剥夺条件：资源不能被强制剥夺
4. 循环等待条件：进程形成循环等待链

四个条件同时满足时发生死锁，破坏任意一个条件即可预防死锁。
```

**总结**：死锁的四个必要条件是理解死锁的关键。

**来源**：408历年真题，考查方式：简答题
