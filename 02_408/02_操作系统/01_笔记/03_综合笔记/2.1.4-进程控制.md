## 一、什么是进程控制？

进程控制的主要功能是对系统中的所有进程实施有效的管理：
- **创建**新进程
- **撤销**已有进程
- 实现进程**状态转换**

> 简化理解：进程控制就是要实现**进程状态转换**

---

## 二、用"原语"实现进程控制

### 1. 原语的概念

```
用户 ─────── 应用程序（软件）
               │
操作系统 ──── 进程管理、存储器管理、设备管理等功能
               │
内核 ──────── 原语（设备驱动、CPU切换等）
               │
裸机 ──────── 纯硬件
```

**原语**是一种特殊的程序，它的执行具有**原子性**。也就是说，这段程序的运行必须**一气呵成，不可中断**。

### 2. 为什么需要原子性？

如果不能一气呵成，就可能导致操作系统中某些关键数据结构信息**不统一**：

> 例：假设进程2等待的事件发生，OS至少需要做两件事：
> ① 将PCB2的 `state` 设为 1（就绪态）
> ② 将PCB2从阻塞队列放到就绪队列
>
> 如果完成了①后收到中断信号，PCB2的 `state=1`，但它却被放在阻塞队列里 → **数据不一致**

### 3. 如何实现原语的"原子性"？

使用 **关中断指令** 和 **开中断指令** 这两个**特权指令**实现：

```
……指令1 指令2 │ 关中断指令 │ 指令a 指令b │ 开中断指令 │ 指令3 ……
               │←── 原语执行期间，不检查中断信号 ──→│
```

- CPU执行关中断指令后，**不再例行检查中断信号**
- 直到执行开中断指令后才恢复检查
- 关中断、开中断之间的指令序列**不可被中断** → 实现了"原子性"

> ⚠️ 这两个特权指令**不允许用户程序使用**，否则用户可以一直关中断，系统将无法正常工作。

## 三、进程控制相关的原语

### 1. 创建原语（创建态 → 就绪态）

操作系统创建一个进程时使用的原语：

```
创建原语的操作：
1. 申请空白PCB
2. 为新进程分配所需资源
3. 初始化PCB
4. 将PCB插入就绪队列
```

引起进程创建的事件：
- 用户登录
- 作业调度
- 提供服务
- 应用请求

### 2. 撤销原语（就绪态/阻塞态/运行态 → 终止态 → 无）

```
撤销原语的操作：
1. 从PCB集合中找到要终止的进程的PCB
2. 若进程正在运行，立即剥夺CPU，将CPU分配给其他进程
3. 终止其所有子进程（进程间的关系是树形结构）
4. 将该进程拥有的所有资源归还给父进程或操作系统
5. 删除PCB
```

引起进程终止的事件：
- 进程自己请求终止（`exit` 系统调用）
- 异常结束（整数除以0、非法使用特权指令，被OS强行杀掉）
- 外界干预（Ctrl+Alt+Delete，用户选择杀掉进程）

### 3. 阻塞原语（运行态 → 阻塞态）

```
阻塞原语的操作：
1. 找到要阻塞的进程的PCB
2. 保护进程运行现场，将PCB state设为"阻塞态"
3. 将PCB插入相应事件的阻塞队列
```

### 4. 唤醒原语（阻塞态 → 就绪态）

```
唤醒原语的操作：
1. 在事件的阻塞队列中找到该进程的PCB
2. 将PCB从阻塞队列移出，state设为"就绪态"
3. 将PCB插入就绪队列
```

> ⚠️ **阻塞原语和唤醒原语必须成对使用！** 因何事阻塞，就应由何事唤醒。

### 5. 切换原语（运行态 ↔ 就绪态）

```
切换原语的操作：
1. 将运行进程的运行环境（寄存器数据）存入PCB
2. 将该进程的PCB移入相应队列
3. 选择另一个进程执行，并更新其PCB
4. 根据PCB恢复新进程的运行环境
```

## 四、进程上下文（Context）

进程切换时需要保存和恢复的信息称为**进程上下文**：

| 信息 | 说明 |
|------|------|
| PC（程序计数器） | 下一条指令的地址 |
| PSW（程序状态字） | 程序状态信息 |
| 通用寄存器 | 运行过程中的中间结果 |
| 其他必要信息 | 如栈指针等 |

### 上下文切换过程

```
进程A运行中
    │
    ▼
保存A的上下文到A的PCB
    │
    ▼
从B的PCB恢复B的上下文
    │
    ▼
进程B开始运行
```

---

## 📝 考点总结

1. 进程控制通过**原语**实现，原语具有**原子性**
2. 原子性通过**关中断/开中断**指令实现（特权指令）
3. **阻塞原语和唤醒原语必须成对使用**
4. 因何事阻塞，就应由**何事唤醒**
5. 进程切换时需要保存和恢复**进程上下文**
6. V操作不会导致进程阻塞，因此两个V操作顺序可以交换
