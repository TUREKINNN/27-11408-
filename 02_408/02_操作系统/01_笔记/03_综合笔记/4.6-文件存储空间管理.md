# 文件存储空间管理

## 📊 考情分析
- **考试频率**：高频考点，几乎每年必考
- **分值**：选择题2-4分，综合题可达8-12分
- **题型**：选择题、综合应用题
- **真题考查方式**：空闲表法、空闲链表法、位示图法、成组链接法等

## 一、基本概念

### 1.1 存储空间的划分与初始化
**存储空间的划分**：将物理磁盘划分为一个个文件卷（逻辑卷、逻辑盘）

**存储空间的初始化**：将各个文件卷划分为目录区、文件区
- **目录区**：主要存放文件目录信息（FCB）、用于磁盘存储空间管理的信息
- **文件区**：用于存放文件数据

### 1.2 空闲空间管理的重要性
- 文件存储空间管理是操作系统的重要功能
- 需要有效地管理外存中的空闲空间
- 需要支持文件的创建、删除、拓展等操作

## 二、核心知识

### 2.1 空闲表法

#### 基本原理
- 适用于连续分配方式
- 空闲表中记录每个空闲区间的**第一个空闲盘块号**和**空闲盘块数**
- 类似于内存管理中的动态分区分配

#### 分配算法
- 可以采用首次适应、最佳适应、最坏适应等算法
- 为一个文件分配连续的存储空间

#### 回收算法
- 与内存管理中的动态分区分配很类似
- 当回收某个存储区时需要有四种情况：
  1. 回收区的前后都没有相邻空闲区
  2. 回收区的前后都是空闲区
  3. 回收区前面是空闲区
  4. 回收区后面是空闲区
- 回收时需要注意表项的合并问题

#### 优缺点
- **优点**：实现简单，适用于连续分配
- **缺点**：不适用于离散分配，空闲表可能很大

### 2.2 空闲链表法

#### 空闲盘块链
**基本原理**：
- 将所有空闲磁盘块用链表链接起来
- 操作系统保存着链头、链尾指针
- 空闲盘块中存储着下一个空闲盘块的指针

**分配算法**：
- 若某文件申请K个盘块，则从链头开始依次摘下K个盘块分配
- 修改空闲链的链头指针

**回收算法**：
- 回收的盘块依次挂到链尾
- 修改空闲链的链尾指针

**特点**：
- 适用于离散分配的物理结构
- 为文件分配多个盘块时可能要重复多次操作

#### 空闲盘区链
**基本原理**：
- 连续的空闲盘块组成一个空闲盘区
- 空闲盘区中的第一个盘块内记录了盘区的长度、下一个盘区的指针

**分配算法**：
- 可以采用首次适应、最佳适应等算法
- 从链头开始检索，按照算法规则找到一个大小符合要求的空闲盘区
- 分配给文件
- 若没有合适的连续空闲块，也可以将不同盘区的盘块同时分配给一个文件

**回收算法**：
- 若回收区和某个空闲盘区相邻，则需要将回收区合并到空闲盘区中
- 若回收区没有和任何空闲区相邻，将回收区作为单独的一个空闲盘区挂到链尾

**特点**：
- 离散分配、连续分配都适用
- 为一个文件分配多个盘块时效率更高

### 2.3 位示图法

#### 基本原理
- 每个二进制位对应一个盘块
- "0"代表盘块空闲，"1"代表盘块已分配（或相反）
- 位示图一般用连续的"字"来表示
- 可以用（字号，位号）对应一个盘块号

#### 盘块号与（字号,位号）的转换
假设字长为n，盘块号、字号、位号从0开始：
- (字号, 位号) = (i, j) 的二进制位对应的盘块号 b = n×i + j
- b号盘块对应的字号 i = b/n，位号 j = b%n

**注意**：要能自己推出盘块号与（字号,位号）相互转换的公式。注意题目条件：盘块号、字号、位号到底是从0开始还是从1开始。

#### 分配算法
1. 顺序扫描位示图，找到K个相邻或不相邻的"0"
2. 根据字号、位号算出对应的盘块号，将相应盘块分配给文件
3. 将相应位设置为"1"

#### 回收算法
1. 根据回收的盘块号计算出对应的字号、位号
2. 将相应二进制制位设为"0"

#### 特点
- 连续分配、离散分配都适用
- 位示图常驻内存，分配和回收速度快
- 适用于各种大小的文件系统

### 2.4 成组链接法

#### 基本原理
- 空闲表法、空闲链表法不适用于大型文件系统
- UNIX系统中采用了成组链接法对磁盘空闲块进行管理
- 文件卷的目录区中专门用一个磁盘块作为"超级块"
- 当系统启动时需要将超级块读入内存
- 要保证内存与外存中的"超级块"数据一致

#### 超级块的结构
- 超级块中记录了下一组空闲盘块的数量和块号
- 超级块中的空闲块号指向下一组空闲盘块
- 最后一组的特殊标记表示没有下一组空闲盘块

#### 分配算法
1. 检查第一个分组的块数是否足够
2. 若足够，则分配第一个分组中的空闲块，并修改相应数据
3. 若不够，则将超级块中的下一组空闲盘块信息读入内存，然后分配

#### 回收算法
1. 检查第一组是否已满
2. 若未满，则将回收的盘块号加入第一组，并修改相应数据
3. 若已满，则将第一组的信息写入回收的盘块中，将回收的盘块作为新的超级块

#### 特点
- 适用于大型文件系统
- 分配和回收效率高
- 需要保证内存与外存中的"超级块"数据一致

## 三、深入理解

### 3.1 四种管理方法的比较

| 管理方法 | 适用分配方式 | 分配效率 | 回收效率 | 适用系统 |
|---------|------------|---------|---------|---------|
| 空闲表法 | 连续分配 | 高 | 高 | 小型系统 |
| 空闲盘块链 | 离散分配 | 低 | 低 | 小型系统 |
| 空闲盘区链 | 连续/离散分配 | 高 | 高 | 中型系统 |
| 位示图法 | 连续/离散分配 | 高 | 高 | 各种系统 |
| 成组链接法 | 连续/离散分配 | 高 | 高 | 大型系统 |

### 3.2 位示图法的计算
假设磁盘块大小为1KB，磁盘容量为1GB：
- 磁盘块数 = 1GB / 1KB = 2^20个
- 位示图大小 = 2^20位 = 2^17字节 = 128KB

### 3.3 成组链接法的设计思想
- 将空闲盘块分组管理
- 每组的第一个盘块记录下一组的信息
- 超级块常驻内存，提高分配效率
- 通过链接方式将各组串联起来

- 成组链接法的分配回收过程是重点

## 五、典型例题

### 概念题
**题目**：简述位示图法的基本原理和优缺点。

**答案**：
**基本原理**：
- 每个二进制位对应一个盘块
- "0"代表盘块空闲，"1"代表盘块已分配
- 可以用（字号，位号）对应一个盘块号

**优点**：
- 连续分配、离散分配都适用
- 位示图常驻内存，分配和回收速度快
- 适用于各种大小的文件系统

**缺点**：
- 位示图需要占用存储空间
- 对于大型文件系统，位示图可能很大

### 计算题
**题目**：假设磁盘块大小为1KB，磁盘容量为1GB，位示图中字号和位号都从0开始。问：
1. 磁盘共有多少个磁盘块？
2. 位示图需要多少个字（假设字长为32位）？
3. 盘块号为1000对应的字号和位号是多少？

**答案**：
1. 磁盘块数 = 1GB / 1KB = 2^20 = 1048576个
2. 字数 = 2^20 / 32 = 2^15 = 32768个
3. 字号 = 1000 / 32 = 31，位号 = 1000 % 32 = 8

### 真题
**题目**（2019年408真题）：某文件系统采用位示图法管理空闲磁盘块，磁盘块大小为1KB，磁盘容量为1GB。若位示图中字号和位号都从0开始，则盘块号为2000对应的字号和位号分别是（）
A. 62，16
B. 63，16
C. 62，17
D. 63，17

**答案**：A
**解析**：字号 = 2000 / 32 = 62，位号 = 2000 % 32 = 16

## 六、易错点

1. **位示图法的盘块号转换**
   - 要注意字号和位号的起始值
   - 要注意字长是多少位

2. **成组链接法的分配回收**
   - 要保证内存与外存中的"超级块"数据一致
   - 分配和回收时要正确修改超级块的内容

3. **空闲表法的回收算法**
   - 有四种情况需要考虑
   - 要注意表项的合并问题

4. **空闲链表法的分配效率**
   - 空闲盘块链：分配多个盘块时效率低
   - 空闲盘区链：分配多个盘块时效率高
