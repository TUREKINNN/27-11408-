# 5.5 I/O 核心子系统

## 一、概述

I/O 核心子系统属于操作系统的**内核部分**，对应 I/O 软件层次结构中的**中间三层**（设备独立性软件、设备驱动程序、中断处理程序）。

### 重点掌握的功能

| 功能 | 说明 |
|------|------|
| **I/O 调度** | 用某种算法确定处理 I/O 请求的顺序 |
| **设备保护** | 不同用户对设备有不同的访问权限 |
| **假脱机技术（SPOOLing）** | 将独占设备改造为共享设备 |
| **设备分配与回收** | 管理设备的分配和回收 |
| **缓冲区管理** | 缓冲与高速缓存 |

---

## 二、I/O 调度

用某种算法确定一个好的顺序来处理各个 I/O 请求。

### 常见调度算法

| 设备类型 | 调度算法 |
|---------|---------|
| **磁盘** | FCFS、SSTF、SCAN、C-SCAN、LOOK、C-LOOK |
| **打印机等** | FCFS、优先级算法、短作业优先 |

> 当多个 I/O 请求到来时，用调度算法确定满足请求的顺序。

---

## 三、设备保护

在 UNIX 系统中，设备被看做**一种特殊的文件**：
- 每个设备有对应的 **FCB**
- 用户请求访问设备时，系统根据 FCB 判断该用户是否有**相应的访问权限**
- 实现方式与**文件保护**相同

---

## 四、功能在层次中的归属

```
用户层软件
    └── 假脱机技术（SPOOLing）

设备独立性软件
    └── I/O 调度、设备保护、设备分配与回收、缓冲区管理

设备驱动程序

中断处理程序

硬件
```

> **注意**：假脱机技术一般在**用户层软件**实现（需要请求磁盘设备的设备独立性软件服务）。
