# 5.7 缓冲区管理

## 一、缓冲区概述

### 1.1 什么是缓冲区

缓冲区是一个**存储区域**：
- **硬件缓冲区**：由专门的硬件寄存器组成（成本高、容量小，仅用于高速场合）
- **内存缓冲区**：利用内存作为缓冲区（一般情况）

### 1.2 缓冲区的作用

```
CPU → 快速放入缓冲区 → 做别的事
                          ↓
                   慢速 I/O 设备慢慢取走数据
```

- **输出时**：CPU 快速把数据放入缓冲区，I/O 设备慢慢取走
- **输入时**：I/O 设备慢慢把数据放入缓冲区，CPU 快速取走

> 核心作用：**缓解 CPU 与 I/O 设备速度不匹配的矛盾**。

---

## 二、单缓冲

### 2.1 概念

OS 在主存中为进程分配**一个缓冲区**（默认大小 = 一个块）。

### 2.2 规则

- 缓冲区**非空**时：只能传出数据，不能冲入数据
- 缓冲区**为空**时：可以冲入数据，但必须**充满后**才能传出

### 2.3 处理流程

```
块设备 → [输入T] → 缓冲区 → [传送M] → 工作区 → [处理C] → CPU
```

### 2.4 性能分析

**假设初始状态**：工作区满，缓冲区空

**情况一：T > C（设备输入慢）**

```
时间轴：  0 ──── T+M ──── 2(T+M) ──── 3(T+M)
操作：    设备→缓冲区(T)  缓冲区→工作区(M)  CPU处理(C)
```

- 处理一块数据平均耗时 = **T + M**

**情况二：T < C（CPU 处理慢）**

```
时间轴：  0 ──── C+M ──── 2(C+M) ──── 3(C+M)
操作：    设备→缓冲区(T)  缓冲区→工作区(M)  CPU处理(C)
```

- 处理一块数据平均耗时 = **C + M**

### 2.5 结论

$$\text{单缓冲处理一块数据平均耗时} = \max(C, T) + M$$

---

## 三、双缓冲

### 3.1 概念

OS 在主存中为进程分配**两个缓冲区**。

### 3.2 性能分析

**假设初始状态**：工作区空，一个缓冲区满，另一个缓冲区空

**情况一：T > C + M（设备输入慢）**

- CPU 处理完一块数据时，设备还没输入完下一块
- 处理一块数据平均耗时 = **T**

**情况二：T < C + M（CPU 处理慢）**

- 设备输入完一块数据时，CPU 还没处理完
- 处理一块数据平均耗时 = **C + M**

### 3.3 结论

$$\text{双缓冲处理一块数据平均耗时} = \max(T, C + M)$$

---

## 四、单缓冲 vs 双缓冲对比

| 方案 | 平均耗时公式 | 适用场景 |
|------|-------------|---------|
| **单缓冲** | max(C, T) + M | 一般情况 |
| **双缓冲** | max(T, C + M) | 设备和 CPU 速度差异大时更优 |

---

## 五、通信中的缓冲区配置

### 5.1 单缓冲通信

```
A 机 [发送缓冲区] → 网络 → [接收缓冲区] B 机
```

- 任一时刻只能实现**数据的单向传输**

### 5.2 双缓冲通信

```
A 机 [发送缓冲区] [接收缓冲区] ←→ [接收缓冲区] [发送缓冲区] B 机
```

- 可以实现**双向的数据传输**

---

## 六、循环缓冲区

将多个**大小相等**的缓冲区链接成一个**循环队列**：

```
    ┌──→ [满] → [满] → [空] → [空] → [满] ──┐
    │                                        │
    └────────────────────────────────────────┘
                in ↑          ↑ out
```

- **in 指针**：指向下一个可冲入数据的**空缓冲区**
- **out 指针**：指向下一个可取出数据的**满缓冲区**

---

## 七、缓冲池

### 7.1 概念

缓冲池由系统中**共用的缓冲区**组成。

### 7.2 按使用状况分类

| 队列 | 说明 |
|------|------|
| **空缓冲队列** | 空闲的缓冲区 |
| **输入队列** | 装满输入数据的缓冲区 |
| **输出队列** | 装满输出数据的缓冲区 |

### 7.3 四种工作缓冲区

| 名称 | 缩写 | 功能 |
|------|------|------|
| 收容输入数据的工作缓冲区 | **hin** | 从空缓冲队列取出，冲满后挂到输入队列 |
| 提取输入数据的工作缓冲区 | **sin** | 从输入队列取出，读空后挂到空缓冲队列 |
| 收容输出数据的工作缓冲区 | **hout** | 从空缓冲队列取出，冲满后挂到输出队列 |
| 提取输出数据的工作缓冲区 | **sout** | 从输出队列取出，读空后挂到空缓冲队列 |

### 7.4 工作流程

```
① 输入进程请求输入数据
   → 从空缓冲队列取一块作为 hin
   → 冲满后挂到输入队列队尾

② 计算进程想要取得输入数据
   → 从输入队列取一块作为 sin
   → 读空后挂到空缓冲队列

③ 计算进程想要输出数据
   → 从空缓冲队列取一块作为 hout
   → 冲满后挂到输出队列队尾

④ 输出进程请求输出数据
   → 从输出队列取一块作为 sout
   → 读空后挂到空缓冲队列
```

## 考点总结

> 1. **单缓冲**：平均耗时 = max(C, T) + M（必考计算）
> 2. **双缓冲**：平均耗时 = max(T, C + M)（必考计算）
> 3. 通信中单缓冲只能单向，双缓冲可以双向
> 4. 循环缓冲区的 in/out 指针
> 5. 缓冲池的三种队列和四种工作缓冲区
