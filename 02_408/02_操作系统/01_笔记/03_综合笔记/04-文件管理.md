## 一、基本概念

### 1.1 文件的定义

**文件** 是具有文件名的一组相关信息的集合。文件是操作系统中信息存储和管理的基本单位。

### 1.2 文件的分类

| 分类方式 | 类型 |
|----------|------|
| 按用途 | 系统文件、用户文件、库文件 |
| 按数据形式 | 源文件、目标文件、可执行文件 |
| 按存取控制 | 只读、读写、执行 |
| 按组织方式 | 顺序文件、索引文件、索引顺序文件、散列文件 |

### 1.3 文件系统的层次结构

```
用户接口（文件操作：open/close/read/write）
    ↓
文件目录系统（目录管理）
    ↓
存取控制验证（权限检查）
    ↓
逻辑文件系统（逻辑结构→物理结构映射）
    ↓
物理文件系统（物理块分配）
    ↓
设备管理程序（I/O操作）
    ↓
辅助存储设备（磁盘/磁带）
```

### 1.4 文件的逻辑结构

| 类型 | 特点 |
|------|------|
| **无结构文件（流式文件）** | 一串字节流，Unix/Linux观点 |
| **有结构文件（记录式文件）** | 由一组记录组成 |

#### 有结构文件的组织

| 类型 | 特点 | 查找方式 |
|------|------|----------|
| **顺序文件** | 记录按关键字排列 | 顺序查找/折半查找 |
| **索引文件** | 建立索引表 | 通过索引快速查找 |
| **索引顺序文件** | 分组+索引 | 索引→组内顺序查找 |
| **散列文件** | 用散列函数计算地址 | 直接计算 |

### 1.5 文件的物理结构（分配方式）

#### 连续分配

- 每个文件占用一段**连续**的磁盘块
- **优点**：顺序访问和随机访问都快
- **缺点**：外部碎片、文件不易扩展

#### 链接分配

**隐式链接：**
- 每个盘块包含指向下一个盘块的指针
- **优点**：无外部碎片
- **缺点**：不能随机访问，指针占用空间，可靠性差

**显式链接（FAT）：**
- 将所有指针集中存放在**文件分配表（FAT）**中
- FAT常驻内存，查找速度快
- **优点**：比隐式链接快，可随机访问
- **缺点**：FAT占用内存空间

#### 索引分配

- 每个文件有一个**索引块**，记录文件各逻辑块对应的物理块号
- **优点**：支持随机访问，无外部碎片
- **缺点**：索引块占空间

**多级索引：**
- 索引块太大时，对索引块再建索引
- **混合索引**（Unix）：直接地址 + 一次间接 + 二次间接 + 三次间接

### 1.6 目录结构

| 结构 | 特点 |
|------|------|
| **单级目录** | 所有文件在同一目录，不允许重名 |
| **两级目录** | 主文件目录(MFD) + 用户文件目录(UFD) |
| **树形目录** | 多级层次结构，绝对路径/相对路径 |
| **无环图目录** | 支持文件共享（硬链接/软链接） |

### 1.7 文件共享

| 方式 | 特点 |
|------|------|
| **基于索引节点（硬链接）** | 多个目录项指向同一索引节点，删除一个不影响其他 |
| **基于符号链（软链接）** | 创建Link类型文件，存储目标文件路径 |

### 1.8 文件保护

| 保护方式 | 说明 |
|----------|------|
| **口令** | 简单但安全性低 |
| **加密** | 安全性高但开销大 |
| **访问控制** | 访问控制表(ACL)，按用户/组/其他设置权限 |

## 二、核心知识

### 2.1 索引分配计算

#### 混合索引方式（Unix风格）

设每个盘块大小1KB，每个盘块号占4B：

```
直接地址：10个 → 10 × 1KB = 10KB
一次间接：1个 → (1KB/4B) × 1KB = 256KB
二次间接：1个 → 256 × 256 × 1KB = 64MB
三次间接：1个 → 256 × 256 × 256 × 1KB = 16GB

最大文件大小 = 10KB + 256KB + 64MB + 16GB
```

#### 访问次数分析

- 直接地址：1次访盘
- 一次间接：2次访盘（索引块+数据块）
- 二次间接：3次访盘（一级索引+二级索引+数据块）
- 三次间接：4次访盘

### 2.2 FAT表计算

#### 示例

磁盘有1000个盘块（0-999），用FAT管理。

设文件A占盘块：4→7→2→10→12

FAT表（部分）：
| 盘块号 | FAT值 |
|--------|-------|
| 2 | 10 |
| 4 | 7 |
| 7 | 2 |
| 10 | 12 |
| 12 | -1（结束标志） |

查找文件A的第3个逻辑块：
- 起始盘块=4
- 第1块：4，第2块：FAT[4]=7，第3块：FAT[7]=2
- 物理块号=2

### 2.3 磁盘空间管理

#### 空闲表法

类似内存的空闲分区表，记录空闲区的起始块号和长度。

#### 空闲链表法

将空闲盘块用指针链接成链表。

#### 位示图法

用一个二进制位表示一个盘块的状态：
- 0=空闲，1=已分配（或反之）

**计算示例：**
盘块号从0开始，位示图[i][j]对应的盘块号 = i × 字长 + j

例如字长32位，位示图[2][5]对应的盘块号 = 2×32+5 = 69

#### 成组链接法（Unix）

将空闲盘块分组，每组的最后一个块记录下一组的空闲块信息。

## 三、深入理解

### 3.1 三种分配方式的比较

| 比较项 | 连续分配 | 链接分配 | 索引分配 |
|--------|----------|----------|----------|
| 顺序访问 | 快 | 快 | 快 |
| 随机访问 | 快 | 慢 | 中等 |
| 外部碎片 | 有 | 无 | 无 |
| 文件扩展 | 困难 | 容易 | 容易 |
| 可靠性 | 高 | 低（指针丢失） | 高 |

### 3.2 为什么Unix采用混合索引？

- 小文件（<10KB）：直接访问，快
- 中等文件（<256KB+10KB）：一次间接，够用
- 大文件：二/三次间接，支持超大文件
- **兼顾了小文件效率和大文件支持**

## 五、典型例题

### 例题1（概念题）

**题目**：下列关于文件的叙述，错误的是（ ）
A. 文件的逻辑结构可分为有结构文件和无结构文件
B. 连续分配适合顺序访问，不适合随机访问
C. 链接分配不支持随机访问
D. 索引分配支持随机访问

**思路**：连续分配既适合顺序访问也适合随机访问。

**解答**：选B。连续分配将文件存放在连续的物理块中，知道起始地址和块大小，可以直接计算任意块的物理地址，因此**既适合顺序访问也适合随机访问**。

**总结**：连续分配的最大优点就是支持随机访问，缺点是外部碎片。

---

### 例题2（计算题/索引分配）

**题目**：某文件系统盘块大小1KB，盘块号占4B，采用混合索引：
- 直接地址：6个
- 一次间接：1个
- 二次间接：1个

求：（1）文件最大长度？（2）访问第500个逻辑块需要几次访盘？

**思路**：计算每个索引层能寻址的范围。

**解答**：

（1）每个索引块能容纳的指针数 = 1KB / 4B = 256个

```
直接地址：6 × 1KB = 6KB（逻辑块0-5）
一次间接：256 × 1KB = 256KB（逻辑块6-261）
二次间接：256 × 256 × 1KB = 65536KB（逻辑块262-65797）

最大文件长度 = 6KB + 256KB + 65536KB ≈ 65800KB ≈ 64.2MB
```

（2）第500个逻辑块：
- 500 > 6（超出直接地址范围）
- 500 ≤ 6+256=262？ 500 > 262，超出一次间接
- 在二次间接范围内（262 ≤ 500 ≤ 65797）

访盘次数：
- 1次读二次间接索引块
- 1次读一次间接索引块（500-262=238，第238个指针）
- 1次读数据块
- 共**3次访盘**

**总结**：关键——先判断逻辑块在哪一层索引范围内，再计算访盘次数。

---

### 例题3（综合题/位示图）

**题目**：磁盘共2048个盘块（0-2047），用位示图管理，字长32位。问：
（1）位示图需要多少个字？
（2）盘块号1024对应位示图的哪个字的哪一位？

**思路**：直接计算。

**解答**：

（1）位示图字数 = 2048 / 32 = **64个字**

（2）盘块号1024：
- 字号 = 1024 / 32 = **32**（第33个字）
- 位号 = 1024 % 32 = **0**（最低位）

**总结**：位示图编号从0开始时，盘块号b对应：字号=b/字长，位号=b%字长。

---

### 例题4（真题改编/FAT）

**题目**：磁盘有500个盘块，用FAT管理，FAT表项占2B。文件F的起始盘块为100，FAT[100]=200，FAT[200]=350，FAT[350]=-1。问：
（1）文件F占几个盘块？
（2）文件F的逻辑块号2对应哪个物理盘块？
（3）FAT至少占多少个盘块？

**思路**：沿FAT链追踪。

**解答**：

（1）100→200→350→结束，共**3个盘块**

（2）逻辑块0→物理块100，逻辑块1→物理块200，逻辑块2→物理块**350**

（3）FAT表项数=500，每个表项2B，FAT总大小=500×2=1000B
设每个盘块1KB，则FAT至少占 ⌈1000/1024⌉ = **1个盘块**

**总结**：FAT是显式链接的核心，所有盘块的链接信息集中存储在FAT中。

## 六、易错点

1. ❌ "连续分配不支持随机访问" → ✅ 连续分配**最支持**随机访问
2. ❌ "硬链接删除源文件后，链接文件仍可访问" → ✅ 硬链接指向同一索引节点，只要有一个链接存在，文件就在
3. ❌ "软链接和硬链接完全相同" → ✅ 软链接是新文件存储路径，硬链接是同一索引节点
4. ❌ "索引分配没有缺点" → ✅ 索引块占用空间，小文件效率不如连续分配
5. ❌ "位示图中盘块号从1开始" → ✅ 具体看题目说明，可能从0或1开始
