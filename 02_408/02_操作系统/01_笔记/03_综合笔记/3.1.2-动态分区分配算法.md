# 3.1.2 动态分区分配算法

## 📌 知识总览

动态分区分配算法：在动态分区分配方式中，当很多个空闲分区都能满足需求时，应该选择哪个分区进行分配？本节介绍四种动态分区分配算法。

---

## 一、首次适应算法（First Fit）

### 1.1 算法思想

> 每次都从**低地址**开始查找，找到**第一个**能满足大小的空闲分区。

### 1.2 实现方式

- 空闲分区以**地址递增**的次序排列
- 每次分配内存时**顺序查找**空闲分区链（或空闲分区表）
- 找到大小能满足要求的**第一个**空闲分区

### 1.3 执行过程示例

```
初始状态：
地址 8M: 空闲分区 20MB
地址 32M: 空闲分区 10MB
地址 60M: 空闲分区 4MB

请求：8MB
分配：从地址8M开始查找，第一个满足的是20MB的分区
结果：地址8M的分区变为 12MB（20-8=12），进程放入地址8M~16M
```

### 1.4 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单 | 低地址部分会产生很多小的空闲分区 |
| 查找速度快 | 每次分配都要从头查找，增加开销 |
| 大分区保留在高地址 | 可能产生外部碎片 |

---

## 二、最佳适应算法（Best Fit）

### 2.1 算法思想

> 为了保证当"大进程"到来时能有连续的大片空间，可以**尽可能多地留下大片的空闲区**，即**优先使用更小的空闲区**。

### 2.2 实现方式

- 空闲分区按**容量递增**次序链接
- 每次分配内存时顺序查找空闲分区链
- 找到大小能满足要求的**第一个**空闲分区（即最小的能满足的分区）

### 2.3 执行过程示例

```
初始状态（按容量递增排序）：
地址 60M: 空闲分区 4MB
地址 32M: 空闲分区 10MB
地址 8M: 空闲分区 20MB

请求：3MB
分配：从4MB的分区开始查找，第一个满足的是4MB的分区
结果：地址60M的分区变为 1MB（4-3=1），进程放入地址60M~63M
```

### 2.4 优缺点

| 优点 | 缺点 |
|------|------|
| 保留大的空闲分区 | 会产生很多很小的、难以利用的内存块 |
| 适合大进程 | **产生很多外部碎片** |
| | 需要排序，开销较大 |

---

## 三、最坏适应算法（Worst Fit）

### 3.1 算法思想

> 为了解决最佳适应算法的问题——即留下太多难以利用的小碎片，可以在每次分配时**优先使用最大的连续空闲区**，这样分配后剩余的空闲区就不会太小，更方便使用。

### 3.2 实现方式

- 空闲分区按**容量递减**次序链接
- 每次分配内存时顺序查找空闲分区链
- 找到大小能满足要求的**第一个**空闲分区（即最大的能满足的分区）

### 3.3 执行过程示例

```
初始状态（按容量递减排序）：
地址 8M: 空闲分区 20MB
地址 32M: 空闲分区 10MB
地址 60M: 空闲分区 4MB

请求：3MB
分配：从20MB的分区开始查找，第一个满足的是20MB的分区
结果：地址8M的分区变为 17MB（20-3=17），进程放入地址8M~11M
```

### 3.4 优缺点

| 优点 | 缺点 |
|------|------|
| 分配后剩余的空闲区更大 | 大的连续空闲区被迅速用完 |
| 更方便使用 | 如果之后有"大进程"到达，就没有内存分区可用 |
| | 需要排序，开销较大 |

---

## 四、邻近适应算法（Next Fit）

### 4.1 算法思想

> 首次适应算法每次都从链头开始查找，这可能会导致低地址部分出现很多小的空闲分区，而每次分配查找时，都要经过这些分区，因此也增加了查找的开销。如果每次都从**上次查找结束的位置**开始检索，就能解决上述问题。

### 4.2 实现方式

- 空闲分区以**地址递增**的顺序排列（可排成一个循环链表）
- 每次分配内存时从**上次查找结束的位置**开始查找
- 找到大小能满足要求的**第一个**空闲分区

### 4.3 执行过程示例

```
初始状态（循环链表）：
地址 8M: 空闲分区 20MB
地址 32M: 空闲分区 10MB
地址 60M: 空闲分区 4MB

上次查找结束位置：地址32M

请求：3MB
分配：从地址32M开始查找，第一个满足的是10MB的分区
结果：地址32M的分区变为 7MB（10-3=7），进程放入地址32M~35M
```

### 4.4 优缺点

| 优点 | 缺点 |
|------|------|
| 查找开销更均匀 | 高地址部分的大分区更可能被使用 |
| 不会每次从头查找 | 可能导致无大分区可用 |

---

## 五、四种算法对比 ⭐⭐⭐

### 5.1 算法特性对比

| 算法 | 算法思想 | 分区排列顺序 | 优点 | 缺点 |
|------|---------|------------|------|------|
| **首次适应** | 从头到尾找适合的分区 | 地址递增 | 实现简单；保留高地址大分区 | 低地址产生小碎片 |
| **最佳适应** | 优先使用更小的分区 | 容量递增 | 保留大分区 | 产生很多小碎片（外部碎片） |
| **最坏适应** | 优先使用最大的分区 | 容量递减 | 分配后剩余空闲区大 | 大分区被迅速用完 |
| **邻近适应** | 从上次结束位置查找 | 地址递增（循环） | 查找开销均匀 | 可能无大分区可用 |

### 5.2 综合评价

> **结论：** 综合来看，四种算法中，**首次适应算法**的效果反而更好。

**原因分析：**
- 首次适应算法的规则决定了当低地址部分有更小的分区可以满足需求时，会更有可能用到低地址部分的小分区
- 也会更有可能把高地址部分的大分区保留下来（类似最佳适应算法的优点）
- 邻近适应算法可能导致无论低地址、高地址部分的空闲分区都有相同的概率被使用
- 这导致了高地址部分的大分区更可能被使用，划分为小分区，最后导致无大分区可用（类似最坏适应算法的缺点）

## 六、重点考点总结

### 6.1 考试常见题型

1. **根据分配序列判断使用了哪种算法**
2. **计算分配后的内存状态**
3. **比较不同算法的性能**
4. **分析外部碎片的产生**

### 6.2 计算技巧

1. **首次适应**：从地址0开始找第一个满足的
2. **最佳适应**：从最小的分区开始找满足的
3. **最坏适应**：从最大的分区开始找满足的
4. **邻近适应**：从上次结束位置开始找满足的

### 6.3 记忆口诀

```
首次适应：地址递增，从头找
最佳适应：容量递增，找最小
最坏适应：容量递减，找最大
邻近适应：地址递增，从上次找
```

---

## 💡 易错点提醒

1. **首次适应和邻近适应**：空闲分区按地址递增排列
2. **最佳适应**：空闲分区按容量递增排列
3. **最坏适应**：空闲分区按容量递减排列
4. **最佳适应产生最多外部碎片**
5. **最坏适应导致大分区被迅速用完**
6. **首次适应综合效果最好**
