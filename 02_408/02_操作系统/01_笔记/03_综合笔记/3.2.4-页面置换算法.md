# 3.2.4 页面置换算法

> [[02_408/02_操作系统/01_笔记/03_综合笔记/03-内存管理]] | [[02_408/02_操作系统/01_笔记/03_综合笔记/3.1.3.1-基本分页存储管理]]

---

## 📌 核心概念

**页面置换算法**是当内存已满，需要调入新页面时，选择哪个页面被淘汰的策略。目标是减少缺页中断次数。

### 性能指标

- **缺页率** = 缺页次数 / 总访问次数
- **缺页中断次数**：越少越好
- **抖动（Thrashing）**：频繁换入换出现象

---

## 🔄 常见算法

### 1. 最佳置换算法（OPT）

| 项目 | 说明 |
|------|------|
| **原理** | 淘汰未来最长时间不再被访问的页面 |
| **优点** | 缺页率最低（理论最优） |
| **缺点** | 无法实现（无法预知未来） |
| **用途** | 作为评价其他算法的基准 |

### 2. 先进先出（FIFO）

| 项目 | 说明 |
|------|------|
| **原理** | 淘汰最早进入内存的页面 |
| **实现** | 队列 |
| **优点** | 实现简单 |
| **缺点** | 性能差，可能淘汰热点页面 |
| **Belady异常** | 物理块增加，缺页次数反而增加 |

### 3. 最近最久未使用（LRU）

| 项目 | 说明 |
|------|------|
| **原理** | 淘汰最长时间未被访问的页面 |
| **实现** | 栈/计数器 |
| **优点** | 性能接近 OPT |
| **缺点** | 硬件实现复杂，开销大 |

### 4. 时钟（CLOCK）/ 最近未使用（NRU）

| 项目 | 说明 |
|------|------|
| **原理** | 循环扫描，使用访问位（A）和修改位（M） |
| **实现** | 循环链表 |
| **优点** | 近似 LRU，开销小 |
| **缺点** | 精度略低 |

---

## 📊 算法对比

| 算法 | 复杂度 | 性能 | 实现难度 | Belady异常 |
|------|--------|------|----------|-----------|
| OPT | - | 最优 | 不可实现 | 无 |
| FIFO | O(1) | 差 | 简单 | 有 |
| LRU | O(1)~O(n) | 好 | 较难 | 无 |
| CLOCK | O(1) | 较好 | 中等 | 无 |

---

## 🔢 计算示例

**访问序列**：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
**物理块数**：3

### FIFO 算法

```
访问 → 内存状态 → 缺页？
1  → [1]      → 缺页（1次）
2  → [1,2]    → 缺页（2次）
3  → [1,2,3]  → 缺页（3次）
4  → [2,3,4]  → 缺页（4次，淘汰1）
1  → [3,4,1]  → 缺页（5次，淘汰2）
2  → [4,1,2]  → 缺页（6次，淘汰3）
5  → [1,2,5]  → 缺页（7次，淘汰4）
1  → [2,5,1]  → 缺页（8次，淘汰2）
2  → [5,1,2]  → 缺页（9次，淘汰5）
3  → [1,2,3]  → 缺页（10次，淘汰1）
4  → [2,3,4]  → 缺页（11次，淘汰2）
5  → [3,4,5]  → 缺页（12次，淘汰3）
```

FIFO 缺页次数：12次

### LRU 算法

```
访问 → 内存状态（最近使用在右）→ 缺页？
1  → [1]                      → 缺页（1次）
2  → [1,2]                    → 缺页（2次）
3  → [1,2,3]                  → 缺页（3次）
4  → [2,3,4]                  → 缺页（4次，淘汰1）
1  → [3,4,1]                  → 缺页（5次，淘汰2）
2  → [4,1,2]                  → 缺页（6次，淘汰3）
5  → [1,2,5]                  → 缺页（7次，淘汰4）
1  → [2,5,1]                  → 命中
2  → [5,1,2]                  → 命中
3  → [1,2,3]                  → 缺页（8次，淘汰5）
4  → [2,3,4]                  → 缺页（9次，淘汰1）
5  → [3,4,5]                  → 缺页（10次，淘汰2）
```

LRU 缺页次数：10次

---

## ⚠️ Belady 异常

**现象**：物理块数增加，缺页次数反而增加

**原因**：FIFO 算法的固有缺陷

**示例**：访问序列 1,2,3,4,1,2,5,1,2,3,4,5
- 3 个物理块：FIFO 缺页 9 次
- 4 个物理块：FIFO 缺页 10 次

**结论**：FIFO 不是栈式算法，可能产生 Belady 异常；LRU 是栈式算法，不会产生 Belady 异常。
