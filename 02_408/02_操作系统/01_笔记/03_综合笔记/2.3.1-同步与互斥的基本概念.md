## 一、进程同步

### 问题背景

> 进程具有**异步性**：各并发执行的进程以各自独立的、不可预知的速度向前推进。

### 同步的含义

操作系统要提供**"进程同步机制"**来解决异步问题。

> **同步**：直接制约关系。多个进程在某些操作上需要按照一定的**先后顺序**执行。

### 示例

```
进程A：① 从缓冲区读数据 → ② 数据预处理 → ③ 写回缓冲区 → ④ 其他任务

进程B：① 准备工作 → ② 从缓冲区读数据 → ③ 后续处理
```

- 进程B的②必须在进程A的③**之后**执行（A写完B才能读）
- 这就是**同步关系**（一前一后）

## 二、进程互斥

### 临界资源

- **临界资源**：一个时间段内只允许**一个进程**访问的资源
- 许多物理设备（如摄像头、打印机）属于临界资源
- 许多变量、数据、内存缓冲区也属于临界资源

### 互斥的含义

> **互斥**：间接制约关系。当一个进程访问某临界资源时，另一个想要访问该临界资源的进程**必须等待**。

### 临界区的逻辑结构

对临界资源的互斥访问，在逻辑上分为**四个部分**：

```
do {
    进入区;      // 检查是否可进入，设置"上锁"标志
    临界区;      // 访问临界资源的代码段
    退出区;      // 解除"锁"标志
    剩余区;      // 做其他处理
} while(true)
```

| 部分 | 说明 |
|------|------|
| **进入区** | 负责检查是否可进入临界区，设置正在访问临界资源的标志（"上锁"） |
| **临界区** | 访问临界资源的代码段（也称"临界段"） |
| **退出区** | 负责解除正在访问临界资源的标志（"解锁"） |
| **剩余区** | 做其他处理 |

### 互斥访问的四个原则

| 原则 | 说明 |
|------|------|
| **空闲让进** | 临界区空闲时，可以允许一个请求进入临界区的进程**立即进入** |
| **忙则等待** | 已有进程进入临界区时，其他进程**必须等待** |
| **有限等待** | 应保证请求访问的进程能在**有限时间内**进入临界区（不饥饿） |
| **让权等待** | 不能进入临界区时，应**立即释放处理机**，防止忙等待 |

> ⚠️ **让权等待**不是所有算法都能满足的，如Peterson算法就不满足让权等待。

---

## 📝 考点总结

1. **同步**：直接制约关系，需要按照先后顺序执行
2. **互斥**：间接制约关系，对临界资源的访问需要互斥
3. 临界区的四个部分：进入区、临界区、退出区、剩余区
4. 互斥的四个原则：**空闲让进、忙则等待、有限等待、让权等待**
5. 进入区和退出区是负责实现**互斥**的代码段
