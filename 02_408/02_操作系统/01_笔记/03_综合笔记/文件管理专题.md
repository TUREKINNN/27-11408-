## 一、索引节点（inode）

### 概念

inode（index node）是 Unix/Linux 文件系统中用于存储文件元数据的数据结构。一个文件对应一个 inode，包含：

| 属性 | 说明 |
|------|------|
| 文件类型 | 普通文件、目录、链接等 |
| 权限 | 读、写、执行权限 |
| 所有者 | 用户 ID (UID) |
| 文件大小 | 字节数 |
| 时间戳 | 创建时间、修改时间、访问时间 |
| 数据块指针 | 指向文件数据块的指针数组 |

### inode 结构

```
struct inode {
    int i_mode;      // 文件类型和权限
    int i_nlinks;    // 硬链接计数
    int i_uid;       // 所有者
    int i_size;      // 文件大小（字节）
    int i_atime;     // 最后访问时间
    int i_mtime;     // 最后修改时间
    int i_ctime;     // 创建时间
    int i_addr[13];  // 数据块指针（12直接 + 1间接）
};
```

### 目录项与 inode 的关系

```
目录项（dentry）：文件名 → inode编号
inode：inode编号 → 文件元数据 + 数据块

文件打开过程：
open("foo.txt") → 查目录项 → 得到inode号 → 读inode → 得到数据块地址 → 读数据
```

### 硬链接与软链接

| 项目 | 硬链接 | 软链接（符号链接） |
|------|--------|-------------------|
| 实质 | 多个目录项指向同一 inode | 新文件存储目标文件路径 |
| inode | 共享 | 不共享 |
| 跨文件系统 | ❌ 不可以 | ✅ 可以 |
| 删除源文件 | 不影响（引用计数减1） | 链接失效 |
| 限制 | 不能链接目录 | 无限制 |

## 二、文件系统全局结构（layout）

### 磁盘布局

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ 引导块       │ 超级块       │ 索引节点区   │ 数据块区     │
│ (Boot Block) │ (Superblock)│ (inode区)   │ (Data Blocks)│
└─────────────┴─────────────┴─────────────┴─────────────┘
```

| 区域 | 说明 |
|------|------|
| **引导块** | 存放引导程序，用于系统启动 |
| **超级块** | 文件系统元数据（块大小、总块数、空闲块信息等） |
| **索引节点区** | 存放所有 inode |
| **数据块区** | 存放文件实际数据 |

### 内存中的文件系统结构

```
打开文件表（系统级）
    ↓
进程打开文件表
    ↓
文件描述符 → 系统级打开文件表项 → inode
```

## 三、虚拟文件系统（VFS）

### 概念

**VFS（Virtual File System）**：在内核中抽象出的一层，为上层提供统一的文件操作接口，屏蔽底层不同文件系统（ext4、NTFS、FAT等）的差异。

### VFS 层次结构

```
用户空间：open(), read(), write(), close()
    ↓
VFS 层：提供统一接口
    ↓
具体文件系统：ext4, NTFS, FAT32, NFS...
    ↓
物理设备：磁盘、SSD、网络存储
```

### VFS 核心对象

| 对象 | 说明 |
|------|------|
| **超级块（super_block）** | 描述已安装文件系统 |
| **索引节点（inode）** | 描述具体文件 |
| **目录项（dentry）** | 描述目录项，建立文件名→inode映射 |
| **文件对象（file）** | 描述由进程打开的文件 |

### VFS 的作用

1. **统一接口**：上层不用关心底层文件系统类型
2. **可扩展**：新增文件系统只需实现 VFS 接口
3. **跨文件系统操作**：支持不同文件系统间的数据复制

## 四、文件系统挂载（mounting）

### 概念

**挂载**：将一个文件系统安装到目录树的某个目录上，使其可被访问。

### 挂载过程

```
① 在目录树中选择一个空目录作为挂载点
② 读取文件系统的超级块，获取元数据
③ 将文件系统的根目录与挂载点关联
④ 用户访问挂载点时，重定向到文件系统根目录
```

### 挂载表示例

```bash
# Linux 挂载示例
mount /dev/sda1 /mnt/data    # 将 sda1 挂载到 /mnt/data
mount -t nfs server:/share /mnt/nfs  # 挂载 NFS 网络文件系统
umount /mnt/data             # 卸载
```

### 挂载表

系统维护一个挂载表，记录所有已安装的文件系统：

| 挂载点 | 设备 | 文件系统类型 | 选项 |
|--------|------|-------------|------|
| / | /dev/sda1 | ext4 | rw |
| /boot | /dev/sda2 | ext4 | ro |
| /mnt/data | /dev/sdb1 | ntfs | rw,uid=1000 |

## 五、内存映射文件（Memory-Mapped Files）

### 概念

**mmap**：将文件映射到进程的虚拟地址空间，使文件访问变为内存访问。

### 原理

```
普通文件访问：
用户进程 → read() → 内核缓冲区 → 磁盘

内存映射文件：
用户进程 → 直接读写内存 → 页式虚拟存储 → 磁盘（按需调页）
```

### mmap 系统调用

```c
void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);
```

| 参数 | 说明 |
|------|------|
| addr | 映射到的虚拟地址（通常为 NULL，由系统分配） |
| length | 映射长度 |
| prot | 保护属性（PROT_READ, PROT_WRITE 等） |
| flags | 映射类型（MAP_SHARED, MAP_PRIVATE） |
| fd | 文件描述符 |
| offset | 文件中的偏移量 |

### 优点

1. **简化文件操作**：无需 read/write，直接内存访问
2. **提高效率**：减少数据拷贝次数（零拷贝）
3. **共享内存**：MAP_SHARED 使多个进程共享同一映射
4. **延迟加载**：文件内容按需从磁盘加载到内存

### 与虚拟内存的关系

- mmap 利用页式虚拟存储机制
- 访问映射区域时，若页面不在内存，触发缺页中断
- 操作系统将文件对应部分调入内存
- 写回策略（MAP_SHARED）：修改内容写回磁盘

## 六、外存空闲空间管理方法

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **空闲表法** | 记录所有空闲区间的起始块号和长度 | 简单 | 表太大 |
| **空闲链表法** | 用链表连接所有空闲块 | 简单 | 查找慢 |
| **位示图法** | 用二进制位表示每个块的状态 | 空间效率高 | 需要扫描 |
| **成组链接法** | 将空闲块分组，组间链接 | Unix 常用 | 实现复杂 |
