# 2.1.3 进程的状态与转换

## 一、进程的五种基本状态

### 1. 创建态（New）

- 进程正在被创建时的状态
- 操作系统为进程**分配资源**、**初始化PCB**

### 2. 就绪态（Ready）

- 进程创建完成后进入就绪态
- 已经**具备运行条件**（获得了除CPU之外的所有资源）
- 但由于**没有空闲CPU**，暂时不能运行
- 系统中可能有**多个**进程处于就绪态，组成**就绪队列**

### 3. 运行态（Running）

- CPU空闲时，操作系统选择一个就绪进程上处理机运行
- 该进程处于运行态
- **单CPU**：同一时刻只会有一个进程处于运行态
- **多核CPU**：同一时刻可能有多个进程处于运行态

### 4. 阻塞态（Blocked/Waiting）

- 进程运行过程中，可能请求等待某个事件发生（如等待I/O设备）
- 在事件发生之前，进程无法继续执行
- 操作系统让该进程**下CPU**，进入阻塞态
- 当等待的事件发生时，从阻塞态回到就绪态

### 5. 终止态（Terminated）

- 进程执行 `exit` 系统调用，请求操作系统终止
- 操作系统让该进程下CPU，**回收内存空间**等资源
- 最后还要**回收PCB**
- 当终止工作完成后，进程彻底消失

## 二、进程状态的转换

### 状态转换图

```
            创建
             │
             ▼
    ┌─────────────────┐     进程被调度      ┌──────────────┐
    │     就绪态        │ ──────────────────> │    运行态      │
    │   (Ready)         │                     │  (Running)     │
    └─────────────────┘ <────────────────── └──────────────┘
             ▲           时间片到/主动让出CPU        │
             │                                     │
     等待事件发生                          进程申请资源/
             │                           等待事件发生
             │                                     │
    ┌─────────────────┐                     ▼
    │     阻塞态        │               终止态(Terminated)
    │   (Blocked)       │
    └─────────────────┘
```

### 关键转换条件

| 转换 | 条件 |
|------|------|
| 创建态 → 就绪态 | 系统完成创建进程的一系列工作（分配资源、初始化PCB） |
| 就绪态 → 运行态 | 进程被调度（被选中上CPU） |
| 运行态 → 就绪态 | **时间片到**，或主动让出CPU（Yield） |
| 运行态 → 阻塞态 | 进程**申请某种系统资源**，或**等待某个事件发生**（主动行为） |
| 阻塞态 → 就绪态 | 申请的资源被分配，或**等待的事件发生**（被动行为，由其他进程或操作系统唤醒） |

> ⚠️ 注意：**阻塞态 → 运行态** 和 **就绪态 → 阻塞态** 这两种转换是**不存在**的！

## 三、进程的组织方式

### 1. 链接方式

按照进程状态将PCB分为多个队列：

```
执行指针 ──────> PCB2（当前运行态的进程，单CPU只有一个）

就绪队列指针 ──> PCB5 → PCB1 → PCB4（优先级高的放队头）

阻塞队列指针 ──> PCB3 → PCB7 → PCB6（可按阻塞原因分多个队列）
```

- **执行指针**：指向当前处于运行态的进程（单CPU只有一个）
- **就绪队列指针**：指向当前处于就绪态的进程
- **阻塞队列指针**：指向当前处于阻塞态的进程
- 多核CPU可有多个进程同时处于运行态（运行队列）

### 2. 索引方式

根据进程状态的不同，建立几张索引表：

```
执行指针 ──────> PCB2

就绪表指针 ────> 就绪索引表 → PCB5, PCB1, PCB4
阻塞表指针 ────> 阻塞索引表 → PCB6, PCB7, PCB3
```

## 四、七状态模型（含挂起态）

在五状态模型基础上引入**挂起态**，形成七状态模型：

| 状态 | 说明 |
|------|------|
| 就绪态 | 在**内存**中，等待CPU |
| 就绪挂起 | 被调到**外存**，但具备运行条件 |
| 阻塞态 | 在**内存**中，等待事件 |
| 阻塞挂起 | 被调到**外存**，等待事件 |

### 挂起与阻塞的区别

| 对比 | 阻塞态 | 挂起态 |
|------|--------|--------|
| 进程映像位置 | 在**内存**中 | 被调到**外存** |
| 激活条件 | 等待的事件发生 | 重新调入内存 |

> 挂起态是将进程映像调到外存去了，而阻塞态下进程映像还在内存中。

---

## 📝 考点总结

1. 进程的三种基本状态：**就绪态、运行态、阻塞态**
2. 进程PCB中有一个变量 `state` 表示进程的当前状态
3. **阻塞态 → 运行态** 和 **就绪态 → 阻塞态** 不存在
4. 运行态 → 阻塞态是进程的**主动行为**
5. 阻塞态 → 就绪态是**被动行为**（由其他进程或OS唤醒）
6. 区分挂起态和阻塞态：映像在内存还是外存
