# 3.1.1 内存的基础知识

## 📌 知识总览

内存是计算机中重要的组成部分，用于存放程序和数据。本节介绍内存的基本概念、地址空间、指令的工作原理以及程序的装入过程。

---

## 一、什么是内存

### 1.1 内存的定义

> 内存可存放数据。程序执行前需要先放到内存中才能被CPU处理——**缓和CPU与硬盘之间的速度矛盾**。

### 1.2 内存的编址

在多道程序环境下，系统中会有多个程序并发执行，需要区分各个程序的数据放在什么地方。

**方案：给内存的存储单元编地址**

- 内存中有"小房间"，每个小房间就是一个"存储单元"
- 内存地址从0开始，每个地址对应一个存储单元

### 1.3 编址方式

| 编址方式 | 存储单元大小 | 说明 |
|---------|------------|------|
| **按字节编址** | 1字节（1B = 8位） | 最常见的方式 |
| **按字编址** | 1个字 | 如字长16位，则每个存储单元大小为16位 |

### 1.4 常用数量单位

```
2^10 = 1K（千）
2^20 = 1M（兆，百万）
2^30 = 1G（十亿，千兆）
```

**例子：** 4GB内存
- 4GB = 4 × 2^30 = 2^32 个字节
- 按字节编址，有2^32个"小房间"
- 需要32个二进制位表示地址（0 ~ 2^32-1）

## 二、指令的工作原理

### 2.1 程序的执行过程

```
源代码 → 编译 → 链接 → 装入模块（可执行文件） → 装入内存 → CPU执行
```

### 2.2 指令中的地址

每条指令告诉CPU：
1. 应该去内存的**哪个地址**读/写数据
2. 这个数据应该做**什么样的处理**

**例子：** `x = x + 1` 的执行过程

```c
// 编译后的指令
指令1：把地址79中的数据读入寄存器3  // LOAD
指令2：把寄存器3的数据加1           // ADD
指令3：把寄存器3的数据存入地址79     // STORE
```

### 2.3 逻辑地址 vs 物理地址

| 地址类型 | 别名 | 说明 |
|---------|------|------|
| **逻辑地址** | 相对地址 | 相对于进程起始地址的偏移量 |
| **物理地址** | 绝对地址 | 在内存中的实际位置 |

> 程序经过编译、链接后生成的指令中指明的是**逻辑地址**（相对地址），即相对于进程起始地址而言的地址。

## 三、程序的装入过程

### 3.1 装入的三种方式

#### 3.1.1 绝对装入

> 在编译时，如果知道程序将放到内存中的哪个位置，编译程序将产生**绝对地址**的目标代码。

**特点：**
- 只适用于**单道程序环境**
- 编译时就知道装入位置
- 程序中使用的绝对地址，可在编译或汇编时给出

**例子：**
```
编译时知道从地址100开始存放
指令0：往地址179的存储单元中写入10
指令1：把地址179中的数据读入寄存器3
```

#### 3.1.2 可重定位装入（静态重定位）

> 编译、链接后的装入模块的地址都是从0开始的。装入时对地址进行"重定位"，将逻辑地址变换为物理地址（地址变换是在**装入时一次完成**的）。

**特点：**
- 一个作业装入内存时，必须分配其要求的**全部内存空间**
- 如果没有足够的内存，就不能装入该作业
- 作业一旦进入内存后，在运行期间**不能再移动**，也不能再申请内存空间

**例子：**
```
装入的起始物理地址为100
逻辑地址79 → 物理地址179（所有地址+100）
```

#### 3.1.3 动态运行时装入（动态重定位）

> 编译、链接后的装入模块的地址都是从0开始的。装入程序把装入模块装入内存后，并不会立即把逻辑地址转换为物理地址，而是把地址转换推迟到**程序真正要执行时**才进行。

**特点：**
- 需要一个**重定位寄存器**的支持
- 装入内存后所有的地址依然是逻辑地址
- 程序执行时，通过重定位寄存器将逻辑地址转换为物理地址

**例子：**
```
重定位寄存器存放起始地址100
执行时：逻辑地址79 → 物理地址79+100=179
```

## 四、程序的链接过程

### 4.1 链接的三种方式

| 链接方式 | 说明 | 特点 |
|---------|------|------|
| **静态链接** | 程序运行前，将各目标模块及所需库函数链接成一个完整的装入模块 | 链接后不再拆开 |
| **装入时动态链接** | 装入内存时，边装入边链接 | 便于修改和更新 |
| **运行时动态链接** | 程序执行中需要该模块时，才对它进行链接 | 便于实现对目标模块的共享 |

---

## 五、地址空间

### 5.1 逻辑地址空间

- 进程的逻辑地址空间从0开始
- 每个地址相对于进程起始地址的偏移量
- 进程看到的地址都是逻辑地址

### 5.2 物理地址空间

- 内存的物理地址空间从0开始
- 每个地址对应内存中的实际位置
- CPU最终访问的是物理地址

### 5.3 地址转换

> 操作系统需要提供**地址重定位**的功能，将逻辑地址转换为物理地址。

---

## 六、重点考点总结

### 6.1 三种装入方式对比

| 装入方式 | 地址变换时机 | 灵活性 | 硬件支持 |
|---------|------------|--------|---------|
| 绝对装入 | 编译时 | 最差 | 无 |
| 静态重定位 | 装入时 | 较差 | 无 |
| 动态重定位 | 执行时 | 最好 | 重定位寄存器 |

### 6.2 逻辑地址与物理地址

```
逻辑地址：程序中使用的地址（相对于进程起始地址）
物理地址：内存中的实际地址（绝对地址）

关系：物理地址 = 逻辑地址 + 基址寄存器的值
```

### 6.3 考试常见题型

1. **计算地址长度**：根据内存大小确定需要多少位表示地址
2. **地址转换**：根据装入方式将逻辑地址转换为物理地址
3. **区分装入方式**：理解三种装入方式的特点和适用场景

---

## 💡 易错点提醒

1. **逻辑地址和物理地址是不同的概念**
2. **绝对装入只适用于单道程序环境**
3. **静态重定位在装入时完成，动态重定位在执行时完成**
4. **动态重定位需要重定位寄存器支持**
5. **4GB内存需要32位地址**
6. **按字节编址是最常见的方式**
