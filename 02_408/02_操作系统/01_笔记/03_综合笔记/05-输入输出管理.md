# 05-输入输出管理

## 考情分析
- **选择题高频**：缓冲区处理时间计算、I/O控制方式对比、磁盘调度算法计算
- **大题重点**：磁盘调度算法（FCFS/SSTF/SCAN/C-SCAN/LOOK/C-LOOK）
- **易混点**：DMA与通道方式区别、单双缓冲时间公式、SCAN与LOOK区别

---

## 一、I/O设备分类

### 1.1 按传输速率分类
| 类别 | 举例 | 速率特征 |
|------|------|----------|
| **低速设备** | 键盘、鼠标 | 几字节~几百字节/s |
| **中速设备** | 打印机 | 几千~几万字节/s |
| **高速设备** | 磁盘、SSD | 几十万~几MB/s |

### 1.2 按信息交换单位分类
| 类别 | 特征 | 举例 |
|------|------|------|
| **块设备** | 以数据块为单位传输，可寻址 | 磁盘、SSD |
| **字符设备** | 以字符为单位传输，不可寻址 | 键盘、打印机、鼠标 |

## 二、I/O控制方式

### 2.1 四种方式详细对比

| 特性 | 程序查询方式 | 中断驱动方式 | DMA方式 | 通道方式 |
|------|------------|-------------|---------|---------|
| **CPU介入程度** | 极高，全程参与 | 较高，每字节中断一次 | 低，仅开始和结束 | 极低，仅启动通道 |
| **数据传送单位** | 字节/字 | 字节/字 | **数据块** | **一组数据块** |
| **数据传送控制** | CPU直接控制 | CPU参与每次传送 | DMA控制器 | 通道程序 |
| **CPU与设备并行** | ❌不能 | 部分并行 | ✅高度并行 | ✅高度并行 |
| **适用场景** | 低速简单设备 | 低速设备 | 高速设备（磁盘） | 多设备复杂系统 |
| **优点** | 简单 | CPU利用率提高 | 效率高 | CPU负担最小 |
| **缺点** | CPU利用率极低 | 中断频繁开销大 | 需专用硬件 | 通道程序复杂 |

### 2.2 各方式核心原理

**程序查询方式**：CPU不断轮询设备状态寄存器（忙等待），效率最低。
**中断驱动方式**：设备准备好后发中断请求，CPU响应中断处理数据。
**DMA方式**：
- DMA控制器接管总线，直接在设备和内存间传输数据
- 传输单位：**数据块**（不是字节）
- CPU仅在传输开始时设置DMA寄存器（内存地址、字节数、方向），传输结束时由DMA发中断
- CPU与DMA控制器交替使用总线（**周期窃取**）

**通道方式**：
- 通道是专用处理器，执行通道程序（通道指令序列）
- CPU只需启动通道，通道自主完成全部I/O
- 传输单位：**一组数据块**

## 三、I/O软件层次结构

```
┌─────────────────────────┐
│     用户层I/O软件        │  ← 用户接口（库函数如printf）
├─────────────────────────┤
│   设备独立性软件          │  ← 统一接口、设备分配、缓冲管理、逻辑→物理映射
├─────────────────────────┤
│     设备驱动程序          │  ← 每类设备一个驱动，将抽象请求转为具体操作
├─────────────────────────┤
│     中断处理程序          │  ← 最底层，响应硬件中断
├─────────────────────────┤
│       硬件               │
└─────────────────────────┘
```

**各层功能**：
- **中断处理程序**：保存/恢复现场，唤醒等待进程，位于最底层
- **设备驱动程序**：将上层的抽象I/O请求转换为对具体设备的操作命令；每类设备对应一个驱动
- **设备独立性软件（设备无关软件）**：实现设备统一命名、设备分配与回收、缓冲区管理、逻辑设备名→物理设备名映射
- **用户层软件**：提供用户接口（库函数），如C的`printf`、`scanf`

## 四、缓冲区管理

### 4.1 单缓冲
- 系统为进程分配**一个**缓冲区
- 处理一块数据时间：**T = max(C, T) + M**
  - C：CPU处理时间
  - T：设备将数据传送到缓冲区时间
  - M：缓冲区数据传送到用户区时间
- C和T可以并行（CPU处理上一块时，设备可输入下一块），M与C/T串行

### 4.2 双缓冲（缓冲区交换）
- 系统分配**两个**缓冲区
- 处理一块数据时间：**T = max(C + M, T)**
- 设备可连续填入缓冲区，不需要等待M完成

### 4.3 循环缓冲
- 多个缓冲区组成循环队列
- 输入和输出可使用各自缓冲区队列

### 4.4 缓冲池
- 系统公用缓冲区池，分为：空缓冲队列、输入队列、输出队列
- 四种工作缓冲区：收容输入(hin)、提取输入(sin)、收容输入(hout)、提取输出(sout)
- 最灵活，现代操作系统广泛采用

### 对比表
| 类型 | 缓冲区数 | 处理时间公式 | 特点 |
|------|---------|-------------|------|
| 单缓冲 | 1 | max(C,T)+M | 最简单 |
| 双缓冲 | 2 | max(C+M,T) | 设备可连续工作 |
| 循环缓冲 | ≥3 | — | 多缓冲区循环 |
| 缓冲池 | 公用 | — | 最灵活 |

## 五、SPOOLing技术（假脱机技术）

### 核心思想
利用磁盘作为中间存储，将独占设备改造为共享设备。

### 组成部分
| 组成部分 | 说明 |
|---------|------|
| **输入井** | 磁盘上模拟输入设备，暂存输入数据 |
| **输出井** | 磁盘上模拟输出设备，暂存输出数据 |
| **输入缓冲区** | 内存中，暂存从输入设备读入的数据 |
| **输出缓冲区** | 内存中，暂存准备输出到设备的数据 |
| **输入进程** | 模拟脱机输入时的外围控制机 |
| **输出进程** | 模拟脱机输出时的外围控制机 |

### 打印机实例
1. 用户进程请求打印 → 数据先送到**输出井**
2. 输出进程从输出井取出数据 → 送**输出缓冲区** → 送打印机
3. 打印机被多进程**共享**（每个进程以为自己独占）
4. 实际上是**虚拟设备**技术

## 六、磁盘调度算法

### 6.1 磁盘地址结构
- **柱面号（磁道号）**：磁头臂移动方向
- **盘面号（磁头号）**：选择哪个磁头
- **扇区号**：磁盘旋转后定位

> 寻道时间最耗时，调度算法主要优化**寻道时间**。

### 6.2 调度算法详解

#### FCFS（先来先服务）
按请求到达顺序服务。

**例题**：磁头当前在53号磁道，请求队列：98,183,37,122,14,124,65,67
```
移动顺序：53→98→183→37→122→14→124→65→67
移动磁道数：|53-98|+|98-183|+|183-37|+|37-122|+|122-14|+|14-124|+|124-65|+|65-67|
= 45+85+146+85+108+110+59+2 = 640
平均寻道长度：640/8 = 80
```

#### SSTF（最短寻道时间优先）
选择离当前磁头最近的请求。

**例题**：同上，当前53号
```
53→37→14→65→67→98→122→124→183
移动：16+23+51+2+31+24+2+59 = 208
平均：208/8 = 26
```
⚠️ 可能产生**饥饿**（远处请求长期得不到服务）。

#### SCAN（电梯算法）
磁头单向移动，到达边界后反向。

**例题**：当前53号，向磁道号增大方向移动
```
53→65→67→98→122→124→183→37→14
移动：12+2+31+24+2+59+146+23 = 299
平均：299/8 ≈ 37.4
```

#### C-SCAN（循环扫描）
磁头单向移动，到达边界后**直接返回起点**（不服务），再单向扫描。

**例题**：当前53号，向增大方向
```
53→65→67→98→122→124→183→14→37
移动：12+2+31+24+2+59+169+23 = 322
（到183后回到0再到14，移动183+14=197？注意C-SCAN直接回0再到14）
```

#### LOOK / C-LOOK
- **LOOK**：SCAN改进，不走到边界，到达请求最远处就反向
- **C-LOOK**：C-SCAN改进，到达请求最远处直接回最小请求处

### 算法对比
| 算法 | 优点 | 缺点 |
|------|------|------|
| FCFS | 公平 | 效率低，移动量大 |
| SSTF | 平均寻道短 | 可能饥饿 |
| SCAN | 较公平，无饥饿 | 两端请求等待久 |
| C-SCAN | 更公平，各处等待均匀 | 返回时不服务 |
| LOOK | SCAN优化 | 同SCAN |
| C-LOOK | C-SCAN优化 | 同C-SCAN |

## 七、SSD基本原理与磨损均衡

### SSD基本原理
- 基于**闪存（NAND Flash）**，无机械部件
- 读取快、无寻道时间、抗震动
- 写入前必须**擦除**（擦除以块为单位，读写以页为单位）
- 有**写入次数限制**（SLC ~10万次，MLC ~1万次，TLC ~3000次）

### 磨损均衡（Wear Leveling）
- **目的**：让所有闪存块均匀擦写，避免某些块过早损坏
- **静态磨损均衡**：将不常修改的数据搬到磨损严重的块，把新块用于频繁写入
- **动态磨损均衡**：每次写入选择擦写次数最少的块

### SSD vs HDD
| 特性 | SSD | HDD |
|------|-----|-----|
| 寻道时间 | 无 | 有（机械臂移动） |
| 随机读写 | 极快 | 慢 |
| 寿命 | 有限写入次数 | 机械磨损 |
| 抗震 | 强 | 弱 |

## 九、典型例题

### 例题1：单缓冲计算
> 磁盘读一块数据到缓冲区用时80μs，缓冲区传到用户区用时20μs，CPU处理用时50μs。求处理一块数据时间。

**解**：T=max(C,T_device)+M = max(50,80)+20 = 100μs

### 例题2：双缓冲计算
> 条件同上。

**解**：T=max(C+M, T_device) = max(50+20, 80) = 80μs

### 例题3：SCAN算法
> 请求队列：23,89,132,42,187,当前在60，向增大方向。求总移动距离。

**解**：60→89→132→187→42→23，移动=29+43+55+145+19=291

### 例题4：I/O方式判断
> DMA方式中，CPU参与的工作是（）
A. 数据传送 B. 中断处理 C. 设置DMA寄存器 D. 数据校验

**解**：C。CPU仅在传输开始时设置DMA控制寄存器。

## 十、易错点
1. 单缓冲时间公式是 **max(C,T)+M**，不是 C+T+M
2. DMA的数据传送单位是**数据块**，中断驱动是**字节/字**
3. SCAN到边界就反向，LOOK到最远请求就反向
4. SPOOLing的输入井输出井在**磁盘**上，缓冲区在**内存**中
5. SSD写入前必须先**擦除**，擦除以块为单位
