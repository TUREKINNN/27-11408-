## 一、基本概念

### 1.1 操作系统的定义

**操作系统（Operating System, OS）** 是控制和管理计算机硬件与软件资源、合理地组织计算机工作流程、方便用户使用的**程序集合**。它是计算机系统中最基本的**系统软件**，是用户与计算机硬件之间的**接口**。

### 1.2 操作系统的特征

| 特征 | 含义 | 关键词 |
|------|------|--------|
| **并发（Concurrence）** | 多个程序在同一时间间隔内交替执行 | 宏观并行、微观串行 |
| **共享（Sharing）** | 系统资源供多个并发进程共同使用 | 互斥共享、同时共享 |
| **虚拟（Virtual）** | 通过技术将物理实体变为逻辑对应物 | 时分复用、空分复用 |
| **异步（Asynchronism）** | 程序以不可预知的速度向前推进 | 走走停停、推进速度不可预知 |

> **并发和共享是操作系统的两个最基本的特征**，二者互为存在条件。

### 1.3 操作系统的目标和功能

| 功能 | 说明 |
|------|------|
| **处理机管理** | 进程控制、进程同步、进程通信、调度 |
| **存储器管理** | 内存分配、地址映射、内存保护、内存扩充 |
| **文件管理** | 文件存储空间管理、目录管理、文件读写保护 |
| **设备管理** | 缓冲管理、设备分配、设备处理 |

### 1.4 操作系统的发展历程

| 阶段 | 特点 | 关键技术 |
|------|------|----------|
| 手工操作 | 无OS，用户独占 | — |
| 单道批处理 | 自动性、顺序性、单道性 | 脱机I/O |
| 多道批处理 | 多道、宏观并行、微观串行 | 中断、通道 |
| 分时系统 | 多路性、独立性、及时性、交互性 | 时间片 |
| 实时系统 | 及时性、可靠性 | 硬实时/软实时 |

### 1.5 操作系统的运行机制

> 考纲中使用"**内核模式**"和"**用户模式**"，与"内核态/用户态"同义。

| 概念 | 说明 |
|------|------|
| **内核态（管态 / 内核模式）** | 可执行特权指令，访问所有资源 |
| **用户态（目态 / 用户模式）** | 只能执行非特权指令，受限访问 |
| **特权指令** | I/O指令、置中断屏蔽指令、存取保护寄存器等 |
| **非特权指令** | 算术运算、逻辑运算等普通指令 |

**内核态↔用户态的切换：**
- 用户态→内核态：**中断/异常**（唯一途径）
- 内核态→用户态：设置程序状态字PSW

### 1.6 中断与异常

| 分类 | 来源 | 例子 |
|------|------|------|
| **中断（外中断）** | CPU外部设备 | I/O中断、时钟中断 |
| **异常（内中断/陷入）** | CPU内部 | 除零错误、缺页异常、系统调用 |

### 1.7 系统调用

**系统调用** 是操作系统提供给应用程序的接口，是用户程序取得操作系统服务的**唯一途径**。

系统调用与一般函数调用的区别：
| 比较项 | 系统调用 | 普通函数调用 |
|--------|----------|-------------|
| 运行状态 | 内核态 | 用户态 |
| 入口 | 中断机制 | 函数调用指令 |
| 返回 | 可能切换进程 | 返回调用者 |

### 1.8 操作系统体系结构

| 类型 | 特点 | 代表 |
|------|------|------|
| **宏内核（单内核）** | 功能模块集中，性能高，可靠性低 | Linux、Windows |
| **微内核** | 仅核心功能在内核，其他在用户态，可靠但性能低 | Mach、QNX |
| **层次式** | 分层设计，每层只能调用下层 | THE |
| **模块化** | 内核模块化，可动态加载 | Solaris |

## 二、核心知识

### 2.1 中断处理流程

```
1. 关中断（防止新中断打断当前处理）
2. 保存断点（PC和PSW入栈）
3. 中断服务程序寻址（查中断向量表）
4. 保存现场和屏蔽字
5. 开中断（允许更高优先级中断嵌套）
6. 执行中断服务程序
7. 关中断
8. 恢复现场和屏蔽字
9. 开中断
10. 返回断点（IRET指令）
```

### 2.2 系统调用的执行过程

```
1. 用户程序调用库函数（如printf）
2. 库函数将系统调用号和参数放入约定寄存器
3. 执行陷入指令（trap/INT），触发中断
4. CPU切换到内核态
5. 通过系统调用号查系统调用表
6. 执行对应的系统调用处理程序
7. 返回结果给用户程序
8. CPU切换回用户态
```

## 三、深入理解

### 3.1 并发与并行的区别

- **并发（Concurrence）**：同一时间间隔内多个程序交替执行（单处理器即可实现）
- **并行（Parallelism）**：同一时刻多个程序同时执行（需多处理器）

### 3.2 为什么系统调用是进入内核态的唯一途径？

因为如果用户程序能直接调用内核函数，就无法实现保护。系统调用通过**陷入指令**实现，每次调用都经过**参数检查**，确保安全性。

### 3.3 微内核的优缺点

**优点**：可靠性高（某服务崩溃不影响内核）、可扩展性好、易于移植
**缺点**：性能较低（频繁的用户态↔内核态切换、消息传递开销大）

## 五、典型例题

### 例题1（概念题）

**题目**：下列关于操作系统的叙述中，正确的是（ ）
A. 操作系统是用户和计算机之间的接口
B. 操作系统是软件和硬件之间的接口
C. 操作系统是源程序和目标程序之间的接口
D. 操作系统是外设和主机之间的接口

**思路**：操作系统是用户与计算机硬件之间的接口，不是单纯的软件或硬件接口。

**解答**：选A。操作系统是配置在计算机硬件上的第一层软件，是对硬件系统的首次扩充，是用户与计算机硬件之间的接口。

**总结**：记住OS的标准定义——"用户与计算机硬件之间的接口"。

---

### 例题2（计算题/辨析题）

**题目**：在单处理器系统中，有n个进程，最多有多少个进程处于就绪态？最多有多少个进程处于运行态？

**思路**：单处理器同一时刻只有一个进程在运行。

**解答**：
- 运行态：**1个**（单处理器同一时刻只能执行一个进程）
- 就绪态：**n-1个**（排除正在运行的那一个，其余最多n-1个处于就绪态）
- 阻塞态：可以是0到n-1个

**总结**：单处理器系统中，运行态进程数 ≤ 1，就绪态 + 运行态 + 阻塞态 = n。

---

### 例题3（综合题）

**题目**：说明中断和异常的区别，并各举两个例子。

**思路**：从来源、触发方式、可否屏蔽等角度区分。

**解答**：

| 比较项 | 中断（外中断） | 异常（内中断） |
|--------|---------------|---------------|
| 来源 | CPU外部 | CPU内部 |
| 触发时机 | 与当前指令无关 | 与当前指令有关 |
| 是否可屏蔽 | 可屏蔽（I/O中断）或不可屏蔽（时钟中断） | 不可屏蔽 |
| 例子 | I/O完成中断、时钟中断 | 除零错误、缺页异常、溢出 |

**总结**：中断是"外部事件引起"，异常是"指令执行引起"。

---

### 例题4（真题改编）

**题目**：下列选项中，不可能在用户态发生的是（ ）
A. 系统调用
B. 外部中断
C. 进程切换
D. 缺页异常

**思路**：系统调用的"执行"发生在内核态（虽然由用户态发起），进程切换也必须在内核态完成。

**解答**：选C。进程切换需要修改PCB、保存/恢复上下文等内核操作，只能在内核态完成。系统调用虽然由用户态发起，但其"发起"（执行trap指令）发生在用户态，"执行"在内核态——题目问的是"发生"，一般理解为"触发/发起"，所以A可在用户态发生。缺页异常是指令执行时触发的，在用户态可能发生。

**总结**：区分"发生"和"执行"——系统调用在用户态发起，在内核态执行。

## 六、易错点

1. ❌ "并发就是并行" → ✅ 并发是交替执行，并行是同时执行
2. ❌ "操作系统的最基本特征是共享" → ✅ **并发**和**共享**互为存在条件，缺一不可
3. ❌ "系统调用在内核态执行，所以是内核态发生" → ✅ 发起在用户态，执行在内核态
4. ❌ "异常可以屏蔽" → ✅ 异常不可屏蔽（因为它由当前指令引起，必须处理）
5. ❌ "分时系统就是实时系统" → ✅ 分时强调交互性，实时强调及时性和可靠性

### 操作系统概念深度补充

#### 操作系统定义的三个维度

从不同角度理解操作系统：

| 维度 | 描述 | 通俗理解 |
|------|------|----------|
| **① 系统资源的管理者** | 控制和管理整个计算机系统的硬件和软件资源 | "管家"——管理CPU、内存、设备等 |
| **② 向上层提供方便易用的服务** | 把丑陋的硬件功能封装成简单易用的服务 | "翻译官"——把硬件指令翻译成用户能理解的操作 |
| **③ 最接近硬件的层次** | 对硬件机器的拓展，将裸机改造成功能更强的机器 | "改装师"——让硬件能协调工作 |

> **一台电脑的诞生**：裸机（纯硬件） → 安装操作系统 → 安装应用程序 → 用户使用

#### 操作系统功能详解

**作为系统资源管理者——用QQ视频聊天的例子**：

| 步骤 | 操作 | OS功能 |
|------|------|--------|
| Step 1 | 在文件夹中找到QQ安装位置 | **文件管理** |
| Step 2 | 双击打开QQ.exe，将相关数据放入内存 | **存储器管理**（内存分配） |
| Step 3 | QQ进程被CPU处理 | **处理机管理**（进程调度） |
| Step 4 | 使用摄像头进行视频 | **设备管理**（设备分配） |

#### 用户接口分类

```
操作系统提供的接口
├── 命令接口
│   ├── 联机命令接口（交互式）：用户说一句，系统做一句（如cmd中的time命令）
│   └── 脱机命令接口（批处理）：用户说一堆，系统做一堆（如.bat批处理文件）
└── 程序接口（系统调用）
    └── 由一系列系统调用组成，是应用程序请求操作系统服务的唯一方式
    └── 例：printf函数底层使用了显示相关的系统调用
```

> GUI（图形化用户接口）使用户可以使用形象的图形界面操作，不需要记忆复杂命令。如在Windows中，删除文件只需拖拽到回收站。GUI、命令接口统称为"用户接口"。

#### 操作系统作为虚拟机

没有任何软件支持的计算机称为**裸机**。覆盖了软件的机器称为**扩充机器**或**虚拟机**。

类比理解：
```
发动机——只会转；轮胎——只会滚
覆盖传动系统 → 让发动机带着轮子转 → 硬件得到拓展

操作系统 → 将CPU、内存、磁盘、显示器、键盘等合理组织 → 硬件相互协调配合
```

### 操作系统四大特征深度解析

#### 并发（Concurrence）

**关键辨析**：

| 概念 | 定义 | 要求 |
|------|------|------|
| **并发** | 两个或多个事件在**同一时间间隔**内发生 | 单处理器即可（宏观并行，微观串行） |
| **并行** | 两个或多个事件在**同一时刻**同时发生 | 需要多处理器 |

> - **单核CPU**：同一时刻只能执行一个程序，各程序只能**并发**执行
> - **多核CPU**（如4核）：同一时刻可同时执行4个程序，可**并行**执行
> - 即使是4核CPU，只要有超过4个程序需要"同时"运行，**并发性**依然是必不可少的
> - 操作系统就是伴随着"多道程序技术"而出现的，**操作系统和程序并发一起诞生**

#### 共享（Sharing）

| 共享方式 | 描述 | 生活例子 |
|----------|------|----------|
| **互斥共享** | 一个时间段内只允许一个进程访问 | 使用QQ和微信视频——摄像头同一时间只能给一个程序 |
| **同时共享** | 宏观上多进程"同时"访问（微观上可能是交替的） | QQ发文件A + 微信发文件B——都在访问硬盘（分时共享） |

#### 并发与共享的关系——互为存在条件

```
并发性：系统中同时存在多个运行着的程序
    ↓ 需要
共享性：资源可供多个并发进程共同使用

失去并发性 → 只有一个程序运行 → 共享性失去意义
失去共享性 → QQ和微信不能同时访问硬盘 → 无法同时发送文件 → 无法并发
```

#### 虚拟（Virtual）

把一个**物理上的实体**变为若干个**逻辑上的对应物**。

| 虚拟技术 | 物理实体 | 逻辑对应物 | 实现方式 |
|----------|----------|-----------|----------|
| **虚拟存储器** | 实际4GB内存 | 用户看来远大于4GB | **空分复用技术** |
| **虚拟处理器** | 1个单核CPU | 用户看来有多个CPU | **时分复用技术** |

> 没有并发性，就谈不上虚拟性。如果只有一个程序运行，虚拟技术就失去了意义。

#### 异步（Asynchronism）

在多道程序环境下，由于**资源有限**，进程的执行不是一贯到底的，而是**走走停停**，以**不可预知的速度**向前推进。

> 只有系统拥有并发性，才有可能导致异步性。如果系统只能串行运行各程序，每个程序的执行会一贯到底。

### 内核态与用户态补充

#### 特权指令示例

| 类型 | 例子 |
|------|------|
| **特权指令**（仅内核态可执行） | I/O指令、置中断屏蔽指令、存取保护寄存器、清内存指令 |
| **非特权指令**（用户态可执行） | 算术运算、逻辑运算、访管指令（陷入指令） |

#### 中断处理与进程切换

| 操作 | 执行状态 | 说明 |
|------|----------|------|
| 系统调用的**发起** | 用户态 | 执行trap/INT指令 |
| 系统调用的**执行** | 内核态 | 在内核中完成服务 |
| 外部中断的**响应** | 内核态 | CPU自动切换到内核态 |
| 进程切换 | **仅内核态** | 修改PCB、保存/恢复上下文 |

> - "不可能在用户态发生的是"——**进程切换**必须在内核态完成
> - 系统调用在用户态**发起**，在内核态**执行**——区分"发生"和"执行"
> - 缺页异常在用户态可能发生（是指令执行时触发的）
