# 3.1.2 连续分配管理方式

## 📌 知识总览

连续分配：指为用户进程分配的必须是一个**连续的内存空间**。本节介绍三种连续分配方式：单一连续分配、固定分区分配、动态分区分配。

---

## 一、单一连续分配

### 1.1 基本概念

在单一连续分配方式中，内存被分为：
- **系统区**：通常位于内存的低地址部分，用于存放操作系统相关数据
- **用户区**：用于存放用户进程相关数据

> 内存中只能有**一道用户程序**，用户程序**独占整个用户区空间**。

### 1.2 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单 | 只能用于**单用户、单任务**的操作系统 |
| 无外部碎片 | 有**内部碎片** |
| 可以采用覆盖技术扩充内存 | 存储器利用率极低 |
| 不一定需要采取内存保护 | 不能用于多道程序环境 |

### 1.3 内部碎片

> 分配给某进程的内存区域中，如果有些部分没有用上，就是**"内部碎片"**。

---

## 二、固定分区分配

### 2.1 基本概念

20世纪60年代出现了支持多道程序的系统，为了能在内存中装入多道程序，且这些程序之间又不会相互干扰，于是将整个用户空间划分为**若干个固定大小的分区**，在每个分区中只装入一道作业。

### 2.2 分区大小的划分方式

| 划分方式 | 特点 | 适用场景 |
|---------|------|---------|
| **分区大小相等** | 缺乏灵活性 | 用一台计算机控制多个相同对象的场合（如钢铁厂的炼钢炉控制程序） |
| **分区大小不等** | 增加了灵活性 | 根据常在系统中运行的作业大小情况进行划分（多个小分区、适量中等分区、少量大分区） |

### 2.3 分区说明表

操作系统需要建立一个数据结构——**分区说明表**，来实现各个分区的分配与回收。

**表项内容：**
- 每个表项对应一个分区
- 通常按分区大小排列
- 每个表项包括：分区的大小、起始地址、状态（是否已分配）

**分配过程：**
1. 当用户程序要装入内存时
2. 由操作系统内核程序根据用户程序大小检索该表
3. 从中找到一个能满足大小的、未分配的分区
4. 将之分配给该程序，然后修改状态为"已分配"

### 2.4 优缺点

| 优点 | 缺点 |
|------|------|
| 实现简单 | 当用户程序太大时，可能所有分区都不能满足需求 |
| 无外部碎片 | 会产生**内部碎片**，内存利用率低 |
| | 可能需要采用覆盖技术 |

## 三、动态分区分配 ⭐⭐⭐

### 3.1 基本概念

> 动态分区分配又称为**可变分区分配**。这种分配方式**不会预先划分**内存分区，而是在进程装入内存时，根据进程的大小**动态地建立分区**，并使分区的大小正好适合进程的需要。

**特点：** 系统分区的大小和数目是**可变的**。

### 3.2 需要解决的问题

1. 系统要用什么样的**数据结构**记录内存的使用情况？
2. 当很多个空闲分区都能满足需求时，应该**选择哪个分区**进行分配？
3. 如何进行分区的**分配与回收**操作？

### 3.3 数据结构

#### 3.3.1 空闲分区表

每个空闲分区对应一个表项，表项中包含：
- 分区号
- 分区大小
- 分区起始地址
- 状态

#### 3.3.2 空闲分区链

每个分区的起始部分和末尾部分分别设置前向指针和后向指针。起始部分处还可记录分区大小等信息。

### 3.4 分配算法（下节详细介绍）

当很多个空闲分区都能满足需求时，需要按照一定的**动态分区分配算法**选择分区：
- 首次适应算法
- 最佳适应算法
- 最坏适应算法
- 邻近适应算法

### 3.5 分配操作

**情况一：空闲分区大小恰好等于请求大小**

```
分配前：空闲分区表项 {大小: 20MB, 起始地址: 8M}
分配后：删除该表项
```

**情况二：空闲分区大小大于请求大小**

```
分配前：空闲分区表项 {大小: 20MB, 起始地址: 8M}
请求：4MB
分配后：修改表项 {大小: 16MB, 起始地址: 12M}
```

### 3.6 回收操作

回收时需要考虑回收区与空闲分区的相邻情况：

#### 情况一：回收区后面有相邻的空闲分区

```
回收前：
- 进程1（20MB）
- 进程4（4MB）← 回收
- 空闲分区（10MB）

回收后：
- 进程1（20MB）
- 空闲分区（14MB）← 合并
```

#### 情况二：回收区前面有相邻的空闲分区

```
回收前：
- 空闲分区（20MB）
- 进程4（4MB）← 回收
- 进程3（18MB）

回收后：
- 空闲分区（28MB）← 合并（前面）
- 进程3（18MB）
```

#### 情况三：回收区前后都有相邻的空闲分区

```
回收前：
- 空闲分区（20MB）
- 进程4（4MB）← 回收
- 空闲分区（10MB）

回收后：
- 空闲分区（34MB）← 三个分区合并
```

#### 情况四：回收区前后都没有相邻的空闲分区

```
回收前：
- 进程1（20MB）
- 进程4（4MB）← 回收
- 进程3（18MB）

回收后：
- 进程1（20MB）
- 空闲分区（4MB）← 新建表项
- 进程3（18MB）
```

## 四、外部碎片与内部碎片

### 4.1 内部碎片

> 分配给某进程的内存区域中，有些部分没有用上，就是**"内部碎片"**。

**例子：** 固定分区分配中，分区大小为10MB，进程只需要7MB，剩余3MB就是内部碎片。

### 4.2 外部碎片

> 内存中的某些空闲分区太小而难以利用，就是**"外部碎片"**。

**例子：** 动态分区分配中，内存中有多个小的空闲分区（如1MB、2MB），但进程需要5MB，这些小分区无法满足需求。

### 4.3 解决外部碎片的方法

| 方法 | 说明 |
|------|------|
| **紧凑（Compaction）** | 将内存中的进程移动到一起，合并空闲分区 |
| **分页存储管理** | 将进程分为多个页面，可以放在不相邻的内存块中 |

## 五、重点考点总结

### 5.1 三种连续分配方式对比

| 分配方式 | 分区数量 | 分区大小 | 内部碎片 | 外部碎片 | 适用环境 |
|---------|---------|---------|---------|---------|---------|
| 单一连续分配 | 1个 | 固定 | 有 | 无 | 单用户单任务 |
| 固定分区分配 | 多个 | 固定 | 有 | 无 | 多道程序 |
| 动态分区分配 | 动态变化 | 可变 | 无 | 有 | 多道程序 |

### 5.2 考试常见题型

1. **区分内部碎片和外部碎片**
2. **分析动态分区分配的回收操作**
3. **计算内存利用率**
4. **选择合适的分配方式**

---

## 💡 易错点提醒

1. **内部碎片**是分配了但没用上的部分
2. **外部碎片**是空闲但太小无法利用的部分
3. 固定分区分配有内部碎片，无外部碎片
4. 动态分区分配无内部碎片，有外部碎片
5. 回收时需要考虑四种相邻情况
6. **紧凑**可以解决外部碎片，但需要开销
