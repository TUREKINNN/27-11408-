# 5.6 假脱机技术（SPOOLing 技术）

## 一、脱机技术背景

### 1.1 手工操作阶段的问题

```
输入设备（慢）→ 主机（快）→ 输出设备（慢）
```

- 主机直接从 I/O 设备获取数据
- 设备速度慢，主机速度快 → **人机速度矛盾**
- 主机浪费大量时间等待设备

### 1.2 批处理阶段的脱机技术

引入**脱机输入/输出技术**（用磁带完成）：

```
输入设备（慢）→ 外围控制机 → 磁带（快）→ 主机
```

- 在外围控制机控制下，慢速设备的数据先输入到**快速磁带**
- 主机从快速磁带读入数据 → **缓解速度矛盾**
- **"脱机"** = 脱离主机控制进行的输入/输出操作

---

## 二、SPOOLing 技术

### 2.1 概念

**SPOOLing 技术**用**软件的方式模拟脱机技术**。

### 2.2 系统组成

```
输入设备                            输出设备
    ↓                                  ↑
输入缓冲区（内存）                    输出缓冲区（内存）
    ↓                                  ↑
输入进程                              输出进程
    ↓                                  ↑
输入井（磁盘）                        输出井（磁盘）
```

| 组件 | 位置 | 功能 |
|------|------|------|
| **输入井** | 磁盘 | 模拟脱机输入时的磁带，收容 I/O 设备输入的数据 |
| **输出井** | 磁盘 | 模拟脱机输出时的磁带，收容用户进程输出的数据 |
| **输入进程** | 内存 | 模拟脱机输入时的外围控制机 |
| **输出进程** | 内存 | 模拟脱机输出时的外围控制机 |
| **输入缓冲区** | 内存 | 暂存从输入设备输入的数据，之后转存到输入井 |
| **输出缓冲区** | 内存 | 暂存从输出井送来的数据，之后传送到输出设备 |

### 2.3 实现条件

必须有**多道程序技术**的支持。

## 三、共享打印机原理

### 3.1 设备分类

| 类型 | 说明 |
|------|------|
| **独占式设备** | 一段时间内只能满足一个进程的请求（如打印机） |
| **共享设备** | 可同时满足多个进程的使用请求（如磁盘） |

### 3.2 SPOOLing 改造打印机

打印机是独占式设备，但可用 SPOOLing 技术改造成**共享设备**。

### 3.3 工作流程

当多个用户进程提出打印请求时：

**假脱机管理进程**为每个进程做两件事：
1. 在磁盘**输出井**中申请一个空闲缓冲区，将打印数据送入其中
2. 申请一张空白的**打印请求表**，填入打印请求信息，挂到**假脱机文件队列**

**输出进程**的工作：
1. 打印机空闲时，从文件队列**队头**取出一张打印请求表
2. 根据表中要求，将数据从**输出井**传送到**输出缓冲区**
3. 输出到打印机进行打印
4. 依次处理完全部打印任务

### 3.4 核心思想

虽然系统中只有一台打印机，但每个进程都觉得在**独占一台打印机**：
- 为每个进程在输出井中分配一个存储区（**相当于分配了一个逻辑设备**）
- SPOOLing 技术可将**独占式设备改造为共享设备**

> **SPOOLing 的本质**：把一台物理设备**虚拟成逻辑上的多台设备**。

---

## 四、SPOOLing 技术的特点

| 特点 | 说明 |
|------|------|
| 提高 I/O 速度 | 利用磁盘作为缓冲 |
| 将独占设备改造为共享设备 | 实现设备的虚拟化 |
| 实现虚拟设备技术 | 每个进程都觉得自己独占设备 |
