# 2.2.1 调度的概念、层次

```
│   └── 低级调度（进程调度）

```

## 一、调度的基本概念

> 当有一堆任务要处理，但由于资源有限，这些事情没法同时处理。需要确定某种**规则**来决定处理这些任务的**顺序**，这就是"调度"研究的问题。

---

## 二、调度的三个层次

### 1. 高级调度（作业调度）

- **外存 → 内存**（面向**作业**）
- 按一定原则从外存的**作业后备队列**中挑选一个作业**调入内存**，并**创建进程**
- 每个作业只调入一次，调出一次
- 作业调入时会建立PCB，调出时才撤销PCB
- 发生频率**最低**

> 简化理解：好几个程序需要启动，到底先启动哪个？

### 2. 低级调度（进程调度/处理机调度）

- **内存 → CPU**（面向**进程**）
- 按照某种策略从**就绪队列**中选取一个进程，将处理机分配给它
- 这是操作系统中**最基本**的一种调度
- 进程调度的频率很高，一般**几十毫秒一次**
- 发生频率**最高**

### 3. 中级调度（内存调度）

- **外存 ↔ 内存**（面向**进程**）
- 内存不够时，可将某些进程的数据**调出外存**
- 等内存空闲或进程需要运行时再**重新调入内存**
- 暂时调到外存等待的进程状态为**挂起状态**
- 被挂起的进程PCB会被组织成**挂起队列**
- 一个进程可能被多次调出、调入
- 发生频率**中等**

---

## 三、三层调度的对比

| 对比项 | 高级调度（作业调度） | 中级调度（内存调度） | 低级调度（进程调度） |
|--------|-------------------|--------------------|--------------------|
| **做什么** | 从后备队列选作业调入内存 | 从挂起队列选进程调回内存 | 从就绪队列选进程分配CPU |
| **调度发生在** | 外存→内存 | 外存→内存 | 内存→CPU |
| **发生频率** | 最低 | 中等 | 最高 |
| **对进程状态的影响** | 无→创建态→就绪态 | 挂起态→就绪态 | 就绪态→运行态 |

## 四、七状态模型

在五状态模型基础上引入**挂起态**：

```
        创建态
           │
           ▼
    ┌──→ 就绪态 ◄──────┐
    │      │            │
    │      ▼            │ 激活
    │    运行态          │
    │      │            │
    │      ▼            │
    │    阻塞态 ────────┘
    │      │
    │      ▼ 挂起
    │    阻塞挂起
    │      │
    │      ▼ 事件出现
    │    就绪挂起
    │      │
    │      ▼ 激活
    └───── 就绪态
           │
           ▼
        终止态
```

### 挂起态的两种细分

| 状态 | 说明 |
|------|------|
| **就绪挂起** | 进程在外存，但已具备运行条件，只需调入内存即可变为就绪态 |
| **阻塞挂起** | 进程在外存，且还在等待某个事件发生 |

> ⚠️ **挂起 ≠ 阻塞**：挂起态的进程映像在外存，阻塞态的进程映像在内存

---

## 📝 考点总结

1. 三层调度：**高级调度**（作业）、**中级调度**（内存）、**低级调度**（进程）
2. **低级调度**是最基本的调度，频率最高
3. 高级调度频率最低，中级调度居中
4. **挂起态**和**阻塞态**的区别：进程映像在外存还是内存
5. 七状态模型：五状态 + 就绪挂起 + 阻塞挂起
