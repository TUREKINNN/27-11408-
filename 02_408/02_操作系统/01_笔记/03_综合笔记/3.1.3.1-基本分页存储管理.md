## 📌 核心概念

**分页存储管理**是操作系统内存管理中的一种非连续分配方式，将进程的逻辑地址空间和内存物理空间都划分为固定大小的块，通过页表实现地址映射。

### 与连续分配对比

| 特性 | 连续分配 | 分页存储 |
|------|----------|----------|
| 内存连续性 | 要求连续 | 不要求连续 |
| 外部碎片 | 有 | 无 |
| 内部碎片 | 无 | 有（最后一页） |
| 实现复杂度 | 简单 | 复杂（需页表） |
| 地址转换 | 简单 | 需查页表 |

---

## 📐 地址结构

### 逻辑地址结构

```
| 页号P | 页内偏移W |
|--------|-----------|
```

- **页号P**：逻辑地址空间中的页编号
- **页内偏移W**：页内的字节偏移

### 地址转换公式

```
已知：逻辑地址 A，页面大小 L
     P = A / L  （整除，取页号）
     W = A % L  （取余，得偏移）

物理地址 = 页框号F × 页面大小 + W
```

---

## 🔄 地址转换过程

```
① CPU 给出逻辑地址 (P, W)
② 查页表：页号P → 页框号F
③ 物理地址 = F × 页面大小 + W
④ 访问物理地址
```

### 示例

页面大小 = 4KB，逻辑地址 = 8196
```
页号P = 8196 / 4096 = 2
偏移W = 8196 % 4096 = 4

假设页表：第2页 → 页框5
物理地址 = 5 × 4096 + 4 = 20484
```

---

## 📊 页表

### 基本页表

| 页号 | 页框号 |
|------|--------|
| 0 | 5 |
| 1 | 2 |
| 2 | 7 |
| 3 | 1 |

### 页表寄存器（PTR）

- 存储在 PCB 中
- 指向当前进程的页表起始地址

---

## ⚠️ 注意事项

1. **页面大小选择**：
   - 太小：页表过大，碎片少
   - 太大：内部碎片多
   - 常见：4KB（2¹²）

2. **最后一页的内部碎片**：
   - 进程大小往往不是页面大小的整数倍
   - 最后一页会有部分空间浪费

3. **访问内存次数**：
   - 无快表：2次（查页表 + 访问数据）
   - 有快表：1次（快表命中时）
