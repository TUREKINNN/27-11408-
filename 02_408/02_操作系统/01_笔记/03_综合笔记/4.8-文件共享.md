# 文件共享

## 📊 考情分析
- **考试频率**：高频考点，几乎每年必考
- **分值**：选择题2-4分，综合题可达8-12分
- **题型**：选择题、综合应用题
- **真题考查方式**：硬链接、软链接、文件共享的实现方式等

## 一、基本概念

### 1.1 文件共享的定义
**文件共享**是指多个用户共享地使用同一个文件。

### 1.2 文件共享与文件复制的区别
- **文件共享**：系统中只有"一份"文件数据，某个用户修改了该文件的数据，其他用户也可以看到文件数据的变化
- **文件复制**：系统中会有"好几份"文件数据，其中一个用户修改了自己的那份文件数据，对其他用户的文件数据并没有影响

### 1.3 文件共享的重要性
- 节省存储空间
- 减少数据冗余
- 方便用户协作

## 二、核心知识

### 2.1 基于索引节点的共享方式（硬链接）

#### 基本原理
- 索引节点中设置一个链接计数变量count
- count表示链接到本索引节点上的用户目录项数
- 若count=2，说明此时有两个用户目录项链接到该索引节点上，或者说是有两个用户在共享此文件

#### 删除文件的过程
- 若某个用户决定"删除"该文件，则只是要把用户目录中与该文件对应的目录项删除，且索引节点的count值减1
- 若count>0，说明还有别的用户要使用该文件，暂时不能把文件数据删除，否则会导致指针悬空
- 当count=0时系统负责删除文件

#### 特点
- 多个目录项指向同一个索引节点
- 删除文件时只是count减1，文件不删除
- 所有共享用户看到的是同一个文件

### 2.2 基于符号链的共享方式（软链接）

#### 基本原理
- 创建一个Link类型的新文件
- 文件中记录了被共享文件的存放路径
- 类似于Windows操作系统的"快捷方式"

#### 访问过程
- 当用户访问Link类型文件时，操作系统判断文件属于Link类型文件
- 于是会根据其中记录的路径层层查找目录
- 最终找到被共享文件的索引节点

#### 特点
- Link类型的文件名可以不同
- 删除原文件后，Link文件依然存在，只是通过路径已经找不到原文件了
- 可以跨文件系统共享

### 2.3 硬链接与软链接的比较

| 特性 | 硬链接 | 软链接 |
|------|--------|--------|
| 实现方式 | 多个目录项指向同一索引节点 | 创建Link类型文件记录路径 |
| 删除原文件 | 只是count减1，文件不删除 | 原文件删除后，链接失效 |
| 跨文件系统 | 不可以 | 可以 |
| 访问速度 | 直接访问，速度快 | 需要根据路径查找，速度慢 |
| 存储空间 | 不占用额外空间 | 需要占用额外空间存放Link文件 |

## 三、深入理解

### 3.1 文件共享的设计思想
1. **索引节点的作用**：通过索引节点实现文件的物理地址共享
2. **链接计数的作用**：通过count变量管理文件的删除时机
3. **路径的作用**：通过路径实现文件的逻辑共享

### 3.2 文件共享的实现细节

#### 硬链接的实现
1. 创建硬链接时，在目录中创建新的目录项
2. 新目录项指向被共享文件的索引节点
3. 索引节点的count加1
4. 删除硬链接时，目录项删除，count减1
5. 当count=0时，删除文件

#### 软链接的实现
1. 创建软链接时，创建新的Link类型文件
2. Link文件中记录被共享文件的路径
3. 访问软链接时，根据路径查找被共享文件
4. 删除软链接时，只是删除Link文件
5. 删除原文件后，软链接失效

### 3.3 文件共享的安全性
- **硬链接**：所有共享用户都有相同的访问权限
- **软链接**：可以为不同用户设置不同的访问权限

## 五、典型例题

### 概念题
**题目**：简述硬链接和软链接的区别。

**答案**：
**硬链接**：
- 多个目录项指向同一个索引节点
- 删除文件时只是count减1，文件不删除
- 所有共享用户看到的是同一个文件
- 不可以跨文件系统

**软链接**：
- 创建Link类型文件记录路径
- 删除原文件后，链接失效
- 可以跨文件系统
- 访问速度较慢

### 分析题
**题目**：说明基于索引节点的文件共享方式中，删除文件的过程。

**答案**：
删除文件的过程：
1. 若某个用户决定"删除"该文件，则只是要把用户目录中与该文件对应的目录项删除
2. 索引节点的count值减1
3. 若count>0，说明还有别的用户要使用该文件，暂时不能把文件数据删除
4. 当count=0时系统负责删除文件

### 真题
**题目**（2021年408真题）：下列关于文件共享的叙述中，正确的是（）
A. 硬链接可以跨文件系统共享文件
B. 软链接不能跨文件系统共享文件
C. 硬链接中删除原文件后，链接失效
D. 软链接中删除原文件后，链接失效

**答案**：D
**解析**：
- 硬链接不能跨文件系统，A错误
- 软链接可以跨文件系统，B错误
- 硬链接中删除原文件后，只是count减1，文件不删除，C错误
- 软链接中删除原文件后，链接失效，D正确

## 六、易错点

1. **硬链接与软链接的区别**
   - 硬链接：多个目录项指向同一索引节点
   - 软链接：创建Link类型文件记录路径

2. **删除文件时的处理**
   - 硬链接：只是count减1，文件不删除
   - 软链接：原文件删除后，链接失效

3. **跨文件系统共享**
   - 硬链接：不可以
   - 软链接：可以

4. **访问速度**
   - 硬链接：直接访问，速度快
   - 软链接：需要根据路径查找，速度慢
