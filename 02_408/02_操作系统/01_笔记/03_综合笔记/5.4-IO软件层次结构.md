# 5.4 I/O 软件层次结构

## 一、总体架构

```
I/O 请求
    ↓
┌───────────────────────┐
│     用户层软件          │ ← 最接近用户
├───────────────────────┤
│   设备独立性软件        │
│   （设备无关性软件）     │  ┐
├───────────────────────┤  │ I/O 核心子系统
│    设备驱动程序         │  │ （操作系统内核部分）
├───────────────────────┤  │
│    中断处理程序         │  ┘
├───────────────────────┤
│       硬件             │ ← 最接近硬件
└───────────────────────┘
    ↑
I/O 应答
```

**封装思想**：每一层利用下层提供的服务，实现某些功能，屏蔽实现细节，向高层提供服务。

---

## 二、用户层软件

### 功能

- 实现与**用户交互的接口**
- 提供方便易用的**库函数**（如 `printf("hello")`）
- 将用户请求翻译成格式化的 I/O 请求
- 通过**系统调用**请求操作系统内核的服务

### 示例

```c
printf("hello, world!");
// 被翻译为等价的 write 系统调用
```

---

## 三、设备独立性软件（设备无关性软件）

### 概念

与设备的**硬件特性无关**的功能几乎都在这一层实现。

### 主要功能

| 功能 | 说明 |
|------|------|
| ① 统一调用接口 | 向上层提供 read/write 系统调用 |
| ② 设备保护 | 设备被视为特殊文件，不同用户有不同的访问权限 |
| ③ 差错处理 | 处理设备错误 |
| ④ 设备分配与回收 | 管理设备的分配和回收 |
| ⑤ 数据缓冲区管理 | 通过缓冲技术屏蔽设备间数据交换差异 |
| ⑥ 逻辑设备名映射 | 建立逻辑设备名→物理设备名的映射，选择驱动程序 |

### 逻辑设备表（LUT）

```
逻辑设备名        物理设备名    驱动程序入口地址
/dev/打印机1      3            1024
/dev/打印机2      5            2046
```

**两种管理方式**：

| 方式 | 适用场景 | 说明 |
|------|---------|------|
| 整个系统一张 LUT | 单用户 OS | 所有用户不能使用相同的逻辑设备名 |
| 每个用户一张 LUT | 多用户 OS | 各用户逻辑设备名可重复，LUT 存于用户管理进程的 PCB |

---

## 四、设备驱动程序

### 功能

- 对**硬件设备的具体控制**
- 将上层命令转化成特定设备**"能听得懂"**的操作
- 包括设置设备寄存器、检查设备状态等

### 为什么需要不同的驱动程序？

不同设备的内部硬件特性不同：
- **佳能打印机**：状态寄存器 0=空闲，1=忙碌；有两个数据寄存器
- **惠普打印机**：状态寄存器 1=空闲，0=忙碌；有一个数据寄存器

> 厂家须提供与设备相对应的驱动程序。驱动程序一般以**独立进程**的方式存在。

---

## 五、中断处理程序

### 工作流程

```
I/O 完成 → 中断信号
    ↓
从控制器读出设备状态
    ↓
┌─────┴─────┐
│ I/O 正常？  │
├─ 是 ─────┤─ 否 ──────────┐
│ 从设备读入  │ 根据异常原因    │
│ 一个字的数据│ 做相应处理      │
│ 放入内存    │               │
└──────────┘───────────────┘
```

---

## 六、I/O 请求的完整流程

```
用户调用库函数（printf）
    ↓ 用户层软件
翻译为系统调用（write）
    ↓ 设备独立性软件
通过 LUT 找到驱动程序
    ↓ 设备驱动程序
向 I/O 控制器发出命令
    ↓ 硬件
设备执行 I/O 操作
    ↓ 完成后
中断信号 → 中断处理程序
    ↓
返回结果给用户
```

---

## 七、层次归属判断

| 操作 | 所在层次 |
|------|---------|
| 直接涉及硬件细节（设置寄存器） | **设备驱动程序** |
| 对各种设备都需要的管理工作 | **设备独立性软件** |
| 与中断相关 | **中断处理程序** |
| 库函数调用 | **用户层软件** |

## 考点总结

> 1. I/O 软件**五个层次的顺序**（必须记住）
> 2. **设备独立性软件**的功能（特别是 LUT）
> 3. **设备驱动程序**的功能和必要性
> 4. 判断某个操作在哪一层完成
> 5. I/O 请求的完整流程
