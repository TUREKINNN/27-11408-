# 2.1.1 进程的概念、组成、特征

## 📌 知识框架

```
进程的概念、组成、特征
├── 进程的概念
├── 进程的组成
│   ├── PCB（进程控制块）
│   ├── 程序段
│   └── 数据段
└── 进程的特征
```

---

## 一、进程的概念

### 1. 程序与进程的区别

| 对比项 | 程序 | 进程 |
|--------|------|------|
| 本质 | **静态**的，存放在磁盘里的可执行文件 | **动态**的，程序的一次执行过程 |
| 示例 | QQ.exe 文件 | 可同时启动多次QQ程序，每次启动对应一个进程 |
| 关系 | 一个程序 | 同一个程序多次执行会对应**多个进程** |

### 2. 进程的正式定义

> **进程**是进程实体的运行过程，是系统进行**资源分配**和**调度**的一个独立单位。

- **进程实体（进程映像）**：由 PCB、程序段、数据段三部分组成，是**静态**的
- **进程**：是进程实体的**运行过程**，是**动态**的
- 进程实体反映了进程在某一时刻的状态（如：x++后，x=2）

---

## 二、进程的组成

### 1. PCB（Process Control Block，进程控制块）

**PCB 是进程存在的唯一标志！** 当进程被创建时，操作系统为其创建 PCB，当进程结束时，会回收其 PCB。

PCB 中包含的信息：

| 信息类别 | 具体内容 | 作用 |
|----------|----------|------|
| 进程描述信息 | PID（进程ID）、UID（用户ID） | 让操作系统**区分各个进程** |
| 进程控制和管理信息 | 进程当前状态、优先级 | 实现操作系统对进程的**管理** |
| 资源分配清单 | 内存使用情况、I/O设备、打开的文件 | 实现操作系统对**资源的管理** |
| 处理机相关信息 | 程序计数器(PC)、通用寄存器、PSW等 | 实现进程**上下文切换** |

### 2. 程序段

- 包含程序的**指令**（机器指令序列）
- 是给**进程自己**用的
- 同一个程序的多个进程共享相同的程序段内容

### 3. 数据段

- 包含运行过程中产生的各种**数据**
- 是给**进程自己**用的
- 与进程自身的运行逻辑有关
- 不同进程的数据段各不相同

### 4. 进程的组成总结

```
进程实体（进程映像）
├── PCB ─────── 给操作系统用的（管理者所需的信息）
├── 程序段 ──── 给进程自己用的（包含指令）
└── 数据段 ──── 给进程自己用的（包含数据）
```

> 💡 更确切地说，应该是"进程实体（进程映像）的组成"

---

## 三、进程的特征

相比于程序（静态），进程拥有以下**五大特征**：

| 特征 | 说明 | 重要程度 |
|------|------|----------|
| **动态性** | 进程最基本的特征。进程是程序的一次执行过程，是动态地产生、变化和消亡的 | ⭐⭐⭐ 最基本 |
| **并发性** | 多个进程实体同存于内存中，能在一段时间内同时运行 | ⭐⭐ |
| **独立性** | 进程是系统进行资源分配和调度的**独立单位** | ⭐⭐ |
| **异步性** | 各进程以各自独立的、不可预知的速度向前推进 | ⭐⭐⭐ 导致结果不确定性 |
| **结构性** | 每个进程都配置一个PCB，从结构上看，进程实体由程序段、数据段和PCB组成 | ⭐ |

## 四、程序是如何运行的？

### 执行过程

```
C语言代码  ──编译器──>  机器指令（二进制）
Int x = 1;              指令1
x++;                    指令2
……                      指令3
                        …..
```

> 一条高级语言的代码翻译过来可能会对应**多条机器指令**

### 运行时的内存布局

```
┌─────────────────────────────────────┐
│              内存                     │
│  ┌───────────────────────────────┐  │
│  │            PCB                 │  │
│  │  PC: 下一条指令地址             │  │
│  │  IR: 当前正在执行的指令          │  │
│  │  PSW: 程序状态字寄存器           │  │
│  │  通用寄存器: 运行中间结果         │  │
│  ├───────────────────────────────┤  │
│  │          程序段                 │  │
│  │  指令1, 指令2, 指令3, ...       │  │
│  ├───────────────────────────────┤  │
│  │          数据段                 │  │
│  │  x=1, ...                     │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 关键寄存器

| 寄存器 | 全称 | 作用 |
|--------|------|------|
| **PC** | 程序计数器 | 存放**下一条**指令的地址 |
| **IR** | 指令寄存器 | 存放当前**正在执行**的指令 |
| **PSW** | 程序状态字寄存器 | 存放程序状态信息 |
| 通用寄存器 | - | 存放运行过程中的中间结果 |

---

## 📝 考点总结

1. **PCB 是进程存在的唯一标志**（超高频考点）
2. 进程实体（进程映像）的组成：PCB + 程序段 + 数据段
3. **动态性**是进程最基本的特征
4. **异步性**会导致并发程序执行结果的不确定性
5. 进程是系统进行**资源分配**和**调度**的独立单位
6. 引入线程后，进程只作为除CPU之外的系统资源的分配单元
