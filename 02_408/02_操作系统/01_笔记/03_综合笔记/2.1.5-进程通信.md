# 2.1.5 进程通信

## 📌 知识框架

```
进程通信（IPC）
├── 共享存储
│   ├── 基于数据结构的共享（低级）
│   └── 基于存储区的共享（高级）
├── 消息传递
│   ├── 直接通信方式
│   └── 间接通信方式（信箱通信）
└── 管道通信
```

---

## 一、什么是进程间通信？

**进程间通信（IPC, Inter-Process Communication）** 是指两个进程之间产生**数据交互**。

### 为什么需要操作系统支持？

进程是分配系统资源的单位（包括内存地址空间），各进程拥有的内存地址空间**相互独立**。

> 为了保证安全，一个进程**不能直接访问**另一个进程的地址空间。

因此，进程通信需要操作系统提供支持。

---

## 二、共享存储

操作系统在内存中划出一块**共享存储区**，两个进程可以通过这片共享区域交换数据。

```
进程P地址空间    共享存储区    进程Q地址空间
    ┌─────┐    ┌─────────┐    ┌─────┐
    │     │    │         │    │     │
    │  P  │◄──►│  共享区  │◄──►│  Q  │
    │     │    │         │    │     │
    └─────┘    └─────────┘    └─────┘
```

### 两种共享方式

| 方式 | 说明 | 特点 |
|------|------|------|
| **基于数据结构的共享** | 共享空间里只能放特定数据结构（如长度为10的数组） | 速度慢、限制多，**低级通信** |
| **基于存储区的共享** | 数据的形式、存放位置由通信进程控制，不是OS | 速度快，**高级通信** |

### 注意事项
- 各进程对共享空间的访问应该是**互斥**的
- 可以使用 **P、V操作** 实现同步互斥
- 通过"增加页表项/段表项"可将同一片共享内存映射到各进程的地址空间

---

## 三、消息传递

进程间的数据交换以**格式化的消息（Message）**为单位。通过OS提供的**发送消息/接收消息**原语进行数据交换。

### 消息的格式

```
┌────────────────┬──────────────┐
│    消息头        │    消息体     │
│  - 发送进程ID    │              │
│  - 接收进程ID    │   实际数据    │
│  - 消息长度      │              │
└────────────────┴──────────────┘
```

### 1. 直接通信方式

发送方**指名道姓**地发送消息给接收方：

```
进程P                          进程Q
  │                              │
  │  send(Q, msg)                │
  │ ─────────────────────────►  │
  │                    消息存入Q的 │
  │                    消息队列    │
  │                              │
  │                    receive(P, &msg)
  │ ◄─────────────────────────  │
```

### 2. 间接通信方式（信箱通信）

以**"信箱"**作为中间实体进行消息传递：

```
进程P         信箱A         进程Q
  │            ┌───┐          │
  │ send(A,msg)│   │          │
  │ ──────────►│   │          │
  │            │   │ receive(A,&msg)
  │            │   │──────────►│
  │            └───┘          │
```

- 可以多个进程往同一个信箱send消息
- 也可以多个进程从同一个信箱中receive消息

## 四、管道通信

"管道"是一个特殊的**共享文件**，又名pipe文件。在内存中开辟一个大小固定的**内存缓冲区**。

```
进程P ──写数据──► [管道（内存缓冲区）] ──读数据──► 进程Q
```

### 管道通信的5个关键规则

| 编号 | 规则 |
|------|------|
| ① | 管道只能采用**半双工**通信，某一时间段内只能**单向**传输。双向通信需设置**两个管道** |
| ② | 各进程要**互斥**地访问管道（由操作系统实现） |
| ③ | 管道**写满**时，写进程**阻塞**，直到读进程取走数据后唤醒写进程 |
| ④ | 管道**读空**时，读进程**阻塞**，直到写进程写入数据后唤醒读进程 |
| ⑤ | 数据一旦被读出就**彻底消失**。一个管道允许多个写进程，但只允许**一个读进程**（408真题官方答案） |

### 补充说明
- 写进程往管道写数据，即使管道没被写满，只要管道**没空**，读进程就可以从管道读数据
- 读进程从管道读数据，即使管道没被读空，只要管道**没满**，写进程就可以往管道写数据

---

## 📝 考点总结

1. **共享存储**：需要互斥访问，用P、V操作实现
2. **消息传递**：分为直接通信和间接通信（信箱）
3. **管道通信**：半双工、互斥访问、写满阻塞写者/读空阻塞读者
4. 管道中数据**读出即消失**
5. 2014年408真题：一个管道允许多个写进程，**一个读进程**
