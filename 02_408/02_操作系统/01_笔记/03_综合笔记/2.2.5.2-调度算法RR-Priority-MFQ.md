# 2.2.5 调度算法：时间片轮转、优先级、多级反馈队列

## 📌 知识框架

```
调度算法（适用于交互式系统）
├── 时间片轮转（RR）
├── 优先级调度
└── 多级反馈队列
```

> 💡 这三种算法适合用于**交互式系统**（分时操作系统、实时操作系统），更注重**响应时间**、**公平性**、**平衡性**

---

## 一、时间片轮转（RR, Round-Robin）

### 算法思想
公平地、轮流地为各个进程服务，让每个进程在一定时间间隔内都可以得到响应

### 算法规则
1. 按各进程到达就绪队列的顺序，轮流让各进程执行一个**时间片**（如100ms）
2. 若进程未在一个时间片内执行完，则**剥夺处理机**，将进程重新放到**就绪队列队尾**
3. 由**时钟装置**发出时钟中断来通知CPU时间片已到

### 是否可抢占？
**抢占式**

### 例题推导（时间片q=2）

| 进程 | 到达时间 | 运行时间 |
|------|---------|---------|
| P1   | 0       | 5       |
| P2   | 2       | 4       |
| P3   | 4       | 1       |
| P4   | 5       | 6       |

```
|P1|P2|P1|P3|P2|P4|P1|P4| P4|
0  2  4  6  7  9  11 12 14 16
```

详细过程（括号内为就绪队列和剩余时间）：

| 时刻 | 事件 | 就绪队列 |
|------|------|---------|
| 0    | P1(5)到达，P1运行 | P1(5) |
| 2    | P2(4)到达，P1时间片到 | P2(4)→P1(3) |
| 4    | P3(1)到达，P2时间片到 | P1(3)→P3(1)→P2(2) |
| 5    | P4(6)到达 | P3(1)→P2(2)→P4(6)，P1在运行 |
| 6    | P1时间片到 | P3(1)→P2(2)→P4(6)→P1(1) |
| 7    | P3运行完（1<2，主动释放） | P2(2)→P4(6)→P1(1) |
| 9    | P2时间片到且运行完 | P4(6)→P1(1) |
| 11   | P4时间片到 | P1(1)→P4(4) |
| 12   | P1运行完 | P4(4) |
| 14   | P4时间片到，就绪队列空 | P4(2) 继续运行 |
| 16   | 全部完成 | - |

### 时间片大小的影响

| 时间片大小 | 影响 |
|-----------|------|
| **太大** | 退化为**FCFS**，增大响应时间 |
| **太小** | 进程切换过于频繁，系统开销大，实际执行时间比例减少 |
| **合适** | 兼顾响应时间和系统效率 |

## 二、优先级调度

### 算法思想
根据任务的**紧急程度**来决定处理顺序

### 算法规则
调度时选择**优先级最高**的作业/进程

### 是否可抢占？
**抢占式和非抢占式都有**

### 优先级类型

| 类型 | 说明 |
|------|------|
| **静态优先级** | 创建时确定，运行期间不变 |
| **动态优先级** | 运行期间可调整 |

### 优先级设置的一般规则

| 进程类型 | 优先级设置 |
|----------|-----------|
| 系统进程 > 用户进程 | 系统进程优先级更高 |
| 交互型 > 非交互型 | 交互型进程优先级更高 |
| I/O型 > 计算型 | I/O型进程优先级更高（I/O设备速度慢，应优先处理） |

### 是否导致饥饿？
**会**：若源源不断地有高优先级进程到来，低优先级进程将永远得不到服务

## 三、多级反馈队列调度算法

### 算法思想
对其他调度算法的**折中权衡**

### 算法规则

```
优先级高 ──→ 第1级队列（时间片=1）
              ┌─────────────────┐
              │ Q1: 时间片=1     │
              └────────┬────────┘
                       │ 时间片用完未完成
                       ▼
              ┌─────────────────┐
              │ Q2: 时间片=2     │
              └────────┬────────┘
                       │ 时间片用完未完成
优先级低 ──→          ▼
              ┌─────────────────┐
              │ Q3: 时间片=4     │
              └─────────────────┘
```

**规则：**
1. 设置**多级就绪队列**，各级队列**优先级从高到低**，**时间片从小到大**
2. 新进程到达时先进入**第1级队列**，按FCFS排队等待被分配时间片
3. 若用完时间片进程还未结束，则进入**下一级队列队尾**
4. 如果已在最下级队列，则重新放回**该队列队尾**
5. 只有第k级队列**为空**时，才会为k+1级队头的进程分配时间片

### 是否可抢占？
**抢占式**：在k级队列的进程运行过程中，若更上级队列（1~k-1级）中进入了一个新进程，则新进程**抢占**处理机，原进程放回k级队列队尾。

### 例题推导

| 进程 | 到达时间 | 运行时间 |
|------|---------|---------|
| P1   | 0       | 8       |
| P2   | 1       | 4       |
| P3   | 5       | 1       |

设置3级队列：Q1(时间片=1)、Q2(时间片=2)、Q3(时间片=4)

```
P1(1)→P2(1)→P1(2)→P2(1)→P3(1)→P2(2)→P1(4)→P1(1)
```

### 优缺点

| 优点 | 说明 |
|------|------|
| 公平 | 对各类型进程相对公平（FCFS的优点） |
| 响应快 | 每个新到达的进程都可以很快得到响应（RR的优点） |
| 短作业优先 | 短进程只需较少时间就可完成（SPF的优点） |
| 无需预估 | 不必事先估计进程的运行时间 |
| 灵活 | 可灵活调整对各类进程的偏好 |

> 🎯 **UNIX操作系统使用的就是多级反馈队列调度算法**

---

## 四、三种算法对比总结

| 算法 | 可抢占 | 优点 | 缺点 | 饥饿 |
|------|--------|------|------|------|
| **RR** | 是 | 公平，适用于分时系统 | 频繁切换有开销，不区分优先级 | 不会 |
| **优先级** | 有抢占/非抢占 | 区分紧急程度，适用于实时系统 | 可能导致饥饿 | **会** |
| **多级反馈队列** | 是 | 综合表现优秀 | 较复杂 | **会** |

---

## 📝 考点总结

1. 时间片轮转：时间片太大退化为FCFS，太小切换开销大
2. 优先级调度：注意抢占式和非抢占式的区别
3. 多级反馈队列：综合了FCFS、RR、SPF的优点
4. 这三种算法适用于**交互式系统**
