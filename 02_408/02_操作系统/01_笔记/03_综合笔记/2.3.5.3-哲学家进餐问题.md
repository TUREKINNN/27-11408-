# 2.3.5.3 哲学家进餐问题

## 📌 问题描述

一张圆桌上坐着**5名哲学家**，每两个哲学家之间的桌上摆一根**筷子**，桌子的中间是一碗米饭。

- 哲学家们倾注毕生的精力用于**思考和进餐**
- 哲学家在思考时，并不影响他人
- 只有当哲学家**饥饿**时，才试图拿起**左、右两根筷子**（一根一根地拿起）
- 如果筷子已在他人手上，则需等待
- 饥饿的哲学家只有**同时拿起两根筷子**才可以开始进餐
- 当进餐完毕后，放下筷子继续思考

---

## 一、关系分析

### 1.1 互斥关系

系统中有**5个哲学家进程**，5位哲学家与左右邻居对其中间筷子的访问是**互斥关系**。

**筷子编号规则：**
- 哲学家按0~4编号
- 哲学家 `i` 左边的筷子编号为 `i`
- 哲学家 `i` 右边的筷子编号为 `(i+1)%5`

```
        哲学家0
        /    \
  筷子0/      \筷子4
      /        \
哲学家4          哲学家1
      \        /
  筷子3\      /筷子1
        \    /
    哲学家3—筷子2—哲学家2
```

### 1.2 问题特殊性

> 这个问题中只有互斥关系，但与之前遇到的问题不同的是：每个哲学家进程需要**同时持有两个临界资源**才能开始吃饭。如何避免临界资源分配不当造成的**死锁**现象，是哲学家问题的精髓。

---

## 二、方案一：朴素实现（存在死锁） ⭐⭐⭐

### 2.1 信号量设置

```c
semaphore chopstick[5] = {1, 1, 1, 1, 1};  // 5根筷子
```

### 2.2 朴素实现

```c
// i号哲学家的进程
Pi() {
    while (1) {
        P(chopstick[i]);         // 拿左
        P(chopstick[(i+1)%5]);   // 拿右
        
        吃饭...;
        
        V(chopstick[i]);         // 放左
        V(chopstick[(i+1)%5]);   // 放右
        
        思考...;
    }
}
```

### 2.3 死锁分析

如果5个哲学家**并发地**拿起了自己左手边的筷子：
- 哲学家0拿到了筷子0
- 哲学家1拿到了筷子1
- 哲学家2拿到了筷子2
- 哲学家3拿到了筷子3
- 哲学家4拿到了筷子4

此时：
- 每位哲学家都在等待自己**右边的人放下筷子**（阻塞）
- 这些哲学家进程都因等待筷子资源而被阻塞
- **发生"死锁"！**

---

## 三、解决方案

### 方案一：限制同时进餐的哲学家数量 ⭐⭐⭐

**思想：** 最多允许**四个**哲学家同时进餐。这样可以保证至少有一个哲学家是可以拿到左右两只筷子的。

```c
semaphore chopstick[5] = {1, 1, 1, 1, 1};
semaphore mutex = 1;  // 限制最多4个哲学家同时进餐

Pi() {
    while (1) {
        P(mutex);                // 限制同时进餐人数
        P(chopstick[i]);         // 拿左
        P(chopstick[(i+1)%5]);   // 拿右
        V(mutex);                // 释放限制
        
        吃饭...;
        
        V(chopstick[i]);         // 放左
        V(chopstick[(i+1)%5]);   // 放右
        
        思考...;
    }
}
```

**分析：** `mutex` 初值为4，最多4个哲学家同时尝试拿筷子。由于只有5根筷子，4个哲学家中至少有一个能拿到两根筷子。

### 方案二：奇偶号哲学家拿筷子顺序不同 ⭐⭐⭐

**思想：** 
- **奇数号**哲学家：先拿**左边**的筷子，再拿**右边**的筷子
- **偶数号**哲学家：先拿**右边**的筷子，再拿**左边**的筷子

```c
semaphore chopstick[5] = {1, 1, 1, 1, 1};

Pi() {
    while (1) {
        if (i % 2 == 1) {           // 奇数号哲学家
            P(chopstick[i]);         // 先拿左
            P(chopstick[(i+1)%5]);   // 再拿右
        } else {                     // 偶数号哲学家
            P(chopstick[(i+1)%5]);   // 先拿右
            P(chopstick[i]);         // 再拿左
        }
        
        吃饭...;
        
        V(chopstick[i]);             // 放左
        V(chopstick[(i+1)%5]);       // 放右
        
        思考...;
    }
}
```

**分析：** 如果相邻的两个奇偶号哲学家都想吃饭，那么只有一个可以拿起第一只筷子，另一个会直接阻塞。避免了占有一支后再等待另一只的情况。

### 方案三：用互斥信号量限制拿起筷子的操作 ⭐⭐⭐

**思想：** 仅当一个哲学家**左右两支筷子都可用**时才允许他抓起筷子。

```c
semaphore chopstick[5] = {1, 1, 1, 1, 1};
semaphore mutex = 1;  // 互斥地取筷子

// i号哲学家的进程
Pi() {
    while (1) {
        P(mutex);                // 互斥地拿起两根筷子
        P(chopstick[i]);         // 拿左
        P(chopstick[(i+1)%5]);   // 拿右
        V(mutex);                // 释放互斥
        
        吃饭...;
        
        V(chopstick[i]);         // 放左
        V(chopstick[(i+1)%5]);   // 放右
        
        思考...;
    }
}
```

**分析：** 用 `mutex` 信号量保证每个哲学家在拿起筷子时是原子操作，避免了同时拿起一根筷子后等待另一根的情况。

---

## 四、方案对比

| 方案 | 思想 | 优点 | 缺点 |
|------|------|------|------|
| 限制进餐人数 | 最多4人同时进餐 | 实现简单 | 并发度降低 |
| 奇偶号不同顺序 | 破坏循环等待 | 并发度较高 | 实现较复杂 |
| 互斥取筷子 | 保证原子性 | 简单有效 | 并发度最低 |

---

## 五、核心考点总结

### 5.1 哲学家进餐问题的本质

> 这是一个**互斥问题**，但与普通互斥问题不同的是：每个进程需要**同时持有多个临界资源**。如何避免**死锁**是这个问题的核心。

### 5.2 死锁的避免方法

1. **限制并发数**：通过信号量限制同时竞争资源的进程数
2. **破坏循环等待**：改变资源申请顺序
3. **保证原子性**：用互斥信号量保证同时获取多个资源的操作是原子的

### 5.3 考试应用

> 哲学家进餐问题的思想可以应用到任何需要**同时持有多个临界资源**的场景。关键是要识别出可能导致死锁的情况，并选择合适的解决方案。

---

## 💡 易错点提醒

1. **朴素实现会导致死锁**，考试中不能直接使用
2. 筷子编号规则：哲学家 `i` 的左边是 `i`，右边是 `(i+1)%5`
3. 三种解决方案各有优缺点，考试中要根据题目要求选择
4. **死锁是哲学家进餐问题的核心考点**
5. 方案三（互斥取筷子）的并发度最低，但实现最简单
6. 考试中常考的是**方案二（奇偶号不同顺序）**
