## 2. I/O控制方式

### 2.1 程序直接控制方式

```
CPU循环检测设备状态
```

**优点**：简单。

**缺点**：CPU利用率低。

### 2.2 中断驱动方式

```
设备完成后发中断通知CPU
```

**优点**：CPU利用率高。

**缺点**：中断开销大。

### 2.3 DMA方式

<span style="background-color:#ccc1d9">**DMA**</span>：直接内存访问。

```
DMA控制器直接在设备和内存间传输数据
```

**优点**：CPU介入少。

**缺点**：需要DMA控制器。

### 2.4 通道方式

<span style="background-color:#ccc1d9">**通道**</span>：专用处理器控制I/O。

```
通道执行通道程序控制I/O
```

**优点**：CPU完全解放。

**缺点**：成本高。

### 2.5 方式比较

| 方式 | CPU介入 | 效率 | 成本 |
|------|---------|------|------|
| **程序直接控制** | 高 | 低 | 低 |
| **中断驱动** | 中 | 中 | 低 |
| **DMA** | 低 | 高 | 中 |
| **通道** | 无 | 高 | 高 |

### 3.1 设备驱动程序的作用

<span style="background-color:#ccc1d9">**设备驱动程序**</span>：操作系统与设备之间的接口。

## 4. 设备分配

### 4.1 设备分配的原则

| 原则 | 描述 |
|------|------|
| **安全性** | 避免死锁 |
| **公平性** | 公平分配 |
| **效率** | 提高利用率 |

### 4.2 设备分配的数据结构

| 结构 | 描述 |
|------|------|
| **设备控制表（DCT）** | 记录设备状态 |
| **控制器控制表（COCT）** | 记录控制器状态 |
| **通道控制表（CHCT）** | 记录通道状态 |
| **系统设备表（SDT）** | 记录所有设备 |

### 4.3 设备分配的过程

```
1. 查SDT，找到设备DCT
2. 查DCT，确认设备空闲
3. 分配设备，更新DCT
4. 查COCT，分配控制器
5. 查CHCT，分配通道（可选）
6. 返回设备句柄
```

## 5. I/O调度

### 5.1 I/O调度的目的

**目的**：优化I/O请求的执行顺序。

### 5.2 I/O调度算法

#### 5.2.1 先来先服务（FCFS）

```
按请求顺序处理
```

**优点**：简单，公平。

**缺点**：效率低。

#### 5.2.2 最短寻道时间优先（SSTF）

```
优先处理最近的请求
```

**优点**：寻道时间短。

**缺点**：可能导致饥饿。

#### 5.2.3 扫描算法（SCAN）

```
从一端扫描到另一端
```

**优点**：公平，效率高。

#### 5.2.4 循环扫描算法（C-SCAN）

```
从一端扫描到另一端，直接返回起点
```

**优点**：均匀响应。

## 6. 缓冲管理

### 6.1 缓冲的作用

<span style="background-color:#ccc1d9">**缓冲**</span>：在内存中开辟的临时存储区域。

**作用**：

1. 缓和CPU与I/O设备的速度差异
2. 减少中断次数
3. 提高CPU和I/O设备的并行性

### 6.2 缓冲的类型

| 类型 | 描述 |
|------|------|
| **单缓冲** | 一个缓冲区 |
| **双缓冲** | 两个缓冲区交替使用 |
| **循环缓冲** | 多个缓冲区循环使用 |
| **缓冲池** | 多个缓冲区统一管理 |

### 6.3 缓冲池的管理

**组成**：

| 队列 | 描述 |
|------|------|
| **空缓冲队列** | 空闲缓冲区 |
| **输入缓冲队列** | 装满输入数据 |
| **输出缓冲队列** | 装满输出数据 |

**操作**：

| 操作 | 描述 |
|------|------|
| **GetBuf** | 从队列取缓冲区 |
| **PutBuf** | 放回缓冲区到队列 |

## 8. 典型例题

### 例题 1：I/O控制方式

**题目**：比较程序直接控制、中断驱动、DMA和通道四种I/O控制方式的特点。

**解答**：

```
程序直接控制：
- CPU循环检测设备状态
- CPU利用率低
- 简单，成本低

中断驱动：
- 设备完成后发中断
- CPU利用率较高
- 中断开销大

DMA：
- DMA控制器直接传输数据
- CPU介入少
- 需要DMA控制器

通道：
- 专用处理器控制I/O
- CPU完全解放
- 成本高
```

**总结**：不同I/O控制方式适用于不同场景。

**来源**：408历年真题，考查方式：简答题

---

### 例题 2：设备分配

**题目**：描述设备分配的过程。

**解答**：

```
1. 查系统设备表（SDT），找到设备对应的设备控制表（DCT）
2. 检查DCT，确认设备是否空闲
3. 若空闲，分配设备，更新DCT状态为忙
4. 查控制器控制表（COCT），分配空闲控制器
5. 若需要通道，查通道控制表（CHCT），分配空闲通道
6. 返回设备句柄给进程
```

**总结**：设备分配需要分配设备、控制器和通道。

**来源**：408历年真题，考查方式：简答题

---

### 例题 3：磁盘性能计算

**题目**：某磁盘转速为7200转/分，平均寻道时间为8ms，求平均访问时间。

**解答**：

```
转速 = 7200转/分 = 120转/秒

平均旋转延迟 = 1/(2 × 120) = 1/240 ≈ 4.17ms

假设传输时间忽略不计：

平均访问时间 = 平均寻道时间 + 平均旋转延迟
             = 8ms + 4.17ms ≈ 12.17ms
```

**总结**：磁盘访问时间主要由寻道时间和旋转延迟组成。

**来源**：408历年真题，考查方式：计算题
