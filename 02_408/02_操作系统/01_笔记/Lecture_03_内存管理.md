## 1. 内存管理的基本概念

### 1.1 内存管理的目的

<span style="background-color:#ccc1d9">**内存管理**</span>：合理分配和使用内存资源。

### 1.2 内存管理的功能

| 功能 | 描述 |
|------|------|
| **内存分配** | 分配内存空间 |
| **内存回收** | 回收不再使用的内存 |
| **内存保护** | 防止越界访问 |
| **内存共享** | 共享内存区域 |
| **内存扩充** | 虚拟内存技术 |

## 2. 分区管理

### 2.1 固定分区

**特点**：内存划分为固定大小的分区。

**优点**：简单。

**缺点**：浪费空间。

### 2.2 可变分区

**特点**：根据进程大小动态分配分区。

**分配算法**：

| 算法 | 描述 |
|------|------|
| **首次适应** | 从头开始找第一个足够大的分区 |
| **最佳适应** | 找最小的足够大的分区 |
| **最坏适应** | 找最大的分区 |

**回收算法**：

1. 上邻接空闲区：合并
2. 下邻接空闲区：合并
3. 上下都邻接：合并为一个

### 2.3 分区管理的优缺点

**优点**：简单，无需硬件支持。

**缺点**：存在碎片问题。

## 3. 分页管理

### 3.1 分页的基本概念

<span style="background-color:#ccc1d9">**分页**</span>：将内存划分为固定大小的页框，程序划分为同样大小的页。

### 3.2 页表

**定义**：记录页号到页框号的映射关系。

**页表项**：

```
页表项 = 页框号 + 有效位 + 修改位 + 访问位
```

### 3.3 地址变换

```
逻辑地址 = 页号 + 页内偏移

物理地址 = 页框号 × 页大小 + 页内偏移
```

### 3.4 快表（TLB）

**定义**：页表的Cache。

**工作原理**：

1. 先查快表
2. 命中：直接得到页框号
3. 未命中：查页表，更新快表

### 3.5 多级页表

**目的**：减少页表占用的内存。

**结构**：

```
一级页表 → 二级页表 → 页表项
```

### 3.6 分页管理的优缺点

**优点**：无外部碎片，内存利用率高。

**缺点**：有内部碎片，地址变换开销大。

---

## 4. 分段管理

### 4.1 分段的基本概念

<span style="background-color:#ccc1d9">**分段**</span>：将程序按逻辑结构划分为段。

### 4.2 段表

**定义**：记录段号到段地址的映射关系。

**段表项**：

```
段表项 = 段基址 + 段长度 + 保护位
```

### 4.3 地址变换

```
逻辑地址 = 段号 + 段内偏移

物理地址 = 段基址 + 段内偏移
```

### 4.4 分段管理的优缺点

**优点**：无内部碎片，便于共享和保护。

**缺点**：有外部碎片。

---

## 5. 段页式管理

### 5.1 段页式的基本概念

<span style="background-color:#ccc1d9">**段页式**</span>：分段和分页的结合。

### 5.2 段页式的地址结构

```
逻辑地址 = 段号 + 页号 + 页内偏移
```

### 5.3 地址变换过程

```
1. 根据段号查段表，得到页表基址
2. 根据页号查页表，得到页框号
3. 物理地址 = 页框号 × 页大小 + 页内偏移
```

### 5.4 段页式的优缺点

**优点**：结合分段和分页的优点。

**缺点**：地址变换开销大。

## 6. 虚拟内存

### 6.1 虚拟内存的概念

<span style="background-color:#ccc1d9">**虚拟内存**</span>：将主存和辅存统一管理，提供比实际主存大的地址空间。

### 6.2 虚拟内存的实现方式

| 方式 | 描述 |
|------|------|
| **分页虚拟内存** | 基于分页 |
| **分段虚拟内存** | 基于分段 |
| **段页式虚拟内存** | 基于段页式 |

### 6.3 页面置换算法

#### 6.3.1 最佳置换算法（OPT）

```
置换未来最久不使用的页
```

**优点**：缺页次数最少。

**缺点**：不可实现。

#### 6.3.2 先进先出算法（FIFO）

```
置换最先进入的页
```

**优点**：简单。

**缺点**：可能出现Belady异常。

#### 6.3.3 最近最少使用算法（LRU）

```
置换最近最少使用的页
```

**优点**：性能好。

**缺点**：实现复杂。

#### 6.3.4 时钟算法（Clock）

```
用位记录使用情况，循环检查
```

**优点**：简单，接近LRU。

**缺点**：性能略差于LRU。

### 6.4 页面分配策略

| 策略 | 描述 |
|------|------|
| **固定分配** | 每个进程分配固定页数 |
| **可变分配** | 根据需要动态分配 |
| **全局置换** | 从所有页中选择置换 |
| **局部置换** | 只从进程自己的页中选择置换 |

### 6.5 抖动（Thrashing）

**定义**：频繁的页面置换导致系统效率急剧下降。

**原因**：分配给进程的页框数不足。

**解决**：

1. 增加页框数
2. 减少进程数量

### 7.1 内存保护的目的

### 7.2 内存保护的方法

## 8. 典型例题

### 例题 1：分页地址变换

**题目**：某系统页面大小为4KB，页表项大小为4B，虚拟地址为123456H，页表起始地址为1000H。求物理地址。

**解答**：

```
页大小 = 4KB = 2^12

页内偏移 = 123456H & 0FFFh = 456H
虚拟页号 = 123456H >> 12 = 123H

页表项地址 = 1000H + 123H × 4 = 1000H + 49CH = 149CH

假设页表项内容为567H，则物理页号 = 567H

物理地址 = 567H × 4096 + 456H = 567456H
```

**总结**：分页地址变换需要两次内存访问。

**来源**：408历年真题，考查方式：计算题

---

### 例题 2：页面置换算法

**题目**：页面访问序列为1,2,3,4,1,2,5,1,2,3,4,5，分配3个页框。分别用FIFO和LRU计算缺页次数。

**解答**：

```
FIFO：
访问序列：1 2 3 4 1 2 5 1 2 3 4 5
页框1：   1 1 1 4 4 4 5 5 5 3 3 3
页框2：     2 2 2 1 1 1 1 1 1 4 4
页框3：       3 3 3 2 2 2 2 2 2 5
缺页：   × × × ×     ×       × ×

缺页次数：6次

LRU：
访问序列：1 2 3 4 1 2 5 1 2 3 4 5
页框1：   1 1 1 4 1 1 5 1 1 3 3 5
页框2：     2 2 2 2 2 2 2 2 2 4 4
页框3：       3 3 3 3 3 3 3 3 3 3
缺页：   × × × ×     ×     × ×

缺页次数：5次
```

**总结**：LRU的缺页次数少于FIFO。

**来源**：408历年真题，考查方式：计算题

### 例题 3：分段地址变换

**题目**：某分段系统中，段表如下：

```
段号 基址   长度
0    200H   100H
1    400H   200H
2    600H   150H
```

求逻辑地址(1, 0A0H)的物理地址。

**解答**：

```
段号 = 1
段内偏移 = 0A0H

检查段内偏移是否越界：
0A0H < 200H，未越界

物理地址 = 基址 + 段内偏移
         = 400H + 0A0H = 4A0H
```

**总结**：分段地址变换需要检查越界。

**来源**：408历年真题，考查方式：计算题
