## 1️⃣ 连续分配管理方式

### 📌 单一连续分配

```
┌──────────┐
│  OS      │
├──────────┤
│ 用户程序  │ ← 整个用户区只有一个程序
│          │
└──────────┘
```

> **特点**：简单，无外部碎片，有内部碎片  
> **适用**：单用户单任务 OS

### 📌 固定分区分配

```
┌──────────┐
│  OS      │
├──────────┤
│ 分区 1   │ ← 大小固定
├──────────┤
│ 分区 2   │
├──────────┤
│ 分区 3   │ ← 空闲
└──────────┘
```

> **特点**：分区大小固定，无外部碎片，有内部碎片

### 📌 动态分区分配（可变分区）

```
┌──────────┐
│  OS      │
├──────────┤
│ 分区 1   │ ← 已分配
├──────────┤
│ 空闲     │ ← 空闲区
├──────────┤
│ 分区 2   │ ← 已分配
├──────────┤
│ 空闲     │
└──────────┘
```

### 📊 4 大分配算法对比

| 算法 | 思想 | 优点 | 缺点 |
|------|------|------|------|
| **首次适应 (First Fit)** | 从头找第一个 ≥ 要求的空闲区 | 简单 / 快速 | 碎片集中在低地址 |
| **最佳适应 (Best Fit)** | 找最小的 ≥ 要求的空闲区 | 减少大块碎片 | 产生大量小碎片 |
| **最坏适应 (Worst Fit)** | 找最大的空闲区 | 保留大块空闲 | 大块被切碎 |
| **邻近适应 (Next Fit)** | 从上次结束位置继续找 | 均衡碎片 | 实现稍复杂 |

## 2️⃣ 基本分页存储管理

### 📌 核心思想

> **页面** = 进程的逻辑地址空间分成等大小的块  
> **页框** = 物理内存分成等大小的块（与页面大小相同）  
> **页表** = 记录页面 → 页框的映射

### 📊 地址结构（32 位为例）

```
┌──────────┬──────────┐
│  页号 P  │ 页内偏移 W │
│  20 位   │  12 位    │
└──────────┴──────────┘
2^20 页    4KB 页大小
```

### 🔁 地址变换过程

```
逻辑地址 LA
    │
    ▼
┌──────────────┐
│ P = LA / 页面大小 │ ← 页号
│ W = LA % 页面大小 │ ← 页内偏移
└──────────────┘
    │
    ▼
┌──────────────┐
│ 查页表：      │
│ 页框号 = 页表[P] │
└──────────────┘
    │
    ▼
┌──────────────┐
│ 物理地址 =    │
│ 页框号 × 页面大小 + W │
└──────────────┘
```

### 🚀 快表（TLB）

```
CPU ───> TLB（快表）
          │ 命中？
          ├── 是 → 直接得页框号（无需访存）
          └── 否 → 查内存中的页表（慢表）
                    │
                    ▼
                  访存 2 次
```

### 📊 多级页表（以二级页表为例）

```
┌──────────┬──────────┬──────────┐
│ 一级页号 │ 二级页号 │ 页内偏移 │
│  10 位   │  10 位   │  12 位   │
└──────────┴──────────┴──────────┘
    │
    ▼
┌──────────────┐
│ 一级页表     │ → 指向二级页表
└──────────────┘
    │
    ▼
┌──────────────┐
│ 二级页表     │ → 指向页框
└──────────────┘
```

> **优点**：节省页表占用空间（一级页表可以不在内存中）  
> **缺点**：访存次数增加（3 次）

## 3️⃣ 基本分段存储管理

### 📌 核心思想

> **段** = 按程序的**逻辑结构**划分（如代码段、数据段、栈段）  
> 每段大小不固定，段内地址连续

### 📊 地址结构

```
┌──────────┬──────────┐
│  段号 S  │ 段内偏移 W │
│  16 位   │  16 位    │
└──────────┴──────────┘
```

### 🔁 段表项

```
┌──────────┬──────────┬──────┬──────┐
│ 段号     │ 段长     │ 基址 │ 权限 │
└──────────┴──────────┴──────┴──────┘
```

### 📊 分页 vs 分段 对比

| 维度 | 分页 | 分段 |
|------|------|------|
| **划分依据** | 固定大小（硬件决定） | 逻辑结构（程序员决定） |
| **地址结构** | (页号, 页内偏移) | (段号, 段内偏移) |
| **大小** | 固定 | 可变 |
| **透明性** | 对程序员透明 | 对程序员可见 |
| **保护** | 简单 | 段级保护 |
| **共享** | 不便 | 容易 |
| **碎片** | 内部碎片 | 外部碎片 |

## 4️⃣ 虚拟存储管理

### 📌 核心思想

> **请求分页** = 只把当前需要的页面调入内存，其他页面在外存  
> **页面置换** = 内存满时，淘汰暂时不用的页面，调入新页面

### 📊 虚拟地址空间 vs 物理内存

```
虚拟地址空间（32 位，4GB）
┌────────────────┐
│ 虚拟页 0       │ → 实际不在内存
├────────────────┤
│ 虚拟页 1       │ → 实际在内存（页框 5）
├────────────────┤
│ 虚拟页 2       │ → 实际不在内存
├────────────────┤
│ ...            │
└────────────────┘
       │ 页表
       ▼
物理内存（1GB）
┌────────────────┐
│ 页框 0         │
├────────────────┤
│ 页框 1         │
├────────────────┤
│ ...            │
└────────────────┘
```

### 📊 4 大页面置换算法

#### OPT（最佳置换）

> **思想**：淘汰**未来最久**不使用的页面  
> **特点**：理论最优，无法实现

#### FIFO（先进先出）

> **思想**：淘汰**最早进入**内存的页面  
> **特点**：实现简单，**Belady 异常**（增加页框可能增加缺页）

#### LRU（最近最久未使用）

> **思想**：淘汰**过去最久**未使用的页面  
> **特点**：性能接近 OPT，**实现复杂**（需硬件支持或软件模拟）

#### CLOCK（时钟算法）

> **思想**：每页有访问位 A，循环扫描  
> **步骤**：
> 1. 找 A=0 的页面淘汰
> 2. 遇到 A=1 的页面，将 A 置 0 后跳过
> 3. 第二轮必能找到 A=0

```
    ┌──→ 页 0
    │    A=1 → A=0, 跳过
    │    
页 N ┼──→ 页 1
    │    A=0 → 淘汰！
    │    
    └──→ 页 2
         A=0 → 淘汰！
```

### 📊 4 大算法性能对比

| 算法 | 缺页率 | 实现难度 | Belady 异常 |
|------|--------|---------|-----------|
| OPT | 最低 | 不可能实现 | - |
| FIFO | 高 | 简单 | 有 |
| LRU | 低 | 复杂 | 无 |
| CLOCK | 中 | 中等 | 无 |

## 🎯 存储管理·必背公式

### 1. 逻辑地址 → 物理地址

```
页号 P = 逻辑地址 / 页面大小
页内偏移 W = 逻辑地址 % 页面大小
物理地址 = 页框号 × 页面大小 + W
```

### 2. 页表大小

```
页表项数 = 2^页号位数
页表大小 = 页表项数 × 页表项大小（通常 4 字节）
```

### 3. 缺页率

```
缺页率 = 缺页次数 / 访问总次数
```

### 4. 访问时间（带 TLB）

```
EAT = 命中率 × TLB访问时间 + (1-命中率) × (TLB访问时间 + 内存访问时间)
```

## ⚠️ 6 大易错点

> 1. ❌ 页号和页内偏移算错（位运算搞反）
> 2. ❌ 页表项大小误算（32 位系统页表项 = 4 字节）
> 3. ❌ 多级页表多访存次数（k 级页表 = k+1 次访存）
> 4. ❌ FIFO 的 Belady 异常
> 5. ❌ 分段 vs 分页混淆
> 6. ❌ 页面大小与地址结构位数不匹配
