## 1️⃣ 文件的逻辑结构

### 📊 3 大逻辑结构对比

| 结构 | 特点 | 适用 | 优缺点 |
|------|------|------|--------|
| **顺序文件** | 记录顺序存放 | 批量访问 | 简单 / 不支持随机访问 |
| **索引文件** | 每文件一个索引表 | 随机访问 | 灵活 / 索引表占空间 |
| **索引顺序文件** | 顺序 + 索引 | 顺序+随机 | 综合两者优点 |

### 📊 索引文件结构

```
文件数据
┌────┬────┬────┬────┬────┐
│ 0  │ 1  │ 2  │ 3  │ 4  │ ← 逻辑记录
└──┬─┴──┬─┴──┬─┴──┬─┴──┬─┘
   ▼    ▼    ▼    ▼    ▼
┌──────────────────────────┐
│     物理磁盘块号          │ ← 索引表
│  100  205  150  300  ...  │
└──────────────────────────┘
```

### 📊 多级索引（UNIX inode）

```
inode（索引节点）
┌──────────────────────┐
│ 13 个地址项：         │
│  [0]-[9]: 直接指针    │ → 10 个数据块
│  [10]: 一级间接       │ → 1 个索引块
│  [11]: 二级间接       │ → 1 个索引块的索引
│  [12]: 三级间接       │
└──────────────────────┘

最大文件大小 = 10 + 1×256 + 1×256² + 1×256³ 块
            (假设块大小 1KB, 地址 4B)
            = 10KB + 256KB + 64MB + 16GB ≈ 16GB
```

## 2️⃣ 目录管理

### 📊 FCB（文件控制块）

```
FCB 结构
┌──────────────┐
│ 文件名       │
├──────────────┤
│ 文件类型     │
├──────────────┤
│ 物理位置     │ ← 起始块号 / inode
├──────────────┤
│ 文件大小     │
├──────────────┤
│ 保护信息     │ ← 权限位
├──────────────┤
│ 时间戳       │ ← 创建/修改/访问
└──────────────┘
```

### 📊 4 大目录结构

#### 1. 单级目录

```
/ root
├── file1
├── file2
└── file3
```

> **问题**：不允许重名，查找慢

#### 2. 二级目录

```
/ root
├── user1
│   ├── file1
│   └── file2
└── user2
    ├── file1  ← 不同用户可重名
    └── file3
```

#### 3. 树形目录

```
/
├── home
│   ├── user1
│   │   ├── docs
│   │   └── photos
│   └── user2
├── usr
│   ├── bin
│   └── lib
└── var
    └── log
```

> **优点**：层次清晰，**绝对路径 / 相对路径**  
> **缺点**：**不便于共享**（不同用户访问需指明路径）

#### 4. 无环图目录

```
/ root
├── docs (共享)
│   ├── a.txt  ─┐
│   └── b.txt   │ 硬链接
├── home         │
│   └── user1    │
│       └── a.txt│ ← 指向同一文件
│       └── b.txt│
└── usr
    └── a.txt  ──┘
```

> **优点**：支持**共享**（硬链接 / 软链接）  
> **缺点**：需解决**删除一致性**问题

### 📊 软链接 vs 硬链接

| 维度 | 硬链接 | 软链接 |
|------|--------|--------|
| **本质** | 同一文件的多个目录项 | 快捷方式（指向路径） |
| **inode** | 相同 inode | 不同 inode |
| **删除原文件** | 仍可访问（其他硬链接） | 不可访问（悬空链接） |
| **跨文件系统** | 不支持 | 支持 |
| **目录** | 不支持（早期） | 支持 |

## 3️⃣ 磁盘调度算法

### 📌 4 大算法对比

假设磁头初始位置 53，磁道请求序列：98, 183, 37, 122, 14, 124, 65, 67

#### FCFS（先来先服务）

> **思想**：按请求顺序访问

```
访问顺序：53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
移动距离：45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = 640
```

#### SSTF（最短寻道优先）

> **思想**：优先访问最近的磁道

```
53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183
移动距离：12 + 2 + 30 + 23 + 84 + 24 + 2 + 59 = 236
```

#### SCAN（电梯算法）

> **思想**：磁头单向移动到底，再反向

```
53 → 65 → 67 → 98 → 122 → 124 → 183 → (到 199) → 37 → 14
移动距离：12 + 2 + 31 + 24 + 2 + 59 + 16 + 162 + 23 = 331
```

#### C-SCAN（循环扫描）

> **思想**：磁头单向移动到末尾，**直接跳回起点**，再单向扫描

```
53 → 65 → 67 → 98 → 122 → 124 → 183 → (到 199) → 跳到 0 → 14 → 37
移动距离：12 + 2 + 31 + 24 + 2 + 59 + 16 + 199 + 14 + 23 = 382
```

### 📊 4 大算法对比表

| 算法 | 寻道 | 公平性 | 饥饿 | 适用 |
|------|------|--------|------|------|
| FCFS | 长 | 公平 | 无 | 简单场景 |
| SSTF | 短 | 不公平 | 有 | 单用户 |
| SCAN | 较短 | 较公平 | 两端可能 | 电梯 |
| C-SCAN | 较短 | 公平 | 均匀 | 均匀分布 |

## 4️⃣ 文件系统实现

### 📊 4 大空闲空间管理

#### 1. 空闲表法

```
┌──────┬──────┬──────┬──────┐
│ 起始 │ 长度 │ 起始 │ 长度 │
│ 块号 │      │ 块号 │      │
├──────┼──────┼──────┼──────┤
│  2   │  4   │  9   │  3   │
└──────┴──────┴──────┴──────┘
```

> **适用**：连续分配

#### 2. 空闲链表法

```
空闲块 2 → 空闲块 5 → 空闲块 10 → 空闲块 15 → NULL
```

> **缺点**：分配/回收需遍历链表，效率低

#### 3. 位示图法 ⭐⭐⭐⭐⭐

```
位示图（每行 8 位）
第 0 行: 0 0 1 1 1 1 0 0  (1=占用, 0=空闲)
第 1 行: 1 1 1 1 0 0 0 0
第 2 行: 1 0 0 0 1 1 1 0
...
```

> **优点**：节省空间，查找快速  
> **计算公式**：
> - 位号 = (起始块号) % 字长
> - 行号 = (起始块号) / 字长
> - 字号 = (行号) / 每行字数
> - 位号 = (行号) % 每行字数

#### 4. 成组链接法（UNIX）

```
组 0（超级块）        组 1                组 2
┌────────────┐      ┌────────────┐      ┌────────────┐
│ 组长：100  │      │ 组长：200  │      │ 组长：300  │
│ 总空闲数：N │      │ 总空闲数：N │      │ 总空闲数：N │
│ 下一组：200 │ ──→ │ 下一组：300 │ ──→ │ 下一组：NIL │
└────────────┘      └────────────┘      └────────────┘
```

> **优点**：兼顾效率和空间，是 **UNIX 文件系统**标准方法

## 🎯 文件系统·必背公式

### 1. 磁盘访问时间

```
T = 寻道时间 + 旋转延迟 + 传输时间
T ≈ T_seek + T_rotation/2 + T_transfer
```

### 2. FAT 表大小

```
FAT 大小 = 磁盘块数 × FAT 项大小
        = 磁盘容量 / 块大小 × FAT 项大小
```

### 3. 位示图大小

```
位示图大小 = 磁盘块数（位）
         = 磁盘容量 / 块大小
```

### 4. 索引节点大小

```
inode 大小 = (直接指针数 + 间接级数) × 指针大小 + 文件元数据
```
