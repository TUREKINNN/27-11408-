## 1️⃣ IO 控制方式（4 大方式对比）

### 📊 4 大方式发展史

```
程序直接控制 → 中断驱动 → DMA → 通道
   简单          CPU 参与少    仅初始化    独立处理
   CPU 忙等      CPU 切换     CPU 不参与  高度并行
```

### 1. 程序直接控制方式（程序查询 / 轮询）

```
CPU ──> 设备 ──> 返回结果
  ↑         ↓
  └──── 等待 ────┘
  （CPU 一直忙等）
```

> **特点**：CPU 100% 占用，**效率极低**  
> **适用**：早期简单系统

### 2. 中断驱动方式

```
CPU ──> 启动 IO ──> 处理其他任务
  ↑
  └──── 设备完成 ──中断──> CPU 处理
```

> **特点**：CPU 利用率高，**每次 IO 字都中断**（开销大）  
> **适用**：字符设备

### 3. DMA 方式（直接存储器访问）

```
CPU ──> 设置 DMA 参数 ──> 去做其他事
  ↑         │
  │         ▼
  │      DMA 控制器 ──> 直接传输数据块到内存
  │         │
  │         ▼
  └── 完成 ──中断──> CPU
```

> **特点**：**块传输**才中断，CPU 干预少  
> **适用**：块设备（磁盘）

### 4. 通道方式

```
CPU ──> 通道程序 ──> 通道控制器
                       │  执行多个 IO 操作
                       │  （独立于 CPU）
                       ▼
                   IO 设备 1, 2, 3, ...
```

> **特点**：**通道是专门处理器**，可执行复杂 IO 程序  
> **适用**：大型机 / 高性能服务器

### 📊 4 大方式对比

| 方式 | CPU 干预 | 中断次数 | 适用 | 效率 |
|------|---------|---------|------|------|
| 程序查询 | 全程 | 0 | 简单 | ★ |
| 中断 | 每次字 | 多 | 字符设备 | ★★ |
| DMA | 块前后 | 2 次 | 块设备 | ★★★★ |
| 通道 | 极少 | 极少 | 大型机 | ★★★★★ |

## 2️⃣ 缓冲管理

### 📊 4 大缓冲方式

#### 1. 单缓冲

```
┌──────────┐
│  缓冲 1  │ ← 一个缓冲区
└──────────┘
T = max(C, T) + M
（每块：max(IO 时间, 处理时间) + 传输时间）
```

> **特点**：设备和处理器**串行**工作

#### 2. 双缓冲

```
┌──────────┬──────────┐
│  缓冲 1  │  缓冲 2  │ ← 两个缓冲区
└──────────┴──────────┘
T = max(C, T)
（IO 和处理可并行）
```

> **特点**：设备和处理器**并行**工作

#### 3. 循环缓冲

```
┌────┬────┬────┬────┬────┬────┐
│ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ ← N 个环形缓冲
└──┬┴──┬─┴──┬─┴──┬─┴──┬─┴──┬─┘
   └─────────────┘
       (指针循环)
```

> **特点**：多个缓冲循环使用，适合**突发 IO**

#### 4. 缓冲池

```
┌─────────────────────────────┐
│  缓冲池（多个缓冲队列）       │
├─────────────┬───────────────┤
│  空闲缓冲   │  输入队列      │
├─────────────┼───────────────┤
│  收容输入   │  提取输入      │
├─────────────┼───────────────┤
│  收容输出   │  提取输出      │
└─────────────┴───────────────┘
```

> **特点**：缓冲池**集中管理**，可双向使用

### 📊 4 大方式对比

| 方式 | 缓冲数 | 适用 | 复杂度 |
|------|--------|------|--------|
| 单缓冲 | 1 | 简单 | ★ |
| 双缓冲 | 2 | 并行 | ★★ |
| 循环缓冲 | N | 突发 | ★★★ |
| 缓冲池 | 多组 | 复杂 | ★★★★ |

## 3️⃣ 设备分配

### 📊 设备分类

```
设备
├── 独占设备
│   └── 打印机、磁带机
│       （分配后独占使用）
├── 共享设备
│   └── 磁盘
│       （可多个进程交替使用）
└── 虚拟设备
    └── SPOOLing 设备
        （通过 SPOOLing 把独占变共享）
```

### 📊 4 大设备分配算法

| 算法 | 思想 | 优缺点 |
|------|------|--------|
| **先来先服务** | 按请求顺序分配 | 公平 / 设备利用率低 |
| **优先级高优先** | 优先级高的先分配 | 实时性 / 可能饥饿 |
| **最短寻道优先** | 选最近的设备 | 高效 / 远端饥饿 |
| **专用/共享/虚拟** | 按设备类型 | 灵活 / 实现复杂 |

### 📊 设备分配考虑因素

| 因素 | 内容 |
|------|------|
| **设备类型** | 独占 / 共享 / 虚拟 |
| **设备状态** | 忙 / 闲 / 故障 |
| **分配算法** | FCFS / 优先级 / SSTF |
| **安全性** | 分配后是否会导致死锁 |
| **独立性** | 用户程序与物理设备无关 |

### 📊 设备分配中的数据结构

```
DCT（设备控制表）          COCT（控制器控制表）
┌──────────────┐         ┌──────────────┐
│ 设备状态     │         │ 控制器状态   │
│ 等待队列     │         │ 等待队列     │
│ 占用进程     │ ──→     │ 占用设备     │
│ ...          │         │ ...          │
└──────────────┘         └──────────────┘
                              │
                              ▼
                         CHCT（通道控制表）
                         ┌──────────────┐
                         │ 通道状态     │
                         │ 等待队列     │
                         │ ...          │
                         └──────────────┘
```

## 4️⃣ SPOOLing（假脱机技术）

### 📌 核心思想

> **磁盘模拟独占设备**  
> 把慢速独占设备"模拟"成共享设备

### 📊 SPOOLing 系统组成

```
┌─────────────────────────────────────┐
│         用户进程                      │
├─────────────────────────────────────┤
│    输入井      │     输出井            │ ← 磁盘区域
│ (输入缓冲区)   │   (输出缓冲区)        │
├─────────────────────────────────────┤
│   输入 SP    │    输出 SP             │ ← 系统进程
│ (预输入)      │    (缓输出)            │
├─────────────────────────────────────┤
│  输入设备    │   输出设备              │
│ (键盘等)     │   (打印机等)            │
└─────────────────────────────────────┘
```

### 📊 SPOOLing 工作流程（以打印机为例）

```
1. 用户进程生成打印数据
2. 写入"输出井"（磁盘文件）
3. 用户进程立即返回（不等待打印）
4. 输出 SP（系统进程）在后台从输出井读取数据
5. 实际控制打印机打印
```

> **优点**：
> - **提高独占设备利用率**
> - **用户进程不需等待慢速 IO**
> - **实现设备独立性**

### 📊 SPOOLing 必背要点

| 概念 | 说明 |
|------|------|
| **输入井 / 输出井** | 磁盘上开辟的存储区域 |
| **输入 SP / 输出 SP** | 系统进程（类似守护进程） |
| **请求打印表** | 记录打印任务的状态信息 |
| **特点** | 预输入 + 缓输出 |
| **适用设备** | 打印机、磁带等独占设备 |

## 🎯 IO 管理·必背公式

### 1. 单缓冲时间

```
T = max(C, T) + M
其中 C = 处理时间, T = IO 时间, M = 传输时间
```

### 2. 双缓冲时间

```
T = max(C, T)
（并行处理）
```

### 3. 系统吞吐量

```
W = 1 / T_block
T_block = max(C, T) + M  （单缓冲）
T_block = max(C, T)      （双缓冲）
```

### 4. 寻道时间

```
T_seek = n × s
n = 移动磁道数
s = 跨一道时间（通常 1ms）
```

## ⚠️ 5 大易错点

> 1. ❌ 4 大 IO 控制方式的发展顺序（程序查询 → 中断 → DMA → 通道）
> 2. ❌ 单/双缓冲时间公式搞混
> 3. ❌ 共享/独占/虚拟设备分类
> 4. ❌ SPOOLing 的"输入井/输出井"是**磁盘**，不是内存
> 5. ❌ DMA 中断次数（块前后各 1 次 = 2 次）
