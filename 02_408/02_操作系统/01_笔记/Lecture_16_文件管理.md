## 2. 文件目录

### 2.1 文件目录的作用

<span style="background-color:#ccc1d9">**文件目录**</span>：管理文件的索引结构。

### 2.2 目录项的内容

| 内容 | 描述 |
|------|------|
| **文件名** | 文件标识 |
| **文件类型** | 文件分类 |
| **文件位置** | 文件存储位置 |
| **文件大小** | 文件长度 |
| **创建时间** | 创建日期 |
| **访问权限** | 读写权限 |

### 2.3 目录结构

#### 2.3.1 单级目录

```
┌─────────────────────────┐
│       根目录            │
│  文件1 文件2 文件3 ...  │
└─────────────────────────┘
```

**优点**：简单。

**缺点**：重名问题，查找效率低。

#### 2.3.2 两级目录

```
┌─────────────────────────┐
│       主目录            │
│  用户1 用户2 用户3 ...  │
├──────────┬──────────────┤
│ 用户1目录 │ 用户2目录... │
│ 文件1文件2│ 文件3文件4...│
└──────────┴──────────────┘
```

**优点**：解决重名问题。

**缺点**：不够灵活。

#### 2.3.3 多级目录（树形结构）

```
┌─────────────────────────┐
│         /               │
├──────┬──────┬───────────┤
│ bin  │ etc  │   home    │
├──┬───┴──┬───┴────┬──────┤
│ ls │cat │hosts │user1  │
└───┴─────┴───────┴──┬────┘
                     │
                 文件1 文件2
```

**优点**：层次清晰，便于管理。

**缺点**：查找路径长。

#### 2.3.4 图状目录

**特点**：支持共享和链接。

**优点**：灵活。

**缺点**：复杂，可能出现环路。

## 3. 文件系统实现

### 3.1 文件的物理结构

#### 3.1.1 连续分配

```
┌──────┬──────┬──────┬──────┬──────┐
│ 文件A │ 文件A │ 文件A │ 文件B │ 文件B │
└──────┴──────┴──────┴──────┴──────┘
```

**优点**：顺序访问快。

**缺点**：有外部碎片，难以扩展。

#### 3.1.2 链接分配

```
┌──────┬──────┬──────┬──────┬──────┐
│ 文件A │ 文件B │ 文件A │ 文件C │ 文件A │
│ →3   │ →4   │ →5   │      │      │
└──────┴──────┴──────┴──────┴──────┘
```

**优点**：无外部碎片，便于扩展。

**缺点**：随机访问慢。

#### 3.1.3 索引分配

```
┌──────┬──────┬──────┬──────┬──────┐
│ 索引表│ 文件A │ 文件A │ 文件A │ 文件B │
│ →1,2,3│      │      │      │      │
└──────┴──────┴──────┴──────┴──────┘
```

**优点**：无外部碎片，随机访问快。

**缺点**：索引表占用空间。

#### 3.1.4 多级索引

```
一级索引 → 二级索引 → 数据块
```

**优点**：支持大文件。

**缺点**：地址变换开销大。

### 3.2 文件存储空间管理

#### 3.2.1 空闲表法

```
┌──────┬──────┬──────┐
│ 起始 │ 长度 │ 状态 │
├──────┼──────┼──────┤
│ 100  │ 5    │ 空闲 │
│ 200  │ 10   │ 空闲 │
└──────┴──────┴──────┘
```

**优点**：简单。

**缺点**：查找效率低。

#### 3.2.2 空闲链表法

```
空闲块1 → 空闲块2 → 空闲块3 → NULL
```

**优点**：便于动态分配。

**缺点**：查找效率低。

#### 3.2.3 位示图法

```
0 1 0 0 1 0 1 0 ...
↑
第0位：第0块是否空闲
```

**优点**：查找效率高。

**缺点**：需要额外空间。

#### 3.2.4 成组链接法

**特点**：结合空闲表和空闲链表。

**优点**：高效。

**缺点**：复杂。

## 4. 文件的存取方法

### 4.1 顺序存取

```
从头开始，顺序读取每个记录
```

**优点**：简单。

**缺点**：查找效率低。

### 4.2 随机存取

```
直接访问指定位置的记录
```

**优点**：查找效率高。

**缺点**：需要索引支持。

### 4.3 索引存取

```
通过索引表间接访问记录
```

**优点**：灵活。

**缺点**：索引表占用空间。

## 7. 磁盘调度算法

### 7.1 先来先服务（FCFS）

```
按请求顺序处理
```

**优点**：简单，公平。

**缺点**：寻道时间长。

### 7.2 最短寻道时间优先（SSTF）

```
优先处理最近的请求
```

**优点**：寻道时间短。

**缺点**：可能导致饥饿。

### 7.3 扫描算法（SCAN）

```
从一端扫描到另一端，处理沿途请求
```

**优点**：公平，寻道时间短。

**缺点**：两端请求等待时间长。

### 7.4 循环扫描算法（C-SCAN）

```
从一端扫描到另一端，然后直接返回起点
```

**优点**：均匀响应。

**缺点**：略复杂。

## 8. 典型例题

### 例题 1：文件物理结构

**题目**：比较连续分配、链接分配和索引分配的优缺点。

**解答**：

```
连续分配：
- 优点：顺序访问快
- 缺点：有外部碎片，难以扩展

链接分配：
- 优点：无外部碎片，便于扩展
- 缺点：随机访问慢

索引分配：
- 优点：无外部碎片，随机访问快
- 缺点：索引表占用空间
```

**总结**：不同物理结构适用于不同场景。

**来源**：408历年真题，考查方式：简答题

### 例题 2：磁盘调度

**题目**：磁盘请求队列：98, 183, 37, 122, 14, 124, 65, 67。当前磁头在53。分别用SSTF和SCAN计算总寻道长度。

**解答**：

```
SSTF：
当前位置：53
最近：65 (53→65, 12)
次近：67 (65→67, 2)
次近：37 (67→37, 30)
次近：14 (37→14, 23)
次近：98 (14→98, 84)
次近：122 (98→122, 24)
次近：124 (122→124, 2)
次近：183 (124→183, 59)

总寻道长度：12+2+30+23+84+24+2+59 = 236

SCAN（假设向磁道号增大方向扫描）：
53→65 (12) →67 (2) →98 (31) →122 (24) →124 (2) →183 (59)
然后转向：183→37 (146) →14 (23)

总寻道长度：12+2+31+24+2+59+146+23 = 299
```

**总结**：SSTF的寻道长度更短。

**来源**：408历年真题，考查方式：计算题

### 例题 3：多级索引

**题目**：某文件系统采用三级索引，每个索引块可存放100个索引项，每个磁盘块大小为1KB。求最大文件大小。

**解答**：

```
一级索引：100个数据块
二级索引：100 × 100 = 10000个数据块
三级索引：100 × 100 × 100 = 1000000个数据块

总数据块数 = 100 + 10000 + 1000000 = 1010100个

每个数据块大小 = 1KB

最大文件大小 = 1010100 × 1KB = 1010100KB ≈ 986.4MB
```

**总结**：多级索引支持大文件存储。

**来源**：408历年真题，考查方式：计算题
