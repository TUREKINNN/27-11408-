# Lecture 01: 操作系统概述

> 本笔记整合自王道《操作系统》、408历年真题、汤子瀛《计算机操作系统》等资料。
> 核心概念用 <span style="background-color: #ccc1d9">紫色</span> 标注，重要定理用 <span style="background-color: #8db3e2">蓝色</span> 标注。

---

## 1. 操作系统的功能

### 1.1 操作系统的定义

<span style="background-color:#ccc1d9">**操作系统**</span>：管理计算机硬件和软件资源，为用户和应用程序提供服务的系统软件。

### 1.2 操作系统的功能

| 功能 | 描述 |
|------|------|
| **进程管理** | 管理进程的创建、调度、同步、通信 |
| **内存管理** | 管理内存分配、回收、保护、共享 |
| **文件管理** | 管理文件的创建、读写、删除、保护 |
| **设备管理** | 管理I/O设备的分配、驱动、调度 |
| **用户接口** | 提供命令行和图形界面 |

---

## 2. 操作系统的分类

### 2.1 按用户数量分类

| 类型 | 描述 |
|------|------|
| **单用户单任务** | 一个用户，一个任务（MS-DOS） |
| **单用户多任务** | 一个用户，多个任务（Windows） |
| **多用户多任务** | 多个用户，多个任务（Linux） |

### 2.2 按使用环境分类

| 类型 | 描述 |
|------|------|
| **批处理系统** | 批量处理，无交互（大型机） |
| **分时系统** | 多个用户共享CPU（UNIX） |
| **实时系统** | 实时响应，确定性（嵌入式） |
| **网络操作系统** | 管理网络资源（Windows Server） |
| **分布式操作系统** | 管理分布式资源（Hadoop） |

---

## 3. 进程与线程

### 3.1 进程的定义

<span style="background-color:#ccc1d9">**进程**</span>：程序在计算机中的一次执行过程。

### 3.2 进程的特征

| 特征 | 描述 |
|------|------|
| **动态性** | 进程有生命周期 |
| **并发性** | 多个进程同时运行 |
| **独立性** | 进程是资源分配的基本单位 |
| **异步性** | 进程以不可预知的速度执行 |

### 3.3 进程的状态

| 状态 | 描述 |
|------|------|
| **就绪态** | 已分配资源，等待CPU |
| **运行态** | 正在执行 |
| **阻塞态** | 等待某个事件（I/O完成） |
| **创建态** | 进程正在创建 |
| **终止态** | 进程执行完毕 |

### 3.4 进程状态转换

```
创建态 → 就绪态 → 运行态 → 阻塞态 → 就绪态 → 运行态 → 终止态
              ↑                          │
              └──────────────────────────┘
```

### 3.5 线程的定义

<span style="background-color:#ccc1d9">**线程**</span>：进程中的执行单元，是CPU调度的基本单位。

### 3.6 进程与线程的区别

| 对比项 | 进程 | 线程 |
|--------|------|------|
| **资源分配** | 基本单位 | 不分配资源 |
| **调度** | 切换开销大 | 切换开销小 |
| **并发性** | 进程间并发 | 线程间并发 |
| **独立性** | 独立地址空间 | 共享进程资源 |

### 3.7 线程的分类

| 类型 | 描述 |
|------|------|
| **用户级线程** | 由用户程序管理 |
| **内核级线程** | 由操作系统管理 |
| **混合线程** | 用户级+内核级 |

## 4. 进程调度

### 4.1 调度的层次

| 层次 | 描述 |
|------|------|
| **高级调度** | 决定哪些作业进入内存 |
| **中级调度** | 决定哪些进程换入/换出 |
| **低级调度** | 决定哪个进程获得CPU |

### 4.2 调度算法的评价标准

| 标准 | 描述 |
|------|------|
| **CPU利用率** | CPU使用时间比例 |
| **吞吐量** | 单位时间完成作业数 |
| **周转时间** | 作业从提交到完成的时间 |
| **响应时间** | 从请求到响应的时间 |
| **公平性** | 公平分配CPU |

### 4.3 调度算法

#### 4.3.1 先来先服务（FCFS）

```
按到达顺序调度
```

**优点**：简单，公平。

**缺点**：不利于短作业。

#### 4.3.2 短作业优先（SJF）

```
优先调度最短的作业
```

**优点**：平均周转时间短。

**缺点**：可能导致长作业饥饿。

#### 4.3.3 时间片轮转（RR）

```
每个进程分配一个时间片，轮流执行
```

**优点**：响应时间快，公平。

**缺点**：时间片选择困难。

#### 4.3.4 优先级调度

```
按优先级调度，优先级高的先执行
```

**优点**：灵活。

**缺点**：低优先级进程可能饥饿。

#### 4.3.5 多级反馈队列

```
多个队列，优先级从高到低，时间片从小到大
```

**优点**：综合了各种算法的优点。

**缺点**：复杂。

## 5. 进程控制块

### 5.1 PCB的定义

<span style="background-color:#ccc1d9">**PCB**</span>：进程控制块，存储进程的所有信息。

### 5.2 PCB的内容

| 信息 | 描述 |
|------|------|
| **进程标识符** | 进程唯一标识 |
| **进程状态** | 当前状态 |
| **CPU寄存器** | 保存的寄存器值 |
| **优先级** | 调度优先级 |
| **内存管理信息** | 页表、段表 |
| **I/O状态信息** | 打开的文件、设备 |
| **进程同步信息** | 信号量、锁 |

## 6. 操作系统的结构

### 6.1 整体式结构

**特点**：所有功能集成在一起。

**优点**：简单。

**缺点**：难以维护。

### 6.2 层次式结构

**特点**：按层次组织，下层为上层服务。

**优点**：清晰，易维护。

**缺点**：效率低。

### 6.3 微内核结构

**特点**：核心功能最小化，其他功能作为服务。

**优点**：灵活，可靠。

**缺点**：通信开销大。

## 7. 典型例题

### 例题 1：进程状态转换

**题目**：描述进程从就绪态到运行态再到阻塞态的转换过程。

**解答**：

```
1. 就绪态 → 运行态：
   - 条件：进程被调度选中
   - 操作：分配CPU，恢复上下文

2. 运行态 → 阻塞态：
   - 条件：进程等待I/O或其他事件
   - 操作：保存上下文，释放CPU

3. 阻塞态 → 就绪态：
   - 条件：等待的事件完成
   - 操作：进程进入就绪队列
```

**总结**：进程状态转换由调度和事件驱动。

**来源**：408历年真题，考查方式：简答题

---

### 例题 2：调度算法比较

**题目**：比较FCFS、SJF、RR三种调度算法的特点。

**解答**：

```
FCFS：
- 优点：简单，公平
- 缺点：不利于短作业，平均周转时间长

SJF：
- 优点：平均周转时间最短
- 缺点：长作业可能饥饿，需要预知作业时间

RR：
- 优点：响应时间快，公平
- 缺点：时间片选择困难，上下文切换开销大
```

**总结**：不同调度算法适用于不同场景。

**来源**：408历年真题，考查方式：简答题

---

### 例题 3：周转时间计算

**题目**：三个作业A、B、C的到达时间和运行时间分别为：A(0, 3)、B(1, 2)、C(2, 1)。分别用FCFS和SJF计算平均周转时间。

**解答**：

```
FCFS：
A: 到达0，开始0，结束3，周转3
B: 到达1，开始3，结束5，周转4
C: 到达2，开始5，结束6，周转4
平均：(3+4+4)/3 = 11/3 ≈ 3.67

SJF：
A: 到达0，开始0，结束3，周转3
C: 到达2，开始3，结束4，周转2
B: 到达1，开始4，结束6，周转5
平均：(3+2+5)/3 = 10/3 ≈ 3.33
```

**总结**：SJF的平均周转时间更短。

**来源**：408历年真题，考查方式：计算题
