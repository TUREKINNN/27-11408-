## 1. 图的基本概念

### 1.1 图的定义

<span style="background-color:#ccc1d9">**图**</span>是由顶点集合 $V$ 和边集合 $E$ 组成的，记为 $G = (V, E)$。

### 1.2 图的基本术语

| 术语 | 定义 |
|------|------|
| **顶点/边** | 图中的数据元素称为顶点，顶点之间的关系称为边 |
| **有向图/无向图** | 边有方向的图为有向图，边无方向的图为无向图 |
| **完全图** | 任意两个顶点之间都存在边的图 |
| **稀疏图/稠密图** | 边数远小于顶点平方的图为稀疏图 |
| **度** | 顶点的度是依附于该顶点的边数 |
| **出度/入度** | 有向图中，出度是从该顶点出发的边数，入度是指向该顶点的边数 |
| **路径** | 从顶点 $v_i$ 到 $v_j$ 的顶点序列 |
| **路径长度** | 路径上边的数目 |
| **简单路径** | 路径中顶点不重复出现 |
| **回路** | 起点和终点相同的路径 |
| **连通图** | 无向图中任意两个顶点都连通 |
| **强连通图** | 有向图中任意两个顶点都双向可达 |
| **权/网** | 边带有数值称为权，带权的图称为网 |

### 1.3 图的性质

<span style="background-color:#8db3e2">**性质1**：无向图的所有顶点度数之和 = $2|E|$</span>

<span style="background-color:#8db3e2">**性质2**：有向图的所有顶点入度之和 = 所有顶点出度之和 = $|E|$</span>

## 2. 图的存储结构

### 2.1 邻接矩阵法

<span style="background-color:#ccc1d9">**邻接矩阵**</span>：用二维数组表示顶点之间的相邻关系。

```c
#define MaxVertexNum 100
typedef struct {
    char Vex[MaxVertexNum];
    int Edge[MaxVertexNum][MaxVertexNum];
    int vexnum, arcnum;
} MGraph;
```

**特点**：
- 空间复杂度：$O(n^2)$
- 判断两顶点是否相邻：$O(1)$
- 查找顶点的所有邻接点：$O(n)$

### 2.2 邻接表法

<span style="background-color:#ccc1d9">**邻接表**</span>：用链表表示每个顶点的邻接点。

```c
typedef struct ArcNode {
    int adjvex;
    struct ArcNode *nextarc;
    int weight;
} ArcNode;

typedef struct VNode {
    char data;
    ArcNode *firstarc;
} VNode;

typedef struct {
    VNode adjlist[MaxVertexNum];
    int vexnum, arcnum;
} ALGraph;
```

**特点**：
- 空间复杂度：$O(n+e)$
- 判断两顶点是否相邻：$O(n)$
- 查找顶点的所有邻接点：$O(e/n)$

## 3. 图的遍历

### 3.1 深度优先遍历（DFS）

**思想**：从起点出发，尽可能深地访问，直到无法继续，再回溯。

```c
void DFS(Graph G, int v) {
    visit(v);
    visited[v] = true;
    for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
        if (!visited[w]) DFS(G, w);
    }
}
```

**时间复杂度**：邻接表 $O(n+e)$，邻接矩阵 $O(n^2)$

### 3.2 广度优先遍历（BFS）

**思想**：从起点出发，逐层访问所有邻接点。

```c
void BFS(Graph G, int v) {
    visit(v);
    visited[v] = true;
    EnQueue(Q, v);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, v);
        for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
            if (!visited[w]) {
                visit(w);
                visited[w] = true;
                EnQueue(Q, w);
            }
        }
    }
}
```

**时间复杂度**：邻接表 $O(n+e)$，邻接矩阵 $O(n^2)$

## 4. 最小生成树

### 4.1 生成树的定义

<span style="background-color:#ccc1d9">**生成树**</span>：包含图中全部顶点的极小连通子图，有 $n-1$ 条边。

### 4.2 Prim算法（普里姆）

**思想**：从某顶点开始，每次将代价最小的新顶点纳入生成树。

**时间复杂度**：$O(n^2)$，适合稠密图

### 4.3 Kruskal算法（克鲁斯卡尔）

**思想**：每次选择权值最小的边，若不形成环则加入生成树。

**时间复杂度**：$O(e \log e)$，适合稀疏图

## 5. 最短路径

### 5.1 Dijkstra算法

**思想**：从源点出发，逐步找到到各顶点的最短路径。

**时间复杂度**：$O(n^2)$

**局限性**：不能处理负权边

### 5.2 Floyd算法

**思想**：动态规划，逐步加入中间顶点，更新最短路径。

**时间复杂度**：$O(n^3)$

**优点**：能处理负权边，可求所有顶点对的最短路径

## 6. 拓扑排序

### 6.1 AOV网

<span style="background-color:#ccc1d9">**AOV网**</span>：用顶点表示活动，用边表示活动之间先后关系的有向无环图。

### 6.2 拓扑排序算法

**步骤**：
1. 计算每个顶点的入度
2. 将入度为0的顶点入队
3. 出队一个顶点，输出，删除该顶点及其所有出边
4. 重复步骤2-3，直到队列为空

**时间复杂度**：$O(n+e)$

---

## 7. 关键路径

### 7.1 AOE网

<span style="background-color:#ccc1d9">**AOE网**</span>：用顶点表示事件，用边表示活动，边上的权表示活动持续时间。

### 7.2 关键路径算法

**步骤**：
1. 拓扑排序，计算每个事件的最早发生时间 $ve$
2. 逆拓扑排序，计算每个事件的最迟发生时间 $vl$
3. 计算每个活动的最早开始时间 $e$ 和最迟开始时间 $l$
4. $e = l$ 的活动为关键活动

## 8. 典型例题

### 例题 1：判断图是否为树

**题目**：判断一个无向图是否为树。

**思路**：树的条件：连通且边数 = 顶点数 - 1

**解答**：

```c
bool isTree(Graph G) {
    int edges = 0;
    for (int i = 0; i < G.vexnum; i++) {
        for (int j = i+1; j < G.vexnum; j++) {
            if (G.Edge[i][j] != 0) edges++;
        }
    }
    if (edges != G.vexnum - 1) return false;
    // 检查连通性
    DFS(G, 0);
    for (int i = 0; i < G.vexnum; i++) {
        if (!visited[i]) return false;
    }
    return true;
}
```

**总结**：树是连通且无环的图。

**来源**：408历年真题，考查方式：选择题

### 例题 2：Dijkstra算法求最短路径

**题目**：用Dijkstra算法求下图中从顶点0到其他顶点的最短路径。

```
邻接矩阵：
   0  1  2  3  4
0  0  4  2  ∞  ∞
1  4  0  1  5  ∞
2  2  1  0  8  10
3  ∞  5  8  0  2
4  ∞  ∞  10 2  0
```

**解答**：

```
dist数组变化：
初始：[0, 4, 2, ∞, ∞]
选2：[0, 3, 2, 10, 12]
选1：[0, 3, 2, 8, 12]
选3：[0, 3, 2, 8, 10]
选4：[0, 3, 2, 8, 10]

最短路径：
0→1：0→2→1 (长度3)
0→2：0→2 (长度2)
0→3：0→2→1→3 (长度8)
0→4：0→2→1→3→4 (长度10)
```

**总结**：贪心策略，每次选当前最短路径。

**来源**：408历年真题，考查方式：综合题

### 例题 3：拓扑排序

**题目**：对下图进行拓扑排序。

```
顶点：A, B, C, D, E, F, G
边：A→C, A→D, B→D, B→E, C→F, D→F, E→G, F→G
```

**解答**：

```
入度：A:0, B:0, C:1, D:2, E:1, F:2, G:2

步骤：
1. 选A，输出：A，入度变化：C:0, D:1
2. 选B，输出：A→B，入度变化：D:0, E:0
3. 选C，输出：A→B→C，入度变化：F:1
4. 选D，输出：A→B→C→D，入度变化：F:0
5. 选E，输出：A→B→C→D→E，入度变化：G:1
6. 选F，输出：A→B→C→D→E→F，入度变化：G:0
7. 选G，输出：A→B→C→D→E→F→G

拓扑序列：A→B→C→D→E→F→G（不唯一）
```

**总结**：拓扑排序结果不唯一，但必须满足所有前驱关系。

**来源**：408历年真题，考查方式：综合题
