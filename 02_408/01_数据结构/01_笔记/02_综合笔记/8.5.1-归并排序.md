# 归并排序

## 📊 考情分析
- 考试频率：**高频考点**，几乎每年必考
- 分值：2~8分（选择题 + 算法设计题）
- 题型：选择题（复杂度、稳定性）、综合应用题（手算归并过程）、算法设计题（代码实现）
- 真题考查方式：归并过程手算、两个有序序列的归并操作、递归调用过程、空间复杂度分析

## 一、基本概念

### 1. 严格定义
**归并排序**（Merge Sort）是基于"**归并**"操作的排序算法。将两个或多个**已经有序的序列**合并成**一个有序序列**。

### 2. 基本术语
- **归并（Merge）**：将两个或多个有序序列合并为一个有序序列的操作
- **2路归并**：每次将两个有序序列合并（内部排序中一般采用2路归并）
- **m路归并**：每次将 m 个有序序列合并（外部排序中常用）
- **归并树**：归并排序过程中形成的倒立二叉树形态

## 二、核心知识

### 1. 2路归并的基本操作

将两个有序序列归并为一个有序序列：

**示例**：
- 序列A：`[12, 16, 24, 33, 45, 56, 60]`（7个元素）
- 序列B：`[7, 10, 21, 24, 37]`（5个元素）

**归并过程**（双指针法）：
1. 设置两个指针 i, j 分别指向A和B的开头
2. 比较 i, j 所指元素，将较小的放入结果数组
3. 移动被选中元素的指针
4. 重复直到一个序列被完全合并
5. 将另一个序列的剩余元素全部追加到结果中

```
A: [12, 16, 24, 33, 45, 56, 60]    B: [7, 10, 21, 24, 37]
    i                                   j

比较12和7 → 选7     结果: [7]
比较12和10 → 选10    结果: [7, 10]
比较12和21 → 选12    结果: [7, 10, 12]
比较16和21 → 选16    结果: [7, 10, 12, 16]
比较24和21 → 选21    结果: [7, 10, 12, 16, 21]
比较24和24 → 选24(A)  结果: [7, 10, 12, 16, 21, 24]（稳定性：相等时选靠前的）
比较33和24 → 选24(B)  结果: [7, 10, 12, 16, 21, 24, 24]
比较33和37 → 选33     结果: [7, 10, 12, 16, 21, 24, 24, 33]
A剩余：45, 56, 60 → 追加
最终结果: [7, 10, 12, 16, 21, 24, 24, 33, 45, 56, 60]
```

**关键结论**：
- **2路归并**：每选出一个最小元素需对比关键字 **1 次**
- **m路归并**：每选出一个最小元素需对比关键字 **m-1 次**

### 2. 归并排序过程

**思路**：
1. 将待排序序列不断**二分**，直到每个子序列只含1个元素（自然有序）
2. 相邻的两个子序列**归并**为一个有序序列
3. 重复归并过程直到整个序列有序

**示例**：初始序列 `[49, 38, 65, 97, 76, 13, 27]`

```
初始序列：[49, 38, 65, 97, 76, 13, 27]

第一趟归并（每两个相邻元素归并）：
[49] [38] → [38, 49]
[65] [97] → [65, 97]
[76] [13] → [13, 76]
[27]      → [27]
结果：[38, 49, 65, 97, 13, 76, 27]

第二趟归并（每两个相邻子序列归并）：
[38, 49] [65, 97] → [38, 49, 65, 97]
[13, 76] [27]     → [13, 27, 76]
结果：[38, 49, 65, 97, 13, 27, 76]

第三趟归并：
[38, 49, 65, 97] [13, 27, 76] → [13, 27, 38, 49, 65, 76, 97]
结果：[13, 27, 38, 49, 65, 76, 97]
```

**关键**：核心操作是**把数组内的两个有序序列归并为一个**

### 3. 代码实现（C语言）

#### （1）归并操作 Merge

```c
// 辅助数组B
int *B = (int *)malloc((n + 1) * sizeof(int));

// 将A[low...mid]和A[mid+1...high]归并为一个有序序列
void Merge(int A[], int low, int mid, int high) {
    // 将A[low...high]复制到辅助数组B
    for (int k = low; k <= high; k++)
        B[k] = A[k];
    
    int i = low;          // 左半部分的指针
    int j = mid + 1;      // 右半部分的指针
    int k = low;          // A数组的写入指针
    
    // 归并：比较B[i]和B[j]，将较小的放入A[k]
    while (i <= mid && j <= high) {
        if (B[i] <= B[j])        // 注意：<= 保证稳定性
            A[k++] = B[i++];
        else
            A[k++] = B[j++];
    }
    
    // 将剩余元素复制到尾部
    while (i <= mid)
        A[k++] = B[i++];
    while (j <= high)
        A[k++] = B[j++];
}
```

**归并操作图解**：
```
A: [low ... mid | mid+1 ... high]
B: (复制)
   i →           j →
   比较B[i]和B[j]，小的放入A[k]

直到i>mid或j>high，将剩余部分全部复制
```

#### （2）归并排序（递归实现）

```c
void MergeSort(int A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;    // 从中间划分
        MergeSort(A, low, mid);         // 对左半部分归并排序
        MergeSort(A, mid + 1, high);    // 对右半部分归并排序
        Merge(A, low, mid, high);       // 归并左右两部分
    }
}

// 调用：MergeSort(A, 0, n-1);
```

**递归调用过程**（以 `[49, 38, 65, 97, 76, 13, 27]` 为例）：

```
MergeSort(A, 0, 6)
├── MergeSort(A, 0, 3)
│   ├── MergeSort(A, 0, 1)
│   │   ├── MergeSort(A, 0, 0) → [49]（单元素，直接返回）
│   │   ├── MergeSort(A, 1, 1) → [38]（单元素，直接返回）
│   │   └── Merge(A, 0, 0, 1) → [38, 49]
│   ├── MergeSort(A, 2, 3)
│   │   ├── MergeSort(A, 2, 2) → [65]
│   │   ├── MergeSort(A, 3, 3) → [97]
│   │   └── Merge(A, 2, 2, 3) → [65, 97]
│   └── Merge(A, 0, 1, 3) → [38, 49, 65, 97]
├── MergeSort(A, 4, 6)
│   ├── MergeSort(A, 4, 5)
│   │   ├── MergeSort(A, 4, 4) → [76]
│   │   ├── MergeSort(A, 5, 5) → [13]
│   │   └── Merge(A, 4, 4, 5) → [13, 76]
│   ├── MergeSort(A, 6, 6) → [27]
│   └── Merge(A, 4, 5, 6) → [13, 27, 76]
└── Merge(A, 0, 3, 6) → [13, 27, 38, 49, 65, 76, 97]
```

## 三、深入理解

### 1. 设计思想
- 归并排序采用**分治法**（Divide and Conquer）
- **分（Divide）**：将序列一分为二
- **治（Conquer）**：递归地对两个子序列排序
- **合（Combine）**：将两个有序子序列归并为一个

### 2. 复杂度分析

#### 时间复杂度

**2路归并排序的归并趟数**：

- 2路归并的归并树形态是**倒立的二叉树**
- 树高为 h，则 $n \leq 2^{h-1}$，即 $h-1 = \lceil \log_2 n \rceil$
- **归并趟数 = $\lceil \log_2 n \rceil$**

| 项目 | 复杂度 |
|------|--------|
| 每趟归并 | O(n)（遍历所有元素一次） |
| 归并趟数 | $\lceil \log_2 n \rceil$ |
| **总时间复杂度** | **O(n log₂n)** |
| 最好情况 | O(n log₂n) |
| 最坏情况 | O(n log₂n) |
| 平均情况 | O(n log₂n) |

> **关键**：归并排序的时间复杂度**与初始序列无关**，始终为 O(n log₂n)

#### 空间复杂度

- **O(n)**：来自于辅助数组 B
- 归并排序不是原地排序算法
- 这是归并排序的主要缺点

### 3. 稳定性分析

**归并排序是稳定的！**

**保证稳定性的关键**：在 Merge 函数中，当两个元素相等时，**优先使用靠前的那个**：
```c
if (B[i] <= B[j])    // 注意是 <= ，不是 <
    A[k++] = B[i++];  // 相等时取左边的，保证稳定性
```

### 4. 归并排序 vs 其他 O(n log n) 排序

| 算法 | 最好 | 最坏 | 平均 | 空间 | 稳定性 |
|------|------|------|------|------|--------|
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(n log n) | O(log n) | ❌ 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 |

**归并排序的独特优势**：
1. 时间复杂度稳定，不会退化
2. 是**唯一**稳定的 O(n log n) 排序算法
3. 特别适合**外部排序**和**链表排序**

### 常考题型
1. **手算归并过程**：给定初始序列，写出每一趟的归并结果
2. **两个有序序列的归并**：给定两个有序数组，写出归并过程和结果

4. **递归调用分析**：分析递归调用的执行过程
5. **复杂度计算**：时间复杂度和空间复杂度

### 命题规律
- **归并操作本身**是高频考点（Merge函数的手算过程）
- **2路归并的趟数 = $\lceil \log_2 n \rceil$** 这个结论要记住
- 归并排序是**唯一**稳定的 O(n log n) 排序，常在选择题中考查
- 算法设计题中，归并排序的代码实现是常考内容
- 外部排序本质上也是基于归并的思想

## 五、典型例题

### 例1：手算归并过程
**题目**：对序列 `[49, 38, 65, 97, 76, 13, 27]` 进行2路归并排序，写出每趟的排序结果。

**答案**：
```

第1趟：[38, 49, 65, 97, 13, 76, 27]
第2趟：[38, 49, 65, 97, 13, 27, 76]
第3趟：[13, 27, 38, 49, 65, 76, 97]
```

### 例2：两个有序序列归并
**题目**：将有序序列 `[2, 5, 8, 12]` 和 `[3, 7, 9, 15, 20]` 归并为一个有序序列。

**答案**：
```
A: [2, 5, 8, 12]    B: [3, 7, 9, 15, 20]

比较2和3 → 选2     结果: [2]
比较5和3 → 选3     结果: [2, 3]
比较5和7 → 选5     结果: [2, 3, 5]
比较8和7 → 选7     结果: [2, 3, 5, 7]
比较8和9 → 选8     结果: [2, 3, 5, 7, 8]
比较12和9 → 选9    结果: [2, 3, 5, 7, 8, 9]
比较12和15 → 选12   结果: [2, 3, 5, 7, 8, 9, 12]
A已空，追加B剩余：15, 20
结果: [2, 3, 5, 7, 8, 9, 12, 15, 20]
```

**比较次数**：7次（因为A用完了4个元素，每次比较消耗1个元素）

### 例3：归并趟数计算
**题目**：对16个元素进行2路归并排序，需要几趟归并？

**答案**：
$$\lceil \log_2 16 \rceil = 4 \text{ 趟}$$

### 例4：算法设计题
**题目**：设计一个算法，判断一个数组是否是由两个长度分别为 $\lfloor n/2 \rfloor$ 和 $\lceil n/2 \rceil$ 的有序子数组归并得到的。

**答案**：
```c
bool IsMergeResult(int A[], int n) {
    int mid = n / 2;
    // 检查A[0...mid-1]是否有序
    for (int i = 0; i < mid - 1; i++) {
        if (A[i] > A[i + 1])
            return false;
    }
    // 检查A[mid...n-1]是否有序
    for (int i = mid; i < n - 1; i++) {
        if (A[i] > A[i + 1])
            return false;
    }
    return true;
}
```

### 例5：真题（408统考）
**题目**：2路归并排序中，若初始序列有 n 个元素，则归并的趟数为？

**答案**：$\lceil \log_2 n \rceil$

## 六、易错点

1. **空间复杂度是 O(n)**，不是 O(log n)！辅助数组占 n 个空间
2. **归并排序是稳定的**：关键在于 `<=` 的使用
3. **每趟归并的时间复杂度是 O(n)**，不是 O(n log n)
4. **归并趟数 = $\lceil \log_2 n \rceil$**，不是 $\lfloor \log_2 n \rfloor$
5. **Merge 操作中**，当一个子序列用完后，要将另一个子序列的**剩余元素全部复制**到结果中
6. **时间复杂度与初始序列无关**：最好、最坏、平均都是 O(n log n)
7. **递归实现**需要额外的栈空间 O(log n)，但通常归入时间复杂度分析
