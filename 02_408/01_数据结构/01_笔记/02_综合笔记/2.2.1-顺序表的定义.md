# 顺序表的定义

## 📊 考情分析
- **考试频率**：高频考点
- **分值**：选择题2-4分，综合题可能结合操作考查
- **题型**：概念辨析、存储结构分析、地址计算
- **真题考查方式**：考查顺序表的存储特性、静态/动态分配的区别、地址计算公式

## 一、基本概念

### 1.1 顺序表的定义

**顺序表**——用**顺序存储**的方式实现线性表。

> **顺序存储**：把逻辑上相邻的元素存储在**物理位置上也相邻**的存储单元中，元素之间的关系由存储单元的邻接关系来体现。

### 1.2 关键特征

| 特征 | 说明 |
|------|------|
| **存储方式** | 连续的存储空间 |
| **元素关系** | 由存储单元的邻接关系体现 |
| **随机访问** | 可在 $O(1)$ 时间内找到第 $i$ 个元素 |
| **存储密度高** | 每个节点只存储数据元素 |

## 二、核心知识

### 2.1 静态分配

```c
#define MaxSize 10  // 定义最大长度

typedef struct {
    ElemType data[MaxSize];  // 用静态的"数组"存放数据元素
    int length;              // 顺序表的当前长度
} SqList;  // 顺序表的类型定义（静态分配方式）
```

**特点**：
- 数组大小固定，编译时确定
- 存储空间为 `MaxSize * sizeof(ElemType)`
- 表长确定后**无法更改**
- 如果声明过大的内存空间，会造成浪费

**初始化**：
```c
void InitList(SqList &L) {
    L.length = 0;  // 将长度设为0
}
```

> ⚠️ 内存中可能有遗留的"脏数据"，更好的做法是使用基本操作来访问数据元素

### 2.2 动态分配

```c
#define InitSize 10  // 顺序表的初始长度

typedef struct {
    ElemType *data;   // 指示动态分配数组的指针
    int MaxSize;      // 顺序表的最大容量
    int length;       // 顺序表的当前长度
} SeqList;  // 顺序表的类型定义（动态分配方式）
```

**关键操作**：
```c
// C语言：malloc、free函数
L.data = (ElemType *)malloc(sizeof(ElemType) * InitSize);

// C++：new、delete关键字
L.data = new ElemType[InitSize];
```

**扩容操作**：
```c
void IncreaseSize(SeqList &L, int len) {
    ElemType *p = L.data;
    L.data = (ElemType *)malloc((L.MaxSize + len) * sizeof(ElemType));
    for (int i = 0; i < L.length; i++) {
        L.data[i] = p[i];  // 将数据复制到新区域
    }
    L.MaxSize = L.MaxSize + len;
    free(p);  // 释放原来的内存空间
}
```

**注意**：`realloc` 函数也可实现扩容，但建议初学者使用 `malloc` 和 `free` 更能理解背后过程。

### 2.3 地址计算公式

设线性表第一个元素的存放位置是 $LOC(L)$，每个数据元素占 $k$ 字节：

$$LOC(a_i) = LOC(L) + (i-1) \times k$$

**示例**：
- $a_1$：$LOC(L)$
- $a_2$：$LOC(L) + sizeof(ElemType)$
- $a_3$：$LOC(L) + 2 \times sizeof(ElemType)$

### 2.4 如何获取数据元素大小

```c
typedef struct {
    int num;     // 号数
    int people;  // 人数
} Customer;

sizeof(int)       // = 4B
sizeof(Customer)  // = 8B
```

## 三、深入理解

### 3.1 顺序表的特点

| 特点 | 说明 |
|------|------|
| ✅ 随机访问 | $O(1)$ 时间内找到第 $i$ 个元素 |
| ✅ 存储密度高 | 每个节点只存储数据元素 |
| ❌ 扩展容量不方便 | 即使动态分配，时间复杂度也较高 |
| ❌ 插入删除不方便 | 需要移动大量元素 |

### 3.2 静态分配 vs 动态分配

| 对比项 | 静态分配 | 动态分配 |
|--------|----------|----------|
| 容量 | 固定不变 | 可动态扩展 |
| 空间分配 | 编译时确定 | 运行时分配 |
| 灵活性 | 低 | 高 |
| 实现方式 | 静态数组 | 指针 + malloc/free |

### 3.3 存储密度

**存储密度** = 数据元素本身占用的存储空间 / 结点占用的存储空间

- 顺序表：存储密度 = 1（只存储数据元素）
- 链表：存储密度 < 1（还需要存储指针）

## 五、典型例题

### 计算题

**Q1**：一个顺序表起始地址为 2000，每个元素占 4 字节，求第 10 个元素的地址。

**解答**：
$$LOC(a_{10}) = 2000 + (10-1) \times 4 = 2000 + 36 = 2036$$

### 概念题

**Q2**：下列关于顺序表的说法，错误的是（）
- A. 顺序表支持随机访问
- B. 顺序表的存储密度高
- C. 顺序表插入删除操作的时间复杂度为 $O(1)$
- D. 顺序表需要大片连续的存储空间

**答案**：C

**解析**：顺序表插入删除需要移动元素，时间复杂度为 $O(n)$。

## 六、易错点

1. **位序 vs 下标**：第 $i$ 个元素对应 `data[i-1]`
2. **静态分配容量固定**：表长确定后无法更改
3. **动态分配并非完美**：扩容需要移动所有元素，时间开销大
4. **malloc 返回指针**：需要强制类型转换为 `ElemType *`
