# 栈的顺序存储实现

## 📊 考情分析
- **考试频率**：高，选择题和代码题常考
- **分值**：2~8分
- **题型**：选择题、代码填空题、算法设计题
- **真题考查方式**：顺序栈的操作实现、栈满/栈空判断、共享栈

## 一、基本概念

### 1. 顺序栈的定义
用**顺序存储**方式实现的栈，分配一片连续的存储空间存放栈中元素。

**存储结构**：
```
data[0]  data[1]  data[2]  data[3]  data[4]  data[5]  data[6]  data[7]  data[8]  data[9]
   a        b        c        d        e
                                                         ↑
                                                        top
```
- `top` 指向栈顶元素
- 栈满时 `top == MaxSize - 1`

### 2. 两种实现方式
| 方式 | top初始值 | 栈空条件 | 栈满条件 | 进栈操作 |
|------|----------|---------|---------|---------|
| 方式一 | -1 | top == -1 | top == MaxSize-1 | top++; data[top]=x |
| 方式二 | 0 | top == 0 | top == MaxSize | data[top]=x; top++ |

## 二、核心知识

### 1. 顺序栈的定义（方式一：top指向栈顶元素）
```c
#define MaxSize 10

typedef struct {
    ElemType data[MaxSize];  // 静态数组存放栈中元素
    int top;                 // 栈顶指针
} SqStack;

// 初始化
void InitStack(SqStack *S) {
    S->top = -1;  // 初始化栈顶指针
}

// 判空
bool StackEmpty(SqStack S) {
    return (S.top == -1);
}

// 进栈
bool Push(SqStack *S, ElemType x) {
    if (S->top == MaxSize - 1)  // 栈满
        return false;
    S->top++;                    // 栈顶指针加1
    S->data[S->top] = x;        // 新元素入栈
    return true;
}

// 出栈
bool Pop(SqStack *S, ElemType *x) {
    if (S->top == -1)  // 栈空
        return false;
    *x = S->data[S->top];  // 读取栈顶元素
    S->top--;               // 栈顶指针减1
    return true;
}

// 读栈顶元素
bool GetTop(SqStack S, ElemType *x) {
    if (S.top == -1)  // 栈空
        return false;
    *x = S.data[S.top];  // 读取栈顶元素
    return true;
}
```

**注意**：出栈操作只是移动栈顶指针，数据还残留在内存中，只是逻辑上被删除了。

### 2. 顺序栈的定义（方式二：top指向栈顶元素的下一个位置）
```c
#define MaxSize 10

typedef struct {
    ElemType data[MaxSize];
    int top;  // top指向下一个可以插入的位置
} SqStack2;

// 初始化
void InitStack2(SqStack2 *S) {
    S->top = 0;  // 初始化栈顶指针为0
}

// 判空
bool StackEmpty2(SqStack2 S) {
    return (S.top == 0);
}

// 进栈
bool Push2(SqStack2 *S, ElemType x) {
    if (S->top == MaxSize)  // 栈满
        return false;
    S->data[S->top] = x;   // 新元素入栈
    S->top++;               // 栈顶指针加1
    return true;
}

// 出栈
bool Pop2(SqStack2 *S, ElemType *x) {
    if (S->top == 0)  // 栈空
        return false;
    S->top--;               // 栈顶指针减1
    *x = S->data[S->top];  // 读取栈顶元素
    return true;
}
```

### 3. 共享栈
**定义**：两个栈共享同一片存储空间，一个栈从数组头部向后增长，另一个栈从数组尾部向前增长。

```c
#define MaxSize 10

typedef struct {
    ElemType data[MaxSize];
    int top0;  // 0号栈栈顶指针
    int top1;  // 1号栈栈顶指针
} ShStack;

// 初始化
void InitShStack(ShStack *S) {
    S->top0 = -1;      // 0号栈栈顶初始化为-1
    S->top1 = MaxSize;  // 1号栈栈顶初始化为MaxSize
}

// 0号栈进栈
bool Push0(ShStack *S, ElemType x) {
    if (S->top0 + 1 == S->top1)  // 栈满
        return false;
    S->top0++;
    S->data[S->top0] = x;
    return true;
}

// 1号栈进栈
bool Push1(ShStack *S, ElemType x) {
    if (S->top0 + 1 == S->top1)  // 栈满
        return false;
    S->top1--;
    S->data[S->top1] = x;
    return true;
}

// 栈满条件：top0 + 1 == top1
// 0号栈空：top0 == -1
// 1号栈空：top1 == MaxSize
```

## 三、深入理解

### 1. 设计思想
顺序栈利用数组的连续存储特性，通过栈顶指针 `top` 来管理栈中元素。

**优点**：
- 存储密度高
- 随机存取（虽然栈不需要）

**缺点**：
- 栈的大小不可变（静态分配）
- 可能浪费空间

### 2. 复杂度分析
| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 初始化 | O(1) | 设置top指针 |
| 判空 | O(1) | 比较top |
| 进栈 | O(1) | top++，赋值 |
| 出栈 | O(1) | 取值，top-- |
| 读栈顶 | O(1) | 取值 |

**所有操作都是O(1)时间复杂度！**

## 五、典型例题

### 概念题
**题目**：若用一个数组data[0..MaxSize-1]存储栈，且top指向栈顶元素，则栈满的条件是（）
A. top == 0  B. top == MaxSize-1  C. top == MaxSize  D. top == -1

**答案**：B
**解析**：top指向栈顶元素，栈满时top达到数组最大下标MaxSize-1。

### 计算题
**题目**：一个栈的入栈序列为1,2,3,4,5，则可能的出栈序列是（）
A. 5,4,3,2,1  B. 1,2,3,4,5  C. 1,3,2,5,4  D. 以上都是

**答案**：D
**解析**：所有选项都是合法的出栈序列。

### 算法题
**题目**：设计一个算法，利用栈判断一个字符串是否是回文。

```c
bool IsPalindrome(char str[], int n) {
    SqStack S;
    InitStack(&S);
    
    // 将前半部分字符入栈
    int mid = n / 2;
    for (int i = 0; i < mid; i++) {
        Push(&S, str[i]);
    }
    
    // 从中间位置开始与栈顶元素比较
    int start = (n % 2 == 0) ? mid : mid + 1;
    for (int i = start; i < n; i++) {
        ElemType topElem;
        Pop(&S, &topElem);
        if (str[i] != topElem)
            return false;
    }
    
    return true;
}
```

## 六、易错点

1. **top指针的含义**：
   - 方式一：top指向栈顶元素，初始为-1
   - 方式二：top指向下一个可插入位置，初始为0
   - ❌ 混淆两种方式的栈满/栈空条件

2. **出栈操作**：
   - 出栈只是移动top指针，数据还残留在内存中
   - ❌ 认为出栈会清除数据

3. **共享栈的栈满条件**：
   - `top0 + 1 == top1`
   - ❌ 认为 `top0 == top1` 时栈满

4. **注意审题**：
   - 题目可能使用不同的top指针定义方式
   - 需要根据题目条件判断
