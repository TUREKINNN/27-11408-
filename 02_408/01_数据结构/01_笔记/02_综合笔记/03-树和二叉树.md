## 一、基本概念

### 1.1 树的定义

树是 n（n≥0）个结点的有限集合：
- n=0 时称为**空树**
- n>0 时，有且仅有一个**根结点**，其余结点可分为 m（m≥0）个互不相交的有限集合，每个集合本身又是一棵树（称为子树）

### 1.2 基本术语

| 术语 | 定义 |
|------|------|
| **度** | 结点拥有的子树个数。树的度 = max(各结点度) |
| **叶子结点（终端结点）** | 度为 0 的结点 |
| **层次** | 根为第 1 层，其子结点为第 2 层，依此类推 |
| **深度（高度）** | 树中结点的最大层次 |
| **森林** | m（m≥0）棵互不相交的树的集合 |
| **有序树/无序树** | 子树之间有/无次序关系 |
| **路径长度** | 路径上经过的边数 |

## 二、二叉树

### 2.1 定义与五种基本形态

二叉树是 n（n≥0）个结点的有限集合，或为空集，或由一个根结点和两棵互不相交的、分别称为**左子树**和**右子树**的二叉树组成。

五种基本形态：
```
① 空二叉树    ② 只有根    ③ 根+左子树    ④ 根+右子树    ⑤ 根+左+右
```

> ⚠️ 二叉树的子树有左右之分，不可颠倒，这是与度为2的树的本质区别。

### 2.2 满二叉树与完全二叉树

**满二叉树**：深度为 k 且有 2^k - 1 个结点的二叉树。每层都满。

**完全二叉树**：深度为 k、有 n 个结点的二叉树，当且仅当其每个结点都与深度为 k 的满二叉树中编号 1~n 的结点一一对应。

完全二叉树的特点：
1. 叶子结点只可能在最后两层出现
2. 若有度为1的结点，最多只有**1个**，且该结点只有**左孩子**
3. 按层序编号后，对任意结点 i：
   - 左孩子编号 = 2i（若 2i ≤ n）
   - 右孩子编号 = 2i+1（若 2i+1 ≤ n）
   - 双亲编号 = ⌊i/2⌋（若 i > 1）

### 2.3 二叉树五大性质

> **02-线性表** 中栈和队列是实现遍历的基础。

**性质1**：第 i 层最多有 2^(i-1) 个结点（i≥1）

**性质2**：深度为 k 的二叉树最多有 2^k - 1 个结点

**性质3**：对任意二叉树 T，若叶子结点数为 n₀，度为2的结点数为 n₂，则：
$$n_0 = n_2 + 1$$

**证明**：
设 n₁ 为度为1的结点数，n = n₀ + n₁ + n₂（总结点数）
设 B 为总分支数（边数），则 B = n - 1（根无入边，其余各一个）
又 B = 0·n₀ + 1·n₁ + 2·n₂
联立：n₀ + n₁ + n₂ - 1 = n₁ + 2n₂
化简得：**n₀ = n₂ + 1** ∎

**性质4**：具有 n 个结点的完全二叉树深度为 ⌊log₂n⌋ + 1

**性质5**：完全二叉树中，对结点 i（1≤i≤⌊n/2⌋）：
- i 有左孩子 ⟺ 2i ≤ n
- i 有右孩子 ⟺ 2i+1 ≤ n
- i 有双亲 ⟺ i > 1

## 三、存储结构

### 3.1 顺序存储

按完全二叉树的层序编号存入数组，适用于完全二叉树。

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 从下标1开始存储
    int n;                   // 结点个数
} SqBiTree;
```

> 缺点：普通二叉树会造成大量空间浪费（最坏 O(2^n)）。

### 3.2 链式存储（二叉链表）

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild;
} BiTNode, *BiTree;
```

n 个结点的二叉链表中，有 n+1 个空指针域（2n 个指针域，n-1 个用于指向孩子）。

## 四、二叉树遍历

### 4.1 递归遍历

```c
// 前序遍历：根→左→右
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);            // 访问根结点
        PreOrder(T->lchild); // 递归遍历左子树
        PreOrder(T->rchild); // 递归遍历右子树
    }
}

// 中序遍历：左→根→右
void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);
        visit(T);
        InOrder(T->rchild);
    }
}

// 后序遍历：左→右→根
void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);
        PostOrder(T->rchild);
        visit(T);
    }
}
```

**时间复杂度**：O(n)，每个结点恰好被访问一次
**空间复杂度**：O(h)，h 为树的高度（递归栈深度）

### 4.2 非递归中序遍历（栈实现）

```c
void InOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !IsEmpty(S)) {
        if (p) {
            Push(S, p);        // 根指针进栈
            p = p->lchild;     // 遍历左子树
        } else {
            Pop(S, p);         // 栈顶弹出
            visit(p);          // 访问根结点
            p = p->rchild;     // 转向右子树
        }
    }
}
```

### 4.3 非递归先序遍历

```c
void PreOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !IsEmpty(S)) {
        if (p) {
            visit(p);          // 先访问
            Push(S, p);
            p = p->lchild;
        } else {
            Pop(S, p);
            p = p->rchild;
        }
    }
}
```

### 4.4 非递归后序遍历

```c
void PostOrder_NonRec(BiTree T) {
    InitStack(S);
    BiTree p = T, r = NULL;  // r记录上次访问的结点
    while (p || !IsEmpty(S)) {
        if (p) {
            Push(S, p);
            p = p->lchild;
        } else {
            GetTop(S, p);
            if (p->rchild && p->rchild != r) {  // 右子树存在且未访问
                p = p->rchild;
            } else {
                Pop(S, p);
                visit(p);
                r = p;
                p = NULL;
            }
        }
    }
}
```

### 4.5 层序遍历（队列实现）

```c
void LevelOrder(BiTree T) {
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);
    while (!IsEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        if (p->lchild) EnQueue(Q, p->lchild);
        if (p->rchild) EnQueue(Q, p->rchild);
    }
}
```

## 五、由遍历序列还原二叉树

> 关键结论：**先序+中序** 或 **后序+中序** 可以唯一确定一棵二叉树。先序+后序**不能**唯一确定。

### 5.1 先序 + 中序还原

**步骤**：
1. 先序序列的**第一个**元素是根结点
2. 在中序序列中找到根结点，其**左边**是左子树中序，**右边**是右子树中序
3. 根据左子树结点数，在先序序列中划分出左子树先序和右子树先序
4. 递归处理左右子树

**示例**：
- 先序：A B D E C F
- 中序：D B E A F C

```
步骤1: 先序首元素 A 为根
步骤2: 中序中 A 左边 D B E 为左子树，F C 为右子树
步骤3: 先序中左子树3个结点 → 左先序 B D E，右先序 C F
步骤4: 递归...
```

### 5.2 后序 + 中序还原

**步骤**：
1. 后序序列的**最后一个**元素是根结点
2. 在中序中划分左右子树
3. 根据左子树结点数划分后序序列
4. 递归处理

## 六、线索二叉树

### 6.1 概念

普通二叉链表中，n 个结点有 n+1 个空指针域。利用这些空指针域存放**前驱**和**后继**信息，称为线索。加了线索的二叉树称为**线索二叉树**。

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 0: 指向孩子  1: 指向线索
} ThreadNode, *ThreadTree;
```

### 6.2 中序线索化

```c
ThreadNode *pre = NULL;  // 全局变量，记录前驱

void InThread(ThreadTree T) {
    if (T != NULL) {
        InThread(T->lchild);
        if (T->lchild == NULL) {      // 左子树为空，建前驱线索
            T->ltag = 1;
            T->lchild = pre;
        }
        if (pre && pre->rchild == NULL) {  // 前驱右子树为空，建后继线索
            pre->rtag = 1;
            pre->rchild = T;
        }
        pre = T;
        InThread(T->rchild);
    }
}

void CreateInThread(ThreadTree T) {
    pre = NULL;
    if (T != NULL) {
        InThread(T);
        pre->rchild = NULL;  // 处理最后一个结点
        pre->rtag = 1;
    }
}
```

### 6.3 中序线索化后遍历

```c
// 找中序下的第一个结点
ThreadNode *FirstNode(ThreadNode *p) {
    while (p->ltag == 0) p = p->lchild;
    return p;
}

// 找后继
ThreadNode *NextNode(ThreadNode *p) {
    if (p->rtag == 0) return FirstNode(p->rchild);
    return p->rchild;
}

// 中序遍历线索二叉树
void InOrder_Thread(ThreadTree T) {
    for (ThreadNode *p = FirstNode(T); p != NULL; p = NextNode(p))
        visit(p);
}
```

## 七、树的存储结构

### 7.1 双亲表示法

```c
typedef struct {
    ElemType data;
    int parent;  // 双亲下标
} PTNode;

typedef struct {
    PTNode nodes[MaxSize];
    int n;  // 结点数
} PTree;
```

### 7.2 孩子表示法

```c
typedef struct CTNode {
    int child;            // 孩子下标
    struct CTNode *next;
} *ChildPtr;

typedef struct {
    ElemType data;
    ChildPtr firstchild;  // 孩子链表头
} CTBox;

typedef struct {
    CTBox nodes[MaxSize];
    int n, r;             // 结点数、根位置
} CTree;
```

### 7.3 孩子兄弟表示法（二叉链表表示法）

```c
typedef struct CSNode {
    ElemType data;
    struct CSNode *firstchild, *nextsibling;  // 第一个孩子、右兄弟
} CSNode, *CSTree;
```

> 这是树转二叉树的理论基础。

## 八、森林与二叉树转换

### 8.1 树 → 二叉树

1. 加线：兄弟之间加线（同一层的结点连起来）
2. 去线：只保留每个结点与**第一个孩子**的连线，去掉与其他孩子的连线
3. 层次调整：第一个孩子为左孩子，兄弟为右孩子

### 8.2 森林 → 二叉树

1. 将每棵树转为二叉树
2. 第一棵树的根作为最终二叉树的根
3. 后一棵树作为前一棵树根的**右子树**依次连接

### 8.3 二叉树 → 森林

逆过程：从根开始，沿右孩子链断开，每棵子树还原为树。

## 九、树和森林的遍历

| 树/森林 | 对应二叉树 |
|---------|-----------|
| 先根遍历 | 先序遍历 |
| 后根遍历 | 中序遍历 |
| 层次遍历 | — |

> ⚠️ 树的先根遍历 = 二叉树先序遍历；树的后根遍历 = 二叉树中序遍历（非后序！）

## 十、哈夫曼树与哈夫曼编码

### 10.1 基本概念

- **带权路径长度（WPL）**：WPL = Σ(wᵢ × lᵢ)，wᵢ 为权值，lᵢ 为路径长度
- **哈夫曼树**：WPL 最小的二叉树，又称最优二叉树
- **前缀编码**：任何一个编码都不是另一个编码的前缀

### 10.2 哈夫曼树构造步骤

1. 将 n 个权值看做 n 棵只有根结点的树，构成森林 F
2. 在 F 中选取**两棵权值最小**的树作为新结点的左右子树，新结点权值 = 左 + 右
3. 从 F 中删除这两棵树，加入新树
4. 重复 2-3，直到 F 中只剩一棵树

### 10.3 完整例题

**题目**：字符集 {A, B, C, D, E}，权值（频率）{5, 29, 7, 8, 14}，构造哈夫曼编码。

**解答**：

```
初始: {5, 7, 8, 14, 29}

第1步: 取5和7合并 → 12
       {8, 12, 14, 29}

第2步: 取8和12合并 → 20
       {14, 20, 29}

第3步: 取14和20合并 → 34
       {29, 34}

第4步: 取29和34合并 → 63
       {63}

       63
      /  \
    29    34
         /  \
       14    20
            /  \
           8    12
               /  \
              5    7
```

**编码**（左0右1）：
| 字符 | 权值 | 编码 |
|------|------|------|
| A | 5 | 0110 |
| B | 29 | 0 |
| C | 7 | 0111 |
| D | 8 | 010 |
| E | 14 | 10 |

WPL = 5×4 + 29×1 + 7×4 + 8×3 + 14×2 = 20 + 29 + 28 + 24 + 28 = **129**

> ⚠️ 哈夫曼树中**没有度为1的结点**，n 个叶子结点的哈夫曼树共有 2n-1 个结点。

## 十一、并查集

并查集用于处理**不相交集合的合并与查询**问题。

```c
int UFSets[Size];  // 集合元素数组

// 初始化：每个元素自成一个集合
void Initial(int S[], int n) {
    for (int i = 0; i < n; i++) S[i] = -1;
}

// Find：查找元素所属集合（根）
int Find(int S[], int x) {
    while (S[x] >= 0) x = S[x];
    return x;
}

// Union：合并两个集合
void Union(int S[], int Root1, int Root2) {
    S[Root2] = Root1;  // Root2挂到Root1下
}
```

**优化**：
- **Find 路径压缩**：查找时将路径上所有结点直接挂到根下
- **Union 按秩合并**：将小树挂到大树下

```c
int Find(int S[], int x) {
    int root = x;
    while (S[root] >= 0) root = root;
    while (x != root) {  // 路径压缩
        int t = S[x];
        S[x] = root;
        x = t;
    }
    return root;
}
```

> 并查集常用于判断图的连通性、Kruskal算法等。见 [[02_408/01_数据结构/01_笔记/02_综合笔记/04-图]]。

1. **手写遍历算法**（递归/非递归）
2. **二叉树还原**并写出某种遍历序列
3. **构造哈夫曼树**并求编码和WPL

## 典型例题

### 例题1：二叉树性质

**题目**：一棵完全二叉树有 1001 个结点，求叶子结点数。

**思路**：利用完全二叉树编号关系。

**解答**：
- n = 1001，最后一个结点编号 1001
- 双亲编号 = ⌊1001/2⌋ = 500
- 编号 501~500 的结点：编号 501 到 1001 共 501 个
- 度为1的结点：只有编号为 500 的结点可能（因为它有左孩子1000但无右孩子1001）
- 实际：n=1001 为奇数，所以 n₁=0
- n₀ = n₂ + 1，n = n₀ + n₁ + n₂ = 2n₀ - 1 = 1001
- **n₀ = 501**

**总结**：n 为偶数时 n₁=1，n 为奇数时 n₁=0。n₀ = ⌈n/2⌉。

### 例题2：遍历序列还原

**题目**：已知先序 ABDECFG，中序 DBEACFG，画出二叉树并求后序。

**思路**：先序首元素为根，在中序中划分左右子树。

**解答**：
```
先序: A B D E C F G    中序: D B E A C F G

A 为根 → 中序中 A 左: D B E (左子树), 右: C F G (右子树)
左子树: 先序 B D E, 中序 D B E → B为根 → 左D, 右E
右子树: 先序 C F G, 中序 C F G → C为根 → 左空, 右F G
右右子树: 先序 F G, 中序 F G → F为根 → 左空, 右G

        A
       / \
      B   C
     / \   \
    D   E   F
             \
              G

后序: D E B G F C A
```

**总结**：先序定根，中序分左右，递归处理。

### 例题3：哈夫曼编码

**题目**：对频率为 A:3 B:5 C:6 D:9 E:11 F:12 的字符构造哈夫曼编码，求 WPL。

**思路**：每次取两个最小权值合并。

**解答**：
```
初始: {3, 5, 6, 9, 11, 12}
3+5=8  → {6, 8, 9, 11, 12}
6+8=14 → {9, 11, 12, 14}
9+11=20 → {12, 14, 20}
12+14=26 → {20, 26}
20+26=46

          46
         /  \
       20    26
      / \   /  \
    9   11 12  14
              /  \
             6    8
                 / \
                3   5
```

WPL = 3×4 + 5×4 + 6×3 + 9×2 + 11×2 + 12×2 = 12+20+18+18+22+24 = **114**

**总结**：构造时始终选最小两个合并，保证 WPL 最小。

### 例题4：非递归后序遍历

**题目**：写出后序非递归遍历的关键思路。

**思路**：后序遍历的难点在于需要判断右子树是否已访问。

**解答**：
核心变量：栈 S，指针 p，标记 r（上一个访问的结点）

1. 沿左子树一路入栈
2. 取栈顶：若右子树**存在且未访问**，转向右子树
3. 否则弹出栈顶并访问，r = p，p = NULL

**总结**：后序非递归的关键是用 r 记录上次访问的结点，判断右子树是否已处理。考试中这是高频编程题。

## 易错点

1. ⚠️ **度为2的树 ≠ 二叉树**：二叉树子树有左右之分，度为2的树无序
2. ⚠️ **n₀ = n₂ + 1 只对二叉树成立**，不能推广到一般树
3. ⚠️ **先序+后序不能唯一确定二叉树**（必须有中序）
4. ⚠️ 树的**后根遍历** = 二叉树的**中序遍历**（非后序！）
5. ⚠️ 哈夫曼树中**没有度为1的结点**
6. ⚠️ 完全二叉树中度为1的结点最多1个，且只有左孩子
7. ⚠️ 非递归后序遍历需要额外标记 r，是常见编程题易错点
8. ⚠️ 并查集 Find 必须做路径压缩，否则退化为 O(n)

### 补充1：树与二叉树转换的详细技巧

#### 树 → 二叉树的转换口诀

转换步骤：
1. 先在二叉树中画一个根节点
2. 按"树的层序"依次处理每个结点
3. 处理方法：如果当前结点在树中有孩子，就把所有孩子结点"用右指针串成糖葫芦"，并在二叉树中把第一个孩子挂在当前结点的左指针下方

**本质**：孩子兄弟表示法（左孩子右兄弟）
- 左指针 → 指向第一个孩子
- 右指针 → 指向右兄弟

#### 森林 → 二叉树的转换

关键：森林中各棵树的根节点视为**平级的兄弟关系**。

步骤：
1. 将森林中每棵树各自转换为二叉树
2. 第一棵树的根作为最终二叉树的根
3. 后面每棵树的根作为前一棵树根的右子树

#### 二叉树 → 树/森林的逆转换

逆过程：从根开始，沿右孩子链断开，每棵子树还原为树。

### 补充2：哈夫曼树的深入理解

#### 哈夫曼树的重要性质

1. **没有度为1的结点**：因为每次合并都产生一个新的度为2的结点
2. **n个叶子结点的哈夫曼树共有 2n-1 个结点**
3. 哈夫曼树**不唯一**（当权值相同时，左右子树可以交换），但 WPL 唯一

#### 哈夫曼编码的前缀性质

哈夫曼编码是**前缀编码**：任何一个编码都不是另一个编码的前缀。

这意味着：在译码时不会产生歧义，可以从左到右逐位扫描即可唯一确定每个字符。

**构造方法**：
- 从根到叶子的路径即为该字符的编码
- 通常约定：左分支为 0，右分支为 1（也可反过来，但必须统一）

#### WPL 的两种计算方法

**方法1**（按定义）：
$$WPL = \sum_{i=1}^{n} w_i \times l_i$$
其中 $w_i$ 是权值，$l_i$ 是从根到叶子的路径长度。

**方法2**（按合并过程）：
$$WPL = \sum_{\text{所有非叶子结点}} \text{该结点的权值}$$

因为每次合并产生的新结点权值 = 两个子树权值之和，而这些权值最终都会被累加到 WPL 中。

### 补充3：并查集的优化

#### 基本并查集的问题

基本的 Find 操作可能退化为 O(n)（树退化为链）。

#### 优化1：按秩合并（Union by Rank）

合并时将**矮树**挂在**高树**下面，避免树过高。

```c
void Union(int S[], int Root1, int Root2) {
    if (S[Root2] < S[Root1])  // Root2 更深（负数更大）
        S[Root1] = Root2;     // 把 Root1 挂到 Root2
    else {
        if (S[Root1] == S[Root2])
            S[Root1]--;       // 高度增加1
        S[Root2] = Root1;     // 把 Root2 挂到 Root1
    }
}
```

#### 优化2：路径压缩（Path Compression）

Find 时将沿途所有结点直接挂到根下面：

```c
int Find(int S[], int x) {
    int root = x;
    while (S[root] >= 0) root = S[root];  // 找根
    while (x != root) {                    // 路径压缩
        int t = S[x];
        S[x] = root;
        x = t;
    }
    return root;
}
```

**时间复杂度**：同时使用两种优化后，Find 和 Union 操作的均摊时间复杂度为 $O(\alpha(n))$，其中 $\alpha$ 是反阿克曼函数，实际可视为常数。

### 补充4：二叉树的遍历序列特征

**由遍历序列判断结点关系**：

若已知先序遍历为 `A B D E C F`，中序遍历为 `D B E A F C`：
- 先序的第一个 `A` 是根
- 中序中 `A` 左边 `D B E` 是左子树，右边 `F C` 是右子树
- 对左子树：先序 `B D E`，中序 `D B E` → 根是 B
- 递归继续...

**重要考点**：给定先序+后序，**不能**唯一确定二叉树。因为无法区分左右子树。

**反例**：先序 AB，后序 BA → 可以是 A 有左孩子 B，也可以是 A 有右孩子 B。
