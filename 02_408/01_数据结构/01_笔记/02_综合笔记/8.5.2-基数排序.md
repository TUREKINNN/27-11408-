# 基数排序

## 📊 考情分析
- 考试频率：中频考点，每隔1~2年出现
- 分值：2~5分（以选择题为主）
- 题型：选择题（排序过程判断、复杂度分析、稳定性）、综合应用题（手算分配收集过程）
- 真题考查方式：分配收集过程手算、时间复杂度分析、基数排序的适用场景判断

## 一、基本概念

### 1. 严格定义
**基数排序**（Radix Sort）是一种**不基于"比较"**的排序算法。它将关键字拆分为若干组，按照各个关键字位权重递增的次序，对每组关键字分别进行"分配"和"收集"。

### 2. 基本术语
- **基数（r）**：每个关键字位可能取得的值的个数。如十进制数的基数 r=10
- **关键字位**：将关键字拆分为 d 个部分，每个部分称为一个关键字位
- **分配（Distribute）**：按当前关键字位的值，将元素分配到对应的队列中
- **收集（Collect）**：将各队列中的元素按顺序依次出队并链接
- **最高位优先（MSD）**：从最高位开始排序
- **最低位优先（LSD）**：从最低位开始排序（**考试中通常采用LSD**）

### 3. 关键字的数学表示

假设长度为 n 的线性表中每个结点的关键字由 d 元组组成：

$$(k_j^{d-1}, k_j^{d-2}, \ldots, k_j^1, k_j^0)$$

其中，$0 \leq k_j^i \leq r-1$（$0 \leq j < n, 0 \leq i \leq d-1$），r 称为**基数**。

> 例如：十进制数 520 可以表示为 $(5, 2, 0)$，其中 d=3，r=10

## 二、核心知识

### 1. 最低位优先（LSD）基数排序过程

**初始序列**：`520, 211, 438, 888, 007, 111, 985, 666, 996, 233, 168`

**要求**：得到按关键字**递减**的有序序列

#### 第一趟：以"个位"进行分配、收集

**分配**：设置 r=10 个队列 Q₀~Q₉，按个位数字分配

```
Q₉  Q₈   Q₇  Q₆   Q₅  Q₄  Q₃   Q₂  Q₁   Q₀
    438  007  666  985      233      211  520
    888       996            168
    168
```

**收集**：从 Q₉ 到 Q₀ 依次出队

**结果**：`438, 888, 168, 007, 666, 996, 985, 233, 211, 111, 520`
（按个位递减排序）

#### 第二趟：以"十位"进行分配、收集

**分配**：

```
Q₉  Q₈   Q₇  Q₆   Q₅  Q₄  Q₃   Q₂  Q₁   Q₀
    985       666       438  233      211  007
    888       168            520      111
    996
```

**收集**：从 Q₉ 到 Q₀ 依次出队

**结果**：`996, 888, 985, 168, 666, 438, 233, 520, 211, 111, 007`
（按十位递减排序，十位相同的按个位递减排列）

#### 第三趟：以"百位"进行分配、收集

**分配**：

```
Q₉  Q₈  Q₇  Q₆  Q₅   Q₄  Q₃  Q₂   Q₁   Q₀
996 888     666 520  438      233  168   007
985                        211  111
```

**收集**：从 Q₉ 到 Q₀ 依次出队

**结果**：`996, 985, 888, 666, 520, 438, 233, 211, 168, 111, 007`
（按百位递减排序，百位相同的按十位递减、个位递减排列）

**最终结果**：`996, 985, 888, 666, 520, 438, 233, 211, 168, 111, 007` ✓

### 2. 排序规则总结

- **LSD（最低位优先）**：按关键字位权重**递增**的次序（个→十→百）
- **递减排序**：收集时从 Q_{r-1} 到 Q₀（从大到小）
- **递增排序**：收集时从 Q₀ 到 Q_{r-1}（从小到大）
- 每一趟的结果都是**按当前关键字位有序**，且**关键字位相同**的元素保持上一趟的相对顺序（稳定性保证）

### 3. 代码实现（C语言）

```c
// 基数排序（LSD，递增排序）
// 假设关键字为十进制正整数，最多d位

#define MAX_DIGIT 10   // 最大位数
#define RADIX     10   // 基数（十进制）

// 链队列结点
typedef struct Node {
    int data;
    struct Node *next;
} Node;

// 链队列
typedef struct {
    Node *front;
    Node *rear;
} LinkQueue;

// 获取关键字的第digit位（从0开始，0表示个位）
int GetDigit(int num, int digit) {
    for (int i = 0; i < digit; i++)
        num /= 10;
    return num % 10;
}

// 基数排序
void RadixSort(int A[], int n, int d) {
    LinkQueue Q[RADIX];
    
    // 初始化r个队列
    for (int i = 0; i < RADIX; i++) {
        Q[i].front = Q[i].rear = NULL;
    }
    
    // d趟分配、收集
    for (int digit = 0; digit < d; digit++) {
        // 分配：顺序扫描各个元素
        for (int i = 0; i < n; i++) {
            int idx = GetDigit(A[i], digit);
            // 将A[i]插入Q[idx]队尾（入队操作）
            Enqueue(&Q[idx], A[i]);
        }
        
        // 收集：从Q[0]到Q[r-1]依次出队
        int k = 0;
        for (int i = 0; i < RADIX; i++) {
            while (!IsEmpty(&Q[i])) {
                A[k++] = Dequeue(&Q[i]);
            }
        }
    }
}
```

**代码关键点**：
- 基数排序通常基于**链式存储**实现
- 每趟需要 r 个辅助队列
- 分配过程：O(n)（扫描所有元素）
- 收集过程：O(r)（遍历所有队列）

## 三、深入理解

### 1. 设计思想
- 基数排序是一种**分配式排序**，不基于比较
- 核心思想是**按位分配、按位收集**
- 从最低位开始，每一趟都使得序列在当前关键字位上有序
- 通过 d 趟操作，最终在所有关键字位上都有序

### 2. 复杂度分析

#### 时间复杂度
- 一趟分配：O(n)（扫描所有 n 个元素）
- 一趟收集：O(r)（遍历所有 r 个队列）
- 共 d 趟
- **总时间复杂度 = O(d(n + r))**

#### 空间复杂度
- 需要 r 个辅助队列
- **空间复杂度 = O(r)**（队列中的指针空间不计）

### 3. 稳定性分析

**基数排序是稳定的！**

**证明**：
- 关键字相同的元素在分配时被放入同一个队列
- 在同一个队列中，**先入队的元素先出队**（FIFO）
- 先入队的元素在原始序列中排在前面
- 因此排序后关键字相同的元素的**相对顺序不变** → 稳定

### 4. 基数排序的适用场景

基数排序擅长解决的问题：
1. ✅ 数据元素的关键字可以方便地拆分为 d 组，且 **d 较小**
2. ✅ 每组关键字的取值范围不大，即 **r 较小**
3. ✅ 数据元素个数 **n 较大**

**反例**：
- ❌ 给5个人的身份证号排序（d=18，太大）
- ❌ 给中文人名排序（每个字可能有上万种取值，r太大）
- ✅ 给**十亿人**的身份证号排序（n很大，d=18，r=10）

### 5. 基数排序的应用示例

**题目**：某学校有10000学生，将学生信息按年龄递减排序。

**分析**：
- 生日可拆分为三组关键字：年(1991~2005)、月(1~12)、日(1~31)
- d=3，r≈30
- 时间复杂度 = O(d(n+r)) = O(3 × 10030) ≈ **O(30000)**
- 若用 O(n²) 排序 ≈ O(10⁸)
- 若用 O(n log n) 排序 ≈ O(140000)

**结论**：基数排序在 d 小、r 小、n 大时效率极高！

## 四、真题考法分析

### 常考题型
1. **分配收集过程手算**：给定初始序列和排序要求，手写每趟的分配和收集结果
2. **时间复杂度计算**：O(d(n+r)) 的理解和应用
3. **稳定性判断**：基数排序是稳定的（常考选择题）
4. **适用场景判断**：判断什么情况下使用基数排序
5. **趟数计算**：d 个关键字位就需要 d 趟

### 命题规律
- 基数排序**不是基于比较**的排序，这个性质常在选择题中考查
- **LSD 和 MSD** 的区别偶尔会考
- 基数排序的**稳定性**是必考知识点
- 有时会结合外部排序考查

### 例2：复杂度分析题
**题目**：对 n=1000 个3位十进制正整数进行基数排序，时间复杂度约为多少？

**答案**：
- d=3（3位数），r=10（十进制）
- O(d(n+r)) = O(3 × 1010) = O(3030)
- 比 O(n²) = O(1000000) 快得多
- 比 O(n log n) = O(10000) 也更快

### 例3：适用场景判断
**题目**：以下哪种情况最适合使用基数排序？
A. 对100个2位数排序
B. 对10亿个18位身份证号排序
C. 对10000个字符串排序
D. 对100万个实数排序

**答案**：B
- A：d=2, r=10, n=100，适合但 n 不够大
- B：d=18, r=10, n=10⁹，d和r都不大，n很大，**最适合**
- C：r太大（字符集太大），不适合
- D：实数不方便拆分为固定位数，不适合

### 例4：稳定性判断
**题目**：基数排序是否稳定？为什么？

**答案**：**稳定**。
- 关键字相同的元素在分配时进入同一个队列
- 同一队列中保持先进先出（FIFO）
- 先进入队列的元素在原始序列中位置靠前
- 因此排序后关键字相同的元素相对顺序不变

## 六、易错点

1. **基数排序不是基于比较的排序**：时间复杂度不受 O(n log n) 下界约束
2. **LSD 的处理顺序**：从最低位到最高位，不是反过来
3. **收集顺序决定排序方向**：
   - 递增排序：从 Q₀ 到 Q_{r-1} 收集
   - 递减排序：从 Q_{r-1} 到 Q₀ 收集
4. **d 趟分配收集**：d 个关键字位就需要 d 趟
5. **空间复杂度是 O(r)**：需要 r 个辅助队列
6. **基数排序是稳定的**：这是它的重要特性
7. **适用条件**：d 小、r 小、n 大
