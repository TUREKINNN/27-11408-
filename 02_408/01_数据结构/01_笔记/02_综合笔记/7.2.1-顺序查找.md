# 7.2.1 顺序查找

## 一、算法思想

顺序查找，又叫"**线性查找**"，通常用于**线性表**。

**算法思想**：从头到尾（或从尾到头）逐个比较关键字，直到找到目标元素或遍历完整个表。

---

## 二、基本实现

### 1. 普通实现

```c
// 在顺序表中查找关键字 key
int Search(SSTable ST, KeyType key) {
    int i;
    for (i = 0; i < ST.length; i++) {
        if (ST.elem[i].key == key)
            return i;  // 查找成功，返回下标
    }
    return -1;  // 查找失败
}
```

**时间复杂度**：$O(n)$

### 2. 带哨兵的实现（优化）

```c
// 数据从下标 1 开始存储，0 号位置存放"哨兵"
int Search(SSTable ST, KeyType key) {
    ST.elem[0].key = key;  // 设置哨兵
    int i = ST.length;
    while (ST.elem[i].key != key) {
        i--;
    }
    return i;  // 返回 0 表示查找失败
}
```

**优点**：无需判断是否越界（`i >= 0`），每次循环少一次比较，**效率更高**。

---

## 三、查找效率分析

### 1. 查找成功

假设每个元素被查概率相同（$p_i = 1/n$）：

$$ASL_{成功} = \sum_{i=1}^{n} \frac{1}{n} \cdot i = \frac{1}{n} \cdot \frac{n(n+1)}{2} = \frac{n+1}{2}$$

### 2. 查找失败

无论目标关键字是什么，都必须遍历整个表才能确定失败：

$$ASL_{失败} = n + 1$$

---

## 四、顺序查找的优化（有序表）

### 1. 前提条件
查找表中元素**有序存放**（递增排列）。

### 2. 优化策略
当比较到某个元素**大于**目标关键字时，即可判定查找失败，无需遍历到表尾。

### 3. 查找判定树分析

对于有序表 `[7, 13, 19, 29, 37, 43]`：

```
              29
           /      \
         13        43
        /  \      /  \
       7   19   37   [失败结点]
      / \  / \  / \
     ... ... ... ...
```

- **n 个成功结点**：每个成功结点的查找长度 = 自身所在层数
- **n+1 个失败结点**：每个失败结点的查找长度 = 其父节点所在层数

> **关键结论**：有序顺序表的顺序查找，查找成功的 ASL 不变，但查找失败的 ASL 可以减少。

### 4. 不等概率优化

当各元素被查概率**不相等**时，将**被查概率大的元素放在靠前位置**，可降低 ASL。

**示例**：
| 元素 | 7 | 13 | 19 | 29 | 37 | 43 |
|------|---|---|---|---|---|---|
| 概率 | 15% | 5% | 10% | 40% | 28% | 2% |

- 顺序排列：$ASL = 1 \times 0.15 + 2 \times 0.05 + 3 \times 0.1 + 4 \times 0.4 + 5 \times 0.28 + 6 \times 0.02 = 3.67$
- 按概率降序排列（29, 37, 7, 19, 13, 43）：$ASL = 1 \times 0.4 + 2 \times 0.28 + 3 \times 0.15 + 4 \times 0.1 + 5 \times 0.05 + 6 \times 0.02 = 2.18$

## 五、考点总结

| 考点 | 内容 |
|------|------|
| 基本实现 | 普通实现 vs 带哨兵实现 |
| ASL 计算 | 成功 $ASL = (n+1)/2$，失败 $ASL = n+1$ |
| 有序表优化 | 利用有序性提前终止查找 |
| 判定树分析 | 成功/失败结点的查找长度 |
| 不等概率 | 概率大的放前面可降低 ASL |
| 空间复杂度 | $O(1)$ |

> 1. 顺序查找的时间复杂度为 $O(n)$，效率较低
> 2. 带哨兵的实现每趟循环可减少一次比较
> 3. 顺序查找**既适用于顺序表，也适用于链表**
