## 一、核心概念

### 1.1 强连通分量（Strongly Connected Component, SCC）

> **定义**：在有向图 $G = (V, E)$ 中，若顶点子集 $V' \subseteq V$ 中任意两个顶点 $u, v$ 之间**互相可达**（即存在 $u \to v$ 和 $v \to u$ 的路径），且 $V'$ 不被更大的集合所包含，则称 $V'$ 为一个**强连通分量**。

**关键性质**：
- 每个顶点恰好属于一个强连通分量
- 将每个SCC缩成一个点后，原图变为一个**有向无环图（DAG）**
- 单独的一个顶点（无自环且无法到达其他顶点）也是一个SCC

**示例**：
```
有向图：
  1 → 2 → 3 → 1
          ↓
  4 → 5 ← 6 → 4

SCC1 = {1, 2, 3}    （互相可达）
SCC2 = {4, 5, 6}    （互相可达）
```

### 1.2 DFS搜索树相关概念

在深度优先搜索（DFS）过程中，会产生一棵**DFS搜索树**（DFS生成森林），需要理解以下概念：

| 概念 | 含义 |
|------|------|
| **树边（Tree Edge）** | DFS过程中首次访问某顶点时经过的边 |
| **返祖边（Back Edge）** | 从后代指向祖先的边 |
| **横叉边（Cross Edge）** | 从一个子树指向另一个子树的边 |
| **前向边（Forward Edge）** | 从祖先指向后代的非树边 |

> **关键定理**：在DFS搜索树中，存在返祖边是形成环（强连通分量）的充要条件。

## 二、Tarjan算法原理

### 2.1 核心数据结构

Tarjan算法使用以下关键数据结构：

#### dfn数组（时间戳）
$$dfn[u] = \text{顶点 } u \text{ 在DFS中被首次访问的次序编号（从1开始）}$$

#### low数组（最低可达时间戳）
$$low[u] = \text{顶点 } u \text{ 及其后代通过一条返祖边能到达的最小 } dfn \text{ 值}$$

**low值的含义**：$low[u]$ 表示从 $u$ 出发，经过若干条树边和**至多一条**返祖边，能够到达的DFS序最小的顶点。

#### 栈（Stack）
- 存储当前正在搜索的强连通分量中的顶点
- 栈中顶点的 $low$ 值尚未最终确定

#### inStack数组
- 标记某个顶点是否在栈中
- 避免重复入栈

### 2.2 核心思想

**Tarjan算法的本质**：利用DFS过程中维护 $low$ 值，当某个顶点 $u$ 满足 $dfn[u] = low[u]$ 时，说明 $u$ 是当前强连通分量的**根**，此时栈中从栈顶到 $u$ 的所有顶点构成一个强连通分量。

**为什么 $dfn[u] = low[u]$ 时 $u$ 是SCC的根？**
- $low[u] = dfn[u]$ 说明 $u$ 及其后代无法通过返祖边到达 $u$ 的祖先
- 意味着没有任何边从 $u$ 的后代连向 $u$ 之前的顶点
- 因此 $u$ 及其后代构成的子图与外界没有"回路"，形成独立的SCC

### 2.3 low值更新规则

对于顶点 $u$ 的每条出边 $u \to v$：

| 条件 | 操作 |
|------|------|
| $v$ 未被访问 | 先递归访问 $v$，回溯后 $low[u] = \min(low[u], low[v])$ |
| $v$ 在栈中 | $low[u] = \min(low[u], dfn[v])$（返祖边或横叉边） |
| $v$ 已访问且不在栈中 | 不做任何操作（$v$ 属于已确定的其他SCC） |

## 三、Tarjan算法步骤详解

```
算法：Tarjan_SCC(G)
输入：有向图 G = (V, E)
输出：所有强连通分量

1. 初始化：
   - index = 0（全局时间戳计数器）
   - 栈 S 为空
   - 对所有顶点 v：dfn[v] = 0, low[v] = 0, inStack[v] = false

2. 对每个未访问的顶点 u，调用 DFS_Tarjan(u)

3. DFS_Tarjan(u) 过程：
   3.1 设 dfn[u] = low[u] = ++index
   3.2 将 u 压入栈 S，inStack[u] = true
   3.3 对 u 的每个邻接顶点 v：
       3.3.1 若 v 未被访问（dfn[v] = 0）：
             - 递归调用 DFS_Tarjan(v)
             - low[u] = min(low[u], low[v])
       3.3.2 若 v 在栈中（inStack[v] = true）：
             - low[u] = min(low[u], dfn[v])
   3.4 若 dfn[u] == low[u]：  ← u 是SCC的根
       - 重复弹栈，直到弹出 u：
           - w = pop(S), inStack[w] = false
           - w 属于当前 SCC
       - 输出当前 SCC
```

**手动模拟过程**（以有向图为例）：

```
有向图：1→2, 2→3, 3→1, 3→4, 4→5, 5→4

步骤 | 当前顶点 | dfn | low | 栈状态（底→顶） | 操作
-----|----------|-----|-----|------------------|-----
 1   |    1     |  1  |  1  | [1]             | 访问1
 2   |    2     |  2  |  2  | [1,2]           | 访问2
 3   |    3     |  3  |  3  | [1,2,3]         | 访问3
 4   |  3→1     |     |  1  | [1,2,3]         | 返祖边，low[3]=min(3,1)=1
 5   |    3→4   |     |     |                 | 递归访问4
 6   |    4     |  4  |  4  | [1,2,3,4]       | 访问4
 7   |    5     |  5  |  5  | [1,2,3,4,5]     | 访问5
 8   |  5→4     |     |  4  | [1,2,3,4,5]     | 返祖边，low[5]=min(5,4)=4
 9   |  回溯5   |     |     | [1,2,3,4,5]     | low[4]=min(4,low[5])=4
10   |  dfn[4]==low[4] |   | [1,2,3]         | 弹出{5,4}为一个SCC
11   |  回溯3   |     |     | [1,2,3]         | low[2]=min(2,low[3])=1
12   |  回溯2   |     |     | [1,2,3]         | low[1]=min(1,low[2])=1
13   |  dfn[1]==low[1] |   | []              | 弹出{3,2,1}为一个SCC

结果：SCC1 = {4, 5}, SCC2 = {1, 2, 3}
```

## 四、C语言实现代码

```c
#include <stdio.h>
#include <string.h>

#define MAXN 100005

// 邻接表
typedef struct {
    int to, next;
} Edge;

Edge edges[MAXN * 2];  // 边数组
int head[MAXN];         // 邻接表头
int edge_cnt = 0;

// Tarjan 相关变量
int dfn[MAXN];      // DFS时间戳
int low[MAXN];      // 最低可达时间戳
int stack[MAXN];    // 栈
int inStack[MAXN];  // 是否在栈中
int scc[MAXN];      // 每个顶点所属的SCC编号
int scc_size[MAXN]; // 每个SCC的大小
int index = 0;      // 时间戳计数器
int top = -1;       // 栈顶指针
int scc_cnt = 0;    // SCC计数器

// 初始化图
void init(int n) {
    memset(head, -1, sizeof(int) * (n + 1));
    memset(dfn, 0, sizeof(int) * (n + 1));
    memset(low, 0, sizeof(int) * (n + 1));
    memset(inStack, 0, sizeof(int) * (n + 1));
    memset(scc, 0, sizeof(int) * (n + 1));
    memset(scc_size, 0, sizeof(int) * (n + 1));
    edge_cnt = 0;
    index = 0;
    top = -1;
    scc_cnt = 0;
}

// 添加有向边
void add_edge(int u, int v) {
    edges[edge_cnt].to = v;
    edges[edge_cnt].next = head[u];
    head[u] = edge_cnt++;
}

// Tarjan 算法核心
void tarjan(int u) {
    // 步骤1：初始化 dfn 和 low
    dfn[u] = low[u] = ++index;

    // 步骤2：入栈
    stack[++top] = u;
    inStack[u] = 1;

    // 步骤3：遍历邻接点
    for (int i = head[u]; i != -1; i = edges[i].next) {
        int v = edges[i].to;

        if (!dfn[v]) {
            // 情况A：v 未被访问，递归
            tarjan(v);
            // 回溯后更新 low[u]
            if (low[v] < low[u])
                low[u] = low[v];
        } else if (inStack[v]) {
            // 情况B：v 在栈中（返祖边或横叉边）
            if (dfn[v] < low[u])
                low[u] = dfn[v];
        }
        // 情况C：v 已访问且不在栈中，忽略
    }

    // 步骤4：判断 u 是否为 SCC 的根
    if (dfn[u] == low[u]) {
        scc_cnt++;
        int w;
        do {
            w = stack[top--];
            inStack[w] = 0;
            scc[w] = scc_cnt;
            scc_size[scc_cnt]++;
        } while (w != u);
    }
}

int main() {
    int n, m;  // n个顶点，m条边
    scanf("%d %d", &n, &m);
    init(n);

    for (int i = 0; i < m; i++) {
        int u, v;
        scanf("%d %d", &u, &v);
        add_edge(u, v);  // 有向边
    }

    // 对每个未访问的顶点执行 Tarjan
    for (int i = 1; i <= n; i++) {
        if (!dfn[i])
            tarjan(i);
    }

    // 输出结果
    printf("强连通分量个数：%d\n", scc_cnt);
    for (int i = 1; i <= scc_cnt; i++) {
        printf("SCC %d (大小=%d)：", i, scc_size[i]);
        for (int j = 1; j <= n; j++) {
            if (scc[j] == i)
                printf("%d ", j);
        }
        printf("\n");
    }

    return 0;
}
```

### 代码要点说明

| 要点 | 说明 |
|------|------|
| **邻接表存储** | 使用静态数组模拟链表，适合稀疏图 |
| **栈操作** | `stack[++top]` 入栈，`stack[top--]` 出栈 |
| **inStack标记** | 区分"未访问"和"已访问但不在栈中"两种状态 |
| **low更新** | 返祖边用 `dfn[v]` 更新，树边回溯用 `low[v]` 更新 |
| **SCC判定** | `dfn[u] == low[u]` 时弹栈至 $u$ |

## 五、时间复杂度分析

| 指标 | 分析 |
|------|------|
| **时间复杂度** | $O(V + E)$，每个顶点和每条边各被访问一次 |
| **空间复杂度** | $O(V)$，dfn、low、栈各需 $O(V)$ 空间 |
| **总访问次数** | 每个顶点入栈、出栈各一次；每条边检查一次 |

> ⚠️ Tarjan算法是**线性时间**算法，是求强连通分量的最优算法之一。

## 六、典型例题

### 例题1：求强连通分量（2023年改编）

**题目**：对下图求所有强连通分量。

```
有向图：
1 → 2 → 3 → 1
          ↓
3 → 4 → 5 → 6 → 4
```

**解答**：

手动模拟Tarjan过程：

| 步骤 | 操作 | dfn | low | 栈 |
|------|------|-----|-----|----|
| 1 | 访问1 | 1→1 | 1→1 | [1] |
| 2 | 访问2 | 2→2 | 2→2 | [1,2] |
| 3 | 访问3 | 3→3 | 3→3 | [1,2,3] |
| 4 | 3→1返祖 | - | low[3]=1 | [1,2,3] |
| 5 | 访问4 | 4→4 | 4→4 | [1,2,3,4] |
| 6 | 访问5 | 5→5 | 5→5 | [1,2,3,4,5] |
| 7 | 访问6 | 6→6 | 6→6 | [1,2,3,4,5,6] |
| 8 | 6→4返祖 | - | low[6]=4 | [1,2,3,4,5,6] |
| 9 | 回溯6 | - | low[5]=4 | [1,2,3,4,5,6] |
| 10 | 回溯5 | - | low[4]=4 | [1,2,3,4,5,6] |
| 11 | dfn[4]=low[4] | - | - | 弹出{6,5,4} |
| 12 | 回溯3 | - | low[3]=1 | [1,2,3] |
| 13 | 回溯2 | - | low[2]=1 | [1,2,3] |
| 14 | dfn[1]=low[1] | - | - | 弹出{3,2,1} |

**结果**：SCC1 = {4, 5, 6}，SCC2 = {1, 2, 3}

### 例题2：选择题

**题目**：有向图 $G$ 有 $n$ 个顶点，$m$ 条边，使用Tarjan算法求所有强连通分量的时间复杂度为（ ）。

A. $O(n \log n)$  B. $O(n + m)$  C. $O(n \times m)$  D. $O(n^2)$

**答案**：B

**解析**：Tarjan算法本质上是一次DFS遍历，每个顶点和每条边各被处理一次，总时间复杂度为 $O(V + E)$。

### 例题3：手写代码题

**题目**：给定有向图，判断图中是否存在环。

**思路**：利用Tarjan算法，若某个SCC包含2个及以上顶点，则图中存在环。

```c
int has_cycle(int n) {
    // 先执行tarjan，得到scc_size数组
    for (int i = 1; i <= n; i++)
        if (!dfn[i]) tarjan(i);

    for (int i = 1; i <= scc_cnt; i++)
        if (scc_size[i] >= 2) return 1;  // 存在环
    return 0;  // 无环
}
```

## 七、Tarjan算法 vs Kosaraju算法对比

| 对比维度 | Tarjan算法 | Kosaraju算法 |
|----------|-----------|-------------|
| **基本思想** | 一次DFS + 栈 | 两次DFS（正图+反图） |
| **时间复杂度** | $O(V + E)$ | $O(V + E)$ |
| **空间复杂度** | $O(V)$ | $O(V + E)$（需存储反图） |
| **DFS次数** | 1次 | 2次 |
| **实现难度** | 较难（需理解low值） | 较简单（直觉清晰） |
| **适用场景** | 实际工程中更常用 | 教学中更易理解 |
| **图的存储** | 只需原图 | 需要原图 + 反图 |
| **能否在线** | 可以（边输入边处理） | 不能（需要完整图） |

**Kosaraju算法简述**：
1. 对原图做DFS，记录顶点的**完成时间**（后序遍历顺序）
2. 构造原图的**反图**（所有边反向）
3. 按完成时间**逆序**对反图做DFS，每次DFS访问的顶点构成一个SCC

> 💡 **408备考建议**：优先掌握Tarjan算法（代码题常考），Kosaraju算法作为理解SCC概念的辅助方法。

## 八、易错点

> - 返祖边/横叉边：用 **dfn[v]** 更新（因为 $v$ 可能已被弹栈，其 $low$ 值无意义）
> - 树边回溯：用 **low[v]** 更新（因为子节点的 $low$ 值已经确定）
> - **常见错误**：统一用 `low[v]` 更新，导致结果错误

> - 必须使用 `inStack` 数组标记顶点是否在栈中
> - 不能仅靠 `dfn[v] != 0` 判断，因为已弹出的顶点 $dfn$ 不为0但不应参与 $low$ 更新

> - 没有任何边的孤立顶点也是一个SCC
> - Tarjan算法中，孤立顶点会满足 `dfn[u] == low[u]` 并被单独弹出

> - Tarjan算法求出的SCC编号顺序是**拓扑逆序**（最后弹出的SCC在DAG中度为0）
> - 这在后续利用SCC做DP时非常有用
