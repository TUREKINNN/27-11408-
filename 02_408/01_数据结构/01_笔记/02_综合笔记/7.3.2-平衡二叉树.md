# 7.3.2 平衡二叉树（AVL）

## 一、定义

**平衡二叉树**（Balanced Binary Tree），简称 AVL 树（Adelson-Velsky and Landis）。

树上**任一结点**的左子树和右子树的**高度之差不超过 1**。

### 平衡因子
$$\text{平衡因子} = \text{左子树高度} - \text{右子树高度}$$

> **判定规则**：只要有任一结点的平衡因子绝对值 > 1，就不是平衡二叉树。
> 
> 平衡因子的值只可能是 **-1、0 或 1**。

---

## 二、插入操作

### 基本思想
在 BST 中插入新结点后，从**插入点往回**找到**第一个不平衡结点**，调整以该结点为根的**最小不平衡子树**。

> **关键结论**：只需调整最小不平衡子树，其他祖先结点都会恢复平衡。

---

## 四种旋转操作

### 1. LL 型（右单旋转）

**触发条件**：在结点 A 的**左孩子**的**左子树**上插入，A 的平衡因子从 1 增至 2。

```
    A (bf=2)           B
   / \                / \
  B   AR    →       BL   A
 / \                    / \
BL  BR                BR   AR
```

**操作**：将 A 的左孩子 B **向右上旋转**代替 A 成为根，A 向右下旋转成为 B 的右子树，B 的原右子树 BR 作为 A 的左子树。

### 2. RR 型（左单旋转）

**触发条件**：在结点 A 的**右孩子**的**右子树**上插入，A 的平衡因子从 -1 减至 -2。

```
  A (bf=-2)            B
 / \                  / \
AL   B      →       A   BR
    / \             / \
   BL  BR         AL  BL
```

**操作**：将 A 的右孩子 B **向左上旋转**代替 A 成为根，A 向左下旋转成为 B 的左子树。

### 3. LR 型（先左后右双旋转）

**触发条件**：在 A 的**左孩子**的**右子树**上插入。

```
    A (bf=2)           C
   / \                / \
  B   AR    →       B    A
 / \               / \  / \
AL  C             AL CL CR AR
   / \
  CL  CR
```

**操作**：先将 C **左旋**提升到 B 的位置，再将 C **右旋**提升到 A 的位置。

### 4. RL 型（先右后左双旋转）

**触发条件**：在 A 的**右孩子**的**左子树**上插入。

```
  A (bf=-2)           C
 / \                 / \
AL   B     →       A    B
    / \            / \  / \
   C   AR        AL CL CR AR
  / \
 CL  CR
```

**操作**：先将 C **右旋**提升到 B 的位置，再将 C **左旋**提升到 A 的位置。

## 三、旋转代码实现

### LL 旋转（右旋）

```c
void R_Rotate(BSTree &p) {
    // p 为最小不平衡子树的根
    BSTree lc = p->lchild;       // lc 指向 p 的左孩子
    p->lchild = lc->rchild;      // lc 的右子树挂到 p 的左边
    lc->rchild = p;              // p 挂到 lc 的右边
    p = lc;                      // lc 成为新的根
}
```

### RR 旋转（左旋）

```c
void L_Rotate(BSTree &p) {
    BSTree rc = p->rchild;
    p->rchild = rc->lchild;
    rc->lchild = p;
    p = rc;
}
```

## 四、查找效率

设 $n_h$ 为高度为 h 的平衡二叉树中**最少结点数**：

$$n_h = n_{h-1} + n_{h-2} + 1$$

其中 $n_0 = 0, n_1 = 1, n_2 = 2$。

这与**斐波那契数列**相关，可以证明：

$$h = O(\log_2 n)$$

> **结论**：AVL 树的查找、插入、删除的时间复杂度均为 $O(\log_2 n)$。

## 五、考点总结

| 考点 | 内容 |
|------|------|
| 平衡因子 | 左子树高 - 右子树高，取值 -1, 0, 1 |
| 四种旋转 | LL(右旋)、RR(左旋)、LR(先左后右)、RL(先右后左) |
| 最小不平衡子树 | 每次调整的对象 |
| 查找效率 | $O(\log_2 n)$ |
| 结点数与高度 | $n_h = n_{h-1} + n_{h-2} + 1$ |

> 2. LL/RR 是单旋转，LR/RL 是双旋转
> 3. 调整最小不平衡子树后，其他祖先结点自动恢复平衡
> 4. AVL 树的高度 $h \approx 1.44 \log_2 n$
