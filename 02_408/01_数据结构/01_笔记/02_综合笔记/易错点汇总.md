## 一、概念混淆点

### 1.1 基本概念
| 易混淆项 | 区别 |
|---------|------|
| 数据 vs 数据元素 vs 数据项 | 数据是集合；数据元素是基本单位；数据项是最小单位 |
| 逻辑结构 vs 存储结构 | 逻辑结构是抽象关系；存储结构是物理实现 |
| 算法 vs 程序 | 算法是解决问题的步骤；程序是算法的代码实现 |
| 时间复杂度 vs 实际运行时间 | 复杂度描述增长趋势，不是具体时间 |

### 1.2 线性表
| 易混淆项 | 区别 |
|---------|------|
| 头结点 vs 头指针 | 头结点是额外的节点；头指针是指向第一个节点的指针 |
| 头插法 vs 尾插法 | 头插法结果与输入相反；尾插法结果与输入相同 |
| 顺序表按位查找 O(1) vs 按值查找 O(n) | 按位可以直接计算地址；按值需要遍历比较 |

### 1.3 栈和队列
| 易混淆项 | 区别 |
|---------|------|
| 栈满 vs 栈空 | 顺序栈中需要区分条件，避免误判 |
| 循环队列判空 vs 判满 | 判空：front==rear；判满：(rear+1)%MaxSize==front |
| 共享栈的栈满条件 | 两个栈的栈顶相邻时满：top1+1==top2 |

### 1.4 树与二叉树
| 易混淆项 | 区别 |
|---------|------|
| 满二叉树 vs 完全二叉树 | 满二叉树每层都满；完全二叉树只有最后一层可能不满且靠左 |
| 二叉树 vs 度为2的树 | 二叉树有左右之分；度为2的树无左右之分 |
| 先序遍历 vs 层次遍历 | 先序是深度优先；层次是广度优先 |
| 线索二叉树中 ltag/rtag | 0表示指向孩子；1表示指向前驱/后继线索 |
| 树的先根遍历 vs 二叉树的先序遍历 | 对应关系：树的先根 = 转换后二叉树的先序 |
| 树的后根遍历 vs 二叉树的中序遍历 | 对应关系：树的后根 = 转换后二叉树的中序（不是后序！） |

### 1.5 图
| 易混淆项 | 区别 |
|---------|------|
| 连通图 vs 强连通图 | 连通图用于无向图；强连通图用于有向图 |
| 连通分量 vs 强连通分量 | 分别是无向图和有向图的极大连通子图 |
| 邻接矩阵 vs 邻接表 | 矩阵适合稠密图；邻接表适合稀疏图 |
| DFS vs BFS | DFS用栈/递归；BFS用队列 |
| Prim vs Kruskal | Prim从顶点出发；Kruskal从边出发 |
| Dijkstra vs Floyd | Dijkstra求单源最短路；Floyd求全源最短路 |
| 拓扑排序 vs 关键路径 | 拓扑排序用于AOV网；关键路径用于AOE网 |

### 1.6 查找
| 易混淆项 | 区别 |
|---------|------|
| BST vs AVL | BST不保证平衡；AVL保证任意节点平衡因子≤1 |
| B树 vs B+树 | B树所有节点存数据；B+树只在叶子存数据，叶子有链表连接 |
| 散列表查找成功 vs 查找失败 | 成功找到时的比较次数 vs 确认不存在时的比较次数 |
| 线性探测 vs 二次探测 | 线性探测容易聚集；二次探测减少聚集但不一定能探测所有位置 |

### 1.7 排序
| 易混淆项 | 区别 |
|---------|------|
| 稳定 vs 不稳定排序 | 稳定排序不改变相等元素的相对位置 |
| 内排序 vs 外排序 | 内排序数据全在内存；外排序数据在外存 |
| 快排最好 vs 最坏情况 | 最好：每次恰好平分；最坏：已有序时退化为O(n²) |
| 堆排序建堆方向 | 大根堆：从最后一个非叶子节点向下调整 |
| 直接插入 vs 折半插入 | 直接插入逐个比较找位置；折半插入用二分法找位置 |

## 二、计算易错点

### 2.1 时间复杂度
- ❌ **错误：** 嵌套循环一定是O(n²)
- ✅ **正确：** 要看内层循环的实际执行次数

```c
// 例：这个是O(n)不是O(n²)
for(i=0; i<n; i++)
    for(j=i; j<n; j++)  // j从i开始，总次数为n(n-1)/2 = O(n²)
    // 但如果改成 j*=2，则为 O(nlogn)
```

- ❌ **错误：** 递归算法复杂度都是O(2^n)
- ✅ **正确：** 要根据递推关系具体分析，如二分递归 T(n)=T(n/2)+1 是 O(logn)

### 2.2 二叉树
- ❌ **错误：** 度为2的树有 n₀ = n₂ + 1
- ✅ **正确：** 这个公式只对二叉树成立，普通树的公式不同
  - 普通树：n₀ + n₁ + n₂ + ... + nₘ = n，所有度之和 = n-1

- ❌ **错误：** 完全二叉树中，有右孩子必有左孩子
- ✅ **正确：** 这是对的。但有左孩子不一定有右孩子

- ❌ **错误：** 由先序和后序序列可以唯一确定二叉树
- ✅ **正确：** 不可以！需要中序+先序或中序+后序

- ❌ **错误：** 二叉树的先序序列和后序序列一定不同
- ✅ **正确：** 当二叉树只有一个节点时，先序=后序

### 2.3 图
- ❌ **错误：** 无向图的邻接矩阵一定是对称的
- ✅ **正确：** 这是对的，因为 (i,j) 和 (j,i) 是同一条边

- ❌ **错误：** Dijkstra算法可以处理负权边
- ✅ **正确：** Dijkstra不能处理负权边！Floyd也不能处理负权回路

- ❌ **错误：** 拓扑排序的结果是唯一的
- ✅ **正确：** 不唯一，可能有多种合法的拓扑序列

- ❌ **错误：** 关键路径上的活动不能延迟
- ✅ **正确：** 关键活动的延迟为0（余量为0），非关键活动有一定余量

- ❌ **错误：** n个顶点的连通图最少有n条边
- ✅ **正确：** 最少 n-1 条边（生成树）。n条边是强连通有向图的最少边数。

### 2.4 查找
- ❌ **错误：** 折半查找只能用于顺序表
- ✅ **正确：** 确实只能用于顺序存储且有序的线性表，不能用于链表

- ❌ **错误：** AVL树查找一定比BST树快
- ✅ **正确：** 通常更快，但如果BST恰好是平衡的，可能一样快

- ❌ **错误：** 散列表的装填因子越大越好
- ✅ **正确：** 装填因子越大，冲突越多，查找效率越低

- ❌ **错误：** B树和B+树是一样的
- ✅ **正确：** B+树的数据只在叶子节点，叶子节点用链表连接，更适合范围查询

### 2.5 排序
- ❌ **错误：** 快速排序时间复杂度总是O(nlogn)
- ✅ **正确：** 平均是O(nlogn)，最坏是O(n²)（已有序时）

- ❌ **错误：** 堆排序建堆是从根开始向下调整
- ✅ **正确：** 建堆是从最后一个非叶子节点（⌊n/2⌋）开始向前调整

- ❌ **错误：** 简单选择排序是稳定的
- ✅ **正确：** 不稳定！例如 [5, 5, 3]，第一趟选3与第一个5交换后，两个5的相对位置变了

- ❌ **错误：** 所有时间复杂度为O(nlogn)的排序都是不稳定的
- ✅ **正确：** 归并排序就是O(nlogn)且稳定

## 三、代码实现易错点

### 3.1 链表操作

**易错1：删除节点时忘记释放内存**
```c
// ❌ 错误
p->next = p->next->next;
// ✅ 正确
LNode *q = p->next;
p->next = q->next;
free(q);
```

**易错2：不带头结点的链表插入删除需要特殊处理头节点**
```c
// ❌ 不带头结点时直接用 p->next 会遗漏头节点的处理
// ✅ 建议使用带头结点的链表，操作统一
```

**易错3：尾插法忘记将最后节点的next置为NULL**
```c
// ❌ 忘记
r->next = s;  // s是新节点
r = s;
// ✅ 正确
r->next = s;
r = s;
r->next = NULL;  // 或者在初始化时确保 s->next = NULL
```

### 3.2 栈和队列

**易错4：循环队列判满条件写错**
```c
// ❌ 错误
if(rear == MaxSize - 1)  // 这是普通队列
// ✅ 正确（牺牲一个空间）
if((rear + 1) % MaxSize == front)
```

**易错5：循环队列元素个数计算**
```c
// ❌ 错误
count = rear - front;  // 可能为负数
// ✅ 正确
count = (rear - front + MaxSize) % MaxSize;
```

**易错6：后缀表达式求值时操作数和运算符顺序**
```c
// ❌ 错误：减法和除法时操作数顺序搞反
a = Pop(); b = Pop();
Push(a / b);  // 错！应该是 b / a
// ✅ 正确
a = Pop(); b = Pop();
Push(b / a);  // 先弹出的是右操作数
```

### 3.3 二叉树

**易错7：递归遍历的终止条件**
```c
// ❌ 错误
void PreOrder(BiTree T) {
    printf("%c", T->data);  // T可能为空！
    PreOrder(T->lchild);
    PreOrder(T->rchild);
}
// ✅ 正确
void PreOrder(BiTree T) {
    if(T == NULL) return;  // 必须先判断
    printf("%c", T->data);
    PreOrder(T->lchild);
    PreOrder(T->rchild);
}
```

**易错8：非递归中序遍历的栈操作**
```c
// ❌ 常见错误：忘记在访问节点后将p置为NULL
// ✅ 正确：访问节点后应 p = NULL，否则会死循环
```

**易错9：由先序和中序构造二叉树时，子树边界计算**
```c
// ❌ 常见错误：左右子树长度计算有误
// ✅ 技巧：画图标注每个子树在两个序列中的范围
// 左子树长度 = 根在中序中的位置 - 中序起始位置
```

### 3.4 图

**易错10：邻接矩阵中顶点编号从0还是1开始**
```c
// ❌ 容易混淆
// 题目中可能从1开始，代码中可能从0开始
// ✅ 仔细审题，统一编号
```

**易错11：DFS/BFS中visited数组的初始化**
```c
// ❌ 忘记初始化visited数组
// ✅ 在遍历前将所有元素置为0
for(int i = 0; i < n; i++) visited[i] = 0;
```

**易错12：Dijkstra算法中负权边导致的错误**
```
已确定最短路径的顶点集合S，不再更新
如果存在负权边，已确定的最短路径可能不是真正的最短路径
```

### 3.5 排序

**易错13：快速排序的枢轴选择和划分**
```c
// ❌ 划分时遗漏边界条件
// ✅ 标准划分算法（Hoare分区）：
// 1. 选第一个元素为枢轴
// 2. 从两端向中间扫描
// 3. 先从右找比枢轴小的，再从左找比枢轴大的
// 4. 交换，直到相遇
```

**易错14：建堆时调整方向搞错**
```c
// ❌ 从根开始向下调整（不完整）
// ✅ 从最后一个非叶子节点（⌊n/2⌋）开始，向前逐个调整
for(i = n/2; i >= 1; i--)
    AdjustDown(i);
```

**易错15：堆排序删除堆顶后的调整**
```c
// ❌ 直接删除堆顶
// ✅ 将堆顶与最后一个元素交换，堆大小减1，再从堆顶向下调整
swap(heap[1], heap[n]);
n--;
AdjustDown(1);
```

## 四、真题常见陷阱

### 4.1 审题陷阱

**陷阱1：题目问的是"最多"还是"最少"**
- 例：n个节点的二叉树最多/最少有多少个叶子节点？
- 最多：⌊n/2⌋ + 1（满二叉树时）
- 最少：1（退化为链表时）

**陷阱2：题目说的是"二叉树"还是"树"**
- 二叉树和树的性质不同，公式不同
- 二叉树：n₀ = n₂ + 1
- 树：没有这个公式

**陷阱3：排序算法的"稳定"与"不稳定"**
- 考试经常考某个排序是否稳定
- 记忆口诀："插冒归基"稳定，"希快选堆"不稳定

**陷阱4：时间复杂度的"最好""平均""最坏"**
- 问的是哪种情况？看清楚！
- 快排最好O(nlogn)、平均O(nlogn)、最坏O(n²)

### 4.2 计算陷阱

**陷阱5：完全二叉树的节点编号**
- 从1开始还是从0开始？看清楚题目！
- 从1开始：左孩子2i，右孩子2i+1
- 从0开始：左孩子2i+1，右孩子2i+2

**陷阱6：散列表查找失败的ASL**
- 不是找到最后才失败
- 是找到一个空位置（或回到起点）时才确认失败
- 线性探测中，查找失败的比较次数取决于第一个空位的位置

**陷阱7：二叉树由遍历序列构造**
- 先序+后序不能唯一确定二叉树
- 但如果是"真二叉树"（每个节点度为0或2），先序+后序可以唯一确定

**陷阱8：循环队列牺牲空间 vs 不牺牲**
- 牺牲一个空间：判满条件 (rear+1)%MaxSize == front
- 不牺牲空间：需要额外标志位区分空和满

### 4.3 算法设计陷阱

**陷阱9：链表算法题忘记考虑边界情况**
- 空链表
- 只有一个节点
- 目标节点在头部/尾部

**陷阱10：二叉树递归算法的返回值**
- 求高度时，返回的是左右子树高度的较大值 + 1
- 求节点个数时，返回的是左右子树节点数之和 + 1

**陷阱11：图的算法题中，顶点编号和数组下标的对应**
- 题目中顶点从v1开始，但数组下标从0开始
- 做题时要标注清楚

### 4.4 选择题常见陷阱

**陷阱12：n个元素的排序，比较次数的下界**
- 基于比较的排序，比较次数下界为 ⌈log₂(n!)⌉ ≈ nlog₂n
- 基数排序不是基于比较的排序，可以突破这个下界

**陷阱13：快速排序的空间复杂度**
- 最好O(logn)，最坏O(n)
- 递归深度决定了空间复杂度

**陷阱14：堆排序建堆的时间复杂度**
- 不是O(nlogn)，而是O(n)！
- 原因：高度为h的节点最多有⌈n/2^(h+1)⌉个，每个调整O(h)
- 总复杂度 = Σ(h=1到⌊log₂n⌋) h × n/2^(h+1) = O(n)

**陷阱15：归并排序的空间复杂度**
- O(n)，不是O(1)
- 需要辅助数组来合并两个有序子序列

## 五、考前速记清单

### 必记结论

1. **n₀ = n₂ + 1**（二叉树）
2. **n个节点完全二叉树高度 = ⌊log₂n⌋ + 1**
3. **卡特兰数**：n个元素进栈，出栈序列数 = C(2n,n)/(n+1)
4. **循环队列**：判空front==rear，判满(rear+1)%MaxSize==front
5. **稳定排序**：插冒归基
6. **不稳定排序**：希快选堆
7. **快排最坏**：已有序时O(n²)
8. **堆排序建堆**：O(n)，从⌊n/2⌋开始向下调整
9. **Dijkstra不能处理负权边**
10. **B+树数据只在叶子节点，叶子有链表连接**

### 易错公式

| 知识点 | 正确公式 | 常见错误 |
|--------|---------|---------|
| 二叉树叶子节点 | n₀ = n₂ + 1 | 混淆为 n₀ = n₂ |
| 完全二叉树高度 | h = ⌊log₂n⌋ + 1 | 忘记取整或+1 |
| 循环队列元素数 | (rear-front+MaxSize)%MaxSize | 忘记+MaxSize |
| 哈夫曼树节点数 | 2n-1（n为叶子数） | 混淆为2n |
| 哈夫曼树无度为1的节点 | — | 容易忽略 |
| 折半查找ASL | ≈ log₂(n+1) - 1 | 混淆为log₂n |

## 六、答题技巧

### 选择题
1. 先看选项排除明显错误的
2. 时间复杂度题：关注循环次数和递归深度
3. 概念题：注意"一定""可能""不一定"等限定词
4. 画图辅助理解

### 大题（算法设计）
1. 先写清楚算法思路（伪代码或文字描述）
2. 注意边界条件
3. 注意时间和空间复杂度分析
4. 代码要规范，变量命名清晰
5. 写完后用简单例子验证

### 大题（综合分析）
1. 画图！画图！画图！
2. 按步骤逐步推导
3. 检查计算结果是否合理
4. 注意题目中的特殊条件
