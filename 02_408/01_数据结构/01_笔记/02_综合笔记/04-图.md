## 一、基本概念

### 1.1 图的定义

图 G = (V, E)，V 是顶点的非空有限集，E 是边的有限集。

| 概念 | 定义 |
|------|------|
| **有向图** | 边有方向，用 <v,w> 表示（弧尾→弧头） |
| **无向图** | 边无方向，用 (v,w) 表示 |
| **完全图** | 任意两点间都有边。无向完全图边数 = n(n-1)/2，有向完全图边数 = n(n-1) |
| **连通图** | 无向图中任意两点间存在路径 |
| **强连通图** | 有向图中任意两点互相可达 |
| **连通分量** | 无向图的极大连通子图 |
| **强连通分量** | 有向图的极大强连通子图 |
| **生成树** | 包含图中所有顶点的极小连通子图（n 个顶点，n-1 条边） |
| **度** | 无向图：顶点的边数。有向图：入度 + 出度 |
| **权和网** | 边带权值的图称为网 |

**重要公式**：
- 无向图：所有顶点度数之和 = 2|E|
- 有向图：所有顶点入度之和 = 所有顶点出度之和 = |E|

## 二、存储结构

### 2.1 邻接矩阵

```c
#define MaxVertexNum 100
typedef struct {
    char Vex[MaxVertexNum];              // 顶点表
    int Edge[MaxVertexNum][MaxVertexNum]; // 邻接矩阵
    int vexnum, arcnum;                  // 顶点数、边数
} MGraph;
```

- 无向图的邻接矩阵**对称**，可压缩存储
- 空间复杂度：**O(n²)**
- 适用：稠密图
- 判断两点间是否有边：O(1)

### 2.2 邻接表

```c
typedef struct ArcNode {
    int adjvex;              // 邻接顶点
    struct ArcNode *next;
    // InfoType info;        // 边权值
} ArcNode;

typedef struct VNode {
    char data;
    ArcNode *first;
} VNode, AdjList[MaxVertexNum];

typedef struct {
    AdjList vertices;
    int vexnum, arcnum;
} ALGraph;
```

- 空间复杂度：**O(n+e)**
- 适用：稀疏图
- 有向图的邻接表：出边表
- **逆邻接表**：入边表，方便求入度

### 2.3 十字链表（有向图）

将邻接表和逆邻接表结合：
```c
typedef struct ArcBox {
    int tailvex, headvex;        // 弧尾、弧头
    struct ArcBox *hlink, *tlink; // 弧头相同链、弧尾相同链
    // InfoType info;
} ArcBox;

typedef struct VexNode {
    char data;
    ArcBox *firstin, *firstout;  // 入边、出边
} VexNode;
```

### 2.4 邻接多重表（无向图）

解决邻接表中每条边存储两次的问题：
```c
typedef struct ENode {
    int ivex, jvex;
    struct ENode *ilink, *jlink;  // 依附于 ivex、jvex 的下一条边
    // InfoType info;
} ENode;

typedef struct VexNode {
    char data;
    ENode *firstedge;
} VexNode;
```

> 对比：邻接矩阵适合稠密图，邻接表适合稀疏图，十字链表适合有向图，邻接多重表适合无向图。

## 三、图的遍历

### 3.1 深度优先搜索（DFS）

```c
bool visited[MaxVertexNum];

void DFS(ALGraph G, int v) {
    visit(v);
    visited[v] = true;
    for (ArcNode *p = G.vertices[v].first; p; p = p->next) {
        if (!visited[p->adjvex])
            DFS(G, p->adjvex);
    }
}

void DFSTraverse(ALGraph G) {
    memset(visited, false, sizeof(visited));
    for (int i = 0; i < G.vexnum; i++)
        if (!visited[i]) DFS(G, i);  // 处理非连通图
}
```

**复杂度**：
- 邻接矩阵：O(n²)
- 邻接表：O(n+e)

### 3.2 广度优先搜索（BFS）

```c
void BFS(ALGraph G, int v) {
    InitQueue(Q);
    visit(v);
    visited[v] = true;
    EnQueue(Q, v);
    while (!IsEmpty(Q)) {
        DeQueue(Q, v);
        for (ArcNode *p = G.vertices[v].first; p; p = p->next) {
            if (!visited[p->adjvex]) {
                visit(p->adjvex);
                visited[p->adjvex] = true;
                EnQueue(Q, p->adjvex);
            }
        }
    }
}
```

**复杂度**：
- 邻接矩阵：O(n²)
- 邻接表：O(n+e)

> 💡 **BFS 可求无权图的最短路径**：BFS 生成的广度优先生成树中，从源点到任意结点的路径即为最短路径。

## 四、最小生成树

### 4.1 Prim 算法（加点法）

从某顶点开始，每次选离已选顶点集**最近**的未选顶点加入。

**例题**：对以下无向网求最小生成树。

```
顶点: A B C D E
边及权: (A,B,6) (A,C,1) (A,D,5) (B,C,5) (B,E,3)
        (C,D,5) (C,E,6) (C,F,4) (D,F,2) (E,F,6)
```

**解答**（从 A 开始）：

| 步骤 | 已选集合 | 候选边 | 选择 | 权值 |
|------|----------|--------|------|------|
| 1 | {A} | (A,C,1),(A,D,5),(A,B,6) | (A,C,1) | 1 |
| 2 | {A,C} | (A,D,5),(A,B,6),(C,B,5),(C,D,5),(C,E,6),(C,F,4) | (C,F,4) | 4 |
| 3 | {A,C,F} | (A,D,5),(A,B,6),(C,B,5),(C,D,5),(C,E,6),(D,F,2),(E,F,6) | (D,F,2) | 2 |
| 4 | {A,C,F,D} | (A,B,6),(C,B,5),(C,E,6),(E,F,6) | (C,B,5) | 5 |
| 5 | {A,C,F,D,B} | (B,E,3),(C,E,6),(E,F,6) | (B,E,3) | 3 |

**总权值** = 1+4+2+5+3 = **15**

**时间复杂度**：O(n²)，适合稠密图。

### 4.2 Kruskal 算法（加边法）

每次选权值最小且**不构成回路**的边加入。

**步骤**：
1. 将所有边按权值排序
2. 依次取最小的边，若两端点不在同一连通分量（用**并查集**判断），则加入
3. 直到选了 n-1 条边

对上例排序：(A,C,1)(D,F,2)(B,E,3)(C,F,4)(A,D,5)(B,C,5)(C,D,5)(A,B,6)(C,E,6)(E,F,6)

依次选：(A,C,1)✓ (D,F,2)✓ (B,E,3)✓ (C,F,4)✓ (C,B,5)✓ — 共5条边，完成。

**时间复杂度**：O(elog₂e)，适合稀疏图。

## 五、最短路径

### 5.1 Dijkstra 算法（单源最短路径）

适用于**有向带权图**（权值非负），求一个源点到其余各点的最短路径。

**核心思想**：每次从未确定最短路径的顶点中选距离最小的，用它松弛其余顶点。

**例题**：求顶点 0 到其余各顶点的最短路径。

```
邻接矩阵:
     0    1    2    3    4
0  [ 0   10   ∞    30   100]
1  [ ∞    0    50   ∞    ∞ ]
2  [ ∞    ∞    0    ∞    10 ]
3  [ ∞    ∞    20    0    60 ]
4  [ ∞    ∞    ∞    ∞    0  ]
```

**完整推导表格**：

| 步骤 | 选定点 | dist[0] | dist[1] | dist[2] | dist[3] | dist[4] | path |
|------|--------|---------|---------|---------|---------|---------|------|
| 初始 | — | 0 | 10 | ∞ | 30 | 100 | p[1]=0, p[3]=0, p[4]=0 |
| 1 | **1** | 0 | **10** | 60 | 30 | 100 | p[2]=1 |
| 2 | **3** | 0 | 10 | 50 | **30** | 90 | p[2]=3, p[4]=3 |
| 3 | **2** | 0 | 10 | **50** | 30 | 60 | p[4]=2 |
| 4 | **4** | 0 | 10 | 50 | 30 | **60** | — |

**结果**：
| 终点 | 最短路径 | 距离 |
|------|----------|------|
| 0→1 | 0→1 | 10 |
| 0→2 | 0→3→2 | 50 |
| 0→3 | 0→3 | 30 |
| 0→4 | 0→3→2→4 | 60 |

**Dijkstra 算法代码**：
```c
void Dijkstra(MGraph G, int v, int dist[], int path[]) {
    bool S[MaxVertexNum];  // 已确定最短路径的顶点集
    for (int i = 0; i < G.vexnum; i++) {
        dist[i] = G.Edge[v][i];
        S[i] = false;
        if (dist[i] < INT_MAX) path[i] = v;
        else path[i] = -1;
    }
    S[v] = true;
    dist[v] = 0;

    for (int i = 1; i < G.vexnum; i++) {
        int min = INT_MAX, u = -1;
        for (int j = 0; j < G.vexnum; j++)  // 找最近的未确定顶点
            if (!S[j] && dist[j] < min) { min = dist[j]; u = j; }
        S[u] = true;
        for (int j = 0; j < G.vexnum; j++)  // 松弛
            if (!S[j] && dist[u] + G.Edge[u][j] < dist[j]) {
                dist[j] = dist[u] + G.Edge[u][j];
                path[j] = u;
            }
    }
}
```

**时间复杂度**：O(n²)
**局限**：不适用于负权边

### 5.2 Floyd 算法（各顶点间最短路径）

**核心思想**：动态规划，逐步允许经过中间顶点来缩短距离。

$$A^{(k)}[i][j] = \min(A^{(k-1)}[i][j],\; A^{(k-1)}[i][k] + A^{(k-1)}[k][j])$$

**例题**：对以下有向网用 Floyd 算法求所有顶点对间最短路径。

```
     0    1    2
0  [ 0    4   11]
1  [ 6    0    2]
2  [ 3    ∞    0]
```

**矩阵推导**：

**A⁽⁻¹⁾（初始，无中间顶点）**：
```
     0    1    2
0  [ 0    4   11]
1  [ 6    0    2]
2  [ 3    ∞    0]
```

**A⁽⁰⁾（允许经过顶点0）**：
- A[1][0]+A[0][2] = 6+11 = 17 > A[1][2]=2，不变
- A[2][0]+A[0][1] = 3+4 = 7 < A[2][1]=∞，更新！

```
     0    1    2
0  [ 0    4   11]
1  [ 6    0    2]
2  [ 3    7    0]
```

**A⁽¹⁾（允许经过顶点0,1）**：
- A[0][1]+A[1][2] = 4+2 = 6 < A[0][2]=11，更新！
- A[2][1]+A[1][0] = 7+6 = 13 > A[2][0]=3，不变

```
     0    1    2
0  [ 0    4    6]
1  [ 6    0    2]
2  [ 3    7    0]
```

**A⁽²⁾（允许经过顶点0,1,2）**：
- A[0][2]+A[2][1] = 6+7 = 13 > A[0][1]=4，不变
- A[1][2]+A[2][0] = 2+3 = 5 < A[1][0]=6，更新！

```
     0    1    2
0  [ 0    4    6]
1  [ 5    0    2]
2  [ 3    7    0]
```

**最终结果**：

| 起点→终点 | 最短距离 | 路径 |
|-----------|----------|------|
| 0→1 | 4 | 0→1 |
| 0→2 | 6 | 0→1→2 |
| 1→0 | 5 | 1→2→0 |
| 1→2 | 2 | 1→2 |
| 2→0 | 3 | 2→0 |
| 2→1 | 7 | 2→0→1 |

**Floyd 算法代码**：
```c
void Floyd(MGraph G, int A[][MaxVertexNum], int path[][MaxVertexNum]) {
    for (int i = 0; i < G.vexnum; i++)
        for (int j = 0; j < G.vexnum; j++) {
            A[i][j] = G.Edge[i][j];
            path[i][j] = -1;
        }

    for (int k = 0; k < G.vexnum; k++)         // 中间顶点
        for (int i = 0; i < G.vexnum; i++)      // 起点
            for (int j = 0; j < G.vexnum; j++)  // 终点
                if (A[i][k] + A[k][j] < A[i][j]) {
                    A[i][j] = A[i][k] + A[k][j];
                    path[i][j] = k;
                }
}
```

**时间复杂度**：O(n³)
**优势**：可处理负权边（不可处理负权回路）

## 六、拓扑排序

### 6.1 概念

对**有向无环图（DAG）**的顶点排成一个线性序列，使得对任意边 <v,w>，v 排在 w 前面。

### 6.2 算法步骤

1. 从图中选一个**入度为 0** 的顶点输出
2. 从图中删除该顶点及其所有出边
3. 重复 1-2，直到所有顶点输出（成功）或找不到入度为 0 的顶点（说明有环）

```c
bool TopologicalSort(ALGraph G) {
    InitStack(S);
    for (int i = 0; i < G.vexnum; i++)
        if (indegree[i] == 0) Push(S, i);

    int count = 0;
    while (!IsEmpty(S)) {
        Pop(S, i);
        printf("%d ", i);
        count++;
        for (ArcNode *p = G.vertices[i].first; p; p = p->next) {
            int v = p->adjvex;
            if (--indegree[v] == 0) Push(S, v);
        }
    }
    return count == G.vexnum;  // false 说明有环
}
```

> **应用**：判断有向图是否有环、工程活动排序、课程安排等。

## 七、关键路径

### 7.1 AOE 网

在带权有向图中，以**顶点表示事件**，**边表示活动**，权值表示活动持续时间，称为 **AOE 网**。

- **源点**：入度为 0 的顶点（工程起点）
- **汇点**：出度为 0 的顶点（工程终点）
- **关键路径**：从源点到汇点的**最长路径**
- **关键活动**：关键路径上的活动（不可延迟）

### 7.2 四个变量

| 变量 | 含义 |
|------|------|
| **etv**（earliest time of vertex） | 事件最早发生时间 |
| **ltv**（latest time of vertex） | 事件最迟发生时间 |
| **ete**（earliest time of edge） | 活动最早开始时间 |
| **lte**（latest time of edge） | 活动最迟开始时间 |

**关系**：
- ete\<v,w\> = etv\<v\>（活动最早开始 = 弧尾事件最早发生）
- lte\<v,w\> = ltv\<w\> - dut\<v,w\>（活动最迟开始 = 弧头事件最迟 - 活动时间）
- **关键活动**：ete = lte 的活动

### 7.3 完整推导例题

**题目**：AOE 网如下，求关键路径。

```
事件（顶点）: v0 ~ v5（6个事件）
活动（边）:
  v0→v1: a0=3
  v0→v2: a1=4
  v1→v3: a2=5
  v1→v4: a3=6
  v2→v3: a4=8
  v2→v4: a5=7
  v3→v5: a6=4
  v4→v5: a7=2
```

**步骤1：求 etv（正向拓扑序）**

```
拓扑序: v0 v1 v2 v3 v4 v5

etv[v0] = 0
etv[v1] = etv[v0] + 3 = 3
etv[v2] = etv[v0] + 4 = 4
etv[v3] = max(etv[v1]+5, etv[v2]+8) = max(8, 12) = 12
etv[v4] = max(etv[v1]+6, etv[v2]+7) = max(9, 11) = 11
etv[v5] = max(etv[v3]+4, etv[v4]+2) = max(16, 13) = 16
```

**步骤2：求 ltv（逆向拓扑序）**

```
ltv[v5] = etv[v5] = 16
ltv[v4] = ltv[v5] - 2 = 14
ltv[v3] = ltv[v5] - 4 = 12
ltv[v2] = min(ltv[v3]-8, ltv[v4]-7) = min(4, 7) = 4
ltv[v1] = min(ltv[v3]-5, ltv[v4]-6) = min(7, 8) = 7
ltv[v0] = min(ltv[v1]-3, ltv[v2]-4) = min(4, 0) = 0
```

**步骤3：求 ete 和 lte**

| 活动 | 边 | dur | ete=etv[尾] | lte=ltv[头]-dur | ete=lte? |
|------|-----|-----|-------------|-----------------|----------|
| a0 | v0→v1 | 3 | etv[v0]=0 | ltv[v1]-3=4 | 0≠4 ✗ |
| a1 | v0→v2 | 4 | etv[v0]=0 | ltv[v2]-4=0 | **0=0 ✓** |
| a2 | v1→v3 | 5 | etv[v1]=3 | ltv[v3]-5=7 | 3≠7 ✗ |
| a3 | v1→v4 | 6 | etv[v1]=3 | ltv[v4]-6=8 | 3≠8 ✗ |
| a4 | v2→v3 | 8 | etv[v2]=4 | ltv[v3]-8=4 | **4=4 ✓** |
| a5 | v2→v4 | 7 | etv[v2]=4 | ltv[v4]-7=7 | 4≠7 ✗ |
| a6 | v3→v5 | 4 | etv[v3]=12 | ltv[v5]-4=12 | **12=12 ✓** |
| a7 | v4→v5 | 2 | etv[v4]=11 | ltv[v5]-2=14 | 11≠14 ✗ |

**关键活动**：a1, a4, a6

**关键路径**：**v0 → v2 → v3 → v5**（总长度 = 4+8+4 = 16）

```
       a1(4)      a4(8)      a6(4)
  v0 ------→ v2 ------→ v3 ------→ v5
  |           |
  |a0(3)     |a5(7)
  ↓           ↓
  v1 ------→ v4 ------→ v5
     a3(6)      a7(2)
  v1 ------→ v3
     a2(5)
```

> ⚠️ **关键路径上所有活动的总浮动时间为 0**，任何关键活动延迟都会导致工期延长。

## 真题考法

### 选择题典型考法

1. **图的性质**：n 个顶点的连通图最少/最多多少条边
2. **邻接矩阵 vs 邻接表**：空间复杂度对比、适用场景
3. **DFS/BFS 序列判断**：给出图的存储结构，判断遍历序列
4. **拓扑排序**：判断哪些序列是合法拓扑序
5. **关键路径**：判断关键活动

### 大题典型考法

1. **Dijkstra 手工推导**（画表格，逐步更新 dist 数组）
2. **Floyd 矩阵推导**（写出每个 A^(k) 矩阵）
3. **关键路径完整计算**（etv→ltv→ete→lte→关键路径）
4. **Prim/Kruskal 构造最小生成树**

## 典型例题

### 例题1：邻接矩阵 DFS/BFS 序列

**题目**：无向图邻接矩阵如下，写出从 A 开始的 DFS 和 BFS 序列。
```
     A  B  C  D  E
A  [ 0  1  1  0  0 ]
B  [ 1  0  0  1  1 ]
C  [ 1  0  0  0  1 ]
D  [ 0  1  0  0  1 ]
E  [ 0  1  1  1  0 ]
```

**思路**：邻接矩阵按列号小的优先访问。

**解答**：
- **DFS**：A→B→D→E→C（A访问B，B访问D，D访问E，E访问C）
- **BFS**：A→B→C→D→E（A出队，入B,C；B出队，入D,E；C出队，无新；D出队，无新；E出队）

**总结**：邻接矩阵中 DFS/BFS 的访问顺序取决于矩阵行列排列顺序，按编号小的先访问。

### 例题2：Kruskal 构造最小生成树

**题目**：无向网有 6 个顶点 10 条边，边权从小到大为：
(2,4,1)(0,2,2)(1,2,3)(3,5,4)(0,1,5)(1,3,6)(2,5,7)(0,3,8)(1,4,9)(4,5,10)
求最小生成树及权值。

**思路**：Kruskal 每次选最小的不构成回路的边。

**解答**：
| 步骤 | 选边 | 权值 | 是否成环 | 操作 |
|------|------|------|----------|------|
| 1 | (2,4,1) | 1 | 否 | 加入 |
| 2 | (0,2,2) | 2 | 否 | 加入 |
| 3 | (1,2,3) | 3 | 否 | 加入 |
| 4 | (3,5,4) | 4 | 否 | 加入 |
| 5 | (0,1,5) | 5 | **是**（0-2-1已连通） | 跳过 |
| 6 | (1,3,6) | 6 | 否 | 加入 |

已选 5 条边（n-1=5），完成。

**最小生成树**：{(2,4,1), (0,2,2), (1,2,3), (3,5,4), (1,3,6)}
**总权值** = 1+2+3+4+6 = **16**

**总结**：Kruskal 算法的关键是用并查集判断是否成环。

### 例题3：Dijkstra 完整推导

**题目**：有向网邻接矩阵如下，求顶点 0 到各顶点的最短路径。
```
     0    1    2    3
0  [ 0    1    ∞    4 ]
1  [ ∞    0    9    2 ]
2  [ 3    5    0    8 ]
3  [ ∞    ∞    6    0 ]
```

**解答**：

| 步骤 | 选定点 | dist[0] | dist[1] | dist[2] | dist[3] | 说明 |
|------|--------|---------|---------|---------|---------|------|
| 初始 | — | 0 | 1 | ∞ | 4 | |
| 1 | **1** | 0 | **1** | 10 | 3 | 1→3: 1+2=3<4, 更新 |
| 2 | **3** | 0 | 1 | 9 | **3** | 3→2: 3+6=9<10, 更新 |
| 3 | **2** | 0 | 1 | **9** | 3 | 完成 |

**结果**：
| 终点 | 最短距离 | 路径 |
|------|----------|------|
| 0→1 | 1 | 0→1 |
| 0→2 | 9 | 0→1→3→2 |
| 0→3 | 3 | 0→1→3 |

**总结**：Dijkstra 关键步骤——选最近点→更新 dist→重复。

### 例题4：关键路径

**题目**：AOE 网，事件 v0~v3，活动：v0→v1(2), v0→v2(3), v1→v3(4), v2→v3(5)。求关键路径。

**思路**：etv→ltv→ete→lte。

**解答**：

**etv（正向）**：
- etv[0]=0, etv[1]=2, etv[2]=3, etv[3]=max(2+4, 3+5)=8

**ltv（逆向）**：
- ltv[3]=8, ltv[2]=8-5=3, ltv[1]=8-4=4, ltv[0]=min(4-2, 3-3)=0

**ete/lte**：
| 活动 | ete | lte | 关键? |
|------|-----|-----|-------|
| v0→v1 | 0 | 4-2=2 | ✗ |
| v0→v2 | 0 | 3-3=0 | **✓** |
| v1→v3 | 2 | 8-4=4 | ✗ |
| v2→v3 | 3 | 8-5=3 | **✓** |

**关键路径**：**v0→v2→v3**，总长度 = **8**

**总结**：ete=lte 的活动即关键活动，它们构成的路径就是关键路径。

## 易错点

1. ⚠️ **Dijkstra 不适用于负权边**，Floyd 可处理负权边但不能有负权回路
2. ⚠️ **最小生成树不唯一**（当有权值相同的边时），但总权值唯一
3. ⚠️ 邻接矩阵 DFS 时间复杂度是 **O(n²)**，不是 O(n+e)
4. ⚠️ 拓扑排序结果**不唯一**（入度为 0 的顶点可能有多个）
5. ⚠️ 关键路径求 etv 用**正向拓扑序**（取 max），求 ltv 用**逆向拓扑序**（取 min）
6. ⚠️ lte = ltv[弧头] - dur，**不是** ltv[弧尾] + dur
7. ⚠️ 无向图连通分量数 = DFS/BFS 调用次数
8. ⚠️ 有 n 个顶点的连通图至少 n-1 条边（生成树），完全连通图 n(n-1)/2 条边

### 补充1：十字链表（Orthogonal List）

十字链表是**有向图**的一种链式存储结构，结合了邻接表和逆邻接表。

**弧结点结构**：
```c
typedef struct ArcBox {
    int tailvex;          // 弧尾顶点编号
    int headvex;          // 弧头顶点编号
    struct ArcBox *hlink; // 弧头相同的下一条弧
    struct ArcBox *tlink; // 弧尾相同的下一条弧
    // InfoType info;     // 权值
} ArcBox;
```

**顶点结点结构**：
```c
typedef struct VexNode {
    char data;
    ArcBox *firstin;   // 该顶点作为弧头的第一条弧
    ArcBox *firstout;  // 该顶点作为弧尾的第一条弧
} VexNode;
```

**性能分析**：
- 空间复杂度：O(|V| + |E|)
- 求入度：沿 firstin 链遍历，O(入度)
- 求出度：沿 firstout 链遍历，O(出度)
- 优势：同时方便求入度和出度

### 补充2：邻接多重表（Adjacency Multilist）

邻接多重表是**无向图**的一种链式存储结构，解决邻接表中每条边存储两次的问题。

**边结点结构**：
```c
typedef struct ENode {
    int ivex, jvex;              // 边的两个顶点
    struct ENode *ilink, *jlink; // 依附于 ivex、jvex 的下一条边
    // InfoType info;
} ENode;
```

**顶点结点结构**：
```c
typedef struct VexNode {
    char data;
    ENode *firstedge;  // 第一条依附于该顶点的边
} VexNode;
```

**与邻接表的对比**：
| 特性 | 邻接表 | 邻接多重表 |
|------|--------|------------|
| 每条边存储次数 | 2次（两个顶点各一次） | 1次 |
| 删除边的操作 | 需要删除两个结点 | 只需删除一个结点 |
| 适用场景 | 通用 | 需要频繁删除边的场景 |

### 补充3：图的四种存储结构对比

| 存储结构 | 空间复杂度 | 适用场景 | 求度的便利性 |
|----------|-----------|----------|-------------|
| 邻接矩阵 | O(\|V\|²) | 稠密图 | 遍历行/列 |
| 邻接表 | O(\|V\|+\|E\|) | 稀疏图 | 出度方便，入度不方便 |
| 十字链表 | O(\|V\|+\|E\|) | 有向图 | 入度出度都方便 |
| 邻接多重表 | O(\|V\|+\|E\|) | 无向图 | 方便 |

### 补充4：BFS 求最短路径（无权图）

对于**无权图**，BFS 可以求出从源点到所有其他顶点的最短路径（边数最少的路径）。

**原理**：BFS 按层次遍历，先访问的顶点距离源点更近。

**算法**：
```c
void BFS_ShortestPath(ALGraph G, int v, int dist[], int path[]) {
    InitQueue(Q);
    for (int i = 0; i < G.vexnum; i++) dist[i] = -1;
    dist[v] = 0;
    EnQueue(Q, v);
    while (!IsEmpty(Q)) {
        DeQueue(Q, v);
        for (ArcNode *p = G.vertices[v].first; p; p = p->next) {
            if (dist[p->adjvex] == -1) {  // 未访问
                dist[p->adjvex] = dist[v] + 1;
                path[p->adjvex] = v;
                EnQueue(Q, p->adjvex);
            }
        }
    }
}
```

**注意**：BFS 只适用于**无权图**或**边权相同**的图。对于带权图，必须用 Dijkstra 或 Floyd。

### 补充5：有向无环图（DAG）描述表达式

DAG 可以用来描述表达式的计算过程，共享公共子表达式，节省存储空间。

例如表达式：`((a+b)*(b*(c+d)) + (c+d)*e) * ((c+d)*e)`

可以构建 DAG，其中 `(c+d)` 和 `(c+d)*e` 等公共子表达式只计算一次，多个父结点共享同一个子结点。

**与表达式树的区别**：
- 表达式树中，相同的子表达式会重复存储
- DAG 中，相同的子表达式共享同一个结点
- DAG 可以节省空间，并且反映了实际计算中的优化策略
