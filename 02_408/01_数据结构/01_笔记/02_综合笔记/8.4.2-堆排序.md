# 堆排序

## 📊 考情分析
- 考试频率：**高频考点**，几乎每年必考
- 分值：3~10分（选择题 + 综合应用题）
- 题型：选择题（判断大根堆/小根堆、排序结果）、综合应用题（建堆过程、手算排序）、算法题（调整堆的代码）
- 真题考查方式：给定序列建堆、建堆后的排序过程、调整操作的比较次数、判断建堆时间复杂度

## 一、基本概念

### 1. 堆的定义
若 n 个关键字序列 $L[1..n]$ 满足以下**任一**性质，则称为**堆（Heap）**：

- **大根堆（大顶堆/最大堆）**：$L(i) \geq L(2i)$ 且 $L(i) \geq L(2i+1)$，其中 $1 \leq i \leq \lfloor n/2 \rfloor$
- **小根堆（小顶堆/最小堆）**：$L(i) \leq L(2i)$ 且 $L(i) \leq L(2i+1)$，其中 $1 \leq i \leq \lfloor n/2 \rfloor$

**通俗理解**：
- 大根堆：完全二叉树中，**根 ≥ 左、右**（每个父节点都不小于其子节点）
- 小根堆：完全二叉树中，**根 ≤ 左、右**（每个父节点都不大于其子节点）

> **对比 BST**：二叉搜索树是 左 ≤ 根 ≤ 右；大根堆是 根 ≥ 左、右

### 2. 基本术语
- **堆顶**：完全二叉树的根节点（数组下标 1 或 0）
- **堆底**：完全二叉树的最后一个叶子节点
- **非终端节点**：有孩子的节点，编号满足 $i \leq \lfloor n/2 \rfloor$
- **下坠（调整）**：将不满足堆性质的节点向下移动的操作

### 3. 二叉树顺序存储中的关键公式（编号从 1 开始）

| 关系 | 公式 |
|------|------|
| i 的左孩子 | $2i$ |
| i 的右孩子 | $2i+1$ |
| i 的父节点 | $\lfloor i/2 \rfloor$ |
| i 所在层次 | $\lceil \log_2(n+1) \rceil$ 或 $\lfloor \log_2 n \rfloor + 1$ |
| 判断 i 是否有左孩子 | $2i \leq n$ ? |
| 判断 i 是否有右孩子 | $2i+1 \leq n$ ? |
| 判断 i 是否是叶子/分支节点 | $i > \lfloor n/2 \rfloor$ ?（是叶子） |

> **注意**：若编号从 0 开始，则左孩子为 $2i+1$，右孩子为 $2i+2$，父节点为 $\lfloor (i-1)/2 \rfloor$

## 二、核心知识

### 1. 建立大根堆

**思路**：把所有非终端节点都检查一遍，是否满足大根堆的要求，如果不满足，则进行调整。

**步骤**：
1. 从最后一个非终端节点（$\lfloor n/2 \rfloor$）开始，从后往前依次检查
2. 检查当前节点是否满足 大根堆（根 ≥ 左、右）
3. 若不满足，将当前节点与**更大的一个孩子**互换
4. 互换后可能导致下层不满足堆性质，需继续向下调整（"下坠"）

**建堆示例**：初始序列 `[53, 17, 78, 09, 45, 65, 87, 32]`（编号从1开始）

```
初始完全二叉树：
         53
       /    \
      17     78
     / \    / \
    09  45 65  87
   /
  32

非终端节点编号：i ≤ ⌊8/2⌋ = 4，即节点 1、2、3、4

从 i=4 开始检查（逆序）：
i=4(09)：左右孩子32、45，45>09，交换 → 09和45互换
i=3(78)：左右孩子65、87，87>78，交换 → 78和87互换
i=2(17)：左右孩子09(已变)、45，45>17，交换 → 17和45互换；
         交换后17在位置4，其孩子为32，32>17，继续交换
i=1(53)：左右孩子45(已变)、87，87>53，交换 → 53和87互换；
         交换后53在位置3，其孩子为65、78，78>53，继续交换

建堆结果：[87, 45, 78, 32, 17, 65, 53, 09]
```

### 2. 堆排序过程（基于大根堆）

**核心思想**：大根堆的堆顶元素关键字最大，每趟将堆顶元素加入有序子序列。

**步骤**：
1. 建立大根堆
2. 每趟：将堆顶（最大元素）与待排序序列的**最后一个元素交换**
3. 待排序序列长度减 1
4. 将堆顶元素向下调整（"下坠"），恢复大根堆性质
5. 重复 n-1 趟

**排序示例**：基于上面建好的大根堆 `[87, 45, 78, 32, 17, 65, 53, 09]`

| 趟数 | 操作 | 待排序区 | 有序区 |
|------|------|---------|--------|
| 初始 | 建堆 | 87 45 78 32 17 65 53 09 | — |
| 第1趟 | 87与09交换，调整 | 78 45 65 32 17 09 53 | **87** |
| 第2趟 | 78与53交换，调整 | 65 45 53 32 17 09 | **78 87** |
| 第3趟 | 65与09交换，调整 | 53 45 09 32 17 | **65 78 87** |
| 第4趟 | 53与17交换，调整 | 45 32 09 17 | **53 65 78 87** |
| 第5趟 | 45与17交换，调整 | 32 17 09 | **45 53 65 78 87** |
| 第6趟 | 32与09交换，调整 | 17 09 | **32 45 53 65 78 87** |
| 第7趟 | 17与09交换 | 09 | **17 32 45 53 65 78 87** |
| 结果 | — | — | **09 17 32 45 53 65 78 87** |

> **注意**：基于大根堆的堆排序得到**递增序列**；基于小根堆得到**递减序列**

### 3. 代码实现（C语言）

```c
// 对以 k 为根的子树进行"下坠"调整（大根堆）
// n 为堆的元素个数
void HeadAdjust(int A[], int k, int n) {
    A[0] = A[k];                     // A[0]暂存子树的根节点
    for (int i = 2 * k; i <= n; i *= 2) {  // 沿key较大的子节点向下筛选
        if (i < n && A[i] < A[i + 1])      // 如果有右孩子且右孩子更大
            i++;                            // 取右孩子
        if (A[0] >= A[i])                   // 如果根节点已经最大
            break;                          // 调整结束
        else {
            A[k] = A[i];                    // 将A[i]调整到双亲节点上
            k = i;                          // 修改k值，以便继续向下筛选
        }
    }
    A[k] = A[0];                     // 被筛选节点的值放入最终位置
}

// 建立大根堆
void BuildMaxHeap(int A[], int n) {
    for (int i = n / 2; i > 0; i--)   // 从最后一个非终端节点开始
        HeadAdjust(A, i, n);
}

// 堆排序（递增排序）
void HeapSort(int A[], int n) {
    BuildMaxHeap(A, n);               // 建立大根堆
    for (int i = n; i > 1; i--) {     // n-1趟
        swap(A[1], A[i]);             // 堆顶与堆底元素交换
        HeadAdjust(A, 1, i - 1);      // 将剩余的 i-1 个元素调整为大根堆
    }
}
```

**代码关键点**：
- `A[0]` 作为暂存单元，不存储有效数据（编号从1开始）
- `HeadAdjust` 的核心是"下坠"操作：每次与较大的孩子比较并交换
- 建堆时从 `n/2` 到 `1` 逆序处理
- 排序时堆顶与堆底交换，然后对 `n-1` 个元素重新调整

## 三、深入理解

### 1. 设计思想
- 堆排序是**简单选择排序的改进**
- 简单选择排序每趟找最大值需要 O(n) 比较
- 堆排序利用**堆的结构特性**，每趟找最大值只需 O(log n) 比较
- 堆本质上是一棵**完全二叉树**，利用数组的下标关系来模拟树结构

### 2. 复杂度分析

#### 建堆的时间复杂度 = O(n)

**推导过程**：
- 一个节点每"下坠"一层，最多只需比较关键字 **2 次**
- 若树高为 h，某结点在第 i 层，则向下调整最多需要下坠 h-i 层，关键字比较次数不超过 2(h-i)
- 第 i 层最多有 $2^{i-1}$ 个结点，只有第 1~(h-1) 层的结点才有可能需要下坠调整

$$\sum = \sum_{i=1}^{h-1} 2^{i-1} \cdot 2(h-i) = \sum_{i=1}^{h-1} 2^{i}(h-i)$$

令 $j = h-i$，则：

$$\sum = \sum_{j=1}^{h-1} 2^{h-j} \cdot j = 2^h \sum_{j=1}^{h-1} \frac{j}{2^j}$$

由等比数列求和公式，$\sum_{j=1}^{\infty} \frac{j}{2^j} = 2$，因此：

$$\sum \leq 2^h \cdot 2 \leq 2n \leq 4n$$

**结论**：建堆的时间复杂度 = **O(n)**

#### 排序的时间复杂度 = O(n log₂n)

- 共需 n-1 趟
- 每趟交换后需要将根节点"下坠"调整
- 根节点最多下坠 h-1 层，每下坠一层最多比较 2 次
- 每趟排序复杂度不超过 O(h) = O(log₂n)
- 总时间复杂度 = (n-1) × O(log₂n) = **O(n log₂n)**

#### 总结

| 项目 | 复杂度 |
|------|--------|
| 建堆 | O(n) |
| 排序 | O(n log₂n) |
| **总时间复杂度** | **O(n) + O(n log₂n) = O(n log₂n)** |
| 最好时间复杂度 | O(n log₂n) |
| 最坏时间复杂度 | O(n log₂n) |
| 平均时间复杂度 | O(n log₂n) |
| **空间复杂度** | **O(1)**（仅用了常数个辅助变量） |

### 3. 稳定性分析

**堆排序是不稳定的！**

**反例**：序列 `[1, 2a, 2b]`

```
建大根堆（左右孩子一样大时优先和左孩子交换）：
         2b
        /  \
      2a    1

第1趟：2b与1交换 → [1, 2a, 2b]（有序区：2b）
调整后大根堆：[2a, 1]

第2趟：2a与1交换 → [1, 2a]（有序区：2a, 2b）

排序结果：1 2a 2b
初始序列：1 2a 2b
→ 本例中相对位置未变

但考虑 [2a, 1, 2b]：
建大根堆：2b 1 2a
第1趟：2b与2a交换 → [2a, 1, 2b] → 但2b在最后
实际得到：1 2a 2b
初始：2a在2b前，排序后2a在2b前 → 本例稳定

经典反例：[2a, 2b, 1]
建大根堆：2b 2a 1（2b成为堆顶，2a在左子树）
第1趟：2b与1交换 → [1, 2a, 2b]（但实际2a在堆中被调换）
```

**简记**：堆排序在调整过程中可能交换不相邻的元素，导致关键字相同的元素相对位置改变。

## 四、真题考法分析

### 常考题型
1. **建堆过程**：给定序列，手画建堆过程（从后往前调整非终端节点）
2. **排序结果**：给定堆，写出排序后的序列
3. **判断大根堆/小根堆**：给定数组，判断是否为堆
4. **调整操作**：插入/删除元素后的堆调整
5. **复杂度分析**：建堆O(n)的推导是高频考点
6. **堆的性质判断**：稳定性、空间复杂度等

### 命题规律
- **建堆过程必考**：几乎每年都会涉及堆的调整过程
- **建堆时间复杂度 = O(n)** 这个结论的推导是难点
- 堆排序通常与优先队列、Top-K问题结合考查
- 注意大根堆得到递增序列，小根堆得到递减序列

## 五、典型例题

### 例1：建堆手算题
**题目**：将序列 `[5, 1, 7, 3, 9, 4, 8, 2]` 建立为大根堆，写出建堆过程。

**答案**：
```
初始完全二叉树（编号1开始）：
         5
       /   \
      1     7
     / \   / \
    3   9 4   8
   /
  2

非终端节点：i ≤ ⌊8/2⌋ = 4，即节点4(3)、3(7)、2(1)、1(5)

i=4：3的孩子是2，3>2，不用调整 → [5,1,7,3,9,4,8,2]
i=3：7的孩子是4和8，8>7，交换7和8 → [5,1,8,3,9,4,7,2]
i=2：1的孩子是3和9，9>1，交换1和9 → [5,9,8,3,1,4,7,2]
     1到了位置5，无孩子，不用继续
i=1：5的孩子是9和8，9>5，交换5和9 → [9,5,8,3,1,4,7,2]
     5到了位置2，孩子是3和1，5>3和1，不用继续

大根堆：[9, 5, 8, 3, 1, 4, 7, 2]
```

### 例2：排序过程题
**题目**：对上面建好的大根堆进行堆排序，写出前3趟的排序结果。

**答案**：
```
大根堆：[9, 5, 8, 3, 1, 4, 7, 2]

第1趟：9和2交换 → [2, 5, 8, 3, 1, 4, 7, | 9]
      调整：2下坠→8上→2到3位，7>2，继续下坠
      → [8, 5, 7, 3, 1, 4, 2, | 9]

第2趟：8和2交换 → [2, 5, 7, 3, 1, 4, | 7, 9]
      调整：7>5>2，7到堆顶
      → [7, 5, 4, 3, 1, 2, | 7, 9]  
      (修正：8与2交换后 [2,5,7,3,1,4,2], 7>5>2, 下坠得 [7,5,4,3,1,2])
      → [7, 5, 4, 3, 1, 2, | 8, 9]

第3趟：7和2交换 → [2, 5, 4, 3, 1, | 7, 8, 9]
      调整：5>4>2，5到堆顶
      → [5, 3, 4, 2, 1, | 7, 8, 9]
```

### 例3：复杂度分析题
**题目**：堆排序最好和最坏情况下的时间复杂度分别是什么？

**答案**：
- 堆排序的最好、最坏、平均时间复杂度**都是 O(n log₂n)**
- 这是堆排序的一个优点：**时间复杂度稳定**
- 不像快速排序最坏情况退化为 O(n²)

### 例4：真题
**题目**：已知关键字序列 `{5, 8, 12, 19, 28, 20, 15, 22}` 是小根堆，插入关键字 3 后调整得到的新小根堆是？

**答案**：
```
原小根堆：
         5
       /   \
      8     12
     / \   / \
    19 28 20 15
   /
  22

插入3到堆底（位置9）：
         5
       /   \
      8     12
     / \   / \
    19 28 20 15
   / \
  22  3

3 < 8（父节点），交换：3和8交换
         5
       /   \
      3     12
     / \   / \
    19 28 20 15
   / \
  22  8

3 < 5（父节点），交换：3和5交换
         3
       /   \
      5     12
     / \   / \
    19 28 20 15
   / \
  22  8

新小根堆：[3, 5, 12, 19, 28, 20, 15, 22, 8]
```

### 例5：算法题
**题目**：设计一个算法，判断一个数组 `A[1..n]` 是否为大根堆。

**答案**：
```c
bool IsMaxHeap(int A[], int n) {
    for (int i = 1; i <= n / 2; i++) {   // 只需检查非终端节点
        int left = 2 * i;
        int right = 2 * i + 1;
        if (left <= n && A[i] < A[left])
            return false;
        if (right <= n && A[i] < A[right])
            return false;
    }
    return true;
}
```

## 六、易错点

1. **建堆时的检查顺序**：从最后一个非终端节点（$\lfloor n/2 \rfloor$）**从后往前**检查，不是从前往后
2. **左右孩子一样大时的处理**：优先和**左孩子**交换
3. **堆排序得到的序列**：大根堆 → 递增序列；小根堆 → 递减序列
4. **建堆时间复杂度是 O(n)**，不是 O(n log n)！这是常见错误
5. **堆排序的空间复杂度是 O(1)**，但归并排序是 O(n)
6. **堆排序是不稳定的**
7. **堆用数组实现**，下标从1开始时，左孩子是 2i，右孩子是 2i+1
8. **"下坠"操作的比较次数**：每下坠一层最多比较 2 次（左、右孩子各比较一次）
