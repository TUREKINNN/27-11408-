# 队列的链式实现

## 📊 考情分析
- **考试频率**：中等，选择题偶尔出现
- **分值**：2~6分
- **题型**：选择题、代码填空题
- **真题考查方式**：链队列的定义、入队出队操作、与循环队列的区别

## 一、基本概念

### 1. 链队列的定义
用**链式存储**方式实现的队列，通常采用**单链表**，队头指针指向链头，队尾指针指向链尾。

**特点**：
- 入队：在链尾插入新结点
- 出队：删除链头结点

### 2. 带头结点 vs 不带头结点
| 方式 | 初始化 | 入队 | 出队 |
|------|--------|------|------|
| 带头结点 | front=rear=头结点 | 尾插法 | 删除第一个结点 |
| 不带头结点 | front=rear=NULL | 尾插法（需特殊处理） | 删除第一个结点 |

## 二、核心知识

### 1. 链队列的定义（带头结点）
```c
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} LinkNode;

typedef struct {
    LinkNode *front, *rear;  // 队头和队尾指针
} LinkQueue;

// 初始化
void InitQueue(LinkQueue *Q) {
    Q->front = Q->rear = (LinkNode *)malloc(sizeof(LinkNode));
    Q->front->next = NULL;  // 头结点next指向NULL
}

// 判空
bool QueueEmpty(LinkQueue Q) {
    return (Q.front == Q.rear);
}

// 入队（尾插法）
void EnQueue(LinkQueue *Q, ElemType x) {
    LinkNode *p = (LinkNode *)malloc(sizeof(LinkNode));
    p->data = x;
    p->next = NULL;
    Q->rear->next = p;  // 新结点链接到队尾
    Q->rear = p;         // 队尾指针指向新结点
}

// 出队
bool DeQueue(LinkQueue *Q, ElemType *x) {
    if (Q->front == Q->rear)  // 队空
        return false;
    LinkNode *p = Q->front->next;  // 指向第一个结点
    *x = p->data;
    Q->front->next = p->next;
    if (Q->rear == p)  // 如果删除的是最后一个结点
        Q->rear = Q->front;  // 队尾指针指向头结点
    free(p);
    return true;
}
```

### 2. 链队列的定义（不带头结点）
```c
// 初始化
void InitQueue2(LinkQueue *Q) {
    Q->front = Q->rear = NULL;
}

// 判空
bool QueueEmpty2(LinkQueue Q) {
    return (Q.front == NULL);
}

// 入队（尾插法）
void EnQueue2(LinkQueue *Q, ElemType x) {
    LinkNode *p = (LinkNode *)malloc(sizeof(LinkNode));
    p->data = x;
    p->next = NULL;
    if (Q->front == NULL) {  // 空队列
        Q->front = Q->rear = p;
    } else {
        Q->rear->next = p;
        Q->rear = p;
    }
}

// 出队
bool DeQueue2(LinkQueue *Q, ElemType *x) {
    if (Q->front == NULL)  // 队空
        return false;
    LinkNode *p = Q->front;
    *x = p->data;
    Q->front = p->next;
    if (Q->rear == p)  // 如果删除的是最后一个结点
        Q->rear = NULL;
    free(p);
    return true;
}
```

## 三、深入理解

### 1. 设计思想
链队列利用单链表的尾插法实现入队，头删法实现出队。

**优点**：
- 不存在队满的问题（除非内存不足）
- 增删操作方便

**缺点**：
- 存储密度低（需要额外的指针空间）
- 不能随机存取

### 2. 复杂度分析
| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 初始化 | O(1) | 创建头结点 |
| 判空 | O(1) | 比较指针 |
| 入队 | O(1) | 尾插法 |
| 出队 | O(1) | 头删法 |
| 读队头 | O(1) | 取第一个结点的值 |

### 3. 链队列 vs 循环队列
| 比较项 | 循环队列 | 链队列 |
|--------|---------|--------|
| 存储方式 | 连续存储 | 离散存储 |
| 队满条件 | (rear+1)%MaxSize==front | 内存不足 |
| 空间利用 | 可能浪费 | 按需分配 |
| 适用场景 | 队列大小可预估 | 队列大小不可预估 |

## 五、典型例题

### 概念题
**题目**：链队列与循环队列相比，其优点是（）
A. 存储密度高  B. 不会出现队满的情况  C. 随机存取  D. 以上都不是

**答案**：B
**解析**：链队列采用动态分配，不会出现队满的情况（除非内存不足）。

### 算法题
**题目**：设计一个算法，利用链队列实现BFS（广度优先搜索）。

```c
void BFS(Graph G, int start) {
    LinkQueue Q;
    InitQueue(&Q);
    
    bool visited[G.vexNum];
    for (int i = 0; i < G.vexNum; i++)
        visited[i] = false;
    
    visited[start] = true;
    EnQueue(&Q, start);
    
    while (!QueueEmpty(Q)) {
        ElemType v;
        DeQueue(&Q, &v);
        printf("%d ", v);
        
        // 访问v的所有邻接结点
        for (int w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
            if (!visited[w]) {
                visited[w] = true;
                EnQueue(&Q, w);
            }
        }
    }
}
```

## 六、易错点

1. **入队操作的指针修改**：
   - 先修改新结点的next，再修改队尾指针
   - ❌ 错误：顺序颠倒会导致丢失链表

2. **出队操作的特殊情况**：
   - 删除最后一个结点时，需要修改队尾指针
   - ❌ 错误：忘记修改队尾指针

3. **带头结点和不带头结点的区别**：
   - 带头结点：`Q.front == Q.rear` 表示队空
   - 不带头结点：`Q.front == NULL` 表示队空

4. **出队操作的内存释放**：
   - 出队后需要 `free` 结点
   - ❌ 错误：忘记释放内存导致内存泄漏
