# 7.4.2 B+ 树

## 一、基本概念

B+ 树是 B 树的变体，主要用于**数据库和文件系统**的索引。

### m 阶 B+ 树的定义

1. 每个**分支结点**最多有 m 棵子树（孩子结点）
2. 非叶根结点**至少有两棵子树**，其他分支结点**至少有 $\lceil m/2 \rceil$ 棵子树**
3. 结点的**子树个数与关键字个数相等**
4. 所有**叶结点包含全部关键字**及指向记录的指针，叶结点按关键字大小**顺序排列**，且**相邻叶结点按大小顺序相互链接**
5. 所有分支结点中仅包含其**各个子结点中关键字的最大值**及指向子结点的指针

---

## 二、B+ 树 vs B 树

| 特性 | B 树 | B+ 树 |
|------|------|-------|
| n 个关键字对应子树数 | n+1 | **n** |
| 根关键字数 | [1, m-1] | [1, m] |
| 其他结点关键字数 | [$\lceil m/2 \rceil$-1, m-1] | [$\lceil m/2 \rceil$, m] |
| 关键字分布 | 各结点中不重复 | **叶结点包含全部关键字**，非叶结点也会出现 |
| 查找路径 | 可能停在任何层 | **一定走到最下面一层** |
| 叶子结点链接 | 无 | **有顺序链表连接** |
| 适用场景 | 内存中的索引 | **数据库/文件系统的磁盘索引** |

---

## 三、查找操作

### 1. 多路查找（从根开始）
从根结点开始，根据关键字值选择合适的子树，直到到达叶子结点。

### 2. 顺序查找（沿叶子链表）
从某个叶子结点开始，沿链表顺序遍历。

> **关键特点**：B+ 树中，无论查找成功与否，**最终都要走到叶子结点**。

---

## 四、为什么 B+ 树适合数据库？

1. **支持范围查询**：叶子结点通过链表连接，可以高效地进行范围扫描
2. **磁盘 I/O 友好**：分支结点不含记录指针，每个结点能容纳更多关键字，树更矮，磁盘 I/O 更少
3. **查询稳定**：任何查询都需要走到叶子结点，查询时间稳定

## 五、考点总结

| 考点 | 内容 |
|------|------|
| B+ 树定义 | 5 条性质 |
| 与 B 树的区别 | 子树数、关键字分布、查找路径、叶子链接 |
| 查找特点 | 一定走到叶子结点 |
| 适用场景 | 数据库索引 |

> B 树：n 个关键字 → n+1 棵子树
> B+ 树：n 个关键字 → n 棵子树
