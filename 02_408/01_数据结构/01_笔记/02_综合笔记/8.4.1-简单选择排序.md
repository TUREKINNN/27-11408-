# 简单选择排序

## 📊 考情分析
- 考试频率：中高频考点，几乎每年都会涉及
- 分值：2~5分（选择题）或结合算法题考查
- 题型：选择题（判断稳定性、复杂度）、综合应用题（手算排序过程）
- 真题考查方式：给定初始序列手写排序过程、判断排序趟数、比较关键字次数、判断稳定性

## 一、基本概念

### 1. 严格定义
**简单选择排序**（Selection Sort）是一种基于"选择"思想的排序算法。每一趟在待排序元素中选取关键字**最小**（或最大）的元素，加入有序子序列。

### 2. 基本术语
- **趟（pass）**：选择排序的一轮完整操作过程
- **有序子序列**：已排好序的前缀部分
- **待排序序列**：尚未处理的后缀部分
- **n个元素需要 n-1 趟处理**（最后一个元素自然有序）

## 二、核心知识

### 1. 算法思想
每一趟排序：
1. 在待排序元素 `[i, n-1]` 中找到关键字最小的元素 `min`
2. 将 `min` 与第 `i` 个位置的元素交换
3. 有序子序列扩展到 `[0, i]`
4. 重复直到只剩一个元素

### 2. 排序过程示例

初始序列：`49 38 65 97 76 13 27`

| 趟数 | 操作 | 排序结果 |
|------|------|---------|
| 初始 | — | 49 38 65 97 76 13 27 |
| 第1趟 | 选最小值13，与49交换 | **13** 38 65 97 76 49 27 |
| 第2趟 | 选最小值27，与38交换 | **13 27** 65 97 76 49 38 |
| 第3趟 | 选最小值38，与65交换 | **13 27 38** 97 76 49 65 |
| 第4趟 | 选最小值49，与97交换 | **13 27 38 49** 76 97 65 |
| 第5趟 | 选最小值65，与76交换 | **13 27 38 49 65** 97 76 |
| 第6趟 | 选最小值76，与97交换 | **13 27 38 49 65 76** 97 |
| 第7趟 | 剩余1个，无需处理 | **13 27 38 49 65 76 97** |

### 3. 代码实现（C语言）

```c
// 简单选择排序
void SelectSort(int A[], int n) {
    int i, j, min;
    for (i = 0; i < n - 1; i++) {       // n-1趟
        min = i;                          // 记录最小元素下标
        for (j = i + 1; j < n; j++) {    // 在A[i...n-1]中找最小元素
            if (A[j] < A[min])
                min = j;                  // 更新最小元素下标
        }
        if (min != i)
            swap(A[i], A[min]);           // 将最小元素交换到第i个位置
    }
}

// 交换函数
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
```

**代码关键点**：
- 外层循环 `i` 从 0 到 `n-2`（共 n-1 趟）
- `min` 只记录下标，不直接交换
- 每趟只做**一次交换**（将最小元素换到正确位置）
- 最后一趟（`i = n-2`）只比较两个元素，之后无需处理

## 三、深入理解

### 1. 设计思想
- 简单选择排序的核心思想是"**选最小、放前面**"
- 每一趟都从剩余元素中选出全局最小的，放入已排序区的末尾
- 这是一种**不稳定的**原地排序算法

### 2. 复杂度分析

#### 时间复杂度

| 情况 | 比较次数 | 交换次数 | 时间复杂度 |
|------|---------|---------|-----------|
| 最好情况 | $\frac{n(n-1)}{2}$ | 0 | O(n²) |
| 最坏情况 | $\frac{n(n-1)}{2}$ | n-1 | O(n²) |
| 平均情况 | $\frac{n(n-1)}{2}$ | — | O(n²) |

**关键结论**：
- **无论有序、逆序还是乱序，都需要 n-1 趟处理**
- 每一趟都需要扫描剩余所有元素找最小值
- 比较次数始终为 $(n-1)+(n-2)+\cdots+1 = \frac{n(n-1)}{2}$
- **元素交换次数 < n-1**（最多 n-1 次，最少 0 次）
- 这是简单选择排序相比冒泡排序的一个优点：**交换次数少**

#### 空间复杂度
- **O(1)**：只用了常数个辅助变量（i, j, min, temp）
- 原地排序算法

### 3. 稳定性分析

**简单选择排序是不稳定的！**

**反例**：序列 `[2, 2, 1]`（带下标标记为 $2_1, 2_2, 1$）

| 步骤 | 操作 | 结果 |
|------|------|------|
| 初始 | — | $2_1$ $2_2$ 1 |
| 第1趟 | 选最小值1，与$2_1$交换 | **1** $2_2$ $2_1$ |
| 排序结果 | — | 1 $2_2$ $2_1$ |

排序前 $2_1$ 在 $2_2$ 之前，排序后 $2_1$ 在 $2_2$ 之后 → **相对位置改变 → 不稳定**

> **易记口诀**：简单选择"选最小"，交换可能乱次序，n-1趟O(n²)，交换少但不稳定。

### 4. 适用性
- 既可以用于**顺序表**，也可以用于**链表**
- 对于链表，只需修改指针而不需要真正交换数据

## 四、真题考法分析

### 常考题型
1. **判断稳定性**：给定序列或关键字，判断排序后是否稳定
2. **比较次数与交换次数计算**：给出初始序列，计算需要多少次比较和交换
3. **排序趟数判断**：n个元素需要几趟
4. **手算排序过程**：给出初始序列，写出每一趟的结果
5. **与其他排序算法的对比**：与冒泡、插入等排序的比较

### 命题规律
- 简单选择排序的**比较次数固定**为 n(n-1)/2，与初始序列无关
- 简单选择排序的**交换次数**与初始序列有关，但最多 n-1 次
- 稳定性判断是高频考点

## 五、典型例题

### 例1：概念题
**题目**：简单选择排序的比较次数与初始排列是否有序有关吗？

**答案**：**无关**。无论初始序列如何，都需要 n-1 趟，每趟从 i 到 n-1 扫描找最小值，总比较次数恒为 $\frac{n(n-1)}{2}$。

### 例2：计算题
**题目**：对序列 `[5, 3, 7, 1, 9, 4]` 进行简单选择排序，问：
1. 共需几趟？
2. 总共需要比较多少次？
3. 最多需要交换几次？

**答案**：
1. 6-1 = **5趟**
2. $\frac{6 \times 5}{2} = 15$ 次
3. 最多 5 次（n-1）

### 例3：稳定性判断题
**题目**：对关键字序列 `[3a, 1, 3b, 2]`（其中 $3_a$ 和 $3_b$ 关键字相同）进行简单选择排序，排序后 $3_a$ 和 $3_b$ 的相对位置是否改变？

**答案**：
- 第1趟选最小值1，与$3_a$交换 → `[1, 3a, 3b, 2]`
- 第2趟选最小值2，与$3_a$交换 → `[1, 2, 3b, 3a]`
- $3_a$ 和 $3_b$ 的相对位置改变 → **不稳定**

### 例4：真题（408统考）
**题目**：以下排序算法中，（ ）算法在最好情况下的时间复杂度与其他三个不同。
A. 冒泡排序　B. 简单选择排序　C. 希尔排序　D. 直接插入排序

**答案**：D
- 冒泡排序最好情况 O(n)（有序时，一趟扫描即可）
- 简单选择排序最好情况 O(n²)（无论什么情况都要 n-1 趟比较）
- 希尔排序最好情况 O(n log n)
- 直接插入排序最好情况 O(n)（有序时，每趟只需比较1次）

## 六、易错点

1. **简单选择排序的比较次数是固定的 O(n²)**，不像冒泡排序和插入排序那样最好情况可以达到 O(n)
2. **不稳定**：不要因为"选择"这个操作看起来很"温和"就认为它是稳定的
3. **交换次数少 ≠ 时间复杂度低**：虽然交换次数少，但比较次数仍然是 O(n²)
4. **最后一趟不需要处理**：n-1 趟后只剩一个元素，自然有序
5. **适用范围广**：既可以用于顺序表也可以用于链表
