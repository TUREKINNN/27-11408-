# 7.4.1 B 树

## 一、基本概念

**B 树**，又称**多路平衡查找树**。B 树中所有允许的孩子个数的最大值称为 B 树的**阶**，通常用 m 表示。

### m 阶 B 树的定义

一棵 m 阶 B 树或为空树，或为满足如下特性的 m 叉树：

1. 每个结点**至多**有 m 棵子树，即至多含有 m-1 个关键字
2. 若根结点不是终端结点，则**至少有两棵子树**
3. 除根结点外的所有非叶结点**至少有 $\lceil m/2 \rceil$ 棵子树**，即至少含有 $\lceil m/2 \rceil - 1$ 个关键字
4. 所有非叶结点的结构为：$n, P_0, K_1, P_1, K_2, P_2, \ldots, K_n, P_n$
   - $K_i$ 为关键字，且 $K_1 < K_2 < \cdots < K_n$
   - $P_i$ 为指向子树的指针，$P_{i-1}$ 所指子树中所有关键字 $< K_i$，$P_i$ 所指子树中所有关键字 $> K_i$
   - $\lceil m/2 \rceil - 1 \leq n \leq m - 1$
5. **所有叶结点都出现在同一层次上**，不带信息（视为查找失败结点）

---

## 二、核心特性

| 条件 | 子树数 | 关键字数 |
|------|--------|----------|
| 根结点 | [2, m] | [1, m-1] |
| 其他非叶结点 | [$\lceil m/2 \rceil$, m] | [$\lceil m/2 \rceil$-1, m-1] |

### 关键字的排列
$$\text{子树}_0 < K_1 < \text{子树}_1 < K_2 < \text{子树}_2 < \cdots < K_n < \text{子树}_n$$

类比 BST：左 < 根 < 右

---

## 三、查找操作

### 查找过程
1. 在当前结点中顺序查找（或折半查找）关键字
2. 若找到 → 查找成功
3. 若未找到 → 沿指针进入相应子树，继续查找
4. 到达叶子结点 → 查找失败

> **注意**：每个结点内的关键字是有序的，可以使用**折半查找**。

---

## 四、B 树的高度

### 含 n 个关键字的 m 阶 B 树

**最小高度**（每个结点尽可能满）：

$$h \geq \log_m(n+1)$$

**最大高度**（每个结点尽可能少）：

$$h \leq \log_{\lceil m/2 \rceil} \frac{n+1}{2} + 1$$

> **综合**：$\log_m(n+1) \leq h \leq \log_{\lceil m/2 \rceil} \frac{n+1}{2} + 1$

---

## 五、考点总结

| 考点 | 内容 |
|------|------|
| B 树定义 | 5 条性质 |
| 结点关键字数 | 根: [1, m-1]，其他: [$\lceil m/2 \rceil$-1, m-1] |
| 查找过程 | 结点内查找 + 沿指针进入子树 |
| 高度范围 | $\log_m(n+1) \leq h \leq \log_{\lceil m/2 \rceil} \frac{n+1}{2} + 1$ |
| 叶结点 | 都在同一层次，是查找失败结点 |

> 1. B 树的核心性质：**尽可能"满"、尽可能"平衡"**
> 2. n 个关键字的 B 树有 **n+1 个叶子结点**（失败结点）
> 3. 大部分学校算 B 树高度**不包括叶子结点**
