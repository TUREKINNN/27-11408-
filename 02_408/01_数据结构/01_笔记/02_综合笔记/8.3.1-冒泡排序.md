# 冒泡排序（Bubble Sort）

## 📊 考情分析
- **考试频率**：高频考点，每年必考
- **分值**：2-4分（选择题）或10分（算法题）
- **题型**：选择题（复杂度分析、稳定性判断）、算法题（手写代码/优化）
- **真题考查方式**：模拟排序过程、分析比较次数和交换次数、优化版本

## 一、基本概念

### 冒泡排序的定义
从后往前（或从前往后）**两两比较相邻元素的值**，若为逆序（即 $A[i-1] > A[i]$），则交换它们，直到序列比较完。称这样过程为"一趟"冒泡排序。

### 核心思想
- 每一趟排序将**关键字最小的元素"冒"到最前面**
- 总共需要进行 $n-1$ 趟冒泡

## 二、核心知识

### 排序过程示例

初始序列：`49, 38, 65, 97, 76, 13, 27, 49`

#### 第1趟（从前往后，两两比较交换）
```
49, 38, 65, 97, 76, 13, 27, 49
38, 49, 65, 97, 76, 13, 27, 49  (49>38, 交换)
38, 49, 65, 97, 76, 13, 27, 49  (49<65, 不换)
...
38, 49, 65, 13, 76, 97, 27, 49  (97>76, 交换)
...
13, 38, 49, 65, 76, 97, 27, 49  (最小元素13冒到最前)
```
第1趟结束：最小元素13到位。

#### 第2趟
从下标1开始继续冒泡，27到位。

#### 第3趟
38到位。

#### 第4趟
49到位（注意两个49的相对位置不变）。

#### 第5趟
无交换，排序完成。

### 代码实现

```c
void BubbleSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;                    // 本趟是否发生交换
        for (int j = n - 1; j > i; j--) {     // 从后往前冒泡
            if (A[j-1] > A[j]) {              // 逆序则交换
                swap(A[j-1], A[j]);
                flag = true;
            }
        }
        if (flag == false)                     // 本趟无交换，已有序
            return;
    }
}
```

### 优化：带标志位
```c
// 关键：若某一趟排序没有发生"交换"，说明此时已经整体有序
if (flag == false)
    return;
```

## 三、深入理解

### 复杂度分析

| 情况 | 比较次数 | 交换次数 | 时间复杂度 |
|------|---------|---------|-----------|
| 最好（有序） | $n-1$ | 0 | $O(n)$ |
| 最坏（逆序） | $\frac{n(n-1)}{2}$ | $\frac{n(n-1)}{2}$ | $O(n^2)$ |
| 平均 | — | — | $O(n^2)$ |

- **空间复杂度**：$O(1)$
- **稳定性**：**稳定**（只有 $A[j-1] > A[j]$ 时才交换，相等时不交换）

### 最坏情况推导
- 第1趟：比较 $n-1$ 次
- 第2趟：比较 $n-2$ 次
- 第i趟：比较 $n-i$ 次
- 总比较次数：$(n-1)+(n-2)+\dots+1 = \frac{n(n-1)}{2}$

### 冒泡排序是否适用于链表？
**可以**。可以从前往后"冒泡"，每一趟将更大的元素"冒"到链尾。

### 常考题型

3. 判断冒泡排序的稳定性

- 常与快速排序对比出题

## 六、易错点

1. **冒泡排序是稳定的**：因为只有严格大于时才交换
2. **最好情况是 $O(n)$**：带标志位优化后，已经有序时只需一趟
3. **每趟确定一个元素的最终位置**：第i趟确定第i小的元素
4. **交换操作需要3次赋值**：每次交换需要3次移动元素
5. **冒泡排序适用于链表**：这与快速排序不同
