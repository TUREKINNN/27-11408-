# 栈在括号匹配中的应用

## 📊 考情分析
- **考试频率**：中等，选择题和算法题常考
- **分值**：2~10分
- **题型**：选择题、算法设计题
- **真题考查方式**：括号匹配算法实现、匹配失败的情况分析

## 一、基本概念

### 1. 括号匹配问题
给定一个字符串，检查其中的括号是否匹配。

**括号类型**：
- 圆括号：`(` 和 `)`
- 方括号：`[` 和 `]`
- 花括号：`{` 和 `}`

### 2. 匹配规则
- 每个左括号必须有一个对应的右括号
- 左右括号必须正确嵌套
- 最后出现的左括号最先被匹配（**LIFO**）

**示例**：
- `{[()()]} `：匹配 ✓
- `{[(])}`：不匹配 ✗（括号交叉）
- `((()`：不匹配 ✗（左括号多余）

## 二、核心知识

### 1. 算法思想
用栈实现括号匹配：
1. 依次扫描所有字符
2. 遇到**左括号**：压入栈顶
3. 遇到**右括号**：
   - 如果栈空：右括号单身，匹配失败
   - 如果栈非空：弹出栈顶元素，检查是否匹配

### 2. 算法实现
```c
bool BracketMatch(char str[], int n) {
    SqStack S;
    InitStack(&S);
    
    for (int i = 0; i < n; i++) {
        char ch = str[i];
        
        // 遇到左括号，压入栈顶
        if (ch == '(' || ch == '[' || ch == '{') {
            Push(&S, ch);
        }
        // 遇到右括号
        else if (ch == ')' || ch == ']' || ch == '}') {
            if (StackEmpty(S))  // 右括号单身
                return false;
            
            char topElem;
            Pop(&S, &topElem);
            
            // 检查是否匹配
            if (ch == ')' && topElem != '(')
                return false;
            if (ch == ']' && topElem != '[')
                return false;
            if (ch == '}' && topElem != '{')
                return false;
        }
    }
    
    // 检查栈是否为空
    if (!StackEmpty(S))  // 左括号单身
        return false;
    
    return true;
}
```

### 3. 匹配失败的三种情况
1. **左括号单身**：扫描完所有字符后，栈非空
2. **右括号单身**：遇到右括号时，栈为空
3. **左右括号不匹配**：弹出的左括号与当前右括号不匹配

## 三、深入理解

### 1. 设计思想
利用栈的**后进先出**特性，保证括号的正确嵌套：
- 最后出现的左括号最先被匹配
- 每出现一个右括号，就"消耗"一个左括号

### 2. 复杂度分析
| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 扫描字符串 | O(n) | 遍历所有字符 |
| 入栈/出栈 | O(1) | 每个字符最多入栈出栈一次 |
| **总体** | **O(n)** | n为字符串长度 |

**空间复杂度**：O(n)，最坏情况下所有字符都是左括号。

## 五、典型例题

### 概念题
**题目**：使用栈实现括号匹配时，遇到右括号应该（）
A. 直接入栈  B. 弹出栈顶元素检查是否匹配  C. 忽略  D. 返回错误

**答案**：B
**解析**：遇到右括号时，应该弹出栈顶的左括号，检查是否匹配。

### 算法题
**题目**：设计一个算法，判断一个表达式中的括号是否匹配，支持三种括号。

```c
bool IsBalanced(char expr[]) {
    SqStack S;
    InitStack(&S);
    
    for (int i = 0; expr[i] != '\0'; i++) {
        char ch = expr[i];
        
        if (ch == '(' || ch == '[' || ch == '{') {
            Push(&S, ch);
        } else if (ch == ')' || ch == ']' || ch == '}') {
            if (StackEmpty(S))
                return false;
            
            char top;
            Pop(&S, &top);
            
            if ((ch == ')' && top != '(') ||
                (ch == ']' && top != '[') ||
                (ch == '}' && top != '{')) {
                return false;
            }
        }
    }
    
    return StackEmpty(S);
}
```

## 六、易错点

1. **匹配失败的判断**：
   - 需要检查三种情况：左括号单身、右括号单身、不匹配
   - ❌ 错误：只检查一种情况

2. **栈空的判断**：
   - 遇到右括号时，需要先判断栈是否为空
   - ❌ 错误：直接弹出栈顶元素

3. **匹配检查**：
   - 弹出的左括号必须与当前右括号类型相同
   - ❌ 错误：不检查类型

4. **最终检查**：
   - 扫描完所有字符后，需要检查栈是否为空
   - ❌ 错误：忘记检查
