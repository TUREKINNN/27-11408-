## 一、基本概念

### 1.1 栈（Stack）

**栈**是限定仅在**表尾**（栈顶）进行插入和删除操作的线性表。

数学描述：设栈 $S = (a_1, a_2, \ldots, a_n)$，其中 $a_1$ 为**栈底**（bottom），$a_n$ 为**栈顶**（top）。

- **后进先出**（LIFO, Last In First Out）
- 插入操作称为 **入栈**（Push）
- 删除操作称为 **出栈**（Pop）
- 只能访问**栈顶元素**

### 1.2 队列（Queue）

**队列**是限定仅在**表尾**插入、**表头**删除的线性表。

数学描述：设队列 $Q = (a_1, a_2, \ldots, a_n)$，其中 $a_1$ 为**队头**（front），$a_n$ 为**队尾**（rear）。

- **先进先出**（FIFO, First In First Out）
- 插入操作称为 **入队**（Enqueue）
- 删除操作称为 **出队**（Dequeue）

### 1.3 分类与比较

| 特性 | 栈 | 队列 |
|------|-----|------|
| 原则 | LIFO | FIFO |
| 插入位置 | 栈顶 | 队尾 |
| 删除位置 | 栈顶 | 队头 |
| 典型应用 | 递归、括号匹配、表达式求值 | BFS、缓冲区、任务调度 |
| 受限方式 | 只能在一端操作 | 两端分别操作 |

## 二、核心知识

### 2.1 栈的存储结构

#### 顺序栈

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];
    int top;                 // 栈顶指针（下标）
} SqStack;
```

**初始化**（top指向-1，即空栈时top = -1）：

```c
void InitStack(SqStack *S) {
    S->top = -1;
}
```

**判空**：

```c
bool StackEmpty(SqStack S) {
    return S.top == -1;
}
```

**入栈**：

```c
bool Push(SqStack *S, ElemType e) {
    if (S->top == MaxSize - 1)  // 栈满
        return false;
    S->data[++S->top] = e;     // 先top加1，再赋值
    return true;
}
```

**出栈**：

```c
bool Pop(SqStack *S, ElemType *e) {
    if (S->top == -1)           // 栈空
        return false;
    *e = S->data[S->top--];    // 先取值，再top减1
    return true;
}
```

**读栈顶**：

```c
bool GetTop(SqStack S, ElemType *e) {
    if (S.top == -1) return false;
    *e = S.data[S.top];
    return true;
}
```

> 约定1：`top = -1` 表示空栈，`++top` 入栈，`top--` 出栈（本笔记采用）
> 约定2：`top = 0` 表示空栈，`data[top++]` 入栈，`data[--top]` 出栈
> **考试时必须先声明采用哪种约定！**

#### 链栈

```c
typedef struct StackNode {
    ElemType data;
    struct StackNode *next;
} StackNode, *LinkStack;
```

- 不存在栈满上溢的问题（除非内存耗尽）
- 入栈/出栈都是 O(1)
- 通常**不需要头节点**（直接在链头操作）

```c
bool Push(LinkStack *S, ElemType e) {
    StackNode *p = (StackNode *)malloc(sizeof(StackNode));
    p->data = e;
    p->next = *S;
    *S = p;
    return true;
}

bool Pop(LinkStack *S, ElemType *e) {
    if (*S == NULL) return false;
    StackNode *p = *S;
    *e = p->data;
    *S = p->next;
    free(p);
    return true;
}
```

### 2.2 队列的存储结构

#### 普通顺序队列的问题

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];
    int front, rear;          // 队头、队尾指针
} SqQueue;
```

**假溢出问题**：不断出队入队后，rear 到达数组末尾，但 front 前面还有空位。此时队列未满却无法入队。

**解决方案**：使用**循环队列**。

#### 循环队列

将数组"首尾相接"，通过**取模运算**实现循环。

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];
    int front, rear;
} SqQueue;
```

**初始化**：

```c
void InitQueue(SqQueue *Q) {
    Q->front = 0;
    Q->rear = 0;
}
```

**队列长度**：

```c
int QueueLength(SqQueue Q) {
    return (Q.rear - Q.front + MaxSize) % MaxSize;
}
```

**入队**：

```c
bool EnQueue(SqQueue *Q, ElemType e) {
    if ((Q->rear + 1) % MaxSize == Q->front)  // 队满
        return false;
    Q->data[Q->rear] = e;
    Q->rear = (Q->rear + 1) % MaxSize;
    return true;
}
```

**出队**：

```c
bool DeQueue(SqQueue *Q, ElemType *e) {
    if (Q->front == Q->rear)  // 队空
        return false;
    *e = Q->data[Q->front];
    Q->front = (Q->front + 1) % MaxSize;
    return true;
}
```

> **方案一**：牺牲一个存储单元
> - **队空条件**：`front == rear`
> - **队满条件**：`(rear + 1) % MaxSize == front`
> - 队列最大容量：MaxSize - 1
>
> **方案二**：增设 size 变量
> - 队空：`size == 0`
> - 队满：`size == MaxSize`
>
> **方案三**：增设 tag 标志
> - 最近操作是入队则 tag = 1，出队则 tag = 0
> - 队空：`front == rear && tag == 0`
> - 队满：`front == rear && tag == 1`

### 2.3 栈的应用

#### （1）括号匹配

```c
bool BracketCheck(char *str, int length) {
    SqStack S;
    InitStack(&S);
    for (int i = 0; i < length; i++) {
        if (str[i] == '(' || str[i] == '[' || str[i] == '{') {
            Push(&S, str[i]);       // 左括号入栈
        } else {
            if (StackEmpty(S))      // 栈空遇右括号
                return false;
            ElemType top;
            Pop(&S, &top);
            if (str[i] == ')' && top != '(') return false;
            if (str[i] == ']' && top != '[') return false;
            if (str[i] == '}' && top != '{') return false;
        }
    }
    return StackEmpty(S);  // 栈空则全部匹配
}
```

#### （2）表达式求值

**中缀表达式** → **后缀表达式**（逆波兰表达式）的转换规则：

1. 从左到右扫描中缀表达式
2. **操作数**直接输出
3. **左括号**入栈
4. **右括号**：弹出栈顶元素并输出，直到遇到左括号（左括号弹出但不输出）
5. **运算符**：
   - 若栈空或栈顶为左括号，直接入栈
   - 若当前运算符优先级 **>** 栈顶，入栈
   - 若当前运算符优先级 **≤** 栈顶，弹出栈顶并输出，然后继续比较

**优先级**：`*`、`/`（2）> `+`、`-`（1）> `(` （0，不比较）

**后缀表达式求值**：

1. 从左到右扫描
2. 操作数入栈
3. 运算符：弹出两个操作数，计算后结果入栈
4. 最终栈中唯一元素即为结果

#### （3）递归

递归的实现依赖**系统栈**（函数调用栈）。每次递归调用将返回地址、参数、局部变量压栈；返回时弹栈恢复。

> 任何递归算法都可以用**显式栈**改写为非递归版本。这是高频考点，如二叉树的非递归遍历。

### 2.4 队列的应用

1. **BFS（广度优先搜索）**：[[02_408/01_数据结构/01_笔记/02_综合笔记/04-图]]中的BFS使用队列
2. **层次遍历二叉树**：[[02_408/01_数据结构/01_笔记/02_综合笔记/03-树和二叉树]]
3. **操作系统中的任务调度**：FCFS调度策略
4. **缓冲区**：打印机缓冲、消息队列

### 2.5 特殊队列

#### 双端队列（Deque）

两端都可以进行入队和出队操作。

- **输入受限的双端队列**：只允许一端入队，两端都可以出队
- **输出受限的双端队列**：只允许一端出队，两端都可以入队

#### 优先队列

每次出队的是优先级最高的元素（通常用堆实现，见[[02_408/01_数据结构/01_笔记/02_综合笔记/06-排序]]）。

## 三、深入理解

### 3.1 出栈序列的合法性（卡特兰数）

n 个不同元素依次入栈，出栈序列的个数为**卡特兰数**：

$$
C_n = \frac{1}{n+1}\binom{2n}{n} = \frac{(2n)!}{(n+1)! \cdot n!}
$$

**判断合法出栈序列的方法**：

模拟法：按照给定的出栈序列，依次判断每个出栈元素：
1. 如果该元素已在栈中且不在栈顶→不合法
2. 如果该元素还未入栈→先将它之前的元素全部入栈，再出栈
3. 如果该元素在栈顶→直接出栈

### 3.2 两个栈共享空间

```c
typedef struct {
    ElemType data[MaxSize];
    int top1;  // 栈1的栈顶（从左往右增长）
    int top2;  // 栈2的栈顶（从右往左增长）
} SharedStack;
```

- 栈1空：`top1 == -1`；栈2空：`top2 == MaxSize`
- 栈满：`top1 + 1 == top2`
- 优点：两个栈此消彼长，空间利用率高

### 3.3 用两个栈模拟队列

栈A负责入队，栈B负责出队：
- 入队：push到A
- 出队：若B非空，从B pop；若B空，将A全部倒入B，再从B pop

均摊时间复杂度：每个元素最多被移动2次，总时间 O(1)（均摊）。

### 3.4 用两个队列模拟栈

方法：保持一个队列始终为空。
- 入栈：入非空队列
- 出栈：将非空队列的前 n-1 个元素转移到空队列，最后一个出队

时间复杂度：O(n)。

### 3.5 与其他知识的联系

- 栈是实现[[02_408/01_数据结构/01_笔记/02_综合笔记/03-树和二叉树]]非递归遍历的关键工具
- 栈是[[02_408/01_数据结构/01_笔记/02_综合笔记/07-算法设计]]中递归转非递归的核心
- 队列是[[02_408/01_数据结构/01_笔记/02_综合笔记/04-图]]BFS的基础
- 中缀转后缀、后缀求值是表达式处理的经典应用

## 四、真题考法分析

### 4.1 常考具体考点

1. **合法出栈序列判断**：给出入栈序列，判断哪些是合法的出栈序列
2. **循环队列的空满判断**：front和rear的关系
3. **栈在表达式中的应用**：中缀→后缀转换过程
4. **栈与递归的关系**：递归过程中栈的变化
5. **双端队列的出队序列**：比单栈更复杂的合法性判断

### 4.2 命题规律和趋势

- 出栈序列几乎是**每年必考**的选择题
- 循环队列的判空/判满条件也是高频选择题
- 表达式转换偶尔作为大题出现

### 4.3 选择题常见陷阱

1. **1,2,3入栈，合法出栈序列**：123, 132, 213, 231, 321 都合法，只有 312 不合法！
2. **循环队列取模不要忘**：rear 前进是 `(rear+1)%MaxSize`
3. **队满牺牲一个单元**：实际容量是 MaxSize-1
4. **栈顶指针初始值**：top = -1 时 `data[++top]` 入栈；top = 0 时 `data[top++]` 入栈，别搞混

## 五、典型例题

### 例题1：概念理解题（选择题风格）

**题目**：若元素 a, b, c, d, e, f 依次进栈，下列哪个不是合法的出栈序列？
A. b, d, c, a, f, e
B. a, c, b, e, d, f
C. c, b, d, a, e, f
D. d, e, c, a, b, f

**思路分析**：逐一模拟每个选项的出栈过程。

**详细解答**：

**选项D模拟**：
- d出栈：a, b, c, d依次入栈，d出栈 → 栈中有[c, b, a]
- e出栈：e入栈，e出栈 → 栈中有[c, b, a]
- c出栈：c在栈顶，c出栈 → 栈中有[b, a]
- a出栈：a不在栈顶（b在栈顶）→ **不合法！**

**答案：D**

**总结归纳**：遇到这种题，关键是**模拟**：维护一个栈，按出栈序列逐个处理。如果需要出栈的元素不在栈顶，且已经过了入栈时机，则不合法。

---

### 例题2：计算分析题（填空题风格）

**题目**：一个循环队列 Q，数组大小 MaxSize = 8，当前 front = 3, rear = 7。
（1）队列中有多少个元素？
（2）若连续执行 2 次入队操作后，rear 的值是多少？
（3）此时（rear = 1）再执行 1 次出队，front 的值是多少？

**思路分析**：用循环队列的取模公式逐步计算。

**详细解答**：

（1）元素个数 = (rear - front + MaxSize) % MaxSize = (7 - 3 + 8) % 8 = **4**

（2）入队2次：rear = (7 + 1) % 8 = 0，rear = (0 + 1) % 8 = **1**

（3）出队1次：front = (3 + 1) % 8 = **4**

**总结归纳**：循环队列的所有位置计算都要 `% MaxSize`，这是出错最多的地方。

### 例题3：算法设计题（大题风格）

**题目**：利用栈的基本操作，编写一个算法，将中缀表达式 `A+B*(C-D)-E/F` 转换为后缀表达式。

**思路分析**：按照中缀转后缀的规则，维护一个运算符栈，逐个处理每个字符。

**详细解答**：

| 步骤 | 当前字符 | 操作 | 输出（后缀） | 栈中内容 |
|------|----------|------|-------------|----------|
| 1 | A | 操作数直接输出 | A | |
| 2 | + | 栈空，入栈 | A | + |
| 3 | B | 操作数直接输出 | AB | + |
| 4 | * | 优先级 > +，入栈 | AB | +* |
| 5 | ( | 左括号入栈 | AB | +*( |
| 6 | C | 操作数直接输出 | ABC | +*( |
| 7 | - | 优先级 > (，入栈 | ABC | +*(- |
| 8 | D | 操作数直接输出 | ABCD | +*(- |
| 9 | ) | 弹出至 ( | ABCD- | +* |
| 10 | - | 优先级 ≤ *，弹出*；优先级 ≤ +，弹出+；入栈 | ABCD-*+ | - |
| 11 | E | 操作数直接输出 | ABCD-*+E | - |
| 12 | / | 优先级 > -，入栈 | ABCD-*+E | -/ |
| 13 | F | 操作数直接输出 | ABCD-*+EF | -/ |
| 结束 | | 依次弹出栈中运算符 | ABCD-*+EF/- | |

**后缀表达式**：`ABCD-*+EF/-`

**总结归纳**：
- 核心规则：操作数直接输出，运算符看优先级
- 左括号入栈后优先级变为最低（只与右括号配对）
- 右括号触发弹出直到遇到左括号
- 结束后栈中运算符全部弹出

### 例题4：历年真题（2016年408）

**题目**：（2016年408第2题）设有如下图所示的火车调度站，入站轨道上车厢编号从左到右为 1, 2, 3, 4, 5。若出站轨道上车厢编号从左到右为 3, 2, 5, 4, 1，则调度站中栈的最小容量至少为（  ）

A. 1  B. 2  C. 3  D. 4

**思路分析**：模拟整个入栈出栈过程，记录栈的最大深度。

**详细解答**：

| 操作 | 栈中内容 | 栈深度 |
|------|----------|--------|
| 1入栈 | [1] | 1 |
| 2入栈 | [1, 2] | 2 |
| 3入栈 | [1, 2, 3] | 3 |
| 3出栈 | [1, 2] | 2 |
| 2出栈 | [1] | 1 |
| 4入栈 | [1, 4] | 2 |
| 5入栈 | [1, 4, 5] | 3 |
| 5出栈 | [1, 4] | 2 |
| 4出栈 | [1] | 1 |
| 1出栈 | [] | 0 |

栈最大深度为 **3**，所以最小容量至少为 3。

**答案：C**

**总结归纳**：求栈的最小容量就是求**模拟过程中的最大栈深度**。逐个模拟入栈出栈过程，记录栈的实时大小。

## 六、易错点

> 牺牲一个空间方案下，队满是 `(rear+1)%MaxSize == front`，不是 `rear == MaxSize-1`（那是普通顺序队列）。

> `(rear - front + MaxSize) % MaxSize`，分子中 **+MaxSize** 不能省！当 rear < front 时会得到负数。

> 对于入栈序列 1, 2, 3，出栈序列 3, 1, 2 是**不合法**的！因为 3 先出栈意味着 1, 2, 3 都入栈了，此时栈中从顶到底为 3, 2, 1。3 出栈后栈顶是 2，不可能直接出 1。

> 左括号入栈后不参与优先级比较，直到遇到右括号才弹出。左括号本身**不输出**。

> 弹出两个操作数时，**先弹出的是右操作数，后弹出的是左操作数**。例如栈中 [3, 5]，遇 `-`，计算 5 - 3 = 2（不是 3 - 5）。

> 顺序栈：入栈前检查 `top == MaxSize-1`
> 链栈：一般不需要检查（动态分配）

### 补充1：双端队列的出栈序列判断

双端队列（Deque）是一种两端都可以进行插入和删除操作的线性表。考试中常考的变体有：

- **输入受限的双端队列**：只允许从一端插入，两端都可以删除
- **输出受限的双端队列**：只允许从两端插入，一端删除

**重要结论**：栈中合法的出栈序列，在双端队列中一定也合法。但双端队列可以产生更多合法的输出序列。

对于输入序列 1,2,3,4：
- 栈的合法出栈序列数 = 卡特兰数 $C_4 = \frac{1}{5}\binom{8}{4} = 14$ 种
- 双端队列可以产生更多的合法输出序列

**判断方法**：
1. 对于输入受限的双端队列：在栈的基础上，还可以利用"从右端删除"的能力来产生额外的合法序列
2. 对于输出受限的双端队列：在栈的基础上，还可以利用"从右端插入"的能力

### 补充2：栈在递归中的深层应用

递归函数的执行过程本质上就是栈（函数调用栈/递归工作栈）的操作过程：

**函数调用栈存储的内容**：
1. 调用返回地址
2. 实参（实际参数）
3. 局部变量

**递归调用的栈操作过程**：
- 每进入一层递归 → 将递归调用所需信息压入栈顶
- 每退出一层递归 → 从栈顶弹出相应信息

**递归的缺点**：
1. 效率低：大量函数调用开销
2. 可能导致栈溢出：太多层递归
3. 可能包含重复计算：如斐波那契递归

**递归转非递归的方法**：
- 可以使用**自定义栈**将递归算法改造成非递归算法
- 典型例子：二叉树的非递归遍历（前序、中序、后序）
- 后缀表达式求值也是递归思想的栈实现

### 补充3：特殊矩阵的压缩存储

这是栈和队列章节中数组部分的重要考点：

#### 二维数组地址计算

**行优先存储**：M行N列的二维数组 b[M][N] 中：
$$b[i][j] 的地址 = LOC + (i \times N + j) \times sizeof(ElemType)$$

**列优先存储**：
$$b[i][j] 的地址 = LOC + (j \times M + i) \times sizeof(ElemType)$$

> ⚠️ 注意：矩阵元素的行、列号通常从1开始，而数组下标通常从0开始。考试时必须仔细审题！

#### 对称矩阵的压缩存储

n阶对称矩阵只需存储下三角区（或上三角区），共 $\frac{n(n+1)}{2}$ 个元素。

**按行优先存储下三角区，映射公式**：

$$k = \begin{cases} \frac{i(i-1)}{2} + j - 1, & i \geq j \text{（下三角区和主对角线）} \\ \frac{j(j-1)}{2} + i - 1, & i < j \text{（上三角区，利用对称性）} \end{cases}$$

#### 三角矩阵的压缩存储

下三角矩阵：除主对角线和下三角区外，其余元素都相同（常数c）。

存储策略：下三角区元素按行优先存入一维数组，最后一个位置存储常量c。

$$k = \begin{cases} \frac{i(i-1)}{2} + j - 1, & i \geq j \\ \frac{n(n+1)}{2}, & i < j \text{（存常量c的位置）} \end{cases}$$

#### 三对角矩阵（带状矩阵）

n阶三对角矩阵中，只有主对角线及其上下各一条对角线有非零元素。

按行优先存储，元素 $a_{i,j}$ 的下标映射：
$$k = 2i + j - 3$$

### 补充4：循环队列的更多考点细节

**循环队列判空判满的三种方案对比**：

| 方案 | 队空条件 | 队满条件 | 优缺点 |
|------|----------|----------|--------|
| 牺牲一个单元 | front == rear | (rear+1)%MaxSize == front | 简单，但浪费一个空间 |
| 增设size变量 | size == 0 | size == MaxSize | 不浪费空间，需额外变量 |
| 增设tag标志 | front==rear && tag==0 | front==rear && tag==1 | 不浪费空间，逻辑稍复杂 |

**tag方案的原理**：
- 最近一次操作是入队 → tag = 1
- 最近一次操作是出队 → tag = 0
- 当 front == rear 时：如果 tag == 1 说明是入队导致的"满"，如果 tag == 0 说明是出队导致的"空"
