# 关键路径（Critical Path）

> - 考试频率：★★★
> - 分值：选择题2-4分，大题8-12分
> - 题型：选择题、综合应用题
> - 重点：AOE网络、四变量推导（etv/ltv/ete/lte）、关键活动识别
> - 难度：偏难，需要细心计算

---

## 一、基本概念

### 1.1 AOE 网络定义

**AOE 网**（Activity On Edge Network）是以**边表示活动**、**顶点表示事件**的**带权有向无环图**。

- **边（弧）**：表示一个活动（Activity），边上的权值表示活动的**持续时间**
- **顶点（事件）**：表示一个事件（Event），是前序活动完成、后续活动开始的标志
- **源点**：入度为 0 的顶点，表示整个工程的开始
- **汇点**：出度为 0 的顶点，表示整个工程的结束

**AOE 网的性质：**
- 只有某个事件（顶点）发生后，从该顶点出发的活动（边）才能开始
- 只有进入某顶点的所有活动都完成后，该事件才能发生
- AOE 网中**不能有环**
- 工程的最短完成时间 = 从源点到汇点的**最长路径**长度

### 1.2 AOV 与 AOE 的区别

| 对比项目 | AOV 网 | AOE 网 |
|---------|--------|--------|
| **顶点含义** | 活动 | 事件 |
| **边的含义** | 活动间的优先关系 | 活动（带权值=持续时间） |
| **边权** | 无权（或仅为先后关系） | 有权（持续时间） |
| **关注问题** | 拓扑排序 | 关键路径 |
| **典型应用** | 任务依赖排序 | 工程进度优化 |

### 1.3 关键路径相关概念

| 术语 | 符号 | 含义 |
|------|------|------|
| 事件最早发生时间 | $etv$（Earliest Time of Vertex） | 事件最早可以发生的时间 |
| 事件最晚发生时间 | $ltv$（Latest Time of Vertex） | 不影响工期的前提下，事件最晚必须发生的时间 |
| 活动最早开始时间 | $ete$（Earliest Time of Edge） | 活动最早可以开始的时间 |
| 活动最晚开始时间 | $lte$（Latest Time of Edge） | 不影响工期的前提下，活动最晚必须开始的时间 |

**关键活动：** $ete = lte$ 的活动（没有余量的活动）

**关键路径：** 由关键活动构成的从源点到汇点的路径

**关键路径长度 = 工程最短完成时间**

## 二、算法详解

### 2.1 四变量推导（核心重点）

#### 2.1.1 第一步：求 etv（事件最早发生时间）

**含义：** 每个事件最早何时能发生（所有前驱活动都完成时）

**计算方法：** 从源点出发，按**拓扑序**正向递推

$$etv[j] = \max_{\langle i,j\rangle \in E}\{etv[i] + w(i,j)\}$$

**理解：** 事件 $j$ 的最早发生时间 = 所有指向 $j$ 的活动中，前驱事件最早时间 + 活动持续时间的**最大值**

> 一个事件要等**所有**前驱活动都完成才能发生，所以取**最大值**。

#### 2.1.2 第二步：求 ltv（事件最晚发生时间）

**含义：** 每个事件最晚何时必须发生（不影响工期的前提下）

**计算方法：** 从汇点出发，按**逆拓扑序**反向递推

$$ltv[i] = \min_{\langle i,j\rangle \in E}\{ltv[j] - w(i,j)\}$$

**理解：** 事件 $i$ 的最晚发生时间 = 所有从 $i$ 出发的活动中，后继事件最晚时间 - 活动持续时间的**最小值**

> 要不影响后续活动，取**最小值**（约束最紧的那个后继决定）。

#### 2.1.3 第三步：求 ete（活动最早开始时间）

**含义：** 活动 $a_k = \langle i, j \rangle$ 最早何时可以开始

$$ete(a_k) = etv[i]$$

**理解：** 活动的开始取决于其前驱事件的发生时间，前驱事件发生了就可以开始。

#### 2.1.4 第四步：求 lte（活动最晚开始时间）

**含义：** 活动 $a_k = \langle i, j \rangle$ 最晚何时必须开始（不影响工期）

$$lte(a_k) = ltv[j] - w(i,j)$$

**理解：** 为了不耽误后续事件 $j$ 的最晚发生时间，活动必须在 $ltv[j] - w(i,j)$ 之前开始。

#### 2.1.5 第五步：识别关键活动

$$\text{关键活动} = \{a_k \mid ete(a_k) = lte(a_k)\}$$

即 $etv[i] = ltv[j] - w(i,j)$ 的活动

**关键活动的特点：** 没有余量（松弛时间 = 0），延迟任何关键活动都会延长工程总工期。

### 2.2 算法完整步骤

```
1. 拓扑排序：得到拓扑序列（同时计算 etv）
2. 逆拓扑序：计算 ltv
3. 遍历每条边（活动）：
   - ete = etv[起点]
   - lte = ltv[终点] - 权值
   - 若 ete == lte，则该活动是关键活动
4. 关键活动构成的路径即为关键路径
```

### 2.3 时间复杂度

| 步骤 | 复杂度 |
|------|--------|
| 拓扑排序 + 求 etv | $O(n+m)$ |
| 求 ltv | $O(n+m)$ |
| 求 ete/lte | $O(m)$ |
| **总复杂度** | **$O(n+m)$** |

## 三、代码实现

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXVEX 100
#define INF 65535

// 边结点
typedef struct ArcNode {
    int adjvex;            // 邻接顶点下标
    int weight;            // 边权（活动持续时间）
    struct ArcNode *next;
} ArcNode;

// 顶点结点
typedef struct {
    int data;
    ArcNode *first;
} VNode;

// AOE网（邻接表）
typedef struct {
    VNode vexs[MAXVEX];
    int numVertexes, numEdges;
} AOEGraph;

/**
 * 求关键路径
 * @param G AOE网（邻接表存储）
 */
void CriticalPath(AOEGraph G) {
    int n = G.numVertexes;
    int inDegree[MAXVEX] = {0};
    int etv[MAXVEX] = {0};   // 事件最早发生时间
    int ltv[MAXVEX];         // 事件最晚发生时间
    int stack[MAXVEX], top = -1;  // 拓扑排序用栈
    int topo[MAXVEX], count = 0;  // 拓扑序列
    
    // ========== 第1步：计算入度 ==========
    for (int i = 0; i < n; i++) {
        ArcNode *p = G.vexs[i].first;
        while (p) {
            inDegree[p->adjvex]++;
            p = p->next;
        }
    }
    
    // ========== 第2步：拓扑排序 + 求 etv ==========
    // 入度为0的顶点入栈
    for (int i = 0; i < n; i++) {
        if (inDegree[i] == 0)
            stack[++top] = i;
    }
    
    while (top != -1) {
        int u = stack[top--];
        topo[count++] = u;  // 记录拓扑序
        
        ArcNode *p = G.vexs[u].first;
        while (p) {
            int v = p->adjvex;
            inDegree[v]--;
            
            // 更新 etv：取最大值
            if (etv[u] + p->weight > etv[v]) {
                etv[v] = etv[u] + p->weight;
            }
            
            if (inDegree[v] == 0)
                stack[++top] = v;
            
            p = p->next;
        }
    }
    
    // 检查是否有环
    if (count < n) {
        printf("AOE网中存在环！\n");
        return;
    }
    
    // ========== 第3步：求 ltv ==========
    // 初始化 ltv 为汇点的 etv
    for (int i = 0; i < n; i++) {
        ltv[i] = etv[topo[n - 1]];  // 工程总工期 = 汇点的 etv
    }
    
    // 逆拓扑序计算 ltv
    for (int i = n - 1; i >= 0; i--) {
        int u = topo[i];
        ArcNode *p = G.vexs[u].first;
        while (p) {
            int v = p->adjvex;
            // 更新 ltv：取最小值
            if (ltv[v] - p->weight < ltv[u]) {
                ltv[u] = ltv[v] - p->weight;
            }
            p = p->next;
        }
    }
    
    // ========== 第4步：求 ete、lte，识别关键活动 ==========
    printf("工程总工期：%d\n", etv[topo[n - 1]]);
    printf("关键活动：\n");
    printf("%-8s %-8s %-8s %-8s %-8s %-8s\n",
           "活动", "权值", "etv[i]", "etv[j]", "ete", "lte");
    
    for (int i = 0; i < n; i++) {
        ArcNode *p = G.vexs[i].first;
        while (p) {
            int j = p->adjvex;
            int ete = etv[i];               // 活动最早开始
            int lte = ltv[j] - p->weight;   // 活动最晚开始
            
            printf("<%d,%d>   %-8d %-8d %-8d %-8d %-8d",
                   i, j, p->weight, etv[i], etv[j], ete, lte);
            
            if (ete == lte) {
                printf(" ★关键活动");
            }
            printf("\n");
            
            p = p->next;
        }
    }
}
```

## 四、典型例题

### 例题 1：关键路径求解（大题）

**题目：** 设 AOE 网如下，求关键路径。

```
顶点：v0, v1, v2, v3, v4, v5
活动（边）：
  <v0,v1> = 3     <v0,v2> = 4
  <v1,v3> = 5     <v2,v3> = 6
  <v3,v4> = 2     <v3,v5> = 3
  <v4,v5> = 1
```

图示：
```
v0 --3--> v1 --5--> v3 --2--> v4 --1--> v5
|                               |
4                               3
|                               |
v         v                     v
v2 ------6--------> v3 ---------> v5
```

**解析：**

**第1步：求 etv（拓扑序：v0, v1, v2, v3, v4, v5）**

| 顶点 | 计算过程 | etv |
|------|---------|-----|
| v0 | 源点 | 0 |
| v1 | etv[v0]+3 = 3 | 3 |
| v2 | etv[v0]+4 = 4 | 4 |
| v3 | max(etv[v1]+5, etv[v2]+6) = max(8, 10) = 10 | 10 |
| v4 | etv[v3]+2 = 12 | 12 |
| v5 | max(etv[v3]+3, etv[v4]+1) = max(13, 13) = 13 | 13 |

工程总工期 = etv[v5] = **13**

**第2步：求 ltv（逆拓扑序：v5, v4, v3, v2, v1, v0）**

| 顶点 | 计算过程 | ltv |
|------|---------|-----|
| v5 | 汇点 | 13 |
| v4 | ltv[v5]-1 = 12 | 12 |
| v3 | min(ltv[v4]-2, ltv[v5]-3) = min(10, 10) = 10 | 10 |
| v2 | ltv[v3]-6 = 4 | 4 |
| v1 | ltv[v3]-5 = 5 | 5 |
| v0 | min(ltv[v1]-3, ltv[v2]-4) = min(2, 0) = 0 | 0 |

**第3步：求 ete 和 lte，识别关键活动**

| 活动 | 权值 | ete=etv[i] | lte=ltv[j]-w | ete=lte? |
|------|------|-----------|--------------|----------|
| <v0,v1> | 3 | 0 | 5-3=2 | ❌ |
| <v0,v2> | 4 | 0 | 4-4=0 | ✅ **关键** |
| <v1,v3> | 5 | 3 | 10-5=5 | ❌ |
| <v2,v3> | 6 | 4 | 10-6=4 | ✅ **关键** |
| <v3,v4> | 2 | 10 | 12-2=10 | ✅ **关键** |
| <v3,v5> | 3 | 10 | 13-3=10 | ✅ **关键** |
| <v4,v5> | 1 | 12 | 13-1=12 | ✅ **关键** |

**关键路径：** v0 → v2 → v3 → v5（总权值 = 4+6+3 = 13）
（注意 v3 → v4 → v5 也是关键路径的一部分）

### 例题 2：综合应用题

**题目：** 某工程由 6 个活动 A~F 组成，AOE 网如下：

```
       A(3)
v0 --------> v1
|             |  C(2)
| B(4)        v
|          v3 ----> v4
v             |  E(4)
v2 -----> v3  v
   D(6)       v4 ----> v5
                  F(1)
```

完整边列表：
| 活动 | 起点 | 终点 | 持续时间 |
|------|------|------|---------|
| A | v0 | v1 | 3 |
| B | v0 | v2 | 4 |
| C | v1 | v3 | 2 |
| D | v2 | v3 | 6 |
| E | v3 | v4 | 4 |
| F | v4 | v5 | 1 |

(1) 求每个事件的最早发生时间 etv 和最晚发生时间 ltv。
(2) 求每个活动的最早开始时间 ete 和最晚开始时间 lte。
(3) 找出关键活动和关键路径。
(4) 工程最短完成时间是多少？

**解析：**

**(1) etv 计算（拓扑序 v0,v1,v2,v3,v4,v5）：**
- etv[v0] = 0
- etv[v1] = 0+3 = 3
- etv[v2] = 0+4 = 4
- etv[v3] = max(3+2, 4+6) = max(5, 10) = 10
- etv[v4] = 10+4 = 14
- etv[v5] = 14+1 = 15

**ltv 计算（逆拓扑序）：**
- ltv[v5] = 15
- ltv[v4] = 15-1 = 14
- ltv[v3] = 14-4 = 10
- ltv[v2] = 10-6 = 4
- ltv[v1] = 10-2 = 8
- ltv[v0] = min(8-3, 4-4) = min(5, 0) = 0

**(2) ete 和 lte：**

| 活动 | ete=etv[i] | lte=ltv[j]-w |
|------|-----------|--------------|
| A<v0,v1> | 0 | 8-3=5 |
| B<v0,v2> | 0 | 4-4=0 |
| C<v1,v3> | 3 | 10-2=8 |
| D<v2,v3> | 4 | 10-6=4 |
| E<v3,v4> | 10 | 14-4=10 |
| F<v4,v5> | 14 | 15-1=14 |

**(3) 关键活动（ete == lte）：**
- B: ete=0, lte=0 ✅
- D: ete=4, lte=4 ✅
- E: ete=10, lte=10 ✅
- F: ete=14, lte=14 ✅

**关键路径：** v0 → v2 → v3 → v4 → v5
路径权值：4 + 6 + 4 + 1 = **15**

**(4) 工程最短完成时间 = 15**

## 五、易错点

### 5.1 常见错误

1. **etv 取 max，ltv 取 min**
   - etv：事件要等所有前驱完成 → 取**最大值**
   - ltv：不影响工期 → 取**最小值**
   - 这是最容易搞混的！

2. **ltv 初始化错误**
   - ltv 应初始化为汇点的 etv（工程总工期）
   - 不是初始化为 0 或 INF

3. **逆拓扑序的含义**
   - 按拓扑序列**从后往前**遍历
   - 不是"逆向拓扑排序"

4. **关键路径上活动的余量为 0**
   - $lte - ete = 0$ 的活动是关键活动
   - 余量 > 0 的活动有缓冲时间

5. **关键路径可能不唯一**
   - 一个 AOE 网可能有多条关键路径
   - 只有缩短**所有**关键路径上的活动，才能缩短工期

### 5.2 考试技巧

1. **画表计算**：列出 etv、ltv、ete、lte 四列，不容易出错
2. **验证**：关键路径长度应等于汇点的 etv
3. **注意**：某活动 ete ≠ lte 不代表可以随便延迟，余量 = lte - ete
4. **题目问"工程最短完成时间"** = 汇点的 etv = 关键路径长度

### 6.3 深入理解

**为什么 etv 取 max？**
- 事件必须等所有前驱活动完成才能发生
- 最慢的那个前驱决定事件发生时间
- 类比：木桶的最长边决定等待时间

**为什么 ltv 取 min？**
- 要满足所有后继活动的最晚约束
- 最紧的那个后继决定事件最晚时间
- 类比：多个截止日期取最早的

**关键路径的工程意义：**
- 关键活动没有余量，必须按时完成
- 非关键活动有一定余量（松弛时间）
- 优化工期：压缩关键活动的持续时间
- 注意：压缩后关键路径可能改变

---

> - AOE 网 = 边表示活动、顶点事件的带权有向无环图
> - **四变量公式：**
>   - $etv[j] = \max\{etv[i] + w\}$（拓扑序正向，取最大）
>   - $ltv[i] = \min\{ltv[j] - w\}$（逆拓扑序，取最小）
>   - $ete = etv[\text{起点}]$
>   - $lte = ltv[\text{终点}] - w$
> - **关键活动：** $ete = lte$（余量为 0）
> - **关键路径长度 = 汇点 etv = 工程最短工期**
> - 时间复杂度：$O(n+m)$
