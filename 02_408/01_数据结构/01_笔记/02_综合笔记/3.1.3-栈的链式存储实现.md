# 栈的链式存储实现

## 📊 考情分析
- **考试频率**：中等，选择题偶尔出现
- **分值**：2~4分
- **题型**：选择题、代码填空题
- **真题考查方式**：链栈的定义、进栈出栈操作、与顺序栈的区别

## 一、基本概念

### 1. 链栈的定义
用**链式存储**方式实现的栈，通常采用**单链表**，将链头作为栈顶。

**特点**：
- 进栈：在链头插入新结点（头插法）
- 出栈：删除链头结点

### 2. 带头结点 vs 不带头结点
| 方式 | 初始化 | 进栈 | 出栈 |
|------|--------|------|------|
| 带头结点 | S->next = NULL | 头插法 | 删除第一个结点 |
| 不带头结点 | S = NULL | 头插法（需特殊处理） | 删除第一个结点 |

## 二、核心知识

### 1. 链栈的定义（带头结点）
```c
typedef struct StackNode {
    ElemType data;
    struct StackNode *next;
} StackNode, *LinkStack;

// 初始化
void InitStack(LinkStack *S) {
    *S = (StackNode *)malloc(sizeof(StackNode));
    (*S)->next = NULL;  // 头结点next指向NULL
}

// 判空
bool StackEmpty(LinkStack S) {
    return (S->next == NULL);
}

// 进栈（头插法）
bool Push(LinkStack S, ElemType x) {
    StackNode *p = (StackNode *)malloc(sizeof(StackNode));
    if (p == NULL)
        return false;
    p->data = x;
    p->next = S->next;  // 新结点指向原第一个结点
    S->next = p;         // 头结点指向新结点
    return true;
}

// 出栈
bool Pop(LinkStack S, ElemType *x) {
    if (S->next == NULL)  // 栈空
        return false;
    StackNode *p = S->next;  // 指向第一个结点
    *x = p->data;
    S->next = p->next;  // 头结点指向第二个结点
    free(p);
    return true;
}

// 读栈顶元素
bool GetTop(LinkStack S, ElemType *x) {
    if (S->next == NULL)  // 栈空
        return false;
    *x = S->next->data;
    return true;
}
```

### 2. 链栈的定义（不带头结点）
```c
// 初始化
void InitStack2(LinkStack *S) {
    *S = NULL;  // 栈为空
}

// 判空
bool StackEmpty2(LinkStack S) {
    return (S == NULL);
}

// 进栈（头插法）
bool Push2(LinkStack *S, ElemType x) {
    StackNode *p = (StackNode *)malloc(sizeof(StackNode));
    if (p == NULL)
        return false;
    p->data = x;
    p->next = *S;  // 新结点指向原栈顶
    *S = p;         // 栈顶指针指向新结点
    return true;
}

// 出栈
bool Pop2(LinkStack *S, ElemType *x) {
    if (*S == NULL)  // 栈空
        return false;
    StackNode *p = *S;  // 指向栈顶结点
    *x = p->data;
    *S = p->next;  // 栈顶指针指向下一个结点
    free(p);
    return true;
}
```

## 三、深入理解

### 1. 设计思想
链栈利用单链表的头插法和头删法实现栈的LIFO特性。

**优点**：
- 不存在栈满的问题（除非内存不足）
- 增删操作方便

**缺点**：
- 存储密度低（需要额外的指针空间）
- 不能随机存取

### 2. 复杂度分析
| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 初始化 | O(1) | 创建头结点 |
| 判空 | O(1) | 比较指针 |
| 进栈 | O(1) | 头插法 |
| 出栈 | O(1) | 头删法 |
| 读栈顶 | O(1) | 取第一个结点的值 |

### 3. 链栈 vs 顺序栈
| 比较项 | 顺序栈 | 链栈 |
|--------|--------|------|
| 存储方式 | 连续存储 | 离散存储 |
| 栈满条件 | top == MaxSize-1 | 内存不足 |
| 空间利用 | 可能浪费 | 按需分配 |
| 适用场景 | 栈大小可预估 | 栈大小不可预估 |

## 四、真题考法分析

### 常考题型
1. **代码填空**：链栈的进栈出栈操作
2. **概念辨析**：链栈与顺序栈的区别
3. **判断题**：链栈的操作正确性

### 命题规律
- 重点考查**头插法进栈**和**头删法出栈**
- 常考**带头结点和不带头结点**的区别
- 注意**进栈出栈的指针操作顺序**

## 五、典型例题

### 概念题
**题目**：链栈与顺序栈相比，其优点是（）
A. 存储密度高  B. 不会出现栈满的情况  C. 随机存取  D. 以上都不是

**答案**：B
**解析**：链栈采用动态分配，不会出现栈满的情况（除非内存不足）。

### 算法题
**题目**：设计一个算法，利用链栈将一个十进制数转换为二进制数。

```c
void DecimalToBinary(int n) {
    LinkStack S;
    InitStack(&S);
    
    // 将余数入栈
    while (n > 0) {
        Push(&S, n % 2);
        n = n / 2;
    }
    
    // 出栈输出
    ElemType bit;
    while (!StackEmpty(S)) {
        Pop(&S, &bit);
        printf("%d", bit);
    }
    printf("\n");
}
```

## 六、易错点

1. **进栈操作的指针顺序**：
   - 先修改新结点的next，再修改头结点的next
   - ❌ 错误：顺序颠倒会导致丢失链表

2. **出栈操作的内存释放**：
   - 出栈后需要 `free` 结点
   - ❌ 错误：忘记释放内存导致内存泄漏

3. **带头结点和不带头结点的区别**：
   - 带头结点：`S->next == NULL` 表示栈空
   - 不带头结点：`S == NULL` 表示栈空

4. **进栈操作的实现**：
   - 链栈的进栈是**头插法**，不是尾插法
   - ❌ 错误：使用尾插法
