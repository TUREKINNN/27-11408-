# 05 - 查找

> - **选择题**：常考BST/AVL性质判断、散列表ASL计算、折半查找判定树
> - **大题**：高频考KMP算法的next数组手推过程

---

## 一、顺序查找（Sequential Search）

### 1.1 基本思想

从表的一端开始，逐个比较关键字，直到找到或查完全表。

```c
int Search_Seq(SSTable ST, KeyType key) {
    ST.elem[0] = key;  // 哨兵
    int i = ST.length;
    while (ST.elem[i] != key) i--;
    return i;  // 返回0表示查找失败
}
```

### 1.2 ASL分析

对n个元素的表，查找第i个元素需比较 **n-i+1** 次。

$$ASL_{成功} = \sum_{i=1}^{n} P_i \cdot (n - i + 1)$$

等概率时（$P_i = \frac{1}{n}$）：

$$ASL_{成功} = \frac{1}{n}\sum_{i=1}^{n}(n-i+1) = \frac{n+1}{2}$$

$$ASL_{失败} = n + 1$$

> 在`elem[0]`存放待查关键字，可省去每次循环的越界检查`i>=1`，减少比较次数。

## 二、折半查找（Binary Search）

### 2.1 前提条件

- 表必须**有序**
- 表必须支持**随机存取**（顺序存储）

### 2.2 算法实现

```c
int Binary_Search(SSTable ST, KeyType key) {
    int low = 1, high = ST.length, mid;
    while (low <= high) {
        mid = (low + high) / 2;
        if (ST.elem[mid] == key)
            return mid;
        else if (ST.elem[mid] < key)
            low = mid + 1;
        else
            high = mid - 1;
    }
    return 0;  // 查找失败
}
```

### 2.3 判定树构造

对有序表`a[1..n]`，构造方法：
1. **根节点** = `mid = ⌊(1+n)/2⌋`
2. 左子树对应`a[1..mid-1]`，右子树对应`a[mid+1..n]`
3. 递归构造

> 对`a[1..11]`，根为`a[6]`，左子树根`a[3]`，右子树根`a[9]`...
>
> ```
>           6
>        /     \
>       3        9
>      / \      / \
>     1    4    7   10
>      \    \    \    \
>       2    5    8   11
> ```

### 2.4 ASL计算

n个结点的判定树深度 $h = \lceil \log_2(n+1) \rceil$

$$ASL_{成功} = \frac{1}{n}\sum_{i=1}^{h} i \cdot 2^{i-1} = \frac{n+1}{n}\log_2(n+1) - 1 \approx \log_2(n+1) - 1$$

**ASL失败**：外部结点（查找失败到达的位置）的层数之和 / 外部结点个数。

## 三、分块查找（Block Search）

### 3.1 思想

将表分为若干**块**，块内无序，但块间有序（第i块最大值 < 第i+1块最小值）。建立**索引表**记录每块最大值和起始地址。

### 3.2 查找过程

1. 在索引表中**顺序/折半查找**确定所在块
2. 在块内**顺序查找**

### 3.3 ASL分析

设n个元素分为b块，每块s个元素：

$$ASL = L_{索引} + L_{块内}$$

- 索引顺序查找 + 块内顺序查找：

$$ASL = \frac{b+1}{2} + \frac{s+1}{2} = \frac{s^2 + 2s + n}{2s}$$

当 $s = \sqrt{n}$ 时，$ASL_{最优} = \sqrt{n} + 1$

## 四、二叉排序树（BST）

### 4.1 定义

二叉排序树（Binary Search Tree）：左子树所有结点值 **<** 根结点值 **<** 右子树所有结点值，且左右子树也是BST。

### 4.2 查找

```c
BSTNode* BST_Search(BiTree T, KeyType key) {
    while (T != NULL && T->data != key) {
        if (key < T->data)
            T = T->lchild;
        else
            T = T->rchild;
    }
    return T;
}
```

### 4.3 插入

```c
int BST_Insert(BiTree &T, KeyType key) {
    if (T == NULL) {
        T = (BSTNode*)malloc(sizeof(BSTNode));
        T->data = key;
        T->lchild = T->rchild = NULL;
        return 1;
    }
    if (key == T->data)
        return 0;  // 已存在，插入失败
    else if (key < T->data)
        return BST_Insert(T->lchild, key);
    else
        return BST_Insert(T->rchild, key);
}
```


### 4.4 删除

| 情况 | 操作 |
|------|------|
| 被删结点是**叶子** | 直接删除 |
| 被删结点只有**左子树或右子树** | 用子树替代 |
| 被删结点有**左右子树** | 用**中序前驱**（左子树最大）或**中序后继**（右子树最小）替代，再删除该前驱/后继 |

### 4.5 复杂度

| | 最好 | 最坏 | 平均 |
|--|------|------|------|
| 查找 | O(log n) | O(n)（退化为链表） | O(log n) |
| 插入 | O(log n) | O(n) | O(log n) |

## 五、平衡二叉树（AVL树）

### 5.1 定义

任意结点的**平衡因子** = 左子树高度 − 右子树高度 ∈ {-1, 0, 1}

### 5.2 四种旋转

插入后找到**最小不平衡子树**的根A，分四种情况：

#### LL型（右旋）

在A的**左孩子B**的**左子树**上插入。

```
  旋转前：        旋转后：
      A              B
     /              / \
    B        →     C   A
   /                    
  C                    
```

```c
// 右旋
B = A->lchild;
A->lchild = B->rchild;
B->rchild = A;
// 更新高度，B成为新根
```

#### RR型（左旋）

在A的**右孩子B**的**右子树**上插入。

```
  旋转前：        旋转后：
    A                B
     \              / \
      B      →     A   C
       \
        C
```

```c
// 左旋
B = A->rchild;
A->rchild = B->lchild;
B->lchild = A;
```

#### LR型（先左旋再右旋）

在A的**左孩子B**的**右子树**上插入，新结点为C。

```
  旋转前：          先对B左旋：       再对A右旋：
      A                 A                  C
     /                 /                  / \
    B        →        C          →      B   A
     \               /
      C             B
```

```c
// 先对B左旋，再对A右旋
A->lchild = LeftRotate(A->lchild);
return RightRotate(A);
```

#### RL型（先右旋再左旋）

在A的**右孩子B**的**左子树**上插入，新结点为C。

```
  旋转前：          先对B右旋：       再对A左旋：
    A                  A                  C
     \                  \                / \
      B        →         C      →      A   B
     /                    \
    C                      B
```

```c
// 先对B右旋，再对A左旋
A->rchild = RightRotate(A->rchild);
return LeftRotate(A);
```

> - **LL右旋，RR左旋**
> - **LR先左后右，RL先右后左**
> - 哪边长往哪边旋

### 5.3 复杂度

AVL树保证树高 $h = O(\log n)$，查找/插入/删除均为 **O(log n)**。

## 六、红黑树（Red-Black Tree）

### 6.1 五大性质

> 1. 每个结点是**红色**或**黑色**
> 2. **根结点**是**黑色**
> 3. **叶子结点**（NIL）是**黑色**
> 4. 红色结点的**两个子结点**都是**黑色**（不存在连续红结点）
> 5. 从任一结点到其所有后代叶子的路径上，**黑色结点数相同**

### 6.2 关键结论

- 从根到叶子的最长路径 ≤ 2 × 最短路径
- n个结点的红黑树高度 $h \leq 2\log_2(n+1)$
- 查找/插入/删除时间复杂度均为 **O(log n)**

---

## 七、B树（B-tree）

### 7.1 m阶B树定义

> 1. 每个结点**最多**有m棵子树、m-1个关键字
> 2. 每个**非根**内部结点**至少**有⌈m/2⌉棵子树、⌈m/2⌉-1个关键字
> 3. **根结点**至少有**2**棵子树（若非叶子）、至少1个关键字
> 4. 所有**叶子结点**在同一层（绝对平衡）
> 5. 结点结构：`n, P₀, K₁, P₁, K₂, P₂, ..., Kₙ, Pₙ`
>    - K₁ < K₂ < ... < Kₙ
>    - Pᵢ所指子树中关键字均在(Kᵢ, Kᵢ₊₁)之间

### 7.2 插入与分裂

插入总是在**叶子层**进行：
- 若插入后关键字数 ≤ m-1，直接插入
- 若插入后关键字数 = m（溢出），则**分裂**：
  1. 取中间关键字 `K⌈m/2⌉` 上提到父结点
  2. 左半部分形成一个结点，右半部分形成一个结点
  3. 若父结点也溢出，递归分裂

### 7.3 B树的高度

$$\lceil \log_m(n+1) \rceil \leq h \leq \lceil \log_{\lceil m/2 \rceil} \frac{n+1}{2} \rceil + 1$$

---

## 八、B+树

### 8.1 定义

- **非叶子结点**仅起索引作用，不存储数据指针
- 所有**关键字**都出现在**叶子结点**中，叶子结点按关键字有序链接成**链表**
- 非叶子结点的每个关键字是其子树中最大（或最小）关键字的副本

### 8.2 B+树 vs B树 对比

| 特性 | B树 | B+树 |
|------|-----|------|
| 关键字与数据 | 每个关键字对应一个数据指针 | 只有叶子存数据指针 |
| 叶子结点链接 | 不链接 | 用指针链接成有序链表 |
| 查找稳定性 | 不稳定（可能在任意层找到） | 稳定（一定查到叶子层） |
| 非叶子结点 | 也存数据 | 仅存索引，可存更多关键字 |
| 范围查询 | 需要中序遍历 | 沿叶子链表遍历即可 |

## 九、散列表（Hash Table）

### 9.1 散列函数构造方法

| 方法 | 公式 | 适用场景 |
|------|------|----------|
| **除留余数法** | H(key) = key % p | 最常用，p取≤m的素数 |
| **直接定址法** | H(key) = a·key + b | 关键字分布连续 |
| **数字分析法** | 取关键字中分布均匀的几位 | 关键字位数多 |
| **平方取中法** | 取key²的中间几位 | 关键字每位分布都不均匀 |

### 9.2 冲突处理

#### 开放定址法

$$H_i = (H(key) + d_i) \% m$$

| 方法 | $d_i$ | 特点 |
|------|-------|------|
| **线性探测** | $d_i = 1, 2, 3, ...$ | 简单，易产生**堆积** |
| **二次探测** | $d_i = 1², -1², 2², -2², ...$ | 减少堆积，但可能有空间遗漏 |

#### 链地址法

将所有关键字为同义词的记录存储在同一个单链表中。

### 9.3 ASL计算

> 关键字序列：{19, 14, 23, 1, 68, 20, 84, 27, 55, 11, 10, 79}
> 散列函数：H(key) = key % 13，用**链地址法**
>
> 计算各位置：
> - H(19)=6, H(14)=1, H(23)=10, H(1)=1, H(68)=3
> - H(20)=7, H(84)=6, H(27)=1, H(55)=3, H(11)=11, H(10)=10, H(79)=1
>
> 散列表：
> ```
> [0] → 空
> [1] → 14 → 1 → 27 → 79
> [2] → 空
> [3] → 68 → 55
> [4] → 空
> [5] → 空
> [6] → 19 → 84
> [7] → 20
> [8] → 空
> [9] → 空
> [10] → 23 → 10
> [11] → 11
> [12] → 空
> ```
>
> 每个关键字的比较次数 = 它在链表中的位置：
> - 14(1次), 1(2次), 27(3次), 79(4次)
> - 68(1次), 55(2次)
> - 19(1次), 84(2次)
> - 20(1次)
> - 23(1次), 10(2次)
> - 11(1次)
>
> $$ASL_{成功} = \frac{1+2+3+4+1+2+1+2+1+1+2+1}{12} = \frac{21}{12} = 1.75$$

> 装填因子 $\alpha = \frac{n}{m}$（n个元素，m个槽位）
>
> - 链地址法：$ASL_{成功} \approx 1 + \frac{\alpha}{2}$
> - 线性探测：$ASL_{成功} \approx \frac{1}{2}(1 + \frac{1}{1-\alpha})$

## 十、KMP算法

### 10.1 核心思想

利用**已匹配的信息**，避免主串指针回溯，只移动模式串指针。

**next数组含义**：`next[j]`表示当模式串第j个字符匹配失败时，应将模式串的第`next[j]`个字符与主串当前字符对齐继续比较。

### 10.2 next数组手动推导

>
> **规则**：
> - `next[1] = 0`（固定）
> - `next[2] = 1`（固定）
> - 对于j ≥ 3：看`P[1..j-1]`的**最长相等前后缀长度**，`next[j]` = 该长度 + 1
>
> | j | P[j] | P[1..j-1] | 最长相等前后缀 | next[j] |
> |---|------|-----------|---------------|---------|
> | 1 | a | （无） | - | **0** |
> | 2 | b | a | 无 | **1** |
> | 3 | a | ab | 无 | **1** |
> | 4 | a | aba | 前缀a=后缀a，长度1 | **2** |
> | 5 | b | abaa | 前缀a=后缀a，长度1 | **2** |
> | 6 | c | abaab | 前缀ab=后缀ab，长度2 | **3** |
> | 7 | a | abaabc | 无 | **1** |
> | 8 | c | abaabca | 前缀a=后缀a，长度1 | **2** |
>
> 结果：`next = {0, 1, 1, 2, 2, 3, 1, 2}`

### 10.3 next数组的计算代码

```c
void GetNext(char *P, int next[]) {
    int j = 1, k = 0;
    next[1] = 0;
    while (j < P[0]) {  // P[0]存串长
        if (k == 0 || P[j] == P[k]) {
            j++; k++;
            next[j] = k;
        } else {
            k = next[k];  // 回溯
        }
    }
}
```

### 10.4 nextval数组（改进的next）

> 在next基础上，若`P[next[j]] == P[j]`，则`nextval[j] = nextval[next[j]]`（继续跳），否则`nextval[j] = next[j]`
>
> 对 "abaabcac"：
>
> | j | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
> |---|---|---|---|---|---|---|---|---|
> | P | a | b | a | a | b | c | a | c |
> | next | 0 | 1 | 1 | 2 | 2 | 3 | 1 | 2 |
> | nextval | **0** | **1** | **0** | **2** | **1** | **3** | **0** | **2** |
>
> 推导过程：
> - j=1: nextval[1]=0（固定）
> - j=2: P[2]=b, P[next[2]]=P[1]=a, b≠a → nextval[2]=next[2]=**1**
> - j=3: P[3]=a, P[next[3]]=P[1]=a, a=a → nextval[3]=nextval[next[3]]=nextval[1]=**0**
> - j=4: P[4]=a, P[next[4]]=P[2]=b, a≠b → nextval[4]=next[4]=**2**
> - j=5: P[5]=b, P[next[5]]=P[2]=b, b=b → nextval[5]=nextval[next[5]]=nextval[2]=**1**
> - j=6: P[6]=c, P[next[6]]=P[3]=a, c≠a → nextval[6]=next[6]=**3**
> - j=7: P[7]=a, P[next[7]]=P[1]=a, a=a → nextval[7]=nextval[next[7]]=nextval[1]=**0**
> - j=8: P[8]=c, P[next[8]]=P[2]=b, c≠b → nextval[8]=next[8]=**2**

### 10.5 KMP匹配代码

```c
int KMP(char *S, char *P, int next[]) {
    int i = 1, j = 1;
    while (i <= S[0] && j <= P[0]) {
        if (j == 0 || S[i] == P[j]) {
            i++; j++;
        } else {
            j = next[j];  // i不回溯！
        }
    }
    if (j > P[0])
        return i - P[0];  // 匹配成功，返回位置
    else
        return 0;
}
```

> KMP算法：**O(n+m)**，其中n为主串长度，m为模式串长度。
> 暴力匹配：**O(n·m)**。

| 查找方法 | 平均时间 | 适用结构 | 是否有序要求 |
|----------|---------|----------|-------------|

| B树 | O(log n) | 多路平衡 | 有序 |
| 散列表 | O(1)（平均） | 散列表 | 否 |

## 红黑树的定义

红黑树是二叉排序树

左子树结点值 ≤ 根结点值 ≤ 右子树结点值

与普通BST相比，有什么要求

①每个结点或是红色，或是黑色的  
②根节点是黑色的  
③叶结点（外部结点、NULL结点、失败结点）均是黑色的  
④不存在两个相邻的红结点（即红结点的父节点和孩子结点均是黑色）  
⑤对每个结点，从该节点到任一叶结点的简单路径上，所含黑结点的数目相同

左根右，根叶黑不红红，黑路同

张口就是freestyle

```txt
struct RBnode { // 红黑树的结点定义

    RBnode* rChild; // 右孩子指针
    int color; // 结点颜色，如：可用 0/1 表示 黑/红，也可使用枚举型enum表示颜色
};
```

红黑树是二叉排序树

左子树结点值 ≤ 根结点值 ≤ 右子树结点值

与普通BST相比，有什么要求

①每个结点或是红色，或是黑色的

②根节点是黑色的  
③叶结点（外部结点、NULL结点、失败结点）均是黑色的  
④不存在两个相邻的红结点（即红结点的父节点和孩子结点均是黑色）  
⑤对每个结点，从该节点到任一叶结点的简单路径上，所含黑结点的数目相同

左根右，根叶黑
不红红，黑路同

张口就是freestyle

性质1：从根节点到叶结点的最长路径不大于最短路径的2倍

性质2：有n个内部节点的红黑树高度 $h \leq 2\log_{2}(n + 1)$

→ 红黑树查找操作时间复杂度 = $O(\log_{2}n)$
