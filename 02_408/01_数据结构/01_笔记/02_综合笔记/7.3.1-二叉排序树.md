# 7.3.1 二叉排序树（BST）

## 一、定义

**二叉排序树**（Binary Search Tree, BST），又称二叉查找树。

一棵二叉树或者是空二叉树，或者是具有如下性质的二叉树：
1. 左子树上所有结点的关键字均**小于**根结点的关键字
2. 右子树上所有结点的关键字均**大于**根结点的关键字
3. 左子树和右子树又各是一棵二叉排序树

> **核心性质**：`左子树结点值 < 根结点值 < 右子树结点值`
> 
> **中序遍历** BST 可得到一个**递增的有序序列**。

---

## 二、查找操作

### 算法思想
若树非空，目标值与根结点的值比较：
- 若相等 → 查找成功
- 若小于根结点 → 在**左子树**上查找
- 若大于根结点 → 在**右子树**上查找
- 查找成功返回结点指针，查找失败返回 NULL

### 递归实现

```c
BSTNode *BST_Search(BiTree T, KeyType key) {
    if (T == NULL) return NULL;
    if (key == T->key) return T;
    else if (key < T->key)
        return BST_Search(T->lchild, key);
    else
        return BST_Search(T->rchild, key);
}
```

### 非递归实现

```c
BSTNode *BST_Search(BiTree T, KeyType key) {
    while (T != NULL && key != T->key) {
        if (key < T->key) T = T->lchild;
        else T = T->rchild;
    }
    return T;
}
```

| 实现方式 | 空间复杂度 |
|----------|-----------|
| 递归 | $O(h)$，h 为树高 |
| 非递归 | $O(1)$ |

## 三、插入操作

### 算法思想
若原 BST 为空，则直接插入结点；否则：
- 若关键字 k **小于**根结点值 → 插入到**左子树**
- 若关键字 k **大于**根结点值 → 插入到**右子树**

> **关键性质**：新插入的结点一定是**叶子结点**。

### 实现

```c
int BST_Insert(BiTree &T, KeyType key) {
    if (T == NULL) {
        T = (BiTree)malloc(sizeof(BSTNode));
        T->key = key;
        T->lchild = T->rchild = NULL;
        return 1;
    }
    else if (key == T->key)
        return 0;  // 已存在，插入失败
    else if (key < T->key)
        return BST_Insert(T->lchild, key);
    else
        return BST_Insert(T->rchild, key);
}
```

---

## 四、构造 BST

### 示例
按照序列 `{50, 26, 21, 30, 66, 60, 70, 68}` 构造 BST：

```
步骤1: 50          步骤2: 50          步骤3: 50
                     /               / \
                    26              26  66
                                  / \
                                21  30
```

逐个插入，最终得到完整的 BST。

> **注意**：不同的关键字序列可能得到**相同的 BST**。

---

## 五、删除操作

### 删除规则
1. **叶子结点**：直接删除
2. **只有左子树或只有右子树**：用子树替换被删结点
3. **既有左子树又有右子树**：
   - 找到**中序遍历的前驱**（左子树中最大的）或**后继**（右子树中最小的）
   - 用前驱/后继的值替换被删结点
   - 然后删除前驱/后继结点

> **关键性质**：删除操作不会改变 BST 的中序有序性。

---

## 六、查找效率分析

### 最好情况
BST 是**平衡的**（完全二叉树），树高 $h = \lceil \log_2(n+1) \rceil$：
$$ASL = O(\log_2 n)$$

### 最坏情况
BST 退化为**单支树**（如插入有序序列），树高 $h = n$：
$$ASL = O(n)$$

---

## 七、考点总结

| 考点 | 内容 |
|------|------|
| BST 性质 | 左 < 根 < 右，中序遍历有序 |
| 查找实现 | 递归/非递归，时间复杂度 $O(h)$ |
| 插入特性 | 新结点一定是叶子结点 |
| 删除操作 | 三种情况的处理 |
| 查找效率 | 最好 $O(\log n)$，最坏 $O(n)$ |

> 1. BST 的查找效率取决于**树的高度**
> 2. 插入有序序列会导致 BST 退化为单支树
> 3. 删除非叶子结点时，要保持 BST 的有序性
