# 外部排序

> [[02_408/01_数据结构/01_笔记/02_综合笔记/06-排序]] | [[01_数学一/01_高等数学/01_笔记/09_数一笔记_高数/索引]]

## 概念

**外部排序**：当数据量太大无法全部装入内存时，需要借助外存（磁盘）进行排序的方法。

### 基本思想

```
① 将外存数据分成若干块
② 每块读入内存，用内部排序排序
③ 写回外存，形成有序归并段
④ 多路归并，形成最终有序文件
```

## 置换-选择排序

### 目的

生成**更长的初始归并段**，减少归并次数。

### 思想

利用一个内存工作区（如大小为 w 的堆），从输入文件依次选择最小元素输出，同时从输入文件读入新元素补充到堆中。

### 步骤

```
① 建立初始堆
② 重复：
   - 输出堆顶元素 rmin
   - 从输入读入新元素 x
   - 若 x >= rmin，替换堆顶，筛选调整
   - 若 x < rmin，标记 rmin 为当前段结束，用 x 替换堆顶
③ 重复直到所有元素处理完
```

### 优点

- 平均归并段长度 = 2 × 工作区大小
- 比简单分块产生的归并段更长

## 最佳归并树

### 概念

**最佳归并树**：将多个有序归并段进行多路归并时，使**总的 I/O 次数（带权路径长度）最小**的归并方案。

### 构建方法

类似于 **Huffman 树**的构建：
- 每个归并段的长度作为权值
- 归并次数作为带权路径长度

### 关键点

#### 1. 多路归并

**m 路归并** = m 个有序段合并为 1 个有序段

#### 2. 虚段处理

- 假设初始有 n 个归并段
- m 路归并时，m 叉树需要 (n-1) mod (m-1) == 0
- 若不满足，需要补充**虚段**（权值为 0）

#### 3. 计算总 I/O 次数

```
总 I/O 次数 = 2 × 读盘 + 写盘
            = 2 × Σ(归并段长度 × 归并轮数)
```

### 示例

假设有 5 个归并段，长度分别为：1, 3, 5, 7, 9
- 三路归并（m=3）
- (5-1) mod (3-1) = 0 ✓ 不需要补虚段

构建 Huffman 树（这里是最小带权路径长度树）：

```
        25
       /  \
      9    16
          /  \
         7    9
            / \
           3   5 (+1)
```

带权路径长度 = (3+5+1) × 2 + 7 × 2 + 9 × 1 = 18 + 14 + 9 = 41

## 多路平衡归并

### 败者树

**败者树**：用于多路归并的高效数据结构，将多路比较的复杂度从 O(m) 降到 O(log m)。

### 步骤

```
① 构建败者树：m 个归并段中选最小者
② 输出最小元素
③ 从该段读入下一个元素
④ 调整败者树
⑤ 重复直到所有段读完
```

## 磁盘排序优化

1. **增大归并段**：置换-选择排序
2. **减少归并次数**：增大 k 路归并的 k
3. **多缓冲区并行**：使用多个输入/输出缓冲区
4. **预读和延后写**：利用磁盘局部性

## 公式速记

- **置换-选择排序平均长度**：2w（w 为工作区大小）
- **m 路归并虚段数**：使 (n - 1) mod (m - 1) == 0
- **总 I/O 次数**：2 × 带权路径长度 × 块大小
