## 一、插入排序

### 1.1 直接插入排序

#### 思想
将待排序元素插入到已排序子表的正确位置。

#### C代码实现

```c
void InsertSort(int A[], int n) {
    int i, j, temp;
    for (i = 2; i <= n; i++) {  // A[1]已有序，从A[2]开始
        if (A[i] < A[i-1]) {
            temp = A[i];  // 哨兵
            for (j = i - 1; j >= 1 && A[j] > temp; j--)
                A[j+1] = A[j];  // 后移
            A[j+1] = temp;  // 插入
        }
    }
}
```

#### 图解过程

```
初始: [49] 38 65 97 76 13 27 49'
i=2:  [38 49] 65 97 76 13 27 49'
i=3:  [38 49 65] 97 76 13 27 49'
i=4:  [38 49 65 97] 76 13 27 49'
i=5:  [38 49 65 76 97] 13 27 49'
i=6:  [13 38 49 65 76 97] 27 49'
i=7:  [13 27 38 49 65 76 97] 49'
i=8:  [13 27 38 49 49' 65 76 97]
```

#### 复杂度

| | 时间 | 空间 |
|--|------|------|
| 最好 | O(n)（已有序） | O(1) |
| 最坏 | O(n²)（逆序） | O(1) |
| 平均 | **O(n²)** | O(1) |

> **稳定性**：✅ **稳定**

### 1.2 折半插入排序

#### 思想
在已排序部分用**折半查找**确定插入位置，减少比较次数，但移动次数不变。

```c
void BinaryInsertSort(int A[], int n) {
    int i, j, low, high, mid, temp;
    for (i = 2; i <= n; i++) {
        temp = A[i];
        low = 1; high = i - 1;
        while (low <= high) {  // 折半查找
            mid = (low + high) / 2;
            if (A[mid] > temp) high = mid - 1;
            else low = mid + 1;
        }
        for (j = i - 1; j >= high + 1; j--)
            A[j+1] = A[j];  // 统一后移
        A[high+1] = temp;
    }
}
```

- 时间：比较次数 O(n log n)，移动次数 O(n²)，总 **O(n²)**
- 空间：O(1)
- **稳定性**：✅ **稳定**

### 1.3 希尔排序（Shell Sort）

#### 思想
将序列按**增量序列**分组，每组内做直接插入排序，逐步缩小增量至1。

```c
void ShellSort(int A[], int n) {
    int i, j, temp, dk;
    for (dk = n/2; dk >= 1; dk /= 2) {  // 增量序列
        for (i = dk+1; i <= n; i++) {
            if (A[i] < A[i-dk]) {
                temp = A[i];
                for (j = i-dk; j >= 1 && A[j] > temp; j -= dk)
                    A[j+dk] = A[j];
                A[j+dk] = temp;
            }
        }
    }
}
```

#### 图解（以dk=4,2,1为例）

```
初始: 49 38 65 97 76 13 27 49' 55 04
dk=4: 分组 [49,76,55] [38,13,04] [65,27] [97,49']
      排序后: 49 13 27 49' 55 04 65 97 76 38
dk=2: 分组 [49,27,55,65,76] [13,49',04,97,38]
      排序后: 27 04 49 38 55 13 65 49' 76 97
dk=1: 直接插入排序
      结果: 04 13 27 38 49 49' 55 65 76 97
```

- 时间：与增量序列有关，一般 **O(n^1.3)** 左右
- 空间：O(1)
- **稳定性**：❌ **不稳定**（相同关键字可能被分到不同组）

## 二、交换排序

### 2.1 冒泡排序（Bubble Sort）

```c
void BubbleSort(int A[], int n) {
    int i, j, temp;
    bool flag;
    for (i = 0; i < n-1; i++) {
        flag = false;
        for (j = n-1; j > i; j--) {  // 从后往前冒泡
            if (A[j-1] > A[j]) {
                temp = A[j-1];
                A[j-1] = A[j];
                A[j] = temp;
                flag = true;
            }
        }
        if (!flag) return;  // 本趟无交换，已有序
    }
}
```

- 最好：**O(n)**（一趟无交换）
- 最坏：**O(n²)**
- 平均：**O(n²)**
- 空间：O(1)
- **稳定性**：✅ **稳定**

### 2.2 快速排序（Quick Sort）

#### 思想
选一个**枢轴**（pivot），将表划分为两部分：左半 ≤ pivot ≤ 右半，递归排序。

#### C代码实现

```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];  // 取第一个元素为枢轴
    while (low < high) {
        while (low < high && A[high] >= pivot) high--;
        A[low] = A[high];
        while (low < high && A[low] <= pivot) low++;
        A[high] = A[low];
    }
    A[low] = pivot;
    return low;
}

void QuickSort(int A[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(A, low, high);
        QuickSort(A, low, pivotpos - 1);
        QuickSort(A, pivotpos + 1, high);
    }
}
```

#### 图解一趟划分

```
初始: [49] 38 65 97 76 13 27 49'
pivot=49, low→              ←high

  low=49, high右移找<49: 27移到low位
  [27] 38 65 97 76 13 [_] 49'  low右移找>49
  27 38 [65] 97 76 13 [_] 49'  high左移找<49
  27 38 [65] 97 76 [13] _ 49'  13移到high位
  27 38 13 97 76 [_] 65 49'    low右移找>49
  ...
最终: 27 38 13 [49] 76 97 65 49'
```

#### 复杂度

| | 时间 | 说明 |
|--|------|------|
| 最好 | O(n log n) | 每次恰好二分 |
| 最坏 | **O(n²)** | 已有序时退化 |
| 平均 | **O(n log n)** | 内部排序中平均性能最优 |

- 空间：O(log n)（递归栈深度），最坏O(n)
- **稳定性**：❌ **不稳定**

## 三、选择排序

### 3.1 简单选择排序

```c
void SelectSort(int A[], int n) {
    int i, j, min, temp;
    for (i = 0; i < n-1; i++) {
        min = i;
        for (j = i+1; j < n; j++)
            if (A[j] < A[min]) min = j;
        if (min != i) {
            temp = A[i]; A[i] = A[min]; A[min] = temp;
        }
    }
}
```

- 时间：最好/最坏/平均均为 **O(n²)**
- 空间：O(1)
- **稳定性**：❌ **不稳定**（例如 `{5, 5, 2}` → 第一趟交换5和2）


### 3.2 堆排序（Heap Sort）

#### 思想
利用**堆**这种完全二叉树结构，反复取出堆顶（最大/最小元素）。

#### 关键操作

**调整堆（向下筛选）**：

```c
void HeapAdjust(int A[], int k, int len) {
    A[0] = A[k];  // 暂存
    for (int i = 2*k; i <= len; i *= 2) {
        if (i < len && A[i] < A[i+1]) i++;  // 取较大子结点
        if (A[0] >= A[i]) break;
        else {
            A[k] = A[i];
            k = i;
        }
    }
    A[k] = A[0];
}

void BuildMaxHeap(int A[], int len) {
    for (int i = len/2; i >= 1; i--)
        HeapAdjust(A, i, len);
}

void HeapSort(int A[], int len) {
    BuildMaxHeap(A, len);
    for (int i = len; i > 1; i--) {
        int temp = A[1]; A[1] = A[i]; A[i] = temp;  // 堆顶与末尾交换
        HeapAdjust(A, 1, i-1);  // 重新调整
    }
}
```

#### 图解建堆过程

```
初始完全二叉树:         建大根堆过程：
      4                  从最后一个非叶子结点(n/2=5)开始
     / \                  向上逐步调整
    8    5
   / \  / \
  2  3 9   1

调整结点5(子树9,1): 5<9, 交换 → 9上, 5下
调整结点8(子树2,3): 8>3, 8>2, 不变
调整结点4(子树8,9): 4<9, 交换 → 9上, 4下
  继续调整4的子树: 4<5? 不确定...
最终大根堆:
      9
     / \
    8    5
   / \  / \
  2  3 4   1
```

#### 复杂度

- 建堆：**O(n)**
- 每次调整：O(log n)
- 总时间：**O(n log n)**（最好/最坏/平均都是）
- 空间：**O(1)**
- **稳定性**：❌ **不稳定**

## 四、归并排序

### 4.1 二路归并排序

#### 思想
将表不断**二分**到单个元素，再逐步**两两归并**为有序表。

```c
int *B = (int*)malloc((n+1)*sizeof(int));  // 辅助数组

void Merge(int A[], int low, int mid, int high) {
    for (int k = low; k <= high; k++)
        B[k] = A[k];
    int i = low, j = mid+1, k = low;
    while (i <= mid && j <= high) {
        if (B[i] <= B[j])
            A[k++] = B[i++];
        else
            A[k++] = B[j++];
    }
    while (i <= mid) A[k++] = B[i++];
    while (j <= high) A[k++] = B[j++];
}

void MergeSort(int A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(A, low, mid);
        MergeSort(A, mid+1, high);
        Merge(A, low, mid, high);
    }
}
```

#### 图解

```
       [49 38 65 97 76 13 27 49']
      /                        \
  [49 38 65 97]          [76 13 27 49']
   /        \              /        \
[49 38]   [65 97]      [76 13]   [27 49']
 / \       / \          / \       / \
[49][38] [65][97]    [76][13]  [27][49']
 \ /       \ /          \ /       \ /
[38 49]   [65 97]      [13 76]   [27 49']
   \        /              \        /
  [38 49 65 97]          [13 27 49' 76]
      \                        /
  [13 27 38 49 49' 65 76 97]
```

- 时间：最好/最坏/平均均为 **O(n log n)**
- 空间：**O(n)**（辅助数组）
- **稳定性**：✅ **稳定**

## 五、基数排序

### 5.1 思想

按关键字各位**从低位到高位**（LSD）依次分配和收集。

### 5.2 过程图解

对 `{278, 109, 063, 930, 589, 184, 505, 269, 008, 083}`：

```
按个位分配→收集: 930 063 083 184 505 278 008 109 589 269
按十位分配→收集: 505 008 109 930 063 269 278 083 184 589
按百位分配→收集: 008 063 083 109 184 269 278 505 589 930
```

### 5.3 复杂度

设关键字有d位，基数为r（通常r=10）：

- 时间：**O(d(n+r))**
- 空间：**O(r)**（r个队列）
- **稳定性**：✅ **稳定**

## 六、外部排序

### 6.1 基本概念

当数据量太大无法全部装入内存时，需要**外部排序**。核心思想是**多路归并**。

### 6.2 多路归并

将内存分为k个输入缓冲区和1个输出缓冲区，做**k路归并**。

- **败者树**：加速k路归并中选出最小/最大关键字的过程，时间从O(k)降到O(log k)

### 6.3 置换-选择排序

用于生成更长的**初始归并段**，使归并段长度大于内存容量。

### 6.4 最佳归并树

用**哈夫曼树**思想构造，使总读写磁盘次数最少。

- 对于k路归并，若初始归并段数m不满足 $(m-1) \% (k-1) = 0$，需补 $(k-1) - (m-1)\%(k-1)$ 个长度为0的虚段

## 七、完整对比总结表

| 算法 | 最好 | 最坏 | 平均 | 空间 | 稳定性 |
|------|------|------|------|------|--------|
| **直接插入** | O(n) | O(n²) | O(n²) | O(1) | ✅ 稳定 |
| **折半插入** | O(n log n) | O(n²) | O(n²) | O(1) | ✅ 稳定 |
| **冒泡排序** | O(n) | O(n²) | O(n²) | O(1) | ✅ 稳定 |
| **简单选择** | O(n²) | O(n²) | O(n²) | O(1) | ❌ 不稳定 |
| **希尔排序** | - | - | O(n^1.3) | O(1) | ❌ 不稳定 |
| **快速排序** | O(n log n) | O(n²) | O(n log n) | O(log n) | ❌ 不稳定 |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ 稳定 |
| **基数排序** | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(r) | ✅ 稳定 |

> - **O(n log n)且稳定**：归并排序（唯一的）
> - **O(n log n)且O(1)空间**：堆排序（唯一的）
> - **平均性能最优**：快速排序
> - **比较次数与初始状态无关**：简单选择排序、归并排序
> - **初始有序时最快**：直接插入O(n)、冒泡O(n)
> - **初始有序时最慢**：快速排序O(n²)

## 八、排序算法选择指南

```
数据量小(n ≤ 50) → 直接插入排序
数据基本有序 → 直接插入/冒泡
要求稳定 → 归并排序
要求O(1)空间 → 堆排序
一般情况(内部排序) → 快速排序
关键字位数少且范围小 → 基数排序
数据量大(外部排序) → 多路归并排序
```

## 知识回顾与重要考点

败者树解决的问题：使用多路平衡归并可减少归并趟数，但是用老土方法从 k 个归并段选出一个最小/最大元素需要对比关键字 k-1 次，构造败者树可以使关键字对比次数减少到 $\lceil \log_2 k \rceil$

败者树可视为一棵完全二叉树（多了一个头头）。k个叶结点分别对应 k 个归并段中当前参加比较的元素，非叶子结点用来记忆左右子树中的“失败者”，而让胜者往上继续进行比较，一直到根结点。
