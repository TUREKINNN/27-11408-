# 双链表

## 📊 考情分析
- **考试频率**：中频考点
- **分值**：选择题2-4分，算法题可结合考查
- **题型**：概念辨析、代码实现、与单链表的对比
- **真题考查方式**：双链表的插入删除代码、循环双链表的特性、指针修改顺序

## 一、基本概念

### 1.1 双链表的定义

**双链表**——在单链表的基础上，每个结点增加一个指向**前驱结点**的指针。

### 1.2 结点结构

```
┌────────┬────────┬────────┐
│ prior  │  data  │  next  │
│ 前驱指针│ 数据域 │ 后继指针│
└────────┴────────┴────────┘
```

### 1.3 单链表 vs 双链表

| 对比项 | 单链表 | 双链表 |
|--------|--------|--------|
| 指针域 | 只有 next | 有 prior 和 next |
| 方向 | 只能向后 | 可进可退 |
| 存储密度 | 更高 | 略低 |
| 空间开销 | 小 | 大（多一个指针） |

## 二、核心知识

### 2.1 双链表的定义

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior;  // 前驱指针
    struct DNode *next;   // 后继指针
} DNode, *DLinkList;
```

### 2.2 双链表的初始化

```c
// 带头结点的双链表初始化
bool InitDLinkList(DLinkList &L) {
    L = (DNode *)malloc(sizeof(DNode));
    if (L == NULL)
        return false;
    L->prior = NULL;  // 头结点的prior永远指向NULL
    L->next = NULL;   // 头结点后暂时没有结点
    return true;
}
```

**空表**：`L->next == NULL && L->prior == NULL`

### 2.3 双链表的插入

```c
// 在结点p之后插入结点s
bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL)
        return false;
    
    s->next = p->next;    // ① s指向p的后继
    if (p->next != NULL)   // 如果p不是最后一个结点
        p->next->prior = s; // ② p的后继指向s
    s->prior = p;          // ③ s指向p
    p->next = s;           // ④ p指向s
    
    return true;
}
```

**指针修改顺序分析**：

```
    x     p     s     y
    ←─→   ←─→   ←─→   ←─→
```

步骤①②③④的顺序可以调整，但有些顺序会导致指针丢失：

```
❌ 错误顺序：先执行④①
p->next = s;    // p的后继丢失
s->next = p->next; // s指向了自己
```

> ⚠️ **关键**：修改指针时要注意顺序，避免丢失指针信息。

### 2.4 双链表的删除

```c
// 删除p的后继结点q
bool DeleteNextDNode(DNode *p) {
    if (p == NULL)
        return false;
    
    DNode *q = p->next;  // q指向p的后继
    if (q == NULL)        // p没有后继
        return false;
    
    p->next = q->next;   // ① p指向q的后继
    if (q->next != NULL)  // 如果q不是最后一个结点
        q->next->prior = p; // ② q的后继指向p
    
    free(q);              // 释放q
    return true;
}
```

**特殊情况**：如果p是最后一个结点（q不存在），无法执行删除。

### 2.5 双链表的遍历

```c
// 后向遍历
while (p != NULL) {
    // 对结点p做相应处理
    p = p->next;
}

// 前向遍历
while (p != NULL) {
    // 对结点p做相应处理
    p = p->prior;
}

// 前向遍历（跳过头结点）
while (p->prior != NULL) {
    // 对结点p做相应处理
    p = p->prior;
}
```

**查找操作**：双链表不可随机存取，按位查找、按值查找都只能用遍历方式实现，时间复杂度 $O(n)$。

### 2.6 双链表的销毁

```c
void DestroyList(DLinkList &L) {
    while (L->next != NULL)
        DeleteNextDNode(L);  // 依次删除头结点的后继
    free(L);                  // 释放头结点
    L = NULL;
}
```

## 三、深入理解

### 3.1 双链表的优势

1. **可进可退**：可以方便地找到前驱和后继
2. **删除操作更方便**：不需要从头遍历找前驱
3. **前插操作简单**：直接通过 prior 找到前驱

### 3.2 双链表的劣势

1. **存储密度低**：每个结点多一个指针域
2. **插入删除更复杂**：需要修改更多指针
3. **代码更容易出错**：指针修改顺序需要注意

### 3.3 什么时候用双链表？

- 需要频繁进行**前向遍历**
- 需要频繁进行**前插操作**
- 需要频繁**删除指定结点**

## 五、典型例题

### 概念题

**Q1**：下列关于双链表的说法，正确的是（）
- A. 双链表支持随机访问
- B. 双链表的存储密度高于单链表
- C. 双链表可以在O(1)时间内找到前驱结点
- D. 双链表的插入操作比单链表简单

**答案**：C

### 代码题

**Q2**：写出双链表在结点p之前插入结点s的代码。

```c
bool InsertPriorDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL)
        return false;
    
    s->prior = p->prior;   // s指向p的前驱
    if (p->prior != NULL)
        p->prior->next = s; // p的前驱指向s
    s->next = p;            // s指向p
    p->prior = s;           // p指向s
    
    return true;
}
```

**Q3**：编写一个函数，将双链表中的结点逆置。

```c
void ReverseDLinkList(DLinkList &L) {
    DNode *p = L->next;
    L->next = NULL;
    
    while (p != NULL) {
        DNode *q = p->next;
        p->next = L->next;
        if (L->next != NULL)
            L->next->prior = p;
        L->next = p;
        p->prior = L;
        p = q;
    }
}
```

## 六、易错点

1. **指针修改顺序**：插入时注意不要丢失指针
2. **头结点的prior**：始终指向NULL
3. **空表判断**：`L->next == NULL`（不是 `L->prior == NULL`）
4. **删除时释放内存**：`free(q)` 不能忘记
5. **p是最后一个结点时的处理**：`p->next == NULL` 需要特殊处理
