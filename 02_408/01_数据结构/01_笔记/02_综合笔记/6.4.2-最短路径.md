# 最短路径（Shortest Path）

> - 考试频率：★★★
> - 分值：选择题2-4分，大题8-12分
> - 题型：选择题、综合应用题
> - 重点：Dijkstra算法执行过程、Floyd算法矩阵推导、负权边问题

---

## 一、基本概念

### 1.1 最短路径定义

在带权图 $G=(V,E)$ 中，从顶点 $u$ 到顶点 $v$ 的所有路径中，**边权之和最小**的路径称为从 $u$ 到 $v$ 的**最短路径**，其边权之和记为 $d(u,v)$。

### 1.2 最短路径问题分类

| 问题类型 | 描述 | 对应算法 |
|---------|------|---------|
| **单源最短路径** | 从一个源点到所有其他顶点的最短路径 | Dijkstra、Bellman-Ford |
| **多源最短路径（全源最短路径）** | 所有顶点对之间的最短路径 | Floyd |

### 1.3 重要概念

- **松弛操作（Relaxation）**：对于边 $(u,v)$，若 $d[u] + w(u,v) < d[v]$，则更新 $d[v] = d[u] + w(u,v)$
- **松弛的本质**：发现更短的路径时，更新最短距离估计值
- **三角不等式**：$d[u,v] \leq d[u,x] + d[x,v]$

## 二、算法详解

### 2.1 Dijkstra 算法（迪杰斯特拉算法）

#### 2.1.1 基本思想（贪心策略）

Dijkstra 算法求解**单源最短路径**问题，基于贪心思想：

1. 维护两个集合：已确定最短路径的顶点集 $S$，未确定的顶点集 $V-S$
2. 每次从 $V-S$ 中选择**距离源点最近**的顶点 $u$，加入 $S$
3. 对 $u$ 的所有邻接顶点执行**松弛操作**
4. 重复直到 $S$ 包含所有顶点

**核心思想：** 每次贪心地选择当前距离源点最近的未处理顶点，该距离即为最终最短距离。

#### 2.1.2 算法步骤（详细）

设源点为 $s$，$dist[i]$ 表示 $s$ 到 $i$ 的当前最短距离估计，$path[i]$ 记录最短路径上的前驱：

```
初始化：
  dist[s] = 0
  dist[i] = w(s, i)  （若 s 到 i 有边），否则 ∞
  path[i] = s（若有边），否则 -1
  S = {s}

重复 n-1 次：
  1. 从 V-S 中选出 dist 值最小的顶点 u
  2. 将 u 加入 S
  3. 对 u 的每个邻接顶点 v ∈ V-S：
     若 dist[u] + w(u,v) < dist[v]，则
       dist[v] = dist[u] + w(u,v)
       path[v] = u
```

#### 2.1.3 负权边问题

> 
> **原因：** Dijkstra 基于贪心策略，一旦顶点加入 $S$ 集合就认为其最短距离已确定。但存在负权边时，后续可能发现更短的路径。
>
> **反例：** A→B 权值为 2，A→C 权值为 3，C→B 权值为 -2。
> - Dijkstra 先将 B 加入 S（dist=2），但实际上 A→C→B = 3+(-2) = 1 < 2
> - 已加入 S 的顶点距离被错误地确定了
>
> **处理负权边的算法：** Bellman-Ford 算法或 SPFA 算法

#### 2.1.4 时间复杂度分析

| 实现方式 | 时间复杂度 | 说明 |
|---------|-----------|------|
| 数组（朴素） | $O(n^2)$ | 每次线性扫描找最小值 |
| 最小堆（二叉堆） | $O((n+m)\log n)$ | 适合稀疏图 |
| Fibonacci 堆 | $O(n\log n + m)$ | 理论最优 |

**考试重点：** $O(n^2)$ 的朴素实现

#### 2.1.5 适用场景

- 边权**非负**的带权有向图或无向图
- 单源最短路径问题
- 稠密图用数组实现，稀疏图用堆优化

### 2.2 Floyd 算法（弗洛伊德算法）

#### 2.2.1 基本思想（动态规划）

Floyd 算法求解**全源最短路径**问题，基于动态规划：

**状态定义：** $d^{(k)}[i][j]$ 表示从 $i$ 到 $j$、中间顶点编号不超过 $k$ 的最短路径长度。

**状态转移方程：**

$$d^{(k)}[i][j] = \min\begin{cases} d^{(k-1)}[i][j] & \text{（不经过顶点 } k\text{）} \\ d^{(k-1)}[i][k] + d^{(k-1)}[k][j] & \text{（经过顶点 } k\text{）} \end{cases}$$

**核心思想：** 依次考虑每个顶点作为中间顶点，逐步优化所有顶点对之间的最短路径。

#### 2.2.2 算法步骤（详细）

```
初始化：
  D[i][j] = w(i,j)  （若 i,j 之间有边）
  D[i][j] = ∞       （若 i,j 之间无边）
  D[i][i] = 0

对于 k = 0, 1, ..., n-1：       // 中间顶点
  对于 i = 0, 1, ..., n-1：     // 起点
    对于 j = 0, 1, ..., n-1：   // 终点
      若 D[i][k] + D[k][j] < D[i][j]：
        D[i][j] = D[i][k] + D[k][j]
```

#### 2.2.3 关键要点

1. **k 必须在最外层循环**：保证逐步考虑每个中间顶点
2. **可以处理负权边**：但不能有负权回路（负权环）
3. **三重循环的顺序**：k-i-j 或 k-j-i 均可，但 k 必须在最外层
4. **路径记录**：用 $path[i][j]$ 记录中间顶点，递归输出完整路径

#### 2.2.4 时间复杂度分析

| 指标 | 复杂度 |
|------|--------|
| 时间复杂度 | $O(n^3)$ |
| 空间复杂度 | $O(n^2)$ |

#### 2.2.5 适用场景

- 顶点数较少（$n \leq 200$ 左右）
- 需要求所有顶点对之间的最短路径
- 可以处理**负权边**（但不能有负权回路）

---

### 2.3 两种算法对比

| 对比项目 | Dijkstra 算法 | Floyd 算法 |
|---------|--------------|-----------|
| **问题类型** | 单源最短路径 | 全源最短路径 |
| **算法思想** | 贪心 | 动态规划 |
| **时间复杂度** | $O(n^2)$ | $O(n^3)$ |
| **空间复杂度** | $O(n)$ | $O(n^2)$ |
| **负权边** | ❌ 不可以处理 | ✅ 可以处理 |
| **负权回路** | ❌ | ❌ 不可以处理 |
| **适用图规模** | 大规模图 | 小规模图 |
| **存储结构** | 邻接矩阵或邻接表 | 邻接矩阵 |
| **代码复杂度** | 中等 | 简单（三重循环） |
| **能否输出路径** | 可以（path数组） | 可以（path矩阵） |

> - **Dijkstra**：贪心，$O(n^2)$，单源，**不能**有负权边
> - **Floyd**：DP，$O(n^3)$，全源，**可以**有负权边
> - 两者都**不能**处理负权回路

## 三、代码实现

### 3.1 Dijkstra 算法（邻接矩阵 + 数组实现）

```c
#include <stdio.h>

#define MAXVEX 100
#define INF 65535

typedef struct {
    int arc[MAXVEX][MAXVEX];
    int numVertexes;
} MGraph;

/**
 * Dijkstra算法：单源最短路径
 * @param G    图（邻接矩阵）
 * @param src  源点下标
 * @param dist dist[i]存储src到i的最短距离
 * @param path path[i]存储最短路径上i的前驱顶点
 */
void Dijkstra(MGraph G, int src, int dist[], int path[]) {
    int visited[MAXVEX];  // 标记是否已加入S集合
    
    // 初始化
    for (int i = 0; i < G.numVertexes; i++) {
        visited[i] = 0;
        dist[i] = G.arc[src][i];
        if (dist[i] < INF && i != src)
            path[i] = src;
        else
            path[i] = -1;
    }
    
    visited[src] = 1;  // 源点加入S
    dist[src] = 0;
    path[src] = -1;
    
    // 循环 n-1 次
    for (int i = 1; i < G.numVertexes; i++) {
        // 步骤1：在V-S中找dist最小的顶点
        int min = INF;
        int u = -1;
        for (int j = 0; j < G.numVertexes; j++) {
            if (!visited[j] && dist[j] < min) {
                min = dist[j];
                u = j;
            }
        }
        
        if (u == -1) break;  // 剩余顶点不可达
        
        visited[u] = 1;  // 步骤2：u加入S
        
        // 步骤3：松弛操作
        for (int j = 0; j < G.numVertexes; j++) {
            if (!visited[j] && 
                dist[u] + G.arc[u][j] < dist[j]) {
                dist[j] = dist[u] + G.arc[u][j];
                path[j] = u;
            }
        }
    }
}

/**
 * 打印从src到dest的最短路径
 */
void PrintPath(int path[], int src, int dest) {
    if (dest == src) {
        printf("%d ", src);
        return;
    }
    if (path[dest] == -1) {
        printf("不可达");
        return;
    }
    PrintPath(path, src, path[dest]);
    printf("-> %d ", dest);
}
```

### 3.2 Floyd 算法（邻接矩阵实现）

```c
#include <stdio.h>

#define MAXVEX 100
#define INF 65535

/**
 * Floyd算法：全源最短路径
 * @param G    图（邻接矩阵）
 * @param dist dist[i][j]存储i到j的最短距离
 * @param path path[i][j]存储i到j最短路径的中间顶点
 */
void Floyd(MGraph G, int dist[][MAXVEX], int path[][MAXVEX]) {
    int n = G.numVertexes;
    
    // 初始化距离矩阵和路径矩阵
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            dist[i][j] = G.arc[i][j];
            path[i][j] = -1;  // -1表示直达，无中间顶点
        }
    }
    
    // 三重循环：k-i-j
    for (int k = 0; k < n; k++) {          // 中间顶点（最外层！）
        for (int i = 0; i < n; i++) {      // 起点
            for (int j = 0; j < n; j++) {  // 终点
                // 经过k可能更短
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                    path[i][j] = k;  // 记录中间顶点
                }
            }
        }
    }
}

/**
 * 递归输出i到j的最短路径（经过的顶点）
 */
void PrintFloydPath(int path[][MAXVEX], int i, int j) {
    if (path[i][j] == -1) {
        // 直达
        printf("%d -> %d\n", i, j);
        return;
    }
    int k = path[i][j];
    PrintFloydPath(path, i, k);  // 递归输出 i -> k
    PrintFloydPath(path, k, j);  // 递归输出 k -> j
}

/**
 * 检测图中是否存在负权回路
 * 若 dist[i][i] < 0，则存在从i出发回到i的负权回路
 */
int HasNegativeCycle(MGraph G, int dist[][MAXVEX]) {
    for (int i = 0; i < G.numVertexes; i++) {
        if (dist[i][i] < 0)
            return 1;  // 存在负权回路
    }
    return 0;
}
```

## 四、典型例题

### 例题 1：Dijkstra 算法执行过程（选择题）

**题目：** 如下有向图，以顶点 0 为源点，用 Dijkstra 算法求最短路径。

```
0 --1--> 1 --2--> 3
|        |
2        4
|        |
v        v
2 --3--> 4 --1--> 3
```

邻接矩阵：
```
     0    1    2    3    4
0  [ 0,   1,   2,   ∞,   ∞]
1  [ ∞,   0,   ∞,   2,   4]
2  [ ∞,   ∞,   0,   ∞,   3]
3  [ ∞,   ∞,   ∞,   0,   ∞]
4  [ ∞,   ∞,   ∞,   1,   0]
```

**解析：**

初始化：dist = [0, 1, 2, ∞, ∞]，visited = {0}

| 步骤 | 选中顶点u | dist[u] | 松弛更新 | dist 数组 |
|------|----------|---------|---------|----------|
| 初始 | 0(源) | 0 | — | [0, **1**, **2**, ∞, ∞] |
| 1 | 1 | 1 | d[3]=1+2=3, d[4]=1+4=5 | [0, 1, 2, **3**, **5**] |
| 2 | 2 | 2 | d[4]=min(5,2+3)=5 | [0, 1, 2, 3, 5] |
| 3 | 3 | 3 | 无邻接 | [0, 1, 2, 3, 5] |
| 4 | 4 | 5 | d[3]=min(3,5+1)=3(不变) | [0, 1, 2, 3, 5] |

**结果：**
- 0→1 最短距离 = 1，路径 0→1
- 0→2 最短距离 = 2，路径 0→2
- 0→3 最短距离 = 3，路径 0→1→3
- 0→4 最短距离 = 5，路径 0→1→4

### 例题 2：Floyd 算法执行过程（选择题）

**题目：** 设有向图的邻接矩阵为：

```
D^(0) =
     0    1    2
0  [ 0,   4,  11]
1  [ 6,   0,   2]
2  [ 3,   ∞,   0]
```

用 Floyd 算法求所有顶点对的最短路径。

**解析：**

**k=0（经过顶点 0）：**
- D[1][0]+D[0][2] = 6+11 = 17 > D[1][2]=2，不更新
- D[2][0]+D[0][1] = 3+4 = 7 < D[2][1]=∞，更新 D[2][1]=7

```
D^(1) =
     0    1    2
0  [ 0,   4,  11]
1  [ 6,   0,   2]
2  [ 3,   7,   0]
```

**k=1（经过顶点 1）：**
- D[0][1]+D[1][2] = 4+2 = 6 < D[0][2]=11，更新 D[0][2]=6
- D[2][1]+D[1][0] = 7+6 = 13 > D[2][0]=3，不更新
- D[2][1]+D[1][2] = 7+2 = 9 > D[2][2]=0，不更新

```
D^(2) =
     0    1    2
0  [ 0,   4,   6]
1  [ 6,   0,   2]
2  [ 3,   7,   0]
```

**k=2（经过顶点 2）：**
- D[0][2]+D[2][1] = 6+7 = 13 > D[0][1]=4，不更新
- D[1][2]+D[2][0] = 2+3 = 5 < D[1][0]=6，更新 D[1][0]=5

```
D^(3) =
     0    1    2
0  [ 0,   4,   6]
1  [ 5,   0,   2]
2  [ 3,   7,   0]
```

**最终结果：** D^(3) 即为所有顶点对之间的最短路径。

### 例题 3：综合应用题（带负权边）

**题目：** 设有向图 $G$ 如下，请回答：
- (1) 能否用 Dijkstra 算法求从顶点 A 出发的最短路径？为什么？
- (2) 用 Floyd 算法求所有顶点对的最短路径。

```
A --5--> B
A --1--> C
C --(-2)--> B
B --3--> D
```

**解析：**

**(1) 不能用 Dijkstra 算法。**

原因：图中存在负权边 $(C,B)$ 权值为 $-2$。Dijkstra 算法基于贪心策略，假设已加入集合 $S$ 的顶点最短距离不会被更新，但负权边可能导致已确定的最短距离被推翻。

具体分析：Dijkstra 从 A 出发，dist[B]=5, dist[C]=1，选择 C 加入 S（dist=1）。然后从 C 松弛到 B：dist[C]+(-2)=1-2=-1 < 5，更新 dist[B]=-1。但 C 已经加入 S，这在标准 Dijkstra 中不会被再次考虑，且 B 的新距离可能影响其他顶点。

实际上对于本例，因为 C 加入 S 后仍然松弛了 B，所以本例 Dijkstra 恰好能给出正确结果。但如果结构更复杂（如负权边影响已加入 S 的顶点），则会出错。**考试中统一回答：有负权边不能用 Dijkstra。**

**(2) Floyd 算法：**

邻接矩阵：
```
     A    B    C    D
A  [ 0,   5,   1,   ∞]
B  [ ∞,   0,   ∞,   3]
C  [ ∞,  -2,   0,   ∞]
D  [ ∞,   ∞,   ∞,   0]
```

k=A: 无更新
k=B: D[C][B]+D[B][D] = -2+3 = 1 < ∞，更新 D[C][D]=1
k=C: D[A][C]+D[C][B] = 1+(-2) = -1 < 5，更新 D[A][B]=-1
     D[A][C]+D[C][D] = 1+1 = 2 < ∞，更新 D[A][D]=2
k=D: 无更新

```
最终：
     A    B    C    D
A  [ 0,  -1,   1,   2]
B  [ ∞,   0,   ∞,   3]
C  [ ∞,  -2,   0,   1]
D  [ ∞,   ∞,   ∞,   0]
```

## 五、易错点

### 5.1 Dijkstra 相关

1. **负权边会导致错误结果**
   - 即使图是连通的，有负权边就不能用 Dijkstra
   - 考试常见陷阱：给一个带负权边的图，问能否用 Dijkstra

2. **已加入 S 的顶点不可再更新**
   - Dijkstra 一旦确定某个顶点的最短路径，就不会再修改
   - 这是贪心正确性的前提，也是不能处理负权边的根本原因

3. **找最小值时注意排除已访问的顶点**
   - `if (!visited[j] && dist[j] < min)` — 必须加 visited 判断

### 5.2 Floyd 相关

1. **k 必须在最外层循环**
   - 若 k 在内层，结果可能不正确
   - 原因：需要逐步考虑每个中间顶点

2. **Floyd 也不能处理负权回路**
   - 负权回路：环上边权之和 < 0
   - 检测方法：对角线元素 `dist[i][i] < 0`

3. **三重循环顺序**
   - 正确：k-i-j 或 k-j-i
   - 错误：i-j-k 或其他 k 不在最外层的顺序

### 5.3 通用易错

1. **最短路径不唯一时**，题目可能要求输出特定的一条
2. **无权图的最短路径**：等价于 BFS 求最少边数
3. **初始化**：dist[src]=0（源点到自己的距离），不要遗漏

---

> - **Dijkstra**：贪心，$O(n^2)$，单源，**禁止**负权边
> - **Floyd**：DP，$O(n^3)$，全源，**允许**负权边（禁止负权回路）
> - Floyd 三重循环：**k 在最外层**
> - 松弛操作：`dist[v] = min(dist[v], dist[u] + w(u,v))`
> - 负权回路检测：Floyd 中 `dist[i][i] < 0` 即存在
