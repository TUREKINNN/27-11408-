# 树的存储结构

## 📊 考情分析
- **考试频率**：中频，常与二叉树存储结构对比考查
- **分值**：选择题2-4分
- **题型**：选择题为主
- **真题考查方式**：三种存储方式的特点对比、双亲表示法与孩子表示法的优缺点辨析

## 一、基本概念

### 树的存储方式总览

树的存储结构有三种主要方式：
1. **双亲表示法**（顺序存储）
2. **孩子表示法**（顺序+链式）
3. **孩子兄弟表示法**（链式，最重要）

## 二、核心知识

### 1. 双亲表示法

**思路**：用数组顺序存储各个结点。每个结点中保存数据元素、指向双亲结点的"指针"（数组下标）。

```c
typedef struct {
    ElemType data;
    int parent;  // 双亲在数组中的下标
} PTNode;

typedef struct {
    PTNode nodes[MaxSize];
    int n;  // 结点数
} PTree;
```

**示例**：
```
树：         A          数组：
           /|\         下标  data  parent
          B C D         0    A     -1
         /|   |         1    B      0
        E F   G         2    C      0
                        3    D      0
                        4    E      1
                        5    F      1
                        6    G      3
```
- 根结点的 `parent = -1`
- 非根结点的 `parent` = 父节点在数组中的下标

**双亲表示法存储森林**：每棵树的根结点 `parent = -1`

| 优点 | 缺点 |
|------|------|
| 找**双亲**很方便，$O(1)$ | 找**孩子**不方便，需遍历整个数组 |
| 适用于"找父亲多、找孩子少"的场景（如并查集） | |

### 2. 孩子表示法

**思路**：用数组顺序存储各个结点。每个结点中保存数据元素、孩子链表的头指针。

```c
// 孩子链表结点
typedef struct CTNode {
    int child;           // 孩子在数组中的下标
    struct CTNode *next;
} *ChildPtr;

// 数组结点
typedef struct {
    ElemType data;
    ChildPtr firstchild;  // 孩子链表头指针
} CTBox;

// 树
typedef struct {
    CTBox nodes[MaxSize];
    int n, r;  // 结点数、根的位置
} CTree;
```

**示例**：
```
下标  data  firstchild
0     A    → 1 → 2 → 3 ↦
1     B    → 4 → 5 ↦
2     C    → 6 ↦
3     D    ↦
4     E    ↦
5     F    ↦
6     G    ↦
```

| 优点 | 缺点 |
|------|------|
| 找**孩子**很方便 | 找**双亲**不方便 |
| | 需要额外的链表指针空间 |

### 3. 孩子兄弟表示法（二叉树表示法）

**思路**：每个结点存储数据元素、**第一个孩子指针**和**右兄弟指针**。

```c
typedef struct CSNode {
    ElemType data;
    struct CSNode *firstchild;  // 第一个孩子
    struct CSNode *nextsibling; // 右兄弟
} CSNode, *CSTree;
```

**示例**：
```
树：         A            孩弟表示：
           /|\           A → firstchild=B, nextsibling=NULL
          B C D          B → firstchild=E, nextsibling=C
         /|   |          C → firstchild=NULL, nextsibling=D
        E F   G          D → firstchild=G, nextsibling=NULL
                         E → firstchild=NULL, nextsibling=F
                         F → firstchild=NULL, nextsibling=NULL
                         G → firstchild=NULL, nextsibling=NULL
```

| 优点 | 缺点 |
|------|------|
| 可以方便地将树转换为**二叉树** | |
| 最灵活、最常用 | |

## 三、深入理解

### 孩子兄弟表示法的重要意义

孩子兄弟表示法将一棵**任意的树**转换为一棵**二叉树**：
- `firstchild` → 二叉树的**左孩子**
- `nextsibling` → 二叉树的**右孩子**

这是**树与二叉树转换**的基础！

### 三种表示法的对比

| | 双亲表示法 | 孩子表示法 | 孩子兄弟表示法 |
|---|---|---|---|
| 存储方式 | 顺序 | 顺序+链式 | 链式 |
| 找双亲 | $O(1)$ | $O(n)$ | $O(n)$ |
| 找孩子 | $O(n)$ | $O(1)$ | $O(1)$（第一个孩子） |
| 找兄弟 | $O(n)$ | $O(1)$（链表中下一个） | $O(1)$ |
| 适用场景 | 并查集 | 通用 | 树↔二叉树转换 |

## 四、真题考法分析

### 常考题型
1. 给定树的图形，写出双亲表示法的数组
2. 孩子兄弟表示法与二叉树的对应关系
3. 三种表示法的优缺点对比
4. 给定双亲表示法，判断某结点的父节点

### 命题规律
- 孩子兄弟表示法是核心，与"树与二叉树的转换"紧密相关
- 双亲表示法常与并查集结合考查

## 五、典型例题

### 概念题

**例1**：一棵有 $n$ 个结点的树采用双亲表示法存储，判断结点 $i$ 是否为根结点的条件是？

**解**：`nodes[i].parent == -1`

**例2**：在孩子兄弟表示法中，如何判断一个结点是否为叶子结点？

**解**：`p->firstchild == NULL`（没有第一个孩子，说明没有孩子）

### 计算题

**例3**：一棵有 $n$ 个结点、度为 $k$ 的树，采用孩子表示法存储，共需要多少个链表结点？

**解**：链表结点数 = 边数 = $n - 1$（每个结点恰好被其父节点的一条边指向）

## 六、易错点

1. **双亲表示法中根的parent为-1**：不是0，0是一个合法的下标
2. **孩子兄弟表示法是重点**：它直接关联到树与二叉树的转换
3. **孩子表示法找双亲不方便**：不是不能，而是需要遍历
4. **孩子兄弟表示法的两个指针含义**：firstchild是"第一个孩子"，nextsibling是"右兄弟"
5. **边数 = n-1**：这对计算孩子表示法的链表结点数很重要
