# 单链表的查找

## 📊 考情分析
- **考试频率**：高频考点
- **分值**：选择题2-4分，算法题可结合考查
- **题型**：时间复杂度分析、代码实现、封装思想
- **真题考查方式**：按位查找的O(n)特性、按值查找的实现、基本操作的封装

---

## 一、基本概念

### 1.1 按位查找

**GetElem(L, i)**：获取表L中第 $i$ 个位置的元素的值。

### 1.2 按值查找

**LocateElem(L, e)**：在表L中查找具有给定关键字值的元素。

> 注：本节只探讨**带头结点**的情况。

---

## 二、核心知识

### 2.1 按位查找实现

```c
// 按位查找，返回第i个元素（带头结点）
LNode *GetElem(LinkList L, int i) {
    if (i < 0)
        return NULL;
    
    LNode *p;
    int j = 0;
    p = L;
    
    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }
    
    return p;
}
```

**边界情况分析**：

| 情况 | i值 | 结果 |
|------|-----|------|
| 头结点 | $i = 0$ | 返回头结点 |
| 第1个数据结点 | $i = 1$ | 返回第1个结点 |
| 超出范围 | $i = 8$（链表长度不足） | 返回 NULL |
| 普通情况 | $i = 3$ | 返回第3个结点 |

**时间复杂度**：$O(n)$

### 2.2 按值查找实现

```c
// 按值查找，返回第一个数据域为e的结点的指针
LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;  // 从第一个数据结点开始
    
    while (p != NULL && p->data != e) {
        p = p->next;
    }
    
    return p;  // 找到返回结点指针，未找到返回NULL
}
```

**执行示例**：
```
L → [头] → [5] → [8] → [10] → [2] → NULL
调用 LocateElem(L, 8)
  p指向5: 5≠8, 继续
  p指向8: 8=8, 返回该结点指针
```

**时间复杂度**：$O(n)$

### 2.3 求表长

```c
// 求表的长度（带头结点）
int Length(LinkList L) {
    int len = 0;
    LNode *p = L->next;
    
    while (p != NULL) {
        p = p->next;
        len++;
    }
    
    return len;
}
```

**时间复杂度**：$O(n)$

### 2.4 封装的好处

将查找操作封装为基本操作，可以在其他操作中复用：

```c
// 在第i个位置插入元素（利用封装）
bool ListInsert(LinkList &L, int i, ElemType e) {
    LNode *p = GetElem(L, i - 1);  // 找到第i-1个结点
    return InsertNextNode(p, e);   // 在p后插入新结点
}

// 考虑非法情况：健壮性
if (p == NULL)
    return false;
```

> 💡 **好处**：避免重复代码，代码逻辑清晰、简洁、易维护。

## 三、深入理解

### 3.1 链表 vs 顺序表的查找对比

| 操作 | 顺序表 | 单链表 |
|------|--------|--------|
| 按位查找 | $O(1)$ ✅ | $O(n)$ ❌ |
| 按值查找 | $O(n)$ | $O(n)$ |

**原因**：
- 顺序表支持随机存取，可通过地址公式直接定位
- 单链表必须从头结点开始依次遍历

### 3.2 为什么链表按位查找是 $O(n)$？

```
L → [头] → [a1] → [a2] → [a3] → [a4] → NULL
```

要找到第 $i$ 个结点，必须：
1. 从头结点出发
2. 沿着 next 指针依次向后移动
3. 移动 $i$ 次

无法跳过中间结点直接访问，这就是**非随机存取**的含义。

### 3.3 ElemType 为结构类型时的比较

```c
typedef struct {
    int num;
    int people;
} Customer;

// 基本类型可以直接用 == 比较
if (p->data == e)

// 结构类型需要逐个分量比较
if (p->data.num == e.num && p->data.people == e.people)
```

> 💡 **考研初试提示**：手写代码可以直接用 `==`，主要考察算法思想。

## 五、典型例题

### 算法题

**Q1**：查找单链表中倒数第 $k$ 个结点。

```c
// 方法：双指针法
LNode *FindKthToTail(LinkList L, int k) {
    LNode *p = L->next;  // 快指针
    LNode *q = L->next;  // 慢指针
    
    // p先走k步
    for (int i = 0; i < k; i++) {
        if (p == NULL)
            return NULL;  // k超出链表长度
        p = p->next;
    }
    
    // p和q同时走
    while (p != NULL) {
        p = p->next;
        q = q->next;
    }
    
    return q;
}
```

**Q2**：查找单链表的中间结点。

```c
// 方法：快慢指针法
LNode *FindMiddle(LinkList L) {
    LNode *slow = L->next;  // 慢指针，每次走1步
    LNode *fast = L->next;  // 快指针，每次走2步
    
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
    }
    
    return slow;
}
```

**Q3**：判断单链表是否有环。

```c
// 方法：快慢指针法
bool HasCycle(LinkList L) {
    LNode *slow = L->next;
    LNode *fast = L->next;
    
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
        
        if (slow == fast)  // 相遇说明有环
            return true;
    }
    
    return false;
}
```

## 六、易错点

1. **按位查找返回指针**：`GetElem` 返回 `LNode *`，不是元素值
2. **按值查找返回指针**：`LocateElem` 返回 `LNode *`，不是位序
3. **不带头结点的处理**：代码逻辑需要调整
4. **边界条件**：$i = 0$、超出范围等特殊情况
5. **结构体比较**：不能直接用 `==`
