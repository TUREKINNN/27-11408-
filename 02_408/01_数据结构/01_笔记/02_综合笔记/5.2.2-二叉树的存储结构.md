# 二叉树的存储结构

## 📊 考情分析
- **考试频率**：中高频，常与其他知识点结合考查
- **分值**：2-4分（选择题）或嵌入算法题中
- **题型**：选择题、算法设计题
- **真题考查方式**：顺序存储的空间分析、链式存储的空链域计算

## 一、基本概念

### 1. 顺序存储

定义一个长度为 $MaxSize$ 的数组 $t$，按照**从上至下、从左至右**的顺序依次存储完全二叉树中的各个结点。

```
完全二叉树：          顺序存储：
    1                t[1]=1
   / \              t[2]=2  t[3]=3
  2   3             ...
 / \ / \            t[8]=8  t[9]=9  t[10]=10 ...
4  5 6  7
```

**编号规则**（从1开始）：
- 结点 $i$ 的**左孩子**：$2i$
- 结点 $i$ 的**右孩子**：$2i+1$
- 结点 $i$ 的**父节点**：$\lfloor i/2 \rfloor$
- 判断 $i$ 是否有**左孩子**：$2i \leq n$ ?
- 判断 $i$ 是否有**右孩子**：$2i+1 \leq n$ ?
- 判断 $i$ 是否是**叶子**：$i > \lfloor n/2 \rfloor$ ?
- 结点 $i$ 的**层次**：$\lceil \log_2(i+1) \rceil$ 或 $\lfloor \log_2 i \rfloor + 1$

### 2. 链式存储（二叉链表）

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild;  // 左右孩子指针
} BiTNode, *BiTree;
```

**重要性质**：$n$ 个结点的二叉链表共有 $n+1$ 个**空链域**。

**推导**：
- 总指针域 = $2n$（每个结点有2个指针域）
- 非空指针域 = $n-1$（边数，每个非根结点被一个指针指向）
- 空链域 = $2n - (n-1) = n+1$

### 3. 三叉链表（带父指针的二叉链表）

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild;
    struct BiTNode *parent;  // 父节点指针
} BiTNode, *BiTree;
```

## 二、核心知识

### 顺序存储的特点

| 优点 | 缺点 |
|------|------|
| 适合完全二叉树 | 非完全二叉树浪费空间 |
| 下标直接反映父子关系 | 最坏情况：高度 $h$ 只有 $h$ 个结点需要 $2^h-1$ 个单元 |
| 无需额外指针开销 | 不方便插入删除 |

**最坏情况**：高度为 $h$ 且只有 $h$ 个结点的单支树（所有结点只有右孩子），需要 $2^h-1$ 个存储单元。

> **结论**：二叉树的顺序存储结构，**只适合存储完全二叉树**

### 链式存储的特点

| 优点 | 缺点 |
|------|------|
| 适用任何二叉树 | 需要额外指针空间 |
| 插入删除方便 | 找父结点不方便 |
| 空间利用率高 | |

**找父结点**：二叉链表中找指定结点 $p$ 的父结点，只能**从根开始遍历**。

> **Tips**：根据实际需求决定要不要加父结点指针（三叉链表）

## 三、深入理解

### 顺序存储的空间分析

对于非完全二叉树的顺序存储，必须按照完全二叉树的编号方式存储，中间可能出现空位：

```
    A
   / \
  B   C
 /
D
```

存储为：`[_, A, B, C, D, _, _]`（7个位置只用了4个）

### 空链域的应用
$n+1$ 个空链域可以用来存储**线索信息**，这就是线索二叉树的基础。

### 存储结构选择

| 场景 | 推荐存储 |
|------|----------|
| 完全二叉树/满二叉树 | 顺序存储 |
| 一般二叉树 | 二叉链表 |
| 需要频繁找父节点 | 三叉链表 |
| 需要线索化 | 线索链表 |

## 四、真题考法分析

### 常考题型
1. 计算顺序存储所需最少空间
2. 计算二叉链表的空链域数目
3. 完全二叉树的父子编号计算
4. 存储结构的适用场景选择

### 命题规律
- 空链域 $n+1$ 是高频考点（常与线索二叉树结合）
- 顺序存储的空间浪费是常考陷阱

## 五、典型例题

### 计算题

**例1**：一棵有 $n$ 个结点的二叉树采用二叉链表存储，其中空指针域有多少个？

**解**：$n+1$ 个空链域。

**例2**：完全二叉树有100个结点，按从1编号，结点50的左孩子编号为？

**解**：$2 \times 50 = 100$，左孩子编号为100（因为 $100 \leq 100$，存在）。

**例3**：一棵高度为5的单支树（只有右孩子），采用顺序存储需要多少空间？

**解**：需要 $2^5 - 1 = 31$ 个存储单元。

### 算法题

**例4**：写出在二叉链表上计算二叉树高度的算法。

```c
int Depth(BiTree T) {
    if (T == NULL)
        return 0;
    int left = Depth(T->lchild);
    int right = Depth(T->rchild);
    return (left > right ? left : right) + 1;
}
```

## 六、易错点

1. **顺序存储必须对应完全二叉树编号**：非完全二叉树也要按完全二叉树方式编号
2. **空链域是 $n+1$ 不是 $n-1$**：$2n$ 个指针域中 $n-1$ 个非空，$n+1$ 个为空
3. **编号从1开始**：如果从0开始，左孩子为 $2i+1$，右孩子为 $2i+2$，父为 $\lfloor(i-1)/2\rfloor$
4. **二叉链表找父节点需遍历**：不能直接找到，除非用三叉链表
5. **顺序存储空间不一定最优**：对一般二叉树可能浪费大量空间
