# 图的关键路径

> [[02_408/01_数据结构/01_笔记/02_综合笔记/04-图]] | [[02_408/01_数据结构/01_笔记/02_综合笔记/图的拓扑排序]]

## 概念

**关键路径**：在 AOE（Activity On Edge）网中，从源点到汇点的**最长路径**（决定整个工程的最短完成时间）。

- **AOE 网**：边表示活动、顶点表示事件的有向无环图
- 关键活动：最早开始时间 = 最晚开始时间的活动

## 关键术语

| 术语 | 含义 |
|------|------|
| **ve(vj)** | 事件 vj 的最早发生时间 |
| **vl(vj)** | 事件 vj 的最晚发生时间 |
| **e(ai)** | 活动 ai 的最早开始时间 |
| **l(ai)** | 活动 ai 的最晚开始时间 |
| **d(ai)** | 活动 ai 的时间余量 = l - e |
| **关键活动** | d(ai) = 0 的活动 |

## 计算步骤

### 1. 求 ve（最早发生时间）—— 正向

从源点开始，拓扑排序：
```
ve(源点) = 0
ve(vj) = max{ve(vi) + dut(<vi, vj>)}  对所有前驱 vi
```

### 2. 求 vl（最晚发生时间）—— 逆向

从汇点开始，逆拓扑排序：
```
vl(汇点) = ve(汇点)
vl(vi) = min{vl(vj) - dut(<vi, vj>)}  对所有后继 vj
```

### 3. 求 e 和 l

```
e(ai) = ve(vj)  // ai 的弧尾 vj
l(ai) = vl(vk) - dut(ai)  // ai 的弧头 vk
```

### 4. 求 d 并判断关键活动

```
d(ai) = l(ai) - e(ai)
d(ai) = 0 ⟹ 关键活动
```

## 示例

```
       a1=3      a2=2      a3=4
   V1 ------→ V2 ------→ V4 ------→ V6
   |          ↑↓          ↑          ↑
   |a4=2   a5=1        a6=3      a7=2
   ↓          |           |          |
   V3 ------→ V4         V5 ------→ V7
       a8=2          (a7连接V5→V7)
```

简化：
```
   1 →(3)→ 2 →(2)→ 4
   ↑      ↓
  (2)    (1)
   |      ↓
   3 →(2)→ 4
```

**ve 计算**：
- ve(1) = 0
- ve(3) = ve(1) + 2 = 2
- ve(2) = max(ve(1)+3, ve(3)+2) = max(3, 4) = 4
- ve(4) = max(ve(2)+2, ve(3)+?) 

**关键路径**：1→2→4→6（权值和最大）

## 关键路径特点

1. 至少有一条关键路径
2. 关键路径上的活动是关键活动
3. 关键活动的时间余量为 0
4. 缩短关键活动的时间可缩短整个工期

## 应用

- 项目管理（PERT/CPM）
- 工程进度控制
- 任务调度优化
