# 单链表的定义

## 📊 考情分析
- **考试频率**：高频考点，必考
- **分值**：选择题2-4分，综合题必涉及
- **题型**：概念辨析、代码实现、带头/不带头结点的区别
- **真题考查方式**：链表结构定义、头结点的作用、带头/不带头结点的代码差异

---

## 一、基本概念

### 1.1 单链表的定义

**单链表**——用**链式存储**的方式实现线性表。

每个结点除了存放数据元素外，还要存储指向下一个结点的**指针**。

### 1.2 结点结构

```
┌────────┬────────┐
│  data  │  next  │
│ 数据域 │ 指针域 │
└────────┴────────┘
```

### 1.3 顺序表 vs 单链表

| 对比项 | 顺序表 | 单链表 |
|--------|--------|--------|
| 存储方式 | 连续空间 | 离散空间 |
| 随机存取 | ✅ 支持 | ❌ 不支持 |
| 存储密度 | 高 | 低（需要存指针） |
| 扩容 | 不方便 | 方便 |
| 空间要求 | 大片连续空间 | 不要求连续空间 |

## 二、核心知识

### 2.1 用代码定义单链表

#### 结点定义
```c
typedef struct LNode {
    ElemType data;        // 数据域
    struct LNode *next;   // 指针域
} LNode, *LinkList;
```

**说明**：
- `typedef struct LNode LNode;` —— 将 `struct LNode` 重命名为 `LNode`
- `typedef struct LNode *LinkList;` —— 将 `struct LNode *` 重命名为 `LinkList`

#### 增加新结点
```c
LNode *p = (LNode *)malloc(sizeof(LNode));
```

**作用**：在内存中申请一个结点所需空间，并用指针 `p` 指向这个结点。

### 2.2 LinkList vs LNode*

| 声明方式 | 含义 | 使用场景 |
|----------|------|----------|
| `LinkList L` | 强调这是一个**单链表** | 声明头指针 |
| `LNode *p` | 强调这是一个**结点** | 操作结点指针 |

> 💡 **代码可读性**：`LinkList` 强调单链表，`LNode *` 强调结点。

### 2.3 不带头结点的单链表

```c
// 初始化：防止脏数据
LinkList L = NULL;  // 头指针指向NULL
```

**特点**：
- 头指针 `L` 直接指向第一个数据结点
- 空表时 `L = NULL`
- 对第一个数据结点和后续数据结点需要用**不同的代码逻辑**
- 空表和非空表需要用**不同的代码逻辑**

### 2.4 带头结点的单链表

```c
// 初始化
LinkList L = (LinkList)malloc(sizeof(LNode));  // 创建头结点
L->next = NULL;  // 头结点next指向NULL
```

**特点**：
- 头结点不存储数据，只是为了操作方便
- 空表时 `L->next = NULL`
- 所有数据结点的处理逻辑**统一**
- **推荐使用带头结点**

### 2.5 带头 vs 不带头对比

| 对比项 | 不带头结点 | 带头结点 |
|--------|-----------|----------|
| 空表判断 | `L == NULL` | `L->next == NULL` |
| 第一个结点 | 特殊处理 | 统一处理 |
| 代码复杂度 | 更复杂 | 更简单 |
| 头指针含义 | 指向第一个数据结点 | 指向头结点 |

> ⚠️ **考试注意**：带头、不带头都有可能考察，注意审题！

## 三、深入理解

### 3.1 为什么要带头结点？

**问题**：不带头结点时，插入/删除第一个元素需要修改头指针 `L`。

**带头结点的优势**：
1. 代码逻辑统一，不需要特殊处理第一个结点
2. 空表和非空表处理方式一致
3. 代码更简洁，不易出错

### 3.2 内存中的单链表

```
内存中（离散分布）：
┌──────┐     ┌──────┐     ┌──────┐     ┌──────┐
│  a1  │ ──→ │  a2  │ ──→ │  a3  │ ──→ │  a4  │ ──→ NULL
└──────┘     └──────┘     └──────┘     └──────┘
  ↑
  L（头指针）
```

### 3.3 指针与结点的关系

```c
LNode *p;  // p是一个指向LNode类型结点的指针
p->data;   // 访问p指向结点的数据域
p->next;   // 访问p指向结点的指针域
```

**等价写法**：
```c
(*p).data   // 等价于 p->data
(*p).next   // 等价于 p->next
```

## 五、典型例题

### 概念题

**Q1**：下列关于单链表的说法，正确的是（）
- A. 单链表支持随机访问
- B. 单链表的存储密度高于顺序表
- C. 带头结点的单链表，空表时 `L->next == NULL`
- D. 单链表需要大片连续的存储空间

**答案**：C

**解析**：
- A错：单链表不支持随机访问
- B错：单链表需要存储指针，存储密度低于顺序表
- C正确：带头结点空表时，头结点的next指向NULL
- D错：单链表的结点在内存中离散分布

### 代码题

**Q2**：写出带头结点单链表的初始化代码。

```c
LinkList InitList() {
    LinkList L = (LinkList)malloc(sizeof(LNode));
    if (L == NULL)  // 内存分配失败
        return NULL;
    L->next = NULL;
    return L;
}
```

---

## 六、易错点

1. **头结点不存数据**：头结点的数据域可以不使用
2. **空表判断**：带头结点是 `L->next == NULL`，不带头结点是 `L == NULL`
3. **malloc可能失败**：需要检查返回值
4. **指针必须初始化**：未初始化的指针是"野指针"
5. **带头/不带头审题**：考试中两种都可能考查
