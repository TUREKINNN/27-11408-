# 顺序表的查找

## 📊 考情分析
- **考试频率**：高频考点
- **分值**：选择题2-4分，算法题10-15分
- **题型**：时间复杂度分析、代码实现、随机存取特性考查
- **真题考查方式**：按位查找的O(1)特性、按值查找的实现、结构体比较方法

---

## 一、基本概念

### 1.1 按位查找

**GetElem(L, i)**：获取表L中第 $i$ 个位置的元素的值。

### 1.2 按值查找

**LocateElem(L, e)**：在表L中查找具有给定关键字值的元素。

---

## 二、核心知识

### 2.1 按位查找实现

#### 静态分配
```c
#define MaxSize 10
typedef struct {
    ElemType data[MaxSize];
    int length;
} SqList;

ElemType GetElem(SqList L, int i) {
    return L.data[i-1];
}
```

#### 动态分配
```c
#define InitSize 10
typedef struct {
    ElemType *data;
    int MaxSize;
    int length;
} SeqList;

ElemType GetElem(SeqList L, int i) {
    return L.data[i-1];  // 和访问普通数组的方法一样
}
```

**时间复杂度**：$O(1)$

> 💡 **随机存取特性**：由于顺序表的各个数据元素在内存中连续存放，因此可以根据起始地址和数据元素大小**立即**找到第 $i$ 个元素。

### 2.2 按值查找实现

```c
// 在顺序表L中查找第一个元素值等于e的元素，并返回其位序
int LocateElem(SeqList L, ElemType e) {
    for (int i = 0; i < L.length; i++) {
        if (L.data[i] == e)
            return i + 1;  // 数组下标为i的元素值等于e，返回其位序i+1
    }
    return 0;  // 退出循环，说明查找失败
}
```

**执行示例**：
```
L.data = [6, 2, 9, 4, 9, 3]  length=6
调用 LocateElem(L, 9)
  i=0: data[0]=6 ≠ 9
  i=1: data[1]=2 ≠ 9
  i=2: data[2]=9 = 9 → 返回 i+1=3
```

### 2.3 结构类型的比较

**注意**：C语言中，结构体的比较**不能直接用 `==`**

```c
typedef struct {
    int num;
    int people;
} Customer;

// 错误写法
if (L.data[i] == e)  // 结构体不能直接用 == 比较

// 正确写法：依次对比各个分量
bool compare(Customer a, Customer b) {
    if (a.num == b.num && a.people == b.people)
        return true;
    return false;
}
```

> 💡 **考研初试提示**：手写代码可以直接用 `==`，无论 `ElemType` 是基本数据类型还是结构类型。主要考察算法思想，不会严格要求代码完全可运行。

## 三、深入理解

### 3.1 按位查找时间复杂度

**时间复杂度**：$O(1)$

**原理**：顺序表的各元素在内存中连续存放，通过地址计算公式直接定位：
$$LOC(a_i) = LOC(L) + (i-1) \times sizeof(ElemType)$$

这就是顺序表的**随机存取**特性。

### 3.2 按值查找时间复杂度

**问题规模**：$n = L.length$（表长）

| 情况 | 条件 | 循环次数 | 时间复杂度 |
|------|------|----------|------------|
| **最好** | 目标元素在表头 | 1次 | $O(1)$ |
| **最坏** | 目标元素在表尾 | $n$次 | $O(n)$ |
| **平均** | 等概率出现在任意位置 | $\frac{n+1}{2}$次 | $O(n)$ |

**平均情况推导**：

假设目标元素出现在任何一个位置的概率相同，都是 $\frac{1}{n}$

- 在第1位，循环1次
- 在第2位，循环2次
- ...
- 在第 $n$ 位，循环 $n$ 次

$$\text{平均循环次数} = 1 \cdot \frac{1}{n} + 2 \cdot \frac{1}{n} + \dots + n \cdot \frac{1}{n} = \frac{n(n+1)}{2n} = \frac{n+1}{2}$$

$$\text{平均时间复杂度} = O(n)$$

### 3.3 指针类型与地址计算

```c
ElemType *data;  // 如果一个ElemType占6B
int *p;          // 一个int占4B
```

当 `data` 指向地址2000时：
- `data[0]`：从2000开始的6B
- `data[1]`：从2006开始的6B
- `data[2]`：从2012开始的6B

如果用 `int *p` 指向同一地址，`p[1]` 会从2004开始（因为int占4B），导致数据错乱。

> 💡 **关键理解**：malloc函数返回的存储空间起始地址要转换为与数据元素的数据类型相对应的指针。

## 五、典型例题

### 算法题

**Q1**：在顺序表中查找值为 $x$ 的元素，若找到则将其与后继元素交换，若找不到则插入表中使表仍有序。

```c
void SearchExchangeInsert(SqList &L, ElemType x) {
    int low = 0, high = L.length - 1, mid;
    // 二分查找
    while (low <= high) {
        mid = (low + high) / 2;
        if (L.data[mid] == x) break;
        else if (L.data[mid] < x) low = mid + 1;
        else high = mid - 1;
    }
    
    if (L.data[mid] == x && mid != L.length - 1) {
        // 找到且不是最后一个，交换
        ElemType temp = L.data[mid];
        L.data[mid] = L.data[mid + 1];
        L.data[mid + 1] = temp;
    } else {
        // 未找到，插入
        int i;
        for (i = L.length - 1; i > high; i--) {
            L.data[i + 1] = L.data[i];
        }
        L.data[i + 1] = x;
        L.length++;
    }
}
```

**Q2**：设计一个算法，找出顺序表中出现次数最多的元素。

```c
ElemType MajorityElem(SqList L) {
    int count = 1;
    ElemType candidate = L.data[0];
    
    // 第一遍：找候选元素
    for (int i = 1; i < L.length; i++) {
        if (L.data[i] == candidate) {
            count++;
        } else {
            count--;
            if (count == 0) {
                candidate = L.data[i];
                count = 1;
            }
        }
    }
    
    // 第二遍：验证候选元素
    count = 0;
    for (int i = 0; i < L.length; i++) {
        if (L.data[i] == candidate) count++;
    }
    
    if (count > L.length / 2)
        return candidate;
    return -1;  // 不存在多数元素
}
```

## 六、易错点

1. **按位查找返回的是元素值**，不是位序
2. **按值查找返回的是位序**，不是下标（位序 = 下标 + 1）
3. **结构体比较不能用 `==`**：需要逐个分量比较
4. **指针类型必须匹配**：否则地址计算会出错
5. **查找失败的返回值**：按值查找失败返回0（不是-1）
