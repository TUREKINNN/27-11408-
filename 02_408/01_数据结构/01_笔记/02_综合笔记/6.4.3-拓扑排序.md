## 一、基本概念

### 1.1 AOV 网络定义

**AOV 网**（Activity On Vertex Network）是以**顶点表示活动**、**边表示活动之间的优先关系**的**有向无环图**（DAG）。

- 若存在边 $\langle v_i, v_j \rangle$，表示活动 $v_i$ 必须在 $v_j$ 之前完成
- $v_i$ 是 $v_j$ 的**前驱**，$v_j$ 是 $v_i$ 的**后继**
- AOV 网中**不能有环**（否则出现循环依赖，活动无法执行）

**典型应用场景：**
- 课程先修关系
- 工程任务调度
- 编译时模块依赖分析

### 1.2 拓扑排序定义

对有向无环图 $G=(V,E)$ 的所有顶点排成一个线性序列，使得对任意边 $\langle v_i, v_j \rangle$，$v_i$ 都排在 $v_j$ 的**前面**。这个线性序列称为**拓扑排序序列**。

**关键性质：**
- **DAG 的拓扑排序序列不唯一**（可能有多个合法序列）
- **有环的图不存在拓扑排序序列**
- 拓扑排序是对 DAG 中所有顶点的一种**线性化**（linearization）

### 1.3 拓扑排序的存在性

| 图类型 | 是否存在拓扑排序 |
|-------|----------------|
| DAG（有向无环图） | ✅ 存在（可能不唯一） |
| 有环的有向图 | ❌ 不存在 |
| 无向图 | ❌ 无意义（无方向概念） |

## 二、算法详解

### 2.1 拓扑排序算法步骤

#### 基于邻接表 + 入度的算法（重点）

```
步骤1：初始化
  - 计算所有顶点的入度 inDegree[i]
  - 建立一个栈（或队列），将所有入度为0的顶点入栈

步骤2：循环（直到栈为空）
  - 从栈中弹出一个顶点 u，输出 u
  - 对 u 的每个邻接顶点 v：
    - inDegree[v]--
    - 若 inDegree[v] == 0，则将 v 入栈

步骤3：判断
  - 若输出的顶点数 == n，排序成功
  - 若输出的顶点数 < n，说明图中有环
```

### 2.2 算法详解

#### 2.2.1 为什么用入度？

- 入度为 0 的顶点**没有前驱**，可以最先执行
- 每输出一个顶点，将其后继的入度减 1
- 若某顶点入度变为 0，说明其所有前驱都已输出，该顶点可以执行

#### 2.2.2 用栈还是用队列？

- **栈**：DFS 风格，得到的拓扑序列不唯一
- **队列**：BFS 风格，得到的拓扑序列不唯一
- **考试中通常使用栈**（或不强调具体容器）
- 无论用什么，得到的都是合法的拓扑排序序列

#### 2.2.3 判断图中是否有环

> **拓扑排序可以判断有向图是否有环：**
> - 若输出顶点数 == $n$：无环（DAG）
> - 若输出顶点数 < $n$：有环

**原理：** 有环时，环中的顶点入度永远不会变为 0（因为环上每个顶点都有来自环内其他顶点的入边），因此不会被输出。

### 2.3 时间复杂度分析

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 计算入度 | $O(n+m)$ | 遍历所有边 |
| 主循环 | $O(n+m)$ | 每个顶点出栈一次 + 每条边被处理一次 |
| **总复杂度** | **$O(n+m)$** | |

**空间复杂度：** $O(n)$（栈 + 入度数组）

## 三、代码实现

### 3.1 邻接表存储结构

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXVEX 100

// 边结点
typedef struct ArcNode {
    int adjvex;            // 邻接顶点下标
    struct ArcNode *next;  // 指向下一条边
} ArcNode;

// 顶点结点
typedef struct {
    int data;         // 顶点数据
    ArcNode *first;   // 指向第一条边
} VNode;

// 邻接表
typedef struct {
    VNode vexs[MAXVEX];
    int numVertexes, numEdges;
} AdjGraph;
```

### 3.2 拓扑排序完整实现

```c
/**
 * 拓扑排序（基于邻接表 + 栈）
 * @param G 图（邻接表存储）
 * @return 1表示排序成功（无环），0表示有环
 */
int TopologicalSort(AdjGraph G) {
    int inDegree[MAXVEX] = {0};  // 入度数组
    int stack[MAXVEX];           // 存放入度为0的顶点
    int top = -1;                // 栈顶指针
    int count = 0;               // 已输出的顶点计数
    int result[MAXVEX];          // 存放拓扑排序结果
    
    // 步骤1：计算所有顶点的入度
    for (int i = 0; i < G.numVertexes; i++) {
        ArcNode *p = G.vexs[i].first;
        while (p != NULL) {
            inDegree[p->adjvex]++;
            p = p->next;
        }
    }
    
    // 步骤2：将所有入度为0的顶点入栈
    for (int i = 0; i < G.numVertexes; i++) {
        if (inDegree[i] == 0) {
            stack[++top] = i;
        }
    }
    
    // 步骤3：主循环
    while (top != -1) {
        // 出栈一个入度为0的顶点
        int u = stack[top--];
        result[count++] = u;
        
        // 遍历u的所有邻接顶点，入度减1
        ArcNode *p = G.vexs[u].first;
        while (p != NULL) {
            int v = p->adjvex;
            inDegree[v]--;
            
            // 若入度变为0，入栈
            if (inDegree[v] == 0) {
                stack[++top] = v;
            }
            p = p->next;
        }
    }
    
    // 步骤4：判断是否有环
    if (count < G.numVertexes) {
        printf("图中存在环，无法完成拓扑排序！\n");
        return 0;
    }
    
    // 输出拓扑排序结果
    printf("拓扑排序结果：");
    for (int i = 0; i < count; i++) {
        printf("%d ", result[i]);
    }
    printf("\n");
    
    return 1;
}
```

### 3.3 基于邻接矩阵的简化实现

```c
/**
 * 拓扑排序（基于邻接矩阵）
 * 适合考试中手写代码
 */
int TopologicalSort_Matrix(MGraph G) {
    int inDegree[MAXVEX] = {0};
    int stack[MAXVEX], top = -1;
    int count = 0;
    
    // 计算入度
    for (int j = 0; j < G.numVertexes; j++) {
        for (int i = 0; i < G.numVertexes; i++) {
            if (G.arc[i][j] != 0 && G.arc[i][j] != INF) {
                inDegree[j]++;
            }
        }
    }
    
    // 入度为0的入栈
    for (int i = 0; i < G.numVertexes; i++) {
        if (inDegree[i] == 0)
            stack[++top] = i;
    }
    
    while (top != -1) {
        int u = stack[top--];
        printf("%d ", u);
        count++;
        
        for (int v = 0; v < G.numVertexes; v++) {
            if (G.arc[u][v] != 0 && G.arc[u][v] != INF) {
                inDegree[v]--;
                if (inDegree[v] == 0)
                    stack[++top] = v;
            }
        }
    }
    
    return count == G.numVertexes;
}
```

## 四、典型例题

### 例题 1：拓扑排序过程（选择题）

**题目：** 对如下有向图进行拓扑排序，给出一个可能的拓扑序列。

```
顶点：1, 2, 3, 4, 5, 6
边：<1,2>, <1,3>, <2,4>, <3,4>, <4,5>, <4,6>, <5,6>
```

**解析：**

计算入度：
| 顶点 | 1 | 2 | 3 | 4 | 5 | 6 |
|------|---|---|---|---|---|---|
| 入度 | 0 | 1 | 1 | 2 | 1 | 2 |

执行过程：

| 步骤 | 操作 | 入度变化 | 输出序列 |
|------|------|---------|---------|
| 初始 | 入度为0的入栈：{1} | — | — |
| 1 | 弹出1，处理后继2,3 | in[2]=0, in[3]=0 | 1 |
| 2 | 弹出2（或3），处理后继4 | in[4]=1 | 1,2 |
| 3 | 弹出3（或2），处理后继4 | in[4]=0 | 1,2,3 |
| 4 | 弹出4，处理后继5,6 | in[5]=0, in[6]=1 | 1,2,3,4 |
| 5 | 弹出5，处理后继6 | in[6]=0 | 1,2,3,4,5 |
| 6 | 弹出6 | — | 1,2,3,4,5,6 |

**输出：** `1, 2, 3, 4, 5, 6`（不唯一，`1, 3, 2, 4, 5, 6` 也是合法的）

### 例题 2：判断有环（选择题）

**题目：** 判断如下有向图是否存在拓扑排序序列。

```
顶点：A, B, C, D
边：<A,B>, <B,C>, <C,D>, <D,B>
```

**解析：**

计算入度：
| 顶点 | A | B | C | D |
|------|---|---|---|---|
| 入度 | 0 | 2 | 1 | 1 |

执行过程：
1. 入度为0的顶点：{A}，弹出 A，处理后继 B → in[B]=1
2. 入度为0的顶点：无

此时栈为空，但只输出了 1 个顶点（共 4 个），**说明图中有环**。

**验证：** B→C→D→B 构成环。

**答案：** 该图不存在拓扑排序序列，因为图中存在环 B→C→D→B。

## 五、易错点

### 5.1 常见错误

1. **忘记计算入度就直接开始**
   - 正确做法：先遍历所有边，统计每个顶点的入度

2. **拓扑排序序列不唯一**
   - 同一个图可能有多个合法的拓扑序列
   - 考试中写出任意一个即可

3. **无环判定条件**
   - 输出顶点数 == $n$ → 无环
   - 输出顶点数 < $n$ → 有环
   - 不要搞反！

4. **拓扑排序只适用于有向图**
   - 无向图没有"前驱后继"关系，拓扑排序无意义

5. **混淆拓扑排序与 DFS 序列**
   - 拓扑排序：基于入度
   - DFS 序列：基于深度优先遍历

### 5.2 考试技巧

- 手动模拟时，**画出入度表**，每步更新
- 有多个入度为0的顶点时，**选择任意一个**即可
- 判断是否有环：检查是否所有顶点都被输出
- **考试常考：给一个图，要求写出拓扑排序序列并判断是否有环**

## 六、关联知识点

### 6.1 前置知识
- **有向图的存储** — 邻接表、邻接矩阵
- **DFS深度优先搜索** — DFS 也可以用于拓扑排序
- **栈** — 存放入度为0的顶点

### 6.2 后续应用
- [[02_408/01_数据结构/01_笔记/02_综合笔记/6.4.4-关键路径]] — 求关键路径前先做拓扑排序
- **AOE网络** — 工程调度问题

### 6.3 拓扑排序的其他应用

1. **判断 DAG**：拓扑排序成功 ⟺ 图是 DAG
2. **关键路径求解**：按拓扑序正向求最早时间，逆拓扑序求最晚时间
3. **任务调度**：保证任务按依赖顺序执行
4. **课程安排**：确定课程学习顺序

### 6.4 DFS 求拓扑排序

利用 DFS 也可以求拓扑排序：
- 对 DAG 进行 DFS，**按完成时间从大到小**排列顶点
- 即 DFS 后序序列的**逆序**就是拓扑排序序列
- 适用于需要同时进行 DFS 遍历的场景

---

> - AOV 网 = 顶点表示活动的有向无环图
> - 拓扑排序核心：**入度为 0 → 输出 → 更新后继入度**
> - 时间复杂度：$O(n+m)$
> - 判环：输出数 < $n$ 则有环
> - 序列不唯一，但都能正确反映偏序关系
