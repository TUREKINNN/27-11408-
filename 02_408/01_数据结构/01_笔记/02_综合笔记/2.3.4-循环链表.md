# 循环链表

## 📊 考情分析
- **考试频率**：中等，选择题常考
- **分值**：2~4分
- **题型**：选择题为主，偶尔出现在算法设计题中
- **真题考查方式**：判断循环链表操作的正确性、循环条件的书写、与普通链表的区别

## 一、基本概念

### 1. 循环单链表
**定义**：循环单链表是单链表的变形，表尾结点的 `next` 指针不是指向 `NULL`，而是**指向头结点**。

**与普通单链表的区别**：
- 普通单链表：表尾结点 `next == NULL`
- 循环单链表：表尾结点 `next == L`（头结点）

**空表判断**：
```c
// 循环单链表空表：L->next == L
bool Empty(LinkList L) {
    return (L->next == L);
}
```

**遍历终止条件**：
- 普通单链表：`p != NULL`
- 循环单链表：`p != L`（回到头结点时停止）

### 2. 循环双链表
**定义**：循环双链表是双链表的变形：
- 表头结点的 `prior` 指向**表尾结点**
- 表尾结点的 `next` 指向**头结点**

**空表判断**：
```c
// 循环双链表空表：L->next == L && L->prior == L
bool Empty(LinkList L) {
    return (L->next == L);
}
```

## 二、核心知识

### 1. 循环单链表的初始化（带头结点）
```c
typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;

// 初始化循环单链表
bool InitList(LinkList *L) {
    *L = (LNode *)malloc(sizeof(LNode));
    if (*L == NULL)
        return false;
    (*L)->next = *L;  // 头结点next指向自己
    return true;
}
```

### 2. 循环单链表的判空
```c
bool Empty(LinkList L) {
    return (L->next == L);
}
```

### 3. 判断结点是否为表尾结点
```c
bool isTail(LinkList L, LNode *p) {
    return (p->next == L);
}
```

### 4. 循环双链表的初始化
```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;
} DNode, *DLinkList;

// 初始化循环双链表
bool InitDLinkList(DLinkList *L) {
    *L = (DNode *)malloc(sizeof(DNode));
    if (*L == NULL)
        return false;
    (*L)->prior = *L;  // 头结点prior指向自己
    (*L)->next = *L;   // 头结点next指向自己
    return true;
}
```

### 5. 循环双链表的插入操作
```c
// 在p结点之后插入s结点
bool InsertNextDNode(DNode *p, DNode *s) {
    s->next = p->next;
    p->next->prior = s;  // 循环双链表不需要判断p->next是否为NULL
    s->prior = p;
    p->next = s;
    return true;
}
```

**关键区别**：普通双链表在表尾插入时需要判断 `p->next` 是否为 `NULL`，而循环双链表不需要，因为**不存在 `NULL` 指针**。

### 6. 循环双链表的删除操作
```c
// 删除p的后继结点q
bool DeleteNextDNode(DNode *p) {
    DNode *q = p->next;
    p->next = q->next;
    q->next->prior = p;  // 循环双链表不需要判断q->next是否为NULL
    free(q);
    return true;
}
```

### 7. 循环双链表的判空
```c
bool Empty(DLinkList L) {
    return (L->next == L);
}
```

### 8. 判断结点是否为表尾结点
```c
bool isTail(DLinkList L, DNode *p) {
    return (p->next == L);
}
```

## 三、深入理解

### 1. 设计思想
循环链表的核心思想是**消除链表中的 `NULL` 指针**，使得从任意结点出发都能遍历整个链表。

**优势**：
- 从任一结点出发可以访问其他任何结点
- 某些操作（如在表尾插入）可以简化代码
- 适合**环形结构**的应用场景

### 2. 循环单链表的优化
如果对链表的操作经常在头部或尾部进行，可以让 `L` 指向**表尾元素**：
- 从尾部找到头部：O(1)
- 从头部找到尾部：O(1)
- 但插入/删除时可能需要修改 `L`

### 3. 复杂度分析
| 操作 | 时间复杂度 |
|------|-----------|
| 初始化 | O(1) |
| 判空 | O(1) |
| 插入 | O(n)（需要找到前驱） |
| 删除 | O(n)（需要找到前驱） |
| 遍历 | O(n) |
| 判断表尾 | O(1) |

## 四、真题考法分析

### 常考题型
1. **判断循环条件**：给出代码片段，判断是否正确遍历循环链表
2. **插入/删除操作**：判断操作序列的正确性
3. **与普通链表的区别**：概念辨析题

### 命题规律
- 重点考查循环链表的**终止条件**和**判空条件**
- 循环双链表的插入/删除操作**不需要判断 `NULL`**
- 常与普通链表的操作进行对比

## 五、典型例题

### 概念题
**题目**：循环单链表 `L` 为空表的条件是（）
A. `L == NULL`  B. `L->next == NULL`  C. `L->next == L`  D. `L->next == L && L->prior == L`

**答案**：C
**解析**：循环单链表空表时，头结点的 `next` 指向自己。D 选项是循环双链表的空表条件。

### 算法题
**题目**：设计一个算法，将循环单链表就地逆置。

```c
void ReverseCircularList(LinkList L) {
    if (L->next == L)  // 空表
        return;
    
    LNode *p = L->next;  // 指向第一个结点
    LNode *q = p->next;
    LNode *temp;
    
    p->next = L;  // 第一个结点变成最后一个结点
    
    while (q != L) {
        temp = q->next;
        q->next = p;
        p = q;
        q = temp;
    }
    
    L->next = p;  // 头结点指向新的第一个结点
}
```

## 六、易错点

1. **判空条件混淆**：
   - 循环单链表：`L->next == L`
   - 循环双链表：`L->next == L`（或 `L->prior == L`）
   - ❌ 错误写法：`L == NULL`

2. **遍历终止条件**：
   - 循环链表：`p != L`（回到头结点时停止）
   - ❌ 错误写法：`p != NULL`（会死循环）

3. **插入/删除操作**：
   - 循环双链表不需要判断 `p->next` 是否为 `NULL`
   - 普通双链表需要判断

4. **表尾判断**：
   - 循环链表：`p->next == L`
   - 普通链表：`p->next == NULL`
