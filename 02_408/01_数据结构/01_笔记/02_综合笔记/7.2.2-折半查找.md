# 7.2.2 折半查找

## 一、基本概念

折半查找，又称"**二分查找**"，**仅适用于有序的顺序表**（不适用于链表）。

> **为什么不能用于链表？** 链表不支持随机访问，无法通过下标直接访问中间元素。

---

## 二、算法思想

在有序表中，每次将待查区间**一分为二**：
1. 取中间元素与目标关键字比较
2. 若相等 → 查找成功
3. 若目标 < 中间值 → 在**左半部分**继续查找
4. 若目标 > 中间值 → 在**右半部分**继续查找
5. 直到 `low > high` → 查找失败

---

## 三、算法实现

```c
// 在有序顺序表中折半查找
int Binary_Search(SSTable ST, KeyType key) {
    int low = 0, high = ST.length - 1, mid;
    while (low <= high) {
        mid = (low + high) / 2;
        if (ST.elem[mid].key == key)
            return mid;        // 查找成功
        else if (key < ST.elem[mid].key)
            high = mid - 1;    // 在左半部分查找
        else
            low = mid + 1;     // 在右半部分查找
    }
    return -1;  // 查找失败
}
```

## 四、查找判定树

### 1. 构造方法

对于有序表 `[7, 10, 13, 16, 19, 29, 32, 33, 37, 41, 43]`（共11个元素）：

```
                    29
                  /    \
               13        41
              /  \      /  \
            10   16   33   43
           /    /    / \
          7   ...  32  37
```

- **根节点** = 中间元素 `mid = (0+10)/2 = 5` → 元素 29
- 递归构造左右子树

### 2. 判定树的性质

| 性质 | 描述 |
|------|------|
| **平衡性** | 左右子树高度差 ≤ 1 |
| **结点数** | n 个内部结点（成功结点），n+1 个外部结点（失败结点） |
| **中序遍历** | 得到有序序列 |
| **树高** | $h = \lceil \log_2(n+1) \rceil$ 或 $h = \lfloor \log_2 n \rfloor + 1$ |

### 3. 查找长度计算

- **查找成功**：比较次数 = 该结点所在**层数**
- **查找失败**：比较次数 = 该失败结点的**父结点**所在层数

---

## 五、ASL 计算

### 1. 查找成功 ASL

对于 n 个元素的有序表，判定树有 $\lceil \log_2(n+1) \rceil$ 层：

$$ASL_{成功} \approx \log_2(n+1) - 1$$

**精确计算**（以 11 个元素为例）：
- 第 1 层：1 个结点，比较 1 次
- 第 2 层：2 个结点，比较 2 次
- 第 3 层：4 个结点，比较 3 次
- 第 4 层：4 个结点，比较 4 次

$$ASL = \frac{1 \times 1 + 2 \times 2 + 4 \times 3 + 4 \times 4}{11} = \frac{33}{11} = 3$$

### 2. 查找失败 ASL

有 n+1 = 12 个失败结点：
- 层 1 的失败结点：0 个
- 层 2 的失败结点：0 个
- 层 3 的失败结点：4 个（父在第 2 层）
- 层 4 的失败结点：8 个（父在第 3 层）

$$ASL_{失败} = \frac{4 \times 2 + 8 \times 3}{12} = \frac{32}{12} \approx 2.67$$

---

## 六、时间复杂度

$$T(n) = O(\log_2 n)$$

每比较一次，搜索范围缩小一半，最多比较 $\lceil \log_2(n+1) \rceil$ 次。

---

## 七、考点总结

| 考点 | 内容 |
|------|------|
| 适用条件 | **有序的顺序表**，不适用于链表 |
| 判定树构造 | 画出折半查找的判定树 |
| ASL 计算 | 成功/失败的平均查找长度 |
| 时间复杂度 | $O(\log_2 n)$ |
| mid 取值 | `mid = (low + high) / 2`（向下取整） |

> 1. 折半查找**只能用于顺序表**，不能用于链表
> 2. 判定树中，失败结点比成功结点多一个
> 3. `low > high` 是查找失败的终止条件
> 4. 折半查找的判定树**一定是平衡的**
