# 外部排序

## 📊 考情分析
- 考试频率：中频考点，每隔1~2年出现
- 分值：2~5分（选择题为主，偶尔综合应用题）
- 题型：选择题（归并趟数计算、磁盘I/O次数、多路归并优化）、综合应用题（归并过程分析）
- 真题考查方式：归并趟数与磁盘I/O次数计算、败者树、置换-选择排序、最佳归并树

## 一、基本概念

### 1. 严格定义
**外部排序**：当数据元素太多，无法一次全部读入内存时，需要利用外存（磁盘等）进行中间结果存储的排序方法。

### 2. 基本术语
- **外存（磁盘）**：大容量存储设备，数据以"**块**"为单位进行读写
- **内存**：小容量高速存储设备，排序过程在此进行
- **缓冲区**：内存中用于暂存磁盘数据的区域
- **归并段（顺串/初始归并段）**：经过内部排序后得到的有序子序列
- **磁盘块**：操作系统以"块"为单位管理磁盘存储空间（如每块1KB）
- **磁盘I/O**：磁盘的读写操作，是外部排序的主要时间开销

### 3. 外存与内存的数据交换
- 操作系统以"**块**"为单位对磁盘存储空间进行管理
- 磁盘的读写以"块"为单位
- 数据读入内存后才能被修改
- 修改完了还要写回磁盘
- 使用归并排序的方法，最少只需在内存中分配 **3块**大小的缓冲区即可对任意一个大文件进行排序

## 二、核心知识

### 1. 外部排序的基本原理

**核心思想**：利用归并排序的方法，通过多次"读入→内部排序→写出→归并"的过程完成排序。

**基本步骤**：
1. **构造初始归并段**：每次读入内存工作区能容纳的数据，进行内部排序后写回磁盘
2. **多趟归并**：将多个归并段两两（或多路）归并，直到只剩一个有序文件

### 2. 外部排序的完整过程

#### 第一步：构造初始归并段

```
磁盘上的原始数据（无序）：
[36, 8, 26, 42, 9, 48, 1, 37, 25, 45, 27, 28, 
 41, 2, 40, 19, 7, 39, 18, 38, 13, 14, 29, 44,
 12, 34, 24, 11, 3, 20, 30, 4, 31, 15, 5, 16,
 33, 23, 35, 22, 21, 10, 32, 6, 47, 43, 46, 17]

内存工作区大小：可以容纳3个块（每块6个记录）

处理过程：
1. 读入前6个记录 [36, 8, 26, 42, 9, 48]，内部排序 → [8, 9, 26, 36, 42, 48]，写回磁盘
2. 读入接下来6个 [1, 37, 25, 45, 27, 28]，内部排序 → [1, 25, 27, 28, 37, 45]，写回磁盘
... 以此类推

结果：8个有序的初始归并段
R1: [8, 9, 26, 36, 42, 48]
R2: [1, 25, 27, 28, 37, 45]
R3: [2, 7, 19, 39, 40, 41]
R4: [13, 14, 18, 29, 38, 44]
R5: [3, 11, 12, 24, 30, 34]
R6: [4, 5, 15, 16, 31, 20]
R7: [10, 21, 22, 23, 33, 35]
R8: [6, 17, 32, 43, 46, 47]
```

**构造初始归并段的I/O次数**：假设N个记录，内存工作区可以容纳L个记录
- 初始归并段数量 r = ⌈N/L⌉
- 需要 **r次读** 和 **r次写**（共2r次磁盘I/O）

#### 第二步：2路归并

```
第1趟归并（两两归并）：
R1 + R2 → R1'  (2个归并段归并为1个)
R3 + R4 → R2'
R5 + R6 → R3'
R7 + R8 → R4'

第2趟归并：
R1' + R2' → R1'' 
R3' + R4' → R2''

第3趟归并：
R1'' + R2'' → 有序文件
```

**2路归并的趟数**：对 r 个归并段做2路归并
- 趟数 = ⌈log₂r⌉

### 3. 外部排序的时间开销

$$\text{外部排序时间开销} = \text{读写外存的时间} + \text{内部排序所需时间} + \text{内部归并所需时间}$$

**关键结论**：**读写外存的时间是外部排序的主要时间开销！**

**示例计算**：
- 8个初始归并段，2路归并需要 ⌈log₂8⌉ = 3 趟
- 每趟需要读写所有数据各一次
- 总的读写次数 = 构造归并段(16次) + 归并过程(3×2×8 = 48次) = 64次
- 若用4路归并：只需 ⌈log₄8⌉ = 2 趟，读写次数 = 16 + 2×2×8 = 48次

### 4. 优化：多路归并

#### 为什么需要多路归并？

- **k路归并**：每次将 k 个归并段归并为1个
- k 越大，归并趟数越少，磁盘I/O次数越少
- 对 r 个初始归并段，做 k 路归并，归并趟数 = ⌈log_k r⌉

#### 多路归并的负面影响

1. **需要更多输入缓冲区**：k 路归并需要 k 个输入缓冲区，内存开销增加
2. **内部归并时间增加**：每挑选一个关键字需要比较关键字 (k-1) 次

#### 优化：败者树

**问题**：k路归并时，每选出一个最小元素需要比较 (k-1) 次

**解决方案**：使用**败者树**（Loser Tree）
- 败者树是一棵完全二叉树
- 可以将 k 路归并中每选出一个元素的比较次数降低到 **⌈log₂k⌉** 次
- 建立败者树的时间复杂度为 O(k)

### 5. 优化：减少初始归并段数量

#### 置换-选择排序（Replacement Selection Sort）

**核心思想**：利用更大的内存工作区，生成**更长的初始归并段**，从而减少归并段数量。

**基本原理**：
- 内存工作区可以容纳 L 个记录
- 使用置换-选择排序，可以生成平均长度为 **2L** 的初始归并段
- 初始归并段数量减少，归并趟数也减少

**结果**：若共 N 个记录，内存工作区可以容纳 L 个记录
- 传统方法：r = ⌈N/L⌉ 个初始归并段
- 置换-选择排序：r = ⌈N/(2L)⌉ 个初始归并段（大约减半）

### 6. 最佳归并树

**问题**：当各初始归并段长度不等时，如何安排归并顺序使总I/O次数最少？

**解决方案**：构造**最佳归并树**（类似哈夫曼树）
- 将每个归并段的长度作为权值
- 构造 k 叉哈夫曼树
- 按照哈夫曼树的结构进行归并

## 三、深入理解

### 1. 设计思想
- 外部排序的核心是**减少磁盘I/O次数**
- 磁盘I/O是瓶颈，内部计算时间可忽略
- 优化思路：
  1. **减少归并趟数**：多路归并
  2. **减少每趟的比较次数**：败者树
  3. **减少初始归并段数量**：置换-选择排序
  4. **优化归并顺序**：最佳归并树

### 2. 复杂度分析

#### 2路归并的磁盘I/O次数

假设 N 个记录，内存工作区可以容纳 L 个记录：

| 步骤 | 磁盘I/O次数 |
|------|------------|
| 构造初始归并段 | 2 × ⌈N/L⌉（读+写） |
| 每趟归并 | 2 × ⌈N/L⌉（读+写） |
| 归并趟数 | ⌈log₂⌈N/L⌉⌉ |
| **总I/O次数** | **2 × ⌈N/L⌉ × (1 + ⌈log₂⌈N/L⌉⌉)** |

#### k路归并的磁盘I/O次数

| 步骤 | 磁盘I/O次数 |
|------|------------|
| 归并趟数 | ⌈log_k r⌉，其中 r = ⌈N/L⌉ |
| **总I/O次数** | **2 × r × (1 + ⌈log_k r⌉)** |

**k 越大，归并趟数越少，I/O次数越少**

### 3. k路平衡归并

**定义**：
1. 最多只能有 k 个段归并为一个
2. 每一趟归并中，若有 m 个归并段参与归并，则经过这一趟处理得到 ⌈m/k⌉ 个新的归并段

**注意**：k路归并 ≠ k路平衡归并！
- k路归并：每次可以从 ≤ k 个段中选择归并
- k路平衡归并：每次必须恰好从 k 个段中归并（最后一轮可能不足k个）

## 四、真题考法分析

### 常考题型
1. **归并趟数计算**：给定 r 个归并段和 k 路归并，计算趟数
2. **磁盘I/O次数计算**：计算总读写次数
3. **败者树的使用**：理解败者树如何减少比较次数
4. **置换-选择排序**：理解如何生成更长的初始归并段
5. **最佳归并树**：构造最优的归并顺序
6. **k路平衡归并的判断**

### 命题规律
- 归并趟数 = ⌈log_k r⌉ 这个公式是必考点
- 磁盘I/O次数的计算是高频考点
- 败者树和置换-选择排序是难点
- 最佳归并树类似于哈夫曼树，需要掌握构造方法

## 五、典型例题

### 例1：归并趟数计算
**题目**：设有16个初始归并段，进行4路归并排序，需要几趟归并？

**答案**：
$$\lceil \log_4 16 \rceil = \lceil 2 \rceil = 2 \text{ 趟}$$

### 例2：磁盘I/O次数计算
**题目**：有10000个记录，内存工作区可容纳500个记录，采用2路归并排序。问总共需要多少次磁盘I/O？

**答案**：
```
初始归并段数量 r = ⌈10000/500⌉ = 20
构造初始归并段的I/O = 2 × 20 = 40次（读+写）
归并趟数 = ⌈log₂20⌉ = 5趟
每趟归并的I/O = 2 × 20 = 40次
总I/O次数 = 40 + 5 × 40 = 240次
```

### 例3：多路归并优化
**题目**：上题中若改为4路归并，总I/O次数是多少？

**答案**：
```
归并趟数 = ⌈log₄20⌉ = ⌈2.16⌉ = 3趟
总I/O次数 = 40 + 3 × 40 = 160次
```
比2路归并减少了80次I/O！

### 例4：败者树
**题目**：在8路归并中，使用败者树每选出一个最小元素需要比较几次？

**答案**：
- 不用败者树：需要 8-1 = 7 次比较
- 用败者树：只需要 ⌈log₂8⌉ = 3 次比较

### 例5：最佳归并树
**题目**：有5个初始归并段，长度分别为 2, 3, 5, 7, 11，进行2路归并。请构造最佳归并树并计算总的读写记录数。

**答案**：
```
类似哈夫曼树构造：
Step 1: 选最小的2和3合并 → 5     序列：[5, 5, 7, 11]
Step 2: 选最小的5和5合并 → 10    序列：[7, 10, 11]
Step 3: 选最小的7和10合并 → 17   序列：[11, 17]
Step 4: 选最小的11和17合并 → 28  序列：[28]

总读写记录数 = 2 × (5 + 10 + 17 + 28) = 2 × 60 = 120次记录读写
```

### 例6：k路平衡归并判断
**题目**：有8个归并段 R1~R8，进行4路归并。第1趟将 R1~R4 归并为 R1'，R5~R6 归并为 R2'，R7~R8 归并为 R3'。这是4路平衡归并吗？

**答案**：**不是**。
- 4路平衡归并要求每趟中，有 m 个归并段参与归并时，得到 ⌈m/4⌉ 个新归并段
- 8个段参与，应该得到 ⌈8/4⌉ = 2 个新归并段
- 但这里得到了3个 → 不是4路平衡归并

## 六、易错点

1. **归并趟数公式**：是 ⌈log_k r⌉ 不是 ⌊log_k r⌋，注意向上取整
2. **初始归并段数量**：r = ⌈N/L⌉，不是 N/L
3. **磁盘I/O的计算**：构造归并段和归并过程都要计算读写
4. **多路归并的折中**：k越大归并趟数越少，但每趟的比较次数增加
5. **败者树的作用**：减少内部归并的比较次数，不是减少I/O次数
6. **置换-选择排序的作用**：减少初始归并段数量，不是减少每段的长度
7. **k路归并 ≠ k路平衡归并**：平衡归并要求每趟都均匀归并
8. **3块缓冲区**：最少只需3块缓冲区（2个输入+1个输出）即可完成外部排序
