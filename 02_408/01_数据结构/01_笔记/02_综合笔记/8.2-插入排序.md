# 插入排序（直接插入排序与折半插入排序）

## 📊 考情分析
- **考试频率**：高频考点，每年必考
- **分值**：2-4分（选择题）或10分（算法设计题）
- **题型**：选择题（复杂度分析、稳定性判断）、算法题（手写代码）
- **真题考查方式**：给出序列要求模拟排序过程、分析时间复杂度、折半插入排序的比较次数

## 一、基本概念

### 插入排序的核心思想
每次将一个待排序的记录按其关键字大小插入到前面**已排好序的子序列**中，直到全部记录插入完成。

### 基本术语
- **有序子序列**：已经排好序的部分
- **待排序元素**：当前需要插入的元素
- **哨兵**：使用 $A[0]$ 作为临时存储，简化边界判断

## 二、核心知识

### 1. 直接插入排序

#### 算法思想
对于 $i = 2, 3, \dots, n$，将 $A[i]$ 插入到前面已排好序的子序列 $A[1..i-1]$ 中。

#### 排序过程示例
初始序列：`49, 38, 65, 97, 76, 13, 27, 49`（从下标1开始）

| 趟数 | 有序子序列 | 插入元素 | 结果 |
|------|-----------|---------|------|
| 第1趟 | [49] | 38 | [38, 49] |
| 第2趟 | [38, 49] | 65 | [38, 49, 65] |
| 第3趟 | [38, 49, 65] | 97 | [38, 49, 65, 97] |
| 第4趟 | [38, 49, 65, 97] | 76 | [38, 49, 65, 76, 97] |
| 第5趟 | [38, 49, 65, 76, 97] | 13 | [13, 38, 49, 65, 76, 97] |
| 第6趟 | [13, 38, 49, 65, 76, 97] | 27 | [13, 27, 38, 49, 65, 76, 97] |
| 第7趟 | [13, 27, 38, 49, 65, 76, 97] | 49 | [13, 27, 38, 49, 49, 65, 76, 97] |

#### 代码实现（带哨兵）

```c
void InsertSort(int A[], int n) {
    int i, j, temp;
    for (i = 2; i <= n; i++) {         // 从第2个元素开始
        if (A[i] < A[i-1]) {           // 需要插入
            A[0] = A[i];               // 哨兵暂存
            for (j = i - 1; A[0] < A[j]; j--)  // 从后往前找插入位置
                A[j+1] = A[j];         // 元素后移
            A[j+1] = A[0];             // 插入到正确位置
        }
    }
}
```

#### 代码实现（不带哨兵）

```c
void InsertSort(int A[], int n) {
    int i, j, temp;
    for (i = 1; i < n; i++) {
        temp = A[i];
        for (j = i - 1; j >= 0 && A[j] > temp; j--)
            A[j+1] = A[j];
        A[j+1] = temp;
    }
}
```

### 2. 折半插入排序

#### 算法思想
先用**折半查找**找到应该插入的位置，再移动元素。

#### 与直接插入排序的区别
| 对比项 | 直接插入排序 | 折半插入排序 |
|--------|------------|------------|
| 查找方式 | 顺序查找 | 折半查找 |
| 比较次数 | 最多 $i-1$ 次 | $\lceil \log_2 i \rceil$ 次 |
| 移动次数 | 不变 | 不变 |
| 时间复杂度 | $O(n^2)$ | $O(n^2)$（移动操作仍是瓶颈）|

#### 代码实现

```c
void HalfInsertSort(int A[], int n) {
    int i, j, low, high, mid, temp;
    for (i = 2; i <= n; i++) {
        A[0] = A[i];
        low = 1; high = i - 1;
        while (low <= high) {          // 折半查找
            mid = (low + high) / 2;
            if (A[mid] > A[0])
                high = mid - 1;
            else
                low = mid + 1;         // 相等时也往右找（保证稳定性）
        }
        for (j = i - 1; j >= low; j--) // 移动元素
            A[j+1] = A[j];
        A[low] = A[0];                 // 插入
    }
}
```

#### 关键细节：稳定性保证
当 $A[mid] == A[0]$ 时，为了保证算法的**稳定性**，应继续在 mid 所指位置**右边**寻找插入位置（即 `low = mid + 1`）。

## 三、深入理解

### 复杂度分析

#### 直接插入排序
- **空间复杂度**：$O(1)$（原地排序）
- **最好时间复杂度**：$O(n)$（原本有序，每趟只比较1次）
- **最坏时间复杂度**：$O(n^2)$（原本逆序）
- **平均时间复杂度**：$O(n^2)$

**最坏情况推导**：
- 第1趟：比较2次，移动3次
- 第2趟：比较3次，移动4次
- 第i趟：比较 $i+1$ 次，移动 $i+2$ 次
- 第 $n-1$ 趟：比较 $n$ 次，移动 $n+1$ 次
- 总比较次数：$2+3+\dots+n = \frac{(n+2)(n-1)}{2} = O(n^2)$

#### 折半插入排序
- **比较次数**：$O(n\log_2 n)$（每趟用折半查找，比较 $\log_2 i$ 次）
- **移动次数**：$O(n^2)$（移动操作不变）
- **总时间复杂度**：$O(n^2)$

### 设计思想
- 直接插入排序利用了"部分有序"的特性
- 折半插入排序用折半查找减少比较次数，但移动次数不变

## 四、真题考法分析

### 常考题型
1. 给出初始序列，要求模拟第k趟后的结果
2. 分析折半插入排序的比较次数
3. 判断排序算法的稳定性

### 命题规律
- 折半插入排序的比较次数与直接插入排序不同，但时间复杂度相同
- 常考"折半插入排序减少的是什么操作"

## 五、典型例题

### 概念题
**Q1**：折半插入排序相比直接插入排序，减少的是（ ）
A. 关键字的比较次数  B. 记录的移动次数  C. 辅助空间  D. 时间复杂度
**答案**：A

**Q2**：直接插入排序在最好情况下的时间复杂度为（ ）
A. $O(n)$  B. $O(n\log_2 n)$  C. $O(n^2)$  D. $O(1)$
**答案**：A

### 计算题
**Q3**：对序列 `49, 38, 65, 97, 76, 13, 27, 49` 进行直接插入排序（带哨兵），写出第5趟结束后前6个元素的值。
**答案**：`13, 38, 49, 65, 76, 97`

### 算法题
**Q4**：编写折半插入排序算法，要求保证排序的稳定性。
**答案**：见上文代码实现，关键是当 $A[mid] == A[0]$ 时 `low = mid + 1`。

## 六、易错点

1. **哨兵的作用**：使用 $A[0]$ 作为哨兵可以省去每轮循环的 `j>=0` 判断
2. **折半插入排序时间复杂度仍是 $O(n^2)$**：虽然比较次数减少，但移动次数不变
3. **稳定性**：直接插入和折半插入都是**稳定**的
4. **下标问题**：带哨兵时数组从 $A[1]$ 开始，不带哨兵时从 $A[0]$ 开始
5. **折半插入中相等时的处理**：必须往右找才能保证稳定性
