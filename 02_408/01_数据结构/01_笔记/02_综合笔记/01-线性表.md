## 一、基本概念

### 1.1 线性表的定义

**线性表**（Linear List）是具有**相同数据类型**的 n（n ≥ 0）个数据元素的**有限序列**。

数学表示：

$$
L = (a_1, a_2, \ldots, a_i, a_{i+1}, \ldots, a_n)
$$

其中：
- $n$ 为表长，$n=0$ 时称为**空表**
- $a_i$ 是第 $i$ 个元素（位序从1开始，非下标）
- $a_1$ 为**表头元素**（首元素），$a_n$ 为**表尾元素**（尾元素）
- 对任意 $a_i$（$1 < i < n$），$a_{i-1}$ 是 $a_i$ 的**直接前驱**，$a_{i+1}$ 是 $a_i$ 的**直接后继**
- $a_1$ 无前驱，$a_n$ 无后继

### 1.2 基本术语

| 术语 | 含义 |
|------|------|
| **表长** | 线性表中元素个数 $n$ |
| **空表** | $n=0$ 的线性表 |
| **位序** | 元素在表中的位置（从1开始计） |
| **前驱/后继** | 逻辑上的前后关系 |
| **存储结构** | 数据在内存中的实际组织方式 |

### 1.3 线性表的分类

按存储方式分类：

| 类型 | 存储方式 | 随机访问 | 插入删除 |
|------|----------|----------|----------|
| **顺序表** | 顺序存储（数组） | ✅ O(1) | ❌ O(n) |
| **单链表** | 链式存储 | ❌ O(n) | ✅ O(1)\* |
| **双链表** | 链式存储（双向） | ❌ O(n) | ✅ O(1)\* |
| **循环链表** | 链式存储（首尾相接） | ❌ O(n) | ✅ O(1)\* |
| **静态链表** | 顺序存储模拟链表 | ❌ O(n) | ✅ O(1)\* |

> \*：已知节点位置时的插入删除为 O(1)，查找位置仍需 O(n)。

## 二、核心知识

### 2.1 顺序表

#### 存储结构

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 静态分配
    int length;              // 当前长度
} SqList;
```

动态分配版本：

```c
#define InitSize 100
typedef struct {
    ElemType *data;    // 动态分配数组指针
    int MaxSize;       // 最大容量
    int length;        // 当前长度
} SeqList;

// 初始化
void InitList(SeqList *L) {
    L->data = (ElemType *)malloc(InitSize * sizeof(ElemType));
    L->length = 0;
    L->MaxSize = InitSize;
}

// 动态扩展
void IncreaseSize(SeqList *L, int len) {
    ElemType *p = L->data;
    L->MaxSize += len;
    L->data = (ElemType *)malloc(L->MaxSize * sizeof(ElemType));
    for (int i = 0; i < L->length; i++)
        L->data[i] = p[i];  // 复制数据
    free(p);
}
```

#### 基本操作及时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| **按位查找** GetElem(L, i) | **O(1)** | `L.data[i-1]`，随机访问 |
| **按值查找** LocateElem(L, e) | **O(n)** | 顺序扫描 |
| **插入** ListInsert(L, i, e) | **O(n)** | 最好O(1)，最坏O(n)，平均O(n/2)=O(n) |
| **删除** ListDelete(L, i, e) | **O(n)** | 最好O(1)，最坏O(n)，平均O(n/2)=O(n) |
| **头插/尾插** | **O(1)/O(1)** | 直接操作 |
| **求表长** | **O(1)** | 返回 length |

**插入操作详解**：

```c
bool ListInsert(SqList *L, int i, ElemType e) {
    if (i < 1 || i > L->length + 1)  // 位序合法性检查
        return false;
    if (L->length >= MaxSize)          // 存储空间已满
        return false;
    for (int j = L->length; j >= i; j--)  // 后移元素
        L->data[j] = L->data[j-1];
    L->data[i-1] = e;                 // 插入（注意下标转换）
    L->length++;
    return true;
}
```

> 在位置 $i$ 插入需移动 $n - i + 1$ 个元素。假设在每个位置插入的概率相等（$p_i = \frac{1}{n+1}$），则平均移动次数为：
> $$E_{insert} = \sum_{i=1}^{n+1} \frac{1}{n+1}(n-i+1) = \frac{n}{2}$$

**删除操作详解**：

```c
bool ListDelete(SqList *L, int i, ElemType *e) {
    if (i < 1 || i > L->length)
        return false;
    *e = L->data[i-1];               // 返回被删元素
    for (int j = i; j < L->length; j++)  // 前移元素
        L->data[j-1] = L->data[j];
    L->length--;
    return true;
}
```

> 删除位置 $i$ 需移动 $n - i$ 个元素，平均移动次数为：
> $$E_{delete} = \sum_{i=1}^{n} \frac{1}{n}(n-i) = \frac{n-1}{2}$$

### 2.2 单链表

#### 存储结构

```c
typedef struct LNode {
    ElemType data;         // 数据域
    struct LNode *next;    // 指针域
} LNode, *LinkList;
```

> `LNode *p` 强调 p 是一个**节点**；`LinkList L` 强调 L 是一个**链表**（头指针）。本质相同，都是指针。

#### 头节点 vs 头指针

| | 头节点 | 头指针 |
|---|--------|--------|
| 是否存储数据 | 通常不存（data无意义） | — |
| 是否必须 | 可选（推荐带头节点） | 必须 |
| 空表判断 | `L->next == NULL` | 取决于是否带头节点 |

> **带头节点的好处**：统一了空表和非空表的操作，无需对第一个节点做特殊处理。

#### 基本操作

**头插法建立链表**（结果**逆序**）：

```c
LinkList List_HeadInsert(LinkList L) {
    L = (LinkList)malloc(sizeof(LNode));
    L->next = NULL;           // 初始化空表
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {       // 输入结束标志
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        s->next = L->next;    // 新节点指向原首节点
        L->next = s;          // 头节点指向新节点
        scanf("%d", &x);
    }
    return L;
}
```

**尾插法建立链表**（结果**正序**）：

```c
LinkList List_TailInsert(LinkList L) {
    L = (LinkList)malloc(sizeof(LNode));
    L->next = NULL;
    LNode *r = L;             // r始终指向尾节点
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        r->next = s;
        r = s;                // r指向新的尾节点
        scanf("%d", &x);
    }
    r->next = NULL;           // 尾节点指针置空
    return L;
}
```

**按位查找**：

```c
LNode *GetElem(LinkList L, int i) {
    if (i < 0) return NULL;
    LNode *p = L;             // 从头节点开始
    int j = 0;
    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }
    return p;                 // i=0返回头节点
}
```

**插入操作**（在第 i 个位置插入）：

```c
bool ListInsert(LinkList L, int i, ElemType e) {
    LNode *p = GetElem(L, i - 1);  // 找到第i-1个节点
    if (p == NULL) return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    s->data = e;
    s->next = p->next;        // ① 先连后
    p->next = s;              // ② 再连前
    return true;
}
```

> 必须先 `s->next = p->next`，再 `p->next = s`。如果先断开 `p->next`，后面就找不到原后继了。

**删除操作**（删除第 i 个节点）：

```c
bool ListDelete(LinkList L, int i, ElemType *e) {
    LNode *p = GetElem(L, i - 1);
    if (p == NULL || p->next == NULL)
        return false;
    LNode *q = p->next;       // q指向待删节点
    *e = q->data;
    p->next = q->next;        // 断开q
    free(q);
    return true;
}
```

#### 时间复杂度汇总

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按位查找 | O(n) | 需要从头遍历 |
| 按值查找 | O(n) | 顺序扫描 |
| 插入（已知前驱） | O(1) | 直接修改指针 |
| 删除（已知前驱） | O(1) | 直接修改指针 |
| 头插 | O(1) | — |
| 尾插（有尾指针） | O(1) | 需要尾指针 |
| 尾插（无尾指针） | O(n) | 需要遍历到尾部 |

### 2.3 双向链表

#### 存储结构

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior;    // 前驱指针
    struct DNode *next;     // 后继指针
} DNode, *DLinkList;
```

#### 插入操作

在节点 p 之后插入节点 s：

```c
s->next = p->next;      // ①
p->next->prior = s;     // ②
s->prior = p;           // ③
p->next = s;            // ④
```

> 四步顺序：①②可以交换，③④可以交换，但 ①② 和 ③④ 不能交叉。

#### 删除操作

删除节点 p 的后继节点 q：

```c
p->next = q->next;      // ①
q->next->prior = p;     // ②
free(q);
```

> 注意：当 q 是尾节点时，`q->next` 为 NULL，第 ② 步要特殊处理（带头节点时不用，因为循环双向链表 tail->next = head）。

### 2.4 循环链表

#### 循环单链表

- 最后一个节点的 `next` 指向头节点（而非 NULL）
- **判空条件**：`L->next == L`（带头节点）
- **判尾条件**：`p->next == L`

#### 循环双链表

- 尾节点的 `next` 指向头节点，头节点的 `prior` 指向尾节点
- **判空条件**：`L->next == L && L->prior == L`
- 优点：从任意节点出发都能遍历整个链表

### 2.5 静态链表

用**数组**模拟链表，每个数组元素包含 `data` 和 `next`（下一个元素的下标）。

```c
#define MaxSize 100
typedef struct {
    ElemType data;
    int next;               // 下一个元素的数组下标
} SLinkList[MaxSize];
```

- 优点：不使用指针，适用于不支持指针的语言
- 缺点：容量固定，不能动态增长

## 三、深入理解

### 3.1 顺序表 vs 链表的全面对比

| 维度 | 顺序表 | 链表 |
|------|--------|------|
| 存储方式 | 连续存储 | 离散存储 |
| 随机访问 | ✅ O(1) | ❌ O(n) |
| 插入/删除 | O(n)需移动元素 | O(1)改指针（已知位置） |
| 空间利用 | 可能浪费（预分配）或溢出 | 按需分配，但有指针开销 |
| 缓存友好 | ✅ 高（连续内存） | ❌ 低（分散内存） |
| 适用场景 | 频繁查找、已知大小 | 频繁增删、大小不确定 |

### 3.2 算法设计技巧

#### 链表逆置（头插法思想）

```c
void Reverse(LinkList L) {
    LNode *p = L->next;
    L->next = NULL;
    while (p != NULL) {
        LNode *q = p->next;    // 保存后继
        p->next = L->next;     // 头插
        L->next = p;
        p = q;
    }
}
```

时间 O(n)，空间 O(1)。

#### 查找倒数第 k 个节点（双指针法）

```c
LNode *FindKthFromEnd(LinkList L, int k) {
    LNode *fast = L->next, *slow = L->next;
    for (int i = 0; i < k; i++) {
        if (fast == NULL) return NULL;
        fast = fast->next;
    }
    while (fast != NULL) {
        fast = fast->next;
        slow = slow->next;
    }
    return slow;
}
```

#### 查找中间节点（快慢指针法）

```c
LNode *FindMiddle(LinkList L) {
    LNode *fast = L->next, *slow = L->next;
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
    }
    return slow;  // 偶数个节点时返回第 n/2 个
}
```

#### 判断链表是否有环（Floyd判圈法）

```c
bool HasCycle(LinkList L) {
    LNode *fast = L->next, *slow = L->next;
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;
    }
    return false;
}
```

### 3.3 复杂度分析方法

- **顺序表操作**：核心在于**移动元素**的次数
- **链表操作**：核心在于**遍历节点**的次数
- 空间复杂度：原地操作指需 O(1) 额外空间（几个指针变量）

### 3.4 与其他知识的联系

- 线性表是[[02_408/01_数据结构/01_笔记/02_综合笔记/02-栈和队列]]的基础
- 链表结构用于[[02_408/01_数据结构/01_笔记/02_综合笔记/03-树和二叉树]]的链式存储
- 邻接表是[[02_408/01_数据结构/01_笔记/02_综合笔记/04-图]]的重要存储方式，本质是链表

## 四、真题考法分析

### 4.1 常考具体考点

1. **顺序表插入/删除的平均移动次数计算**
2. **带头节点vs不带头节点链表的操作差异**
3. **头插法/尾插法建表的元素顺序**
4. **循环链表的判空条件**
5. **双向链表插入/删除的指针操作顺序**
6. **链表算法设计**（逆置、合并、去重、查找公共节点等）

### 4.2 命题规律和趋势

- 选择题：偏爱考**概念辨析**（如带头节点的好处）和**复杂度判断**
- 大题：链表操作题是**最稳定的算法设计题来源**，几乎每隔一年出现
- 近年趋势：要求**时间O(n)空间O(1)**的链表操作越来越多

### 4.3 选择题常见陷阱

1. **位序 vs 下标**：位序从1开始，下标从0开始
2. **头节点的data域**：头节点的data一般无意义，不要用它存数据
3. **插入操作的顺序**：单链表插入必须先连后继再连前驱
4. **带头节点的空表判断**：`L->next == NULL`，不是 `L == NULL`
5. **循环链表判空**：不是 `p->next == NULL`，而是 `p->next == 头节点`

## 五、典型例题

### 例题1：概念理解题（选择题风格）

**题目**：设一个带头节点的单链表 L，判断该链表为空的条件是（  ）
A. `L == NULL`
B. `L->next == NULL`
C. `L->next == L`
D. `L != NULL && L->next == NULL`

**思路分析**：头节点不存数据，空表时头节点存在但其next为NULL。

**详细解答**：
- A错误：L是头指针，始终不为NULL（已分配节点）
- B正确：头节点的next为NULL说明没有数据节点
- C错误：这是循环链表的判空条件
- D多余且不精确

**答案：B**

**总结归纳**：带头节点的单链表，空表即"只有头节点、无数据节点"，判空为 `L->next == NULL`。

---

### 例题2：计算分析题（填空题风格）

**题目**：一个长度为 n 的顺序表，在位置 i（1 ≤ i ≤ n+1）插入一个元素，平均需要移动 ___ 个元素；删除位置 i（1 ≤ i ≤ n）的元素，平均需要移动 ___ 个元素。

**思路分析**：计算等概率下各位置操作的移动次数期望值。

**详细解答**：

**插入**：在位置 i 插入需移动 $n - i + 1$ 个元素。

$$E_{insert} = \frac{1}{n+1}\sum_{i=1}^{n+1}(n-i+1) = \frac{1}{n+1} \cdot \frac{n(n+1)}{2} = \frac{n}{2}$$

**删除**：删除位置 i 需移动 $n - i$ 个元素。

$$E_{delete} = \frac{1}{n}\sum_{i=1}^{n}(n-i) = \frac{1}{n} \cdot \frac{n(n-1)}{2} = \frac{n-1}{2}$$

**答案：n/2 ；(n-1)/2**

**总结归纳**：插入有 n+1 个可选位置，删除有 n 个可选位置。两个公式容易记混，注意分母不同。

### 例题3：算法设计题（大题风格）

**题目**：给定一个带头节点的单链表 L，设计一个算法将所有元素逆置，要求时间复杂度 O(n)，空间复杂度 O(1)。

**思路分析**：
方法一：**头插法重建**——依次摘下每个节点，用头插法重新插入。
方法二：**就地反转指针**——遍历链表，逐个反转 next 指针。

**详细解答**（方法一：头插法）：

```c
void ReverseList(LinkList L) {
    LNode *p = L->next;      // p指向第一个数据节点
    L->next = NULL;           // 断开头节点
    while (p != NULL) {
        LNode *q = p->next;  // 保存后继（防止断链）
        p->next = L->next;   // 头插：新节点指向原首节点
        L->next = p;         // 头节点指向新节点
        p = q;               // 处理下一个
    }
}
```

**详细解答**（方法二：三指针法）：

```c
void ReverseList2(LinkList L) {
    LNode *prev = NULL;
    LNode *curr = L->next;
    while (curr != NULL) {
        LNode *next = curr->next;  // 保存后继
        curr->next = prev;         // 反转指针
        prev = curr;               // 前进
        curr = next;
    }
    L->next = prev;  // 头节点指向原尾节点
}
```

- 时间复杂度：O(n)，遍历一次
- 空间复杂度：O(1)，只用了固定几个指针

**总结归纳**：链表逆置的核心思想是**改变指针方向**。头插法更直观，三指针法更灵活。两种方法都是O(n)时间O(1)空间，考试中写哪种都可以。

### 例题4：历年真题（2019年408）

**题目**：（2019年408第41题，13分）设线性表 L = (a₁, a₂, a₃, ..., aₙ) 采用带头节点的单链表存储。请设计一个尽可能高效的算法，将链表中所有元素逆序排列，即将 (a₁, a₂, ..., aₙ) 变为 (aₙ, aₙ₋₁, ..., a₁)。要求：
（1）给出算法的基本设计思想
（2）根据设计思想，采用C或C++语言描述算法
（3）说明你所设计算法的时间复杂度和空间复杂度

**思路分析**：此题即为链表逆置的标准题。核心要求"尽可能高效"即要求 O(n) 时间 O(1) 空间。

**详细解答**：

（1）基本设计思想：
遍历链表，依次将每个节点从原位置摘下，采用**头插法**插入到头节点之后。遍历完成后，所有节点的顺序即被反转。

（2）算法实现：

```c
void Reverse(LinkList L) {
    LNode *p = L->next;       // p指向第一个数据节点
    L->next = NULL;            // 初始化为空表
    while (p != NULL) {
        LNode *q = p->next;   // 用q保存p的后继
        p->next = L->next;    // 将p插入到头节点之后
        L->next = p;
        p = q;                // 继续处理下一个节点
    }
}
```

（3）复杂度分析：
- **时间复杂度**：O(n)。算法对链表进行一次遍历，每个节点的处理时间是 O(1)。
- **空间复杂度**：O(1)。仅使用了两个辅助指针 p 和 q。

**总结归纳**：此类题的得分关键在于：① 写清楚设计思想；② 代码简洁正确；③ 复杂度分析有理有据。头插法逆置是最标准的解法，务必熟练默写。

## 六、易错点

> 链表操作中第 i 个位置对应的是**位序**（从1开始），而数组中用的是**下标**（从0开始）。`GetElem(L, i)` 找的是位序 i 的元素，实际对应下标 i-1。

> `s->next = p->next; p->next = s;` 这两步**不能交换**！先断开 p->next 会导致原后继丢失。

> 带头节点空表判 `L->next == NULL`，不是 `L == NULL`。不带头节点才判 `L == NULL`。

> 循环链表中 `p->next != NULL` **永远不会成立**（因为循环！）。应写 `p->next != L`（带头节点）或 `p->next != L`（头指针）。

> 链表逆置、删除等操作中，修改指针前**必须先用临时变量保存后续节点**的地址，否则会断链。

> 四步操作：① `s->next=p->next` ② `p->next->prior=s` ③ `s->prior=p` ④ `p->next=s`。其中①②必须在③④之前执行，否则会丢失对原后继的访问。

### 数据结构基础概念补充

#### 核心术语辨析

| 术语 | 定义 | 关键区分 |
|------|------|----------|
| **数据** | 信息的载体，描述客观事物属性的数、字符及所有能输入到计算机中并被程序识别和处理的符号集合 | 是程序加工的原料 |
| **数据元素** | 数据的**基本单位**，通常作为一个整体进行考虑和处理 | 一个数据元素可由若干数据项组成 |
| **数据项** | 构成数据元素的**不可分割的最小单位** | 是数据结构中最小的标识单位 |
| **数据对象** | 具有**相同性质**的数据元素的集合，是数据的一个子集 | 不要求元素间有特定关系 |
| **数据结构** | 相互之间存在**一种或多种特定关系**的数据元素的集合 | 重点在元素之间的**关系** |

> 数据结构这门课着重关注的是**数据元素之间的关系**和**对这些数据元素的操作**，而不关心具体的数据项内容。线性表就是一种典型的数据结构——数据元素之间是一对一的线性关系。

#### 数据结构的三要素

讨论任何一种数据结构时，需要关注三个方面：

**1. 逻辑结构** —— 数据元素之间的逻辑关系

| 类型 | 关系 | 对应章节 |
|------|------|----------|
| **集合** | 各元素同属一个集合，别无其他关系 | — |
| **线性结构** | 一对一关系，除首元素外每个元素有唯一前驱，除尾元素外每个元素有唯一后继 | 第2-3章（线性表、栈、队列） |
| **树形结构** | 一对多关系 | 第4-5章（树、图） |
| **图结构** | 多对多关系 | 第5章 |

> **线性表的逻辑结构**：数据元素之间是一对一的关系，除了第一个元素外所有元素都有唯一前驱，除了最后一个元素外所有元素都有唯一后继。

**2. 物理结构（存储结构）** —— 如何用计算机表示数据元素的逻辑关系

| 存储方式 | 描述 | 特点 |
|----------|------|------|
| **顺序存储** | 逻辑上相邻的元素存储在物理位置上也相邻的存储单元中 | 物理上必须连续；方便随机访问 |
| **链式存储** | 逻辑上相邻的元素在物理位置上可以不相邻，借助指针表示逻辑关系 | 物理上可以离散；需要额外指针空间 |
| **索引存储** | 存储元素信息的同时建立索引表（关键字，地址） | 便于快速查找 |
| **散列存储** | 根据元素关键字直接计算出存储地址（Hash） | 查找效率高，但可能冲突 |

> 1. 若采用顺序存储，则各数据元素在物理上**必须是连续的**；若采用非顺序存储，则各数据元素在物理上**可以是离散的**
> 2. 数据的存储结构**会影响**存储空间分配的方便程度
> 3. 数据的存储结构**会影响**对数据运算的速度
> 4. 顺序表采用顺序存储，单链表采用链式存储——同一逻辑结构可以有不同的存储结构

**3. 数据的运算** —— 施加在数据上的运算

- **运算的定义**是针对**逻辑结构**的，指出运算的功能（如：在线性表中插入元素、删除元素）
- **运算的实现**是针对**存储结构**的，指出运算的具体操作步骤（如：顺序表插入需要移动元素，链表插入需要修改指针）

#### 数据类型与抽象数据类型（ADT）

| 概念 | 定义 |
|------|------|
| **数据类型** | 一个**值的集合**和定义在此集合上的一组**操作**的总称 |
| **原子类型** | 其值不可再分的数据类型（如 int、bool、float） |
| **结构类型** | 其值可以再分解为若干分量的数据类型（如 struct） |
| **抽象数据类型（ADT）** | 用数学化的语言定义数据的逻辑结构、定义运算，与具体实现无关 |

> **程序 = 数据结构 + 算法**
> - 数据结构：如何用数据正确地描述现实世界的问题，并存入计算机
> - 算法：如何高效地处理这些数据，以解决实际问题

#### 算法的五大特性

| 特性 | 含义 |
|------|------|
| **有穷性** | 算法必须在执行有穷步之后结束，每一步在有穷时间内完成 |
| **确定性** | 每条指令必须有确切的含义，对于相同的输入只能得出相同的输出 |
| **可行性** | 算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现 |
| **输入** | 一个算法有零个或多个输入 |
| **输出** | 一个算法有一个或多个输出 |

#### 时间复杂度速查表

| 复杂度 | 名称 | 典型算法示例 |
|--------|------|-------------|
| $O(1)$ | 常数阶 | 顺序表按位查找、直接计算 |
| $O(\log_2 n)$ | 对数阶 | 折半查找 |
| $O(n)$ | 线性阶 | 顺序表遍历、链表遍历 |
| $O(n\log_2 n)$ | 线性对数阶 | 快速排序、归并排序 |
| $O(n^2)$ | 平方阶 | 冒泡排序、选择排序 |
| $O(n^3)$ | 立方阶 | Floyd最短路径 |
| $O(2^n)$ | 指数阶 | 递归求斐波那契数列 |

> **加法规则**：$O(f(n)) + O(g(n)) = O(\max(f(n), g(n)))$
> **乘法规则**：$O(f(n)) \times O(g(n)) = O(f(n) \times g(n))$

> **数据结构与线性表的关系**：线性表是最基本的数据结构，它的逻辑结构（一对一关系）和存储结构（顺序存储/链式存储）是后续学习栈、队列、串等结构的基础。
