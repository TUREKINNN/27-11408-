# 串的模式匹配

> [[01_数学一/01_高等数学/01_笔记/09_数一笔记_高数/索引]]

## 概念

**模式匹配（Pattern Matching）**：在主串中查找子串（模式串）的过程。

- **主串 S**：被搜索的字符串
- **模式串 T**：待查找的字符串
- **匹配成功**：返回 T 在 S 中的位置

## 朴素模式匹配算法（BF 算法）

### 思想

从主串 S 的每一个字符开始，逐个与模式串 T 比较。

### 步骤

```
① i = 0（主串指针），j = 0（模式串指针）
② 比较 S[i] 和 T[j]：
   - 相等：i++, j++，继续比较
   - 不等：i 回退到 i - j + 1，j 重置为 0
③ 若 j == len(T)，匹配成功，返回 i - j
```

### 时间复杂度

- 最坏：O(n × m)
- 平均：O(n × m)
- 最好：O(n)

### C 代码

```c
int naiveMatch(char *S, char *T) {
    int i = 0, j = 0;
    while (S[i] != '\0' && T[j] != '\0') {
        if (S[i] == T[j]) {
            i++; j++;
        } else {
            i = i - j + 1;
            j = 0;
        }
    }
    if (T[j] == '\0') return i - j;
    return -1;
}
```

## KMP 算法

### 核心思想

利用已经部分匹配的信息，避免回退主串指针。

### 关键概念：next 数组

**next[j]** = 当模式串第 j 个字符匹配失败时，模式串指针 j 应该回退到的位置。

### next 数组计算

```
next[0] = -1（固定）
next[j] = k  ⟹  T[0..k-1] == T[j-k..j-1]  且  T[k] != T[j]
```

#### 手动计算示例

模式串 T = "abaabc"

| j | 0 | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|---|
| T[j] | a | b | a | a | b | c |
| next[j] | -1 | 0 | 0 | 1 | 1 | 2 |

**解释**：
- j=0: -1
- j=1: b 前面没有相同前后缀，next=0
- j=2: "a" 与 "a" 相同，next=1
- j=3: "ab" 前后无相同，next=1
- j=4: "a" 匹配 "a"，next=2
- j=5: "ab" 匹配 "ab"，next=3

### 改进 next 数组（nextval）

当 T[j] == T[next[j]] 时，需要继续递归。

### 时间复杂度

- O(n + m)

### C 代码

```c
void getNext(char *T, int *next) {
    int j = 0, k = -1;
    next[0] = -1;
    while (T[j] != '\0') {
        if (k == -1 || T[j] == T[k]) {
            j++; k++;
            next[j] = k;
        } else {
            k = next[k];
        }
    }
}

int kmpMatch(char *S, char *T, int *next) {
    int i = 0, j = 0;
    while (S[i] != '\0' && T[j] != '\0') {
        if (j == -1 || S[i] == T[j]) {
            i++; j++;
        } else {
            j = next[j];
        }
    }
    if (T[j] == '\0') return i - j;
    return -1;
}
```

## 算法对比

| 项目 | BF 算法 | KMP 算法 |
|------|---------|----------|
| 时间复杂度 | O(nm) | O(n+m) |
| 主串指针 | 回退 | 不回退 |
| 适用场景 | 短串 | 长串/多次匹配 |
