# 线索二叉树

## 📊 考情分析
- **考试频率**：中高频，选择题常考
- **分值**：选择题2-4分
- **题型**：选择题为主
- **真题考查方式**：线索二叉树的概念辨析、线索化后指针的指向判断、空链域计算

## 一、基本概念

### 1. 问题背景

在普通二叉链表中：
- $n$ 个结点有 $n+1$ 个**空链域**（浪费了）
- 找指定结点在某种遍历序列中的**前驱/后继**不方便
- 必须从根重新遍历才能找到

**解决方案**：利用空链域存储遍历的前驱/后继信息 → **线索二叉树**

### 2. 线索二叉树的定义

在二叉树的结点上加上线索的二叉树称为**线索二叉树**。

对二叉树以某种次序遍历使其变为线索二叉树的过程称为**线索化**。

### 3. 线索的规则

对 $n$ 个结点的二叉树：
- 若结点的**左子树为空**（`lchild == NULL`），则令 `lchild` 指向该结点在**遍历序列中的前驱**
- 若结点的**右子树为空**（`rchild == NULL`），则令 `rchild` 指向该结点在**遍历序列中的后继**

这些指向前驱/后继的指针称为"**线索**"。

### 4. 线索链表的存储结构

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 左右线索标志
} ThreadNode, *ThreadTree;
```

**tag标志位**：
- `tag == 0`：表示指针指向**孩子**
- `tag == 1`：表示指针是**线索**（前驱或后继）

## 二、核心知识

### 1. 中序线索二叉树

按照**中序遍历**的次序对二叉树进行线索化。

**中序遍历序列**：D G B E A F C

线索化过程：
- 遇到空指针，根据中序遍历的顺序填入前驱/后继
- 前驱线索用**左孩子指针**充当
- 后继线索用**右孩子指针**充当

### 2. 中序线索化代码实现

```c
ThreadNode *pre = NULL;  // 全局变量，记录前驱

// 中序线索化二叉树
void InThread(ThreadTree T) {
    if (T != NULL) {
        InThread(T->lchild);     // 线索化左子树
        visit(T);                 // 访问根，建立线索
        InThread(T->rchild);     // 线索化右子树
    }
}

void visit(ThreadNode *q) {
    if (q->lchild == NULL) {     // 左子树为空，建立前驱线索
        q->lchild = pre;
        q->ltag = 1;
    }
    if (pre != NULL && pre->rchild == NULL) {  // 前驱的右子树为空
        pre->rchild = q;         // 建立后继线索
        pre->rtag = 1;
    }
    pre = q;
}

// 建立中序线索二叉树（主函数）
void CreateInThread(ThreadTree T) {
    pre = NULL;
    if (T != NULL) {
        InThread(T);
        if (pre->rchild == NULL)
            pre->rtag = 1;       // 处理最后一个结点
    }
}
```

### 3. 在中序线索二叉树上找前驱/后继

**找中序后继**：
```c
// 找以p为根的子树中，中序遍历下的第一个结点
ThreadNode *Firstnode(ThreadNode *p) {
    while (p->ltag == 0)  // 一路向左
        p = p->lchild;
    return p;
}

// 在中序线索二叉树中找p的后继
ThreadNode *Nextnode(ThreadNode *p) {
    if (p->rtag == 1)    // 右指针是线索
        return p->rchild;
    else                  // 右子树中序下的第一个结点
        return Firstnode(p->rchild);
}
```

**找中序前驱**：
```c
// 找以p为根的子树中，中序遍历下的最后一个结点
ThreadNode *Lastnode(ThreadNode *p) {
    while (p->rtag == 0)  // 一路向右
        p = p->rchild;
    return p;
}

// 在中序线索二叉树中找p的前驱
ThreadNode *Prenode(ThreadNode *p) {
    if (p->ltag == 1)
        return p->lchild;
    else
        return Lastnode(p->lchild);
}
```

### 4. 中序线索二叉树的遍历

```c
void InOrder(ThreadTree T) {
    for (ThreadNode *p = Firstnode(T); p != NULL; p = Nextnode(p))
        visit(p);
}
```

### 5. 先序线索二叉树和后序线索二叉树

- **先序线索化**：按照先序遍历次序建立线索
- **后序线索化**：按照后序遍历次序建立线索
- 注意：先序线索化时要注意避免循环指针

## 三、深入理解

### 设计思想
- 利用**空链域**存储遍历信息，空间不浪费
- 线索化后可以在 $O(1)$ 时间找到某些前驱/后继（最坏仍需沿线索走）
- 本质上是**空间换时间**的思想

### 空链域利用
- $n$ 个结点的二叉树有 $n+1$ 个空链域
- 线索化后，这些空链域被利用为线索指针
- 线索链表中总有 **2个**指针无法线索化（第一个结点的前驱和最后一个结点的后继）

### 线索二叉树 vs 普通二叉树

| 特性 | 普通二叉树 | 线索二叉树 |
|------|----------|----------|
| 空链域 | 浪费 | 利用为线索 |
| 找前驱/后继 | 需从根遍历 | 沿线索直接找到 |
| 额外空间 | 无 | 每个结点2个tag位 |
| 适用场景 | 通用 | 需要频繁找前驱/后继 |

## 四、真题考法分析

### 常考题型
1. 给定二叉树，画出中序线索化后的结果
2. 判断线索化后某指针是指向孩子还是线索
3. 在线索二叉树中找指定结点的前驱/后继
4. 计算线索二叉树中线索的个数

### 命题规律
- 中序线索二叉树最常考
- tag标志位的判断是核心考点
- 常与遍历序列结合考查

## 五、典型例题

### 概念题

**例1**：$n$ 个结点的线索二叉树中共有几条线索？

**解**：$n+1$ 个空链域都可以线索化，但首尾各有一个无法线索化，所以最多 $n-1$ 条线索（有些结点可能左/右孩子都非空，不产生线索）。实际上线索数 = $n+1-2 = n-1$ 条（一般情况）。

**例2**：中序线索二叉树中，若结点 $p$ 有右孩子，则 $p$ 的中序后继是？

**解**：$p$ 的右子树中的**中序第一个结点**（即右子树中最左边的结点）。

### 计算题

**例3**：对如下二叉树进行中序线索化：
```
    A
   / \
  B   C
 / \
D   E
```
中序遍历：D B E A C

线索化结果：
- D 的左指针 → 线索（无前驱，为 NULL）
- D 的右指针 → 线索（后继 B）
- E 的左指针 → 线索（前驱 B）
- E 的右指针 → 线索（后继 A）
- C 的左指针 → 线索（前驱 A）
- C 的右指针 → 线索（无后继，为 NULL）

## 六、易错点

1. **tag位含义**：`0` 指向孩子，`1` 是线索，不要记反
2. **先序线索化注意循环**：如果当前结点的左子树被线索化为前驱，递归时要注意跳过
3. **线索化需要pre指针**：遍历过程中需要记录上一个访问的结点
4. **首尾特殊处理**：第一个结点的前驱线索和最后一个结点的后继线索为NULL
5. **线索二叉树遍历不需栈**：利用线索可以不用栈和递归完成遍历
6. **中序线索找后继**：有右孩子时，后继在右子树的最左结点，不是右孩子本身
