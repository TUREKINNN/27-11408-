# 快速排序（Quick Sort）

## 📊 考情分析
- **考试频率**：极高频考点，每年必考
- **分值**：4-10分（选择题+算法题）
- **题型**：选择题（复杂度分析、划分过程）、算法题（手写Partition函数）
- **真题考查方式**：模拟划分过程、分析递归深度、手写快速排序代码、分析最坏情况

## 一、基本概念

### 快速排序的定义
在待排序表 $L[1..n]$ 中任取一个元素 **pivot** 作为**枢轴**（或基准，通常取首元素），通过一趟排序将待排序表划分为独立的两部分 $L[1..k-1]$ 和 $L[k+1..n]$，使得：
- $L[1..k-1]$ 中所有元素**小于** pivot
- $L[k+1..n]$ 中所有元素**大于等于** pivot

则 pivot 放在了其最终位置 $L(k)$ 上，这个过程称为一次"**划分**"。然后分别递归地对两个子表重复上述过程，直至每部分内只有一个元素或空为止。

### 核心思想
- **分治法**：每次划分确定一个元素的最终位置
- **递归**：对左右两个子表分别递归排序

## 二、核心知识

### 划分过程示例

初始序列：`49, 38, 65, 97, 76, 13, 27, 49`（下标0~7）

**第一趟划分**（pivot = 49）：
```
49, 38, 65, 97, 76, 13, 27, 49
↑low                            ↑high

high从右往左找 < 49 的元素：27
27, 38, 65, 97, 76, 13, _, 49

low从左往右找 ≥ 49 的元素：65
27, 38, _, 97, 76, 13, 65, 49

high继续找 < 49 的元素：13
27, 38, 13, 97, 76, _, 65, 49

low继续找 ≥ 49 的元素：97
27, 38, 13, _, 76, 97, 65, 49

low和high相遇，pivot放到该位置
27, 38, 13, [49], 76, 97, 65, 49
```

pivot=49已确定最终位置，递归处理左子表`27,38,13`和右子表`76,97,65,49`。

### 代码实现

```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];           // 取第一个元素为枢轴
    while (low < high) {
        while (low < high && A[high] >= pivot) high--;
        A[low] = A[high];         // 比pivot小的移到左边
        while (low < high && A[low] < pivot) low++;
        A[high] = A[low];         // 比pivot大的移到右边
    }
    A[low] = pivot;               // 枢轴放到最终位置
    return low;
}

void QuickSort(int A[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(A, low, high);
        QuickSort(A, low, pivotpos - 1);   // 递归左子表
        QuickSort(A, pivotpos + 1, high);  // 递归右子表
    }
}
```

## 三、深入理解

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | $O(n\log_2 n)$ | 每次划分均匀 |
| 最坏 | $O(n^2)$ | 序列已有序或逆序 |
| 平均 | $O(n\log_2 n)$ | 随机数据 |

- **空间复杂度**：$O(\log_2 n)$（递归栈深度）~ $O(n)$（最坏情况）
- **稳定性**：**不稳定**

### 最坏情况
当序列**已有序或逆序**时，每次划分只能确定一个元素的位置，退化为 $O(n^2)$。

### 最好情况
每次划分都能将序列均匀分成两半，递归深度为 $\log_2 n$，每层需要 $O(n)$ 的比较，总时间为 $O(n\log_2 n)$。

### 递归工作栈
快速排序需要递归工作栈，栈的深度等于递归深度：
- 最好：$O(\log_2 n)$
- 最坏：$O(n)$

### 不稳定的原因
划分过程中，与pivot相等的元素可能会被交换到不同侧，导致相对位置改变。

## 四、真题考法分析

### 常考题型
1. 模拟Partition函数的执行过程
2. 分析快速排序在特定序列下的时间复杂度
3. 手写快速排序代码
4. 判断快速排序的稳定性

### 命题规律
- 快速排序是**平均性能最好**的内部排序算法
- 最坏情况（有序/逆序）是高频考点
- Partition函数的手写是算法题常考

## 五、典型例题

### 概念题
**Q1**：快速排序在最坏情况下的时间复杂度为（ ）
A. $O(n)$  B. $O(n\log_2 n)$  C. $O(n^2)$  D. $O(n^3)$
**答案**：C

**Q2**：以下哪种情况下快速排序的效率最低？
A. 随机序列  B. 基本有序  C. 元素互不相同  D. 元素大量重复
**答案**：B（或D，大量重复时也可能退化）

**答案**：`27, 38, 13, [49], 76, 97, 65, 49`，pivot在下标3

## 六、易错点

1. **快速排序是不稳定的**：划分过程中可能改变相等元素的相对位置
2. **最坏情况是 $O(n^2)$**：不是 $O(n\log_2 n)$
3. **空间复杂度是 $O(\log_2 n)$**：因为递归栈
4. **pivot的选择影响性能**：选首元素在有序序列下会退化
5. **快速排序不适合链表**：因为需要随机访问（low和high指针）
