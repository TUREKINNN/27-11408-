# 最小生成树（Minimum Spanning Tree）

> - 考试频率：★★★
> - 分值：选择题2-4分，大题8-12分
> - 题型：选择题、综合应用题
> - 重点：Prim和Kruskal算法的执行过程、时间复杂度分析、适用场景

---

## 一、基本概念

### 1.1 生成树定义

连通图 $G$ 的**生成树**是包含 $G$ 中**全部顶点**的一个**极小连通子图**。
- 包含 $n$ 个顶点的连通图，其生成树恰好包含 $n-1$ 条边
- 在生成树中添加任意一条边必定形成回路
- 去掉生成树中的任意一条边，图将不再连通

### 1.2 最小生成树定义

对于一个**带权连通无向图** $G=(V,E)$，其所有生成树中**边权之和最小**的那棵生成树称为**最小生成树**（MST）。

**性质：**
- 最小生成树不唯一（当图中存在权值相同的边时）
- 最小生成树的边权之和是唯一的
- 最小生成树的边数 = $|V|-1$

### 1.3 最小生成树的存在性

- 只有**连通图**才有生成树
- 只有**带权连通无向图**才有最小生成树
- 若图不连通，则每个连通分量各自有最小生成树，合称为**最小生成森林**

## 二、算法详解

### 2.1 Prim 算法（普里姆算法）

#### 2.1.1 基本思想（贪心策略）

Prim 算法从**顶点**出发，逐步扩展生成树：

1. 从图中任选一个顶点作为起始点，加入生成树集合 $U$
2. 在所有**一端在 $U$ 中、另一端不在 $U$ 中**的边中，选择权值最小的边
3. 将该边及其另一端顶点加入生成树集合
4. 重复步骤 2-3，直到 $U$ 包含所有顶点

**核心思想：** 每一步都选择"连接已选顶点集与未选顶点集"的最小权边。

#### 2.1.2 算法步骤（详细）

设图 $G=(V,E)$，顶点编号 $0$ 到 $n-1$：

```
初始化：
  lowcost[]: 各顶点到生成树的最小距离，lowcost[0]=0，其余为∞
  adjvex[]:  各顶点在生成树中的邻接顶点

循环 n-1 次：
  1. 在 lowcost[] 中找到值最小且不为0的顶点 j
  2. 输出边 (adjvex[j], j)，权值为 lowcost[j]
  3. 将 j 加入生成树（lowcost[j] = 0）
  4. 更新：对于 j 的每个邻接点 k，
     若 edge[j][k] < lowcost[k]，则
       lowcost[k] = edge[j][k]
       adjvex[k] = j
```

#### 2.1.3 时间复杂度分析

| 存储结构 | 时间复杂度 | 说明 |
|---------|-----------|------|
| 邻接矩阵 | $O(n^2)$ | 外层循环 $n-1$ 次 × 内层找最小值 $O(n)$ |
| 邻接表 + 最小堆 | $O((n+m)\log n)$ | 优化实现 |

**空间复杂度：** $O(n)$（lowcost 和 adjvex 数组）

#### 2.1.4 适用场景

- **稠密图**（边数较多时）效率更高
- 时间复杂度只与顶点数有关，与边数无关（邻接矩阵实现）

---

### 2.2 Kruskal 算法（克鲁斯卡尔算法）

#### 2.2.1 基本思想（贪心策略）

Kruskal 算法从**边**出发，逐步构建生成树：

1. 将图中所有边按**权值从小到大**排序
2. 依次考察每条边：
   - 若该边的两个端点属于**不同的连通分量**，则加入该边（合并两个分量）
   - 若该边的两个端点属于**同一连通分量**，则跳过（否则会形成环）
3. 重复步骤 2，直到选够 $n-1$ 条边

**核心思想：** 每一步都选择当前权值最小且不会形成环的边。

#### 2.2.2 并查集应用

Kruskal 算法需要高效判断两个顶点是否属于同一连通分量，**并查集**是最佳选择：

**并查集的三个操作：**
- `Init(S)`: 初始化，每个元素自成一个集合
- `Find(S, x)`: 查找元素 $x$ 所属集合的代表元素
- `Union(S, x, y)`: 合并 $x$ 和 $y$ 所在的集合

**优化技巧：**
- **按秩合并**：将深度小的树合并到深度大的树下
- **路径压缩**：查找时将路径上的所有节点直接挂到根节点

#### 2.2.3 时间复杂度分析

| 操作 | 复杂度 |
|------|--------|
| 边排序 | $O(m\log m)$ |
| 并查集操作（$m$ 次查找 + $n-1$ 次合并） | $O(m\alpha(n))$ ≈ $O(m)$ |
| **总复杂度** | **$O(m\log m)$** |

其中 $\alpha(n)$ 为反阿克曼函数，实际可视为常数。

#### 2.2.4 适用场景

- **稀疏图**（边数较少时）效率更高
- 时间复杂度主要取决于边数

---

### 2.3 两种算法对比

| 对比项目 | Prim 算法 | Kruskal 算法 |
|---------|-----------|-------------|
| **出发点** | 从顶点出发 | 从边出发 |
| **核心思想** | 扩展顶点集 | 扩展边集 |
| **数据结构** | lowcost 数组 | 并查集 |
| **时间复杂度** | $O(n^2)$（邻接矩阵） | $O(m\log m)$ |
| **适用场景** | 稠密图 | 稀疏图 |
| **是否需要排序** | 不需要 | 需要对边排序 |
| **实现难度** | 较简单 | 需要并查集 |
| **与边数关系** | 无关 | 直接相关 |
| **与顶点数关系** | 直接相关 | 弱相关 |

> - **Prim**：以**点**为核心，"步步为营"扩展顶点集
> - **Kruskal**：以**边**为核心，"见缝插针"选最小边

## 三、代码实现

### 3.1 Prim 算法（邻接矩阵实现）

```c
#include <stdio.h>
#include <limits.h>

#define MAXVEX 100
#define INF 65535

typedef struct {
    int vexs[MAXVEX];           // 顶点表
    int arc[MAXVEX][MAXVEX];    // 邻接矩阵
    int numVertexes, numEdges;  // 顶点数和边数
} MGraph;

/**
 * Prim算法求最小生成树
 * @param G 图（邻接矩阵存储）
 * @param start 起始顶点下标
 */
void MiniSpanTree_Prim(MGraph G, int start) {
    int lowcost[MAXVEX];  // 各顶点到生成树的最小权值
    int adjvex[MAXVEX];   // 各顶点对应最小权边在生成树中的另一端
    
    // 初始化：将起始顶点的邻接边权值存入lowcost
    for (int i = 0; i < G.numVertexes; i++) {
        lowcost[i] = G.arc[start][i];
        adjvex[i] = start;
    }
    lowcost[start] = 0;  // start已加入生成树
    
    printf("Prim最小生成树的边：\n");
    
    for (int i = 1; i < G.numVertexes; i++) {
        int min = INF;
        int j = start;  // 记录当前找到的最小权值顶点
        
        // 在lowcost中找最小值（即离生成树最近的顶点）
        for (int k = 0; k < G.numVertexes; k++) {
            if (lowcost[k] != 0 && lowcost[k] < min) {
                min = lowcost[k];
                j = k;
            }
        }
        
        // 输出找到的边
        printf("(%d, %d) 权值=%d\n", adjvex[j], j, lowcost[j]);
        
        lowcost[j] = 0;  // 顶点j加入生成树
        
        // 更新：j的邻接点到生成树的最小距离
        for (int k = 0; k < G.numVertexes; k++) {
            if (lowcost[k] != 0 && G.arc[j][k] < lowcost[k]) {
                lowcost[k] = G.arc[j][k];
                adjvex[k] = j;
            }
        }
    }
}
```

### 3.2 Kruskal 算法（并查集实现）

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXEDGE 200
#define MAXVEX 100

// 边集数组
typedef struct {
    int begin;   // 边的起点
    int end;     // 边的终点
    int weight;  // 边的权值
} Edge;

// 并查集：查找（带路径压缩）
int Find(int parent[], int x) {
    while (parent[x] >= 0) {
        x = parent[x];
    }
    return x;
    // 路径压缩优化写法：
    // if (parent[x] >= 0)
    //     parent[x] = Find(parent, parent[x]);
    // return parent[x] >= 0 ? parent[x] : x;
}

// 并查集：合并（按秩合并）
void Union(int parent[], int root1, int root2) {
    // 简单合并（将root2挂到root1下）
    parent[root2] = root1;
}

/**
 * 边按权值升序比较函数（用于qsort）
 */
int cmp(const void *a, const void *b) {
    return ((Edge *)a)->weight - ((Edge *)b)->weight;
}

/**
 * Kruskal算法求最小生成树
 * @param edges 边集数组
 * @param n 顶点数
 * @param m 边数
 */
void MiniSpanTree_Kruskal(Edge edges[], int n, int m) {
    int parent[MAXVEX];
    
    // 初始化并查集：每个顶点自成一个集合
    for (int i = 0; i < n; i++) {
        parent[i] = -1;  // -1表示该节点是根
    }
    
    // 将所有边按权值升序排序
    qsort(edges, m, sizeof(Edge), cmp);
    
    printf("Kruskal最小生成树的边：\n");
    
    int count = 0;  // 已选边数
    
    for (int i = 0; i < m && count < n - 1; i++) {
        int root1 = Find(parent, edges[i].begin);
        int root2 = Find(parent, edges[i].end);
        
        // 两端点不在同一连通分量中，不会形成环
        if (root1 != root2) {
            printf("(%d, %d) 权值=%d\n", 
                   edges[i].begin, edges[i].end, edges[i].weight);
            Union(parent, root1, root2);
            count++;
        }
        // 同一连通分量则跳过，避免成环
    }
}
```

## 四、典型例题

### 例题 1：Prim 算法执行过程（选择题）

**题目：** 如下图所示的无向连通图，从顶点 A 出发执行 Prim 算法，给出最小生成树的边的选择顺序。

```
    A---3---B---6---C
    |       |       |
    5       4       2
    |       |       |
    D---7---E---1---F
```

顶点：A, B, C, D, E, F
边及权值：(A,B)=3, (B,C)=6, (A,D)=5, (B,E)=4, (C,F)=2, (D,E)=7, (E,F)=1

**解析：**

从 A 出发，初始化 lowcost：B=3, D=5, 其余=∞

| 步骤 | 选择的边 | 权值 | lowcost 更新 |
|------|---------|------|-------------|
| 1 | (A,B) | 3 | D=5, C=6, E=4 |
| 2 | (B,E) | 4 | D=5, C=6, F=1 |
| 3 | (E,F) | 1 | D=5, C=2 |
| 4 | (F,C) | 2 | D=5 |
| 5 | (A,D) | 5 | 完成 |

最小生成树边：(A,B)=3, (B,E)=4, (E,F)=1, (F,C)=2, (A,D)=5
总权值 = 3+4+1+2+5 = **15**

---

### 例题 2：Kruskal 算法执行过程（选择题）

**题目：** 对例题 1 的图执行 Kruskal 算法，给出边的选择顺序。

**解析：**

将所有边按权值升序排序：
1. (E,F)=1 ✓（E、F 不在同一集合）
2. (C,F)=2 ✓（C、F 不在同一集合）
3. (A,B)=3 ✓（A、B 不在同一集合）
4. (B,E)=4 ✓（B、E 不在同一集合）
5. (A,D)=5 ✓（A、D 不在同一集合）

此时已选 5 = 6-1 条边，算法结束。

最小生成树：(E,F), (C,F), (A,B), (B,E), (A,D)，总权值 = **15**

> Prim 和 Kruskal 算法得到的最小生成树总权值相同（都是 15），但具体选择的边可能不同（当存在权值相同的边时）。

### 例题 3：综合应用题

**题目：** 设有连通网 $G$，顶点集 $V=\{0,1,2,3,4\}$，邻接矩阵如下（$\infty$ 表示无边）：

```
     0    1    2    3    4
0  [ 0,  10,   ∞,   6,   5]
1  [10,   0,   7,   4,   ∞]
2  [ ∞,   7,   0,   8,   3]
3  [ 6,   4,   8,   0,   9]
4  [ 5,   ∞,   3,   9,   0]
```

(1) 画出图 $G$。
(2) 用 Prim 算法从顶点 0 出发求最小生成树，写出每步选择的边。
(3) 用 Kruskal 算法求最小生成树，写出每步选择的边。

**解析：**

**(1) 图 G 如下：**
- 边：(0,1)=10, (0,3)=6, (0,4)=5, (1,2)=7, (1,3)=4, (2,3)=8, (2,4)=3, (3,4)=9

**(2) Prim 算法（从 0 出发）：**

初始化 lowcost：[0, 10, ∞, 6, 5]

| 步骤 | 最小值顶点 | 选择边 | 权值 | lowcost 更新 |
|------|-----------|--------|------|-------------|
| 1 | 4 | (0,4) | 5 | [0, 10, 3, 6, 0] |
| 2 | 2 | (4,2) | 3 | [0, 7, 0, 6, 0] |
| 3 | 3 | (0,3) | 6 | [0, 4, 0, 0, 0] |
| 4 | 1 | (3,1) | 4 | [0, 0, 0, 0, 0] |

最小生成树：(0,4)=5, (4,2)=3, (0,3)=6, (3,1)=4，总权值 = **18**

**(3) Kruskal 算法：**

按权值排序的边：
1. (2,4)=3 ✓
2. (1,3)=4 ✓
3. (0,4)=5 ✓
4. (0,3)=6 ✓ — 此时已选 4 条边，算法结束

最小生成树：(2,4)=3, (1,3)=4, (0,4)=5, (0,3)=6，总权值 = **18**

## 五、易错点

### 5.1 常见错误

1. **Prim 算法选错起点**
   - 起点选择不影响最终总权值，但会影响具体选边顺序
   - 考试时严格按照题目给定的起点执行

2. **Kruskal 算法忘记判断是否成环**
   - 必须用并查集判断两端点是否在同一连通分量
   - 同一分量的边直接跳过

3. **混淆最小生成树的唯一性**
   - 当存在权值相同的边时，最小生成树不唯一
   - 但最小生成树的总权值唯一

4. **时间复杂度记混**
   - Prim（邻接矩阵）：$O(n^2)$ — 与顶点数平方成正比
   - Kruskal：$O(m\log m)$ — 与边数有关

5. **Prim 算法中 lowcost 理解错误**
   - lowcost[k] 表示顶点 k 到**生成树集合**的最小边权，不是到某个特定顶点的距离

### 5.2 考试技巧

- 选择题：画表逐步模拟，注意每步更新 lowcost 或并查集
- 大题：写出完整过程，不要跳步
- 求"最小权值之和"时，验证是否等于正确值
