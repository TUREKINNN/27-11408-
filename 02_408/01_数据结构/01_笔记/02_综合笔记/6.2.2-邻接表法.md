# 6.2.2 邻接表法

## 基本思想

**顺序 + 链式存储**，结合了数组和链表的优势。

### 结构组成

```
顶点表（顺序存储）          边表（链式存储）
┌──────┬──────────┐
│ data │ *first   │──→ [adjvex | next] → [adjvex | next] → ^
├──────┼──────────┤
│  A   │  ──────  │──→ [B|→] → [C|→] → [D|→] → ^
├──────┼──────────┤
│  B   │  ──────  │──→ [A|→] → [E|→] → [F|→] → ^
├──────┼──────────┤
│ ...  │   ...    │
└──────┴──────────┘
```

- **顶点表**：数组存储各顶点信息，每个结点包含 `data` 和 `*first` 指针
- **边表**：单链表存储与该顶点相连的边/弧信息

## 无向图的邻接表

每条边在邻接表中出现**两次**。

- 边结点数量 = $2|E|$
- 整体空间复杂度 = $O(|V| + 2|E|)$

## 有向图的邻接表

每条弧在邻接表中只出现**一次**。

- 边结点数量 = $|E|$
- 整体空间复杂度 = $O(|V| + |E|)$

> **逆邻接表**：若需要方便地找入边，可以建立逆邻接表（每个顶点的链表存储以该顶点为终点的弧）

## 与树的孩子表示法对比

邻接表的结构与树的**孩子表示法**完全一致：

- 孩子表示法：顺序存储各结点，每个结点保存孩子链表头指针
- 邻接表：顺序存储各顶点，每个顶点保存邻接边链表头指针

## 重要性质

> 确定了顶点编号后，邻接表中各边结点的链接顺序可以不同。
> 而邻接矩阵表示方式是**唯一**的。

## 求顶点的度

| 操作 | 无向图 | 有向图 |
|------|--------|--------|
| 求度 | 遍历该顶点的边链表，O(degree) | 需结合邻接表和逆邻接表 |
| 求出度 | 同上 | 遍历邻接表中该顶点的边链表 |
| 求入度 | 同上 | 遍历所有顶点的边链表，O(|V|+|E|)，**不方便** |

## 邻接表 vs 邻接矩阵 对比

| 特性 | 邻接表 | 邻接矩阵 |
|------|--------|----------|
| **空间复杂度** | 无向图 O(|V|+2\|E\|)；有向图 O(|V|+\|E\|) | O(|V|²) |
| **适合场景** | **稀疏图** | **稠密图** |
| **表示方式** | **不唯一** | **唯一** |
| **求度/入度/出度** | 有向图求入度不方便，其余方便 | 必须遍历对应行或列 |
| **找相邻边** | 有向图找入边不方便，其余方便 | 必须遍历对应行或列 |

> - 判断何时用邻接表 vs 邻接矩阵
> - 邻接表表示不唯一，邻接矩阵唯一
> - 空间复杂度的区别及其适用场景
