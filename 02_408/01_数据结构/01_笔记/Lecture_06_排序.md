## 1. 排序的基本概念

### 1.1 排序的定义

<span style="background-color:#ccc1d9">**排序**</span>：将数据元素按关键字递增或递减的顺序排列。

### 1.2 基本术语

| 术语 | 定义 |
|------|------|
| **稳定性** | 若两个元素关键字相同，排序后相对位置不变，则为稳定排序 |
| **内部排序** | 排序过程中数据全部在内存中 |
| **外部排序** | 排序过程中需要访问外存 |

### 1.3 排序算法分类

- **插入排序**：直接插入排序、希尔排序
- **交换排序**：冒泡排序、快速排序
- **选择排序**：简单选择排序、堆排序
- **归并排序**：二路归并排序
- **基数排序**：按关键字的每一位进行排序

---

## 2. 插入排序

### 2.1 直接插入排序

**思想**：将未排序元素插入到已排序序列的正确位置。

```c
void InsertSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

**时间复杂度**：$O(n^2)$

**空间复杂度**：$O(1)$

**稳定性**：稳定

### 2.2 希尔排序

**思想**：按增量分组，每组进行插入排序，增量递减至1。

**时间复杂度**：$O(n^{1.3})$ ~ $O(n^2)$

**稳定性**：不稳定

---

## 3. 交换排序

### 3.1 冒泡排序

**思想**：重复遍历，比较相邻元素，交换逆序对。

```c
void BubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
                flag = true;
            }
        }
        if (!flag) break;
    }
}
```

**时间复杂度**：$O(n^2)$

**空间复杂度**：$O(1)$

**稳定性**：稳定

### 3.2 快速排序

**思想**：选择基准元素，将数组分成两部分，递归排序。

```c
int Partition(int arr[], int low, int high) {
    int pivot = arr[low];
    while (low < high) {
        while (low < high && arr[high] >= pivot) high--;
        arr[low] = arr[high];
        while (low < high && arr[low] <= pivot) low++;
        arr[high] = arr[low];
    }
    arr[low] = pivot;
    return low;
}

void QuickSort(int arr[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(arr, low, high);
        QuickSort(arr, low, pivotpos - 1);
        QuickSort(arr, pivotpos + 1, high);
    }
}
```

**时间复杂度**：
- 最好：$O(n \log n)$
- 最坏：$O(n^2)$
- 平均：$O(n \log n)$

**空间复杂度**：$O(\log n)$ ~ $O(n)$

**稳定性**：不稳定

## 4. 选择排序

### 4.1 简单选择排序

**思想**：每次从未排序部分选择最小元素放到已排序部分末尾。

```c
void SelectSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min]) min = j;
        }
        if (min != i) swap(arr[i], arr[min]);
    }
}
```

**时间复杂度**：$O(n^2)$

**空间复杂度**：$O(1)$

**稳定性**：不稳定

### 4.2 堆排序

**思想**：利用堆的性质，每次取出堆顶元素，调整堆。

**时间复杂度**：$O(n \log n)$

**空间复杂度**：$O(1)$

**稳定性**：不稳定

## 5. 归并排序

### 5.1 二路归并排序

**思想**：将数组分成两半，分别排序，然后合并。

```c
void Merge(int arr[], int low, int mid, int high) {
    int *temp = (int*)malloc((high - low + 1) * sizeof(int));
    int i = low, j = mid + 1, k = 0;
    while (i <= mid && j <= high) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= high) temp[k++] = arr[j++];
    for (i = low, k = 0; i <= high; i++, k++) arr[i] = temp[k];
    free(temp);
}

void MergeSort(int arr[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(arr, low, mid);
        MergeSort(arr, mid + 1, high);
        Merge(arr, low, mid, high);
    }
}
```

**时间复杂度**：$O(n \log n)$

**空间复杂度**：$O(n)$

**稳定性**：稳定

## 6. 基数排序

### 6.1 基数排序的思想

**思想**：按关键字的每一位进行排序，从低位到高位。

**时间复杂度**：$O(d(n + r))$，$d$ 为位数，$r$ 为基数

**稳定性**：稳定

## 7. 排序算法总结

### 7.1 算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 |
|------|------------|------------|--------|
| **直接插入排序** | $O(n^2)$ | $O(1)$ | 稳定 |
| **希尔排序** | $O(n^{1.3})$ ~ $O(n^2)$ | $O(1)$ | 不稳定 |
| **冒泡排序** | $O(n^2)$ | $O(1)$ | 稳定 |
| **快速排序** | $O(n \log n)$（平均） | $O(\log n)$ | 不稳定 |
| **简单选择排序** | $O(n^2)$ | $O(1)$ | 不稳定 |
| **堆排序** | $O(n \log n)$ | $O(1)$ | 不稳定 |
| **归并排序** | $O(n \log n)$ | $O(n)$ | 稳定 |
| **基数排序** | $O(d(n + r))$ | $O(n + r)$ | 稳定 |

### 7.2 选择策略

| 场景 | 推荐算法 |
|------|----------|
| 数据量小 | 直接插入排序、冒泡排序 |
| 数据基本有序 | 直接插入排序、冒泡排序 |
| 数据量大 | 快速排序、堆排序、归并排序 |
| 稳定性要求 | 归并排序、基数排序 |
| 空间受限 | 堆排序、快速排序 |

## 8. 外部排序

### 8.1 外部排序的过程

1. **生成初始归并段**：将外存数据分段读入内存，排序后写回外存
2. **多路归并**：对多个归并段进行合并

### 8.2 败者树

<span style="background-color:#ccc1d9">**败者树**</span>：一种树形选择结构，用于多路归并时选择最小元素。

### 8.3 置换-选择排序

**思想**：生成长度可变的初始归并段。

### 8.4 最佳归并树

**思想**：构造哈夫曼树，使归并代价最小。

## 9. 典型例题

### 例题 1：快速排序

**题目**：对数组 `[49, 38, 65, 97, 76, 13, 27, 49]` 进行快速排序。

**解答**：

```c
int Partition(int arr[], int low, int high) {
    int pivot = arr[low];
    while (low < high) {
        while (low < high && arr[high] >= pivot) high--;
        arr[low] = arr[high];
        while (low < high && arr[low] <= pivot) low++;
        arr[high] = arr[low];
    }
    arr[low] = pivot;
    return low;
}

// 第一趟划分：
// 基准：49
// 结果：[27, 38, 13, 49, 76, 97, 65, 49]
// pivotpos = 3

// 递归排序左半部分 [27, 38, 13]
// 基准：27
// 结果：[13, 27, 38]

// 递归排序右半部分 [76, 97, 65, 49]
// 基准：76
// 结果：[49, 65, 76, 97]

// 最终结果：[13, 27, 38, 49, 49, 65, 76, 97]
```

**总结**：快速排序的关键是划分操作。

**来源**：408历年真题，考查方式：综合题

### 例题 3：排序算法选择

**题目**：分析不同场景下应选择哪种排序算法。

**解答**：

```
场景1：数据量小（n < 100）
→ 选择：直接插入排序或冒泡排序
→ 原因：简单，常数因子小

场景2：数据基本有序
→ 选择：直接插入排序或冒泡排序
→ 原因：时间复杂度接近O(n)

场景3：数据量大（n > 10000）
→ 选择：快速排序、堆排序或归并排序
→ 原因：时间复杂度O(n log n)

场景4：稳定性要求高
→ 选择：归并排序或基数排序
→ 原因：稳定排序算法

场景5：空间受限
→ 选择：堆排序或快速排序
→ 原因：空间复杂度O(log n)或O(1)
```

**总结**：根据数据特点和性能要求选择合适的排序算法。

**来源**：408历年真题，考查方式：选择题
