## 第一章 绪论

### 一、选择题

> (A) 动态结构和静态结构
> (B) 紧凑结构和非紧凑结构
> (C) 线性结构和非线性结构
> (D) 内部结构和外部结构

**答案**：C

**解析**：数据结构按逻辑结构分为线性结构（如线性表、栈、队列、串）和非线性结构（如树、图）。动态/静态是按存储方式划分，紧凑/非紧凑是按存储密度划分，内部/外部是按数据存储位置划分。

> (A) 问题的规模
> (B) 待处理数据的初态
> (C) 问题的规模和待处理数据的初态
> (D) 计算机的运算速度

**答案**：C

**解析**：算法的时间复杂度是一个函数，它定性描述该算法的运行时间。时间复杂度不仅与问题规模 n 有关，还与输入数据的初始状态有关（如冒泡排序最好 O(n) 最坏 O(n²)）。计算机运算速度影响的是实际运行时间，而非时间复杂度。

> (A) 算法必须有输入
> (B) 算法必须有输出
> (C) 算法必须在有限步内结束
> (D) 算法的每一步必须确切定义

**答案**：A

**解析**：算法的五大特性：有穷性、确定性、可行性、输入（0 个或多个）、输出（至少 1 个）。算法可以没有输入（如 `print("hello")` ），但必须至少有一个输出。

> ```c
> for (i = 1; i <= n; i++)
>     for (j = 1; j <= i; j++)
>         x++;
> ```
> (A) O(n)
> (B) O(n²)
> (C) O(n³)
> (D) O(log n)

**答案**：B

**解析**：内层循环执行次数为 1+2+3+...+n = n(n+1)/2，因此时间复杂度为 O(n²)。注意虽然内层循环上界是 i 而非 n，但累加后仍是平方级别。

> ```c
> for (i = 1; i <= n; i *= 2)
>     for (j = 1; j <= n; j++)
>         x++;
> ```
> (A) n log₂n
> (B) n²
> (C) log₂n
> (D) n

**答案**：A

**解析**：外层循环 i 每次乘以 2，执行次数为 ⌈log₂n⌉；内层循环每次执行 n 次。总执行次数约为 n·log₂n。

### 二、填空题


**答案**：集合、线性结构、树形结构、图状结构（或网状结构）

**解析**：四种基本逻辑结构：集合（元素同属一个集合，无其他关系）；线性结构（一对一关系）；树形结构（一对多关系）；图状结构（多对多关系）。


**答案**：有穷性、确定性、可行性、输入、输出

**解析**：有穷性指算法必须在有限步后终止；确定性指每条指令必须有确切的含义；可行性指算法中的操作都可以通过基本运算有限次实现。


**答案**：时间复杂度、空间复杂度

**解析**：时间复杂度衡量算法运行时间随问题规模增长的趋势；空间复杂度衡量算法所需额外存储空间。此外还有可读性、健壮性等指标。

## 第二章 线性表

### 一、选择题

> (A) 有限序列
> (B) 无限序列
> (C) 有序序列
> (D) 无序序列

**答案**：A

**解析**：线性表是由 n（n≥0）个具有相同特性的数据元素构成的有限序列。关键点：①有限（元素个数有限）；②序列（元素之间有先后顺序）。注意"有序"指元素值按大小排列，与线性表的"序列"概念不同——线性表强调的是位置顺序，不要求值有序。

> (A) 数据表
> (B) 字符
> (C) 数据元素
> (D) 数据项

**答案**：C

**解析**：线性表由数据元素组成。数据元素是数据的基本单位，可由多个数据项组成。数据项是数据的不可分割的最小单位。注意区分：数据元素 > 数据项。

> (A) 插入删除方便
> (B) 存储空间利用率高
> (C) 随机访问速度快
> (D) 不需要预先分配空间

**答案**：C

**解析**：顺序表（数组）支持随机存取，可在 O(1) 时间内访问任意位置的元素。链表需要从头遍历，访问第 i 个元素需要 O(n)。注意：顺序表存储密度为 1，确实空间利用率高，但"随机访问速度快"是它相比链表最核心的优势，也是 408 常考点。

> (A) 随机访问速度快
> (B) 插入删除不需要移动元素
> (C) 存储空间连续
> (D) 存储空间利用率高

**答案**：B

**解析**：链表插入和删除只需修改指针，O(1) 时间（给定插入位置的前提下）。顺序表插入删除需要移动大量元素，平均移动 n/2 个。但链表存储密度低（每个结点需额外存储指针），且不支持随机存取。

> (A) 方便链表的建立
> (B) 方便链表的遍历
> (C) 方便链表的插入和删除
> (D) 节省存储空间

**答案**：C

**解析**：头结点使得在首元结点之前插入新结点或删除首元结点时，操作与其他位置的操作统一，无需特殊处理。头结点不存储数据（或存储附加信息），其指针域指向首元结点。

> (A) n - i
> (B) n - i + 1
> (C) i
> (D) i - 1

**答案**：A

**解析**：删除第 i 个元素（1≤i≤n）后，第 i+1 到第 n 个元素需依次前移一位，共移动 n-i 个元素。插入第 i 个位置需移动 n-i+1 个元素。二者差 1，务必区分。

> (A) i
> (B) n - i
> (C) n - i + 1
> (D) n - i - 1

**答案**：C

**解析**：在第 i 个位置（1≤i≤n+1）插入新元素，原第 i 到第 n 个元素需依次后移一位，共移动 n-i+1 个元素。当 i=n+1 时（表尾插入），移动 0 个元素。

> (A) O(1)
> (B) O(n)
> (C) O(n²)
> (D) O(log n)

**答案**：A

**解析**：给定指向某结点的指针 p，在其后插入新结点只需修改指针，时间复杂度 O(1)。若未给定插入位置指针，需先遍历找到插入位置，则为 O(n)。注意：在单链表中，在某结点前插入新结点需要 O(n)（需找到前驱），除非采用"后插再交换数据"技巧。

> (A) O(1)
> (B) O(n)
> (C) O(n²)
> (D) O(log n)

**答案**：A

**解析**：双向链表每个结点有前驱和后继指针，给定 p 可直接访问其前驱和后继，删除操作只需修改前驱的 next 和后继的 prior，O(1) 即可完成。

### 二、填空题


**答案**：LOC(a₁) + (i-1) × L

**解析**：顺序表元素连续存储，第 i 个元素（下标从 1 开始）前面有 i-1 个元素，所以偏移量为 (i-1)×L。这是随机存取的基础。


**答案**：数据域、指针域

**解析**：数据域存储数据元素本身，指针域存储指向直接后继结点的指针。最后一个结点的指针域为空（NULL）。


**答案**：最后一个结点的指针指向头结点（或第一个结点）

**解析**：循环单链表：最后一个结点的 next 指向头结点；循环双向链表：头结点的 prior 指向尾结点，尾结点的 next 指向头结点。循环链表的优点是：从任一结点出发可以访问到表中所有其他结点。


**答案**：前驱指针（prior）、数据域（data）、后继指针（next）

**解析**：prior 指向直接前驱，next 指向直接后继。双向链表可双向遍历，在 O(1) 时间内找到前驱和后继。


**答案**：0

**解析**：线性表可以为空表（n=0）。空顺序表 length=0；空链表头指针指向 NULL（或仅有头结点，其 next 为 NULL）。

### 三、综合题


**答案**：

```c
void delX(SqList *L, ElemType x) {
    int k = 0;
    for (int i = 0; i < L->length; i++) {
        if (L->data[i] != x) {
            L->data[k] = L->data[i];
            k++;
        }
    }
    L->length = k;
}
```

**解析**：使用双指针（k 为慢指针，i 为快指针），遍历一遍顺序表，将不等于 x 的元素依次前移。时间复杂度 O(n)，每个元素最多移动一次；空间复杂度 O(1)，仅使用常量辅助空间。这是顺序表删除元素的经典高效算法。


**答案**：

```c
void Reverse(SqList *L) {
    ElemType temp;
    for (int i = 0; i < L->length / 2; i++) {
        temp = L->data[i];
        L->data[i] = L->data[L->length - 1 - i];
        L->data[L->length - 1 - i] = temp;
    }
}
```

**解析**：交换首尾对称位置的元素，向中间靠拢。i 从 0 到 length/2 - 1（当 length 为奇数时中间元素不用交换）。时间复杂度 O(n)，空间复杂度 O(1)。


**答案**：

```c
void ReverseList(LinkList L) {
    LNode *p = L->next;  // p 指向首元结点
    LNode *q;
    L->next = NULL;      // 将头结点与后续断开
    while (p != NULL) {
        q = p->next;     // 暂存后继
        p->next = L->next;  // 头插
        L->next = p;
        p = q;
    }
}
```

**解析**：使用头插法，依次将原链表的每个结点摘下，插入到头结点之后。时间复杂度 O(n)，空间复杂度 O(1)。注意需保存 p->next 再修改指针，否则会断链。这是单链表反转的经典方法。


**答案**：

```c
LinkList Merge(LinkList La, LinkList Lb) {
    LinkList Lc;
    LNode *pa = La->next, *pb = Lb->next, *pc;
    Lc = La;           // 以 La 的头结点作为 Lc 的头结点
    pc = Lc;
    while (pa != NULL && pb != NULL) {
        if (pa->data <= pb->data) {
            pc->next = pa;
            pc = pa;
            pa = pa->next;
        } else {
            pc->next = pb;
            pc = pb;
            pb = pb->next;
        }
    }
    pc->next = (pa != NULL) ? pa : pb;  // 连接剩余部分
    free(Lb);  // 释放 Lb 的头结点
    return Lc;
}
```

**解析**：类似归并排序的合并过程，同时遍历两个链表，每次取较小值的结点接到结果链表尾部。时间复杂度 O(m+n)，空间复杂度 O(1)。注意释放无用头结点。

## 第三章 栈和队列

### 一、选择题

> (A) 先进先出
> (B) 后进先出
> (C) 随机访问
> (D) 顺序访问

**答案**：B

**解析**：栈是后进先出（LIFO，Last In First Out）的线性表，只允许在栈顶进行插入和删除。类比：一摞盘子，后放的先取走。

> (A) 先进先出
> (B) 后进先出
> (C) 随机访问
> (D) 顺序访问

**答案**：A

**解析**：队列是先进先出（FIFO，First In First Out）的线性表，只允许在队尾插入、队头删除。类比：排队购物，先到先服务。

> (A) 入栈和出栈
> (B) 入队和出队
> (C) 插入和删除
> (D) 查找和排序

**答案**：A

**解析**：栈的基本操作：Push（入栈/压栈）将元素放入栈顶；Pop（出栈/弹栈）移除并返回栈顶元素。此外还有 GetTop（取栈顶元素，不移除）、StackEmpty（判空）等辅助操作。

> (A) 入栈和出栈
> (B) 入队和出队
> (C) 插入和删除
> (D) 查找和排序

**答案**：B

**解析**：队列的基本操作：EnQueue（入队）在队尾插入元素；DeQueue（出队）从队头删除元素。此外还有 GetHead（取队头元素）、QueueEmpty（判空）等辅助操作。

> (A) 表达式求值
> (B) 括号匹配
> (C) 函数调用
> (D) 打印队列

**答案**：D

**解析**：栈的典型应用：①表达式求值（中缀转后缀、后缀计算）；②括号匹配（遇左括号入栈，右括号出栈匹配）；③函数调用（递归调用栈）；④DFS 搜索；⑤撤销操作（Undo）。打印队列是队列的应用（FIFO）。

### 二、填空题


**答案**：栈顶元素（或栈顶元素的下一个位置，取决于实现约定）

**解析**：两种常见约定：①top 指向栈顶元素所在位置，初始 top = -1；②top 指向栈顶元素的下一个位置，初始 top = 0。408 考试中通常采用第一种约定。入栈时 top++ 再赋值，出栈时先取值再 top--。


**答案**：队头元素、队尾元素的下一个位置

**解析**：循环队列通常约定：front 指向队头元素，rear 指向队尾元素的下一个位置。这样判空为 front == rear，判满为 (rear+1) % maxSize == front（牺牲一个单元）。另一种约定是 rear 指向队尾元素，此时需要不同的判空判满逻辑。


**答案**：n

**解析**：顺序栈用数组实现，大小为 n 的数组最多存放 n 个元素。注意：循环队列如果是"牺牲一个单元"的判满方式，容量为 n 的数组最多存放 n-1 个元素。


**答案**：front == rear（队头指针等于队尾指针）

**解析**：当 front == rear 时，队列中没有元素。注意区分：若采用 rear 指向队尾元素本身（而非下一位置），则判空条件可能不同。


**答案**：(rear + 1) % n == front

**解析**：循环队列中，若 rear 的下一个位置是 front，则认为队列已满。此时数组中实际最多存放 n-1 个元素。若不牺牲单元，可另设一个 size 变量记录元素个数，或设 tag 标记最后一次操作是入队还是出队。

### 三、综合题


**答案**：

```c
#define MAXSIZE 100
typedef struct {
    ElemType data[MAXSIZE];
    int top;
} SqStack;

// 入栈
int Push(SqStack *S, ElemType e) {
    if (S->top == MAXSIZE - 1) return 0;  // 栈满
    S->data[++S->top] = e;
    return 1;
}

// 出栈
int Pop(SqStack *S, ElemType *e) {
    if (S->top == -1) return 0;  // 栈空
    *e = S->data[S->top--];
    return 1;
}
```

**解析**：top 初始为 -1（空栈）。入栈时先 top++，再放入元素；出栈时先取出栈顶元素，再 top--。注意边界检查：栈满不能入栈，栈空不能出栈。


**答案**：

```c
#define MAXSIZE 100
typedef struct {
    ElemType data[MAXSIZE];
    int front, rear;
} SqQueue;

// 入队
int EnQueue(SqQueue *Q, ElemType e) {
    if ((Q->rear + 1) % MAXSIZE == Q->front) return 0;  // 队满
    Q->data[Q->rear] = e;
    Q->rear = (Q->rear + 1) % MAXSIZE;
    return 1;
}

// 出队
int DeQueue(SqQueue *Q, ElemType *e) {
    if (Q->front == Q->rear) return 0;  // 队空
    *e = Q->data[Q->front];
    Q->front = (Q->front + 1) % MAXSIZE;
    return 1;
}
```

**解析**：采用牺牲一个单元的判满方式。front 指向队头，rear 指向队尾下一位置。入队：先存值再 rear 后移；出队：先取值再 front 后移。`% MAXSIZE` 实现循环。


**答案**：

```c
int EvalPostfix(char *exp) {
    int stack[MAXSIZE], top = -1;
    for (int i = 0; exp[i] != '\0'; i++) {
        if (isdigit(exp[i])) {
            stack[++top] = exp[i] - '0';
        } else {
            int b = stack[top--];  // 右操作数
            int a = stack[top--];  // 左操作数
            switch (exp[i]) {
                case '+': stack[++top] = a + b; break;
                case '-': stack[++top] = a - b; break;
                case '*': stack[++top] = a * b; break;
                case '/': stack[++top] = a / b; break;
            }
        }
    }
    return stack[top];
}
```

**解析**：后缀表达式（逆波兰式）求值：从左到右扫描，遇到操作数入栈；遇到运算符弹出两个操作数（先弹出的是右操作数），计算后将结果入栈。最终栈中只剩一个元素，即表达式结果。注意减法和除法中操作数顺序。

## 第四章 串

### 一、选择题

> (A) 可以顺序存储
> (B) 数据元素是单个字符
> (C) 可以链式存储
> (D) 数据元素可以是多个字符

**答案**：B

**解析**：串（字符串）是由零个或多个字符组成的有限序列。它是线性表的一种，但数据元素限定为单个字符。一般线性表的数据元素可以是任意类型，而串的元素只能是字符。

> (A) 空串长度为 0，空格串长度不为 0
> (B) 二者相同
> (C) 空串不可存储，空格串可以存储
> (D) 无区别

**答案**：A

**解析**：空串是长度为 0 的串（""），不含任何字符；空格串是由一个或多个空格字符组成的串（" "），长度 ≥ 1。空格是 ASCII 码为 32 的合法字符。

> (A) 不需要回溯主串指针
> (B) 时间复杂度总是 O(n)
> (C) 适用于任意字符集
> (D) 实现简单

**答案**：A

**解析**：KMP 算法的核心优势是主串指针 i 不回溯，模式串指针 j 根据 next 数组回退。朴素匹配算法中主串指针需要回溯，最坏时间复杂度 O(mn)；KMP 算法时间复杂度 O(m+n)。但 KMP 算法实现相对复杂，需要在 O(m) 时间内计算 next 数组。

> (A) 01123422

### 三、综合题


**答案**：

next 数组（next[1]=0）：

| j | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
|---|---|---|---|---|---|---|---|---|
| 模式串 | a | b | a | a | b | c | a | c |
| next[j] | 0 | 1 | 1 | 2 | 2 | 3 | 1 | 2 |

nextval 数组：

| j | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
|---|---|---|---|---|---|---|---|---|
| 模式串 | a | b | a | a | b | c | a | c |
| nextval[j] | 0 | 1 | 0 | 2 | 1 | 3 | 0 | 2 |

**解析**：nextval 是 next 的改进：若 p[j] == p[next[j]]，则 nextval[j] = nextval[next[j]]，否则 nextval[j] = next[j]。例如 j=3 时 p[3]='a' 等于 p[next[3]=1]='a'，所以 nextval[3]=nextval[1]=0。这避免了连续相同字符时的无效回退。


**答案**：

```c
int Index_BF(char S[], char T[]) {
    int i = 1, j = 1;  // 从下标 1 开始存储
    while (i <= strlen(S+1) && j <= strlen(T+1)) {
        if (S[i] == T[j]) {
            i++; j++;
        } else {
            i = i - j + 2;  // 主串回溯
            j = 1;          // 模式串回到开头
        }
    }
    if (j > strlen(T+1))
        return i - strlen(T+1);  // 匹配成功
    else
        return 0;  // 匹配失败
}
```

**解析**：BF（Brute Force）算法最坏时间复杂度 O(mn)（如 S="aaaaaaaaab"，T="aaab"）。i 回溯公式 i = i-j+2 的含义：i 回到本次匹配起始位置的下一个位置。

```c
void BFS(ALGraph G, int v) {
    int visited[MAXV] = {0};
    int queue[MAXV], front = 0, rear = 0;
    printf("%d ", v);
    visited[v] = 1;
    queue[++rear] = v;
    while (front != rear) {
        int w = queue[++front];
        ArcNode *p = G.adjlist[w].firstarc;
        while (p != NULL) {
            if (!visited[p->adjvex]) {
                printf("%d ", p->adjvex);
                visited[p->adjvex] = 1;
                queue[++rear] = p->adjvex;
            }
            p = p->nextarc;
        }
    }
}
```

**答案**：

```c
void Dijkstra(MGraph G, int v0, int dist[], int path[]) {
    int final[MAXV] = {0};
    int min, v, w;

    // 初始化
    for (v = 0; v < G.n; v++) {
        final[v] = 0;
        dist[v] = G.edge[v0][v];
        path[v] = (dist[v] < INF && v != v0) ? v0 : -1;
    }
    dist[v0] = 0;
    final[v0] = 1;

    // 主循环
    for (int i = 1; i < G.n; i++) {
        min = INF;
        for (w = 0; w < G.n; w++) {
            if (!final[w] && dist[w] < min) {
                min = dist[w];
                v = w;
            }
        }
        final[v] = 1;

        for (w = 0; w < G.n; w++) {
            if (!final[w] && dist[v] + G.edge[v][w] < dist[w]) {
                dist[w] = dist[v] + G.edge[v][w];
                path[w] = v;
            }
        }
    }
}
```

**解析**：Dijkstra 算法核心步骤：①初始化 dist 数组（源点到各点的直接距离）；②每次选择未访问且 dist 最小的顶点 v，将其标记为已访问；③以 v 为中间点，更新所有未访问邻接顶点的 dist。时间复杂度 O(n²)。path 数组记录最短路径的前驱，可用于回溯完整路径。

## 第八章 排序

### 一、选择题

> (A) 算法时间复杂度稳定
> (B) 相同关键字的相对位置在排序前后不变
> (C) 不会改变原有顺序
> (D) 以上都是

**答案**：B

**解析**：稳定性：若待排序序列中有两个关键字相同的记录 Ri 和 Rj（i<j），排序后 Ri 仍在 Rj 之前，则称该排序算法是稳定的。稳定排序：冒泡、插入、归并、基数；不稳定排序：快排、堆排、希尔、选择。

> (B) O(n²)
> (C) O(n)
> (D) O(log n)

**答案**：B

**解析**：快速排序在最好和平均情况下为 O(n log n)，最坏情况（每次划分极不平衡，如序列已有序且每次选第一个为基准）退化为 O(n²)。可通过随机选择基准或三数取中法避免最坏情况。

> (B) O(n log n)
> (C) O(n²)
> (D) O(log n)

**答案**：B

**解析**：堆排序建堆 O(n)，每次调整 O(log n)，共 n-1 次调整，总时间复杂度 O(n log n)。无论最好、最坏、平均情况，堆排序时间复杂度始终为 O(n log n)。但堆排序不稳定，且由于跳跃访问，实际性能通常不如快排。

> (B) O(log n)
> (C) O(n)
> (D) O(n²)

**答案**：C

**解析**：归并排序需要与原始序列等长的辅助数组来暂存合并结果，空间复杂度 O(n)。这是归并排序的主要缺点。时间复杂度 O(n log n)，且是稳定排序。

> (B) 堆排序
> (C) 希尔排序
> (D) 归并排序

**答案**：D

**解析**：归并排序在合并两个有序序列时，若遇到相等元素，先取左侧的元素，保证稳定性。快排的划分过程跳跃式交换，堆排序的堆调整涉及跳跃，希尔排序的分组插入，均不稳定。口诀：快堆希选不稳定，冒插归基稳定。

> (D) 希尔排序

→ {97, 76, 65, 49, 49, 13, 27, 38}

```
    /  \   / \

  /
 38
```

## 第一章 绪论（补充题）

### 一、选择题

> (A) 数据项
> (B) 数据元素
> (C) 数据对象

**解析**：数据元素是数据的基本单位。数据项是数据的不可分割的最小单位。数据对象是相同性质的数据元素的集合。注意三者的层次关系：数据结构 > 数据对象 > 数据元素 > 数据项。

**解析**：逻辑结构分为四类：集合、线性、树形、图状。顺序结构是**存储结构**（物理结构）的一种，不是逻辑结构。常见错误是把"顺序"当成逻辑结构。**逻辑结构**关注的是数据元素之间的逻辑关系，**存储结构**关注的是数据在计算机中的存储方式。

**答案**：D

**解析**：时间复杂度是算法执行时间与问题规模的**函数关系**，反映算法的**渐近行为**。它与机器的时钟频率无关（频率影响实际执行时间，但不改变算法复杂度）。算法的复杂度由算法本身决定。

> (D) 数据的存储结构独立于逻辑结构

**答案**：B

**解析**：数据的**逻辑结构**是面向用户的，反映数据元素之间的**逻辑关系**；**存储结构**（物理结构）是面向计算机的，是逻辑结构在计算机中的**映像（表示）**。同一逻辑结构可对应不同的存储结构（线性表 → 顺序表/链表）。但存储结构**依赖于**逻辑结构（要正确反映逻辑关系）。

```c
    if (k <= 0 || L == NULL) return NULL;
    LNode *fast = L->next, *slow = L->next;
    // fast 先走 k-1 步
```

```c
    LNode *slow = L, *fast = L;
    while (fast != NULL && fast->next != NULL) {
```

```c
void MoveOdd(SqList &L) {
    int k = 0;  // k 指向已排好序的奇数区末尾
    for (int i = 0; i < L.length; i++) {
        if (L.data[i] % 2 != 0) {  // 奇数
            // 将 data[i] 插入到 k 位置（k 之前都是奇数）
            ElemType temp = L.data[i];
            for (int j = i; j > k; j--)
                L.data[j] = L.data[j-1];
            L.data[k] = temp;
            k++;
        }
    }
}
```

**解析**：插入排序思想。`k` 维护"已排好序的奇数区"边界。遇到奇数时，将其插入到 `k` 位置，同时后移 `k..i-1` 区间。时间复杂度 $O(n^2)$，空间复杂度 $O(1)$。**注**：若不要求稳定，可用双指针 $O(n)$ 实现。

```c
void Split(LinkList L, LinkList &L1, LinkList &L2) {
    L1 = (LinkList)malloc(sizeof(LNode)); L1->next = NULL;
    L2 = (LinkList)malloc(sizeof(LNode)); L2->next = NULL;
    LNode *p = L->next, *p1 = L1, *p2 = L2;
    ElemType x;  // x 由调用者设定
    while (p != NULL) {
        LNode *q = p->next;  // 暂存后继
        if (p->data < x) {
            p1->next = p; p1 = p;
        } else {
            p2->next = p; p2 = p;
        }
        p = q;
    }
    p1->next = NULL;  // 切断
    p2->next = NULL;
}
```

**解析**：遍历原链表，按 `p->data < x` 判别，分别挂到 `L1`、`L2` 尾部。**注意**：必须保存 `p->next`（因为 `p` 即将脱离原链），并最后将新链表的尾结点 next 置 NULL。时间复杂度 $O(n)$，空间复杂度 $O(1)$。

```c
    Stack in;   // 入队栈
    Stack out;  // 出队栈
} MyQueue;

}

ElemType deQueue(MyQueue *q) {
    // 若 out 栈为空，将 in 栈全部倒入 out 栈
    if (StackEmpty(&q->out)) {
        while (!StackEmpty(&q->in)) {
            ElemType x = Pop(&q->in);
            Push(&q->out, x);
        }
    }
    if (StackEmpty(&q->out)) return ERROR;  // 队列空
    return Pop(&q->out);
}
```

```c
void InfixToPostfix(char *infix, char *postfix) {
    Stack s; InitStack(&s);
    int j = 0;
    for (int i = 0; infix[i] != ' '; i++) {
        char c = infix[i];
        if (isdigit(c)) {  // 操作数
            postfix[j++] = c;
        } else if (c == '(') {
            Push(&s, c);
        } else if (c == ')') {
            while (GetTop(&s) != '(') {
                postfix[j++] = Pop(&s);
            }
            Pop(&s);  // 弹出 '('

            while (!StackEmpty(&s) && GetTop(&s) != '(' && 
                   Priority(GetTop(&s)) >= Priority(c)) {
                postfix[j++] = Pop(&s);
            }
            Push(&s, c);
        }
    }
    while (!StackEmpty(&s)) {
        postfix[j++] = Pop(&s);
    }
    postfix[j] = ' ';
}
```

```c
typedef struct {
    Stack data;  // 数据栈
    Stack min;   // 辅助栈（栈顶是当前最小值）
} MinStack;

void push(MinStack *s, ElemType x) {
    Push(&s->data, x);
    if (StackEmpty(&s->min) || x <= GetTop(&s->min)) {
        Push(&s->min, x);
    }
}

ElemType pop(MinStack *s) {
    ElemType x = Pop(&s->data);
    if (x == GetTop(&s->min)) {
        Pop(&s->min);
    }
    return x;
}

ElemType top(MinStack *s) {
    return GetTop(&s->data);
}

ElemType getMin(MinStack *s) {
    return GetTop(&s->min);
}
```

**解析**：**单调栈**应用。辅助栈与数据栈**同步**维护最小值。`push(x)`：x 小于等于辅助栈顶时才入辅助栈（保证辅助栈顶始终是当前最小值）。`pop`：数据栈和辅助栈顶相等时同步弹出。`getMin`：直接返回辅助栈顶。时间复杂度 $O(1)$，空间复杂度 $O(n)$。

```c
        if (S[i] == T[j]) {
            i++; j++;
        } else {
            i = i - j + 1;  // i 回退
```

```c
void LevelOrder(BiTree T) {
    if (T == NULL) return;
    Queue q; InitQueue(&q);
    EnQueue(&q, T);
    while (!QueueEmpty(&q)) {
        BiTree p = DeQueue(&q);
        Visit(p);
        if (p->lchild) EnQueue(&q, p->lchild);
        if (p->rchild) EnQueue(&q, p->rchild);
    }
}
```

**解析**：**层序遍历**借助**队列**实现：① 根入队；② 循环：出队、访问、子结点入队。**关键**：先访问再入队子结点，保证按层顺序。时间复杂度 $O(n)$，空间复杂度 $O(n)$（队列最坏存一层结点）。


**答案**：

```c
void InOrderNonRec(BiTree T) {
    Stack s; InitStack(&s);
    BiTree p = T;
    while (p != NULL || !StackEmpty(&s)) {
        if (p != NULL) {
            Push(&s, p);
            p = p->lchild;  // 一路向左
        } else {
            p = Pop(&s);     // 弹出栈顶
            Visit(p);        // 访问根
            p = p->rchild;   // 转向右子树
        }
    }
}
```

**解析**：**非递归中序遍历**思路：① 沿左子树链一路下入栈；② 到达最左后弹出访问；③ 转向右子树重复。**关键**：用一个指针 `p` 配合栈，明确"何时入栈"、"何时出栈访问"、"何时转向右子树"。时间复杂度 $O(n)$，空间复杂度 $O(h)$。


**答案**：

构造过程：
- 取最小两个 `2, 3` 合并为 `5`，得到新权值 `{5, 5, 7, 11, 13}`（原来的 5 和新建的 5）
- 取最小两个 `5, 5` 合并为 `10`，得到 `{7, 10, 11, 13}`
- 取最小两个 `7, 10` 合并为 `17`，得到 `{11, 13, 17}`
- 取最小两个 `11, 13` 合并为 `24`，得到 `{17, 24}`
- 取 `17, 24` 合并为 `41`（根）

WPL = 2×3 + 3×3 + 5×2 + 7×2 + 11×2 + 13×2 = 6+9+10+14+22+26 = 87

**解析**：**哈夫曼树构造**：每次取两个最小权值的结点合并，新结点权值为两者之和。**WPL = $\sum w_i \times l_i$**，其中 $l_i$ 是叶子到根的路径长度。WPL 越小编码越短。


**答案**：

**转换规则**：
- 每个结点的**长子**（最左孩子）作为该结点的**左子**；
- 该结点的**其他孩子**（兄弟）依次**右链**相连（右子的右子的右子...）。

关系：
- 树的**前序遍历** = 二叉树的**前序遍历**
- 树的**后序遍历** = 二叉树的**中序遍历**

**解析**：**树 → 二叉树**的转换（孩子兄弟表示法）：长子变左子，兄弟变右子。**重要结论**：树的前序对应二叉树的前序，树的后序对应二叉树的中序。这是 408 高频考点。


**答案**：

```c
BiTree BSTSearch(BiTree T, ElemType key) {
    BiTree p = T;
    while (p != NULL) {
        if (key == p->data) return p;
        else if (key < p->data) p = p->lchild;
        else p = p->rchild;
    }
    return NULL;
}
```

**解析**：BST 查找利用**有序性**：与根比较，小于根走左，大于根走右，等于返回。**时间复杂度**：平均 $O(\log n)$，最坏 $O(n)$（退化为链表）。**空间复杂度 $O(1)$**（非递归版）。

**答案**：

```c
void DFS(GraphAdjList G, int v) {
```

**答案**：

```c
void Dijkstra(Graph G, int v0, int dist[], int path[]) {
    bool final[N];  // 标记已确定最短路径
    for (int i = 0; i < G.n; i++) {
        dist[i] = G.edge[v0][i];
        final[i] = false;
        path[i] = (dist[i] < INF) ? v0 : -1;
    }
    dist[v0] = 0; final[v0] = true;
    for (int i = 1; i < G.n; i++) {
        // 找未确定中 dist 最小的
        int min = INF, u = v0;
        for (int j = 0; j < G.n; j++)
            if (!final[j] && dist[j] < min) {
                min = dist[j]; u = j;
            }
        final[u] = true;
        // 更新邻接点
        for (int k = 0; k < G.n; k++) {
            if (!final[k] && dist[u] + G.edge[u][k] < dist[k]) {
                dist[k] = dist[u] + G.edge[u][k];
                path[k] = u;
            }
        }
    }
}
```

**解析**：**Dijkstra 算法**：① 初始化：dist = 直接边长，final 全 false；② 循环 n-1 次：找未确定最小 dist 的顶点 u，标 final，松弛其邻接点。**不能处理负权边**。时间复杂度 $O(n^2)$（邻接矩阵版）。


**答案**：

```c
bool TopoSort(GraphAdjList G) {
    int inDegree[MAXV] = {0};
    for (int i = 0; i < G.n; i++)  // 计算入度
        for (ArcNode *p = G.adjlist[i].firstarc; p; p = p->nextarc)
            inDegree[p->adjvex]++;
    Stack s; InitStack(&s);
    for (int i = 0; i < G.n; i++)
        if (inDegree[i] == 0) Push(&s, i);
    int cnt = 0;
    while (!StackEmpty(&s)) {
        int v = Pop(&s);
        topo[cnt++] = v;  // 输出
        for (ArcNode *p = G.adjlist[v].firstarc; p; p = p->nextarc) {
            int k = p->adjvex;
            if (--inDegree[k] == 0) Push(&s, k);
        }
    }
    return cnt == G.n;  // cnt == n 则无环
}
```

**解析**：**Kahn 算法**（基于 BFS/入度）：① 计算所有顶点入度；② 入度为 0 的入栈；③ 循环：弹栈，输出，邻接点入度 -1，若为 0 入栈；④ 最后输出的顶点数 == 总结点数 → 无环，否则有环。**关键**：入度为 0 才是当前"无前驱"的顶点，可以输出。

```c
            if (!inMST[j] && G.edge[k][j] < lowcost[j]) {
                lowcost[j] = G.edge[k][j];
                closest[j] = k;
            }
    }
}
```

**解析**：**Prim 算法**：从一个顶点出发，逐步扩展 MST。每次找"未在 MST 中且到 MST 边权最小"的顶点加入，更新其他顶点的 lowcost。**贪心**思想保证 MST 最优。**时间复杂度 $O(n^2)$**，适合稠密图。

```c
    if (left >= right) return true;
    int root = arr[right];

    while (i < right && arr[i] < root) i++;
```

### 三、综合题


**答案**：

```c
int Partition(int arr[], int low, int high) {
    int pivot = arr[low];  // 以第一个元素为 pivot
    while (low < high) {
        while (low < high && arr[high] >= pivot) high--;
        arr[low] = arr[high];
        while (low < high && arr[low] <= pivot) low++;
        arr[high] = arr[low];
    }
    arr[low] = pivot;
    return low;
}

void QuickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = Partition(arr, low, high);
        QuickSort(arr, low, pi - 1);
        QuickSort(arr, pi + 1, high);
    }
}
```

**解析**：**Partition** 用 Hoare 法：选 pivot，两端向中间扫，将小元素移左、大元素移右，最终 pivot 到位。**QuickSort** 递归处理左右子数组。**平均 $O(n \log n)$**，**最坏 $O(n^2)$**（如已排序 + 选首元素为 pivot）。**改进**：随机选 pivot、三数取中。


**答案**：

```c
void Merge(int arr[], int L, int M, int R) {
    int left_size = M - L, right_size = R - M + 1;
    int Left[left_size], Right[right_size];
    for (int i = 0; i < left_size; i++) Left[i] = arr[L + i];
    for (int j = 0; j < right_size; j++) Right[j] = arr[M + j];
    int i = 0, j = 0, k = L;
    while (i < left_size && j < right_size) {
        if (Left[i] <= Right[j]) arr[k++] = Left[i++];
        else arr[k++] = Right[j++];
    }
    while (i < left_size) arr[k++] = Left[i++];
    while (j < right_size) arr[k++] = Right[j++];
}

void MergeSort(int arr[], int L, int R) {
    if (L < R) {
        int M = (L + R) / 2;
        MergeSort(arr, L, M);
        MergeSort(arr, M + 1, R);
        Merge(arr, L, M + 1, R);
    }
}
```

**解析**：**归并排序** = 分治。**分**：将数组分两半递归排序；**治**：合并两个有序子数组。**时间复杂度始终 $O(n \log n)$**，**空间复杂度 $O(n)$**（需要辅助数组）。**稳定排序**。


**答案**：

```c
void SiftDown(int arr[], int n, int i) {
    int largest = i;
    int l = 2*i + 1, r = 2*i + 2;
    if (l < n && arr[l] > arr[largest]) largest = l;
    if (r < n && arr[r] > arr[largest]) largest = r;
    if (largest != i) {
        swap(&arr[i], &arr[largest]);
        SiftDown(arr, n, largest);
    }
}

void HeapSort(int arr[], int n) {
    // 建堆
    for (int i = n/2 - 1; i >= 0; i--)
        SiftDown(arr, n, i);
    // 排序
    for (int i = n - 1; i > 0; i--) {
        swap(&arr[0], &arr[i]);
        SiftDown(arr, i, 0);
    }
}
```

**解析**：**堆排序**：① 建堆（自底向上）$O(n)$；② 排序：交换堆顶与末尾，堆大小减 1，sift down $O(\log n)$，n-1 次共 $O(n \log n)$。**总时间 $O(n \log n)$**，**空间 $O(1)$**。**不稳定**（交换可能改变相同元素相对位置）。


**答案**：

```c
void ShellSort(int arr[], int n) {
    for (int dk = n/2; dk >= 1; dk /= 2) {  // 增量序列
        // 插入排序（增量为 dk）
        for (int i = dk; i < n; i++) {
            int temp = arr[i];
            int j = i;
            while (j >= dk && arr[j - dk] > temp) {
                arr[j] = arr[j - dk];
                j -= dk;
            }
            arr[j] = temp;
        }
    }
}
```

**解析**：**希尔排序** = 插入排序 + 缩小增量。每次按增量 $d$ 分组，组内做插入排序；$d$ 从大到小（$n/2, n/4, ..., 1$）。**最末** $d=1$ 时退化为插入排序（此时已基本有序）。**时间复杂度**取决于增量序列，平均 $O(n^{1.3})$。**不稳定**。

### 一、第一章 绪论 冲刺题

> ```c
> int i = n, s = 0;
> while (s < n) {
>     s += i;
>     i--;
> }
> ```
> A. $O(n)$
> B. $O(\log n)$
> C. $O(\sqrt{n})$
> D. $O(n^2)$

**答案**：C

**解析**：循环条件 `s < n`，而 `s` 每次增加 `i`，`i` 从 n 递减到 0。所以 `s = n + (n-1) + ... + 1 = n(n+1)/2 ≈ n^2/2`。需要 `s ≥ n`，即 `n(n+1)/2 ≥ n`，解得 $i \geq \sqrt{n}$ 量级。**时间复杂度 $O(\sqrt{n})$**。

> A. $O(n^2)$
> B. $O(n^2 \log n)$
> C. $O(n^3)$
> D. $O(n^{1.5})$

**答案**：A

**解析**：主定理：$a=3, b=3, n^{\log_b a} = n$。$f(n) = n^2$。$f(n) = \Omega(n^{\log_b a + \epsilon})$，$\epsilon = 1 > 0$。检查 $af(n/b) \leq cf(n)$：$3(n/3)^2 = n^2/3 \leq c \cdot n^2$（取 $c = 1/3 < 1$），成立。**结论**：$T(n) = \Theta(f(n)) = O(n^2)$。

> A. $O(n^2)$
> B. $O(n^2 \log n)$
> C. $O(n^3)$
> D. $O(n^{1.5})$

**答案**：B

**解析**：主定理：$a=4, b=2, n^{\log_b a} = n^2$。$f(n) = n^2 = \Theta(n^{\log_b a})$，属**情况 2**。$T(n) = \Theta(n^{\log_b a} \log n) = \Theta(n^2 \log n) = O(n^2 \log n)$。

### 二、第二章 线性表 冲刺题

>
> **答案**：
>
> ```c
> void Reverse(LinkList &L) {
>     LNode *p = L->next, *q;
>     L->next = NULL;
>     while (p != NULL) {
>         q = p->next;
>         p->next = L->next;
>         L->next = p;
>         p = q;
>     }
> }
> ```
>
> **解析**：经典"头插法"逆置。**关键**：必须先保存 `p->next`（即 q），再修改 `p->next`，否则断链。时间复杂度 $O(n)$，空间复杂度 $O(1)$。


**答案**：

```c
LinkList Merge(LinkList La, LinkList Lb) {
    LNode *pa = La->next, *pb = Lb->next, *pc;
    La->next = NULL;  // 切断 La（用作 Lc）
    pc = La;
    while (pa && pb) {
        if (pa->data <= pb->data) {
            pc->next = pa;
            pc = pa;
            pa = pa->next;
        } else {
            pc->next = pb;
            pc = pb;
            pb = pb->next;
        }
    }
    pc->next = pa ? pa : pb;
    free(Lb);
    return La;
}
```

**解析**：归并思想的链表版。**关键**：用 La 的头结点作为 Lc 的头结点，最后释放 Lb 的头结点。时间 $O(m+n)$，空间 $O(1)$。


**答案**：

```c
void DelMin(LinkList &L) {
    LNode *pre = L, *p = pre->next;
    LNode *minPre = pre, *minP = p;
    while (p != NULL) {
        if (p->data < minP->data) {
            minP = p;
            minPre = pre;
        }
        pre = p;
        p = p->next;
    }
    minPre->next = minP->next;
    free(minP);
}
```

**解析**：**双指针 + 最小值追踪**。`pre`、`p` 是当前结点及其前驱；`minPre`、`minP` 是当前最小结点及其前驱。遍历一次找到最小值，再删除。时间 $O(n)$，空间 $O(1)$。


**答案**：

```c
void Unique(LinkList &L) {
    LNode *p = L->next, *q;
    while (p != NULL) {
        q = p;
        while (q->next != NULL) {
            if (q->next->data == p->data) {
                LNode *r = q->next;
                q->next = r->next;
                free(r);
            } else {
                q = q->next;
            }
        }
        p = p->next;
    }
}
```

**解析**：**双重循环 + 释放**。外层 p 遍历每个结点，内层 q 遍历 p 之后的所有结点，若值等于 p->data 则删除。**时间复杂度 $O(n^2)$**。若先排序再去重可优化为 $O(n \log n)$。

### 三、第三章 栈和队列 冲刺题


**答案**：

```c
int EvalInfix(char *exp) {
    Stack opnd, optr;  // 操作数栈、运算符栈
    InitStack(&opnd); InitStack(&optr);
    Push(&optr, '#');  // 表达式起始符
    Push(&optr, exp[0]);
    // ... 中缀求值主体（参考教材）
    // 1. 若是操作数：进 opnd
    // 2. 若是运算符：与 optr 栈顶比较优先级
    //    栈顶优先级低：当前运算符进 optr
    //    栈顶优先级高：弹 optr 栈顶，弹 opnd 两个操作数，计算结果进 opnd
    //    栈顶等于：弹 optr 栈顶（用于左括号匹配）
    return GetTop(&opnd);
}
```

**解析**：**中缀求值**是栈的经典综合应用。**关键**：定义运算符优先级，'(' 优先级最高（先入栈），'#' 用于结尾。**双栈法**：操作数栈存中间值，运算符栈存待处理运算符。


**答案**：

```c
typedef struct {
    Deque data;  // 双端队列存数据
    Deque maxd;  // 单调双端队列存最大值
} MaxQueue;

void push_back(MaxQueue *q, ElemType x) {
    push_back_deque(&q->data, x);
    while (!empty_deque(&q->maxd) && back_deque(&q->maxd) < x)
        pop_back_deque(&q->maxd);
    push_back_deque(&q->maxd, x);
}

ElemType pop_front(MaxQueue *q) {
    ElemType x = pop_front_deque(&q->data);
    if (x == front_deque(&q->maxd))
        pop_front_deque(&q->maxd);
    return x;
}

ElemType getMax(MaxQueue *q) {
    return front_deque(&q->maxd);
}
```

**解析**：**单调队列**应用。`maxd` 维护单调递减的双端队列，队首是当前最大值。`push_back` 时弹出 `maxd` 末尾所有小于 x 的元素，将 x 入队尾；`pop_front` 时若弹出的元素等于 maxd 队首，则同步弹出。


**答案**：

```c
void ReverseQueue(Queue &Q) {
    Stack s; InitStack(&s);
    while (!QueueEmpty(&Q)) {
        Push(&s, DeQueue(&Q));
    }
    while (!StackEmpty(&s)) {
        EnQueue(&Q, Pop(&s));
    }
}
```

**解析**：**队列 + 栈**实现队列倒置。先全部出队入栈（栈顶是最后入队的元素），再全部出栈入队（队头是栈顶，即最后入队的）。时间 $O(n)$，空间 $O(n)$（栈）。**注**：仅用队列（无栈）也可以，但需要用 `size` 模拟递归。

### 五、第五章 树与二叉树 冲刺题


**答案**：

```c
void Mirror(BiTree &T) {
    if (T == NULL) return;
    swap(T->lchild, T->rchild);
    Mirror(T->lchild);
    Mirror(T->rchild);
}
```

**解析**：**递归翻转**。先交换当前结点的左右子树，再递归翻转左右子树。**时间复杂度 $O(n)$**（每个结点访问一次），**空间复杂度 $O(h)$**（递归栈深度，h 为树高）。


**答案**：

```c
bool isSame(BiTree T1, BiTree T2) {
    if (T1 == NULL && T2 == NULL) return true;
    if (T1 == NULL || T2 == NULL) return false;
    if (T1->data != T2->data) return false;
    return isSame(T1->lchild, T2->lchild) && isSame(T1->rchild, T2->rchild);
}
```

**解析**：**递归比较**。空 + 空 → 相同；一空一不空 → 不同；值不同 → 不同；值相同 → 递归比较左右子树。**时间复杂度 $O(n)$**。


**答案**：

```c
BiTree Delete(BiTree &T, ElemType key) {
    if (T == NULL) return NULL;
    if (key < T->data) T->lchild = Delete(T->lchild, key);
    else if (key > T->data) T->rchild = Delete(T->rchild, key);
    else {  // 找到要删除的结点
        if (T->lchild == NULL) {
            BiTree p = T; T = T->rchild; free(p);
        } else if (T->rchild == NULL) {
            BiTree p = T; T = T->lchild; free(p);
        } else {  // 左右子树都不空
            // 找右子树的最小值（或左子树的最大值）替代
            BiTree p = T->rchild;
            while (p->lchild != NULL) p = p->lchild;
            T->data = p->data;
            T->rchild = Delete(T->rchild, p->data);
        }
    }
    return T;
}
```

**解析**：**BST 删除**三种情况：① 叶子：直接删；② 只有一个子：用子替代；③ 都有子：用**右子树最小值**（或左子树最大值）替代该结点，再删除替代结点。**关键**：替代后要保证 BST 性质。

**答案**：

```c
bool hasCycleUndirected(GraphAdjList G) {
    bool visited[MAXV] = {false};
    for (int v = 0; v < G.n; v++) {
        if (!visited[v]) {
            if (DFS_Cycle(G, v, -1, visited)) return true;
        }
    }
    return false;
}

bool DFS_Cycle(GraphAdjList G, int v, int parent, bool visited[]) {
    visited[v] = true;
    for (ArcNode *p = G.adjlist[v].firstarc; p; p = p->nextarc) {
        int k = p->adjvex;
        if (!visited[k]) {
            if (DFS_Cycle(G, k, v, visited)) return true;
        } else if (k != parent) {
            return true;  // 访问到已访问且非父结点 → 有环
        }
    }
    return false;
}
```

**解析**：**DFS 判断无向图环**。DFS 过程中，若访问到一个**已访问但不是父结点**的顶点，则有环。**关键**：用 `parent` 区分"回头"（合法）和"环"（非法）。

```c
BiTree Insert(BiTree &T, ElemType key) {
```

### 八、第八章 排序 冲刺题


**答案**：

```c
void CountSort(int arr[], int n, int k) {  // k 是关键字范围 [0, k)
    int count[k];
    int output[n];
    memset(count, 0, sizeof(count));
    for (int i = 0; i < n; i++) count[arr[i]]++;
    for (int i = 1; i < k; i++) count[i] += count[i-1];  // 累计
    for (int i = n - 1; i >= 0; i--) {
        output[--count[arr[i]]] = arr[i];
    }
    for (int i = 0; i < n; i++) arr[i] = output[i];
}
```

**解析**：**计数排序**非比较排序。① 计数每个值出现次数；② 累计（前缀和）得到每个值的最后位置；③ 反向填充 output 保持稳定性。**时间 $O(n + k)$**，**空间 $O(n + k)$**。**稳定**。**适用**：关键字范围 k 较小的情况。


**答案**：

```c
void RadixSort(int arr[], int n) {
    int max = arr[0];
    for (int i = 1; i < n; i++) if (arr[i] > max) max = arr[i];
    for (int exp = 1; max / exp > 0; exp *= 10) {
        // 按 exp 位（个/十/百...）计数排序
        int output[n], count[10] = {0};
        for (int i = 0; i < n; i++) count[(arr[i] / exp) % 10]++;
        for (int i = 1; i < 10; i++) count[i] += count[i-1];
        for (int i = n - 1; i >= 0; i--)
            output[--count[(arr[i] / exp) % 10]] = arr[i];
        for (int i = 0; i < n; i++) arr[i] = output[i];
    }
}
```

**解析**：**基数排序**：按位数（个、十、百...）依次做**稳定**的计数排序。**时间复杂度 $O(d \times (n + k))$**，$d$ 是位数，$k$ 是基数（10）。**稳定**。**空间 $O(n + k)$**。


**答案**：

```c
void MoveOdd(SqList &L) {
    int left = 0, right = L.length - 1;
    while (left < right) {
        // 左边找偶数
        while (left < right && L.data[left] % 2 != 0) left++;
        // 右边找奇数
        while (left < right && L.data[right] % 2 == 0) right--;
        if (left < right) {
            swap(L.data[left], L.data[right]);
            left++; right--;
        }
    }
}
```

```c
int Majority(SqList L) {
    int candidate = L.data[0], count = 1;
    // 阶段1：找候选
    for (int i = 1; i < L.length; i++) {
        if (count == 0) {
            candidate = L.data[i];
            count = 1;
        } else if (L.data[i] == candidate) {
            count++;
        } else {
            count--;
        }
    }
    // 阶段2：验证候选
    if (count == 0) return -1;
    count = 0;
    for (int i = 0; i < L.length; i++)

    return (count > L.length / 2) ? candidate : -1;
}
```

```c
bool Merge(SqList &A, SqList &B) {
    if (A.length + B.length > MAXSIZE) return false;
    int i = A.length - 1, j = B.length - 1, k = A.length + B.length - 1;
    // B 复制到 A 末尾
    for (int x = 0; x <= j; x++) A.data[A.length + x] = B.data[x];
    // 从后向前合并
    while (i >= 0 && j >= 0) {
        if (A.data[i] > B.data[j]) A.data[k--] = A.data[i--];
        else A.data[k--] = B.data[j--];
    }
    while (j >= 0) A.data[k--] = B.data[j--];
    A.length += B.length;
    return true;
}
```

**解析**：**逆序合并**。先把 B 复制到 A 末尾，从后向前取大者，避免覆盖。**时间 $O(m + n)$**，**空间 $O(1)$**（不开额外数组）。

```c
void Tarjan(GraphAdjList G, int u) {
    dfn[u] = low[u] = ++Time;
    Push(&S, u);
    inStack[u] = true;
    for (ArcNode *p = G.adjlist[u].firstarc; p; p = p->nextarc) {
        int v = p->adjvex;
        if (!dfn[v]) {
            Tarjan(G, v);
            low[u] = min(low[u], low[v]);
        } else if (inStack[v]) {
            low[u] = min(low[u], dfn[v]);
        }
    }

        sccCount++;
        int w;
        do { Pop(&S, w); inStack[w] = false; } while (w != u);
    }
}
```

```c
    bool adj[MAXV][MAXV] = {false};
    // 标记 G 的边
    for (int u = 0; u < G.n; u++) {
        for (ArcNode *p = G.adjlist[u].firstarc; p; p = p->nextarc) {
            adj[u][p->adjvex] = true;
            adj[p->adjvex][u] = true;  // 无向图对称
        }
    }
    // 构建补图
    InitGraph(&G2, G.n);
    for (int u = 0; u < G.n; u++) {
        for (int v = u + 1; v < G.n; v++) {
            if (!adj[u][v]) AddEdge(&G2, u, v);
        }
    }
}
```

```c
    for (int i = 0; i < B.length; i++) A.data[A.length + i] = B.data[i];
```
