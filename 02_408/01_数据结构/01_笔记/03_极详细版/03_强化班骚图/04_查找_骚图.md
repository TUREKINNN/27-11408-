| # | 算法 | 时间复杂度 | 适用 | 关键 |

## 1️⃣ 顺序查找

### 🚀 哨兵版（避免越界检查）

```c
int Search_Seq(int a[], int n, int key) {
    a[0] = key;  // 哨兵
    int i = n;
    while (a[i] != key) i--;
    return i;     // 0 表示未找到
}
```

### 📊 ASL（平均查找长度）

```
等概率查找成功：ASL = (n+1)/2
```

## 2️⃣ 折半查找（二分查找）

### 🚀 递归 / 迭代

```c
int BinarySearch(int a[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (a[mid] == key) return mid;
        else if (a[mid] < key) low = mid + 1;
        else high = mid - 1;
    }
    return -1;  // 未找到
}
```

### 📊 ASL

```
ASL = log₂(n+1) - 1
判定树为完全二叉树
```

### 📊 折半查找判定树（n=10）

```
构造规则：
- mid = ⌊(low+high)/2⌋ 作为根
- 左子树的 mid 作为左子树的根
- 右子树的 mid 作为右子树的根

n=10, 数组 [0..9]：
         (0+9)/2=4
        /              (0+3)/2=1   (5+9)/2=7
      /    \      /        (0+0)/2=0 (2+3)/2=2 (5+6)/2=5 (8+9)/2=8
                  \                                (2+2)/2=2     (8+8)/2=8
                  ←重复←
```

> 1. mid 计算用 `low + (high - low) / 2` 防溢出  
> 2. low ≤ high 是条件（不是 low < high）  
> 3. 失败时返回 -1（不是 0）

## 3️⃣ 分块查找

### 📌 核心思想

> 将数据分成**若干块**，块间有序（前一块最大 < 后一块最小），块内无序  
> 索引表记录每块的最大关键字

### 📊 结构

```
数据：   [块1: 10 13 21] [块2: 25 28 36] [块3: 40 45 50]
索引表：   21              36              50
           ↓               ↓               ↓
```

### 📊 ASL

```
ASL = L_b + L_w
L_b = 块间查找长度（顺序/折半）
L_w = 块内查找长度（顺序）
最优块大小 = √n, ASL_min = √n + 1
```

## 5️⃣ B+ 树

### 📊 与 B 树的关键区别

| 维度 | B 树 | B+ 树 |
|------|------|-------|
| 数据位置 | 节点都存 | **只在叶子** |
| 叶子链表 | 无 | **链表连接** |
| 关键字数 | n 个关键字 n+1 个孩子 | n 个关键字 n 个孩子 |
| 查找 | 中间节点可能停 | **必须到叶子** |
| 范围查询 | 中序遍历 | **沿链表** |

### 📊 B+ 树结构

```
         [20 30 40]
        /    |    [10 15 20][25 30 35][40 45 50]   ← 叶子（含数据 + 链表）
  ↑       ↑       ↑       ↑
 ←─────→ ←─────→ ←─────→ ←─── 链表连接
```

## 6️⃣ Hash 查找

### 📌 核心思想

> **散列函数**：将关键字映射到数组下标  
> **冲突**：不同关键字映射到同一位置

### 📊 4 大散列函数

| 函数 | 公式 | 适用 |
|------|------|------|
| **直接定址** | H(k) = a*k + b | 关键字连续 |
| **除留余数** | H(k) = k % p（p 为素数） | 最常用 |
| **数字分析** | 取关键字某几位 | 关键字已知 |
| **平方取中** | 取关键字平方的中间几位 | 关键字分布未知 |

### 📊 4 大冲突处理

#### 1. 拉链法（链地址法）

```
数组下标  链表
0:  → 18 → 36 → NULL
1:  → 25 → NULL
2:  → NULL
3:  → 49 → NULL
4:  → NULL
```

> **特点**：实现简单，无堆积；**适合经常插入删除**

#### 2. 开放定址法

$$H_i = (H(key) + d_i) \% m$$

| 探测序列 | $d_i$ | 特点 |
|---------|-------|------|
| **线性探测** | 0, 1, 2, ... | 易堆积 |
| **平方探测** | 0, 1², -1², 2², -2², ... | 不能探测所有位置 |
| **双散列** | i * H2(key) | 灵活 |

#### 3. 公共溢出区

```
基本表：   [10, 25, , 49, ]  （H(10)=H(25)=0，用线性探测）
溢出表：   [10, 25]            （存冲突元素）
```

#### 4. 再散列

> 准备多个散列函数，冲突时换下一个

### 📊 ASL 计算（重点！）

#### 线性探测（等概率）

```
成功：ASL_succ = (1 + 1/(1-α))/2
失败：ASL_fail = (1 + 1/(1-α)²)/2
其中 α = 填装因子 = n/m
```

#### 拉链法（等概率）

```
成功：ASL_succ = 1 + 链长/2
失败：ASL_fail = 链长
```

## 🎯 查找·必背公式

### 1. ASL（平均查找长度）

```
ASL = Σ pᵢ × cᵢ
pᵢ = 查找第 i 个元素的概率
cᵢ = 查找第 i 个元素的比较次数
```

### 2. 折半查找 ASL

```
ASL = ⌈log₂(n+1)⌉ - 1
```

### 3. 散列表填装因子

```
α = n / m  （n=关键字数, m=表长）
α 越大 → 冲突越多
```

---

## ⚠️ 6 大易错点

> 1. ❌ 折半查找**必须有序**
> 2. ❌ 散列函数取模 `p` 选**素数**
> 3. ❌ 线性探测**易堆积**
> 4. ❌ 拉链法装填因子 α > 1 也可以
> 5. ❌ B+ 树和 B 树数据存储位置
> 6. ❌ 分块查找的**块间有序，块内无序**
