## 1️⃣ 图的存储

### 📊 4 大存储方式对比

| 存储 | 空间 | 找邻接点 | 找边 | 适用 |
|------|------|---------|------|------|
| **邻接矩阵** | O(V²) | O(V) | O(1) | 稠密图 |
| **邻接表** | O(V+E) | O(度数) | O(度数) | 稀疏图 |
| **十字链表** | O(V+E) | O(出度) | O(1)入边 | 有向图 |
| **邻接多重表** | O(V+E) | O(度数) | O(1) | 无向图 |

### 🚀 邻接矩阵

```c
// 邻接矩阵（适用于稠密图）
#define MAXV 100
typedef struct {
    int n;                      // 顶点数
    int edges[MAXV][MAXV];      // 邻接矩阵
} MGraph;

// 初始化
void InitMGraph(MGraph *G, int n) {
    G->n = n;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            G->edges[i][j] = 0;  // 0 表示无边
}
```

### 🚀 邻接表

```c
// 邻接表（适用于稀疏图）
typedef struct ArcNode {       // 边节点
    int adjvex;                 // 邻接点下标
    struct ArcNode *nextarc;    // 下一条边
    int info;                   // 权值等信息
} ArcNode;

typedef struct VexNode {       // 顶点节点
    int data;
    ArcNode *firstarc;          // 第一条边
} VexNode, AdjList[MAXV];

typedef struct {
    AdjList vertices;
    int n, e;                   // 顶点数、边数
} ALGraph;
```

### 📊 邻接表示例

```
无向图：
  1 - 2
  |   |
  3 - 4

邻接表：
[1] → 2 → 3 → NULL
[2] → 1 → 4 → NULL
[3] → 1 → 4 → NULL
[4] → 2 → 3 → NULL
```

## 2️⃣ 图的遍历

### 📊 DFS vs BFS 对比

| 维度 | DFS | BFS |
|------|-----|-----|
| **数据结构** | 栈（递归） | 队列 |
| **访问顺序** | 深入到底 | 层层推进 |
| **空间** | O(h) | O(宽度) |
| **最短路径** | ❌（无权图除外） | ✅（无权图最短） |
| **应用** | 拓扑、连通分量 | 最短路、层序 |

### 🚀 DFS（深度优先）

```c
void DFS(ALGraph G, int v) {
    visit[v] = true;            // 标记访问
    visit_func(v);
    ArcNode *p = G.vertices[v].firstarc;
    while (p) {
        if (!visit[p->adjvex])
            DFS(G, p->adjvex);  // 递归
        p = p->nextarc;
    }
}
```

### 🚀 BFS（广度优先）

```c
void BFS(ALGraph G, int v) {
    InitQueue(Q);
    visit[v] = true;
    EnQueue(Q, v);
    while (!IsEmpty(Q)) {
        DeQueue(Q, v);
        visit_func(v);
        ArcNode *p = G.vertices[v].firstarc;
        while (p) {
            if (!visit[p->adjvex]) {
                visit[p->adjvex] = true;
                EnQueue(Q, p->adjvex);
            }
            p = p->nextarc;
        }
    }
}
```

## 3️⃣ 最小生成树（MST）

### 📌 核心概念

> **生成树**：包含图的所有顶点，n-1 条边，**无环**的连通子图  
> **最小生成树**：边权之和最小的生成树

### 📊 Prim vs Kruskal

| 算法 | 思想 | 时间 | 适用 |
|------|------|------|------|
| **Prim** | 选最小边，扩展点集 | O(V²) 或 O(ElogV) | 稠密图 |
| **Kruskal** | 选最小边，不形成环 | O(ElogE) | 稀疏图 |

### 🚀 Prim 算法

```
初始：V = {1}，TE = {}

步骤：
1. 选 V 到 V-V 的最小边 (1, 2) → V = {1, 2}
2. 选 V 到 V-V 的最小边 (1, 3) → V = {1, 2, 3}
3. 选 V 到 V-V 的最小边 (3, 4) → V = {1, 2, 3, 4}
4. 选 V 到 V-V 的最小边 (2, 5) → V = {1, 2, 3, 4, 5}

结束：MST = {(1,2), (1,3), (3,4), (2,5)}
```

### 🚀 Kruskal 算法

```
初始：TE = {}，各顶点自成一个连通分量

步骤（按边权排序）：
1. 选 (1,2) 权 1，不形成环 → 加入
2. 选 (1,3) 权 2，不形成环 → 加入
3. 选 (2,5) 权 3，不形成环 → 加入
4. 选 (3,4) 权 4，不形成环 → 加入
5. 选 (4,5) 权 5，形成环 → 跳过

结束：MST = {(1,2), (1,3), (2,5), (3,4)}
```

## 4️⃣ 最短路径

### 📊 3 大算法对比

| 算法 | 单源/多源 | 负权 | 时间 | 适用 |
|------|---------|------|------|------|
| **Dijkstra** | 单源 | ❌ | O(V²) 或 O((V+E)logV) | 非负权图 |
| **Floyd** | 多源 | ✅（无负环） | O(V³) | 任意图 |
| **SPFA** | 单源 | ✅ | O(VE) | 负权图 |

### 🚀 Dijkstra 算法

```
求顶点 1 到其他所有顶点的最短路径

初始：dist[1] = 0, dist[其他] = ∞
      找到未访问中 dist 最小的点，更新其邻接点的 dist

步骤：
- 访问 1（dist=0），更新邻接点 2（dist=2）、3（dist=4）
- 访问 2（dist=2），更新 4（dist=2+3=5）
- 访问 3（dist=4），更新 4（min(5, 4+1)=5）→ 5 不变
- 访问 4（dist=5），更新 5（5+3=8）

结果：dist = [0, 2, 4, 5, 8]
```

> 负权边可能让"已确定最短"的点被更新

### 🚀 Floyd 算法

```c
// 三重循环
for (k = 0; k < n; k++)           // 中间点
    for (i = 0; i < n; i++)       // 起点
        for (j = 0; j < n; j++)   // 终点
            if (A[i][j] > A[i][k] + A[k][j])
                A[i][j] = A[i][k] + A[k][j];  // 更新最短路
```

> **逐步允许中间点**经过第 k 个点，更新 i 到 j 的最短路径

## 5️⃣ 拓扑排序

### 📌 核心思想

> AOV 网（有向无环图）→ 按**前驱后继关系**排成线性序列

### 🚀 拓扑排序算法

```c
void TopoSort(ALGraph G) {
    InitStack(S);
    for (i = 0; i < G.n; i++)
        if (indegree[i] == 0)    // 入度为 0 入栈
            Push(S, i);
    int cnt = 0;
    while (!IsEmpty(S)) {
        Pop(S, i);
        topo[cnt++] = i;         // 输出
        for (ArcNode *p = G.vertices[i].firstarc; p; p = p->nextarc) {
            if (--indegree[p->adjvex] == 0)
                Push(S, p->adjvex);
        }
    }
    if (cnt < G.n) printf("存在环！");
}
```

### 📊 拓扑排序示例

```
AOV 网：
    A → B → D
    A → C → D
    B → C

拓扑序列：
1. 入度 0：A
2. 删除 A，更新 B、C 入度
3. 入度 0：B
4. 删除 B，更新 C、D 入度
5. 入度 0：C
6. 删除 C，更新 D 入度
7. 入度 0：D

序列：A → B → C → D ✓
```

## 6️⃣ 关键路径

### 📌 核心概念

> AOE 网（边表示活动）→ 求**完成整个工程的最短时间**  
> **关键路径** = 最长路径（决定总工期）

### 📊 关键术语

| 术语 | 含义 | 公式 |
|------|------|------|
| **ve(i)** | 事件 i 最早发生 | `max(ve(k) + w(k,i))` |
| **vl(i)** | 事件 i 最迟发生 | `min(vl(k) - w(i,k))` |
| **e(a)** | 活动 a 最早开始 | `ve(i)` |
| **l(a)** | 活动 a 最迟开始 | `vl(j) - w(a)` |
| **关键活动** | e(a) == l(a) 的活动 | 关键路径上的活动 |

### 📊 关键路径示例

```
AOE 网：
v1 →[a1=3]→ v2 →[a2=2]→ v4
v1 →[a3=2]→ v3 →[a4=4]→ v4
v2 →[a5=3]→ v3

ve:
- v1 = 0
- v2 = 3
- v3 = max(0+2, 3+3) = 6
- v4 = max(3+2, 6+4) = 10

vl:
- v4 = 10
- v3 = 10 - 4 = 6
- v2 = min(6-3, 10-2) = 3
- v1 = min(3-3, 6-2) = 0

关键活动（e == l）：
- a3 (e=0, l=0) ✓
- a4 (e=6, l=6) ✓
- a2 (e=3, l=8) ✗
- a5 (e=3, l=3) ✓

关键路径：v1 → v3 → v4 (a3, a4)
总工期：10
```

## 🎯 图结构·必背公式

### 1. 邻接表空间

```
O(V + E)  ← 稀疏图优选
```

### 2. Dijkstra 时间

```
O(V²)        邻接矩阵
O(ElogV)     邻接表 + 优先队列
```

### 3. Floyd 时间

```
O(V³)  ← 适合 V ≤ 200
```

### 4. 拓扑排序存在性

```
排序结果 < n → 存在环
```

---

## ⚠️ 7 大易错点

> 1. ❌ 邻接表找邻接点是**顺着 firstarc 链表找**
> 2. ❌ DFS 用**栈**，BFS 用**队列**
> 3. ❌ Prim 与 Kruskal 时间复杂度搞混
> 4. ❌ Dijkstra 不能处理**负权**
> 5. ❌ Floyd 路径矩阵 P 的含义
> 6. ❌ 拓扑排序结果**不唯一**
> 7. ❌ 关键路径是**最长路径**（不是最短）
