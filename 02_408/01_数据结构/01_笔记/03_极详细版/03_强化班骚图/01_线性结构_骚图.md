## 1️⃣ 顺序表

### 📌 存储结构

```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│ a₁  │ a₂  │ a₃  │ ... │ aₙ  │     │ ← 连续内存
└─────┴─────┴─────┴─────┴─────┴─────┘
0     1     2     3     n-1   长度 N
```

### 📊 顺序表基本操作

| 操作 | 时间复杂度 | 关键点 |
|------|-----------|--------|
| 访问第 i 个元素 | O(1) | `data[i-1]` |
| 尾插 | O(1) | 摊销 |
| 中间插入 | O(n) | 移动后续元素 |
| 查找 | O(n) | 顺序扫描 |
| 排序 | O(nlogn) | 各种排序算法 |

### 🚀 动态顺序表（动态数组）

```c
typedef struct {
    ElemType *data;     // 动态数组指针
    int MaxSize;        // 最大容量
    int length;         // 当前长度
} SqList;

// 扩容策略（通常 2 倍）
void Expand(SqList *L) {
    L->data = realloc(L->data, 2 * L->MaxSize * sizeof(ElemType));
    L->MaxSize *= 2;
}
```

> 动态数组尾插的**摊销时间复杂度为 O(1)**，虽然扩容时为 O(n)，但 n 次操作总共 O(n)。

## 2️⃣ 链表

### 📊 4 大链表类型对比

#### 1. 单链表

```
head → [data|next] → [data|next] → [data|next] → NULL
       节点 1        节点 2        节点 3
```

#### 2. 双链表

```
NULL ← [prev|data|next] ⇄ [prev|data|next] ⇄ [prev|data|next] → NULL
        节点 1            节点 2            节点 3
```

#### 3. 循环单链表

```
head → [data|next] → [data|next] → [data|next]
       ↑                                │
       └────────────────────────────────┘
```

#### 4. 循环双链表

```
    ┌────────────────────────────────────┐
    ↓                                    │
[prev|data|next] ⇄ [prev|data|next] ⇄ [prev|data|next]
  节点 1            节点 2            节点 3
```

### 📊 4 大链表对比

| 链表 | 找前驱 | 找后继 | 空间 | 操作 |
|------|--------|--------|------|------|
| 单链表 | O(n) | O(1) | 1 指针 | 简单 |
| 双链表 | O(1) | O(1) | 2 指针 | 灵活 |
| 循环单链表 | O(n) | O(1) | 1 指针 | 适合环形 |
| 循环双链表 | O(1) | O(1) | 2 指针 | 最灵活 |

### 🚀 链表常用技巧

#### 1. 头插法（逆序建表）

```c
// 头插法建立单链表（结果与输入顺序相反）
LinkList Create_List_Head() {
    LinkList L = InitList();
    for (i = 0; i < n; i++) {
        LNode *p = (LNode*)malloc(sizeof(LNode));
        scanf(&p->data);
        p->next = L->next;  // 新节点指向首元节点
        L->next = p;         // 头节点指向新节点
    }
    return L;
}
```

#### 2. 尾插法（顺序建表）

```c
// 尾插法建立单链表（结果与输入顺序相同）
LinkList Create_List_Tail() {
    LinkList L = InitList();
    LNode *r = L;  // r 始终指向尾节点
    for (i = 0; i < n; i++) {
        LNode *p = (LNode*)malloc(sizeof(LNode));
        scanf(&p->data);
        p->next = NULL;
        r->next = p;
        r = p;  // 更新 r 指向新尾节点
    }
    return L;
}
```

#### 3. 双向链表插入

```
插入前：... ⇄ A ⇄ C ⇄ ...
插入后：... ⇄ A ⇄ B ⇄ C ⇄ ...

操作：
B->next = C;
B->prev = A;
A->next = B;
C->prev = B;
```

## 3️⃣ 栈与队列

### 📊 栈（Stack）

> **LIFO**（后进先出）  
> 应用：函数调用、表达式求值、括号匹配、DFS

#### 顺序栈

```
┌─────┐
│  3  │ ← top
├─────┤
│  2  │
├─────┤
│  1  │
├─────┤
│  0  │ ← bottom
└─────┘
```

#### 链栈

```
top → [data|next] → [data|next] → [data|next] → NULL
       栈顶          中间         栈底
```

#### 栈的应用

| 应用 | 关键思路 |
|------|---------|
| **括号匹配** | 左括号入栈，右括号出栈匹配 |
| **表达式求值** | 操作数栈 + 运算符栈 |
| **函数调用** | 调用栈保存现场 |
| **DFS 递归** | 递归 = 栈 |
| **浏览器后退** | 访问历史入栈 |

### 📊 队列（Queue）

> **FIFO**（先进先出）  
> 应用：广度优先遍历、消息队列、打印队列

#### 顺序队列（循环队列）

```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│     │  A  │  B  │  C  │     │     │
└─────┴─────┴─────┴─────┴─────┴─────┘
  ↑   front                  ↑  rear
 队首                       队尾（指向下一个入队位置）
```

> **循环队列**解决假溢出问题  
> 公式：
> - 队空：`front == rear`
> - 队满：`(rear + 1) % MaxSize == front`（牺牲一个存储单元）
> - 队长：`(rear - front + MaxSize) % MaxSize`

#### 链队列

```
front → [data|next] → [data|next] → [data|next] ← rear
       队首（出队）                  队尾（入队）
```

#### 双端队列（Deque）

> 两端都可入队出队  
> 应用：滑动窗口最大/最小值

#### 队列的应用

| 应用 | 关键思路 |
|------|---------|
| **BFS 遍历** | 队列存待访问节点 |
| **消息队列** | 生产者-消费者模式 |
| **打印队列** | 先到先打 |
| **树的层序遍历** | 一层一层入队出队 |

## 4️⃣ 串（String）

### 📌 串的存储

```c
// 定长顺序存储
typedef struct {
    char data[MaxSize];
    int length;
} SString;

// 堆分配存储
typedef struct {
    char *data;
    int length;
} HString;
```

### 🚀 KMP 算法（重点！）

#### 1. 朴素模式匹配（BF）

```c
// 时间复杂度 O(n*m)
int Index_BF(String S, String T) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (S.data[i] == T.data[j]) { i++; j++; }
        else { i = i - j + 2; j = 1; }  // i 回退
    }
    if (j > T.length) return i - T.length;
    return 0;
}
```

#### 2. KMP 算法（O(n+m)）

**核心思想**：当匹配失败时，**模式串向右滑动尽可能少的距离**继续匹配

##### next 数组定义

```
next[j] = 当 T[j] 与主串失配时，j 应回退到的位置
```

##### next 数组计算

```
j:       1 2 3 4 5 6 7 8
T:       a b a a b c a c
next[j]: 0 1 1 2 2 3 1 2

手算方法：
- next[1] = 0（固定）
- next[j] = max{k | T[1..k] = T[j-k+1..j-1], 1 ≤ k < j}
- 若不存在则为 1
```

##### KMP 匹配过程

```
主串 S:  a b a c a a b a a b c a b c
模式串 T:a b a b c
next:   0 1 1 2 2
                ↑
              失配时 j 回退到 next[j]

匹配过程：
- i=1, j=1: a==a, i=2, j=2
- i=2, j=2: b==b, i=3, j=3
- i=3, j=3: a==a, i=4, j=4
- i=4, j=4: c≠b 失配，j=next[4]=2
- i=4, j=2: c≠b 失配，j=next[2]=1
- i=4, j=1: c≠a 失配，j=next[1]=0
- i++, j=1
- ...
```

> 1. next[1] = 0（不是 1）  
> 2. next 数组下标从 1 开始（不是 0）  
> 3. 改进的 nextval 数组处理字符相等的情况

## 🎯 线性结构·必背公式

### 1. 顺序表插入

```
平均移动元素数 = n/2
时间复杂度 = O(n)
```

### 2. 链表头插法

```
时间复杂度 = O(n)（建立链表）
```

### 3. 循环队列

```
队空：Q.front == Q.rear
队满：(Q.rear + 1) % MaxSize == Q.front
```

### 4. KMP 时间复杂度

```
T(n) = O(n + m)  ← 比 BF 的 O(nm) 优
```

## ⚠️ 6 大易错点

> 1. ❌ 顺序表插入忘记移动元素
> 2. ❌ 链表插入顺序错误（应先连后断）
> 3. ❌ 循环队列队满条件搞混
> 4. ❌ next 数组下标从 1 开始
> 5. ❌ 链栈/链队判空条件（不是 L->next == NULL，而是 L->next == NULL）
> 6. ❌ 双链表的 prev 和 next 都需更新
