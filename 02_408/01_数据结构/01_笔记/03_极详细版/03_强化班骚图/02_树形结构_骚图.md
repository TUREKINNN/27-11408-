## 1️⃣ 二叉树基础

### 📌 5 大性质

| 性质 | 内容 |
|------|------|
| 1 | 第 i 层最多 $2^{i-1}$ 个节点 |
| 2 | 深度为 k 的二叉树最多 $2^k - 1$ 个节点 |
| 3 | $n_0 = n_2 + 1$（叶子 = 度为 2 + 1） |
| 4 | 完全二叉树 $n$ 个节点深度 $\lfloor \log_2 n \rfloor + 1$ |
| 5 | 顺序存储：节点 i 的左孩子 2i，右孩子 2i+1 |

### 📊 3 大遍历方式

| 遍历 | 顺序 | 递归 / 迭代 |
|------|------|----------|
| **先序** | 根 → 左 → 右 | 入栈出栈 |
| **中序** | 左 → 根 → 右 | 迭代（栈模拟） |
| **后序** | 左 → 右 → 根 | 迭代（双栈或标记） |
| **层序** | 一层一层 | BFS + 队列 |

### 📊 由遍历反推二叉树

> **关键结论**：**已知中序 + 另一序 = 唯一二叉树**

```
先序：ABDECF
中序：DBEAFC

反推过程：
- 先序首 A = 根
- 中序中 A 位置：左 DBE | A | 右 FC
- 先序剩余：BDE | CF
- 左子树先序 BDE，中序 DBE
  - B = 根，左 D | B | 右 E
- 右子树先序 CF，中序 FC
  - C = 根，左 F | C | 右空

        A
       /       B   C
     /   /
    D   F
           E
```

### 🚀 非递归遍历（重点！）

#### 中序遍历（非递归）

```c
void InOrder(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !IsEmpty(S)) {
        if (p) {
            Push(S, p);    // 入栈
            p = p->lchild; // 一路向左
        } else {
            Pop(S, p);     // 出栈访问
            visit(p);
            p = p->rchild; // 转向右子树
        }
    }
}
```

#### 后序遍历（非递归，难点！）

```c
void PostOrder(BiTree T) {
    InitStack(S);
    BiTree p = T, r = NULL;  // r 记录最近访问节点
    while (p || !IsEmpty(S)) {
        if (p) {
            Push(S, p);
            p = p->lchild;
        } else {
            GetTop(S, p);   // 取栈顶（不弹出）
            if (p->rchild && p->rchild != r) {
                p = p->rchild;  // 右子树未访问
            } else {
                Pop(S, p);
                visit(p);
                r = p;       // 标记已访问
                p = NULL;    // 触发出栈
            }
        }
    }
}
```

## 2️⃣ 线索二叉树

### 📌 核心思想

> 利用二叉树中 n+1 个**空指针**域，存放**前驱 / 后继**指针

### 📊 线索二叉树节点结构

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 0=孩子, 1=线索
} ThreadNode;
```

### 🚀 中序线索化

```c
void InThread(ThreadNode *p, ThreadNode *&pre) {
    if (p) {
        InThread(p->lchild, pre);  // 左子树线索化
        if (p->lchild == NULL) {   // 左孩子为空 → 前驱线索
            p->lchild = pre;
            p->ltag = 1;
        }
        if (pre && pre->rchild == NULL) {  // 前驱的右孩子为空 → 后继线索
            pre->rchild = p;
            pre->rtag = 1;
        }
        pre = p;  // 更新 pre
        InThread(p->rchild, pre);  // 右子树线索化
    }
}
```

### 🚀 线索二叉树遍历

```c
// 中序线索二叉树的遍历（不需要递归！）
void InOrder_Thr(ThreadNode *T) {
    ThreadNode *p = T->lchild;  // 找中序首节点
    while (p != T) {            // 回到头节点结束
        while (p->ltag == 0) p = p->lchild;  // 一路向左
        visit(p);
        while (p->rtag == 1 && p->rchild != T) {
            p = p->rchild;       // 沿后继线索
            visit(p);
        }
        p = p->rchild;           // 进入右子树
    }
}
```

## 3️⃣ 树与森林

### 📊 树 → 二叉树转换

```
树：
      A
    / |    B  C  D
  /|     |
 E F     G

转二叉树（左孩子右兄弟）：
      A
     /
    B
   /   E   C
               D
       /
      G
```

> **规则**：树的**第一个孩子**作为左子树，**下一个兄弟**作为右子树

### 📊 4 大遍历对应

| 树的遍历 | 对应二叉树遍历 |
|---------|---------------|
| 先根 | 先序 |
| 后根 | 中序 |
| 层序 | ❌（二叉树没有） |

## 4️⃣ 堆（Heap）

### 📌 核心性质

> **大根堆**：父节点 ≥ 子节点  
> **小根堆**：父节点 ≤ 子节点

### 📊 堆的存储（数组）

```
完全二叉树：
        1
       /       2   3
     /\   /    4 5  6 7

数组存储：a[1..7] = {1, 2, 3, 4, 5, 6, 7}
父节点 i：左孩子 2i，右孩子 2i+1
子节点 i：父节点 ⌊i/2⌋
```

### 🚀 堆的插入（向上调整）

```c
void Heap_Insert(int a[], int *n, int x) {
    a[++(*n)] = x;        // 插入到末尾
    int i = *n;
    while (i > 1 && a[i] > a[i/2]) {  // 大根堆：上浮
        swap(a[i], a[i/2]);
        i = i / 2;
    }
}
```

### 🚀 堆的删除（向下调整）

```c
// 删除堆顶元素（最大/最小）
int Heap_Delete(int a[], int *n) {
    int x = a[1];         // 保存堆顶
    a[1] = a[(*n)--];     // 末元素移到堆顶
    int i = 1;
    while (2*i <= *n) {   // 向下调整
        int child = 2*i;
        if (child < *n && a[child+1] > a[child]) child++;  // 选大孩子
        if (a[i] >= a[child]) break;  // 满足堆序
        swap(a[i], a[child]);
        i = child;
    }
    return x;
}
```

### 🚀 堆排序

```
1. 建堆（O(n)）
2. 反复取堆顶 + 调整（O(nlogn)）
```

### 📊 堆的应用

| 应用 | 思路 |
|------|------|
| **堆排序** | O(nlogn)，不稳定 |
| **优先队列** | 堆 = 优先队列 |
| **Top K 问题** | 维护大小 K 的小顶堆 |
| **Dijkstra 最短路** | 优先队列优化 |
| **Huffman 编码** | 优先队列选最小权 |

## 5️⃣ 哈夫曼树

### 📌 核心概念

> **WPL**（带权路径长度）= $\sum w_i \times l_i$  
> 哈夫曼树 = **WPL 最小的二叉树**

### 🚀 哈夫曼树构造

```
初始：5, 2, 7, 4（权值）

步骤 1：选最小两个 2, 4 → 合并为 6
步骤 2：5, 6, 7 → 选 5, 6 合并为 11
步骤 3：7, 11 → 合并为 18

哈夫曼树：
          18
         /          7   11
           /            5    6
              /              2   4

WPL = 7×1 + 5×2 + 2×3 + 4×3 = 7 + 10 + 6 + 12 = 35
```

### 📊 哈夫曼编码

> **前缀编码**：任一字符编码不是其他字符编码的前缀  
> 哈夫曼树构造前缀编码：左 0 右 1

```
字符:    A  B  C  D
频次:    5  2  7  4
编码:   00 010 11 011
```

## 6️⃣ B 树 / B+ 树

### 📌 m 阶 B 树性质

> 1. 每个节点**最多 m 个孩子**（m-1 个关键字）  
> 2. 除根外，每个节点**至少 ⌈m/2⌉ 个孩子**（⌈m/2⌉-1 个关键字）  
> 3. 根至少 2 个孩子（除非是叶子）  
> 4. 所有叶子节点在同一层  
> 5. 节点内关键字**升序排列**

### 📊 B 树 vs B+ 树

| 维度 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储 | 节点都存数据 | **数据只在叶子** |
| 叶子链表 | 无 | **叶子形成链表** |
| 查找 | 可能在中间节点停 | **必须到叶子** |
| 范围查询 | 需多次遍历 | **沿链表即可** |
| 适用 | 数据库索引 | 数据库索引（更优） |

### 🚀 B 树插入（分裂）

```
插入 25 到 5 阶 B 树（每个节点最多 4 个关键字）：

初始：
        [20 30 40]
       /    |    
插入 25：
        [20 25 30 40]   ← 关键字数 = 4，仍合法

继续插入 27：
        [20 25 27 30 40] ← 关键字数 = 5，**超载！**

分裂：中间关键字 27 上移
              [27]
             /            [20 25]  [30 40]
```

### 🚀 B 树删除（合并/借位）

> 删除关键字后，若节点关键字数 < ⌈m/2⌉-1，需**借位**或**合并**  
> 借位：从兄弟节点借  
> 合并：与父节点 + 兄弟节点合并

## 🎯 树形结构·必背公式

### 1. 完全二叉树深度

```
⌊log₂n⌋ + 1
```

### 2. 二叉树节点关系

```
n₀ = n₂ + 1  （叶子节点 = 度2节点 + 1）
```

### 3. 哈夫曼 WPL

```
WPL = Σ wᵢ × lᵢ
```

### 4. B 树节点数

```
m 阶 B 树，根外节点关键字数 ∈ [⌈m/2⌉-1, m-1]
```

## ⚠️ 7 大易错点

> 1. ❌ 二叉树遍历反推：必须**中序 + 另一序**
> 2. ❌ 线索二叉树 ltag/rtag 含义搞混
> 3. ❌ 树转二叉树时**第一个孩子当左子树，下一个兄弟当右子树**
> 4. ❌ 堆的插入/删除调整方向反
> 5. ❌ 哈夫曼树 WPL 计算错误（叶节点深度算错）
> 6. ❌ B 树分裂时**中间关键字上移**（不是父节点关键字下移）
> 7. ❌ B+ 树和 B 树搞混
