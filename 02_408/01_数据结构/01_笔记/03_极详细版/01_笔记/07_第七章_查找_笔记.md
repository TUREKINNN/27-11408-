# 第七章 查找

## 7.1 查找的基本概念

> **查找表（Search Table）**：由同一类型的数据元素（或记录）构成的集合。
> - **静态查找表**：仅做"查询"、"检索"操作（如顺序查找、折半查找）。
> - **动态查找表**：在查找过程中同时进行插入、删除操作（如二叉排序树、B 树、散列表）。

> - **关键字**：数据元素中某个数据项的值，用以标识一个数据元素。
> - **平均查找长度 ASL**：查找过程中**关键字比较次数的平均值**。
>
> $$ASL = \sum_{i=1}^{n} p_i \cdot c_i$$
>
> 其中 $p_i$ 是查找第 $i$ 个元素的概率，$c_i$ 是找到第 $i$ 个元素所需的比较次数。
> 考试默认 $p_i = 1/n$（等概率）。

## 7.2 顺序查找

### 7.2.1 一般顺序查找（无哨兵）

```c
typedef struct {
    ElemType *data;
    int length;
} SSTable;

int Search_Seq(SSTable ST, ElemType key) {
    for (int i = 0; i < ST.length; ++i)
        if (ST.data[i] == key) return i + 1;
    return 0;
}
```

### 7.2.2 哨兵顺序查找

> 在数组 0 号位（最前）放入待查元素 `key` 作为哨兵，从后往前扫描，可避免每次循环都检查下标越界，性能更优。

```c
int Search_Seq2(SSTable ST, ElemType key) {
    ST.data[0] = key;                          // 设置哨兵
    int i;
    for (i = ST.length; ST.data[i] != key; --i);
    return i;                                  // i == 0 表示未找到
}
```

> - **成功 ASL**（等概率）：$\dfrac{n+1}{2}$；
> - **失败 ASL**（到达哨兵）：$n + 1$；
> - 时间复杂度 $O(n)$；空间复杂度 $O(1)$；
> - 优点：算法简单、对存储结构无要求（顺序/链式皆可）；
> - 缺点：$n$ 较大时查找效率低。

## 7.3 折半查找（二分查找）

> 仅适用于**有序的顺序表**（链式存储无法随机访问）。
> 1. 设 `low=0, high=n-1`；
> 2. `mid = (low+high)/2`；比较 `data[mid]` 与 `key`：
>    - 相等 → 成功；
>    - `key < data[mid]` → `high = mid - 1`；
>    - `key > data[mid]` → `low = mid + 1`；
> 3. 循环直到 `low > high`（查找失败）。

```c
int Binary_Search(SSTable ST, ElemType key) {
    int low = 0, high = ST.length - 1, mid;
    while (low <= high) {
        mid = (low + high) / 2;
        if (ST.data[mid] == key) return mid;
        else if (key < ST.data[mid]) high = mid - 1;
        else low = mid + 1;
    }
    return -1;                                 // 失败
}
```

> - **判定树**为高度 $\lfloor \log_2 n \rfloor + 1$ 的二叉平衡树；
> - **成功 ASL** $\approx \log_2(n+1) - 1$；
> - **失败 ASL** $\approx \log_2(n+1)$；
> - 时间复杂度 $O(\log_2 n)$。

> 查找表 $L = (10, 20, 30, 40, 50, 60, 70, 80, 90)$ 的判定树为：
>
> ```
>              50
>            /    \
>          20      70
>         /  \    /  \
>       10   30  60  80
>                 \      \
>                  40     90
> ```
>
> 判定树形态**只与表的元素个数有关，与具体元素值无关**（折半查找的判定树是唯一的）。

## 7.4 分块查找（索引顺序查找）

> 将查找表分成若干**块**，块内元素**无序**，但块间**有序**（即前一块的所有元素关键字均小于后一块中的最大关键字）。再建立一个**索引表**（每块含最大关键字 + 起始地址），先在索引表上二分/顺序查找定位块，再在块内顺序查找。

> 若将 $n$ 个元素均匀分为 $\sqrt{n}$ 块，每块 $\sqrt{n}$ 个元素：
> - ASL = 索引查找 + 块内查找 $\approx \log_2(\sqrt{n}) + \dfrac{\sqrt{n}+1}{2}$ = $O(\sqrt{n})$。

## 7.5 二叉排序树（BST）

### 7.5.1 定义

> 一棵**空树**，或者是具有以下性质的二叉树：
> 1. 若左子树非空，则左子树上所有结点的关键字均**小于**根结点的关键字；
> 2. 若右子树非空，则右子树上所有结点的关键字均**大于**根结点的关键字；
> 3. 左、右子树也分别是二叉排序树。
>
> **特性**：**中序遍历** BST 得到的序列是**递增有序**的。

### 7.5.2 查找

```c
BSTNode *BST_Search(BSTNode *T, ElemType key) {
    while (T != NULL && key != T->data) {
        if (key < T->data) T = T->lchild;
        else T = T->rchild;
    }
    return T;
}
```

### 7.5.3 插入

> 从根结点开始，与 `key` 比较：
> - 若树为空 → 创建新结点作为根；
> - 若 `key < T->data` → 递归插入左子树；
> - 若 `key > T->data` → 递归插入右子树；
> - 若 `key == T->data` → 不插入（保证唯一性）。

### 7.5.4 删除

> 1. **叶结点**：直接删除；
> 2. **仅有左/右子树**：让该子树替代被删结点；
> 3. **左右子树均存在**：在**右子树中找到中序遍历的第一个结点**（即右子树最小值，又叫**直接后继**）填补到被删结点，再删除该后继（该后继必无左子树）。

```c
bool BST_Delete(BSTNode *&T, ElemType key) {
    if (T == NULL) return false;
    if (key == T->data) return DeleteNode(T);
    else if (key < T->data) return BST_Delete(T->lchild, key);
    else return BST_Delete(T->rchild, key);
}

bool DeleteNode(BSTNode *&p) {
    if (p->lchild == NULL) { BSTNode *q = p; p = p->rchild; free(q); }
    else if (p->rchild == NULL) { BSTNode *q = p; p = p->lchild; free(q); }
    else {
        BSTNode *s = p->rchild;
        while (s->lchild != NULL) s = s->lchild;
        p->data = s->data;
        BST_Delete(p->rchild, s->data);
    }
    return true;
}
```

> - 平均查找长度：$O(\log_2 n)$（平衡时）；
> - 最坏（单支树）：$O(n)$。

## 7.6 平衡二叉树（AVL）

### 7.6.1 定义

> 树中任意结点的**左、右子树高度差（平衡因子 BF）的绝对值不超过 1**。
> $$|h_L - h_R| \leq 1$$
> 其中平衡因子 $BF = h_L - h_R$。AVL 树是**高度平衡的 BST**。

### 7.6.2 平衡调整（4 种旋转）

> | 不平衡类型 | 触发条件 | 调整方法 |
> |:----|:----|:----|
> | LL | 在 A 的**左孩子**的**左子树**中插入 | **右单旋** |
> | RR | 在 A 的**右孩子**的**右子树**中插入 | **左单旋** |
> | LR | 在 A 的**左孩子**的**右子树**中插入 | **先左后右双旋** |
> | RL | 在 A 的**右孩子**的**左子树**中插入 | **先右后左双旋** |

> **"谁在不平衡结点哪边，哪边就旋转；LR/RL 是先反向旋转子树，再正向旋转根。"**
> - LL → 整体**右旋**（左孩子上提）；
> - RR → 整体**左旋**（右孩子上提）；
> - LR → 先左子树左旋，再整体右旋；
> - RL → 先右子树右旋，再整体左旋。

> - 高度为 $h$ 的 AVL 树，最少结点数 $N_h = N_{h-1} + N_{h-2} + 1$（类似斐波那契）；
> - 查找时间复杂度 $O(\log_2 n)$；
> - 插入删除后最多 $O(1)$ 次旋转即可重新平衡。

## 7.7 B 树和 B+ 树

### 7.7.1 B 树的基本概念

> 一棵 m 阶 B 树是**平衡**的多路查找树，满足：
> 1. 每个结点至多有 $m$ 棵子树，$m - 1$ 个关键字；
> 2. 除根外，至少有 $\lceil m/2 \rceil$ 棵子树，$\lceil m/2 \rceil - 1$ 个关键字；
> 3. 根结点至少有 2 棵子树（除非 B 树只有 1 个结点）；
> 4. 所有叶结点出现在同一层（不带信息的**失败结点**）；
> 5. 结点内关键字**递增**排列，左子树关键字 < 父关键字 < 右子树关键字。

> 结点关键字数范围：$2 \leq n \leq 4$；子树数范围：$3 \leq n+1 \leq 5$。

### 7.7.2 B 树的插入与删除

> 1. 找到应插入的**叶结点**（最下层）；
> 2. 插入后若该结点关键字数 $> m - 1$ → **分裂**：取中间关键字上提到父结点，原结点一分为二；
> 3. 若父结点也超 → 继续上提分裂，最坏分裂到根 → 树增高一层。

> 1. 删除非叶结点：用**直接前驱 / 后继**填补后转为叶结点删除；
> 2. 删除叶结点：若关键字数仍 $\geq \lceil m/2 \rceil - 1$ → 直接删；
> 3. 否则**借**兄弟（旋转）或**合并**兄弟 + 父关键字；合并可能导致父结点关键字不足，递归处理。

### 7.7.3 B+ 树

> | 特征 | B 树 | B+ 树 |
> |:----|:----|:----|
> | 关键字数 vs 子树数 | $n$ 个关键字，$n+1$ 棵子树 | $n$ 个关键字，$n$ 棵子树 |
> | 数据存放 | 每个结点都存数据 | **数据只存于叶结点**，非叶结点仅作索引 |
> | 叶结点链接 | 无 | **叶结点按关键字大小链接**（链表） |
> | 查找方式 | 不唯一，可能中途命中 | 必须查找到叶结点（**成功/失败统一在叶结点**） |
> | 适用场景 | 文件系统 | **数据库索引** |

## 7.8 散列表（哈希表）

### 7.8.1 基本概念

> - **散列函数（Hash Function）**：将关键字映射到存储地址的函数 $H(key)$。
> - **散列表（Hash Table）**：根据散列函数建立的表。
> - **冲突（Collision）**：不同关键字映射到同一地址。**同义词** = 发生冲突的两个关键字。
> - 装填因子 $\alpha = \dfrac{n}{m}$（$n$ 关键字数，$m$ 表长）。

### 7.8.2 散列函数的构造

> 1. **直接定址法**：$H(key) = a \cdot key + b$（关键字分布基本连续）；
> 2. **除留余数法**：$H(key) = key \mod p$（$p$ 为不超过表长的最大素数，**最常用**）；
> 3. **数字分析法**：取关键字中分布均匀的几位作为地址（事先知道关键字）；
> 4. **平方取中法**：取关键字平方后的中间几位（位数适中）；
> 5. **折叠法**：将关键字分割为几段，叠加求和（位数多、不必知道分布）。

### 7.8.3 处理冲突的方法

#### 1. 开放定址法

> 1. **线性探测法**：$H_i = (H(key) + d_i) \mod m$，$d_i = 0, 1, 2, \dots$。
> 2. **平方探测法（二次探测）**：$d_i = 0^2, 1^2, -1^2, 2^2, -2^2, \dots$。
> 3. **双散列法**：$H_i = (H_1(key) + i \cdot H_2(key)) \mod m$。
> 4. **伪随机序列法**：$d_i$ 为伪随机数序列。

> 线性探测易产生**堆积（primary clustering）**：连续被占用的块形成"堆"，导致探测序列长。平方探测可缓解，但**不能探测到所有单元**（当 $m$ 不是 4k+3 形式的素数时，探测可能失败）。

#### 2. 拉链法（链地址法 / 链式哈希）

> 将所有同义词链接在同一个**单链表**中，链表头指针存于散列表的对应槽位。
> - **插入** $O(1)$（头插）；
> - **查找** $O(L)$，$L$ 为链表长度；
> - **删除** $O(L)$；
> - 装填因子 $\alpha = n / m$ 可**大于 1**（区别于开放定址法 $\alpha \leq 1$）。

> 关键字集 $\{19, 14, 23, 1, 68, 20, 84, 27, 55, 11, 10, 79\}$，$H(key) = key \mod 13$：
>
> | 下标 | 链表 |
> |:----|:----|
> | 0 | |
> | 1 | $1 \to 27 \to 79$ |
> | 2 | $14 \to$ |
> | 3 | |
> | 4 | $68 \to$ |
> | 5 | $55 \to$ |
> | 6 | |
> | 7 | $20 \to$ |
> | 8 | $84 \to$ |
> | 9 | $23 \to 10 \to$ |
> | 10 | $11 \to$ |
> | 11 | |
> | 12 | $19 \to$ |

### 7.8.4 散列查找性能分析

> - **线性探测法成功 ASL** $\approx \dfrac{1}{2}\left(1 + \dfrac{1}{1-\alpha}\right)$
> - **平方探测法成功 ASL** $\approx -\dfrac{1}{\alpha} \ln(1-\alpha)$
> - **拉链法成功 ASL** $\approx 1 + \dfrac{\alpha}{2}$
> - 散列查找的**平均性能**为 $O(1)$（$\alpha$ 较小时），但**最坏** $O(n)$。

#### 知识回顾

> - 顺序查找 ASL = $(n+1)/2$；折半查找 ASL = $\log_2(n+1) - 1$；分块查找 ASL = $\sqrt{n}$ 量级。
> - BST 中序有序；删除时用直接前驱/后继填补。
> - AVL 4 种旋转：LL 右旋、RR 左旋、LR 先左后右、RL 先右后左。
> - m 阶 B 树：除根外结点数 $\geq \lceil m/2 \rceil$；B+ 树数据存于叶结点且叶结点链表。
> - 散列表：除留余数法构造 + 线性/平方/双散列/拉链法处理冲突；ASL 取决于 $\alpha$。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 查找 = "找东西"
- **ASL**（平均查找长度）：查找一次的平均比较次数
- **时间复杂度**：用 $O$ 表示

#### 2. 3 大查找方法
| 方法 | 适用 | 时间复杂度 | 备注 |
|------|------|-----------|------|
| 顺序查找 | 无序/有序 | $O(n)$ | 最简单 |
| 折半查找 | 有序顺序表 | $O(\log_2 n)$ | 必考 |
| 哈希查找 | 散列表 | $O(1)$ 期望 | 冲突解决 |

#### 3. 折半查找"3 步法"
1. 取中点 `mid = (low+high)/2`
2. 比较目标值与 `data[mid]`
3. 目标值 < `data[mid]` → 左半；目标值 > `data[mid]` → 右半

#### 4. 二叉排序树（BST）
- 左子树所有值 < 根
- 右子树所有值 > 根
- 中序遍历 = **递增序列**

#### 5. 平衡二叉树（AVL）
- 左右子树高度差 $\le 1$
- 失衡时**旋转**：LL/RR/LR/RL

#### 6. 红黑树（408 偶尔考）
- 节点：红/黑
- 5 大性质：根黑/叶黑/红父黑子/黑高相同
- 插入/删除后调整

#### 7. B 树和 B+ 树
- **B 树**：多路平衡查找树
- **B+ 树**：所有数据在叶子节点（**数据库索引**）

#### 8. 哈希表
- $h(\text{key}) = \text{key} \mod p$
- **冲突**：不同 key 映射到同一地址
- **解决**：
  - 拉链法（链地址法）
  - 开放定址法（线性/二次/双重哈希）

### 📚 天勤·严瑾推导

#### 1. ASL 计算
- **顺序查找**（无序）：$\text{ASL} = (n+1)/2$
- **顺序查找**（有监视哨）：$\text{ASL} = n/2$
- **折半查找**：$\text{ASL} = \log_2(n+1) - 1$
- **哈希查找**（线性探测）：$\text{ASL} = (1 + \frac{1}{1-\alpha})/2$

#### 2. 折半查找判定树
- **平衡二叉树**
- 失败节点 = $n+1$ 个
- 树高 $h = \lfloor \log_2 n \rfloor + 1$

#### 3. B 树的 5 大性质
1. 每个节点最多 $m$ 个孩子（$m$ 阶 B 树）
2. 除根外，每个节点至少 $\lceil m/2 \rceil$ 个孩子
3. 根节点至少 2 个孩子（除非是叶子）
4. 所有叶子在同一层
5. 节点内关键字**升序**

#### 4. 哈希表"装填因子"
$$\alpha = \frac{n}{m}$$
- $n$ = 已存元素数
- $m$ = 哈希表长度
- $\alpha$ 越大，冲突越多

#### 5. 各种探测法
- **线性探测**：`H(key, i) = (h(key) + i) mod m`
- **二次探测**：`H(key, i) = (h(key) + i²) mod m`
- **双重哈希**：`H(key, i) = (h₁(key) + i·h₂(key)) mod m`
