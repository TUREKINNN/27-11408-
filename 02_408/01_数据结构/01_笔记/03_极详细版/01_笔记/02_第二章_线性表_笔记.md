# 第二章 线性表

## 2.1 线性表的定义和基本操作

### 2.1.1 线性表的定义

> **线性表（Linear List）**：是具有**相同数据类型的 $n$（$n \geq 0$）个数据元素**的**有限序列**。
>
> 一般表示为：
> $$L = (a_1, a_2, \dots, a_i, a_{i+1}, \dots, a_n)$$
>
> 其中 $n$ 为表长；当 $n = 0$ 时为空表。
>
> - $a_i$ 是线性表的"第 $i$ 个"元素（位序，从 1 开始）
> - $a_1$ 是**表头元素**，$a_n$ 是**表尾元素**
> - 除第一个元素外，集合中每个元素有且仅有一个**直接前驱**；除最后一个元素外，每个元素有且仅有一个**直接后继**

### 2.1.2 线性表的基本操作

| 操作 | 含义 |
|:----|:----|
| `InitList(&L)` | 初始化表，构造一个空的线性表 L |
| `Length(L)` | 求表长，返回 L 中数据元素的个数 |
| `LocateElem(L, e)` | 按值查找，在 L 中查找与 e 相等的元素，返回位序 |
| `GetElem(L, i)` | 按位查找，获取 L 中第 i 个位置元素的值 |
| `ListInsert(&L, i, e)` | 在 L 的第 i 个位置插入新元素 e |
| `ListDelete(&L, i, &e)` | 删除 L 的第 i 个位置元素，并用 e 返回 |
| `PrintList(L)` | 输出 L 中的所有元素 |
| `Empty(L)` | 判空，若 L 为空表返回 true，否则 false |
| `DestroyList(&L)` | 销毁线性表，释放 L 所占的内存空间 |

> 线性表的操作可归纳为：**创（Init）**、**销（Destroy）**、**增（Insert）**、**删（Delete）**、**查（Locate/Get）** 五类。考题常要求：给出操作的时间复杂度。

## 2.2 线性表的顺序表示

### 2.2.1 顺序表的定义

> 用**一段地址连续的存储单元**依次存储线性表中的数据元素，使得**逻辑上相邻的两个元素在物理位置上也相邻**。
> - 元素地址计算（首地址 $LOC(a_1)$，每个元素占 $L$ 个存储单元）：
> $$LOC(a_i) = LOC(a_1) + (i-1) \times L$$
> - **随机访问**（按下标访问）的时间复杂度为 $O(1)$。

```c
#define MaxSize 50              // 定义线性表的最大长度
typedef struct {
    ElemType data[MaxSize];     // 顺序表的元素（静态数组实现）
    int length;                 // 顺序表的当前长度
} SqList;                       // Sq: sequence
```

> - **静态分配**：数组大小在编译时确定（如 `data[MaxSize]`），一旦空间占满，再加入新数据就会溢出。
> - **动态分配**：用 `malloc` 在运行时分配空间，当空间满时再 `realloc` 扩展。
>
> ```c
> #define InitSize 100
> typedef struct {
>     ElemType *data;            // 动态分配数组的指针
>     int MaxSize, length;       // 最大容量、当前长度
> } SeqList;
> ```

### 2.2.2 顺序表的插入与删除

#### 1. 插入操作

> 1. 检查 i 的合法性（$1 \leq i \leq L.length + 1$）；
> 2. 检查存储空间是否已满；
> 3. 将第 $i \sim n$ 个元素**后移一位**；
> 4. 将新元素 e 放入第 i 个位置；
> 5. 表长 +1。

```c
bool ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length + 1) return false;       // i 越界
    if (L.length >= MaxSize) return false;             // 存储空间满
    for (int j = L.length; j >= i; --j)                // j 指向待移动元素
        L.data[j] = L.data[j - 1];                     // 后移一位
    L.data[i - 1] = e;                                 // 插入新元素
    ++L.length;                                        // 长度 +1
    return true;
}
```

#### 2. 删除操作

> 1. 检查 i 的合法性（$1 \leq i \leq L.length$）；
> 2. 取出第 i 个元素到 e；
> 3. 将第 $i+1 \sim n$ 个元素**前移一位**；
> 4. 表长 -1。

```c
bool ListDelete(SqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length) return false;           // i 越界
    e = L.data[i - 1];                                 // 取出被删元素
    for (int j = i; j < L.length; ++j)                 // 前移
        L.data[j - 1] = L.data[j];
    --L.length;                                        // 长度 -1
    return true;
}
```

#### 3. 按值查找（顺序查找）

```c
int LocateElem(SqList L, ElemType e) {
    for (int i = 0; i < L.length; ++i)
        if (L.data[i] == e) return i + 1;              // 返回位序
    return 0;                                          // 未找到
}
```

> - 插入 / 删除：平均移动约 $\dfrac{n}{2}$ 个元素，**时间复杂度 $O(n)$**（最坏 $O(n)$，最好 $O(1)$）。
> - 按值查找：**时间复杂度 $O(n)$**（无序时只能顺序扫描）。
> - 按位查找：**时间复杂度 $O(1)$**（数组下标直接访问）。

> - 在第 $i$ 个位置**插入**：需移动 $n - i + 1$ 个元素。
> - 在第 $i$ 个位置**删除**：需移动 $n - i$ 个元素。
> - 等概率下，平均移动次数 = $\dfrac{\sum_{i=1}^{n+1}(n-i+1)}{n+1} = \dfrac{n}{2}$。

## 2.3 线性表的链式表示

### 2.3.1 单链表的定义

> 通过**任意的存储单元**（可以不连续）存储线性表中的数据元素，每个结点包含**数据域**和**指针域**（指向后继结点）。
> - 单链表的**头结点**（可有可无）：为操作方便而在链表的第一个有效结点之前附设的一个结点，其指针域指向第一个有效结点。
> - **头指针**：指向链表中第一个结点（若带头结点则指向头结点，否则指向第一个数据结点）的指针。

```c
typedef struct LNode {                 // 定义单链表结点类型
    ElemType data;                     // 数据域
    struct LNode *next;                // 指针域，指向后继
} LNode, *LinkList;                    // *LinkList 为指向单链表的指针
```

> - **不带头结点**：第一个结点就是数据结点，对第一个结点的操作需要特殊处理（修改头指针）。
> - **带头结点**：所有结点操作统一（不需要修改头指针），代码更简洁。
> - **推荐**：**带头结点**。考试建议默认带。

### 2.3.2 单链表的基本操作

#### 1. 头插法建立单链表

> 每读入一个数据元素，生成一个新结点，将新结点**插入到当前链表的表头**（头结点之后）。
> - 时间复杂度：$O(n)$
> - 特点：读入数据的顺序与生成链表的顺序**相反**。

```c
LinkList List_HeadInsert(LinkList &L) {
    L = (LinkList)malloc(sizeof(LNode));   // 创建头结点
    L->next = NULL;                         // 初始为空链表
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {                     // 9999 为输入结束标志
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        s->next = L->next;                  // 新结点指向原首元结点
        L->next = s;                        // 头结点指向新结点
        scanf("%d", &x);
    }
    return L;
}
```

#### 2. 尾插法建立单链表

> 每读入一个数据元素，生成一个新结点，将新结点**插入到当前链表的表尾**。
> - 需要维护一个**尾指针** $r$，始终指向当前链表的最后一个结点。
> - 时间复杂度：$O(n)$
> - 特点：读入数据的顺序与生成链表的顺序**相同**。

```c
LinkList List_TailInsert(LinkList &L) {
    L = (LinkList)malloc(sizeof(LNode));
    LNode *r = L;                           // r 始终指向尾结点
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        r->next = s;                        // 接入新结点
        r = s;                              // r 指向新尾结点
        scanf("%d", &x);
    }
    r->next = NULL;                         // 尾结点指针置空
    return L;
}
```

#### 3. 按序号查找

```c
LNode *GetElem(LinkList L, int i) {
    if (i < 0) return NULL;                 // 非法位序
    LNode *p = L;                           // p 指向头结点（位序 0）
    int j = 0;
    while (p != NULL && j < i) {            // 沿 next 域向后扫描
        p = p->next;
        ++j;
    }
    return p;                               // 找到则返回第 i 个结点指针
}
```

#### 4. 按值查找

```c
LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;                     // p 指向首元结点
    while (p != NULL && p->data != e)
        p = p->next;
    return p;
}
```

#### 5. 插入（后插）

```c
bool InsertNextNode(LNode *p, ElemType e) {
    if (p == NULL) return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) return false;            // 内存分配失败
    s->data = e;
    s->next = p->next;
    p->next = s;
    return true;
}
```

#### 6. 前插（关键技巧：偷天换日）

> 单链表无法快速获取前驱结点（需要从头遍历）。**前插的 O(1) 技巧**：
> 1. 在 p 之后插入新结点 s；
> 2. 将 s->data 赋为 p->data（**交换数据**）；
> 3. 将 s->next 指向原 p->next；
> 4. p->next = s。
> 效果上等价于"在 p 之前插入"，时间复杂度 $O(1)$。

#### 7. 删除（按位序）

```c
bool ListDelete(LinkList &L, int i, ElemType &e) {
    if (i < 1) return false;
    LNode *p = GetElem(L, i - 1);           // 找前驱
    if (p == NULL || p->next == NULL) return false;
    LNode *q = p->next;
    e = q->data;
    p->next = q->next;
    free(q);
    return true;
}
```

#### 8. 求表长

```c
int Length(LinkList L) {
    int len = 0;
    LNode *p = L;
    while (p->next != NULL) {
        ++len;
        p = p->next;
    }
    return len;
}
```

> | 操作 | 顺序表 | 单链表 |
> |:----|:----|:----|
> | 按位查找 | $O(1)$ | $O(n)$ |
> | 按值查找 | $O(n)$ | $O(n)$ |
> | 插入 | $O(n)$ | $O(1)$（已知前驱）/ $O(n)$（未知前驱） |
> | 删除 | $O(n)$ | $O(1)$（已知前驱）/ $O(n)$（未知前驱） |

### 2.3.3 双链表

> 每个结点含**前驱指针 `prior`** 和**后继指针 `next`**，可以从任一结点出发快速找到前驱/后继。

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;            // 前驱、后继
} DNode, *DLinkList;
```

> ```c
> bool InsertNextDNode(DNode *p, DNode *s) {
>     s->next = p->next;
>     if (p->next != NULL) p->next->prior = s;
>     s->prior = p;
>     p->next = s;
>     return true;
> }
> ```
> **必须先处理后继指针，再处理前驱指针**，避免断链。

> ```c
> bool DeleteNextDNode(DNode *p) {
>     DNode *q = p->next;
>     if (q == NULL) return false;
>     p->next = q->next;
>     if (q->next != NULL) q->next->prior = p;
>     free(q);
>     return true;
> }
> ```

### 2.3.4 循环链表

#### 1. 循环单链表

> 将单链表的**尾结点指针 `next` 由 `NULL` 改为指向头结点**，整个链表形成一个环。
> - **判空条件**：`L == L->next`（带头结点）；`L == NULL`（不带头结点）。
> - 经常对**尾结点**操作时，时间复杂度 $O(1)$（尾指针 `R` 始终指向尾结点）。

#### 2. 循环双链表

> 双链表的尾结点 `next` 指向头结点，头结点 `prior` 指向尾结点。
> - **判空条件**：`L->next == L`（即头结点的 next 指向自己）。
> - 优点：从任一结点出发，可访问链表中所有结点。

### 2.3.5 静态链表

> 用**数组**来描述线性表的链式存储结构：每个数组元素含 `data`（数据域）和 `next`（下一个结点的数组下标，0 表示空）。
> - 优点：不需要指针，分配在栈区（数组）即可。
> - 缺点：容量固定、不能动态扩展、查找仍要遍历。

```c
#define MaxSize 50
typedef struct {
    ElemType data;
    int next;                              // 下一个元素的数组下标
} SLinkList[MaxSize];
```

> - `next == -1` 表示空指针；
> - `next == 0` 表示指向头结点；
> - 头结点位置通常固定在 `SLinkList[0]`。

### 2.3.6 顺序表与链表的比较

| 比较维度 | 顺序表 | 链表 |
|:----|:----|:----|
| 存储空间 | 预先分配，可能浪费 | 动态分配，不会浪费 |
| 存储密度 | 高（=1） | 低（含指针开销） |
| 存取（读） | 随机访问，$O(1)$ | 顺序访问，$O(n)$ |
| 插入/删除 | 需大量移动元素，$O(n)$ | 仅修改指针，$O(1)$（已知前驱） |
| 适用场景 | 频繁查找、较少改动 | 频繁插入/删除、长度变化大 |

> 1. 需频繁查找 / 很少插入删除 → 顺序表；
> 2. 元素个数变化大、经常插入删除 → 链表；
> 3. 事先确定长度、需要随机访问 → 顺序表；
> 4. 频繁在表头/表尾操作 → 带尾指针的循环单链表。

#### 知识回顾

> - 顺序表 = 数组 + length；支持随机访问 $O(1)$，但插入/删除需移动元素 $O(n)$。
> - 单链表 = 头结点 + 数据结点 + 尾结点 `next = NULL`；插入/删除 $O(1)$，查找 $O(n)$。
> - 头插法 O(n) 逆序、尾插法 O(n) 顺序；双链表 O(1) 找前驱。
> - 静态链表用数组模拟指针（`next` 存下标），考研偶尔考。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 顺序表 = "火车车厢"
- 车厢**紧挨着**（连续存储）
- 找第 $i$ 节车厢**极快** $O(1)$（按号入座）
- 插入/删除要**挪车** $O(n)$

> - 顺序表 = 火车车厢：一节接一节
> - 链表 = 寻宝地图：每节车厢一张"下一节"指示牌

#### 2. 链表 = "寻宝图"
- 结点**不连续**，靠**指针**串起来
- 找第 $i$ 个**要数** $O(n)$
- 插入/删除**改指针** $O(1)$

#### 3. 头结点 vs 头指针
- **头指针**：必有的指针，指向首元结点（或头结点）
- **头结点**：可有可无的"哨兵"，**不存数据**
- **目的**：让所有位置的操作**统一**

#### 4. 链表 5 大分类
- 单链表：每个结点 1 个指针
- 双链表：每个结点 2 个指针
- 循环单链表：尾结点的 next 指向头
- 循环双链表：双向循环
- 静态链表：数组 + 游标

### 📚 天勤·严瑾推导

#### 1. 顺序表的"地址公式"（必背）
$$LOC(a_i) = LOC(a_1) + (i-1) \times L$$

#### 2. 单链表"前插 O(1) 技巧"
单链表无法快速获取前驱。**技巧**：
1. 在 p 后插新结点 s
2. 交换数据 `s->data ↔ p->data`
3. `p->next = s`
> 效果：在 p 前插，等价 $O(1)$。

#### 3. 顺序表插入/删除"移动次数"公式
- **插入第 i 个位置**：移动 $n - i + 1$
- **删除第 i 个位置**：移动 $n - i$
- **平均移动次数**：$n/2$
