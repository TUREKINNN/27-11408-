# 第四章 串

## 4.1 串的定义和基本操作

### 4.1.1 串的定义

> 串是由**零个或多个字符**组成的**有限序列**，一般记为：
> $$S = 'a_1 a_2 \dots a_n' \quad (n \geq 0)$$
> - $n$ 为串的长度；$n = 0$ 时为空串（$\emptyset$ 或 `""`）。
> - 子串：串中任意**连续**的字符组成的子序列。
> - 主串：包含子串的串。
> - 字符在串中的位置：字符在串中的序号（从 1 开始）。
> - 子串在主串中的位置：子串第一个字符在主串中的位置。

> - 串是**数据元素限定为字符**的线性表；
> - 串的基本操作**以子串为操作对象**，而非单个元素。

### 4.1.2 串的基本操作

| 操作 | 含义 |
|:----|:----|
| `StrAssign(&T, chars)` | 把字符串常量赋给串 T（生成串） |
| `StrCopy(&T, S)` | 由串 S 复制得到串 T |
| `StrEmpty(S)` | 判空 |
| `StrLength(S)` | 求串长 |
| `ClearString(&S)` | 清空 |
| `Concat(&T, S1, S2)` | 串连接 |
| `SubString(&Sub, S, pos, len)` | 求子串 |
| `Index(S, T)` | 定位（子串 T 在主串 S 中的位置） |
| `StrCompare(S, T)` | 串比较 |
| `Replace(&S, T, V)` | 替换 |
| `StrInsert(&S, pos, T)` | 插入 |
| `StrDelete(&S, pos, len)` | 删除 |

## 4.2 串的存储结构

### 4.2.1 定长顺序存储

```c
#define MAXLEN 255
typedef struct {
    char ch[MAXLEN + 1];     // 存储串的一维数组（0 号单元闲置或存放长度）
    int length;              // 串的实际长度
} SString;
```

### 4.2.2 堆分配存储

```c
typedef struct {
    char *ch;                // 按串长分配存储区
    int length;
} HString;

HString S;
S.ch = (char *)malloc(MAXLEN * sizeof(char));
S.length = 0;
```

### 4.2.3 链式存储

```c
#define CHUNKSIZE 80
typedef struct Chunk {
    char ch[CHUNKSIZE];
    struct Chunk *next;
} Chunk;
typedef struct {
    Chunk *head, *tail;      // 串的头尾指针
    int length;              // 串的当前长度
} LString;
```

> - 一个结点可存储多个字符，提高存储密度。
> - 操作（求长度、定位）相对复杂，仅在特殊场景使用。

## 4.3 串的模式匹配

### 4.3.1 简单模式匹配算法（BF / Brute Force）

> 用主串 S 的每一个字符作为起始位置，依次与模式串 T 的字符进行匹配：
> 1. 设 `i, j` 分别指示主串和模式串中当前比较的字符位置（下标 0 起）；
> 2. 若 `S[i] == T[j]`，则 `i++, j++` 继续比较；
> 3. 若 `S[i] != T[j]`，则主串指针 `i` 回退到 `i - j + 1`（即本次匹配起始位置的下一个），模式串指针 `j` 重置为 0；
> 4. 当 `j == T.length` 时匹配成功，返回 `i - j`。

```c
int Index_BF(SString S, SString T) {
    int i = 0, j = 0;
    while (i < S.length && j < T.length) {
        if (S.ch[i] == T.ch[j]) { ++i; ++j; }
        else { i = i - j + 1; j = 0; }
    }
    if (j == T.length) return i - j;     // 匹配成功
    return -1;                           // 失败
}
```

> - **最好**：$O(n)$，每次第一个字符就不匹配；
> - **最坏**：$O(n \times m)$，主串长度 $n$，模式串长度 $m$，每个位置都要比较 $m$ 次。

### 4.3.2 KMP 算法

> 当 `S[i] != T[j]` 时，**主串指针 `i` 不回退**，仅让模式串指针 `j` 退到合适的位置（由 `next[j]` 决定），从而利用已经部分匹配的结果跳过无效比较。
> - KMP 时间复杂度 **$O(n + m)$**。

#### 1. 字符串的前缀、后缀与部分匹配值

> - **前缀**：除最后一个字符外，字符串的所有头部子串。
> - **后缀**：除第一个字符外，字符串的所有尾部子串。
> - **部分匹配值（PM）**：字符串的**所有前缀集合与后缀集合的交集中最长元素的长度**。

> | 长度 | 字符串 | 前缀集合 | 后缀集合 | PM |
> |:----|:----|:----|:----|:----|
> | 1 | a | $\emptyset$ | $\emptyset$ | 0 |
> | 2 | ab | {a} | {b} | 0 |
> | 3 | aba | {a, ab} | {a, ba} | 1 |
> | 4 | abab | {a, ab, aba} | {b, ab, bab} | 2 |
> | 5 | ababa | {a, ab, aba, abab} | {a, aba, baba, ababa} | 3 |

#### 2. next 数组（手算方法）

> - `next[1] = 0`，`next[2] = 1`（**固定值**）。
> - 之后每一位的 `next[j]`：看模式串 $T[1 \sim j-1]$ 这一段子串，**前缀集合 = 后缀集合**的最长元素长度 $+1$。
> - **最长相等前后缀长度 + 1**（等价描述）。

> | j | 1 | 2 | 3 | 4 | 5 | 6 |
> |:--|:-:|:-:|:-:|:-:|:-:|:-:|
> | T[j] | a | b | a | a | b | c |
> | next[j] | 0 | 1 | 1 | 2 | 2 | 3 |
>
> **解释**：
> - `next[1]=0, next[2]=1`（固定）。
> - `next[3]`：`T[1..2]="ab"`，最长相等前后缀=0，+1=1。
> - `next[4]`：`T[1..3]="aba"`，最长相等前后缀=1（"a"），+1=2。
> - `next[5]`：`T[1..4]="abaa"`，最长相等前后缀=1（"a"），+1=2。
> - `next[6]`：`T[1..5]="abaab"`，最长相等前后缀=2（"ab"），+1=3。

#### 3. KMP 算法实现

```c
int Index_KMP(SString S, SString T, int next[]) {
    int i = 0, j = 0;
    while (i < S.length && j < T.length) {
        if (j == 0 || S.ch[i] == T.ch[j]) { ++i; ++j; }
        else j = next[j];                    // i 不变, j 回退到 next[j]
    }
    if (j == T.length) return i - j;
    return -1;
}
```

#### 4. 求 next 数组

```c
void get_next(SString T, int next[]) {
    int i = 1, j = 0;                        // T 下标从 1 开始
    next[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) { ++i; ++j; next[i] = j; }
        else j = next[j];
    }
}
```

### 4.3.3 KMP 算法优化（nextval 数组）

> 在 KMP 中，当 `T[j] == T[next[j]]` 时，跳到 `next[j]` 仍会再次失配。**nextval** 数组解决此问题：
> - `nextval[j] = next[ next[j] ]`，直到 `T[j] != T[next[j]]`。
> - 实现中：`if (T.ch[j] == T.ch[next[j]]) nextval[j] = nextval[next[j]]; else nextval[j] = next[j]`。

```c
void get_nextval(SString T, int nextval[]) {
    int i = 1, j = 0;
    nextval[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            ++i; ++j;
            if (T.ch[i] != T.ch[j]) nextval[i] = j;
            else nextval[i] = nextval[j];     // 优化核心
        } else j = next[j];
    }
}
```

#### 知识回顾

> - 串是限定字符的线性表，操作以**子串**为对象。
> - 简单模式匹配 BF 最坏 $O(nm)$，KMP 利用最长相等前后缀将主串不回退，复杂度 $O(n+m)$。
> - 手算 `next`：固定 `next[1]=0, next[2]=1`，其他位看前缀 = 后缀的最长长度 + 1。
> - `nextval` 在 KMP 基础上进一步优化，避开 `T[j] == T[next[j]]` 的重复比较。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 串 = "字符序列"
- $S = \text{"a}_1\text{a}_2\ldots\text{a}_n\text{"}$
- 子串：串中**连续**的字符序列
- 主串：包含子串的串
- 空串：长度为 0（**不同于空格串**）

#### 2. 模式匹配 = "找子串"
- **BF（朴素）**：$O(mn)$，暴力
- **KMP**：$O(m+n)$，**next 数组**

#### 3. next 数组（KMP 核心）
- `next[j]`：当第 j 个字符失配时，**应回退到哪个位置**
- 公式：$\text{next}[j] = \text{最长相等前后缀长度} + 1$

> | j | 1 | 2 | 3 | 4 |
> |---|---|---|---|---|
> | 模式 | A | B | A | B |
> | next[j] | 0 | 1 | 1 | 3 |

### 📚 天勤·严瑾推导

#### 1. BF 算法
```c
int BF(S, T) {
    i = 1; j = 1;
    while (i <= S.length && j <= T.length) {
        if (S[i] == T[j]) { i++; j++; }
        else { i = i - j + 2; j = 1; }
    }
    if (j > T.length) return i - T.length;
    return 0;
}
```

#### 2. KMP 关键
- **next[j]** 含义：模式串 T 中，前 j-1 个字符的**最长相等前后缀**长度 + 1
- 失配时，主串指针 **i 不回退**，模式串指针 j 回退到 next[j]

#### 3. 改进的 nextval
- 解决 `T[j] = T[next[j]]` 时再次失配的问题
- `nextval[j] = next[j]` 对应的 `nextval[next[j]]`
