# 第六章 图

## 6.1 图的基本概念

> **图 $G$** 由顶点集 $V$ 和边集 $E$ 组成，记为 $G = (V, E)$。
> - $V$：顶点（vertex）集合，非空有穷
> - $E$：边（edge）集合，可以为空

### 6.1.1 有向图 vs 无向图

| 类型 | 边记法 | 关系 |
|:----|:----|:----|
| **无向图** | $(v_i, v_j) \in E$ | 顶点 $v_i$ 和 $v_j$ 之间的**无向边**（边对称） |
| **有向图** | $\langle v_i, v_j \rangle \in E$ | 顶点 $v_i$ 到 $v_j$ 的**有向边 / 弧**，$v_i$ 是弧尾，$v_j$ 是弧头 |

### 6.1.2 简单图与多重图

> - **简单图**：① 不存在重复边；② 不存在顶点到自身的边
> - **多重图**：反之（两个条件至少一个不满足）
> - 408 考纲默认**简单图**

### 6.1.3 顶点的度、入度、出度

> - **度（TD）**：与该顶点相关联的边的数目
> - **入度（ID）**：以该顶点为终点的弧的数目（有向图）
> - **出度（OD）**：以该顶点为起点的弧的数目（有向图）
> - 关系：$\text{TD}(v) = \text{ID}(v) + \text{OD}(v)$

> - 无向图：$\sum \text{TD}(v) = 2|E|$（**所有顶点的度之和等于边数的 2 倍**）
> - 有向图：$\sum \text{ID}(v) = \sum \text{OD}(v) = |E|$

### 6.1.4 路径、回路、连通

> - **路径**：顶点 $v_p$ 到 $v_q$ 之间的一条路径是顶点序列 $v_p, v_{i1}, v_{i2}, \dots, v_{im}, v_q$。
> - **路径长度**：路径上**边的数目**（与顶点个数差 1）。
> - **简单路径**：路径上**顶点不重复出现**。
> - **回路 / 环**：第一个顶点和最后一个顶点**相同**的路径。

### 6.1.5 连通图、强连通图

> - **连通图（无向图）**：图中任意两个顶点都是**连通的**（有路径）
> - **强连通图（有向图）**：图中任意两个顶点 $u, v$ 之间既有 $u \to v$ 也有 $v \to u$ 的有向路径
> - **连通分量（无向图）**：无向图中的**极大连通子图**
> - **强连通分量（有向图）**：有向图中的**极大强连通子图**

### 6.1.6 完全图、子图、生成树

> - **无向完全图**：任意两个顶点之间都存在边，**边数 = $\dfrac{n(n-1)}{2}$**
> - **有向完全图**：任意两个顶点之间都存在**方向相反**的两条弧，**弧数 = $n(n-1)$**

> - **子图**：$V' \subseteq V$、$E' \subseteq E$
> - **连通图的生成树**：包含**全部顶点**和**恰好 $n-1$ 条边**的**极小连通子图**

## 6.2 图的存储

### 6.2.1 邻接矩阵法

> 结点数为 $n$ 的图 $G = (V, E)$ 的邻接矩阵 $\mathbf{A}$ 是 $n \times n$ 的。将 $G$ 的顶点编号为 $v_1, v_2, \ldots, v_n$，则：
>
> $$\mathrm{A}[i][j] = \begin{cases} 1, & \text{若} (v_i, v_j) \text{或} \langle v_i, v_j \rangle \text{是} E(G) \text{中的边} \\ 0, & \text{若} (v_i, v_j) \text{或} \langle v_i, v_j \rangle \text{不是} E(G) \text{中的边} \end{cases}$$

#### 存储结构

```c
#define MaxVertexNum 100
typedef struct {
    char Vex[MaxVertexNum];
    int Edge[MaxVertexNum][MaxVertexNum];
    int vexnum, arcnum;
} MGraph;
```

> - 顶点中可以存**更复杂的信息**
> - 可以用 `bool` 型或**枚举型**变量表示边（如表示"无 / 有 / 自环"等）

#### 度 / 入度 / 出度

| 图类型 | 第 $i$ 个结点的度 | 出度 | 入度 |
|:------|:----------------|:-----|:----|
| **无向图** | 第 $i$ 行（=第 $i$ 列）非零元素个数 | — | — |
| **有向图** | — | 第 $i$ **行**的非零元素个数 | 第 $i$ **列**的非零元素个数 |

> 邻接矩阵法求顶点的**度 / 出度 / 入度**的时间复杂度为 $O(|V|)$。

#### 邻接矩阵存储带权图（网）

```c
#define MaxVertexNum 100
#define INFINITY 最大的 int 值
typedef char VertexType;
typedef int EdgeType;
typedef struct {
    VertexType Vex[MaxVertexNum];
    EdgeType Edge[MaxVertexNum][MaxVertexNum];
    int vexnum, arcnum;
} MGraph;
```

> - **优点**：直观、简单；查边 $O(1)$；适合**稠密图**。
> - **缺点**：存储 $O(|V|^2)$ 空间，**稀疏图浪费大**；统计边的数目需扫描全矩阵 $O(|V|^2)$。

### 6.2.2 邻接表法（顺序 + 链式存储）

> 对图中每个顶点 $v_i$ 建立一个单链表，链表中**所有相邻于 $v_i$ 的顶点**。
> - 第 $i$ 个单链表：与 $v_i$ **直接相邻**的所有顶点
> - 每个单链表附设一个**头结点**（存 $v_i$ 的信息）；表结点 = 邻接点域 + 链域
> - 所有头结点用**顺序存储**（一维数组）组织

```c
typedef struct ArcNode {                 // 边 / 弧
    int adjvex;                          // 邻接点下标
    struct ArcNode *nextarc;             // 指向下一条弧
    OtherInfo info;                      // 边上信息（权值等）
} ArcNode;

typedef struct VNode {                   // 顶点
    VertexType data;
    ArcNode *firstarc;                   // 指向第一条弧
} VNode, AdjList[MaxVertexNum];

typedef struct {
    AdjList vertices;
    int vexnum, arcnum;
} ALGraph;
```

> - **无向图**中同一条边出现两次（两个链表各一次），空间 $O(|V| + 2|E|)$
> - **有向图**中每条弧出现一次，空间 $O(|V| + |E|)$
> - 度：遍历第 $i$ 个链表，统计结点数（无向图）= TD($v_i$)；有向图 = OD($v_i$)，ID($v_i$) 需遍历所有链表

> - **优点**：节省空间，适合**稀疏图**；找邻接点快
> - **缺点**：判断两个顶点**是否相邻**需遍历第 $i$ 个链表，$O(\text{degree}(v_i))$
> - 邻接表的**表示不唯一**（边输入顺序不同则链表顺序不同）

### 6.2.3 十字链表（针对有向图）

> 整合**邻接表 + 逆邻接表**：每个弧结点同时属于两个链表（出弧链表 + 入弧链表）。
> - 顶点结点：`data, firstin, firstout`
> - 弧结点：`tailvex, headvex, hlink, tlink, info`
> - 空间 $O(|V| + |E|)$；求 ID / OD 都方便。

### 6.2.4 邻接多重表（针对无向图）

> 每条边只对应一个结点（避免邻接表中边重复存储两次）。
> - 边结点：`mark, ivex, ilink, jvex, jlink, info`
> - 顶点结点：`data, firstedge`

### 6.2.5 图的基本操作

| 操作 | 邻接矩阵 | 邻接表 |
|:----|:----|:----|
| `FirstNeighbor(G, v)` | 扫描第 v 行第一个 1 | 顺着 firstarc 找第一个 |
| `NextNeighbor(G, v, w)` | 扫描第 v 行，w 之后第一个 1 | 顺着 w 的 nextarc |
| `InsertVertex(G, v)` | 直接数组末尾追加 | 直接数组末尾追加 |
| `DeleteVertex(G, v)` | 删除一行一列 | 删除所有涉及 v 的边 |
| `AddEdge(G, u, v)` / `RemoveEdge` | 矩阵对应位置赋值 0/1 | 头插 / 链表中删除 |
| `GetNeighborCount(G, v)` | 扫第 v 行 $O(\|V\|)$ | 扫第 v 个链表 $O(\text{degree})$ |

## 6.3 图的遍历

### 6.3.1 广度优先遍历（BFS）

> 从图的某一顶点出发，**逐层访问**与该顶点相邻的所有未被访问的顶点，再依次从这些被访问的顶点出发，逐层访问其未被访问的邻接点……直到所有连通分量中的顶点都被访问为止。

```c
bool visited[MAX_VERTEX_NUM];
void BFS(Graph G, int v) {
    visit(v); visited[v] = TRUE; EnQueue(Q, v);
    while (!isEmpty(Q)) {
        DeQueue(Q, v);
        for (w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w))
            if (!visited[w]) {
                visit(w); visited[w] = TRUE; EnQueue(Q, w);
            }
    }
}
void BFSTraverse(Graph G) {              // 处理非连通图
    for (i = 0; i < G.vexnum; ++i) visited[i] = FALSE;
    for (i = 0; i < G.vexnum; ++i)
        if (!visited[i]) BFS(G, i);
}
```

> - 空间 $O(|V|)$；时间：邻接矩阵 $O(|V|^2)$；邻接表 $O(|V| + |E|)$

### 6.3.2 深度优先遍历（DFS）

> 从顶点 $v$ 出发，访问 $v$ 后递归地沿**任一未访问的邻接点**深度遍历。

```c
bool visited[MAX_VERTEX_NUM];
void DFS(Graph G, int v) {
    visit(v); visited[v] = TRUE;
    for (w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w))
        if (!visited[w]) DFS(G, w);
}
void DFSTraverse(Graph G) {
    for (i = 0; i < G.vexnum; ++i) visited[i] = FALSE;
    for (i = 0; i < G.vexnum; ++i)
        if (!visited[i]) DFS(G, i);
}
```

> - 空间 $O(|V|)$（递归栈）；时间：邻接矩阵 $O(|V|^2)$；邻接表 $O(|V| + |E|)$

> | 比较 | BFS | DFS |
> |:----|:----|:----|
> | 数据结构 | 队列 | 栈（递归） |
> | 访问顺序 | 逐层 | 一条路走到黑 |
> | 邻接矩阵时间 | $O(\|V\|^2)$ | $O(\|V\|^2)$ |
> | 邻接表时间 | $O(\|V\|+\|E\|)$ | $O(\|V\|+\|E\|)$ |
> | 最小生成树 | Prim 用类 BFS 思想 | Kruskal 不基于 DFS |
> | 单源最短路径 | **BFS 可求无权图**最短路径 | 不能直接求最短路径 |

## 6.4 图的应用

### 6.4.1 最小生成树

> 一个**连通无向图**的生成树中，**所有边的权值之和最小**的那棵。
> - 性质：① 生成树不唯一；② 最小生成树**也不唯一**，但**权值之和唯一**。
> - 含 $n$ 个顶点的最小生成树有 $n-1$ 条边。

#### 1. Prim 算法（加点法）

> 从**任意顶点**开始，每次选取**与当前已选顶点集距离最小**的边，将对应的未选顶点加入。
> - 时间复杂度 $O(|V|^2)$，**与边数无关**，适合**稠密图**。
> - 关键数组：`lowcost[]`（到已选集的最小权值）、`closest[]`（对应的已选顶点）。

```c
void Prim(MGraph G, int u) {
    for (i = 0; i < G.vexnum; ++i) {
        lowcost[i] = G.Edge[u][i];
        closest[i] = u;
    }
    lowcost[u] = 0;                      // 起点入已选集
    for (i = 1; i < G.vexnum; ++i) {
        k = Min(lowcost);                // 找最小权值
        lowcost[k] = 0;                  // k 入已选集
        for (j = 0; j < G.vexnum; ++j)
            if (lowcost[j] != 0 && G.Edge[k][j] < lowcost[j]) {
                lowcost[j] = G.Edge[k][j];
                closest[j] = k;
            }
    }
}
```

#### 2. Kruskal 算法（加边法）

> 每次选取**权值最小**且**不构成回路**的边加入生成树。
> - 时间复杂度 $O(|E| \log |E|)$，**与顶点数无关**，适合**稀疏图**。
> - 判断回路：用**并查集**（两端点属于同一集合则跳过）。

```c
void Kruskal(MGraph G) {
    Init(UFSet);
    SortEdgesByWeight(G);                // 边按权值升序
    for (each edge (u, v, w) in sorted order)
        if (Find(u) != Find(v)) {
            Union(u, v);
            AddEdgeToMST((u, v, w));
        }
}
```

> | | Prim | Kruskal |
> |:----|:----|:----|
> | 思想 | 点 | 边 |
> | 时间 | $O(\|V\|^2)$ | $O(\|E\|\log\|E\|)$ |
> | 适用 | 稠密图 | 稀疏图 |

### 6.4.2 最短路径

#### 1. 单源最短路径 - BFS 算法（无权图）

> 对无权图，用 BFS 逐层访问，第一次到达即为最短路径。
> - 用 `dist[]` 记录距离，`path[]` 记录前驱
> - 时间 $O(|V| + |E|)$

#### 2. 单源最短路径 - Dijkstra 算法（带权图，权非负）

> 集合 $S$ 存放已找到最短路径的顶点。每次从 $V - S$ 中选**距离源点最短**的顶点 $u$ 加入 $S$，并**松弛**所有 $u$ 的邻接点。
> - 时间 $O(|V|^2)$，**不适用于含负权边的图**。
> - 求一个顶点到所有其他顶点的最短路径。

```c
void Dijkstra(Graph G, int v0) {
    Init dist[], path[], final[];
    for (i = 0; i < G.vexnum; ++i) {
        dist[i] = G.Edge[v0][i];
        path[i] = (dist[i] < INFINITY) ? v0 : -1;
    }
    dist[v0] = 0; final[v0] = TRUE;
    for (i = 1; i < G.vexnum; ++i) {
        min = INFINITY;
        for (j = 0; j < G.vexnum; ++j)
            if (!final[j] && dist[j] < min) { min = dist[j]; k = j; }
        final[k] = TRUE;
        for (j = 0; j < G.vexnum; ++j)
            if (!final[j] && min + G.Edge[k][j] < dist[j]) {
                dist[j] = min + G.Edge[k][j];
                path[j] = k;
            }
    }
}
```

> 一旦有负权边，"已确定最短路径"的顶点之后可能因新路径（含负权边）而**更短**，导致算法失效。
> 含负权边应使用 **Bellman-Ford** 或 **SPFA**（不考）。

#### 3. 各顶点间最短路径 - Floyd 算法

> 设 $D^{(k)}[i][j]$ 为只允许经过前 $k$ 个顶点时 $i \to j$ 的最短路径长度，递推：
> $$D^{(k)}[i][j] = \min(D^{(k-1)}[i][j], D^{(k-1)}[i][k] + D^{(k-1)}[k][j])$$
> 时间 $O(|V|^3)$，**可处理负权边**，但**不能处理负权回路**。

```c
void Floyd(Graph G) {
    for (i = 0; i < G.vexnum; ++i)
        for (j = 0; j < G.vexnum; ++j) {
            D[i][j] = G.Edge[i][j];
            path[i][j] = -1;
        }
    for (k = 0; k < G.vexnum; ++k)
        for (i = 0; i < G.vexnum; ++i)
            for (j = 0; j < G.vexnum; ++j++)
                if (D[i][k] + D[k][j] < D[i][j]) {
                    D[i][j] = D[i][k] + D[k][j];
                    path[i][j] = k;
                }
}
```

### 6.4.3 有向无环图描述表达式

> 用**有向无环图（DAG）**描述表达式，可**合并公共子表达式**，节省存储。
> - 算术表达式 → 二叉树 → DAG（公共子表达式只画一次）
> - 拓扑排序可以得到一种**合法的运算顺序**（无后效性）

> 用 DAG 表示 $(a+b) \times (b \times (c+d))$：
> 公共子表达式 $(b+c+d)$（实际上是 $(b+(c+d))$）会被合并。

### 6.4.4 拓扑排序

> 用顶点表示活动、用弧表示活动间优先关系的有向图，称为 **AOV 网**（Activity On Vertex Network）。
> - AOV 网**不能有环**（否则拓扑序列不存在）。

> 1. 在 AOV 网中选一个**入度为 0** 的顶点输出
> 2. 删除该顶点及其所有出边
> 3. 重复 1-2 直到所有顶点都已输出，或当前不存在入度为 0 的顶点
> - 若最后所有顶点都输出 → **存在拓扑序列**
> - 若仍存在未输出的顶点 → **存在环**（无拓扑序列）

```c
void TopologicalSort(Graph G) {
    InitStack(S);                        // 存放入度为 0 的顶点
    for (i = 0; i < G.vexnum; ++i)
        if (indegree[i] == 0) Push(S, i);
    count = 0;
    while (!StackEmpty(S)) {
        Pop(S, i); print(i); ++count;
        for (each w adjacent to i)
            if (--indegree[w] == 0) Push(S, w);
    }
    if (count < G.vexnum) printf("存在环！");
}
```


### 6.4.5 关键路径

> 用顶点表示**事件**、用弧表示**活动**、弧的权表示活动持续时间的**有向无环图**。
> - 关键路径：AOE 网中**从源点到汇点的最长路径**（所有活动按序进行的最短完成时间）。
> - **关键活动**：关键路径上的活动（缩短或延长会直接影响整个工程工期）。

> 1. 求所有事件的**最早发生时间 $ve$**（从前往后，取最大）：
>    $ve[j] = \max_{i \to j} \{ ve[i] + a_{ij} \}$
> 2. 求所有事件的**最迟发生时间 $vl$**（从后往前，取最小）：
>    $vl[i] = \min_{i \to j} \{ vl[j] - a_{ij} \}$
> 3. 求所有活动的**最早开始时间 $e$** / **最迟开始时间 $l$**：
>    - 活动 $a_k$（$i \to j$）: $e_k = ve[i]$, $l_k = vl[j] - a_{ij}$
> 4. 关键活动 = $e_k = l_k$ 的活动；关键路径 = 关键活动组成的路径。

> - 关键路径可能**不止一条**（多关键路径情况下，加快非关键活动不能缩短工期）
> - 关键路径上活动的时间延长多少，整个工程的完成时间也就随之延长多少

#### 知识回顾

> - 无向图边数 $\leq n(n-1)/2$，有向完全图弧数 $= n(n-1)$。
> - 无向图度数之和 $= 2|E|$，有向图入度之和 $=$ 出度之和 $= |E|$。
> - 邻接矩阵 $O(|V|^2)$ 空间，适合稠密图；邻接表 $O(|V|+|E|)$ 空间，适合稀疏图。
> - 邻接表的表示**不唯一**。
> - BFS 找无权图最短路径；Dijkstra 求带非负权图单源最短路径；Floyd 求全源最短路径。
> - Prim $O(|V|^2)$ 适合稠密图；Kruskal $O(|E|\log|E|)$ 适合稀疏图。
> - 拓扑排序针对 AOV 网；关键路径针对 AOE 网。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 图 = "网络"
- **顶点（Vertex）**：节点
- **边（Edge）**：连接
- **度**：连接顶点的边数
- **路径**：顶点序列
- **环**：起点=终点的路径

#### 2. 图的分类
| 类型 | 边 | 关系 |
|------|----|------|
| 有向图 | 有方向 | 单向 |
| 无向图 | 无方向 | 双向 |
| 加权图 | 带权 | 距离/费用 |
| 无权图 | 无权 | 0/1 |

#### 3. 图的 4 种存储
- **邻接矩阵**：$O(n^2)$ 空间，**稠密图**用
- **邻接表**：$O(n+e)$ 空间，**稀疏图**用
- **十字链表**：有向图
- **邻接多重表**：无向图

#### 4. 2 种遍历
- **DFS**（深度优先）：栈/递归
- **BFS**（广度优先）：队列

> - DFS = "走迷宫：一直走，撞墙回头"
> - BFS = "涟漪扩散：一圈一圈"

#### 5. 最小生成树（MST）
- **Prim**：适合**稠密图**，$O(n^2)$
- **Kruskal**：适合**稀疏图**，$O(e\log e)$

#### 6. 最短路径
- **Dijkstra**：单源最短路径，**正权图**，$O(n^2)$
- **Floyd**：任意两点，$O(n^3)$

#### 7. 拓扑排序 + 关键路径
- **AOV 网络**：顶点表示活动，边表示先后
- **AOE 网络**：边表示活动，顶点表示事件
- **关键路径**：最长路径

#### 2. 关键路径"4 步法"
1. 求**事件最早发生时间 ve**
2. 求**事件最晚发生时间 vl**
3. 求**活动最早开始时间 e**
4. 求**活动最晚开始时间 l**
- 关键活动 = $e = l$ 的活动
- 关键路径 = 关键活动组成的路径

#### 3. 最短路径 Dijkstra 算法
```c
void Dijkstra(int v) {
    S = {v};
    for (i=0; i<n; ++i) dist[i] = edge[v][i];
    for (i=0; i<n-1; ++i) {
        u = min{dist[k]|k∉S};
        S = S ∪ {u};
        for (w∉S) if (dist[u]+edge[u][w]<dist[w])
            dist[w] = dist[u]+edge[u][w];
    }
}
```


#### 4. 拓扑排序"3 步法"
1. 选**入度为 0** 的顶点
2. 输出该顶点
3. 删除该顶点的所有出边
- 重复直到所有顶点都输出（**有环则不能输出所有顶点**）
