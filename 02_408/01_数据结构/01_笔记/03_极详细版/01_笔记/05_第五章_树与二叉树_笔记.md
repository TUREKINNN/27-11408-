# 第五章 树与二叉树

## 5.1 树

### 5.1.1 + 5.1.2 树的定义和基本术语

> 树是 $n\ (n \geq 0)$ 个结点的有限集合，$n = 0$ 时称为**空树**。
> 在任意一棵非空树中应满足：
> 1. 有且仅有一个特定的称为**根**的结点
> 2. 当 $n > 1$ 时，其余结点可分为 $m\ (m > 0)$ 个**互不相交**的有限集合 $T_1, T_2, \ldots, T_m$，其中每个集合本身又是一棵树，并且称为根结点的**子树**

> 树是一种**递归定义**的数据结构。

**非空树特性**：
- 有且仅有一个根节点
- 没有后继的结点称为"**叶子结点**"（或终端结点）
- 有后继的结点称为"**分支结点**"（或非终端结点）
- **除根节点外**，任何一个结点都有且仅有一个前驱
- 每个结点可以有 0 个或多个后继

#### 基本术语

| 术语 | 定义 |
|:----|:----|
| **祖先 / 双亲 / 孩子** | 从根到该结点所经分支上的所有结点为其**祖先**；某结点前一层的结点为其**双亲**；该结点后继结点为**孩子** |
| **兄弟 / 堂兄弟** | 同一双亲的孩子之间互为**兄弟**；双亲在同一层的结点互为**堂兄弟** |
| **结点的度** | 一个结点的孩子个数 |
| **树的度** | 树中所有结点的度的最大值 |
| **结点的层次** | 根为第一层，其孩子为第二层，以此类推 |
| **结点的高度** | 从该结点到叶子结点的最长路径上的**边的数目**（自底向上） |
| **结点的深度** | 从根结点到该结点的路径上的**边的数目**（自顶向下） |
| **树的高度（深度）** | 根结点的高度（也等于最大层次） |
| **有序树 / 无序树** | 树中结点的各子树**从左到右有次序**为有序树，否则为无序树 |
| **森林** | $m\ (m \geq 0)$ 棵互不相交的树的集合（去掉根节点后剩下的子树集合） |

> - **深度** = 根到该结点的路径上的边数（自顶向下数）
> - **高度** = 该结点到最深叶子的路径上的边数（自底向上数）
> - 树的**高度 = 最大深度 = 根节点的高度**

### 5.1.3 树的常考性质

> 1. 树中的结点数 = **所有结点的度数之和 + 1**（加 1 是因为根节点没有前驱需要被计入）
> 2. 度为 $m$ 的树中第 $i$ 层上至多有 $m^{i-1}$ 个结点
> 3. 高度为 $h$ 的 $m$ 叉树至多有 $\dfrac{m^h - 1}{m - 1}$ 个结点
> 4. 具有 $n$ 个结点的 $m$ 叉树的**最小高度**为 $\lceil \log_m (n(m-1) + 1) \rceil$

> 一棵度为 4 的树 T 中，若有 20 个度为 4 的结点，10 个度为 3 的结点，1 个度为 2 的结点，10 个度为 1 的结点，则树 T 的叶子结点个数为？
>
> **解**：设叶子结点个数为 $n_0$。总结点数 $n = n_0 + 20 + 10 + 1 + 10 = n_0 + 41$
> 树的边数 = $n - 1 = n_0 + 40$
> 又边数 = $4 \times 20 + 3 \times 10 + 2 \times 1 + 1 \times 10 = 80 + 30 + 2 + 10 = 122$
> 所以 $n_0 + 40 = 122 \Rightarrow n_0 = 82$

## 5.2 二叉树

### 5.2.1 二叉树的定义和基本术语

> 二叉树是 $n\ (n \geq 0)$ 个结点的有限集合：
> ① 或者为空二叉树，即 $n = 0$。
> ② 或者由一个根结点和两个互不相交的被称为根的**左子树**和**右子树**组成。左子树和右子树又分别是一棵二叉树。
>
> **特点**：
> ① 每个结点至多只有两棵子树
> ② 左右子树**不能颠倒**（二叉树是**有序树**）

> | 度为 2 的有序树 | 二叉树 |
> |:--------------|:-------|
> | 任意结点的度 $\leq 2$ | 任意结点的度 $\leq 2$ |
> | **至少有 3 个结点** | 可以是空树（$n=0$） |
> | 子树**无左右之分** | 左右子树**有严格区分** |

> 二叉树是**递归定义**的数据结构。

#### 几种特殊的二叉树

##### 满二叉树

> 一棵高度为 $h$、含 $2^h - 1$ 个结点的二叉树称为**满二叉树**。
> - 叶子结点都在最底层
> - 除叶子结点外，每个结点的度都为 2
> - 编号：从上到下、从左到右按 1, 2, …, $2^h-1$ 编号

##### 完全二叉树

> 当且仅当其每个结点都与高度为 $h$ 的满二叉树中编号为 $1 \sim n$ 的结点**一一对应**时，称为完全二叉树。
>
> **特点**：
> ① 叶子结点**只在最下两层**出现
> ② 最下层叶子结点**集中在左边**
> ③ 如果有度为 1 的结点，只可能有一个，且该结点**只有左孩子**
> ④ 按层序编号后，一旦出现某结点（编号 $i$）为叶子结点或只有左孩子，则编号大于 $i$ 的结点均为叶子结点
> ⑤ 当结点总数 $n$ 为奇数时，$n_1 = 0$；$n$ 为偶数时，$n_1 = 1$

##### 二叉排序树（BST）

> 左子树上所有关键字 < 根关键字；右子树上所有关键字 > 根关键字；左右子树也是 BST。详见 7.4 节。

##### 平衡二叉树（AVL）

> 树上任一结点的左、右子树深度之差不超过 1。详见 7.6 节。

### 5.2.2 二叉树的常考性质

> 设非空二叉树中度为 0、1、2 的结点数分别为 $n_0, n_1, n_2$，则 $n_0 = n_2 + 1$。
> **证明**：边数 = $n - 1$（树的边数），又边数 = $n_1 + 2 n_2$，故 $n - 1 = n_1 + 2n_2$，又 $n = n_0 + n_1 + n_2$，得 $n_0 = n_2 + 1$。

> 二叉树第 $i$ 层（$i \geq 1$）上至多有 $2^{i-1}$ 个结点。

> 高度为 $h$ 的二叉树至多有 $2^h - 1$ 个结点（满二叉树）。

> 具有 $n$ 个结点的完全二叉树的高度（深度）为 $\lfloor \log_2 n \rfloor + 1$。
> 即 $h = \lfloor \log_2 n \rfloor + 1 = \lceil \log_2(n+1) \rceil$。

> ① $i = 1$：根节点；无父节点
> ② $i > 1$：父节点编号 $\lfloor i/2 \rfloor$
> ③ $2i \leq n$：左孩子编号 $2i$；否则无左孩子
> ④ $2i + 1 \leq n$：右孩子编号 $2i+1$；否则无右孩子
> ⑤ $i$ 是叶子结点或仅有左孩子 $\Leftrightarrow$ $i > \lfloor n/2 \rfloor$

### 5.3.1 二叉树的先/中/后序遍历

> | 遍历方式 | 顺序 | 记忆 |
> |:--------|:----|:----|
> | **先序**（根左右） | 根 → 左 → 右 | **根**在**前** |
> | **中序**（左根右） | 左 → 根 → 右 | **根**在**中** |
> | **后序**（左右根） | 左 → 右 → 根 | **根**在**后** |

#### 1. 递归实现

```c
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);                       // 先序：访问根
        PreOrder(T->lchild);            // 递归左子树
        PreOrder(T->rchild);            // 递归右子树
    }
}

void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);             // 中序：先左
        visit(T);                       // 访问根
        InOrder(T->rchild);             // 再右
    }
}

void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);           // 后序：先左
        PostOrder(T->rchild);           // 再右
        visit(T);                       // 最后访问根
    }
}
```

#### 2. 非递归实现（用栈）

> 1. 沿着根的左孩子，依次入栈，直到左孩子为空
> 2. 栈顶元素出栈并访问
> 3. 将当前指针指向该结点的右孩子
> 4. 重复 1-3 直到栈空且指针为空

```c
void InOrder2(BiTree T) {
    InitStack(S); BiTree p = T;
    while (p || !StackEmpty(S)) {
        if (p) { Push(S, p); p = p->lchild; }     // 一路向左入栈
        else { Pop(S, p); visit(p); p = p->rchild; }
    }
}
```

#### 3. 由遍历序列构造二叉树

> - **中序 + 先序** → 唯一
> - **中序 + 后序** → 唯一
> - **中序 + 层序** → 唯一
> - **先序 + 后序** → **不能**唯一确定（必须配合中序）

> - 先序首元素 A 是根；中序中 A 左为 DBE，右为 FC
> - 左子树先序 BDE，中序 DBE → B 为根，D 为左，E 为右
> - 右子树先序 CF，中序 FC → C 为根，F 为左
> - 还原：根 A，左子树（B 左 D 右 E），右子树（C 左 F）
> - 后序：DEB + FCA → **DEBFCA**

### 5.3.2 线索二叉树

> 若某结点的左孩子为空，则将左指针指向其**中序前驱**；若右孩子为空，则将右指针指向其**中序后继**。这种"指向前驱/后继"的指针称为**线索**。
> - 加一个标志域 `ltag` / `rtag`：`tag=0` 表示指向孩子，`tag=1` 表示指向前驱/后继（线索）。
> - **中序线索二叉树**最常见（以前驱/后继关系为主）。

> | 节点情况 | 中序前驱 | 中序后继 |
> |:----|:----|:----|
> | 有左孩子 | 左子树**最右下**的结点 | — |
> | 有右孩子 | — | 右子树**最左下**的结点 |
> | 无左孩子 | `ltag==1` 时，`lchild` 指向 | — |
> | 无右孩子 | — | `rtag==1` 时，`rchild` 指向 |

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;                      // 0 孩子，1 线索
} ThreadNode, *ThreadTree;
```

> - 线索二叉树节省了栈空间（遍历无需递归/栈），但**插入/删除**复杂。
> - **不能从先序/后序**直接得到前驱/后继（因为缺少"根在中间"的天然顺序），所以**中序线索二叉树**最实用。

### 5.4.1 树的存储结构

> 1. **双亲表示法**（顺序存储）：每个结点存 data + 父结点下标。找父 $O(1)$，找子 $O(n)$。
> 2. **孩子表示法**（顺序 + 链式）：每个结点的孩子用单链表串起来，链表头指针存在顺序表里。找子 $O(1)$，找父 $O(n)$。
> 3. **孩子兄弟表示法**（二叉链表 / 左孩子右兄弟）：每个结点含 `firstchild` 和 `nextsibling` 指针。**树 → 二叉树的桥梁**。

#### 1. 双亲表示法

```c
#define MAX_TREE_SIZE 100
typedef struct {
    ElemType data;
    int parent;                          // 父结点在数组中的下标
} PTNode;
typedef struct {
    PTNode nodes[MAX_TREE_SIZE];
    int n;                               // 结点数
} PTree;
```

#### 2. 孩子表示法

```c
typedef struct CNode {                   // 孩子链表结点
    int child;                           // 孩子结点在数组中的下标
    struct CNode *next;
} CNode, *ChildPtr;

typedef struct {                         // 顺序表结点
    ElemType data;
    ChildPtr firstchild;                 // 指向第一个孩子
} CBox;

typedef struct {
    CBox nodes[MAX_TREE_SIZE];
    int n, root;                         // 结点数 + 根的位置
} CTree;
```

#### 3. 孩子兄弟表示法（链式 / 二叉链表）

```c
typedef struct CSNode {
    ElemType data;
    struct CSNode *firstchild, *nextsibling;
} CSNode, *CSTree;
```

> 任何一棵树都可唯一对应一棵二叉树（二叉链表表示），反过来**不一定**。
> - 树中某结点的**第一个孩子** → 二叉树中该结点的**左孩子**
> - 树中某结点的**下一个兄弟** → 二叉树中该结点的**右孩子**
> - 根节点无兄弟，故根节点的右子树为空

### 5.4.2 树、森林与二叉树的转换

> - **树 → 二叉树**：在兄弟之间加线；对每个结点，只保留与**第一个孩子**的连线；**调整**使其左孩子右兄弟。
> - **二叉树 → 树**：右孩子 → 兄弟连线恢复；左孩子保持父子关系。

> - **森林 → 二叉树**：将每棵树转为二叉树，再将**后一棵二叉树的根作为前一棵二叉树根的右孩子**。
> - **二叉树 → 森林**：去掉根的右链，把右链上的每棵子树还原成森林中的一棵树。

### 5.4.3 树和森林的遍历

> - **先根遍历**：先访问根，再依次先根遍历每棵子树 → 与该树对应二叉树的**先序序列**相同
> - **后根遍历**：先依次后根遍历每棵子树，再访问根 → 与该树对应二叉树的**中序序列**相同
> - **层次遍历**：用队列

> - **先序遍历森林**：先序遍历第一棵树 + 先序遍历剩余森林
> - **中序遍历森林**：中序遍历第一棵树的子树森林 + 访问第一棵树根 + 中序遍历剩余森林

> 树的先根遍历 ≡ 对应二叉树的**先序**遍历；树的后根遍历 ≡ 对应二叉树的**中序**遍历。

### 5.5.1 哈夫曼树和哈夫曼编码

> - **结点的权**：给叶子结点赋予的有意义的数值
> - **结点的带权路径长度**：从根结点到该结点的路径长度 × 结点权值
> - **树的带权路径长度 WPL**：树中所有**叶子结点**的带权路径长度之和
>
> $$WPL = \sum_{i=1}^{n} w_i \cdot l_i$$
>
> - **哈夫曼树（最优二叉树）**：在含有 $n$ 个带权叶子的二叉树中，**WPL 最小的二叉树**。
> - 哈夫曼树中**权值越大的叶子越靠近根**。

> 1. 将 $n$ 个权值看作 $n$ 棵只含一个结点的二叉树集合 $F$
> 2. 每次从 $F$ 中选**两棵根权值最小**的树作为左右子树，构造新树（根权值=左右子树根权值之和）
> 3. 删除这两棵树，将新树加入 $F$
> 4. 重复 2-3 直到 $F$ 中只剩一棵树

> - **没有度为 1 的结点**（合并出来的结点都有左右子树，且都是合并的）
> - $n$ 个叶子结点的哈夫曼树共 $2n - 1$ 个结点
> - 哈夫曼树**不唯一**，但 WPL 唯一最小

> - 一种**可变字长编码**，使出现概率大的字符用短码、概率小的字符用长码，从而**整体编码长度最短**。
> - 哈夫曼编码是**前缀码**（任一字符的编码都不是另一字符编码的前缀），由哈夫曼树的**左 0 右 1** 路径得到。
> - 哈夫曼编码在数据压缩、文件压缩、ZIP/JPEG 中广泛应用。

> 给定字符集 {A, B, C, D, E, F}，出现频率分别为 {45, 13, 12, 16, 9, 5}，构造哈夫曼树并求 WPL。
>
> **解**：取最小的两个：5 + 9 = 14 → {14, 12, 13, 16, 45}
> 取最小的两个：12 + 13 = 25 → {14, 16, 25, 45}
> 取最小的两个：14 + 16 = 30 → {25, 30, 45}
> 取最小的两个：25 + 30 = 55 → {45, 55}
> 取剩下的：45 + 55 = 100 → {100}
> WPL = 5×4 + 9×4 + 12×3 + 13×3 + 16×2 + 45×1 = 20+36+36+39+32+45 = **208**

### 5.5.2 并查集（Union-Find Set）

> 一种处理**不相交集合的合并与查询**问题的树形数据结构。三个基本操作：
> - `Make-Set(x)`：建一个只含 x 的集合
> - `Find-Set(x)`：找 x 所在集合的**代表（根）**
> - `Union(x, y)`：合并 x、y 所在的集合

> - **路径压缩**：Find 时让路径上所有结点直接指向根
> - **按秩合并**：Union 时让小树合并到大树（秩为树高）

```c
int parent[MAXN];                        // 父结点

void Init(int n) {
    for (int i = 0; i < n; ++i) parent[i] = i;
}

int Find(int x) {
    while (parent[x] != x) {
        parent[x] = parent[parent[x]];   // 路径压缩
        x = parent[x];
    }
    return x;
}

void Union(int x, int y) {
    int rx = Find(x), ry = Find(y);
    if (rx != ry) parent[rx] = ry;      // 简单合并（按秩合并需 size 数组）
}
```

> - 单次 Find：$O(\alpha(n))$，$\alpha$ 为反阿克曼函数，**实际 < 5**。
> - 整体：接近 $O(1)$。

#### 知识回顾

> - 树的度 = 各结点度的最大值；树中**叶子结点数 = 度为 2 的结点数 + 1**。
> - 完全二叉树的高度 = $\lfloor \log_2 n \rfloor + 1$。
> - 遍历方式 4 种：先/中/后序（递归）+ 层次（队列）。
> - **中序 + 另一序**才能唯一确定二叉树。
> - 树的**先根遍历 ≡ 对应二叉树的先序遍历**；**后根遍历 ≡ 对应二叉树的中序遍历**。
> - 哈夫曼树无度为 1 的结点，$n$ 叶子的哈夫曼树共 $2n-1$ 个结点。
> - 并查集路径压缩后 Find $O(\alpha(n))$。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 树 = "倒过来的家谱"
- **根节点**：唯一，无父节点
- **叶子节点**：无子节点
- **度**：节点的子节点数
- **深度**：从根开始的层数
- **高度**：从节点到最深叶子的路径长度

#### 2. 二叉树 5 大性质（必背）
- 第 $i$ 层至多 $2^{i-1}$ 个节点
- 深度 $k$ 的二叉树至多 $2^k - 1$ 个节点
- 叶子数 $n_0$ = 度为 2 的节点数 $n_2 + 1$
- $n$ 个节点的完全二叉树深度 $\lfloor \log_2 n \rfloor + 1$

#### 3. 4 类二叉树
- **满二叉树**：深度 $k$，$2^k - 1$ 节点
- **完全二叉树**：除最后一层不满，其他满，**左对齐**
- **二叉排序树（BST）**：左 < 根 < 右
- **平衡二叉树（AVL）**：左右子树高度差 $\le 1$

#### 4. 4 种遍历
- **先序**：根 → 左 → 右
- **中序**：左 → 根 → 右
- **后序**：左 → 右 → 根
- **层序**：按层从左到右

> - 先序 = "自我介绍"
> - 中序 = "按身高排队"
> - 后序 = "部门汇报"
> - 层序 = "点名"

#### 5. 哈夫曼树（最优二叉树）
- **带权路径长度（WPL）最小**的二叉树
- **构造**：每次选两个最小权值合并
- 应用：数据压缩（哈夫曼编码）

### 📚 天勤·严瑾推导

#### 1. 二叉树存储
- **顺序存储**：完全二叉树最合适（节省空间）
- **链式存储**：每个节点 2 个指针（左/右孩子）

#### 2. 二叉树遍历"递归 → 非递归"
- **中序非递归**：用栈模拟递归
```c
void InOrder(BiTree T) {
    InitStack(S); p = T;
    while (p || !StackEmpty(S)) {
        if (p) { Push(S, p); p = p->lchild; }
        else { Pop(S, p); visit(p); p = p->rchild; }
    }
}
```

#### 3. 线索二叉树
- 利用 `n+1` 个空链域存前驱/后继
- **前驱线索** + **后继线索** = 线索二叉树
- 节省空间，加快遍历

#### 4. 树 → 二叉树转换
- 左孩子 = 第一个子节点
- 右孩子 = 下一个兄弟节点

#### 5. AVL 树 4 种调整
- **LL**：右旋
- **RR**：左旋
- **LR**：左旋 + 右旋
- **RL**：右旋 + 左旋
