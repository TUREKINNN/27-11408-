# 第一章 绪论

## 1.1 数据结构的基本概念

### 1.1.1 数据、数据元素、数据对象、数据类型、抽象数据类型

> - **数据（Data）**：信息的载体，是描述客观事物属性的数、字符及所有能输入到计算机中并被程序识别和处理（识别、存储、加工）的符号集合。
> - **数据元素（Data Element）**：数据的基本单位，通常作为一个整体进行考虑和处理。一个数据元素可由若干**数据项**组成。
> - **数据项（Data Item）**：构成数据元素的不可分割的最小单位。
> - **数据对象（Data Object）**：具有**相同性质**的数据元素的集合，是数据的一个子集。
> - **数据结构（Data Structure）**：相互之间存在一种或多种特定关系的数据元素的集合。
> - **数据类型（Data Type）**：一个值的集合和定义在此集合上的一组操作的总称。
> - **抽象数据类型（Abstract Data Type, ADT）**：定义一个数据对象、数据对象中各元素间的结构关系以及处理数据元素的操作。其定义不依赖于具体实现。

> **问**：数据对象与数据元素有何区别？
> **答**：数据对象是**性质相同**的数据元素的**集合**（如整数集），而数据元素是集合中的**个体**。数据结构研究的是数据元素之间的**关系**，数据对象是数据的子集。

### 1.1.2 数据结构的三要素

> 数据结构包括以下三个要素：
> 1. **逻辑结构**：数据元素之间的逻辑关系（与存储无关）。
> 2. **物理结构（存储结构）**：数据结构在计算机中的表示（映像）。
> 3. **数据的运算**：施加在数据上的操作（定义 + 实现）。

#### 1. 逻辑结构（4 类）

| 类别 | 关系特征 | 典型结构 |
|:----|:--------|:--------|
| 集合 | 同属一个集合，无其它关系 | 集合 |
| 线性结构 | 一对一 | 线性表、栈、队列 |
| 树形结构 | 一对多 | 树、二叉树 |
| 图状结构（网状结构） | 多对多 | 图 |

#### 2. 物理结构（4 类）

| 存储方式 | 特点 | 典型结构 |
|:----|:----|:----|
| 顺序存储 | 逻辑相邻，物理相邻；可随机访问 | 顺序表 |
| 链式存储 | 逻辑相邻，物理不一定相邻；不可随机访问 | 链表 |
| 索引存储 | 存储信息+索引表，检索快 | 索引顺序表 |
| 散列存储（哈希存储） | 根据关键字计算存储地址 | 哈希表 |

> - 逻辑结构独立于计算机，物理结构依赖于计算机。
> - **同一逻辑结构可对应多种物理结构**（如线性表 → 顺序表 / 链表）。
> - 数据的**运算**包括运算的**定义**（针对逻辑结构）和运算的**实现**（针对存储结构）。

### 1.1.3 抽象数据类型的表示与实现

> ```
> ADT 抽象数据类型名{
>     数据对象：<数据对象的定义>
>     数据关系：<数据关系的定义>
>     基本操作：<基本操作的定义>
> }ADT 抽象数据类型名
> ```

## 1.2 算法和算法评价

### 1.2.1 算法的基本概念

> 1. **有穷性**：算法必须在执行有穷步之后结束，且每步都在有穷时间内完成。
> 2. **确定性**：每条指令必须有确切的含义，相同输入只能得到相同输出。
> 3. **可行性**：算法中描述的操作都可通过已实现的基本运算执行有限次来实现。
> 4. **输入**：零个或多个输入。
> 5. **输出**：一个或多个输出（与输入有某种特定关系的量）。

> - 算法是**有穷**的，程序可以是无穷的（如操作系统）。
> - 算法可用伪代码、流程图、自然语言描述；程序必须用某种语言实现。
> - 程序 = 数据结构 + 算法。

### 1.2.2 算法效率的度量

#### 1. 时间复杂度

> **时间复杂度 $T(n)$**：算法执行时间与问题规模 $n$ 之间的函数关系。
> **渐近时间复杂度（大 O 表示法）**：$T(n) = O(f(n))$ 表示存在常数 $C, n_0$，当 $n \geq n_0$ 时 $T(n) \leq C \cdot f(n)$。

> 1. **加法规则**：$T(n) = T_1(n) + T_2(n) = O(\max(f(n), g(n)))$
> 2. **乘法规则**：$T(n) = T_1(n) \times T_2(n) = O(f(n) \times g(n))$
> 3. **常对幂指阶**：数量级递增顺序为：$O(1) < O(\log_2 n) < O(n) < O(n\log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$

> ```c
> // ① O(1): 与问题规模无关的常数次操作
> int x = a + b;
>
> // ② O(log₂n): 循环变量按倍数递增/递减
> int i = 1;
> while (i <= n) i = i * 2;
>
> // ③ O(n): 单层循环
> for (i = 0; i < n; ++i) sum += i;
>
> // ④ O(n log n): 双层嵌套（一层 O(n)，一层 O(log n)）
> for (i = 0; i < n; ++i)
>     for (j = 1; j <= n; j *= 2) sum++;
>
> // ⑤ O(n²): 双层循环
> for (i = 0; i < n; ++i)
>     for (j = 0; j < n; ++j) sum++;
> ```

> - **最坏时间复杂度**：输入为最坏情况时的复杂度（**考研默认**）。
> - **平均时间复杂度**：所有输入等概率出现时的期望复杂度。
> - **最好时间复杂度**：输入为最好情况时的复杂度。
> - 通常只讨论**最坏**与**平均**复杂度。

#### 2. 空间复杂度

> **空间复杂度 $S(n)$**：算法运行过程中临时占用的存储空间大小与问题规模 $n$ 之间的函数关系。
> - **原地工作**：算法所需的辅助空间为常量，记 $O(1)$。
> - $S(n) = O(g(n))$，表示存在常数 $C, n_0$，当 $n \geq n_0$ 时算法占用的存储量 $\leq C \cdot g(n)$。

> - 空间复杂度包括**程序代码**所占空间 + **输入数据**所占空间 + **辅助变量**所占空间。
> - 算法原地工作指**除输入/输出占用的空间外**，额外需要的辅助空间为 $O(1)$。

> ```c
> // 递归深度为 n, 空间复杂度 O(n)
> void func(int n) {
>     if (n > 1) func(n - 1);
>     // 递归工作栈保存 n 个栈帧
> }
> ```

#### 3. 知识回顾

> - **数据结构 = 逻辑结构 + 存储结构 + 数据的运算**。
> - **逻辑结构 4 类**：集合、线性、树形、图状。
> - **存储结构 4 类**：顺序、链式、索引、散列。
> - 时间复杂度分析三规则：顺序（加法 max）、循环（乘法）、分支（取耗时最大的分支）。
> - 408 考题常考：递归算法时间/空间复杂度分析、二分查找类 $O(\log_2 n)$ 分析。

## 多源补充：四大教辅核心差异

### 🎓 王道·通俗讲解


#### 1. 数据结构 = "数据的家 + 房子布局"
- **数据**：住户（信息）
- **数据结构**：住户 + 房子布局（关系）
- **数据对象**：某一类住户的集合
- **数据类型**：住户的类型（教师/学生）

#### 2. 算法五大特性（408 常考选择）
```
有穷性：算法必须"做完"
确定性：输入相同，输出相同
可行性：能用代码实现
输入：0 个或多个
输出：至少 1 个
```

> 算法 ≈ 菜谱：
> - **有穷性**：菜不能做一辈子
> - **确定性**：同一道菜谱，做出来味道一样
> - **可行性**：家里有锅有灶
> - **输入**：食材
> - **输出**：成品

#### 3. 时间复杂度"加法/乘法" 规则
- **加法**：$T(n) = T_1 + T_2 = O(\max(f, g))$
- **乘法**：$T(n) = T_1 \times T_2 = O(f \times g)$
- **顺序**：取最大
- **嵌套**：相乘

> $O(1) < O(\log n) < O(n) < O(n\log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$

#### 4. 频度统计法 3 种复杂度
- **最坏**：默认讨论
- **平均**：期望值
- **最好**：理想情况

### 📚 天勤·严瑾推导


#### 1. 逻辑结构 4 大类
| 类别 | 关系 | 例 |
|------|------|-----|
| 集合 | 同属 | 集合 |
| 线性 | 一对一 | 线性表 |
| 树 | 一对多 | 二叉树 |
| 图 | 多对多 | 无向图 |

#### 2. 存储结构 4 大类
| 方式 | 特点 | 适用 |
|------|------|------|
| 顺序 | 连续、随机访问 $O(1)$ | 顺序表 |
| 链式 | 任意、动态 $O(1)$ 插入 | 链表 |
| 索引 | 索引表 | 字典 |
| 散列 | 哈希函数 | 哈希表 |

#### 3. 算法的"有穷性 vs 程序的可循环性"
- **算法**：必须有限步结束
- **程序**：可以无限循环（如 OS）

#### 4. 复杂度分析"3 步法"
1. 找**基本语句**：执行次数最多的
2. 计算**频度**：$T(n)$
3. 取**最高阶**：$O(f(n))$
