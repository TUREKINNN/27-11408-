# 第八章 排序

## 8.1 排序的基本概念

> 将一串记录按照其中的某个或某些关键字的**递增**或**递减**次序重新排列的过程。

> - **稳定排序**：关键字相等的两个记录在排序前后的相对次序**不变**。
> - **不稳定排序**：次序可能改变。

> - **内部排序**：待排记录全部存放在内存中进行的排序。
> - **外部排序**：待排记录数量很大，内存无法一次容纳全部记录，需要**内外存**交换数据的排序。

## 8.2 插入排序

### 8.2.1 直接插入排序

> 将待排序的记录插入到已排好序的子序列中。
> 1. 从第 2 个元素开始，往前扫描；
> 2. 比前面元素小时，将其插入到前面有序序列的合适位置。

```c
void InsertSort(ElemType A[], int n) {
    int i, j;
    for (i = 2; i <= n; ++i) {                 // A[1] 视为已排序
        if (A[i] < A[i - 1]) {
            A[0] = A[i];                        // 哨兵
            for (j = i - 1; A[0] < A[j]; --j)
                A[j + 1] = A[j];
            A[j + 1] = A[0];
        }
    }
}
```

> - 时间复杂度：$O(n^2)$（最坏逆序）；$O(n)$（最好顺序）；
> - 空间复杂度：$O(1)$；
> - **稳定**排序。

### 8.2.2 折半插入排序

> 在有序子表中用**折半查找**确定插入位置，再统一后移。比较次数降为 $O(n \log_2 n)$，但**移动次数仍为 $O(n^2)$**。

### 8.2.3 希尔排序（Shell Sort）

> 把记录按一定**增量分组**，对每组做直接插入排序；缩小增量再排，直到增量为 1。
> - **不稳定**；
> - 时间复杂度 $O(n^{1.3})$ ~ $O(n^2)$，与增量序列有关；
> - 空间复杂度 $O(1)$。

## 8.3 交换排序

### 8.3.1 冒泡排序

```c
void BubbleSort(ElemType A[], int n) {
    for (int i = 0; i < n - 1; ++i) {
        bool flag = false;                       // 提前终止标记
        for (int j = n - 1; j > i; --j)
            if (A[j] < A[j - 1]) {
                swap(A[j], A[j - 1]);
                flag = true;
            }
        if (!flag) return;                       // 本轮无交换，已有序
    }
}
```

> - 最坏 $O(n^2)$，最好 $O(n)$（已有序）；
> - **稳定**。

### 8.3.2 快速排序

> 任选一个元素作为**枢轴（pivot）**，通过一次划分将序列分成左右两部分：左边所有元素 $\leq$ pivot，右边所有元素 $\geq$ pivot，再对左右子序列递归排序。

```c
int Partition(ElemType A[], int low, int high) {
    ElemType pivot = A[low];                     // 第一个元素作枢轴
    while (low < high) {
        while (low < high && A[high] >= pivot) --high;
        A[low] = A[high];
        while (low < high && A[low] <= pivot) ++low;
        A[high] = A[low];
    }
    A[low] = pivot;
    return low;
}

void QuickSort(ElemType A[], int low, int high) {
    if (low < high) {
        int p = Partition(A, low, high);
        QuickSort(A, low, p - 1);
        QuickSort(A, p + 1, high);
    }
}
```

> - 平均时间 $O(n \log_2 n)$，空间 $O(\log_2 n)$（递归栈）；
> - 最坏 $O(n^2)$（序列已有序或逆序）；
> - **不稳定**。

> 1. **随机化枢轴**：避免最坏情况；
> 2. **三数取中法**：取左端、中间、右端的中位数作 pivot；
> 3. **小区间插入排序**：当子序列长度 < 阈值（如 8）时改用插入排序；
> 4. **尾递归优化**。

## 8.4 选择排序

### 8.4.1 简单选择排序

> 第 $i$ 趟在 $A[i \sim n]$ 中选最小元素与 $A[i]$ 交换。
> - 时间复杂度 $O(n^2)$；
> - **不稳定**（反例：$2\, 2\, 1$）。

### 8.4.2 堆排序

> - **大根堆**：$\forall$ 结点 $\geq$ 其孩子结点（升序用大根堆）；
> - **小根堆**：$\forall$ 结点 $\leq$ 其孩子结点（降序用小根堆）；
> - 堆是**完全二叉树**的顺序存储结构（数组下标从 1 开始）。

#### 1. 堆的调整

```c
void HeapAdjust(ElemType A[], int k, int len) {
    A[0] = A[k];                                 // A[0] 暂存
    for (int i = 2 * k; i <= len; i *= 2) {
        if (i < len && A[i] < A[i + 1]) ++i;     // 取较大的孩子
        if (A[0] >= A[i]) break;                 // 已满足堆序
        else { A[k] = A[i]; k = i; }             // 调整
    }
    A[k] = A[0];
}
```

#### 2. 建堆与堆排序

```c
void BuildMaxHeap(ElemType A[], int len) {
    for (int i = len / 2; i > 0; --i)            // 从最后一个非叶结点起
        HeapAdjust(A, i, len);
}

void HeapSort(ElemType A[], int len) {
    BuildMaxHeap(A, len);
    for (int i = len; i > 1; --i) {
        swap(A[1], A[i]);                        // 堆顶与堆底交换
        HeapAdjust(A, 1, i - 1);                 // 调整剩余堆
    }
}
```

> - 建堆时间 $O(n)$，堆排序总时间 $O(n \log_2 n)$；
> - 空间 $O(1)$；
> - **不稳定**。

## 8.5 归并排序和基数排序

### 8.5.1 归并排序

> **分治法**：将两个**有序**子序列合并成一个有序序列。
> - 核心操作：两路归并 `Merge(A, low, mid, high)`；
> - 时间 $O(n \log_2 n)$，空间 $O(n)$（辅助数组）；
> - **稳定**。

```c
void Merge(ElemType A[], int low, int mid, int high) {
    ElemType *B = (ElemType *)malloc((high - low + 1) * sizeof(ElemType));
    int i, j, k;
    for (k = low; k <= high; ++k) B[k] = A[k];   // 复制
    for (i = low, j = mid + 1, k = i; i <= mid && j <= high; ++k) {
        if (B[i] <= B[j]) A[k] = B[i++];
        else A[k] = B[j++];
    }
    while (i <= mid) A[k++] = B[i++];
    while (j <= high) A[k++] = B[j++];
}

void MergeSort(ElemType A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(A, low, mid);
        MergeSort(A, mid + 1, high);
        Merge(A, low, mid, high);
    }
}
```

### 8.5.2 基数排序

> 1. 设置 $r$ 个桶（$r$ 为基数，如十进制 $r = 10$）；
> 2. 按关键字**最低位**分配到桶中、再收集；
> 3. 依次对**次低位**、**次高位**……直到**最高位**；
> 4. 最终得到有序序列。
> - 时间复杂度 $O(d(n+r))$，$d$ 为关键字位数；
> - 空间 $O(r)$；
> - **稳定**；
> - 适用：关键字可以分割为若干"位"，且位有**有限范围**。

### 8.5.3 计数排序

> 计数排序是**特殊的桶排序**（每个桶大小为 1）。统计每个值出现的次数，再按次数还原。
> - 时间 $O(n + k)$（$k$ 为数据范围）；
> - 空间 $O(k)$；
> - **稳定**。

## 8.6 各种内部排序算法的比较

| 排序算法 | 最好时间 | 平均时间 | 最坏时间 | 空间 | 稳定性 | 适用场景 |
|:----|:----|:----|:----|:----|:----|:----|
| 直接插入 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 稳定 | 数据基本有序 / 小规模 |
| 希尔 | $O(n)$ | $O(n^{1.3})$ | $O(n^2)$ | $O(1)$ | 不稳定 | 中等规模 |
| 冒泡 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 稳定 | 教学 |
| 快速 | $O(n\log n)$ | $O(n\log n)$ | $O(n^2)$ | $O(\log n)$ | 不稳定 | **大规模首选** |
| 简单选择 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不稳定 | 选最小/最大 |
| 堆 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | 不稳定 | **TopK / 大数据** |
| 归并 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(n)$ | 稳定 | 链表排序 / 外排 |
| 基数 | $O(d(n+r))$ | $O(d(n+r))$ | $O(d(n+r))$ | $O(r)$ | 稳定 | 多关键字 / 字符串 |

> 1. 数据量 $n$ 较小（$\leq 50$）→ 直接插入或简单选择；
> 2. 数据基本有序 → 直接插入或冒泡（带 flag）；
> 3. $n$ 较大且要求稳定 → 归并；
> 4. $n$ 较大且无稳定性要求 → 快速 / 堆 / 希尔；
> 5. 取最大/最小 $K$ 个 → 堆；
> 6. 多关键字 / 字符串 → 基数。

## 8.7 外部排序

### 8.7.1 外部排序的基本概念

> 外部排序由两个阶段构成：
> 1. **生成初始归并段（有序段）**：将文件分批读入内存排序后写回外存（生成 $r$ 个有序段）；
> 2. **多路归并**：将多个有序段反复归并，最终形成整体有序文件。
>
> 时间代价：内部排序时间 + 外存读写时间 + 内部归并时间。
> - $r$ 个初始归并段、$k$ 路归并，则归并树的高度 $\lceil \log_k r \rceil$，总读写次数 $2n \cdot \lceil \log_k r \rceil$。

### 8.7.2 败者树

> 败者树是**完全二叉树**，叶子结点是 $k$ 个归并段当前关键字，内部结点记录**失败者**（即该位置不进入下一轮比较），根结点记录**最终胜者**。
> - **优点**：$k$ 路归并时，**选出最小关键字**只需 $\lceil \log_2 k \rceil$ 次比较（不随 $k$ 增大而显著增加）。

### 8.7.3 置换-选择排序

> 在生成初始归并段时，使用**最小堆** + **缓冲区**：
> 1. 读入 $k$ 个元素，建堆输出堆顶（最小值），并尝试从外存读入新元素；
> 2. 若新元素 $<$ 刚输出的元素 → 放入"待选"堆（下一归并段）；
> 3. 若新元素 $\geq$ 刚输出的元素 → 替换堆顶，继续输出。
>
> - 优点：**平均归并段长度 $2 \times $ 缓冲区大小**，远大于普通分段；
> - 缺点：算法复杂，需要额外的"待选"堆。

### 8.7.4 最佳归并树

> $r$ 个初始归并段长度不一，将归并过程看作一棵** Huffman 树**：让**短段优先归并**，最小化总读写次数。
> - **最佳归并树 WPL** = 总读写次数 = $2n - 1$ 次归并（$n$ 为段数）。
> - 当 $k$ 路归并且初始段数 $r$ 不足时，**需要补充虚段（长度为 0）**使 $(r - 1) \% (k - 1) = 0$。

> 补充 $(5 - 1) \% (3 - 1) = 0$ → 无需补；
> Huffman 树：依次归并 $5+10=15, 15+20=35, 30+30=60, 35+60=95$；
> WPL = $95 \times 2 = 190$ 次读写。

#### 知识回顾

> - 内部排序 8 种：插入（直接/折半/希尔）、交换（冒泡/快速）、选择（简单/堆）、归并、基数。
> - 稳定性比较：插入、冒泡、归并、基数稳定；希尔、快速、简单选择、堆不稳定。
> - 快速排序**最常用**但最坏 $O(n^2)$，堆排序时间恒定 $O(n \log n)$ 适合 TopK。
> - 外部排序 = 内部排序（生成有序段）+ 多路归并；**败者树**优化选最小关键字、**置换-选择**延长归并段、**最佳归并树**减少总读写。

### 🎓 王道·通俗讲解


#### 1. 排序 = "理顺序"
- **稳定性**：相同关键字的相对位置**不变**
- **时间复杂度**
- **空间复杂度**
- 408 常考 8 大排序

#### 2. 8 大排序对照表
| 排序 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 直接插入 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✓ |
| 希尔 | $O(n)$ | $O(n^{1.3})$ | $O(n^2)$ | $O(1)$ | × |
| 冒泡 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | ✓ |
| 快速 | $O(n\log n)$ | $O(n\log n)$ | $O(n^2)$ | $O(\log n)$ | × |
| 简单选择 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | × |
| 堆 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(1)$ | × |
| 归并 | $O(n\log n)$ | $O(n\log n)$ | $O(n\log n)$ | $O(n)$ | ✓ |
| 基数 | $O(d(n+r))$ | $O(d(n+r))$ | $O(d(n+r))$ | $O(r)$ | ✓ |

#### 3. 快速排序"3 步法"
1. 选**基准**（第一个 / 随机 / 三数取中）
2. **Partition**：小的放左，大的放右
3. **递归**对左右子表排序

> 快速排序 = "选一个人站中间，比他矮的站左边，高的站右边"

#### 4. 堆排序"3 步"
1. **建堆**（大根堆/小根堆）
2. **取顶**（堆顶 = 最大/最小）
3. **调整**（剩余元素再调整成堆）

#### 5. 归并排序"分治法"
- **分**：把数组对半分
- **治**：对左右子数组分别排序
- **合**：合并两个有序数组

> 归并排序 = "两队排好的人合并成一队"

### 📚 天勤·严瑾推导

#### 1. 快速排序 Partition（Hoare 法）
```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];  // 基准
    while (low < high) {
        while (low < high && A[high] >= pivot) high--;
        A[low] = A[high];
        while (low < high && A[low] <= pivot) low++;
        A[high] = A[low];
    }
    A[low] = pivot;
    return low;
}
```

#### 2. 堆排序"调整"算法
```c
void HeapAdjust(int A[], int k, int len) {
    A[0] = A[k];
    for (i = 2*k; i <= len; i *= 2) {
        if (i < len && A[i] < A[i+1]) i++;
        if (A[0] >= A[i]) break;
        else { A[k] = A[i]; k = i; }
    }
    A[k] = A[0];
}
```

#### 3. 归并排序"Merge"
```c
void Merge(int A[], int low, int mid, int high) {
    n1 = mid - low + 1;
    n2 = high - mid;
    L[1..n1+1] = A[low..mid];
    R[1..n2+1] = A[mid+1..high];
    L[n1+1] = R[n2+1] = INF;
    i = j = 1;
    for (k = low; k <= high; k++)
        A[k] = (L[i] <= R[j]) ? L[i++] : R[j++];
}
```

#### 4. 各种排序的"使用场景"
- **基本有序 + 小规模**：直接插入
- **大规模 + 一般场景**：快速排序
- **大规模 + 要求稳定**：归并排序
- **求前 k 大/小**：堆排序
- **大量数据 + 关键字位数少**：基数排序
