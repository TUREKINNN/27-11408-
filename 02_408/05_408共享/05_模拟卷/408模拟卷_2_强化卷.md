> (A) `s->next = p; s->prior = p->prior; p->prior->next = s; p->prior = s;`
> (B) `s->prior = p; s->next = p->next; p->next->prior = s; p->next = s;`
> (C) `s->next = p->next; s->prior = p; p->next->prior = s; p->next = s;`
> (D) `p->next = s; s->prior = p; s->next = p->next; p->next->prior = s;`

**答案**：C

**解析**：**双向循环链表 p 后插入 s**（4 步）：
1. `s->next = p->next`（s 的后继指向 p 的后继）
2. `s->prior = p`（s 的前驱指向 p）
3. `p->next->prior = s`（p 的后继的前驱指向 s）
4. `p->next = s`（p 的后继指向 s）
> **关键**：必须**先接 s 的指针**，再修改原链表

> (B) `CDBFEAG`
> (C) `CDBFAGE`
> (D) `CDBGAFE`

**答案**：A

**解析**：**还原二叉树**（前序定根，中序切分）：

前序 `ABCDEFG` → 根 = A
中序 `CBDAEGF` → 切分：`CBD | A | EGF`
- 根 A 左侧 = `CBD`（左子树）
- 根 A 右侧 = `EGF`（右子树）

**递归左子树 `CBD`**：
- 前序 `BCD` → 根 = B
- 中序 `CBD` → 切分：`C | B | D`（B 左侧 = C，B 右侧 = D）
- C 是 B 的左子，D 是 B 的右子

**递归右子树 `EGF`**：
- 前序 `EFG` → 根 = E
- 中序 `EGF` → 切分：`E | G | F` 或 `| E | GF`
  - 实际上：`EGF` 中 E 在最前，所以 E 左侧 = 空，E 右侧 = `GF`
  - 递归 `GF`：前序 `FG` → 根 = F
  - 中序 `GF` → 切分：`G | F`（F 左侧 = G）
  - G 是 F 的左子

**完整二叉树**：
```
        A
       / \
      B   E
     / \   \
    C   D   F
           /
          G
```

**后序遍历** = 左 + 右 + 根 = `C + D + B + G + F + E + A` = **CDBGFEA**

### 题目 1（11 分）

>
> 要求：
> (1) 写出算法思想（3 分）
> (2) 用 C 语言实现算法（5 分）
> (3) 分析算法的时间复杂度和空间复杂度（3 分）

**答案与解析**：

**(1) 算法思想**：

采用**双指针法**：
- `p` 指向当前检查的结点（前驱）
- `q` 指向 p 的后继（待检查的结点）
- 当 q 的值为 x 时，删除 q（让 p->next = q->next，free(q)，q = p->next）
- 当 q 的值不为 x 时，p 和 q 一起前进
- 遍历完整个链表，返回删除计数

**(2) C 语言实现**：

```c
int DeleteX(LinkList &L, ElemType x) {
    int count = 0;
    LNode *p = L;       // p 指向头结点（哨兵）
    LNode *q = p->next; // q 指向第一个实际结点
    
    while (q != NULL) {
        if (q->data == x) {
            p->next = q->next;  // 跳过 q
            free(q);
            count++;
            q = p->next;        // q 重新指向 p 的后继
        } else {
            p = q;              // p 和 q 一起前进
            q = q->next;
        }
    }
    return count;
}
```

**(3) 复杂度分析**：

- **时间复杂度**：$O(n)$（每个结点最多被访问一次）
- **空间复杂度**：$O(1)$（只使用了 2 个指针变量）

### 题目 3（11 分）

>
> (1) 完成 **100 条指令**的总时间是多少？（3 分）
> (2) 若 EX 段要访问主存，需要 200 ns，**采用资源重复**（双端口主存）后，每段仍为 100 ns，问此时完成 100 条指令的总时间？（3 分）
> (3) 在 (2) 的基础上，如果 EX 段和 MEM 段都需访问主存，能否都用双端口解决？为什么？（5 分）

**答案与解析**：

**(1) 100 条指令总时间**：

$T = (n + k - 1) \times \Delta t = (100 + 5 - 1) \times 100 = 104 \times 100 = 10400$ ns

**(2) 资源重复后**：

- 假设双端口主存使 EX 段仍为 100 ns
- 流水线结构不变
- $T = 10400$ ns（不变）

**(3) EX 和 MEM 都访问主存**：

**不能完全用双端口解决**。原因：
- **资源相关**：EX 和 MEM 都在流水线中
- 即便用**双端口**（同时 2 个访问），但 EX 和 MEM 可能**同时访问同一存储单元**
- 需要**分时使用**或**数据旁路**

> **正确解法**：
> - **结构相关**：用双端口 RAM 或分体交叉存储器
> - **数据相关**：用 forwarding（数据旁路）

> **答**：单纯用双端口不能完全解决 EX 和 MEM 的资源冲突，需要**分时访问**或**将 MEM 访问合并到 EX 段**（如 RISC 架构的 load/store 设计）

### 题目 4（12 分）

>
> (1) 写出主存地址格式（标记、组号、块内地址位数）（4 分）
> (2) 主存地址 `1A2B3CH` 映射到 Cache 哪一组？标记位是什么？（4 分）
> (3) 若 Cache 采用**写回法**和**LRU 替换算法**，则 CPU 写一个不在 Cache 中的数据时，需要进行哪些操作？（4 分）

**答案与解析**：

**(1) 地址格式**：

- 主存 4 GB = $2^{32}$ B → 主存地址 **32 位**
- 块大小 128 B = $2^7$ B → 块内地址 **7 位**
- Cache 块数 = 64 KB / 128 B = 512 块
- 8 路组相联 → 组数 = 512 / 8 = 64 组 = $2^6$ → 组号 **6 位**
- 标记位数 = 32 - 6 - 7 = **19 位**

**地址格式**：

| Tag（19 位） | Index（6 位） | Offset（7 位） |

**(2) `1A2B3CH` 映射**：

`1A2B3CH` = `0001 1010 0010 1011 0011 1100`（24 位）→ 补足 32 位：`0000 0000 0001 1010 0010 1011 0011 1100`

- 19 位 Tag = `0000 0000 0001 1010 001`
- 6 位 Index = `0 0010` = 2 → **第 2 组**
- 7 位 Offset = `1011 100` = 0x2C

**答**：`1A2B3CH` 映射到 Cache **第 2 组**，**Tag = 0x00035**

> 实际上 19 位标记 = `0 0000 0000 0110 1000 1` = `0x000D1`

**(3) 写回法 + LRU 替换**：

CPU 写 Cache 不命中的操作（**写分配** + 写回法）：

1. **查找空块**：在目标组中查找**空闲块**（标记位为"无效"）
2. **如有空块**：
   - 调入主存块到该 Cache 块
   - 设置标记位为"有效"
   - 修改 Cache 块
   - 设置"脏"位（Dirty bit）= 1
3. **如无空块**（组内 8 块都有效）：
   - 用 **LRU** 找出**最久未使用**的块
   - 若该块"脏"位 = 1 → 先将数据**写回主存**
   - **调入**新主存块
   - 修改 Cache 块
   - 设置"脏"位 = 1

## 二、综合应用题（共 9 分）

### 题目 7（9 分）

>
> - 子网 1：192.168.1.0/24（含主机 A：192.168.1.10）
> - 子网 2：192.168.2.0/24（R1 接口 IP：192.168.2.1，R2 接口 IP：192.168.2.2）
> - 子网 3：192.168.3.0/24（含主机 B：192.168.3.20，R2 接口 IP：192.168.3.1）
>
> 主机 A 要向主机 B 发送一个 IP 数据报（数据 800 字节，IP 头 20 字节）。
>
> (1) 子网 1 的**子网掩码**和**最大主机数**（2 分）
> (2) A 发送给 B 的过程中，**IP 数据报首部**中的 TTL、源 IP、目的 IP 在各段是否变化（3 分）
> (3) 若子网 2 的 MTU = 500 字节，是否需要**分片**？如何分片？（4 分）

**答案与解析**：

**(1) 子网 1 的子网掩码和最大主机数**：

- 子网掩码 = **255.255.255.0**（/24）
- 主机号位数 = 32 - 24 = 8 位
- 最大主机数 = $2^8 - 2$ = **254**（减去全 0 和全 1）
- 实际可用范围：192.168.1.1 ~ 192.168.1.254

**(2) IP 首部各字段变化**：

| 字段 | A → R1 段 | R1 → R2 段 | R2 → B 段 | 是否变化 |
|:-----|:----------|:-----------|:----------|:---------|
| 源 IP | 192.168.1.10 | 192.168.1.10 | 192.168.1.10 | **不变** |
| 目的 IP | 192.168.3.20 | 192.168.3.20 | 192.168.3.20 | **不变** |
| TTL | 64 | 63 | 62 | **每段 -1** |
| 标识符 | 1234 | 1234 | 1234 | **不变**（用于分片重组） |
| 标志/片偏移 | 0,0 | 0,0 | 0,0 | 可能变化（分片） |

> **关键**：TTL 逐段递减，IP 标识符用于分片重组

**(3) 分片分析**：

- IP 数据报总长 = 20（IP 头）+ 800（数据）= **820 字节**
- 子网 2 的 MTU = 500 字节
- **每个分片的总长 ≤ 500 字节** → 数据部分 ≤ 500 - 20 = 480 字节
- 800 / 480 = 1.67 → 需要 2 个分片

**分片**：
- **分片 1**：20 字节头 + 480 字节数据 = 500 字节，MF = 1，Offset = 0
- **分片 2**：20 字节头 + 320 字节数据 = 340 字节，MF = 0，Offset = 480/8 = 60

> **片偏移单位 = 8 字节**
