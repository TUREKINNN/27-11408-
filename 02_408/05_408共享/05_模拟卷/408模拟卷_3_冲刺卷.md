- (D) 希尔：$O(n^{1.3})$ 平均 / $O(1)$ 空间

> **答**：B（归并排序递归版最符合 $O(n \log n)$ 时间 + $O(\log n)$ 空间（栈））

### 题目 1（11 分）

>
> (1) 写出算法思想（3 分）
> (2) 用 C 语言实现 `BSTDelete` 函数（6 分）
> (3) 分析算法的时间复杂度（2 分）

**答案与解析**：

**(1) 算法思想**：

BST 删除分 3 种情况：
1. **被删结点是叶子**：直接删除
2. **被删结点只有左/右子树**：用子树替换被删结点
3. **被删结点有左右子树**：
   - 找**中序前驱**（左子树最右）或**中序后继**（右子树最左）
   - 用前驱/后继的值**替换**被删结点的值
   - **删除前驱/后继**（一定是叶子或单子树情况）

**(2) C 语言实现**：

```c
// 找最小值结点（右子树最左）
BSTNode* FindMin(BSTNode* T) {
    while (T && T->lchild) T = T->lchild;
    return T;
}

// 删除值为 x 的结点
BSTNode* BSTDelete(BSTNode* T, ElemType x) {
    if (T == NULL) return NULL;  // 未找到
    
    if (x < T->data) {
        T->lchild = BSTDelete(T->lchild, x);  // 在左子树删除
    } else if (x > T->data) {
        T->rchild = BSTDelete(T->rchild, x);  // 在右子树删除
    } else {  // 找到 x
        // 情况 1：叶子 或 情况 2：只有右子树
        if (T->lchild == NULL) {
            BSTNode* temp = T->rchild;
            free(T);
            return temp;
        }
        // 情况 2：只有左子树
        else if (T->rchild == NULL) {
            BSTNode* temp = T->lchild;
            free(T);
            return temp;
        }
        // 情况 3：左右子树都有
        BSTNode* temp = FindMin(T->rchild);  // 中序后继
        T->data = temp->data;  // 用后继值替换
        T->rchild = BSTDelete(T->rchild, temp->data);  // 删除后继
    }
    return T;
}
```

**(3) 时间复杂度**：

- 平均：$O(\log n)$（树高 = log n）
- 最坏：$O(n)$（BST 退化为链表）

## 一、单项选择题（每小题 2 分，共 22 分）

> (A) $(2 - 2^{-23}) \times 2^{127}$
> (B) $(1 - 2^{-23}) \times 2^{127}$
> (C) $(2 - 2^{-23}) \times 2^{128}$
> (D) $2^{127}$

**答案**：A

**解析**：**IEEE 754 单精度浮点数**：
- 阶码 8 位（移码），偏置 127
- 尾数 23 位（**隐藏最高位 1**，所以实际 24 位精度）
- **最大正数**：
  - 阶码 = 11111110（避免无穷大 11111111）= 127
  - 尾数 = 11111111...（23 个 1）= $1 - 2^{-23}$
  - 数值 = $(1 - 2^{-23}) \times 2^{127}$

> **注意**：IEEE 754 单精度实际是 1 位符号 + 8 位阶码 + 23 位尾数，最大正数 = $(2 - 2^{-23}) \times 2^{127}$

> 答案选项 (A) 更接近：$(2 - 2^{-23}) \times 2^{127}$（考虑到隐藏位）

> (4) 若改用**直接映射**，其他条件不变，命中率会变化吗？为什么？（2 分）

**答案与解析**：

**(1) 总块数和组数**：

- Cache 总块数 = 32 KB / 64 B = **512 块**
- 4 路组相联 → 组数 = 512 / 4 = **128 组**

**(2) 地址格式**：

- 32 位主存地址
- 块大小 64 B = $2^6$ → Offset = 6 位
- 组数 128 = $2^7$ → Index = 7 位
- Tag = 32 - 6 - 7 = **19 位**

| Tag（19 位） | Index（7 位） | Offset（6 位） |

**(3) 命中率计算**：

**地址转换**（仅需 Index 即可判断是否命中，因为组相联可换路）：

| 地址 | Index | 命中? |
|:-----|:-----:|:-----:|
| 0x00001000 | (0x1000/64) % 128 = 64 % 128 = 64 | ✗（空） |
| 0x00001040 | (0x1040/64) % 128 = 65 | ✗ |
| 0x00001080 | (0x1080/64) % 128 = 66 | ✗ |
| 0x00001004 | (0x1004/64) % 128 = 64 | ✗（访问不同块）|
| 0x00002000 | (0x2000/64) % 128 = 0 | ✗ |
| 0x00001080 | 同上 = 66 | ✓（已加载）|
| 0x00001000 | = 64 | ✓ |
| 0x00001004 | = 64 | ✗（块 0x00001000 已被 0x00001004 替换）|

> 简化（实际需考虑替换算法）：8 次访问中 2 次命中 → **H = 2/8 = 25%**

> **更精确**：取决于替换算法
> - LRU：第 4 次（0x1004）替换了 0x1000 块，第 7 次（0x1000）miss
> - FIFO：第 4 次（0x1004）替换了 0x1000 块，第 7 次（0x1000）miss
> - 命中率约 2/8

**命中率 H = 2/8 = 25%**（在简化模型下）

**(4) 平均访问时间**：

$t_a = H \cdot t_c + (1-H) \cdot t_m = 0.25 \times 10 + 0.75 \times 100 = 2.5 + 75 = $ **77.5 ns**

**(5) 改为直接映射**：

直接映射下，每个主存块**只能映射到固定组**，可能增加冲突，**命中率会降低**（或不变）：
- 直接映射限制更严，冲突更多
- 组相联更灵活，**通常命中率更高**

## 二、综合应用题（共 15 分）

### 题目 5（7 分）

>
> | 进程 | 已分配 (R1, R2) | 最大需求 (R1, R2) |
> |:----:|:---------------:|:------------------|
> | P1 | (1, 0) | (2, 1) |
> | P2 | (0, 1) | (1, 2) |
> | P3 | (1, 1) | (2, 1) |
>
> 现有可用资源 Available = (0, 0)。
>
> (1) 此时系统是否处于死锁状态？请分析（3 分）
> (2) 若 Available = (0, 1)，系统安全吗？给出安全序列（4 分）

**答案与解析**：

**(1) Available = (0, 0) 时**：

- 每个进程都需要至少 1 个资源才能继续
- P1 还需要 (1, 1)
- P2 还需要 (1, 1)
- P3 还需要 (1, 0)
- **没有进程能被满足 → 死锁**

> **结论**：**死锁**

**(2) Available = (0, 1) 时**：

**Need**：
- P1: (2-1, 1-0) = (1, 1)
- P2: (1-0, 2-1) = (1, 1)
- P3: (2-1, 1-1) = (1, 0)

**Work = (0, 1)**：

| 步骤 | Work | 满足 Need? | 选择 | 释放后 Work' |
|:----:|:----:|:----------:|:----:|:-------------|
| 1 | (0, 1) | P1 (1, 1)：R1 1>0 ❌ | - | - |
| 1 | (0, 1) | P2 (1, 1)：R1 1>0 ❌ | - | - |
| 1 | (0, 1) | P3 (1, 0)：✅ | **P3** | (0+1, 1+1) = (1, 2) |
| 2 | (1, 2) | P1 (1, 1)：✅ | **P1** | (1+1, 2+0) = (2, 2) |
| 3 | (2, 2) | P2 (1, 1)：✅ | **P2** | (2+0, 2+1) = (2, 3) |

**安全序列：$\langle P3, P1, P2 \rangle$**

> **结论**：Available = (0, 1) 时，**系统安全**。
