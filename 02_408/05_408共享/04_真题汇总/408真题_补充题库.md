### 【2009 408 高频】 Flynn 分类

> (A) SISD
> (B) SIMD
> (C) MIMD
> (D) MISD

**答案**：C

**解析**：**Flynn 分类法**（按指令流 I 和数据流 D 多倍性分 4 类）：

| 类型 | 指令流 | 数据流 | 例子 |
|:-----|:------:|:------:|:-----|
| **SISD** | 1 | 1 | 传统单处理器 |
| **SIMD** | 1 | 多 | 阵列机、向量机、GPU |
| **MISD** | 多 | 1 | 流水线（极少实际例子）|
| **MIMD** | 多 | 多 | **多处理机**、多计算机 |

**多处理机**有多个独立指令流和数据流 → **MIMD**。

### 【2010 408 高频】 寻址范围

> (A) 2 GB
> (B) 4 GB
> (C) 8 GB
> (D) 16 GB

**答案**：B

**解析**：地址总线 n 位 → 可寻址 **2ⁿ 字节**（按字节编址）。  
- 32 位 → 2³² B = **4 GB**  
- 2³² = 4 × 2³⁰ = 4 × 1G = **4 GB**  
> **记忆口诀**：n 位地址 = 2ⁿ 字节（B）、2ⁿ⁻³ KB、2ⁿ⁻¹³ MB、2ⁿ²³ GB

### 【2011 408 高频】 补码范围

> (A) -128 ~ +127
> (B) -127 ~ +127
> (C) -128 ~ +128
> (D) -127 ~ +128

**答案**：A

**解析**：**8 位补码**：
- 最小：10000000 = **-2⁷ = -128**
- 最大：01111111 = **2⁷-1 = +127**
- 共 2⁸ = 256 个数，对称但不均衡（负数多 1 个）

> **n 位补码范围**：-2^(n-1) ~ 2^(n-1)-1

### 【2012 408 高频】 中断响应

> (A) 保存程序断点（PC）
> (B) 关中断
> (C) 转向中断服务程序
> (D) 恢复现场

**答案**：D

**解析**：**中断响应**（硬件自动完成）：
1. 关中断（IF = 0）
2. 保存 PC（断点）
3. 转向 ISR（中断服务程序）入口

**恢复现场**是**中断返回**（IRET）时做的，不属于中断响应。

> **高频考点**：区分"中断响应"和"中断处理"——响应是硬件，处理是软件。

### 【2013 408 高频】 DRAM 刷新

> (A) 刷新按行进行
> (B) 刷新期间不允许访存
> (C) 刷新由内存控制器自动完成
> (D) 刷新周期一般为 2 ms

**答案**：B

**解析**：**DRAM 刷新方式**：
- **集中刷新**：一段时间内集中刷新所有行，期间不能访存（有"死区"）
- **分散刷新**：每个存取周期刷新一行，无死区
- **异步刷新**：每 2 ms 内每行至少刷新一次（折中）

**集中刷新有"死区"**，但分散刷新和异步刷新**无死区**，所以 B 错误。

> **重要数据**：DRAM 标准刷新周期 **2 ms**

### 【2015 408 高频】 IEEE 754 阶码

> (A) 127
> (B) 128
> (C) 255
> (D) 256

**答案**：A

**解析**：**IEEE 754 阶码**采用**移码**（Excess-K）：
- **单精度（32位）**：8 位阶码，偏移 = 2⁷ - 1 = **127**
- **双精度（64位）**：11 位阶码，偏移 = 2¹⁰ - 1 = **1023**
- 移码 = 真值 + 偏移

> **为何用移码**：移码无符号比较即可比阶码大小（阶码越大，指数越大）

### 【2020 408 高频】 流水线时间

> (A) n × 5T
> (B) 5T + (n-1)T
> (C) 5T × (n-1) + T
> (D) 5T × n

**答案**：B

**解析**：**流水线公式**（n 条指令，k 级，每级 T）：
- 第 1 条：kT
- 之后每条：T（满载后每周期 1 条）
- **总时间 = kT + (n-1)T = (n + k - 1)T**
- 例：5 级 + 100 条 → 5T + 99T = **104T**

> **吞吐率** = n / (kT + (n-1)T)，n→∞ 时趋近 1/T

## 1.2 计算机组成原理 综合应用题（2 道）

### 【2020 408 风格综合题】 Cache 映射计算

>
> (1) 主存地址如何划分？（标出 Tag、Index、Offset 字段位数）
> (2) 主存地址 0x12345 映射到 Cache 哪一组？
> (3) 给出 Tag、Index、Offset 的二进制。

**答案与解析**：

**已知条件**：
- 主存：1 MB = 2²⁰ B → 主存地址 20 位
- Cache：4 KB = 2¹² B
- 块大小：64 B = 2⁶ B → **Offset 6 位**
- 4 路组相联 → 每组 4 块

**计算**：
- Cache 块数 = 4 KB / 64 B = 2⁶ = 64 块
- 组数 = 64 / 4 = 16 组 = 2⁴ → **Index 4 位**
- Tag = 20 - 6 - 4 = **10 位**

**地址划分**：

| Tag（10 位）| Index（4 位）| Offset（6 位）|
|:-----------:|:-----------:|:------------:|
| 主存区号 | 组号 | 块内地址 |

**(2) 0x12345 映射到 Cache 哪一组？**

0x12345 = 0001 0010 0011 0100 0101（二进制）共 20 位

取中间 4 位（Index）：

`0001 0010 0011 0100 0101`  
`^^^^^^^^^^----^^^^^^`  
`     Tag    Index Offset`  
        `0011` = **第 3 组**（0-indexed）

**(3) 完整字段**：

| 字段 | 二进制 | 十进制 |
|:-----|:-------|:------:|
| **Tag** | `0000010010` | 18 |
| **Index** | `0011` | 3 |
| **Offset** | `01000101` | 0x45 = 69 |

### 【2022 408 风格综合题】 流水线吞吐率

>
> (1) 流水线时钟周期应为多少？
> (2) 若执行 100 条指令，求总时间、加速比、吞吐率、效率。

**答案与解析**：

**(1) 时钟周期 = 最慢阶段 = max(80, 60, 100, 70, 50) = 100 ns**

**(2) 100 条指令总时间**：
- T = 5T_clock + (n-1) × T_clock
- T = 5 × 100 + 99 × 100
- T = 500 + 9900 = **10,400 ns**

**对比串行**（1 条 = 5 阶段总和 = 80+60+100+70+50 = 360 ns）：
- 串行 100 条 = 100 × 360 = 36,000 ns

**加速比**：
- S = 36,000 / 10,400 ≈ **3.46**

**吞吐率**：
- TP = n / T = 100 / 10,400 ns = 100 / 10.4 μs ≈ **9.6 MIPS**

**效率**：
- η = T_clock / 阶段平均 = 100 / 72 = **72.2%**

## 2.2 操作系统 综合应用题（2 道）

### 【2018 408 风格综合题】 PV 操作（生产者-消费者）

>
> (1) 写出 PV 操作描述的同步与互斥关系
> (2) 用信号量实现，给出初始值

**答案与解析**：

**同步关系**：
- 消费者消费时，仓库中必须**有产品** → 等待 `product` 信号量
- 生产者生产时，仓库中必须**有空位** → 等待 `space` 信号量

**互斥关系**：
- 仓库是**临界资源**，必须互斥访问 → `mutex` 信号量

**信号量定义**：
```c
semaphore mutex = 1;       // 互斥访问仓库
semaphore space = N;       // 空位数（初始 N）
semaphore product = 0;     // 产品数（初始 0）
```

**生产者**：
```c
Producer() {
    while (1) {
        P(space);          // 等待空位
        P(mutex);          // 互斥进入
        生产产品放入仓库;
        V(mutex);          // 释放
        V(product);        // 增加产品数
    }
}
```

**消费者**：
```c
Consumer() {
    while (1) {
        P(product);        // 等待产品
        P(mutex);          // 互斥进入
        从仓库取产品;
        V(mutex);          // 释放
        V(space);          // 增加空位数
    }
}
```

**关键点**：
- 互斥 P 操作**必须在同步 P 之后**（避免死锁）
- V 操作顺序不影响（先 V(mutex) 还是 V(product) 都可以）

### 【2020 408 风格综合题】 请求分页地址转换


| 页号 | 0 | 1 | 2 | 3 | 4 | 5 |
|:----:|:-:|:-:|:-:|:-:|:-:|:-:|
| 块号 | 2 | 5 | - | 8 | 9 | - |

> (1) 给出逻辑地址 0x1234 的物理地址
> (2) 给出物理地址 0x4321 的逻辑地址
> (3) 若访问页 2，会发生什么？

**答案与解析**：

**(1) 逻辑地址 0x1234 → 物理地址**

逻辑地址 0x1234 = 0001 0010 0011 0100（16 位）

地址划分：
- 页内偏移 = 12 位（4 KB）→ `010 0011 0100` = 0x234
- 页号 = 16 - 12 = 4 位 → `0001` = 1

查页表：页号 1 → 块号 5

物理地址 = 块号 × 页大小 + 页内偏移  
= 5 × 4 KB + 0x234  
= 5 × 0x1000 + 0x234  
= 0x5000 + 0x234  
= **0x5234**

**(2) 物理地址 0x4321 → 逻辑地址**

物理地址 0x4321 = 0100 0011 0010 0001

地址划分：
- 块内偏移 = 12 位 → `0011 0010 0001` = 0x321
- 块号 = 16 - 12 = 4 位 → `0100` = 4

查页表：块号 4 → 必须是某页的物理块号

但题目给出的是"块号"（物理块号），需要**反向查询**：
- 块号 4 不在页表中（页表只有 2, 5, 8, 9）
- 说明该物理地址**不在任何已映射的页中**

**答案**：物理地址 0x4321 **无对应的逻辑地址**（未映射的页无法访问）。

**(3) 访问页 2**

页表显示：页号 2 的块号为 `-`（无有效位或块号）

**会发生缺页中断（Page Fault）**：
1. CPU 发出访存请求
2. MMU 查页表，发现页 2 不在内存
3. 触发缺页异常
4. OS 启动页面调度：
   - 选择一页换出（如 LRU）
   - 调入页 2
   - 更新页表
5. 重新执行访存

> **缺页率计算**：缺页次数 / 访问总次数

### 【2017 408 高频】 二叉树遍历

> (A) **ABCD**
> (B) DCBA
> (C) CDAB
> (D) ABDC

**答案**：A

**解析**：**还原二叉树步骤**：

**(1) 后序 BDCA → 根 = A**（后序最后访问根）

**(2) 中序 BADC 配合根 A 切分左右**：
- 中序中 A 出现位置：`B | A | DC`
- 根 A 左侧 = `B`（左子树）
- 根 A 右侧 = `DC`（右子树）

**(3) 右子树 `DC` 在后序中 = `CA`**
- 右子树后序 `CA` → 根 = A？  
- 实际：后序是 `BDCA`，A 已是总根
- 右子树中序 `DC` 对应后序中的 `C`（单独节点，因为 D 是它的左子）

**(4) 右子树结构**：C 为根，D 为左子
- 验证后序：`D` (左) + `C` (根) ✓

**(5) 完整二叉树**：
```
       A
      / \
     B   C
        /
       D
```

**(6) 前序遍历**：根 + 左 + 右 = **A** + B + **C(D)** = **ABCD**

> **关键**：后序定根，中序切左右，递归还原

|:-----|:-----|
